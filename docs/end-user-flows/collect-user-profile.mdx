# Collect user profile

This feature adds a final "Tell us about yourself" step to collect extra user profile on sign-up. New users must complete all required fields here before registration is considered finished.

## Key benefits and use cases

- Capture extra business or compliance attributes (fullname, date of birth, address, website, job title, etc.).
- Mix built-in and custom fields; control display order and required status.
- Use composite fields (address, fullname) to gather structured data in one step.
- Add placeholders and descriptions to improve clarity and support multiple languages.

## Quick start

1. In the Console go to: Sign-in Experience → Collect user profile.
2. Click Add (or Create) and choose a built-in field or define a custom one (alphanumeric key).
3. Open field details to set label, description, required flag, and type-specific settings (length, range, format, options, etc.).
4. For Address or fullname, enable only the parts you need and configure each part’s order and required status.
5. Drag and drop to reorder; save to apply.
6. Users see these fields in the final step; every required field must validate before completion.

## Field types

We can simply categorize fields by simplicity:

- Primitive: Text, number, date, select, URL, regex, checkbox.
- Composite: Address, fullname.

## Validation

Field validation is performed on user submission in the final step of registration. The following checks are enforced:

- Required presence (except Checkbox).
- Length boundaries (Text).
- Numeric range (Number).
- Pattern match (Regex, Date with format, URL, Select option membership).
- URL structure (basic syntactic validation, not reachability).

Also applies to the sub-components of the two composite fields: `address` and `fullname`. The individual components must also meet the same validation criteria.

## Localization

### Built-in vs custom

Logto provides system level translations for the basic user data fields, such as `firstName`, `lastName`, `gender`, and `birthdate`. The label of these fields are not configurable in Logto Console.
You can override these labels by providing your own translations in [Logto > Sign-in Experience > Content](https://cloud.logto.io/to/sign-in-experience/content)

On the other hand, custom fields rely entirely on the labels/descriptions you provide.

### Managing translations

Add new labels/descriptions to your translation workflow immediately after creation to avoid mixed-language UI states.

## Best practices

- Collect only essential data at sign-up; defer nice-to-have info to later profile completion.
- Use composite fields for structured data (address, fullname) for consistency and localization.
- Provide clear examples/placeholders for format-sensitive fields (regex, date, numeric ranges).
- Review field usage analytics (if available) to prune rarely completed optional fields.

## FAQs

<details>

<summary>

### Do the profile fields collect existing user data?

</summary>

No, they only collect information from the final step of new user registration.

</details>

<details>

<summary>

### What if I delete a field? Does it remove the user data?

</summary>

No, it will not remove the existing user data. Only the field will be removed from the form.

</details>

<details>

<summary>

### Can a checkbox be required?

</summary>

No, the checkbox field will be always optional.

</details>

<details>

<summary>

### Can I collect country with a controlled list instead of a regular text field?

</summary>

Yes, you can switch the "Country" component to a Select field with standardized options.

</details>
