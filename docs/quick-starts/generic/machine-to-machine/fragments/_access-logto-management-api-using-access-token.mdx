<TabItem value="with-@logto-api-sdk" label="Node.js with `@logto/api` SDK">

With the `@logto/api` SDK, you can directly interact with any Management API using the code snippet below. The access token is cached internally and refreshed automatically if needed.

```js
import { createManagementApi } from '@logto/api/management';

const { apiClient } = createManagementApi('your-tenant-id', {
  clientId: 'your-client-id',
  clientSecret: 'your-client-secret',
});

const response = await apiClient.GET('/api/applications');
console.log(response.data);
```

</TabItem>

<TabItem value="nodejs" label="Node.js">

<AccessTokenUsage />

```js
const logtoEndpoint = 'https://your.logto.endpoint'; // Replace with your Logto endpoint
const accessToken = 'eyJhb...2g'; // Access Token

const fetchLogtoApplications = async () => {
  return await fetch(`${logtoEndpoint}/api/applications`, {
    method: 'GET',
    headers: {
      Authorization: `Bearer ${accessToken}`,
    },
  });
};
```

</TabItem>

<TabItem value="curl" label="cURL">

<AccessTokenUsage />

```bash
curl --location \
  --request GET 'https://your.logto.endpoint/api/applications' \
  --header 'Authorization: Bearer eyJhbG...2g'
```

Remember to replace the actual values with your own. The value after `Bearer` should be the access token (JWT) you received.
