# Logto OSS einrichten

Dieser Abschnitt behandelt grundlegende Einrichtungsschritte und wichtige Aktionen, um dein Produkt effektiv bereitzustellen und deinen Entwicklungsworkflow für den [Logto Open-Source-Dienst (OSS)](https://github.com/logto-io/logto) zu verwalten.

## Erste Schritte mit Logto OSS \{#get-started-with-logto-oss}

Folge der [Anleitung für den Einstieg](/logto-oss/get-started-with-oss/), um Logto heute zu starten, und später kannst du auf die vollständige [Bereitstellungsanleitung](/logto-oss/deployment-and-configuration) für den Produktionseinsatz verweisen.

## Funktionen, die von Logto OSS unterstützt werden \{#feature-supported-by-logto-oss}

Logto OSS unterstützt die meisten Kernfunktionen des Logto-Dienstes und wird regelmäßig aktualisiert und gewartet.

Einige erweiterte Funktionen sind derzeit exklusiv für die Logto Cloud-Version verfügbar, darunter:

| Funktionsbeschränkungen in OSS                                                          | Beschreibung                                                                                                                                                            |
| --------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto-Konsole: mehrere Mandanten](/logto-cloud/tenant-settings)                        | Erstellen und Verwalten mehrerer Logto-Mandanten in einer Konsole.                                                                                                      |
| [Logto-Konsole: Kollaboratoreneinladung](/logto-cloud/tenant-member-management)         | Einladen und Verwalten mehrerer Mitglieder eines Logto-Mandanten.                                                                                                       |
| Logto-Konsole: MFA aktivieren                                                           | Erhöhe die Sicherheit, indem du bei der Anmeldung in der Logto-Konsole Multi-Faktor-Authentifizierung verlangst.                                                        |
| [Logto Protected App](/integrate-logto/protected-app)                                   | Schnelle und nicht-SDK-Authentifizierungsintegration, unterstützt von Cloudflare.                                                                                       |
| [Logto integrierter E-Mail-Dienst](/connectors/email-connectors/built-in-email-service) | Nutze den integrierten und kostenlosen E-Mail-Dienst für die E-Mail-Zustellung.                                                                                         |
| [Bring your UI](/customization/bring-your-ui)                                           | Passe die Benutzeroberfläche oder Abläufe der Anmeldeerfahrung mit dieser Funktion an (OSS kann den Code auf GitHub forken, um die Anmeldeerfahrung direkt anzupassen). |
| [IdP-initiiertes SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)                 | Ermögliche identitätsanbieter-initiiertes Enterprise Single Sign-On.                                                                                                    |
| [SAML-Apps](/integrate-logto/saml-app)                                                  | Integriere Anwendungen über das SAML-Protokoll. Die OSS-Version ist auf 3 SAML-Apps beschränkt.                                                                         |

Tipps: Multi-Tenancy, Mitglieder-Einladungen und MFA sind nicht verfügbar, damit dein Team sich in eine Open-Source-Logto-Konsole anmelden kann. Du kannst jedoch diese Funktionen in deinem eigenen Produkt mit Logto OSS implementieren und sie deinen Endbenutzern zur Verfügung stellen.

## Bleibe auf dem Laufenden mit Logto-Veröffentlichungen \{#stay-updated-with-logto-releases}

Um deine Logto-Instanz mit den neuesten Funktionen auf dem neuesten Stand zu halten, folge der [Logto GitHub Releases](https://github.com/logto-io/logto/releases) Seite, dort findest du alle Veröffentlichungsprotokolle.

Lies die [Anleitung zum Upgrade](/logto-oss/upgrading-oss-version), um zu erfahren, wie du Logto aktualisieren kannst, ohne deinen Code oder das Datenbankschema zu ändern.
