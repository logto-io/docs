# Fehlerbehebung für OSS

<details>

<summary>

### Mein Browser kann die Admin-Konsole nicht laden (zeigt Fehler `Crypto.subtle is unavailable...`) \{#my-browser-cannot-load-admin-console-showing-error-cryptosubtle-is-unavailable}

</summary>

Die Admin-Konsole verwendet die [Web Crypto API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Crypto_API), die [sichere Kontexte](https://developer.mozilla.org/en-US/docs/Web/Security/Secure_Contexts) erfordert, d.h. HTTPS oder HTTP mit `localhost`.

Wenn du HTTP mit einer IP-Adresse oder einer benutzerdefinierten Domain verwendest, kann der Browser die Admin-Konsole nicht laden.

</details>

<details>

<summary>

### Ich verwende eine benutzerdefinierte Domain, aber mein Browser kann die Admin-Konsole nicht laden (zeigt Fehler `"code": "oidc.invalid_redirect_uri"`) \{#im-using-a-custom-domain-but-my-browser-cannot-load-admin-console-showing-error-code-oidcinvalid_redirect_uri}

</summary>

Wenn du eine benutzerdefinierte Domain anstelle von `localhost` verwendest, musst du die Umgebungsvariable `ENDPOINT` auf die Logto-URL setzen. Dies liegt an der strengen Anforderung der Redirect-URI in OIDC. Siehe [Konfiguration](/concepts/core-service/configuration/) für Details.

</details>

<details>

<summary>

### Ich verwende einen HTTPS-Proxy (z.B. Nginx) vor Logto, aber die Anmeldung schlägt fehl (zeigt Fehler `TypeError: Failed to fetch`) \{#im-using-an-https-proxy-eg-nginx-in-front-of-logto-but-i-failed-on-sign-in-showing-error-typeerror-failed-to-fetch}

</summary>

- Stelle zunächst sicher, dass du die Node-Umgebungsvariable `TRUST_PROXY_HEADER` auf `true` gesetzt hast. Siehe [Konfiguration](/concepts/core-service/configuration/#using-a-https-proxy) für Details.
- Außerdem musst du den Header `X-Forwarded-Proto` in deiner Proxy-Konfiguration auf `https` setzen. Siehe [Vertrauen in TLS-Offloading-Proxies](https://github.com/panva/node-oidc-provider/blob/main/docs/README.md#trusting-tls-offloading-proxies) für Details.

</details>

<details>

<summary>

### Ich habe CORS-Probleme. \{#im-facing-cors-issues}

</summary>

- Wenn `ADMIN_ENDPOINT` nicht angegeben ist, wird `localhost:[admin-port]` in Logto für Cross-Origin Resource Sharing (CORS) zugelassen.
- Wenn `ADMIN_ENDPOINT` angegeben ist, werden nur Anfragen vom Ursprung von `ADMIN_ENDPOINT` zugelassen.

</details>

<details>

<summary>

### Was ist das schon wieder? `Error: Invalid id token` \{#what-is-this-again-error-invalid-id-token}

</summary>

Wenn du nicht mit deinen Tokens herumspielst, wird dies in den meisten Fällen durch eine Zeitabweichung zwischen deinem Server und dem Client verursacht. Synchronisiere die Zeit sowohl auf deinem Server als auch auf dem Client und versuche es erneut.

</details>
