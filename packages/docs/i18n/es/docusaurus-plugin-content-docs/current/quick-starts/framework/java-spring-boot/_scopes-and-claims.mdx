Logto utiliza las [convenciones de alcances y reclamos (scopes and claims)](https://openid.net/specs/openid-connect-core-1_0.html#Claims) de OIDC para definir los alcances y reclamos para obtener información del usuario del Token de ID y del <a href="https://openid.net/specs/openid-connect-core-1_0.html#UserInfo">endpoint de userinfo</a> de OIDC. Tanto "alcance" como "reclamo" son términos de las especificaciones de OAuth 2.0 y OpenID Connect (OIDC).

En resumen, cuando solicitas un alcance, obtendrás los reclamos correspondientes en la información del usuario. Por ejemplo, si solicitas el alcance `email`, obtendrás los datos `email` y `email_verified` del usuario.

<ScopeClaimList />

Agrega alcances y reclamos adicionales en el archivo `application.properties` para solicitar más información del usuario. Por ejemplo, para solicitar el alcance `urn:logto:scope:organizations`, añade la siguiente línea al archivo `application.properties`:

```properties title="application.properties"
spring.security.oauth2.client.registration.logto.scope=openid,profile,offline_access,urn:logto:scope:organizations
```

Los reclamos de la organización del usuario se incluirán en el token de autorización.
