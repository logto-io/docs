# Configura la verificación por correo electrónico a través de llamada HTTP

El conector oficial de Logto para correo electrónico HTTP.

<GuideTip />

## Comenzar \{#get-started}

El conector de correo electrónico HTTP te permite enviar correos electrónicos a través de una llamada HTTP. Para usar el conector de correo electrónico HTTP, necesitarás tener tu propio servicio de correo electrónico que exponga una API HTTP para enviar correos electrónicos. Logto llamará a esta API cuando necesite enviar un correo electrónico. Por ejemplo, cuando un usuario se registra, Logto llamará a la API HTTP para enviar un correo electrónico de verificación.

## Configurar el conector de correo electrónico HTTP \{#set-up-http-email-connector}

Para usar el conector de correo electrónico HTTP, necesitas configurar un endpoint HTTP al que Logto pueda llamar. Y un token de autorización opcional para el endpoint.

## Carga útil \{#payload}

El conector de correo electrónico HTTP envía una carga útil al endpoint al activar un correo electrónico. Incluye el correo electrónico del destinatario (`to`), el [tipo de uso](/connectors/email-connectors/email-templates#email-template-types) del correo electrónico que indica el flujo de autenticación (`type`), y una `payload` que contiene [variables](/connectors/email-connectors/email-templates#email-template-variables) disponibles (por ejemplo, código de verificación, enlace, información de la aplicación, información de la organización, información del usuario) y la [localización](/connectors/email-connectors/email-templates#email-template-localization) del usuario.

Ejemplo:

```json
{
  "to": "foo@logto.io",
  "type": "SignIn",
  "payload": {
    "code": "123456"
    "locale": "de"
  }
}
```

Puedes usar esta información para personalizar plantillas de correo electrónico en tu proveedor de entrega de correos electrónicos.

Para todos los tipos disponibles, consulta [Plantilla de correo electrónico](/connectors/email-connectors/email-templates#email-template-types). La definición completa del tipo `SendMessageData` está en [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts).
