# Parâmetros de autenticação

Em um fluxo de login OIDC padrão, aplicativos clientes iniciam uma solicitação de autenticação que redireciona o usuário para a página web de experiência de login universal hospedada pelo Logto. Com base nas suas configurações de experiência de login, os usuários podem fazer login ou se registrar usando vários identificadores, métodos de verificação e conectores de SSO social ou corporativo de terceiros.

Além dos [parâmetros de autenticação OIDC](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest) padrão, nosso produto introduz vários parâmetros de autenticação personalizados que permitem adaptar a experiência de login desejada para os usuários finais.

Isso é particularmente útil quando você deseja impor fluxos de login específicos para diferentes segmentos de usuários. Incluindo, mas não se limitando aos seguintes cenários:

### Registro direto para novos usuários \{#direct-sign-up-for-new-users}

Para uma campanha de registro direcionada, você pode querer direcionar novos usuários diretamente para a página de registro, ignorando o formulário de login padrão, para garantir uma experiência de integração perfeita.

**Uso**: Parâmetro [Primeira tela](/end-user-flows/authentication-parameters/first-screen) → `first_screen=register`

### Registro com email preenchido a partir de um link de assinatura \{#email-prefilled-sign-up-from-a-subscription-link}

Se você estiver coletando endereços de email através de um formulário de assinatura auto-hospedado, esses parâmetros permitem redirecionar os novos assinantes para a página de registro com o campo de email pré-preenchido, facilitando o processo de registro.

**Uso**: Parâmetro [Primeira tela](/end-user-flows/authentication-parameters/first-screen) → `first_screen=identifier:sign_up&identifier=email&login_hint=foo@gmail.com`

### Link direto para redefinição de senha \{#direct-password-reset-link}

Para novos usuários que precisam configurar sua nova senha, envie um link direto e redirecione-os para a página de redefinição de senha com o campo de email pré-preenchido.

**Uso**: Parâmetro [Primeira tela](/end-user-flows/authentication-parameters/first-screen) → `first_screen=reset_password`

### Login social instantâneo \{#instant-social-login}

Se você fornecer botões de login de mídia social diretamente no seu site, pode usar esses parâmetros para pular o formulário de login padrão e permitir que os usuários se autentiquem diretamente com o provedor social escolhido.

**Uso**: Parâmetro [Login direto](/end-user-flows/authentication-parameters/direct-sign-in#social-sign-in) → `direct_sign_in=social:google`

### Login direto SSO para clientes corporativos \{#direct-sso-sign-in-for-enterprise-customers}

Para produtos que atendem a vários clientes corporativos, esses parâmetros permitem exibir um botão SSO dedicado que faz login dos usuários com seu provedor SSO específico, pulando o formulário de login universal e proporcionando uma experiência perfeita para cada organização.

**Uso**: Parâmetro [Login direto](/end-user-flows/authentication-parameters/direct-sign-in#enterprise-sso) → `direct_sign_in=sso:1234567890`

### Métodos de login personalizados exibidos para diferentes sites \{#custom-sign-in-methods-displayed-for-different-sites}

Se seu produto atende tanto equipes internas quanto consumidores, ou diferentes organizações, você pode adaptar a experiência de login de acordo. Por exemplo: - Para ferramentas internas, mostrar apenas login por email / nome de usuário e senha. - Para páginas voltadas para o consumidor, oferecer todas as opções de login disponíveis, incluindo login com Google.

**Uso**: Diferentes configurações de [primeira tela](/end-user-flows/authentication-parameters/first-screen) ou [login direto](/end-user-flows/authentication-parameters/direct-sign-in) com base no tipo de site.

Continue lendo para saber mais sobre os parâmetros de autenticação para personalizar a [primeira tela](/end-user-flows/authentication-parameters/first-screen) ou habilitar uma experiência de [login direto](/end-user-flows/authentication-parameters/direct-sign-in) para seus usuários.
