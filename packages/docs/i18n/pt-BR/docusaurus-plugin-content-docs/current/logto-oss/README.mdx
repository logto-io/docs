# Logto OSS

Logto oferece versões open-source (OSS) e [cloud](/logto-cloud). Esta seção se concentrará na solução OSS: <Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS é uma solução de identidade de uso gratuito que fornece aos desenvolvedores capacidades de personalização total e controle completo sobre seus dados. Logto OSS oferece a flexibilidade de adaptar a base de código para atender a necessidades específicas, sendo uma escolha perfeita para aqueles que priorizam a soberania dos dados e desejam uma solução de provedor de identidade personalizada que possa crescer com seu aplicativo.

## Recursos suportados pelo Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS suporta a maioria das capacidades principais do serviço Logto e é regularmente atualizado e mantido.

Alguns recursos avançados são atualmente exclusivos da versão Logto Cloud, incluindo:

| Limitações de recursos no OSS                                                          | Descrição                                                                                                                                                              |
| -------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Console Logto: múltiplos tenants](/logto-cloud/tenant-settings)                       | Crie e gerencie múltiplos tenants Logto em um console.                                                                                                                 |
| [Console Logto: convite de colaborador](/logto-cloud/tenant-member-management)         | Convide e gerencie múltiplos membros de um tenant Logto.                                                                                                               |
| Console Logto: Habilitar MFA                                                           | Aumente a segurança exigindo autenticação multifatorial ao fazer login no Console Logto.                                                                               |
| [Aplicativo Protegido Logto](/integrate-logto/protected-app)                           | Integração rápida e sem SDK de autenticação alimentada pelo Cloudflare.                                                                                                |
| [Serviço de email embutido Logto](/connectors/email-connectors/built-in-email-service) | Utilize o serviço de email embutido e gratuito para entrega de emails.                                                                                                 |
| [Traga sua UI](/customization/bring-your-ui)                                           | Personalize a UI ou os fluxos da experiência de login usando este recurso (OSS pode bifurcar o código no GitHub para personalizar a experiência de login diretamente.) |
| [SSO iniciado por IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)               | Habilite a autenticação única corporativa iniciada pelo provedor de identidade.                                                                                        |
| [Aplicativos SAML](/integrate-logto/saml-app)                                          | Integre aplicativos via o protocolo SAML. A versão OSS é limitada a 3 aplicativos SAML.                                                                                |

Dicas: Multi-tenancy, convites de membros e MFA não estão disponíveis para sua equipe fazer login em um console Logto open-source. No entanto, você pode implementar esses recursos em seu próprio produto usando Logto OSS, tornando-os disponíveis para seus usuários finais.
