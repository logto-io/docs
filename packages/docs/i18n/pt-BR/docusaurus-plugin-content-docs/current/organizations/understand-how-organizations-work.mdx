# Entenda como as organizações funcionam

Para obter uma visão holística do entendimento, é importante compreender as entidades envolvidas nas funcionalidades de organização, e esses são os fundamentos para integrar organizações ao seu aplicativo.

## Organização (Organization) \{#organization}

Organização consiste em um grupo de [usuários (identidades)](/user-management/user-data). Pode representar as equipes, clientes empresariais e empresas parceiras que podem acessar seu aplicativo.

A introdução de uma organização como uma entidade é importante, pois não apenas agrupa usuários, mas também fornece um contexto para isolamento de locatários em aplicativos multi-tenant.

## Membro da organização \{#organization-member}

No Logto, um usuário que possui a associação de uma organização é referido como um membro da organização (ou seja, membro) dentro do contexto dessa organização.

## Aplicativo M2M da organização \{#organization-m2m-application}

Aplicativo M2M da organização refere-se aos [aplicativos máquina para máquina](/quick-starts/m2m) que estão associados a uma organização. Semelhante aos [membros da organização](#organization-member), os aplicativos da organização estão delimitados dentro do contexto de uma organização.

Uma vez que você associa um aplicativo a uma organização, o aplicativo pode ser atribuído a [papéis da organização](/authorization/organization-template/configure-organization-template#create-organization-role). Isso é útil quando você deseja permitir acesso sem interação aos recursos da organização.

:::note

Outros tipos de aplicativos não suportam associação de organização e papéis de organização, pois são projetados para interação do usuário.

:::

Consulte esta [seção](/organizations/organization-management) para gerenciar organizações, membros e aplicativos.

## Modelo de organização \{#organization-template}

O modelo de organização é projetado para necessidades de controle de acesso em aplicativos multi-tenant. Inclui papéis de organização e permissões de organização. Confira este [modelo de organização](/authorization/organization-template) para saber mais sobre o design de controle de acesso na organização.
