## Suporte Apenas em Coreano no Site do Desenvolvedor \{#developer-site-only-korean-support-now}

Atualmente, o site `Naver Developers` suporta apenas coreano. Considere usar um tradutor.

## Para a Produção \{#for-the-production}

- Para a produção, você deve obter uma revisão da equipe da Naver.
  Caso contrário, apenas usuários registrados podem fazer login.
  - Você pode adicionar um testador no menu `맴버관리(Member Manage)`.
- Para obter uma revisão, verifique `애플리케이션 개발 상태(Application Development Status)`
  em `API 설정(API Setting)` nas configurações do projeto do seu aplicativo.

## Configurar um projeto no Naver Developers \{#set-up-a-project-in-the-naver-developers}

- Visite o [Naver Developers](https://developers.naver.com/main/) e faça login com sua conta Naver.
- Clique em **Application -> 어플리케이션 등록** no menu para criar um novo projeto.
- Siga as instruções abaixo para criar o aplicativo.

### Nome do Aplicativo (어플리케이션 이름) \{#application-name-어플리케이션-이름}

- Digite o nome do seu aplicativo em `어플리케이션 이름` (Este nome é exibido quando um usuário faz login.)

### Uso da API (사용 API) \{#api-usage-사용-api}

- Escolha `네이버 로그인(Naver Login)` para `사용 API(API Usage)`
- Marque `이메일 주소(Email Address), 별명(Nickname), 프로필 사진(Profile Image)` como `필수(Necessário)` em `권한(Role)` (Você pode marcar `추가(Add)` como opcional para essas opções, mas não poderá obter as informações do usuário.)

### Ambiente de Serviço da Open API de Login (로그인 오픈 API 서비스 환경) \{#sign-in-open-api-service-environment-로그인-오픈-api-서비스-환경}

- Para `로그인 오픈 API 서비스 환경(Sign in Open API Service Environment)`, adicione dois ambientes `PC웹(PC Web)` e `모바일웹(Mobile Web)`.

#### PC Web (PC 웹) \{#pc-web-pc-웹}

- Para `서비스 URL(Service URL)`, digite `http(s)://YOUR_URL` (ex. https://logto.io)
- Para `네이버 로그인(Naver Login) Callback URL`, digite `http(s)://YOUR_URL/callback/${connector_id}` (ex. https://logto.io/callback/${connector_id})

#### Mobile Web (Mobile 웹) \{#mobile-web-mobile-웹}

- Para `서비스 URL(Service URL)`, digite `http(s)://YOUR_URL` (ex. https://logto.io)
- Para `네이버 로그인(Naver Login) Callback URL`, digite `http(s)://YOUR_URL/callback/${connector_id}` (ex. https://logto.io/callback/${connector_id})

:::caution

O `connector_id` pode ser encontrado na barra superior da página de detalhes do conector no Logto Admin Console.

:::

## Configurar Logto \{#configure-logto}

### Tipos de configuração \{#config-types}

| Nome         | Tipo   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |

#### clientId \{#clientid}

`clientId` é o `Client ID` do seu projeto.
(Você pode encontrá-lo em `애플리케이션 정보(Application Info)` do seu projeto nos desenvolvedores da Naver.)

#### clientSecret \{#clientseceret}

`clientSecret` é o `Client Secret` do seu projeto.
(Você pode encontrá-lo em `애플리케이션 정보(Application Info)` do seu projeto nos desenvolvedores da Naver.)
