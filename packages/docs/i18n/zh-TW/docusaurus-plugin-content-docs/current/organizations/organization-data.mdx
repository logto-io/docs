# 組織資料結構

在組織層級，有多個屬性，其值可以被建立、更新、讀取和刪除（CRUD）。

## 組織 ID \{#organization-id}

_組織 id_ 是每個組織的唯一識別符。它對於實現組織層級的登入體驗和檢索組織權杖 (Organization tokens) 很有用。

## 名稱 \{#name}

_名稱_ 支援組織層級的登入，並可根據需要整合到組織層級的產品介面中。

## 描述 \{#description}

_描述_ 欄位允許你添加文字以幫助識別和標記組織。

## 組織標誌 \{#organization-logos}

為了在登入體驗中動態顯示客戶的組織標誌，你可以將組織標誌上傳到組織設定頁面。

詳情請參閱 [組織專屬標誌](/customization/match-your-brand/#organization-specific-branding)。

## 自訂資料 \{#custom-data}

_自訂資料_ 是一個 JSON 物件，用於儲存有關組織的額外資訊。它為開發人員提供了靈活性，以儲存額外的組織詳細資訊以供開發使用。

## 是否需要 MFA \{#is-mfa-required}

_isMfaRequired_ 指示組織是否必須啟用多重要素驗證 (MFA)。如果設為 `true`，所有成員在登入以存取組織時必須通過 MFA。此安全性政策設定在組織層級進行配置。

詳情請參閱 [管理組織](/organizations/organization-management#require-mfa-for-organization-members)。

## 建立時間 \{#created-at}

_createdAt_ 是組織建立時的時間戳記，包含時區資訊。

## 租戶 ID \{#tenant-id}

_tenantId_ 識別組織所屬的租戶。
