# 第三方應用程式

Logto 的第三方應用程式整合功能使你能夠將 Logto 作為外部應用程式的 [身分提供者 (IdP, Identity Provider)](https://auth.wiki/identity-provider)。

身分提供者 (IdP) 是一種服務，用於驗證使用者身分並管理其登入憑證。在確認使用者身分後，IdP 生成驗證權杖或斷言，允許使用者存取各種應用程式或服務，而無需再次登入。

與你在 [將 Logto 整合到你的應用程式中](/integrate-logto/integrate-logto-into-your-application) 指南中建立的應用程式不同，這些應用程式由你開發並完全控制，第三方應用程式則是由外部開發者或業務合作夥伴開發的獨立服務。

這種整合方法非常適合常見的業務場景。你可以讓使用者使用他們的 Logto 帳戶存取合作夥伴應用程式，就像企業使用者使用 Google Workspace 登入 Slack 一樣。你也可以建立一個開放平台，讓第三方應用程式可以新增「使用 Logto 登入」功能，類似於「使用 Google 登入」。

## 在 Logto 中建立第三方應用程式 \{#create-an-third-party-application-in-logto}

1. 前往 <CloudLink to="/applications">控制台 > 應用程式</CloudLink>
2. 選擇「第三方應用程式」作為應用程式類型，並選擇以下其中一種整合協議：
   - OIDC / OAuth
3. 輸入應用程式的名稱和描述，然後點擊「建立」按鈕。新的第三方應用程式將被建立。

所有建立的第三方應用程式將在「應用程式」頁面的「第三方應用程式」標籤下列出。這種安排有助於你將它們與自己的應用程式區分開來，使你能更輕鬆地在一個地方管理所有應用程式。
