# 整合 Logto 驗證 (Authentication)

Logto 為網頁、行動和桌面應用程式提供全面的驗證 (Authentication) 解決方案，支援服務之間的 [機器對機器 (M2M)](/quick-starts/m2m) 驗證 (Authentication)，並可透過 [OpenID Connect (OIDC)](https://auth.wiki/openid-connect) 和 [OAuth 2.0](https://auth.wiki/oauth-2.0) 等標準協議作為 [第三方應用程式](/integrate-logto/third-party-applications) 的身分提供者 (IdP)。

選擇最符合你需求的解決方案開始整合：

## 為你的應用程式新增驗證 (Authentication) \{#add-authentication-for-your-application}

無論你正在構建面向使用者的應用程式（如網頁、行動或桌面應用程式），還是用於服務間通信的機器對機器 (M2M) 應用程式，你都可以透過整合 Logto 快速實現全面的 [驗證 (Authentication)](/end-user-flows) 和 [使用者管理](/user-management) 功能。

## 使用 Logto 作為身分提供者 (IdP) \{#use-logto-as-an-identity-provider}

當你希望啟用第三方應用程式使用你的 Logto 身分系統進行驗證 (Authentication) 時，可以將 Logto 配置為 [身分提供者 (IdP)](https://auth.wiki/identity-provider)。

Logto 可以透過 OAuth、OIDC 等標準協議作為 IdP，外部服務可以實現「使用 Logto 登入」，就像「使用 Google 登入」一樣，允許他們的使用者使用你的 Logto 使用者系統進行驗證 (Authentication)。

透過在 Logto 中創建第三方應用程式，你可以在你的身分系統與外部服務之間建立安全連接。

## 使用 Management API 擴展 Logto 功能 \{#extend-logto-features-with-management-api}

Logto [Management API](https://openapi.logto.io/) 使你能夠以程式化方式控制和自動化應用程式中的 Logto 功能。

當你需要擴展 Logto 的能力以增強你的產品（例如 [管理多租戶組織](/organizations) 或使用 Logto 自動化工作流程）時，可以利用 Logto Management API 構建符合產品需求的高級功能。
