為了建立新的 OIDC 憑證，你需要為應用程式設定使用者授權頁面。

1. 前往 [OAuth 使用者授權頁面](https://console.cloud.google.com/apis/credentials/consent) 並選擇 `Internal` 使用者類型。這將使 OAuth 應用程式僅對你的組織內的使用者可用。

<img
  src="/img/assets/google_workspace_consent_screen_user_type.webp"
  alt="Google Workspace 使用者授權頁面使用者類型"
/>

2. 根據頁面上的指示填寫 `Consent Screen` 設定。你需要提供以下最低限度的資訊：

- **Application name**：你的應用程式名稱，將顯示在使用者授權頁面上。
- **Support email**：你的應用程式支援電子郵件，將顯示在使用者授權頁面上。

<img
  src="/img/assets/google_workspace_consent_screen_settings.webp"
  alt="Google Workspace 使用者授權頁面設定"
/>

3. 設定應用程式的 `Scopes`。為了正確從 IdP 檢索使用者的身分資訊和電子郵件地址，Logto SSO 連接器需要從 IdP 授予以下權限範圍：

<img
  src="/img/assets/google_workspace_consent_screen_scopes.webp"
  alt="Google Workspace 使用者授權頁面權限範圍"
/>

- **openid**：此權限範圍是 OIDC 驗證所需，用於檢索 ID 權杖並存取 IdP 的 userInfo 端點。
- **profile**：此權限範圍是存取使用者基本個人資料資訊所需。
- **email**：此權限範圍是存取使用者電子郵件地址所需。

點擊 `Save` 按鈕以保存使用者授權頁面設定。
