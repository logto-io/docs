## 使用 Hugging Face 帳戶登入 \{#sign-in-with-hugging-face-account}

前往 [Hugging Face 網站](https://huggingface.co/)並使用你的 Hugging Face 帳戶登入。如果你還沒有帳戶，可以註冊一個新帳戶。

## 在 Hugging Face 中建立 OAuth 應用程式 \{#create-an-oauth-app-in-the-hugging-face}

按照 [建立 OAuth 應用程式](https://huggingface.co/docs/hub/en/oauth#creating-an-oauth-app) 指南，註冊一個新應用程式。

在建立過程中，你需要提供以下資訊：

- **Application Name**：你的應用程式名稱。
- **Homepage URL**：你的應用程式首頁或登陸頁的 URL。
- **Logo URL**：你的應用程式標誌的 URL。
- **Scopes**：OAuth 應用程式允許的權限範圍。對於 Hugging Face 連接器，通常使用 `profile` 來獲取使用者的個人資料資訊，使用 `email` 來獲取使用者的電子郵件地址。確保這些權限範圍在你的 Hugging Face OAuth 應用程式中被允許使用。
- **Redirect URI**：使用者驗證後重定向的 URL。當你在 Logto 管理控制台中建立 Hugging Face 連接器或在已建立的 Hugging Face 連接器詳細資訊頁面中，可以找到重定向 URI。

## 管理 Hugging Face OAuth 應用程式 \{#managing-hugging-face-oauth-apps}

前往 [已連接的應用程式](https://huggingface.co/settings/connected-applications) 頁面，你可以新增、編輯或刪除現有的 OAuth 應用程式。
你也可以在相應的 OAuth 應用程式設定頁面中找到 `Client ID` 並生成 `App secrets`。

## 配置你的連接器 \{#configure-your-connector}

在前一節提到的 OAuth 應用程式詳細資訊頁面中，填寫 `clientId` 和 `clientSecret` 欄位，使用獲得的 _Client ID_ 和 _App Secret_。

`scope` 是一個以空格分隔的 [Hugging Face 支援的權限範圍](https://huggingface.co/docs/hub/en/oauth#currently-supported-scopes) 列表。如果未提供，預設的 scope 為 `profile`。對於 Hugging Face 連接器，你可能想使用的 scope 是 `profile` 和 `email`。`profile` scope 是獲取使用者個人資料資訊所需的，`email` scope 是獲取使用者電子郵件地址所需的。確保你已在 Hugging Face OAuth 應用程式中允許這些權限範圍（在 [在 Hugging Face 中建立 OAuth 應用程式](#create-an-oauth-app-in-the-hugging-face) 章節中配置）。

### 配置類型 \{#config-types}

| 名稱         | 類型   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
