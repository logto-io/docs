## 疑難排解 \{#troubleshooting}

### 無法解析 `@logto/client/shim` \{#unable-to-resolve-logtoclientshim}

:::note

- 對於使用 SDK 版本 \<= v0.3.0 的使用者，請按照以下指示解決問題。
- 對於使用 SDK 版本 \>= v0.4.0 的使用者，請忽略此部分。此問題已在最新版本中修復。

:::

對於 Expo 專案，如果遇到錯誤 `Unable to resolve "@logto/client/shim" from "node_modules/@logto/rn/lib/index.js"`，可以通過在 `metro.config.js` 文件中添加以下內容來解決：

```js title="metro.config.js"
const config = {
  // ...
  resolver: {
    unstable_enablePackageExports: true,
  },
};

module.exports = config;
```

此錯誤表示 `@logto/rn` 套件無法解析 `@logto/client/shim` 模組。

由於在 `@logto/client` 套件中使用了節點匯出，而 Metro bundler 預設未啟用套件匯出，因此需要手動啟用。

詳情請參閱 [React Native 套件匯出支持](https://reactnative.dev/blog/2023/06/21/package-exports-support#enabling-package-exports-beta)。
