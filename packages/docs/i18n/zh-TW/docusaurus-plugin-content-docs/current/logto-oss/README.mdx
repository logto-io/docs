# Logto OSS

Logto 提供開源 (OSS) 和 [雲端](/logto-cloud) 版本。本節將重點介紹 OSS 解決方案：<Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS 是一個免費使用的身分解決方案，為開發者提供完整的自訂能力和對其資料的完全控制。Logto OSS 提供靈活性，可以調整代碼庫以滿足特定需求，非常適合那些重視資料主權並希望擁有一個可以隨應用程式成長的量身定制的身分提供者解決方案。

## Logto OSS 支援的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支援 Logto 服務的大多數核心功能，並定期更新和維護。

一些高級功能目前僅限於 Logto Cloud 版本，包括：

| OSS 中的功能限制                                                              | 描述                                                                                       |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| [Logto 控制台：多租戶](/logto-cloud/tenant-settings)                          | 在控制台中創建和管理多個 Logto 租戶。                                                      |
| [Logto 控制台：協作者邀請](/logto-cloud/tenant-member-management)             | 邀請和管理一個 Logto 租戶的多個成員。                                                      |
| Logto 控制台：啟用 MFA                                                        | 通過要求在登入 Logto 控制台時進行多重要素驗證來增強安全性。                                |
| [Logto 受保護應用程式](/integrate-logto/protected-app)                        | 由 Cloudflare 提供支持的快速且無 SDK 驗證集成。                                            |
| [Logto 內建電子郵件服務](/connectors/email-connectors/built-in-email-service) | 使用內建且免費的電子郵件服務進行電子郵件傳遞。                                             |
| [自定義你的 UI](/customization/bring-your-ui)                                 | 使用此功能自定義登入體驗的 UI 或流程（OSS 可以在 GitHub 上分叉代碼以直接自定義登入體驗）。 |
| [IdP 發起的 SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)            | 啟用身分提供者發起的企業單一登入 (SSO)。                                                   |
| [SAML 應用程式](/integrate-logto/saml-app)                                    | 通過 SAML 協議集成應用程式。OSS 版本限制為 3 個 SAML 應用程式。                            |

提示：多租戶、成員邀請和 MFA 不適用於你的團隊登入開源 Logto 控制台。然而，你可以使用 Logto OSS 在自己的產品中實現這些功能，讓你的終端使用者可以使用。
