# 升級你的 OSS 版本

Logto 遵循 [semver](https://semver.org/)，因此如果我們在變更日誌中未提及，你可以放心升級 Logto 而無需更改程式碼或資料庫結構。你可以在 [Github Release](https://github.com/logto-io/logto/releases) 中找到發行列表。

## 步驟 1：更新至新版本 \{#step-1-update-to-the-new-version}

要更新，請根據你的部署方式選擇適當的方法：

- **Docker Image**：拉取最新的映像。
- **NPM Package**：升級至最新版本。
- **Tar File**：從發行頁面下載最新的 `.tar` 文件。

## 步驟 2：應用資料庫更改 \{#step-2-apply-database-alterations}

使用 [Logto CLI](/logto-oss/using-cli/database-alteration/) 執行必要的資料庫更改。為了順利升級，無需停止生產應用程式。Logto 確保新的資料庫結構與現有程式碼相容，因此生產環境將繼續順利運行而不會中斷。

## 步驟 3：重新啟動應用程式 \{#step-3-restart-the-application}

在更新程式碼並應用資料庫更改後，重新啟動你的應用程式以應用更改。確保新版本運行順利，且所有組件均按預期運作。

**提示**：為實現零停機升級，考慮使用「交換」策略，逐步將流量從舊實例轉移到新實例。這樣，使用者在升級過程中不會受到干擾。

你的升級現已完成！
