# 停用使用者註冊

在某些情況下，你可能希望限制應用程式的使用者註冊。Logto 允許你停用匿名使用者註冊，並限制使用者登入僅限於由管理員邀請的使用者。

要停用終端使用者註冊，請按照以下步驟操作：

1. 前往 <CloudLink to="/sign-in-experience/sign-up-and-sign-in">控制台 > 登入體驗 > 註冊與登入</CloudLink> 頁面。
2. 在 **進階選項** 區域，關閉 **啟用使用者註冊** 開關以停用使用者註冊。（預設為啟用）

一旦使用者註冊被停用：

- 「建立帳戶」連結將從登入頁面移除。
- 註冊頁面將被停用，訪問該頁面的使用者將被重定向至登入頁面。
- 社交和企業級單一登入 (SSO) 註冊流程也將被停用。未註冊的社交帳戶使用者在社交驗證後將收到「找不到帳戶」錯誤。
- 對於系統中有匹配電子郵件地址或電話號碼帳戶的使用者，社交連結帳戶功能仍然可用。

## 實現僅限邀請的註冊流程 \{#implement-an-invitation-only-sign-up-flow}

Logto 不提供內建的邀請流程。然而，你可以輕鬆地利用 Logto 的使用者管理 API 實現僅限邀請的註冊流程。

### 選項 1：通過電子郵件邀請使用者 \{#option-1-invite-users-via-email}

1. 在 <CloudLink to="/sign-in-experience/sign-up-and-sign-in">控制台 > 登入體驗</CloudLink> 中停用使用者註冊。
2. 在你的應用程式中建立自訂的基於邀請的使用者註冊表單。
3. 向使用者發送包含唯一邀請連結的邀請電子郵件。
4. 當使用者點擊註冊連結時，讓你的應用程式調用 [Logto Management API](https://openapi.logto.io/operation/operation-createuser) 來建立使用者帳戶。
5. 使用者帳戶建立後，將他們重定向至 Logto 登入或重設密碼頁面以完成註冊。請參閱 [驗證參數](/end-user-flows/authentication-parameters/) 以獲取自訂首屏和預填使用者電子郵件以簡化登入流程的指導。

### 選項 2：通知使用者手動註冊 \{#option-2-notify-users-on-manual-registration}

1. 在 Logto 控制台中停用使用者註冊。
2. 在 <CloudLink to="/users">控制台 > 使用者管理</CloudLink> 中或通過 [Management API](https://openapi.logto.io/operation/operation-createuser) 手動建立使用者帳戶。
3. 通過電子郵件通知使用者他們的帳戶已建立。
4. 將使用者重定向至 Logto 登入或重設密碼頁面以完成註冊。請參閱 [驗證參數](/end-user-flows/authentication-parameters/) 以獲取自訂首屏和預填使用者電子郵件以簡化登入流程的指導。

## 僅限邀請註冊的使用者體驗 \{#user-experience-for-invitation-only-sign-up}

當使用者註冊被停用時，訪問註冊頁面的使用者將被重定向至登入頁面。使用者只能由管理員或使用 Management API 建立。根據你配置的註冊和登入方法設置，使用者在登入時可能會有不同的體驗：

- **通過驗證碼使用電子郵件或電話號碼登入**：使用 **電子郵件地址** 或 **電話號碼** 建立使用者帳戶。啟用電子郵件或電話號碼驗證碼登入方法。使用者建立後，他們可以直接使用驗證碼驗證方法通過電子郵件或電話號碼登入。
- **使用特定電子郵件或電話號碼的社交登入**：使用與其社交帳戶相關聯的 **電子郵件地址** 或 **電話號碼** 建立使用者帳戶。為特定社交提供者建立並啟用社交連接。啟用 [**自動帳戶連結**](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking) 功能，以根據匹配的電子郵件或電話號碼自動將社交身分與使用者帳戶連結。使用者建立後，他們可以直接使用與電子郵件或電話號碼匹配的社交帳戶登入。
- **使用者名 / 電子郵件 / 電話號碼與密碼**：使用 **使用者名**、**電子郵件地址** 或 **電話號碼** 建立使用者帳戶，並啟用對應的識別符以進行基於密碼的登入。
  - 使用預設密碼：在帳戶建立時設置密碼，然後安全地與使用者分享。使用者可以直接使用其使用者名、電子郵件或電話號碼以及預設密碼登入，並在需要時重設密碼。
  - 無預設密碼：建立帳戶時不設置密碼，並提示使用者在帳戶啟用時設置密碼。（密碼重設流程需要 **電子郵件地址** 或 **電話號碼**。）建立使用者後，將他們重定向至預填電子郵件或電話號碼的密碼重設頁面。有關將使用者重定向至預填詳細資訊的重設密碼頁面的指導，請參閱 [驗證參數](/end-user-flows/authentication-parameters/)。使用者設置新密碼後，可以直接使用其使用者名、電子郵件或電話號碼和新密碼登入。
