# B2B 服務的多租戶架構

## 架構 \{#architecture}

B2B 應用程式通常使用[多租戶](https://auth.wiki/multi-tenancy)架構。在這些應用程式中，使用者擁有自己的帳戶並管理其身分和驗證，並涉及其他方如企業或組織。終端使用者的身分通常不是個別消費者，而是企業組織內的員工或合作者。

## 將你的需求連接到 Logto 的支援工具包 \{#connect-your-requirements-to-logtos-support-toolkit}

此架構包含三個主要參與管理場景的角色。根據你的具體需求和目標，這些角色可能全部或部分參與。

我們總結了常見的使用案例，突出了每個使用者管理任務的主要目標以及我們提供的相關產品和 API。你可以將你的需求映射到我們的服務，以快速開始。

| 使用者              | 目標                                                                                                                                                                                                                                                                                                                                                                        | Logto 產品和 API                                                                                                                                 |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| 開發者              | 管理和保護使用者身分池，直接與身分資料庫合作。                                                                                                                                                                                                                                                                                                                              | <ul><li>[Logto Console](https://cloud.logto.io)</li><li>[Logto Management API](https://openapi.logto.io)</li></ul>                               |
| 組織管理員          | 自助式組織管理，具有自訂功能，讓組織管理員完全控制身分管理。<br /><br /><ol><li>[自行創建組織](/end-user-flows/organization-experience/create-organization)</li><li>[邀請成員加入](/end-user-flows/organization-experience/invite-organization-members)</li><li>[新增或移除成員資訊並更新其角色](/end-user-flows/organization-experience/organization-management)</li></ol> | <ul><li>[Logto Management API](https://openapi.logto.io)</li></ul>                                                                               |
| 終端使用者/組織成員 | [管理他們自己的系統級驗證和個人資訊。](/end-user-flows)                                                                                                                                                                                                                                                                                                                     | <ul><li>[Logto Management API](https://openapi.logto.io)</li><li>[Account API](https://openapi.logto.io/group/endpoint-account-center)</li></ul> |
