# B2C 服務的單租戶架構

## 架構 \{#architecture}

在消費者應用程式中，使用者完全擁有其帳戶並控制其身分和驗證，無需其他「中間層」方如企業或組織的參與。這是 B2C 與 B2B 身分架構的關鍵區別。

## 將你的需求連接到 Logto 的支援工具包 \{#connect-your-requirements-to-logtos-support-toolkit}

此架構包含兩個主要參與管理場景的方。根據你的具體需求和目標，這些方可能全部或僅部分參與。

我們總結了常見的使用案例，突顯每個使用者身分管理任務的主要目標以及我們提供的相關產品和 API。你可以將你的需求對應到我們的服務，以快速開始。

| 使用者            | 目標                                              | Logto 產品和 API                                                                                                                                                 |
| ----------------- | ------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 開發者            | 管理和保護使用者身分系統，直接與身分資料庫交互。  | <ul><li>[Logto Console](https://cloud.logto.io)</li><li>[Logto Management API](https://openapi.logto.io/)</li></ul>                                              |
| 終端使用者/消費者 | [管理他們自己的驗證和個人資訊。](/end-user-flows) | <ul><li>[Logto Management API](https://openapi.logto.io/)</li><li>[Account API](https://openapi.logto.io/operation/operation-getaccountcentersettings)</li></ul> |
