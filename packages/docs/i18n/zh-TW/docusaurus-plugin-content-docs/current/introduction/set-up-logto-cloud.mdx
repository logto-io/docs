# 設定 Logto Cloud

本節涵蓋基本設定步驟和關鍵操作，以有效地交付你的產品並管理開發工作流程。

## 建立 Logto 租戶 \{#create-logto-tenant}

首先，註冊一個業務並建立一個 Logto 租戶。Logto Cloud 的租戶是一個隔離的環境，你可以在其中管理使用者身分、應用程式和所有其他 Logto 資源。

你的第一個租戶將自動設置為 [**開發**](/logto-cloud/tenant-settings#development) 環境。這非常適合測試和開發用途，並且免費使用。

查看租戶設定並[了解更多](/logto-cloud)關於 Logto Cloud 服務的資訊。

## 邀請協作者 \{#invite-collaborators}

在入門過程中，你有機會邀請協作者一起進行開發工作。如果想要進行更多操作或稍後再邀請，請前往 <CloudLink to="/tenant-settings/members">租戶設定 > 成員</CloudLink>。你將看到當前的成員，並可以邀請更多人。輸入電子郵件地址並分配角色（可以批量操作），然後發送邀請。

## 移至生產租戶 \{#move-to-production-tenant}

你可能會花一些時間開發和測試 Logto 的功能，以組建概念驗證。

在你的專案完成測試並準備進入下一階段後，記得創建一個新的 [生產型租戶](/logto-cloud/tenant-settings#production)（需要 [免費或專業租戶](https://logto.io/pricing)）。因為開發租戶由於其[限制和約束](/logto-cloud/tenant-settings#development)不適合用於生產環境。

:::note

確保在創建生產租戶時立即輸入[自訂網域](/logto-cloud/custom-domain)，因為這將影響你的下一步配置。

:::

此設置允許你的專案同時擁有開發測試和生產環境，幫助你順利管理開發工作流程。

## 從現有系統遷移 \{#migrate-from-the-existing-system}

如果你正在處理現有專案並需要切換驗證提供者，Logto 支援從其他平台進行使用者遷移。你可以遷移[基本資料](/user-management/user-data#basic-data)、[自訂資料](/user-management/user-data#custom-data)、[社交身分](/user-management/user-data#social-identities)和密碼雜湊。詳情請參閱 [遷移到 Logto](../user-management/user-migration.mdx) 指南。
