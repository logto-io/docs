# 設定 Logto OSS

本節涵蓋基本設定步驟和關鍵操作，以有效地交付產品並管理你的開發工作流程，適用於 [Logto 開源服務 (OSS)](https://github.com/logto-io/logto)。

## 開始使用 Logto OSS \{#get-started-with-logto-oss}

按照 [入門指南](/logto-oss/get-started-with-oss/) 立即啟動 Logto，稍後你可以參考完整的 [部署指南](/logto-oss/deployment-and-configuration) 以供生產使用。

## Logto OSS 支援的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支援 Logto 服務的大多數核心功能，並定期更新和維護。

一些高級功能目前僅限於 Logto Cloud 版本，包括：

| OSS 的功能限制                                                                | 描述                                                                                       |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| [Logto 控制台：多租戶](/logto-cloud/tenant-settings)                          | 在控制台中創建和管理多個 Logto 租戶。                                                      |
| [Logto 控制台：協作者邀請](/logto-cloud/tenant-member-management)             | 邀請和管理一個 Logto 租戶的多個成員。                                                      |
| Logto 控制台：啟用 MFA                                                        | 通過要求在登入 Logto 控制台時進行多重要素驗證來增強安全性。                                |
| [Logto Protected App](/integrate-logto/protected-app)                         | 由 Cloudflare 提供支持的快速且非 SDK 驗證集成。                                            |
| [Logto 內建電子郵件服務](/connectors/email-connectors/built-in-email-service) | 使用內建且免費的電子郵件服務進行電子郵件傳送。                                             |
| [自定義你的 UI](/customization/bring-your-ui)                                 | 使用此功能自定義登入體驗的 UI 或流程（OSS 可以在 GitHub 上分叉代碼以直接自定義登入體驗）。 |
| [IdP 發起的 SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)            | 啟用身分提供者發起的企業單一登入 (SSO)。                                                   |
| [SAML 應用程式](/integrate-logto/saml-app)                                    | 通過 SAML 協議集成應用程式。OSS 版本限於 3 個 SAML 應用程式。                              |

提示：多租戶、成員邀請和 MFA 無法用於你的團隊登入開源 Logto 控制台。然而，你可以使用 Logto OSS 在自己的產品中實現這些功能，讓你的終端使用者可以使用。

## 保持 Logto 版本更新 \{#stay-updated-with-logto-releases}

為了讓你的 Logto 實例保持最新功能，請務必關注 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 頁面，你可以在那裡找到所有的發佈日誌。

閱讀 [升級指南](/logto-oss/upgrading-oss-version) 以了解如何在不更改代碼或資料庫結構的情況下升級 Logto。
