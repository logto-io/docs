# 配置權限

## 通過 Logto Console 配置 API 權限 \{#configure-api-permissions-via-logto-console}

權限設置在 **API 資源 (API resource)** 層級。導航至 <CloudLink to="/api-resources">Console > API resources</CloudLink>，點擊你想管理的 API 資源，然後進入「Permissions」標籤。

為權限提供名稱和詳細資訊很重要，這將使識別和管理權限更容易。

## 通過 Logto Management API 配置 API 權限 \{#configure-api-permissions-via-logto-management-api}

使用 Logto Management API 進行管理。調用相關端點。查看此 [參考資料](https://openapi.logto.io/operation/operation-listresourcescopes)。

| method | path                                                                                                                 | description                                                   |
| ------ | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------- |
| GET    | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-listresourcescopes)              | 獲取為 API 資源定義的權限範圍 (Scopes)。                      |
| POST   | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-createresourcescope)             | 為 API 資源創建新的權限範圍 (Scope)。                         |
| DELETE | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-deleteresourcescope) | 從給定資源中刪除 API 資源的權限範圍 (Scope)。                 |
| PATCH  | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-updateresourcescope) | 更新給定資源的 API 資源權限範圍 (Scope)。此方法執行部分更新。 |
