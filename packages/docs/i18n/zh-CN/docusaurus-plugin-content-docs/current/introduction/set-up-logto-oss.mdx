# 设置 Logto OSS

本节涵盖基本设置步骤和关键操作，以有效交付你的产品并管理你的开发工作流程，适用于 [Logto 开源服务 (OSS)](https://github.com/logto-io/logto)。

## 开始使用 Logto OSS \{#get-started-with-logto-oss}

按照 [入门指南](/logto-oss/get-started-with-oss/) 今天就启动 Logto，之后你可以参考完整的 [部署指南](/logto-oss/deployment-and-configuration) 以用于生产环境。

## Logto OSS 支持的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支持 Logto 服务的大多数核心功能，并定期更新和维护。

一些高级功能目前仅限于 Logto Cloud 版本，包括：

| OSS 中的功能限制                                                              | 描述                                                                                         |
| ----------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| [Logto 控制台：多个租户](/logto-cloud/tenant-settings)                        | 在控制台中创建和管理多个 Logto 租户。                                                        |
| [Logto 控制台：协作者邀请](/logto-cloud/tenant-member-management)             | 邀请和管理一个 Logto 租户的多个成员。                                                        |
| Logto 控制台：启用 MFA                                                        | 通过在登录 Logto 控制台时要求多因素认证 (MFA) 来增强安全性。                                 |
| [Logto 受保护应用](/integrate-logto/protected-app)                            | 由 Cloudflare 提供支持的快速且无需 SDK 的认证 (Authentication) 集成。                        |
| [Logto 内置电子邮件服务](/connectors/email-connectors/built-in-email-service) | 利用内置且免费的电子邮件服务进行电子邮件发送。                                               |
| [自定义你的 UI](/customization/bring-your-ui)                                 | 使用此功能自定义登录体验的 UI 或流程（OSS 可以在 GitHub 上 fork 代码以直接自定义登录体验）。 |
| [IdP 发起的 SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)            | 启用身份提供商 (IdP) 发起的企业单点登录 (SSO)。                                              |
| [SAML 应用](/integrate-logto/saml-app)                                        | 通过 SAML 协议集成应用程序。OSS 版本限制为 3 个 SAML 应用。                                  |

提示：多租户、成员邀请和 MFA 不适用于你的团队登录开源 Logto 控制台。然而，你可以使用 Logto OSS 在自己的产品中实现这些功能，使它们对你的终端用户可用。

## 保持 Logto 版本更新 \{#stay-updated-with-logto-releases}

为了使你的 Logto 实例保持最新功能，请务必关注 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 页面，你可以在那里找到所有的发布日志。

阅读 [升级指南](/logto-oss/upgrading-oss-version) 以了解如何在不更改代码或数据库架构的情况下升级 Logto。
