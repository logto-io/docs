# 升级你的 OSS 版本

Logto 遵循 [semver](https://semver.org/)，所以如果我们在更新日志中没有提到，你可以放心地升级 Logto，而无需更改你的代码或数据库架构。你可以在 [Github Release](https://github.com/logto-io/logto/releases) 中找到发布列表。

## 步骤 1：更新到新版本 \{#step-1-update-to-the-new-version}

要更新，请根据你的部署方式选择合适的方法：

- **Docker Image**：拉取最新的镜像。
- **NPM Package**：升级到最新版本。
- **Tar File**：从发布页面下载最新的 `.tar` 文件。

## 步骤 2：应用数据库更改 \{#step-2-apply-database-alterations}

使用 [Logto CLI](/logto-oss/using-cli/database-alteration/) 执行必要的数据库更改。为了实现无缝升级，不需要停止生产应用。Logto 确保新的数据库架构与现有代码兼容，因此生产将继续顺利运行而不会中断。

## 步骤 3：重启应用程序 \{#step-3-restart-the-application}

在更新代码并应用数据库更改后，重启你的应用程序以应用更改。确保新版本运行顺利，并且所有组件都按预期工作。

**提示**：为了实现零停机时间升级，可以考虑使用“交换”策略，逐步将流量从旧实例转移到新实例。这样，用户在升级过程中不会受到干扰。

你的升级现已完成！
