# Logto をインストールする

## インタラクティブ \{#interactive}

以下のコマンドを実行して、ターミナルでインタラクティブなインストールを進めます：

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npm init @logto@latest
```

  </TabItem>

</Tabs>

## オプションとサイレントインストール \{#options-and-silent-installation}

代わりに、以下のオプションを追加していくつかの質問をスキップすることができます：

```bash
# これらのオプションはすべて任意です
--db-url      Logto データベースへの Postgres URL
-p, --path    Logto インスタンスへのパス、存在しないパスである必要があります
--ss          Logto データベースのシーディングをスキップ
--oc          インストール後に公式コネクターを追加
```

完全なヘルプ情報については、`logto init --help` を実行してください。

サイレントインストールを実行する例を以下に示します（例：CI で）：

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npx @logto/cli init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>

</Tabs>

これにより、以下のことが順番に行われます：

1. Logto を `./logto` にダウンロードして抽出
2. URL `postgresql://your-postgres-dsn:5432/logto` でデータベースを初期化してシードしようとする
3. Logto 公式コネクターを `./logto/packages/core/connectors` に追加
