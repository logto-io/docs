# Logto CLI

Logto CLI は、面倒な技術的作業を手動で処理することなく、Logto インスタンスをインストール、管理、更新するための便利な方法を提供します。

:::note
現在、バイナリは提供していないため、環境には Node.js `^18.12.0` が必要です。
:::

### CLI をグローバルにインストールする \{#install-cli-globally}

お好みのパッケージマネージャーを使用して `logto` コマンドをグローバルに使用できるようにインストールします：

<Tabs>

  <TabItem value="npm" label="npm">

```bash
npm i @logto/cli -g
```

  </TabItem>
  <TabItem value="yarn" label="Yarn">

```bash
yarn global add @logto/cli
```

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

```bash
pnpm add -g @logto/cli
```

  </TabItem>

</Tabs>

各コマンドのヘルプ情報を表示するには、常に `--help` を追加できます：

```bash
logto --help
logto db seed --help
```

### ローカル CLI \{#local-cli}

CLI の組み込みバージョンは、v1.0.0-beta.11 以降のすべての Logto リリースに同梱されています。例えば、`~/logto` に Logto インスタンスが既にある場合、以下のコマンドを実行できます：

```bash
cd ~/logto
npm run cli # ローカル CLI を呼び出す
```

オプションを指定して実行したい場合は、Logto CLI のオプションの前に `--` を追加する必要があります。例：

```bash
npm run cli db seed -- --db-url postgresql://your-database-url
```

### `npx` で CLI を使用する \{#use-cli-by-npx}

Logto CLI を使用するもう一つの方法は `npx` で、インストールせずにパッケージを実行できます。

```bash
npx @logto/cli
```

これは一度限りの呼び出しに役立ちます。例：

```bash
npx @logto/cli db seed --db-url postgresql://your-database-url
```
