# データベース設定を管理する

Logto は、OIDC や Cookie キーなどの技術的な設定を維持するのに役立ちます。これらを Admin Console や Management API を介して管理する前に、CLI を使用して目的を達成できます。

## キーで設定を取得する \{#get-config-by-key}

使用法：

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config get <key> [keys...]
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config get <key> [keys...]
```

  </TabItem>

</Tabs>

OIDC プライベートキーを取得したい場合：

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config get oidc.privateKeys
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config get oidc.privateKeys
```

  </TabItem>

</Tabs>

出力は次のようになります：

```bash
oidc.privateKeys=["\n-----BEGIN PRIVATE KEY-----\nMIIJRAIBA..."]
```

利用可能なすべてのキーについては `logto db config get --help` を実行するか、詳細な説明については [Configuration](/concepts/core-service/configuration/#database-configs) を参照してください。

## キーで設定を設定する \{#set-config-by-key}

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config set <key> [keys...]
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config set <key> [keys...]
```

  </TabItem>

</Tabs>

OIDC Cookie キーを設定したい場合：

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config set oidc.cookieKeys "[\"key1\",\"key2\"]"
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config set oidc.cookieKeys "[\"key1\",\"key2\"]"
```

  </TabItem>

</Tabs>

通常、設定する値は有効な JSON 文字列であり、事前に定義された型定義に従う必要があります。利用可能なすべてのキーについては `logto db config set --help` を実行するか、詳細な説明については [Configuration](/concepts/core-service/configuration/) を参照してください。
