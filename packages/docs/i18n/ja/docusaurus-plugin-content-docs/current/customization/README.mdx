# カスタマイズ

Logto をあなた自身のものにするために、ブランディングやユーザーインターフェースから言語やドメイン設定まで、あらゆる側面をカスタマイズしましょう。

## 事前構築された UI \{#prebuilt-ui}

事前構築された UI コンポーネントから始め、基本的なブランド要素をカスタマイズします。次のいずれかを作成できます：

- プラットフォーム全体で統一された認証 (Authentication) 体験。
- 特定のアプリケーションまたは組織のための専用のサインインページ。

<DocCardList
  items={[
    {
      type: 'link',
      label: 'ブランドに合わせる',
      href: '/customization/match-your-brand',
      description:
        'すぐに使用できるサインイン体験のために基本的なブランド要素をカスタマイズする方法を学びます。',
      customProps: {
        icon: <CustomizationIcon />,
      },
    },
  ]}
/>

## カスタム UI \{#custom-ui}

認証 (Authentication) 体験を完全にコントロールするための 2 つのオプション：

  items={[

    {

/>

## コンテンツ & 言語 \{#content--languages}

Logto はデフォルトで 15 の言語をサポートしています。言語サポートを拡張したり、既存の翻訳を変更してニーズに合わせることができます。

<DocCardList
  items={[
    {
      type: 'link',
      label: '言語 & コンテンツ',
      href: '/customization/localized-languages',
      description: '新しい言語を追加し、ユーザー向けにコンテンツをカスタマイズする方法を学びます。',
      customProps: {
        icon: <LanguagesIcon />,
      },
    },
  ]}
/>

## カスタムドメイン \{#custom-domain}

認証 (Authentication) URL に独自のドメインを使用することで、ブランドの存在感を強化します。デフォルトの Logto ドメインをブランド化されたドメインに置き換えます。

<DocCardList
  items={[
    {
      type: 'link',
      label: 'カスタムドメイン',
      href: '/logto-cloud/custom-domain',
      description:
        'サインインまたはサインアップ URL として独自のドメインを使用することで、ブランドとユーザーの信頼を向上させます。',
      customProps: {
        icon: <UserFlowIcon />,
      },
    },
  ]}
/>

## ライブプレビュー \{#live-preview}

さまざまな認証 (Authentication) フロー（[サインアップ](/end-user-flows/sign-up-and-sign-in/sign-up)、[サインイン](/end-user-flows/sign-up-and-sign-in/sign-in)、[ソーシャルログイン](/end-user-flows/sign-up-and-sign-in/social-sign-in)、[パスワードを忘れた場合](/end-user-flows/sign-up-and-sign-in/reset-password)、[MFA](/end-user-flows/mfa)、[SSO](/end-user-flows/enterprise-sso) など）をライブプレビューフィーチャーを使用して試してみましょう。組み込みのデモアプリを通じて、サインイン体験を即座にテストできます。
