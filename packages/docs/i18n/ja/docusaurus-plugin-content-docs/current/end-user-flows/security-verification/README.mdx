# セキュリティ検証

認証 (Authentication) されたユーザーがパスワードの変更、支払いの実行、給与明細や銀行口座の詳細などの機密情報へのアクセスといった、機密性の高いまたはリスクの高い操作を試みる際には、追加のセキュリティ対策が不可欠です。このフローは、特に銀行、医療、政府サービスなどの機密性の高いアプリケーションにおいて重要です。

このプロセスは _セキュリティ検証_ として知られ、ユーザーに再度アイデンティティを確認させ、認可 (Authorization) されたアカウント保有者であることを確認する必要があります。セキュリティ検証を実装することで、システムは不正アクセスに対する保護を強化し、リスクの高い操作に対して重要なセキュリティ層を追加し、機密データを保護するのに役立ちます。
