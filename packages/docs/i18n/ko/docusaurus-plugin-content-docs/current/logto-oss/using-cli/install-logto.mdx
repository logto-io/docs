# Logto 설치

## 상호작용 설치 \{#interactive}

터미널에서 상호작용 설치를 진행하려면 아래 명령어를 실행하세요:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npm init @logto@latest
```

  </TabItem>

</Tabs>

## 옵션 및 무인 설치 \{#options-and-silent-installation}

대안으로, 아래 옵션을 추가하여 일부 질문을 건너뛸 수 있습니다:

```bash
# 모든 옵션은 선택 사항입니다
--db-url      Logto 데이터베이스에 대한 Postgres URL
-p, --path    Logto 인스턴스의 경로, 존재하지 않는 경로여야 합니다
--ss          Logto 데이터베이스 시딩 건너뛰기
--oc          설치 후 공식 커넥터 추가
```

전체 도움말 정보는 `logto init --help`를 실행하세요.

다음은 무인 설치를 수행하는 예시입니다 (예: CI에서):

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npx @logto/cli init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>

</Tabs>

다음 작업을 순서대로 수행합니다:

1. Logto를 `./logto`에 다운로드하고 추출합니다.
2. URL `postgresql://your-postgres-dsn:5432/logto`로 데이터베이스를 초기화하고 시딩을 시도합니다.
3. Logto 공식 커넥터를 `./logto/packages/core/connectors`에 추가합니다.
