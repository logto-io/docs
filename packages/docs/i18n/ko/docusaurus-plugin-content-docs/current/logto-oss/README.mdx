# Logto OSS

Logto는 오픈 소스 (OSS) 버전과 [클라우드](/logto-cloud) 버전을 모두 제공합니다. 이 섹션에서는 OSS 솔루션에 중점을 둡니다: <Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS는 개발자에게 완전한 커스터마이징 기능과 데이터에 대한 완전한 제어를 제공하는 무료 아이덴티티 솔루션입니다. Logto OSS는 코드베이스를 특정 요구에 맞게 조정할 수 있는 유연성을 제공하며, 데이터 주권을 우선시하고 애플리케이션과 함께 성장할 수 있는 맞춤형 아이덴티티 제공자 솔루션을 원하는 사람들에게 적합합니다.

## Logto OSS가 지원하는 기능 \{#feature-supported-by-logto-oss}

Logto OSS는 Logto 서비스의 대부분의 핵심 기능을 지원하며 정기적으로 업데이트 및 유지 관리됩니다.

일부 고급 기능은 현재 Logto Cloud 버전에만 독점적으로 제공되며, 다음을 포함합니다:

| OSS의 기능 제한                                                                 | 설명                                                                                                                                                 |
| ------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto 콘솔: 다중 테넌트](/logto-cloud/tenant-settings)                         | 콘솔에서 여러 Logto 테넌트를 생성하고 관리합니다.                                                                                                    |
| [Logto 콘솔: 협력자 초대](/logto-cloud/tenant-member-management)                | 하나의 Logto 테넌트의 여러 멤버를 초대하고 관리합니다.                                                                                               |
| Logto 콘솔: MFA 활성화                                                          | Logto 콘솔에 로그인할 때 다단계 인증 (MFA)을 요구하여 보안을 강화합니다.                                                                             |
| [Logto 보호 앱](/integrate-logto/protected-app)                                 | Cloudflare에 의해 지원되는 빠르고 비 SDK 인증 통합입니다.                                                                                            |
| [Logto 내장 이메일 서비스](/connectors/email-connectors/built-in-email-service) | 이메일 전송을 위한 내장 및 무료 이메일 서비스를 활용합니다.                                                                                          |
| [UI 가져오기](/customization/bring-your-ui)                                     | 이 기능을 사용하여 로그인 경험의 UI 또는 흐름을 커스터마이징하세요 (OSS는 GitHub에서 코드를 포크하여 로그인 경험을 직접 커스터마이징할 수 있습니다.) |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)           | 아이덴티티 제공자 (IdP) 시작 엔터프라이즈 싱글 사인온 (SSO)을 활성화합니다.                                                                          |
| [SAML 앱](/integrate-logto/saml-app)                                            | SAML 프로토콜을 통해 애플리케이션을 통합합니다. OSS 버전은 3개의 SAML 앱으로 제한됩니다.                                                             |

팁: 다중 테넌시, 멤버 초대 및 MFA는 오픈 소스 Logto 콘솔에 팀이 로그인하는 데 사용할 수 없습니다. 그러나 Logto OSS를 사용하여 이러한 기능을 자체 제품에 구현하여 최종 사용자에게 제공할 수 있습니다.
