# 권한 구성

## Logto 콘솔을 통해 API 권한 구성하기 \{#configure-api-permissions-via-logto-console}

권한은 **API 리소스** 수준에서 설정됩니다. <CloudLink to="/api-resources">콘솔 > API 리소스</CloudLink>로 이동하여 관리하려는 API 리소스를 클릭한 다음 "Permissions" 탭으로 이동하세요.

권한에 대한 이름과 세부 정보를 제공하는 것이 중요합니다. 이는 권한을 식별하고 관리하는 데 도움이 됩니다.

## Logto Management API를 통해 API 권한 구성하기 \{#configure-api-permissions-via-logto-management-api}

Logto Management API를 사용하여 관리하세요. 관련 엔드포인트에 호출을 수행하세요. 이 [참조](https://openapi.logto.io/operation/operation-listresourcescopes)를 확인하세요.

| method | path                                                                                                                 | description                                                                                      |
| ------ | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| GET    | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-listresourcescopes)              | API 리소스에 대해 정의된 스코프를 가져옵니다.                                                    |
| POST   | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-createresourcescope)             | API 리소스에 대한 새로운 스코프를 생성합니다.                                                    |
| DELETE | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-deleteresourcescope) | 주어진 리소스에서 API 리소스 스코프를 삭제합니다.                                                |
| PATCH  | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-updateresourcescope) | 주어진 리소스에 대한 API 리소스 스코프를 업데이트합니다. 이 메서드는 부분 업데이트를 수행합니다. |
