# 결제 및 가격

Logto에서는 [Pro 요금제](https://logto.io/pricing)가 완전히 셀프 서비스로 제공되며, 투명성을 위해 설계되어 있어 결제를 쉽게 이해하고 관리할 수 있습니다.

## 구조 이해하기 \{#understand-the-structure}

기본 구조는 다음과 같습니다:

```math
\begin{align*}
\text{Your next bill} &= \$16 \text{(기본 가격)} \\
&+ \text{추가 비용 (청구되지 않은 사용량)} \\
&+ \text{추가 비용 (다음 주기 요금)}
\end{align*}
```

콘솔에서는 테넌트 설정 섹션에서 "Your next bill" 섹션을 찾아 다가오는 요금을 모니터링하고 제어할 수 있습니다.

## 항목 분해 \{#item-breakdown}

몇 가지 기능만이 추가 구조의 일부임을 유념하세요.

동시에, Logto는 사용한 만큼 지불하는 방식과 비례 배분된 가격 모델을 제공하여 서비스를 편리하게 최대한 활용할 수 있도록 합니다.

|                          | 포함된 할당량 | 추가 비용             |
| ------------------------ | ------------- | --------------------- |
| **토큰**                 | 100K          | $0.08 / mo / 100 이후 |
| **기계 간 애플리케이션** | 1             | $8 **_각각_** / 월    |
| **API 리소스**           | 3             | $4 **_각각_** / 월    |
| **엔터프라이즈 SSO**     | 0             | $48 **_각각_** / 월   |
| **다단계 인증**          | _N/A_         | $48 / 월              |
| **조직**                 | 무제한        | $48 / 월              |
| **테넌트 멤버**          | 3             | $8 **_각각_** / 월    |

이러한 요소들을 고려하면, 알고리즘은 다음과 같습니다:

```math
\begin{align*}
\text{Your next bill} &= \$16 \\
&+ \text{추가 단가} \times \\
\quad & (\text{추가된 수량} \times \text{생성 시 남은 비례 배분 시간} \\
\quad &- \text{제거된 수량} \times \text{삭제 시 남은 비례 배분 시간}) \\
\quad &+ \text{추가 단가} \times \text{남은 수량}
\end{align*}
```

1. **추가 단가**: 이 추가 기능의 단가
2. **추가된 수량**: **_현재_** 청구 주기에서 사용자가 추가한 추가 기능의 양
3. **제거된 수량**: **_현재_** 청구 주기에서 사용자가 제거한 추가 기능의 양
4. **생성 시 남은 비례 배분 시간**: 리소스를 생성할 때 **_현재_** 주기에 남은 비례 배분 시간
5. **삭제 시 남은 비례 배분 시간**: 리소스를 삭제할 때 **_현재_** 주기에 남은 비례 배분 시간
6. **남은 수량**: **_다음_** 주기에서 일관된 청구를 위해 필요한 양

두 가지 예를 들어 어떻게 작동하는지 살펴보겠습니다.

### 예제 1: 변경 없이 일관된 청구 \{#example-1-consistent-billing-with-no-changes}

항목을 추가하거나 제거하지 않는 한, 청구서는 일관되게 유지됩니다.

예를 들어, Pro 요금제를 사용 중이고 2개의 SSO 서비스를 사용 중이라면, 변경하지 않는 한 청구서는 일관되게 유지됩니다.

```math
\text{Your next bill} = \$ 16 + \$ 48 \times 2 = \$ 112
```

### 예제 2: 추가 기능 추가 또는 변경은 다가오는 청구서에 영향을 미칩니다 \{#example-2-adding-add-ons-or-making-changes-will-affect-your-upcoming-bills}

이 경우, 다가오는 청구서는 다양한 경우에 따라 약간 다르게 나타날 수 있습니다. 알고리즘은 여전히 작동합니다.

**사례 1: 추가 기능 없이 Pro 요금제를 구독하고 엔터프라이즈 SSO를 10일 동안만 테스트했습니다.**

매월 5일에 구독이 갱신되고, 20일에 엔터프라이즈 SSO를 추가하면 그 달에 15일 동안 사용하게 됩니다. 30일에 엔터프라이즈 SSO를 삭제하면 10일 동안만 사용한 것입니다.

```math
\$16 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$32
```

**사례 2: 추가 기능 없이 Pro 요금제를 구독했습니다. API 리소스를 추가하고 제거한 후, 5개의 API 리소스를 사용했습니다.**

이 경우, 이 청구 주기 동안 추가 기능을 업데이트했으므로 변경 사항은 다음 청구서에 반영됩니다. 변경 후 첫 달은 약간 더 높을 수 있습니다. 청구서에는 $16 기본 가격, 청구되지 않은 사용량에 대한 추가 비용, 다음 반복 주기에 대한 전체 요금이 포함됩니다.

청구 주기는 매월 1일에 시작되며, 처음 3개의 무료 API 리소스를 사용했습니다. 이번 달에는 두 가지 작업을 수행했습니다:

1. 5일에 4개의 리소스를 추가했습니다.
2. 15일에 그 중 2개의 리소스를 삭제했습니다.

결과적으로, 다가오는 주기에는 2개의 API 리소스를 일관되게 사용할 수 있습니다.

```math
\$16 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$50.67
```

### 추가 기능 사용 시간은 어떻게 결정되나요? \{#how-is-the-add-on-usage-time-determined}

우리는 사용한 만큼 지불하는 모델을 사용하며, 자원을 자주 추가하거나 제거할 경우 어떻게 되는지 궁금할 수 있습니다. 추가 기능은 활성 상태인 정확한 기간 동안 사용된 것으로 간주됩니다. 걱정하지 마세요. Stripe로 구동되는 우리의 시스템은 사용량을 초 단위로 계산하여 각 리소스가 사용된 정확한 시간에 기반하여 정확한 청구를 보장합니다.

Pro 요금제를 구독 취소하면 사용하지 않은 추가 기능 비용을 환불해 드리며, 해당 금액은 최종 청구서에서 차감됩니다.

## 업그레이드 또는 다운그레이드 \{#upgrade-or-downgrade}

### 무료 요금제에서 Pro로 업그레이드 \{#free-plan-upgrade-to-pro}

업그레이드 및 다운그레이드는 프로덕션 테넌트에만 적용됩니다. 무료 요금제에서 업그레이드를 고려해야 할 상황은 다음과 같습니다:

1. MAU 또는 토큰 사용량이 할당량을 초과하면, 적절한 시기에 업그레이드가 필요하다는 알림 이메일을 보내드립니다.
2. 프리미엄 요금제에서만 사용할 수 있는 기능에 접근하고 싶을 때.

### Pro 요금제에서 엔터프라이즈로 업그레이드 \{#pro-plan-upgrade-to-enterprise}

Pro 요금제가 요구 사항을 충족하지 못하고 전체 엔터프라이즈 수준의 지원을 원하신다면, 맞춤형 엔터프라이즈 요금제를 논의하기 위해 [문의하세요](https://logto.io/contact).

### Pro 요금제에서 무료 요금제로 다운그레이드 \{#pro-plan-downgrade-to-free-plan}

Pro 요금제 사용을 중단하기로 결정한 경우, 무료 요금제로 전환할 수 있지만, 무료 요금제의 할당량에 맞게 사용량을 조정해야 합니다.

## 향후 가격 계획 변경 \{#future-pricing-plan-change}

안정적이고 예측 가능한 청구 경험을 제공하기 위해, 가격 변경이 발생할 때마다 현재 요금제를 유지할 것입니다.

### 토큰 소비와 MAU의 연결은 무엇인가요? \{#what-is-the-connection-between-token-consumption-and-mau}

</summary>

Logto는 토큰 기반 인증을 사용합니다. 한 사용자가 한 번 로그인할 때 ID 토큰, 액세스 토큰, 리프레시 토큰을 포함하여 두세 개의 토큰을 소비할 수 있습니다. M2M 또는 조직과 같은 기능을 사용하는 경우, M2M 토큰 또는 조직 토큰과 같은 추가 토큰이 포함될 수 있습니다. 토큰 소비는 사용자가 시스템과 상호작용하는 방식에 따라 달라집니다. 이를 추적할 수 있도록, <CloudLink to="/dashboard">Console > Dashboard</CloudLink>에서 실시간 토큰 사용량 모니터링을 제공합니다. 예를 들어, 무료 요금제는 100k 토큰을 무료로 제공하며, 이는 약 30k-50k MAU (월간 활성 사용자)를 지원할 수 있습니다.

</details>
