# 테넌트 멤버 관리

## 협업자 초대 \{#invite-collaborators}

<CloudLink to="/tenant-settings/members">콘솔 > 테넌트 설정 > 멤버</CloudLink>로 이동하세요. 현재
멤버를 확인하고 더 많은 멤버를 초대할 수 있습니다. 이메일 주소를 입력하고 역할을 할당한 후
(일괄적으로 할 수 있습니다) 전송하세요.

Logto는 Logto Cloud에서 두 가지 다른 역할로 팀 멤버를 테넌트에 초대할 수 있습니다:

- **관리자 (Admin):** 모든 테넌트 리소스에 대한 전체 접근 및 관리 권한.
- **협업자 (Collaborator):** 아이덴티티 관련 리소스를 관리할 수 있지만 제한이 있습니다:
  - 청구 정보에 접근할 수 없음
  - 다른 멤버를 초대하거나 관리할 수 없음
  - 테넌트 이름을 변경할 수 없음
  - 테넌트를 삭제할 수 없음

### 멤버 제한 \{#members-limit}

| 테넌트 유형          | 멤버 제한                                      |
| -------------------- | ---------------------------------------------- |
| 개발 테넌트          | 최대 3명 (더 많은 멤버를 원할 경우 문의하세요) |
| 무료 프로덕션 테넌트 | 1명 (관리자만)                                 |
| 프로 프로덕션 테넌트 | 3명으로 시작, 추가 멤버당 $8                   |

### 초대 관리 \{#manage-invitation}

초대 탭에서는 과거 초대와 그 상태를 보여줍니다

| 상태    | 설명                                    | 가능한 작업 |
| ------- | --------------------------------------- | ----------- |
| 대기 중 | 초대가 발송되었으나 아직 수락되지 않음. | 재전송      |
| 수락됨  | 사용자가 이제 테넌트 멤버가 됨.         | 삭제        |
| 만료됨  | 초대가 7일 제한을 초과함.               | 삭제        |
| 취소됨  | 관리자가 초대를 취소함.                 | 삭제        |

이미 대기 중인 초대가 있거나 멤버인 사람은 초대할 수 없습니다.

### 테넌트 멤버 관리 (관리자 전용) \{#manage-tenant-members-admin-only}

관리자로서 팀을 관리할 수 있는 여러 방법이 있습니다:

- **멤버 초대 또는 제거:** 필요에 따라 새로운 팀 멤버를 추가하거나 접근을 제거합니다.
- **멤버 역할 변경:** 필요한 접근 수준에 따라 '관리자' 또는 '협업자' 역할을 할당합니다.
- **테넌트 소유권 이전:** 필요시 테넌트의 소유권을 다른 멤버에게 이전할 수 있습니다. 먼저 다른 멤버에게 관리자 역할을 부여한 후, 일반 멤버로 남거나 테넌트를 완전히 떠날 수 있습니다.
- **여러 관리자 추가:** 다른 팀 멤버와 함께 관리 책임을 공유하여 협력적인 관리를 할 수 있습니다.
