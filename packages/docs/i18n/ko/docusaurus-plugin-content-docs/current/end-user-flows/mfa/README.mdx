# 다단계 인증 (MFA)

## MFA란 무엇인가요? \{#what-is-mfa}

[다단계 인증 (MFA)](https://auth.wiki/mfa)은 로그인 과정에서 추가적인 보호 계층을 제공하는 보안 방법입니다. 사용자는 디지털 아이덴티티를 확립하기 위해 여러 자격 증명을 제공해야 합니다.

주요 인증 유형은 두 가지입니다:

- **SFA/1FA (단일 요소 인증)**: 이는 초기 로그인 방법으로, 일반적으로 [사용자 이름 / 이메일 / 전화번호와 비밀번호](/end-user-flows/sign-up-and-sign-in/sign-in) 또는 [소셜 로그인](/end-user-flows/sign-up-and-sign-in/social-sign-in)을 요구합니다.
- **MFA/2FA (다단계 인증 / 이중 요소 인증)**: MFA는 계정에 접근하기 위해 최소 두 가지 다른 인증 방법을 요구하여, 무단 접근에 대한 방어를 효과적으로 강화합니다.

인증 요소는 아이덴티티를 확인하는 수단입니다. 선택할 수 있는 다양한 요소가 속성에 따라 분류됩니다:

| 유형   | 의미           | 인증 요소 (Logto 지원)                             |
| ------ | -------------- | -------------------------------------------------- |
| 지식   | 알고 있는 것   | 비밀번호, 이메일 인증 코드, 백업 코드              |
| 소유   | 가지고 있는 것 | SMS 인증 코드, 인증 앱 OTP, 하드웨어 OTP (보안 키) |
| 고유성 | 본인인 것      | 지문, 얼굴 인식과 같은 생체 인식                   |

MFA 흐름에서는 두 번째 인증 단계가 첫 번째와 다른 속성 유형 (지식 / 소유 / 고유성)을 사용해야 합니다. 예를 들어, 첫 번째 요소로 "비밀번호 (지식)"를 사용하고 두 번째 요소로 "인증 앱 OTP (소유)"를 사용하는 것은 다양한 공격 벡터를 효과적으로 완화할 수 있습니다.

## 왜 MFA가 필요한가요? \{#why-do-we-need-an-mfa}

MFA는 특히 B2B 및 B2E 서비스에 중요한 보안 조치입니다. 오늘날의 디지털 환경에서 여러 이유로 널리 채택되고 있습니다:

- **계정 해킹**: 무단 계정 접근은 여전히 중요한 보안 위협입니다. 그러나 MFA는 강력한 보호를 제공하여, 특히 비밀번호 유출로 인한 계정 해킹의 99.9%를 효과적으로 차단합니다. 이는 비밀번호 없는 로그인, 강력한 비밀번호 정책, 비밀번호 관리자, 공격에 대한 보호 조치와 같은 전략으로 보완되는 비용 효율적인 보안 강화책입니다.
- **SaaS 채택**: 많은 기업이 직원 보호 및 민감한 회사 데이터와 자산을 보호하기 위해 MFA를 점점 더 많이 구현하고 있습니다. LastPass의 설문 조사에 따르면, 전 세계 기업의 57%가 현재 MFA를 사용하고 있으며, 이는 전년 대비 12% 증가한 수치입니다.
- **규제 준수**: MFA는 조직이 GDPR 및 NIST와 같은 데이터 보호 규정을 준수하는 데 도움을 주어 사용자 데이터의 보안을 보장합니다.

## Logto 지원 \{#logto-support}

Logto는 복잡한 설정 없이 원클릭 토글로 MFA 활성화 과정을 간소화합니다. [인증 요소 활성화](/end-user-flows/mfa/configure-mfa)에 대한 빠른 가이드를 시작하세요.

**지원되는 MFA 요소**:

- [인증 앱 OTP](/end-user-flows/mfa/authenticator-app-otp): Google Authenticator 또는 Authy와 같은 인증 앱에서 생성된 시간 기반 일회용 비밀번호 (TOTP)를 사용하세요.
- [패스키 (WebAuthn)](/end-user-flows/mfa/webauthn): 보안 키 또는 생체 인증을 사용하여 비밀번호 없는 경험을 제공합니다.
- [백업 코드](/end-user-flows/mfa/backup-codes): 비상 접근을 위한 일회용 백업 코드를 생성하세요.
