# 인증 (Authentication) 매개변수

표준 OIDC 로그인 흐름에서 클라이언트 애플리케이션은 인증 요청을 시작하여 사용자를 Logto 호스팅의 범용 로그인 경험 웹 페이지로 리디렉션합니다. 로그인 경험 설정에 따라 사용자는 다양한 식별자, 인증 방법 및 타사 소셜 또는 엔터프라이즈 SSO 커넥터를 사용하여 로그인하거나 가입할 수 있습니다.

표준 [OIDC 인증 매개변수](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest) 외에도, 우리의 제품은 최종 사용자를 위한 원하는 로그인 경험을 맞춤화할 수 있는 여러 사용자 정의 인증 매개변수를 도입합니다.

이는 특히 다른 사용자 세그먼트에 대해 특정 로그인 흐름을 강제하고자 할 때 유용합니다. 다음 시나리오를 포함하되 이에 국한되지 않습니다:

### 신규 사용자에 대한 직접 가입 \{#direct-sign-up-for-new-users}

    특정 등록 캠페인을 위해, 신규 사용자를 기본 로그인 양식을 우회하여 직접 가입 페이지로 안내하여 원활한 온보딩 경험을 보장할 수 있습니다.

    **사용**: [첫 화면](/end-user-flows/authentication-parameters/first-screen) 매개변수 → `first_screen=register`

### 구독 링크에서 이메일이 미리 채워진 가입 \{#email-prefilled-sign-up-from-a-subscription-link}

    자체 호스팅 구독 양식을 통해 이메일 주소를 수집하는 경우, 이러한 매개변수를 사용하여 신규 구독자를 이메일 필드가 미리 채워진 상태로 가입 페이지로 리디렉션하여 등록 과정을 쉽게 완료할 수 있습니다.

    **사용**: [첫 화면](/end-user-flows/authentication-parameters/first-screen) 매개변수 → `first_screen=identifier:sign_up&identifier=email&login_hint=foo@gmail.com`

### 직접 비밀번호 재설정 링크 \{#direct-password-reset-link}

    새로운 비밀번호를 설정해야 하는 신규 사용자에게 직접 링크를 보내고 이메일 필드가 미리 채워진 상태로 비밀번호 재설정 페이지로 리디렉션합니다.

    **사용**: [첫 화면](/end-user-flows/authentication-parameters/first-screen) 매개변수 → `first_screen=reset_password`

### 즉시 소셜 로그인 \{#instant-social-login}

    사이트에 소셜 미디어 로그인 버튼을 직접 제공하는 경우, 이러한 매개변수를 사용하여 기본 로그인 양식을 건너뛰고 사용자가 선택한 소셜 제공자로 직접 인증할 수 있습니다.

    **사용**: [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in#social-sign-in) 매개변수 → `direct_sign_in=social:google`

### 엔터프라이즈 고객을 위한 직접 SSO 로그인 \{#direct-sso-sign-in-for-enterprise-customers}

    여러 엔터프라이즈 클라이언트를 서비스하는 제품의 경우, 이러한 매개변수를 사용하여 특정 SSO 제공자로 사용자를 로그인시키는 전용 SSO 버튼을 표시하여 범용 로그인 양식을 건너뛰고 각 조직에 원활한 경험을 제공합니다.

    **사용**: [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in#enterprise-sso) 매개변수 → `direct_sign_in=sso:1234567890`

### 다른 사이트에 표시되는 맞춤형 로그인 방법 \{#custom-sign-in-methods-displayed-for-different-sites}

    제품이 내부 팀과 소비자 또는 다른 조직을 모두 서비스하는 경우, 로그인 경험을 이에 맞게 조정할 수 있습니다. 예를 들어: - 내부 도구의 경우, 이메일 / 사용자 이름 및 비밀번호 로그인만 표시합니다. - 소비자 대상 페이지의 경우, Google 로그인을 포함한 모든 사용 가능한 로그인 옵션을 제공합니다.

    **사용**: 웹사이트 유형에 따라 다른 [첫 화면](/end-user-flows/authentication-parameters/first-screen) 또는 [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in) 설정을 사용하세요.

계속 읽어 [첫 화면](/end-user-flows/authentication-parameters/first-screen)을 맞춤화하거나 사용자에게 [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in) 경험을 제공하기 위한 인증 매개변수에 대해 자세히 알아보세요.
