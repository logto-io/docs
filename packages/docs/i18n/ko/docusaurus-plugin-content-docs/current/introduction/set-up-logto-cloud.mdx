# Logto Cloud 설정

이 섹션에서는 제품을 효과적으로 제공하고 개발 워크플로우를 관리하기 위한 기본 설정 단계와 주요 작업을 다룹니다.

## Logto 테넌트 생성 \{#create-logto-tenant}

먼저, 비즈니스에 가입하고 Logto 테넌트를 생성하세요. Logto Cloud의 테넌트는 사용자 아이덴티티, 애플리케이션 및 기타 모든 Logto 리소스를 관리할 수 있는 독립된 환경입니다.

첫 번째 테넌트는 자동으로 [**개발**](/logto-cloud/tenant-settings#development) 환경으로 설정됩니다. 이는 테스트 및 개발 목적으로 이상적이며 무료로 사용할 수 있습니다.

테넌트 설정을 확인하고 Logto 클라우드 서비스에 대해 [더 알아보세요](/logto-cloud).

## 협력자 초대 \{#invite-collaborators}

온보딩 중에 협력자를 초대하여 함께 개발 작업을 수행할 수 있는 기회가 있습니다. 더 많은 작업을 하거나 나중에 초대하려면 <CloudLink to="/tenant-settings/members">테넌트 설정 > 멤버</CloudLink>로 이동하세요. 현재 멤버를 확인하고 더 많은 멤버를 초대할 수 있습니다. 이메일 주소를 입력하고 역할을 할당한 후 (일괄적으로 할 수 있습니다) 전송하세요.

## 프로덕션 테넌트로 이동 \{#move-to-production-tenant}

Logto의 기능과 특징을 개발하고 테스트하여 개념 증명을 구성하는 데 시간을 보낼 수 있습니다.

프로젝트가 테스트를 완료하고 다음 단계로 준비되면, 새로운 [프로덕션 유형 테넌트](/logto-cloud/tenant-settings#production)를 생성하는 것을 잊지 마세요 ([무료 또는 프로 테넌트](https://logto.io/pricing)가 필요합니다). 개발 테넌트는 [제한 및 제약](/logto-cloud/tenant-settings#development) 때문에 프로덕션 용도로 사용되지 않기 때문입니다.

:::note

프로덕션 테넌트를 생성하자마자 [커스텀 도메인](/logto-cloud/custom-domain)을 입력해야 합니다. 이는 다음 단계의 구성에 영향을 미칩니다.

:::

이 설정은 프로젝트가 개발 테스트 및 프로덕션 환경을 모두 갖추도록 하여 개발 워크플로우를 원활하게 관리할 수 있도록 도와줍니다.

## 기존 시스템에서 마이그레이션 \{#migrate-from-the-existing-system}

기존 프로젝트에서 작업 중이고 인증 제공자를 변경해야 하는 경우, Logto는 다른 플랫폼에서 사용자 마이그레이션을 지원합니다. [기본 데이터](/user-management/user-data#basic-data), [커스텀 데이터](/user-management/user-data#custom-data), [소셜 아이덴티티](/user-management/user-data#social-identities) 및 비밀번호 해시를 마이그레이션할 수 있습니다. 자세한 내용은 [Logto로 마이그레이션](../user-management/user-migration.mdx) 가이드를 참조하세요.
