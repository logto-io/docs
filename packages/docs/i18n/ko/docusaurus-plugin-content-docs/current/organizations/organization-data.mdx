# 조직 데이터 구조

조직 수준에서는 생성, 업데이트, 읽기 및 삭제 (CRUD)할 수 있는 여러 속성이 있습니다.

## 조직 ID \{#organization-id}

*조직 ID*는 각 조직에 대한 고유 식별자입니다. 이는 조직 수준의 로그인 경험을 구현하고 조직 토큰을 검색하는 데 유용합니다.

## 이름 \{#name}

*이름*은 조직 수준의 로그인을 지원하며 필요에 따라 조직 수준의 제품 인터페이스에 통합될 수 있습니다.

## 설명 \{#description}

_설명_ 필드는 조직을 식별하고 라벨링하는 데 도움이 되는 텍스트를 추가할 수 있습니다.

## 조직 로고 \{#organization-logos}

로그인 경험에서 클라이언트의 조직 로고를 동적으로 표시하려면, 조직 설정 페이지에 조직 로고를 업로드할 수 있습니다.

자세한 내용은 [조직별 로고](/customization/match-your-brand/#organization-specific-branding)를 참조하세요.

## 사용자 정의 데이터 \{#custom-data}

*사용자 정의 데이터*는 조직에 대한 추가 정보를 저장하는 데 사용되는 JSON 객체입니다. 이는 개발자가 개발 목적으로 추가 조직 세부 정보를 저장할 수 있는 유연성을 제공합니다.

## MFA가 필요한가요? \{#is-mfa-required}

*isMfaRequired*는 조직에 대해 다단계 인증 (MFA)이 필수인지 여부를 나타냅니다. `true`로 설정된 경우, 모든 구성원은 조직에 접근하기 위해 로그인 시 MFA를 통과해야 합니다. 이 보안 정책 설정은 조직 수준에서 구성됩니다.

자세한 내용은 [조직 관리](/organizations/organization-management#require-mfa-for-organization-members)를 참조하세요.

## 생성 시각 \{#created-at}

*createdAt*은 조직이 생성된 시간대가 포함된 타임스탬프입니다.

## 테넌트 ID \{#tenant-id}

*tenantId*는 조직이 속한 테넌트를 식별합니다.
