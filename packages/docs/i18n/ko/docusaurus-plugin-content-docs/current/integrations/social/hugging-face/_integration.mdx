## Hugging Face 계정으로 로그인하기 \{#sign-in-with-hugging-face-account}

[Hugging Face 웹사이트](https://huggingface.co/)로 이동하여 Hugging Face 계정으로 로그인하세요. 계정이 없으면 새 계정을 등록할 수 있습니다.

## Hugging Face에서 OAuth 앱 생성하기 \{#create-an-oauth-app-in-the-hugging-face}

[OAuth 앱 생성](https://huggingface.co/docs/hub/en/oauth#creating-an-oauth-app) 가이드를 따라 새 애플리케이션을 등록하세요.

생성 과정에서 다음 정보를 제공해야 합니다:

- **애플리케이션 이름**: 애플리케이션의 이름.
- **홈페이지 URL**: 애플리케이션의 홈페이지 또는 랜딩 페이지 URL.
- **로고 URL**: 애플리케이션의 로고 URL.
- **스코프**: OAuth 앱에 허용된 스코프. Hugging Face 커넥터의 경우, 사용자의 프로필 정보를 얻기 위해 `profile`을, 사용자의 이메일 주소를 얻기 위해 `email`을 주로 사용합니다. 이러한 스코프를 사용하려면 Hugging Face OAuth 앱에서 허용되었는지 확인하세요.
- **리디렉션 URI**: 사용자가 인증된 후 리디렉션할 URL. Hugging Face 커넥터를 생성할 때 Logto 관리 콘솔에서 또는 생성된 Hugging Face 커넥터 세부 정보 페이지에서 리디렉션 URI를 찾을 수 있습니다.

## Hugging Face OAuth 앱 관리하기 \{#managing-hugging-face-oauth-apps}

[연결된 애플리케이션](https://huggingface.co/settings/connected-applications) 페이지로 이동하여 기존 OAuth 앱을 추가, 편집 또는 삭제할 수 있습니다. 해당 OAuth 앱 설정 페이지에서 `Client ID`를 찾고 `App secrets`를 생성할 수 있습니다.

## 커넥터 구성하기 \{#configure-your-connector}

이전 섹션에서 언급한 OAuth 앱 세부 정보 페이지에서 얻은 *Client ID*와 *App Secret*으로 `clientId`와 `clientSecret` 필드를 작성하세요.

`scope`는 [Hugging Face 지원 스코프](https://huggingface.co/docs/hub/en/oauth#currently-supported-scopes)의 공백으로 구분된 목록입니다. 제공되지 않으면 기본적으로 `profile`로 설정됩니다. Hugging Face 커넥터의 경우, 사용하고자 하는 스코프는 `profile`과 `email`입니다. `profile` 스코프는 사용자의 프로필 정보를 얻기 위해 필요하며, `email` 스코프는 사용자의 이메일 주소를 얻기 위해 필요합니다. Hugging Face OAuth 앱에서 이러한 스코프가 허용되었는지 확인하세요 ( [Hugging Face에서 OAuth 앱 생성하기](#create-an-oauth-app-in-the-hugging-face) 섹션에서 구성됨).

### 구성 유형 \{#config-types}

| 이름         | 유형   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
