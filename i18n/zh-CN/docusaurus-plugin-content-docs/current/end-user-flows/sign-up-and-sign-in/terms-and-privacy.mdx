# 条款和隐私

为了确保你的产品保持开放和透明，在登录和注册页面添加指向使用条款和隐私政策的链接。这可以让你满足特定行业的合规要求。

## 配置条款和隐私 \{#configure-terms--privacy}

1. 导航到 <CloudLink to="/sign-in-experience/content">控制台 > 登录体验 > 内容</CloudLink>。
2. 添加“使用条款”和“隐私政策”的外部链接。
3. 根据特定地区或行业的合规需求设置“同意条款”的策略。

:::note
如果该值为空，你无需在 Logto 的登录流程中管理条款和隐私协议。你可以在用户登录到你的应用后处理这些问题。
:::

## 用户同意条款的体验 \{#user-experience-for-agreeing-to-terms}

Logto 提供多种预构建流程，以根据不同的合规要求处理用户同意条款的体验：

### 选项 1：在继续登录 / 注册时自动同意条款 \{#option-1-agree-to-terms-automatically-on-continue-sign-insign-up}

此策略通过在用户继续登录或注册过程时自动同意条款，提供最流畅的用户体验。
用户不会被明确提示同意条款。

![自动同意条款](./assets/auto-agree-to-terms.png)

### 选项 2：仅在注册时同意条款 \{#opetion-2-agree-to-terms-on-sign-up-only}

此策略要求新用户手动同意**条款和隐私政策**。根据欧盟的 [GDPR](https://gdpr-info.eu/art-4-gdpr/) 和加州的 [CCPA](https://oag.ca.gov/privacy/ccpa)，企业在收集个人信息之前必须获得用户的知情同意。

当用户首次使用社交注册（例如，Google 或 GitHub）时，在重定向到 Logto 后，会弹出一个窗口提示他们同意**条款和隐私政策**。此同意对于后续的社交登录不再需要。

![仅在注册时同意条款](./assets/sign-up-only-agree-to-terms.png)

### 选项 3：在登录和注册时同意条款 \{#option-3-agree-to-terms-on-sign-in-and-sign-up}

此策略要求用户在每次登录或注册时同意**条款和隐私政策**。此策略适用于需要用户每次登录时同意条款的应用程序。用户必须勾选框以同意条款后才能继续。此方法符合中国的 PIPL 和巴西的 LGPD。

![在登录和注册时同意条款](./assets/sign-in-agree-to-terms.png)
