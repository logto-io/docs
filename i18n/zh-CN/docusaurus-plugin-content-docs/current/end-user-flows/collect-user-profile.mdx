# 收集用户资料

除了用于用户登录的认证 (Authentication) 标识符和凭证外，你的应用可能还需要收集额外的用户资料信息，这些信息存储在 Logto 中，便于通过 JWT 声明 (Claims) 或 API 检索，以提供个性化的产品体验。你可以通过以下方式收集用户数据：

- **新用户注册时**：使用 [收集用户资料](#quick-start) 功能，在注册流程中添加一个开箱即用的“告诉我们你的信息”步骤，收集额外的用户资料信息。新用户必须填写所有必填项，注册才算完成。本篇文档重点介绍这种方式。

- **用户注册后**：通过 [Account API](/end-user-flows/account-settings/by-account-api) 在引导流程、账户中心或产品使用过程中实现自助更新用户资料信息的体验。

## 主要优势 \{#key-benefits}

**收集用户资料**功能让你可以在终端用户注册体验中收集额外的用户信息。我们建议只收集产品所需的关键信息，避免冗长的注册流程影响用户转化率。

该功能可以帮助你：

- **获取全面的用户数据**：收集任何[用户数据](/user-management/user-data)以满足业务或合规需求，包括 [OIDC 标准用户属性](#basic-user-data-fields)和[自定义数据](#custom-user-data-fields)。

- **灵活的字段自定义**：可选择多种[字段类型](#field-types)，如文本、数字、日期、复选框、下拉选择（单选）、URL 和正则校验，以满足你的具体数据需求。

- **优化用户体验**：可自定义标签、描述、占位符和校验规则。根据业务需求将字段设置为必填或选填。

- **内置字段配置**：为常见用户属性提供预设的基础数据字段，开箱即用。利用复合字段（地址、全名）高效收集结构化数据。

## 快速开始 \{#quick-start}

1. 前往 <CloudLink to="/sign-in-experience/collect-user-profile">Logto 控制台 > 登录体验 > 收集用户资料</CloudLink>。
2. 点击“添加资料字段”，选择一个[内置字段](#basic-user-data-fields)或定义一个[自定义数据](#custom-user-data-fields)（字母数字键）进行创建。
3. 打开字段详情，设置字段类型、标签、描述、必填标志及类型特定设置（长度、范围、格式、选项等）。点击“保存更改”。
4. 返回 登录体验 > 收集用户资料，通过拖拽排序字段，变更会自动生效。
5. 使用 [Logto 实时预览](/customization/live-preview) 或你的测试应用体验用户流程。无论用户通过[标识符（邮箱 / 手机号 / 用户名）](/end-user-flows/sign-up-and-sign-in/sign-up)、[社交登录](/end-user-flows/sign-up-and-sign-in/social-sign-in)还是[企业单点登录 (SSO)](/end-user-flows/enterprise-sso)创建新账户，注册时都会看到“告诉我们你的信息”页面。

## 用户数据类型 \{#user-data-types}

用户数据可在每个[用户资料](/user-management/manage-users)中查看。关于所有用户数据类别的完整概览，请参考[用户数据结构](/user-management/user-data)文档。收集用户资料功能允许你在注册时收集基础用户数据和自定义用户数据。

### 基础用户数据字段 \{#basic-user-data-fields}

Logto 提供的基础用户数据字段直接存储在 `user` 或 `user.profile` 属性中。这些字段自带默认配置值和 i18n 翻译，创建后即可直接使用。你也可以根据具体需求自定义字段类型和参数。

| 字段名称 (13%) | 用户数据键 (44%)                                                                                                                                                                                                                                                                                | 描述                                                                                                                                                       |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 姓名           | <code>user.name</code>                                                                                                                                                                                                                                                                          | 用户的全名，包含所有姓名组成部分（如 "Jane Doe"）。                                                                                                        |
| 全名           | <code>user.profile.givenName</code><br /><code>user.profile.middleName</code><br /><code>user.profile.familyName</code>                                                                                                                                                                         | 用户的完整法定姓名。可根据配置灵活组合 familyName、givenName 和 middleName。                                                                               |
| 昵称           | <code>user.profile.nickname</code>                                                                                                                                                                                                                                                              | 用户的昵称，可能与法定姓名不同。                                                                                                                           |
| 出生日期       | <code>user.profile.birthdate</code>                                                                                                                                                                                                                                                             | 用户的出生日期，指定格式（如 "MM-dd-yyyy"）。                                                                                                              |
| 性别           | <code>user.profile.gender</code>                                                                                                                                                                                                                                                                | 用户自我认同的性别（如 "女"、"男"、“不愿透露”）。                                                                                                          |
| 个人主页       | <code>user.profile.profile</code>                                                                                                                                                                                                                                                               | 用户可读的个人主页 URL（如社交媒体主页）。                                                                                                                 |
| 个人网站       | <code>user.profile.website</code>                                                                                                                                                                                                                                                               | 用户的个人网站或博客 URL。                                                                                                                                 |
| 地址           | <code>user.profile.address.formatted</code><br /><code>user.profile.address.streetAddress</code><br /><code>user.profile.address.region</code><br /><code>user.profile.address.zoneinfo</code><br /><code>user.profile.address.postalCode</code><br /><code>user.profile.address.country</code> | 用户的完整地址（如 "123 Main St, Anytown, USA 12345"）。可选择单行（<code>address.formatted</code>）或多行并配置组成部分（街道、城市、省份、邮编、国家）。 |

### 自定义用户数据字段 \{#custom-user-data-fields}

自定义用户数据字段是用户自定义的字段，用于收集基础字段未涵盖的额外信息，如偏好、兴趣、公司规模或其他业务特定属性。

创建自定义字段时，用户数据键仅支持字母数字（如 `customData.companySize`）。你可以配置[字段类型](#field-types)并自定义其他属性以满足具体需求。

关于用户创建后如何获取和更新自定义用户数据，详见[用户数据结构](/user-management/user-data#custom-data)。

## 字段类型 \{#field-types}

我们可以简单地将字段按复杂度分类：

- 基础类型：文本、数字、日期、复选框（布尔）、下拉选择（单选）、URL、正则表达式。
- 复合类型：全名、地址。

### 通用属性 \{#common-properties}

| 属性 (14%) | 适用范围 (36%)     | 说明                                                           |
| ---------- | ------------------ | -------------------------------------------------------------- |
| 标签       | 全部               | 展示给用户的显示名称。字段有值或聚焦时浮动到顶部。此项为必填。 |
| 占位符     | 全部（复选框除外） | 标签浮动时显示的内联提示（示例文本 / 格式指引）。              |
| 描述       | 全部（复选框除外） | 字段下方的辅助文本，用于更长的说明或补充上下文。               |
| 必填       | 全部（复选框除外） | 启用后为空时禁止提交。禁用时标签后会追加“（选填）”标记。       |

### 类型特定配置 \{#type-specific-configurations}

| 类型 (14%)       | 可配置项 (36%)            | 说明                                                                                                                                                                                                                |
| ---------------- | ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 文本             | 最小 / 最大长度           | 限制用户输入文本的最小和最大长度。                                                                                                                                                                                  |
| 数字             | 最小 / 最大值             | 限制输入值的数值范围。                                                                                                                                                                                              |
| 日期             | 日期格式（预设 / 自定义） | 可选预设格式（MM/dd/yyyy、dd/MM/yyyy、yyyy-MM-dd）。更改占位符会覆盖默认交互。也可使用完全自定义的 <a href="https://date-fns.org/v2.30.0/docs/format" target="_blank" rel="noopener noreferrer">date-fns</a> 格式。 |
| 复选框           | 默认值                    | 选中（True）/ 未选中（False）。                                                                                                                                                                                     |
| 下拉选择（单选） | 选项列表（≥1）            | 每行使用 <code>value:label</code>。如省略 label，则显示 value。                                                                                                                                                     |
| URL              | 无                        | 无                                                                                                                                                                                                                  |
| 正则表达式       | 校验模式                  | 校验规则（无需首尾斜杠）。如 <code>^\d{3}-\d{2}-\d{4}$</code> 用于 SSN。                                                                                                                                            |
| 地址             | 组成部分                  | 单行或多行；子项包括：街道、城市、省份、邮编、国家。                                                                                                                                                                |
| 全名             | 组成部分                  | 子项包括：名、中间名、姓。                                                                                                                                                                                          |

## 校验 \{#validation}

字段校验在注册最后一步用户提交时进行。强制执行以下检查：

- 必填校验（复选框除外）。
- 长度边界（文本）。
- 数值范围（数字）。
- 模式匹配（正则、日期格式、URL、下拉选项）。
- URL 结构（基础语法校验，不校验可达性）。

同样适用于两个复合字段（address 和 fullname）的子项。各组成部分也需满足相同校验标准。

## 本地化 \{#localization}

Logto 为基础用户数据字段（如 `firstName`、`lastName`、`gender`、`birthdate`）提供系统级翻译。这些字段的标签可在 Logto 控制台中配置。
你可以在 [Logto > 登录体验 > 内容](https://cloud.logto.io/to/sign-in-experience/content) 中自定义这些标签的翻译。

自定义字段完全依赖你提供的标签、占位符和描述。自定义字段的本地化可通过 [自定义你的 UI](/customization/bring-your-ui) 或[联系我们](https://logto.io/contact)反馈实现。

## Management API（关键端点） \{#management-api-key-endpoints}

| 方法 (10%) | 端点                                                         | 作用                         |
| ---------- | ------------------------------------------------------------ | ---------------------------- |
| GET        | <code>/api/custom-profile-fields</code>                      | 列出所有字段。               |
| GET        | <code>/api/custom-profile-fields/:name</code>                | 按名称获取单个字段定义。     |
| POST       | <code>/api/custom-profile-fields</code>                      | 创建字段。                   |
| POST       | <code>/api/custom-profile-fields/batch</code>                | 批量创建（每次 ≤20 个）。    |
| PUT        | <code>/api/custom-profile-fields/:name</code>                | 按名称和完整数据集更新字段。 |
| DELETE     | <code>/api/custom-profile-fields/:name</code>                | 删除字段。                   |
| POST       | <code>/api/custom-profile-fields/properties/sie-order</code> | 更新登录体验中的字段顺序。   |

## 最佳实践 \{#best-practices}

- 注册时只收集必要数据，其他信息可在后续完善资料时补充。
- 结构化数据（地址、全名）建议使用复合字段，便于一致性和本地化。
- 对格式敏感的字段（正则、日期、数值范围）提供清晰的示例 / 占位符。

## 常见问题 \{#faqs}

<details>

<summary>

### 资料字段会收集已有用户的数据吗？ \{#do-the-profile-fields-collect-existing-user-data}

</summary>

不会，仅在新用户注册最后一步收集信息。

</details>

<details>

<summary>

### 如果我删除了一个字段，会删除用户数据吗？ \{#what-if-i-delete-a-field-does-it-remove-the-user-data}

</summary>

不会，已有用户数据不会被删除。只是在终端用户注册表单中移除了该字段。

</details>

<details>

<summary>

### 我能否用受控列表收集国家而不是普通文本字段？ \{#can-i-collect-country-with-a-controlled-list-instead-of-a-regular-text-field}

</summary>

可以，你可以将“国家”组件切换为带标准选项的“下拉选择（单选）”字段。

</details>
