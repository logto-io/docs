# 管理组织 (Organization)

本节介绍开发者如何通过 Logto 控制台或 Logto Management API 管理他们的组织 (Organization)，而不是组织 (Organization) 管理员如何在你的应用内自助管理成员。要了解如何开发你的组织 (Organization) 体验，请查阅[本指南](/end-user-flows/organization-experience/organization-management)。

## 通过 Logto 控制台管理 \{#manage-via-logto-console}

### 创建组织 (Organization) \{#create-an-organization}

导航到 <CloudLink to="/organizations">控制台 > 组织 (Organizations)</CloudLink> 并点击“创建组织 (Organization)”按钮。

### 基本设置 \{#basic-settings}

你可以配置组织 (Organization) 的基本属性，如名称、描述、Logo 等。

### 要求组织 (Organization) 成员启用 MFA \{#require-mfa-for-organization-members}

你可以要求组织 (Organization) 的所有成员启用多因素认证 (MFA)。这是一个安全措施，确保所有成员在访问组织 (Organization) 资源时有额外的保护层。

要启用此功能，请进入组织 (Organization) 详情页并打开“多因素认证 (MFA)”开关。

:::note

你需要[至少启用一种 MFA 方式](/end-user-flows/mfa)才能使此功能正常工作。

:::

启用后，未配置 MFA 的成员在设置 MFA 之前将无法交换组织令牌。关于组织令牌何时被交换的更多细节，请参见[授权 (Authorization)](/authorization)。

请注意：

- 此功能仅检查用户是否已配置 MFA，不会强制用户在交换访问令牌时使用 MFA。
- 此功能不限制用户可以使用哪些 MFA 方式。

### 即时 (Just-in-Time) 供应 \{#just-in-time-provisioning}

[即时 (Just-in-Time) 供应](https://auth.wiki/jit-provisioning) 会在用户首次登录应用时自动将其添加到组织 (Organization)。在 Logto 中，这对[企业单点登录 (SSO)](/end-user-flows/enterprise-sso) 和基于邮箱域的供应均支持。当用户满足特定条件（如通过特定企业 IdP 登录或使用特定域名的邮箱登录）时，会自动被加入组织 (Organization)。

你还可以为首次加入组织 (Organization) 的成员设置默认组织角色 (Role)。

关于即时 (Just-in-Time) 供应的更多细节及设置方法，请参考[本节](/organizations/just-in-time-provisioning)。

### 管理组织 (Organization) 成员 \{#manage-organization-members}

用户可以拥有一个或多个角色 (Role)。添加成员到组织 (Organization) 时，你可以为多个用户分配角色 (Role)。如果不分配，新增用户将不会获得任何角色 (Role)。

在 <CloudLink to="/users">控制台 > 用户管理 > 用户详情页</CloudLink>，你可以看到该用户属于哪些组织 (Organization) 以及拥有的组织角色 (Role)。

### 管理组织 (Organization) M2M 应用程序 \{#manage-organization-m2m-applications}

机器对机器 (M2M) 应用程序也可以被添加到组织 (Organization)。你可以像为用户分配角色 (Role) 一样，[为机器对机器 (M2M) 应用程序分配角色 (Role)](/organizations/understand-how-organizations-work#organization-m2m-application)。

在 <CloudLink to="/applications">控制台 > 应用程序 > 应用详情页</CloudLink>，你可以看到该应用关联了哪些组织 (Organization) 以及拥有的组织角色 (Role)。

## 通过 Logto Management API 管理 \{#manage-via-logto-management-api}

你在 Logto 控制台能做的所有操作，也都可以通过 [Management API](/integrate-logto/interact-with-management-api) 完成，包括但不限于：

1. 创建、删除或编辑组织 (Organization)。
2. 管理组织 (Organization) 模板：创建、删除或编辑组织权限 (Permission) 和角色 (Role)。
3. 向组织 (Organization) 添加或移除成员。
4. 分配或移除用户的组织角色 (Role)。
5. 向组织 (Organization) 添加或移除机器对机器 (M2M) 应用程序。
6. 分配或移除机器对机器 (M2M) 应用程序的组织角色 (Role)。

你还可以查阅本节，了解如何使用 Management API 实现更多组织 (Organization) 级体验和管理。[了解更多](/end-user-flows/organization-experience)

完整能力列表请参考我们的 [API 参考文档](https://openapi.logto.io/group/endpoint-organizations)。
