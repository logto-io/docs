# 理解组织 (Organizations) 的工作原理

为了全面了解，理解组织功能中涉及的实体是很重要的，这些是将组织集成到你的应用中的基础。

## 组织 (Organization) \{#organization}

组织 (Organization) 由一组[用户 (身份)](/user-management/user-data)组成。它可以代表可以访问你的应用的团队、商业客户和合作公司。

引入组织 (Organization) 作为一个实体很重要，因为它不仅可以将用户分组，还提供了多租户应用中的租户隔离上下文。

## 组织成员 \{#organization-member}

在 Logto 中，拥有组织 (Organization) 成员资格的用户在该组织的上下文中被称为组织成员（即成员）。

## 组织 M2M 应用程序 \{#organization-m2m-application}

组织 M2M 应用程序指的是与组织相关联的[机器对机器应用程序](/quick-starts/m2m)。类似于[组织成员](#organization-member)，组织应用程序在组织的上下文中具有范围。

一旦你将应用程序与组织关联，该应用程序可以被分配[组织角色](/authorization/organization-template/configure-organization-template#create-organization-role)。当你想允许对组织资源的非交互访问时，这很有用。

:::note

其他类型的应用程序不支持组织关联和组织角色，因为它们是为用户交互设计的。

:::

请参考这个[部分](/organizations/organization-management)来管理组织、成员和应用程序。

## 组织模板 \{#organization-template}

组织模板是为多租户应用中的访问控制需求而设计的。它包括组织角色和组织权限。查看这个[组织模板](/authorization/organization-template)以了解更多关于组织中访问控制设计的信息。
