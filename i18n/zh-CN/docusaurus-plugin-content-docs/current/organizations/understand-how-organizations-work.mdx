# 了解组织如何运作

为了全面了解，重要的是要理解组织功能中涉及的实体，这些是将组织集成到你的应用程序中的基础。

## 组织 (Organization)

组织由一组用户（身份）组成。它可以代表可以访问你的应用程序的团队、企业客户和合作公司。

引入组织作为一个实体很重要，因为它不仅可以对用户进行分组，还为多租户应用中的租户隔离提供了上下文。

## 组织成员 (Organization member)

在 Logto 中，在某个组织上下文中拥有组织成员资格的用户被称为该组织的组织成员（即成员）。

## 组织 M2M 应用程序

组织 M2M 应用程序指与组织关联的机器对机器应用程序。与组织成员类似，组织应用程序在组织的上下文中进行范围限定。

一旦你将应用程序与组织关联，该应用程序就可以被分配组织角色。这在你希望允许对组织资源进行非交互访问时非常有用。

:::note

其他类型的应用程序不支持组织关联和组织角色，因为它们是为用户交互而设计的。

:::

请参阅此 [部分](/organizations/organization-management) 以管理组织、成员和应用程序。

## 组织模板 (Organization template)

组织模板是为多租户应用中的访问控制需求而设计的。它包括组织角色和组织权限。查看此 [组织模板](/authorization/organization-template) 以了解组织中访问控制设计的更多信息。
