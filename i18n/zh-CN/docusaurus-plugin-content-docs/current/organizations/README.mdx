# 组织 (Organizations)

组织 (Organizations) 功能在多租户 SaaS 和 [企业对企业 (B2B) 应用](/introduction/plan-your-architecture/b2b)中特别有效。除了个人用户外，你的客户还可以由团队、组织或整个公司组成。

通过这个基本元素，你可以为[多租户应用](https://auth.wiki/multi-tenancy)构建必备功能，例如：

- 一个可以被多个组织使用的产品。
- 基于[邀请](/end-user-flows/organization-experience/invite-organization-members)或[即时](/organizations/just-in-time-provisioning)的组织成员供应。
- 由分配给组织内成员的角色定义的[访问控制](/authorization/organization-template)。
- 将组织与[企业单点登录 (SSO)](/end-user-flows/enterprise-sso)体验链接。

“组织 (Organizations)”一词也以其他形式使用，如“工作区”、“团队”、“公司”等。在 Logto 中，我们使用“组织 (Organizations)”作为表示多租户概念的通用术语。

## 规划考虑

在使用组织 (Organizations) 时，集成可能涉及不同程度的努力，我们建议你交叉参考这些以定制你的开发。

- **组织体验**：组织体验会是什么样的？组织管理员可以创建自己的组织吗？组织管理员可以邀请成员吗？你是否打算构建一个自定义仪表板，允许管理员自我管理其组织成员的身份相关信息？
- **角色和访问控制**：应用程序是否需要用户在其组织内分配特定角色？
- **通过 Logto 控制台和 Management API 管理组织**：你希望直接在 Logto 控制台和 Management API 中设置哪些配置？
