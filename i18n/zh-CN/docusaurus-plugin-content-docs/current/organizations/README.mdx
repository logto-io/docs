# 组织 (Organizations)

组织 (Organizations) 功能在多租户 SaaS 和 [企业对企业 (B2B) 应用](/introduction/plan-your-architecture/b2b) 中尤其有效。除了个人用户之外，你的客户还可以由团队、组织 (Organizations) 或整个公司组成。

有了这个基础元素，你可以构建 [多租户应用](https://auth.wiki/multi-tenancy) 的必备功能，例如：

- 一个可以被多个组织 (Organizations) 使用的产品。
- 基于[邀请](/end-user-flows/organization-experience/invite-organization-members)或[即时 (JIT)](/organizations/just-in-time-provisioning)的组织 (Organizations) 成员配置。
- 由分配给组织 (Organizations) 成员的角色定义的[访问控制](/authorization/organization-template)。
- 将组织 (Organizations) 与[企业单点登录 (SSO)](/end-user-flows/enterprise-sso)体验关联。

“组织 (Organizations)”这个术语也以其他形式出现，如“工作区”、“团队”、“公司”等。在 Logto 中，我们使用“组织 (Organizations)”作为表示多租户概念的通用术语。

## 规划注意事项 \{#planning-considerations}

在使用组织 (Organizations) 时，集成可能涉及不同程度的工作量，建议你参考以下内容以定制你的开发。

- **组织 (Organizations) 体验**：组织 (Organizations) 的体验会是什么样？组织 (Organizations) 管理员可以创建自己的组织 (Organizations) 吗？组织 (Organizations) 管理员可以邀请成员吗？你是否打算构建一个自定义仪表盘，让管理员自助管理其组织 (Organizations) 成员的身份相关信息？
- **角色与访问控制**：应用是否需要用户在其组织 (Organizations) 内被分配特定角色？
- **通过 Logto 控制台和 Management API 管理组织 (Organizations)：** 你希望在 Logto 控制台和 Management API 中直接设置哪些配置？
