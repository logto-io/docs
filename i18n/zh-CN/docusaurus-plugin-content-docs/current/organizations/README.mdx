# 组织 (Organizations)

组织功能在多租户 SaaS 和企业对企业 (B2B) 应用中尤其有效。除了个人用户之外，你的客户还可以由团队、组织或整个公司组成。

通过这个基本元素，你可以为多租户应用构建必备功能，例如：

- 一个可以被多个组织使用的产品。
- 基于邀请或即时 (just-in-time) 的组织成员提供。
- 由分配给组织内成员的角色定义的访问控制。
- 将组织与企业单点登录 (Enterprise SSO) 体验链接。

术语“组织”也以其他形式使用，例如“工作区”、“团队”、“公司”等。在 Logto 中，我们使用“组织”作为通用术语来表示多租户的概念。

## 规划考虑

使用组织时，集成可能涉及不同程度的努力，我们建议你交叉参考这些以定制你的开发。

- **组织体验 (Organization experience)**：组织体验会是什么样的？组织管理员可以创建自己的组织吗？组织管理员可以邀请成员吗？你是否打算构建一个自定义仪表板，允许管理员自我管理其组织成员的身份相关信息？
- **角色和访问控制 (Roles and access control)**：应用程序是否需要用户在其组织中分配特定角色？
- **通过 Logto Console 和 Management API 管理组织**：你希望直接在 Logto Console 和 Management API 中设置哪些配置？
