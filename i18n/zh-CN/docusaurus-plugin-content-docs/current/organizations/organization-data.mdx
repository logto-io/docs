# 组织 (Organization) 数据结构

在组织 (Organization) 层面，有几个属性，其值可以被创建、更新、读取和删除 (CRUD)。

## Organization ID \{#organization-id}

_organization id_ 是每个组织 (Organization) 的唯一标识符。它对于实现组织 (Organization) 级别的登录体验和检索组织令牌 (Organization tokens) 很有用。

## Name \{#name}

_name_ 支持组织 (Organization) 级别的登录，并可以根据需要集成到组织 (Organization) 级别的产品界面中。

## Description \{#description}

_description_ 字段允许你添加文本以帮助识别和标记组织 (Organization)。

## Organization logos \{#organization-logos}

为了在登录体验中动态显示客户的组织 (Organization) 标志，你可以将组织 (Organization) 标志上传到组织 (Organization) 设置页面。

有关更多详细信息，请参见 [组织特定标志](/customization/match-your-brand/#organization-specific-branding)。

## Custom data \{#custom-data}

_Custom data_ 是一个 JSON 对象，用于存储有关组织 (Organization) 的额外信息。它为开发人员提供了灵活性，以便为开发目的存储额外的组织 (Organization) 详细信息。

## Is MFA required \{#is-mfa-required}

_isMfaRequired_ 表示组织 (Organization) 是否必须启用多因素认证 (MFA)。如果设置为 `true`，则所有成员在登录时必须通过 MFA 才能访问组织 (Organization)。此安全策略设置在组织 (Organization) 层面配置。

有关更多详细信息，请参见 [管理组织 (Organization)](/organizations/organization-management#require-mfa-for-organization-members)。

## Created at \{#created-at}

_createdAt_ 是组织 (Organization) 创建时的带有时区的时间戳。

## Tenant ID \{#tenant-id}

_tenantId_ 标识组织 (Organization) 所属的租户。
