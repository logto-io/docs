## 开始之前 \{#before-you-start}

你的客户端应用需要从 Logto 获取访问令牌 (Access tokens)。如果你还没有完成客户端集成，请查看我们的 [快速开始](/quick-starts)，适用于 React、Vue、Angular 或其他客户端框架，或者参考我们的 [机器对机器指南](/quick-starts/m2m) 以实现服务器到服务器的访问。

本指南聚焦于在你的 {getFrameworkName(props.framework)} 应用中对这些令牌进行**服务端验证**。

### 你将学到什么 \{#what-you-will-learn}

- **JWT 验证**：学习如何验证访问令牌 (Access tokens) 并提取认证 (Authentication) 信息
- **中间件实现**：创建可复用的中间件以保护 API
- **权限模型**：理解并实现不同的授权 (Authorization) 模式：
  - 应用级端点的全局 API 资源
  - 用于租户特定功能控制的组织权限
  - 多租户数据访问的组织级 API 资源
- **RBAC 集成**：在你的 API 端点中强制执行基于角色的权限 (Permissions) 和权限范围 (Scopes)

### 前置条件 \{#prerequisites}

- 已安装 {getFrameworkLanguageName(props.framework)} 的最新稳定版本
- 基本了解 {getFrameworkName(props.framework)} 及 Web API 开发
- 已配置 Logto 应用（如有需要请参见 [快速开始](/quick-starts)）
