# 集成 Logto 认证 (Authentication)

Logto 为 Web、移动和桌面应用程序提供全面的认证 (Authentication) 解决方案，支持服务之间的 [机器对机器 (M2M)](/quick-starts/m2m) 认证 (Authentication)，并可通过 [OpenID Connect(OIDC)](https://auth.wiki/openid-connect) 和 [OAuth 2.0](https://auth.wiki/oauth-2.0) 等标准协议，作为身份提供商 (IdP) 为[第三方应用程序](/integrate-logto/third-party-applications)提供服务。

请选择最适合你需求的方案，开始集成：

## 为你的应用程序添加认证 (Authentication) \{#add-authentication-for-your-applications}

无论你是在构建面向用户的应用程序（如 Web、移动或桌面应用），还是用于服务间通信的机器对机器 (M2M) 应用程序，你都可以通过集成 Logto，快速实现全面的[认证 (Authentication)](/end-user-flows)和[用户管理](/user-management)功能。

## 添加 SAML 应用 \{#add-saml-apps}

## 添加 OIDC/OAuth 应用 \{#add-oidc-oauth-apps}

当你希望让第三方应用程序通过你的 Logto 身份系统进行认证 (Authentication) 时，你可以将 Logto 配置为 [身份提供商 (IdP)](https://auth.wiki/identity-provider)。

Logto 可以通过 OAuth、OIDC 等标准协议作为 IdP 使用，外部服务可以像“使用 Google 登录”一样实现“使用 Logto 登录”，让他们的用户通过你的 Logto 用户系统进行认证 (Authentication)。

通过在 Logto 中创建第三方应用程序，你可以在你的身份系统与外部服务之间建立安全连接。

## 使用 Management API 扩展 Logto 功能 \{#extend-logto-features-with-management-api}

Logto [Management API](https://openapi.logto.io/) 让你可以在应用程序中以编程方式控制和自动化 Logto 的功能。

当你需要扩展 Logto 的能力以增强你的产品（如[管理多租户组织](/organizations)或通过 Logto 自动化工作流）时，你可以利用 Logto Management API 构建高级功能，以满足你的产品需求。
