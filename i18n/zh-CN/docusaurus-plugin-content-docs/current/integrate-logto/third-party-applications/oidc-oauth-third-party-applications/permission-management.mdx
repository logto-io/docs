# 权限管理

第三方应用程序（不属于你的服务）作为身份提供商集成到 Logto (Logto) 中以认证用户。这些应用程序通常来自外部服务提供商，需要仔细的权限管理以保护用户数据。

Logto (Logto) 使你能够控制授予第三方应用程序的特定权限。这包括管理用户资料、API 资源和组织权限。与第一方应用程序不同，请求未经授权权限的第三方应用程序将被拒绝访问。

通过启用特定权限，你可以决定第三方应用程序可以访问哪些用户信息。用户将在用户授权页面上查看并批准这些权限，然后才授予访问权限。

## 管理你的 OIDC 第三方应用程序的权限

前往你的 OIDC 第三方应用程序的 **应用程序详情** 页面，导航到 **权限** 选项卡，然后点击 **添加权限** 按钮以管理第三方应用程序的权限。

第三方应用程序请求始终需要基本用户数据。此外，Logto (Logto) 支持分配组织资源，使其成为 B2B 服务的理想选择。

### 授予用户数据的权限

分配用户级别的权限，包括 [用户资料权限](#user-permissions-user-profile-scopes)（例如，电子邮件、姓名和头像）和 [API 资源权限](#api-resource-permissions-api-resource-scopes)（例如，对特定资源的读取或写入访问）。

请求的资源名称（例如，个人用户数据、API 名称）和特定权限描述（例如，你的电子邮件地址）将出现在用户授权页面上供用户查看。

通过点击 **授权** 按钮，用户同意授予第三方应用程序指定的权限。

### 授予组织数据的权限

分配组织级别的权限，包括 [组织权限](#organization-permissions-organization-scopes) 和 [API 资源权限](#api-resource-permissions-api-resource-scopes)。Logto (Logto) 允许将 API 资源分配给特定的组织角色。

在用户授权页面上，组织数据与用户数据分开显示。在授权流程中，用户必须选择一个特定的组织以授予访问权限。用户可以在确认前在组织之间切换。第三方应用程序将仅获得对所选组织的数据和相关权限的访问。

## 权限类型

### 用户权限（用户资料权限）

这些权限是 OIDC 标准和 Logto (Logto) 的基本用户资料权限，用于访问用户声明。用户声明将相应地在 ID 令牌和 userinfo 端点中返回。

- `profile`：OIDC 标准权限，用于访问用户名和头像。
- `email`：OIDC 标准权限，用于访问用户电子邮件。
- `phone`：OIDC 标准权限，用于访问用户电话号码。
- `custom_data`：Logto (Logto) 用户资料权限，用于访问 [用户自定义数据](/user-management/user-data/#custom-data)。
- `identity`：Logto (Logto) 用户资料权限，用于访问用户关联的 [社交身份](/user-management/user-data/#social-identities) 信息。
- `role`：Logto (Logto) 用户资料权限，用于访问用户 [角色](/authorization/role-based-access-control/#roles) 信息。
- `urn:logto:scope:organizations`：Logto (Logto) 用户组织权限，用于访问用户组织信息。如果启用并由第三方应用程序请求，将在用户授权页面上显示一个组织选择器。这允许用户查看并选择他们希望授予访问权限的组织。有关更多详细信息，请参阅 [组织](/organizations)。
- `urn:logto:scope:organization_roles`：Logto (Logto) 用户组织权限，用于访问用户组织角色信息。

:::warning

在授权请求中请求未启用的用户资料权限将导致错误。

:::

### API 资源权限（API 资源权限）

Logto (Logto) 为 API 资源提供基于角色的访问控制 (RBAC)。API 资源是由你的服务拥有并由 Logto (Logto) 保护的资源。你可以为第三方应用程序分配自定义 API 权限以访问你的 API 资源。请参阅 [RBAC](/authorization/role-based-access-control)、组织模板和 [保护你的 API](/authorization/api-resources/protect-your-api/) 以获取更多详细信息。

你可以在 <CloudLink to="/api-resources">控制台 > API 资源</CloudLink> 下创建和管理你的 API 资源权限。

:::warning

未启用给第三方应用程序的 API 资源权限在发送授权请求时将被忽略。它不会显示在用户授权页面上，也不会被 Logto (Logto) 授予。

:::

### 组织权限（组织权限）

[组织权限](/authorization/organization-template/#organization-permission) 是专为 Logto (Logto) 组织定义的权限。它们用于访问组织信息和资源。

:::note

为了使用 Logto (Logto) 组织权限，你需要启用 `urn:logto:scope:organizations` 用户权限。否则，在发送授权请求时，组织权限将被忽略。

:::

你可以在组织模板设置页面下定义自己的组织权限。请参阅 [配置组织模板](/authorization/organization-template/configure-organization-template/) 以获取更多详细信息。

:::warning

未启用给第三方应用程序的组织权限在发送授权请求时将被忽略。它不会显示在用户授权页面上，也不会被 Logto (Logto) 授予。

:::
