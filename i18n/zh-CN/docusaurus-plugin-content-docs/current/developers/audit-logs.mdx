# 审计日志

Logto 的审计日志让你可以轻松监控用户活动和事件。它为各种用户管理和健康检查业务场景提供了坚实的基础。

## 查看所有日志

导航到 <CloudLink to="/audit-logs">控制台 > 审计日志</CloudLink>。 Logto 将认证 (Authentication) 事件捕获并组织成一个表格。它记录了事件名称、用户、应用程序和时间戳。你可以通过基于事件名称和应用程序名称的过滤来缩小结果范围。点击特定事件将提供更多详细信息。

:::warning
审计日志仅包含在用户认证 (Authentication) 过程中发生的日志，Management API 操作的日志不会被记录。
:::

## 在租户级别捕获用户活动

Logto 的日志提供了全面的细节，确保操作的便捷性和客户的安全性。它们捕获并记录以下信息：

- 事件类型（完整的审计日志事件列表可以在 [这里](https://github.com/logto-io/logto/blob/master/packages/console/src/consts/logs.ts) 找到）
- 涉及的应用程序
- IP 地址
- 涉及的用户
- 日志 ID
- 时间戳
- 用户代理

通过维护这些事件记录，组织可以有效地检测可能的安全风险，并及时处理以防止未经授权的系统访问。

![审计日志成功详情页面](./assets/audit-log-success-details-page.png)

## 在用户级别进行详细分析

管理员可以对与特定用户相关的日志进行详细分析，便于对特定事件进行全面调查。导航过程简单且用户友好。

要访问特定用户的日志，请按照以下步骤操作：

1. 导航到 <CloudLink to="/users">控制台 > 用户管理</CloudLink>。
2. 选择所需的用户并进入详情页面。
3. 点击“用户日志”。结果表将仅显示该特定用户执行和触发的日志事件。

![与特定用户相关的审计日志](./assets/audit-logs-related-to-specific-user.png)
