# Logto OSS

Logto 提供开源 (OSS) 和 [云端](/logto-cloud) 版本。本节将重点介绍 OSS 方案：<Url href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS 是一款免费使用的身份解决方案，为开发者提供完全的自定义能力和对数据的完全控制。Logto OSS 允许你灵活调整代码库以满足特定需求，非常适合那些重视数据主权并希望拥有可随应用成长的定制身份提供商解决方案的用户。

## Logto OSS 支持的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支持 Logto 服务的大多数核心能力，并且会定期更新和维护。

部分高级功能目前仅限于 Logto Cloud 版本，包括：

| OSS 版本功能限制                                                             | 描述                                                                                     |
| ---------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| [Logto 控制台：多租户](/logto-cloud/tenant-settings)                         | 在控制台中创建和管理多个 Logto 租户。                                                    |
| [Logto 控制台：协作者邀请](/logto-cloud/tenant-member-management)            | 邀请和管理一个 Logto 租户的多个成员。                                                    |
| Logto 控制台：启用 MFA                                                       | 登录 Logto 控制台时通过多因素认证 (MFA) 增强安全性。                                     |
| [Logto Protected App](/integrate-logto/protected-app)                        | 由 Cloudflare 提供支持的快速无 SDK 认证 (Authentication) 集成。                          |
| [Logto 内置邮件服务](/connectors/email-connectors/built-in-email-service)    | 利用内置且免费的邮件服务进行邮件发送。                                                   |
| [自定义你的 UI](/customization/bring-your-ui)                                | 使用此功能自定义登录体验的 UI 或流程（OSS 可在 GitHub 上 fork 代码直接自定义登录体验）。 |
| [IdP 发起的单点登录 (SSO)](/end-user-flows/enterprise-sso/idp-initiated-sso) | 启用身份提供商 (IdP) 发起的企业单点登录 (SSO)。                                          |
| [SAML 应用](/integrate-logto/saml-app)                                       | 通过 SAML 协议集成应用。OSS 版本最多支持 3 个 SAML 应用。                                |

提示：多租户、成员邀请和 MFA 不适用于你的团队登录开源 Logto 控制台。但你可以在自己的产品中通过 Logto OSS 实现这些功能，让你的终端用户可以使用。
