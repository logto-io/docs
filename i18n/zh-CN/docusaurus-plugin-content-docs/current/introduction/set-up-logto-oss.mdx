# 设置 Logto OSS

本节涵盖基本设置步骤和关键操作，以有效交付你的产品并管理 [Logto 开源服务 (OSS)](https://github.com/logto-io/logto) 的开发工作流程。

## 开始使用 Logto OSS \{#get-started-with-logto-oss}

按照 [入门指南](/logto-oss/get-started-with-oss/) 今天启动 Logto，之后你可以参考完整的 [部署指南](/logto-oss/deployment-and-configuration) 以用于生产环境。

## Logto OSS 支持的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支持 Logto 服务的大多数核心功能，并定期更新和维护。

一些高级功能目前仅限于 Logto Cloud 版本，包括 [Logto 受保护应用](/integrate-logto/protected-app)、[免费电子邮件服务](/connectors/email-connectors/built-in-email-service)、[IdP 发起的单点登录 (SSO)](/end-user-flows/enterprise-sso/idp-initiated-sso) 和 [SAML IdP](/integrate-logto/third-party-applications)。

## 保持 Logto 版本更新 \{#stay-updated-with-logto-releases}

为了让你的 Logto 实例保持最新功能，请务必关注 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 页面，你可以在那里找到所有的发布日志。

阅读 [升级指南](/logto-oss/upgrading-oss-version) 以了解如何在不更改代码或数据库架构的情况下升级 Logto。
