# 设置 Logto OSS

本节涵盖了 [Logto 开源服务 (OSS)](https://github.com/logto-io/logto) 的基本设置步骤和关键操作，帮助你高效交付产品并管理开发流程。

## 开始使用 Logto OSS \{#get-started-with-logto-oss}

按照 [入门指南](/logto-oss/get-started-with-oss/) 立即启动 Logto，后续你可以参考完整的 [部署指南](/logto-oss/deployment-and-configuration) 以用于生产环境。

## Logto OSS 支持的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支持 Logto 服务的大多数核心能力，并会定期更新和维护。

部分高级功能目前仅限于 Logto Cloud 版本，包括：

| OSS 功能限制                                                              | 描述                                                                                     |
| ------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| [Logto 控制台：多租户](/logto-cloud/tenant-settings)                      | 在控制台中创建和管理多个 Logto 租户。                                                    |
| [Logto 控制台：协作者邀请](/logto-cloud/tenant-member-management)         | 邀请并管理一个 Logto 租户的多个成员。                                                    |
| Logto 控制台：启用 MFA                                                    | 登录 Logto 控制台时要求多因素认证 (MFA) 以增强安全性。                                   |
| [Logto Protected App](/integrate-logto/protected-app)                     | 由 Cloudflare 提供支持的快速、无 SDK 认证 (Authentication) 集成。                        |
| [Logto 内置邮件服务](/connectors/email-connectors/built-in-email-service) | 使用内置且免费的邮件服务进行邮件发送。                                                   |
| [自定义你的 UI](/customization/bring-your-ui)                             | 使用此功能自定义登录体验的 UI 或流程（OSS 可在 GitHub 上 fork 代码直接自定义登录体验）。 |
| [IdP 发起的 SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)        | 启用身份提供商 (IdP) 发起的企业单点登录 (SSO)。                                          |
| [SAML 应用](/integrate-logto/saml-app)                                    | 通过 SAML 协议集成应用。OSS 版本最多支持 3 个 SAML 应用。                                |
| [隐藏 Logto 品牌](/customization/match-your-brand#hide-logto-branding)    | 从登录体验中移除 “Powered by Logto” 标识。                                               |

提示：多租户、成员邀请和 MFA 不适用于你的团队登录开源版 Logto 控制台。但你可以在自己的产品中通过 Logto OSS 实现这些功能，使其对你的终端用户可用。

## 关注 Logto 发布动态 \{#stay-updated-with-logto-releases}

为确保你的 Logto 实例拥有最新功能，请关注 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 页面，你可以在此找到所有发布日志。

阅读 [升级指南](/logto-oss/upgrading-oss-version) 了解如何在不更改代码或数据库结构的情况下升级 Logto。
