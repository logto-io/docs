# 设置 Logto Cloud

本节涵盖基本设置步骤和关键操作，以有效交付你的产品并管理你的开发工作流程。

## 创建 Logto 租户

首先，注册一个业务并创建一个 Logto 租户。Logto Cloud 的租户是一个隔离的环境，你可以在其中管理用户身份、应用程序和所有其他 Logto 资源。

你的第一个租户将自动设置为 [**开发**](/logto-cloud/tenant-settings#development) 环境。这非常适合测试和开发目的，并且可以免费使用。

查看租户设置并[了解更多](/logto-cloud)关于 Logto 云服务的信息。

## 邀请协作者

在入职期间，你有机会邀请协作者一起进行开发工作。如果你想做更多或稍后进行，请前往 <CloudLink to="/tenant-settings/members">租户设置 > 成员</CloudLink>。你将看到当前的成员，并可以邀请更多成员。输入电子邮件地址并分配角色（你可以批量进行），然后发送邀请。

## 转移到生产租户

你可能需要花一些时间开发和测试 Logto 的功能和特性，以完成概念验证。

在你的项目完成测试并准备进入下一个阶段后，记得创建一个新的[生产类型租户](/logto-cloud/tenant-settings#production)（需要一个[免费或专业租户](https://logto.io/pricing)）。因为开发租户由于其[限制和约束](/logto-cloud/tenant-settings#development)不适合用于生产。

:::note

确保在创建生产租户时立即输入[自定义域](/logto-cloud/custom-domain)，因为这将影响你的下一步配置。

:::

这种设置允许你的项目同时拥有开发测试和生产环境，帮助你顺利管理开发工作流程。

## 从现有系统迁移

如果你正在处理一个现有项目并需要切换认证 (Authentication) 提供商，Logto 支持从其他平台进行用户迁移。你可以迁移[基本数据](/user-management/user-data#basic-data)、[自定义数据](/user-management/user-data#custom-data)、[社交身份](/user-management/user-data#social-identities)和密码哈希。有关详细信息，请参阅[迁移到 Logto](../user-management/user-migration.mdx)指南。
