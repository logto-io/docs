# 设置 Logto Cloud

本节涵盖基本的设置步骤和关键操作，帮助你高效交付产品并管理开发流程。

## 创建 Logto 租户 \{#create-logto-tenant}

首先，注册一个企业账号并创建一个 Logto 租户。Logto Cloud 的租户是一个隔离的环境，你可以在其中管理用户身份、应用程序以及所有其他 Logto 资源。

你的第一个租户会自动设置为[**开发环境**](/logto-cloud/tenant-settings#development)。这非常适合测试和开发用途，并且免费使用。

查看租户设置并[了解更多](/logto-cloud)关于 Logto 云服务的信息。

## 邀请协作者 \{#invite-collaborators}

在入门过程中，你有机会邀请协作者一起进行开发工作。如果你想邀请更多人或稍后再邀请，请前往 <CloudLink to="/tenant-settings/members">租户设置 > 成员</CloudLink>。你会看到当前成员，并可以邀请更多成员。输入电子邮件地址并分配角色（支持批量操作），然后发送邀请。

## 迁移到生产租户 \{#move-to-production-tenant}

你可能会花一些时间开发和测试 Logto 的能力和功能，以组建一个概念验证项目。

当你的项目完成测试并准备进入下一阶段时，记得创建一个新的[生产类型租户](/logto-cloud/tenant-settings#production)（需要一个[免费或专业租户](https://logto.io/pricing)）。因为开发租户不适用于生产环境，受其[限制和约束](/logto-cloud/tenant-settings#development)。

:::note

在创建生产租户后，请务必尽快填写[自定义域名](/logto-cloud/custom-domain)，因为这会影响你后续的配置。

:::

这种设置让你的项目同时拥有开发测试和生产环境，帮助你顺利管理开发流程。

## 从现有系统迁移 \{#migrate-from-the-existing-system}

如果你正在维护一个现有项目并需要切换认证 (Authentication) 提供商，Logto 支持从其他平台迁移用户。你可以迁移[基础数据](/user-management/user-data#basic-data)、[自定义数据](/user-management/user-data#custom-data)、[社交身份](/user-management/user-data#social-identities)和密码哈希。详情请参阅 [迁移到 Logto](../user-management/user-migration.mdx) 指南。

## 使用 Logto MCP Server 搭配 AI 工具 \{#use-logto-mcp-server-with-ai-tools}

如果你在使用 VS Code with Copilot、Cursor 或 Claude Desktop 等 AI 驱动的开发工具，你可以连接到 [Logto MCP Server](/logto-cloud/logto-mcp-server)，直接通过 AI 助手与 Logto 交互。AI 可以检测你的框架、创建应用程序，并为你生成可用的集成代码。
