## 使用 Patreon 账户登录 \{#sign-in-with-patreon-account}

前往 [Patreon 官网](https://www.patreon.com/)，并使用你的 Patreon 账户登录。如果你还没有账户，可以注册一个新账户。

## 创建并配置 OAuth 应用 \{#create-and-configure-oauth-app}

按照 [创建 Patreon OAuth 应用](https://www.patreon.com/portal/registration/register-clients) 的指南，注册一个新应用。

在 **App Name** 中为你的 OAuth 应用命名，并填写应用的 **App URL**。你可以将 **App Description** 字段留空，并将 **Redirect URIs** 自定义为 `${your_logto_origin}/callback/${connector_id}`。`connector_id` 可以在 Logto 管理控制台连接器详情页顶部栏找到。

:::note
如果你在登录时遇到错误信息 “The redirect_uri MUST match the registered callback URL for this application.”，请尝试将 Patreon OAuth 应用的 Redirect URI 与 Logto 应用的重定向 URL（包括协议）保持一致，以解决该问题。
:::

## 管理 OAuth 应用 \{#managing-oauth-apps}

前往 Patreon 的 [Clients & API Keys 页面](https://www.patreon.com/portal/registration/register-clients)，你可以在这里添加、编辑或删除现有的 OAuth 应用。你还可以在 OAuth 应用详情页找到 `Client ID` 并生成 `Client secrets`。

## 配置你的连接器 (Connector) \{#configure-your-connector}

将上节中 OAuth 应用详情页获得的 _Client ID_ 和 _Client Secret_ 分别填写到 `clientId` 和 `clientSecret` 字段中。

`scope` 是一个以空格分隔的 [权限 (Scopes)](https://docs.patreon.com/#scopes) 列表。如果未提供，scope 默认为 `identity identity[email]`。

### 配置类型 \{#config-types}

| Name         | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
