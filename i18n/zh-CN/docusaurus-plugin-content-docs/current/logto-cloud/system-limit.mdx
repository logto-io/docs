# 系统限制

系统限制保障 Logto 的稳定性和公平性。这些限制与计费无关。

**简而言之**

- **开发租户**：固定上限（硬性限制）
- **专业租户**：起始上限（软性限制）。如有合理需求，我们可以提升上限。
- **企业租户**：可配置，由合同约定

在定价页面，**无限制** 意味着 _在合理使用下没有预设的计费上限_。为保护平台稳定性，我们会设置**初始保护措施**（如实体上限和速率限制），对于合理的工作负载可以提升这些上限。

## 限制类型 \{#types-of-limits}

### 系统资源和实体上限 \{#system-resource-and-entity-caps}

针对对象（如应用程序、角色、组织 (Organizations)、连接器 (Connectors)、Webhook）以及某些基于用量的系统资源（如访问令牌 (Access token) 发放）的最大数量。

### 速率限制 \{#rate-limit}

针对每个租户的请求级保护。

默认值：**每个租户每 10 秒 200 次请求**（令牌桶）。大致为**持续 20 RPS**，允许短时突发至桶容量。

如果你的流量合理且持续超出此限制，我们可以评估是否放宽。

## 达到限制时会发生什么 \{#what-happens-when-you-hit-a-limit}

当触发保护措施时，请求可能会因如下错误而失败：

- **HTTP 429**（速率限制）
- **HTTP 4xx**，带有与限制相关的消息（实体上限或功能保护）

这不会删除数据，只会阻止超出限制的操作。

## 如何申请提升限制 \{#request-a-limit-increase}

如果你遇到限制且用例真实，请联系我们。对于专业版和企业版，我们通常可以快速提升限制。

请提供：

- 租户 ID（或租户域名）
- 你遇到的限制（功能名称 + 你需要的数量）
- （如为 RPS 申请）预期流量：平均 RPS、峰值 RPS、突发模式
- 受影响的接口（如登录、令牌、JWKS、userinfo、Management API）
- 时间需求（你需要提升的时间点）

[联系我们](https://logto.io/contact)
