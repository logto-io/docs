# 系统限制

系统限制定义了每个租户可创建的实体最大数量。它们用于保护平台稳定性，并且与计费无关。定价页面上的某些项目标记为 _无限_ 或 _按需付费_，这意味着没有计费上限，但内部系统上限仍然适用，以确保公平使用。我们可以根据具体情况审核提升请求。

## 限制类型 \{#types-of-limits}

### 系统资源和实体上限 \{#system-resource-and-entity-caps}

如应用程序、角色、组织 (Organizations)、连接器 (Connectors) 以及某些基于用量的系统资源（如令牌总消耗量）的最大数量。这些限制因套餐不同而异。

### 速率限制 \{#rate-limit}

每个租户级别的请求保护。一般规则是**每个租户每 10 秒 200 次请求**。如果你的流量是合法的且持续超出此限制，我们可以根据具体情况评估放宽。

---

如果你的用例合理且遇到限制，请联系我们。这有助于我们调整限制并支持真实场景。

## 租户级别限制保护 \{#tenant-level-rate-limit-protection}

### 开发租户 \{#dev-tenants}

[开发租户](/logto-cloud/tenant-settings#development)为测试提供免费访问。为防止滥用并保持预期明确，开发租户有固定的实体上限。如果你需要更多，可以申请提升或升级到 Pro 以获得完整访问权限。
