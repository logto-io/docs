# 开发租户数据保留政策

Logto Cloud 遵循一个明确的原则：**生产用于持久化，开发用于测试**。

- 在**生产租户**（免费版、专业版和企业版）中，所有对象和实体会在账户整个生命周期内保留，除非你主动删除或有其他请求。
- 在**开发租户**中，目标不同。它们是**安全的测试沙盒**，更注重安全性、性能和环境整洁，而不是长期存储数据。

为了贯彻这一原则，Logto 在开发租户中实施了 90 天的数据保留政策：**开发租户中的用户**会在 90 天后自动删除。

在开发环境中，开发者在集成过程中经常会创建大量测试账户。这些账户会产生大量数据，但很少有长期价值。90 天政策确保 Logto 平台资源被高效利用，保持测试环境的整洁和高性能，防止未使用账户堆积，并清晰地区分临时测试数据与长期生产数据。

## 理解 90 天政策 \{#understanding-the-90-day-policy}

### 只删除超过 90 天的用户账户 \{#only-user-accounts-older-than-90-days-are-deleted}

删除用户意味着移除所有个人资料信息、认证 (Authentication) 数据及相关信息。

### 保持不变的内容 \{#what-stays-intact}

你的开发工作和配置不会受到影响。这让你可以轻松将开发和集成设置迁移到未来的生产环境。我们提供了直接[将开发租户转换为生产租户](./tenant-settings#tenant-types-dev-vs-prod)的选项。

## 自动用户删除对你的意义 \{#what-automatic-user-deletion-means-for-you}

- 隐私保护：测试账户在开发期间经常被团队共享或长时间闲置。我们建议每 90 天刷新一次。
- 资源高效：自动删除释放存储和处理能力，保持环境整洁高效。
- 环境清晰分离：开发租户是临时沙盒。长期、持久性数据应存储在生产租户中。

## 开发租户最佳实践 \{#best-practices-for-dev-tenants}

### 开发期间 \{#during-development}

- 将 Logto 开发租户用作你的测试沙盒。
- 记录重要的测试场景，方便随时重现。
- 合理规划开发生命周期，时机成熟时迁移到生产环境。

### 长期项目 \{#for-long-term-projects}

- 按需创建新的测试用户或账户，而不是保留旧的。
- 如果需要带有持久数据的预发布环境，请使用生产租户。

## 常见问题 \{#faqs}

<details>
<summary>

### 我可以延长 90 天的保留期吗？ \{#can-i-extend-the-90-day-retention-period}

</summary>

不可以。90 天政策对所有开发租户都是固定的，以保持一致性和安全性。如果你有特殊需求，请联系我们获取指导。

</details>

<details>
<summary>

### 这会影响生产租户吗？ \{#does-this-affect-production-tenants}

</summary>

不会。生产租户中的所有数据，包括用户和配置（适用于免费版、专业版和企业版）都不会受到影响。

</details>

<details>
<summary>

### 对于企业客户，私有实例下的开发租户会被删除吗？ \{#for-enterprise-customers-will-dev-tenants-under-private-instances-be-deleted}

</summary>

不会。私有实例中的开发租户不受 90 天删除政策影响。

</details>

## 总结 \{#summary}

1. 开发租户中的用户会在 90 天后自动清理
2. 你的所有配置、应用和设置都将永久保留
3. 此政策确保安全、性能和干净的测试环境
4. 生产租户对持久数据有不同的保留政策
5. 请围绕这一自动清理周期规划你的开发流程

此保留政策确保你的开发环境始终是安全、高效、可靠的测试场所，同时保护用户隐私并维护系统性能。

如需持久用户数据存储，请考虑升级为生产租户，届时数据保留将遵循你组织的具体要求。
