# 计费和定价

在 Logto 中，我们的专业计划是完全自助的，旨在提供透明性，以便你可以轻松理解和管理你的账单。

## 理解结构

基本结构如下所示：

```math
\text{Your next bill} = \$16\text{ (base price)} + \text{add-on cost (unbilled usage)} + \text{add-on cost (next cycle charge)}
```

在控制台中，你会在租户设置部分找到“你的下一个账单”部分，在这里你可以监控和控制即将到来的费用。

## 项目细分

请记住，只有少数功能属于附加结构。

同时，Logto 还提供按需付费和按比例定价模型，以确保你能够方便地充分利用我们的服务。

|                         | 免费配额 | 附加费用       |
| ----------------------- | -------- | -------------- |
| **令牌 (Tokens)**       | 100 万   | 每百万令牌 $80 |
| **机器对机器应用**      | 1        | 每月每个 $8    |
| **API 资源**            | 3        | 每月每个 $4    |
| **企业单点登录 (SSO)**  | 0        | 每月每个 $48   |
| **多因素认证 (MFA)**    | _N/A_    | 每月 $48       |
| **组织 (Organization)** | _N/A_    | 每月 $48       |
| **租户成员**            | 3        | 每月每个 $8    |

如果我们考虑这些因素，算法是

```math
\begin{align*}
\text{Your next bill} &= \$16 \\
&+ \text{add on unit price} \times \\
\quad & (\text{quantity added} \times \text{pro-rated time remaining upon creation} \\
\quad &- \text{quantity removed} \times \text{pro-rated time remaining upon deletion}) \\
\quad &+ \text{add on unit price} \times \text{quantity remaining}
\end{align*}
```

1. **add on unit price**: 此附加项的单价
2. **quantity added**: 用户在**当前**计费周期中添加的附加项数量
3. **quantity removed**: 用户在**当前**计费周期中移除的附加项数量
4. **pro-rated time remaining upon creation**: 在你创建资源时**当前**周期中剩余的按比例时间
5. **pro-rated time remaining upon deletion**: 在你删除资源时**当前**周期中剩余的按比例时间
6. **quantity remaining**: 在**下一个**周期中保持一致计费所需的数量

让我们通过两个例子来看看它是如何工作的。

### 示例 1：没有变化的一致计费

如果你不进行任何更改，比如添加或移除项目，你的账单将保持一致。

例如，如果你使用的是专业计划并使用了 2 个 SSO 服务，只要你不进行任何更改，账单将保持一致。

```math
\text{Your next bill} = \$ 16 + \$ 48 \times 2 = \$ 112
```

### 示例 2：添加附加项或进行更改将影响你即将到来的账单

这将使你的即将到来的账单在不同情况下略有不同。算法仍然有效。

**情况 1：你订阅了没有任何附加项的专业计划，并且只测试了 10 天的企业 SSO。**

如果你的订阅在每月 5 日续订，并且你在 20 日添加了一个企业 SSO，那么你将在当月使用 15 天。如果你在 30 日删除了企业 SSO，这意味着你只使用了 10 天。

```math
\$16 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$32
```

**情况 2：你订阅了没有任何附加项的专业计划。在添加和移除 API 资源后，你最终使用了 5 个 API 资源**

在这种情况下，你在本计费周期内更新了附加功能，因此更改将在你的下一个账单中显示。更改后的第一个月可能会略高。你的账单将包括 $16 的基本价格、未计费使用的附加费用以及下一个周期的全额费用。

你的计费周期从每月的 1 日开始，你已经使用了前 3 个免费的 API 资源。本月，你进行了两次操作：

1. 在 5 日，你添加了 4 个资源。
2. 在 15 日，你删除了其中的 2 个资源。

因此，你将在即将到来的周期中始终有 2 个 API 资源可用。

```math
\$16 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$50.67
```

### 如何确定附加项的使用时间？

我们使用按需付费模型，你可能会想知道如果你频繁添加或移除资源会发生什么。附加项被认为在其激活的确切持续时间内使用。别担心——我们的系统由 Stripe 提供支持，精确计算你的使用时间，确保根据每个资源的实际使用时间进行准确计费。

如果你取消订阅专业计划，我们将退还未使用的附加费用，并从你的最终账单中扣除该金额。

## 升级或降级

### 免费计划升级到专业计划

升级和降级仅适用于生产租户。以下是你可能需要考虑从免费计划升级的情况：

1. 如果你的 MAU 或令牌使用量超过配额，我们会在适当的时候向你发送电子邮件提醒，通知你需要升级。
2. 如果你想访问仅在高级计划中可用的功能。

### 专业计划升级到企业计划

如果专业计划不能满足你的需求，并且你正在寻找全面的企业级支持，请联系我们讨论定制的企业计划。

### 专业计划降级到免费计划

如果你选择停止使用专业计划，你可以切换到免费计划，但你需要调整你的使用量以适应免费计划的配额。

## 未来定价计划变更

为了为你提供稳定和可预测的计费体验，我们将在价格变更时保留你的当前计划。
