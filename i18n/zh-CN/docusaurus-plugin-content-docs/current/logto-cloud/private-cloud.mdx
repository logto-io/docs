# Logto 私有云 (Private Cloud)

在专属的 Azure 环境中部署 Logto，以实现更强的隔离性、合规性和基础设施控制。

## 概述 \{#overview}

Logto 私有云 (Private Cloud) 是为那些比共享公有云环境需要更多基础设施隔离和控制的客户提供的专属部署选项。

它专为具有更严格安全、合规、采购或部署要求的组织设计，同时保持与 Logto Cloud 相同的核心产品体验。

以下是不同云选项的区别：

- **Logto Cloud**：托管的公有云，最快的生产路径
- **Logto 私有云 (Private Cloud)**：为有更严格数据驻留或合规要求的客户提供的专属部署

## 何时选择私有云 (Private Cloud) \{#when-to-choose-private-cloud}

当你的组织有以下一项或多项需求时，Logto 私有云 (Private Cloud) 通常是合适的选择：

- 更强的环境隔离
- 由 Logto 团队支持和协作的、带有 SLA 承诺的可用性保障
- 共享公有云无法满足的特定合规要求
- 在 Logto Cloud 不支持的特定区域或基础设施中部署
- 需要连接到公有云环境无法访问的内部资源

如果你没有这些需求，[Logto Cloud](https://cloud.logto.io/?sign_up=1) 通常是更好的选择，因为它更易于采用和运维。

### 部署范围 \{#deployment-scope}

Logto 私有云 (Private Cloud) 目前仅支持 **Azure**。

支持的区域取决于 Azure 区域是否提供 Logto 所需的服务。如果你有目标区域，请联系我们，我们会根据你的需求确认可行性。

### 安全与合规 \{#security-and-compliance}

私有云 (Private Cloud) 提供了比共享公有云部署更清晰隔离边界的专属环境。

我们始终以实际需求为导向讨论安全与合规。如果你的团队需要安全审查、合规讨论或针对部署的具体说明，请联系我们，我们可以直接详细沟通。
