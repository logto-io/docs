# Logto MCP Server

Logto MCP Server 是一个远程 [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) 服务器。将该 URL 添加到你的 AI 应用中，使用你的 Logto Cloud 账户登录，并开始与 Logto 对话。

:::tip
Logto MCP Server 基于 Logto 和 [MCP Auth](https://mcp-auth.dev) 构建。如果你想搭建自己的远程 MCP 服务器，请参阅 [构建你自己的远程 MCP 服务器](#build-your-own-remote-mcp-server)。
:::

## 你可以做什么 \{#what-you-can-do}

- **为你的项目添加认证 (Authentication)**：AI 会检测你的框架，创建一个 Logto 应用，并生成可用的集成代码。支持 38+ 种框架，包括 React、Next.js、Vue、Nuxt、SvelteKit、Express、Go、Python、iOS、Android、Flutter 等。
- **咨询 Logto 相关问题**：AI 会检索 Logto 文档，并为你提供带有来源的答案。

更多功能即将上线，包括登录体验配置、基于角色的访问控制 (RBAC) 和权限管理最佳实践指导、多租户组织 B2B 应用搭建等。

## 在你的 AI 应用中设置 \{#set-up-in-your-ai-application}

### Claude Desktop \{#claude-desktop}

1. 从主菜单打开 **设置**，导航到 **连接器**，点击 **添加自定义连接器**。
2. 输入名称 `Logto MCP Server` 和远程 MCP 服务器 URL：`https://mcp.logto.io`。
3. 展开 **高级设置** 并输入 OAuth Client ID：`mcp-client`。点击 **添加**。
4. 在 **连接器** 列表中，找到 **Logto MCP Server** 并点击 **连接**。
5. 你将被重定向到 Logto Cloud 登录页面。登录并授权以完成连接。

### Cursor \{#cursor}

1. 打开命令面板（`Command + Shift + P`），选择 **View: Open MCP Settings**。
2. 在 **已安装的 MCP 服务器** 部分，点击 **Add Custom MCP** 按钮。
3. 在打开的 `mcp.json` 文件中，添加如下配置：

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud 登录页面会自动弹出。使用你的 Logto Cloud 账户登录以完成连接。

### OpenCode \{#opencode}

1. 在你的 `opencode.json` 文件中添加如下配置：

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. 运行以下命令进行认证 (Authentication)：
   ```bash
   opencode mcp auth Logto
   ```
   注意 `Logto` 对应你配置文件中的 MCP 服务器名称。

浏览器窗口会自动打开。使用你的 Logto Cloud 账户登录以授权并连接。

### VS Code \{#vs-code}

1. 打开命令面板（`Command + Shift + P`），选择 **MCP: Add Server...**。
2. 选择 **HTTP** 作为服务器类型。
3. 输入 URL：`https://mcp.logto.io`。你可以选择将其安装到你的工作区，以便更好地隔离和管理。
4. 当 OAuth 流程启动时，输入 App ID：`mcp-client`。App Secret 保持为空，按回车跳过。
5. 你将被重定向到 Logto Cloud 登录页面。使用你的 Logto Cloud 账户登录以完成连接。

### Windsurf \{#windsurf}

1. 打开命令面板（`Command + Shift + P`），选择 **Windsurf: MCP Marketplace**。
2. 在 **已安装的 MCP** 部分，点击右侧的设置图标，创建并打开 `~/.codeium/windsurf/mcp_config.json` 文件。
3. 添加如下 [配置](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json)：

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud 登录页面会自动弹出。使用你的 Logto Cloud 账户登录以完成连接。

### 其他 MCP 客户端 \{#other-mcp-clients}

Logto MCP Server 可与任何支持 OAuth 授权 (Authorization) 的 MCP 客户端配合使用。请使用以下配置：

- **MCP 服务器 URL：** `https://mcp.logto.io`
- **OAuth client ID：** `mcp-client`
- **Resource：** `https://mcp.logto.io/`（如果你的客户端需要）
- **Scope：** `mcp:all`（如果你的客户端需要）

如果你是 AI 应用开发者，希望支持连接 Logto MCP Server，但 Logto MCP Server 还未支持你的应用的重定向 URI，请 [联系我们](https://logto.io/contact) 并提供你的应用详情。

## 构建你自己的远程 MCP 服务器 \{#build-your-own-remote-mcp-server}

Logto MCP Server 基于 Logto 和 [MCP Auth](https://mcp-auth.dev) 构建。如果你想用认证 (Authentication) 搭建自己的远程 MCP 服务器，请参考以下资源：

- [Remote MCP server in action: a new entry point for SaaS products in the AI era](https://blog.logto.io/remote-mcp-server-in-action)
- [加入我们的 Discord](https://discord.gg/fUQmaFyAam)：分享搭建远程 MCP 服务器的经验与最佳实践
- [MCP Auth](https://mcp-auth.dev)：为你的 MCP 服务器添加生产级认证 (Authentication) 所需的一切，无需花费数周阅读规范或手动集成。
