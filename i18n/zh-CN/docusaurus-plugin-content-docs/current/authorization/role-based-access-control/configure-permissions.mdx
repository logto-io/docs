# 配置权限 (Permissions)

## 通过 Logto 控制台配置 API 权限 (Permissions) \{#configure-api-permissions-via-logto-console}

权限 (Permissions) 是在**API 资源**级别设置的。导航到 <CloudLink to="/api-resources">控制台 > API 资源</CloudLink>，点击你想要管理的 API 资源，然后进入“权限 (Permissions)”选项卡。

为权限 (Permissions) 提供名称和详细信息很重要，因为这将使识别和管理权限 (Permissions) 更加容易。

## 通过 Logto Management API 配置 API 权限 (Permissions) \{#configure-api-permissions-via-logto-management-api}

使用 Logto Management API 进行管理。调用相关端点。查看此[参考](https://openapi.logto.io/operation/operation-listresourcescopes)。

| 方法   | 路径                                                                                                                 | 描述                                                      |
| ------ | -------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| GET    | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-listresourcescopes)              | 获取为 API 资源定义的权限 (Scopes)。                      |
| POST   | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-createresourcescope)             | 为 API 资源创建新的权限 (Scope)。                         |
| DELETE | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-deleteresourcescope) | 从给定资源中删除 API 资源权限 (Scope)。                   |
| PATCH  | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-updateresourcescope) | 更新给定资源的 API 资源权限 (Scope)。此方法执行部分更新。 |
