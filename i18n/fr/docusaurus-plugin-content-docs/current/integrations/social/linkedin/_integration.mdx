### Configurer une application LinkedIn \{#setup-a-linkedin-app}

Allez sur [LinkedIn Developers](https://www.linkedin.com/developers) et connectez-vous avec votre compte LinkedIn. Si vous n'avez pas de compte, vous pouvez en créer un.

Ensuite, créez une application.

**Étape 1 :** Remplissez les détails de l'application.

Complétez le formulaire et créez l'application.

**Étape 2 :** Configurez les URLs de rappel.

Allez sur la page des détails de l'application et trouvez l'onglet "Auth", section "OAuth 2.0" et trouvez le champ "Authorized redirect URLs for your app".

Dans notre cas, ce sera `${your_logto_endpoint}/callback/${connector_id}`. par exemple `https://foo.logto.app/callback/${connector_id}`. Le `connector_id` peut être trouvé sur la barre supérieure de la page des détails du connecteur dans la Logto Admin Console.

**Étape 3 :** Ajoutez le produit.

Allez dans l'onglet "Products" et ajoutez le produit "Sign In with LinkedIn using OpenID Connect".

### Configurer votre connecteur \{#configure-your-connector}

Dans la configuration de votre connecteur Logto, remplissez les champs suivants avec les valeurs obtenues dans l'onglet "Auth" de votre application, section "Application credentials" :

- **clientId :** L'ID client de votre application.
- **clientSecret :** Le secret client principal de votre application.

`scope` est une liste d'espaces délimités de portées OIDC. Si non fourni, la portée par défaut est `openid profile email`.

### Types de configuration \{#config-types}

| Nom          | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
