# Configurer la connexion sociale avec GitHub via GitHub App

Intégrez GitHub App pour activer la connexion avec GitHub, la liaison de comptes et l'accès sécurisé aux API GitHub avec des permissions fines et des jetons de rafraîchissement.

<GuideTip />

## GitHub App vs OAuth App \{#github-app-vs-oauth-app}

GitHub propose deux types d'applications pour l'authentification et l'accès à l'API : **GitHub Apps** et **OAuth Apps**. Voici un comparatif pour vous aider à choisir :

| Fonctionnalité                                  | GitHub App                                                                                        | OAuth App                                                         |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| **Permissions (Permissions)**                   | Permissions fines — accès uniquement aux ressources spécifiques                                   | Portées larges — accorde souvent plus d'accès que nécessaire      |
| **Gestion des permissions**                     | Configurée uniquement dans le tableau de bord GitHub ; le champ `Scope` de Logto peut rester vide | Configurée via les portées dans le connecteur Logto               |
| **Jetons de rafraîchissement (Refresh tokens)** | Toujours émis lors du flux OAuth                                                                  | Non pris en charge — les jetons n'expirent jamais sauf révocation |
| **Expiration des jetons**                       | Les jetons d’accès expirent après 8 heures ; les jetons de rafraîchissement sont valides 6 mois   | Les jetons d’accès n’expirent jamais                              |
| **Contrôle utilisateur**                        | Les utilisateurs choisissent les dépôts auxquels l’application peut accéder                       | Accès à tous les dépôts auxquels l’utilisateur a accès            |
| **Limites de taux**                             | Limites de taux plus élevées qui évoluent avec les installations                                  | Limites de taux plus basses (5 000 requêtes/heure/utilisateur)    |
| **Installation**                                | Installée sur des comptes/organisations avec accès granulaire aux dépôts                          | Autorisée par les utilisateurs avec un accès large                |
| **Webhooks**                                    | Prise en charge centralisée des webhooks intégrée                                                 | Doit être configurée séparément pour chaque dépôt                 |
| **Agir de façon indépendante**                  | Peut agir en son propre nom (serveur à serveur)                                                   | Agit toujours au nom d’un utilisateur                             |

**Quand utiliser GitHub App :**

- Vous avez besoin d’un contrôle fin sur les permissions et l’accès aux dépôts
- Vous souhaitez des jetons de rafraîchissement pour un accès API à long terme avec une sécurité renforcée
- Vous avez besoin de limites de taux API plus élevées
- Vous souhaitez effectuer des tâches automatisées indépendamment des sessions utilisateur

**Quand utiliser OAuth App :**

- Intégration de connexion simple avec un accès API minimal
- Vous avez besoin de jetons qui n’expirent pas (pour des intégrations héritées)
- Vous devez accéder à des ressources de niveau entreprise (les GitHub Apps ne prennent pas encore en charge les permissions de niveau entreprise)

:::tip
GitHub recommande les GitHub Apps plutôt que les OAuth Apps pour la plupart des cas d’utilisation en raison de leurs fonctionnalités de sécurité renforcées et de leurs permissions fines. En savoir plus sur les [différences entre GitHub Apps et OAuth Apps](https://docs.github.com/en/apps/oauth-apps/building-oauth-apps/differences-between-github-apps-and-oauth-apps).
:::

## Commencer \{#get-started}

Le connecteur GitHub App permet l’intégration OAuth 2.0 pour que votre application puisse :

- Ajouter l’authentification "Connexion avec GitHub"
- Lier les comptes utilisateurs aux identités GitHub
- Synchroniser les informations de profil utilisateur depuis GitHub
- Accéder aux API GitHub via le stockage sécurisé des jetons dans le Secret Vault de Logto pour des tâches d’automatisation (par exemple, créer des issues GitHub, gérer des dépôts depuis votre application)
- Utiliser des **jetons de rafraîchissement (Refresh tokens)** (toujours émis par les GitHub Apps) pour maintenir un accès API à long terme sans nécessiter une nouvelle authentification utilisateur

Pour configurer ces fonctionnalités d’authentification, créez d’abord un connecteur GitHub dans Logto :

1. Rendez-vous sur <CloudLink to="/connectors/social">Console Logto > Connecteur > Connecteur social</CloudLink>.
2. Cliquez sur **Ajouter un connecteur social**, sélectionnez **GitHub App**, cliquez sur **Suivant** et suivez le tutoriel étape par étape pour finaliser l’intégration.

## Utiliser le connecteur GitHub \{#utilize-the-github-connector}

Une fois que vous avez créé un connecteur GitHub et que vous l’avez relié à GitHub, vous pouvez l’intégrer dans vos parcours utilisateurs. Choisissez les options qui correspondent à vos besoins :

### Activer "Connexion avec GitHub" \{#enable-sign-in-with-github}

1. Dans la Console Logto, allez sur <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Expérience de connexion > Inscription et connexion</CloudLink>.
2. Ajoutez le connecteur GitHub dans la section **Connexion sociale** pour permettre aux utilisateurs de s’authentifier avec GitHub.

En savoir plus sur [l’expérience de connexion sociale](/end-user-flows/sign-up-and-sign-in/social-sign-in).

### Lier ou délier un compte GitHub \{#link-or-unlink-a-github-account}

Utilisez l’Account API pour construire un Centre de compte personnalisé dans votre application permettant aux utilisateurs connectés de lier ou délier leur compte GitHub. [Suivez le tutoriel Account API](/end-user-flows/account-settings/by-account-api#link-a-new-social-connection)

:::tip
Il est possible d’activer le connecteur GitHub uniquement pour la liaison de compte et l’accès à l’API, sans l’activer pour la connexion sociale.
:::

### Accéder aux API GitHub et effectuer des actions \{#access-github-apis-and-perform-actions}

Votre application peut récupérer les jetons GitHub stockés dans le Secret Vault pour appeler les API GitHub et automatiser des tâches backend (par exemple, créer des issues, gérer des dépôts ou automatiser des workflows). Consultez le guide sur la récupération des jetons stockés pour l’accès API.

Comme les GitHub Apps émettent toujours des jetons de rafraîchissement lors du flux OAuth, Logto stocke à la fois les jetons d’accès et les jetons de rafraîchissement. Les jetons d’accès expirent après 8 heures, mais Logto utilise automatiquement le jeton de rafraîchissement (valide 6 mois) pour obtenir de nouveaux jetons d’accès, garantissant ainsi un accès API ininterrompu.

## Gérer l’identité GitHub de l’utilisateur \{#manage-user-s-github-identity}

Après qu’un utilisateur a lié son compte GitHub, les administrateurs peuvent gérer cette connexion dans la Console Logto :

1. Accédez à <CloudLink to="/users">Console Logto > Gestion des utilisateurs</CloudLink> et ouvrez le profil de l’utilisateur.
2. Sous **Connexions sociales**, localisez l’élément GitHub et cliquez sur **Gérer**.
3. Sur cette page, les administrateurs peuvent gérer la connexion GitHub de l’utilisateur, voir toutes les informations de profil accordées et synchronisées depuis son compte GitHub, et vérifier le statut du jeton d’accès et du jeton de rafraîchissement.

:::note
Contrairement aux OAuth Apps qui utilisent des portées, les GitHub Apps utilisent des permissions fines configurées dans le tableau de bord GitHub. Le jeton d’accès utilisateur est limité aux permissions que votre application et l’utilisateur possèdent tous les deux. Logto ne peut pas afficher directement la liste des permissions, mais votre application aura accès selon les permissions configurées dans les paramètres de votre GitHub App.
:::
