# Configurer la connexion sociale avec Apple

Le connecteur officiel Logto pour la connexion sociale Apple.

<GuideTip />

## Commencer \{#get-started}

Si vous ne connaissez pas le concept du connecteur ou si vous ne savez pas comment ajouter ce connecteur à votre expérience de connexion, veuillez consulter le [tutoriel Logto](/connectors/social-connectors).

<Integration />

### Pièges de la configuration de la portée \{#pitfalls-of-configuring-scope}

Si vous avez configuré votre application pour demander les adresses e-mail des utilisateurs après qu'ils se soient déjà connectés avec Apple, vous ne pourrez pas récupérer les adresses e-mail de ces utilisateurs existants, même s'ils se reconnectent en utilisant l'identifiant Apple. Pour résoudre ce problème, vous devez demander à vos utilisateurs de visiter la [page de gestion du compte Apple ID](https://appleid.apple.com/account/manage) et de supprimer votre application de la section "Se connecter avec Apple". Cela peut être fait en sélectionnant "Arrêter d'utiliser Apple Sign In" sur la page de détails de votre application.

Par exemple, si votre application demande à la fois l'e-mail et le nom des utilisateurs (portée `email name`), l'écran de consentement que les nouveaux utilisateurs voient lors de leur première connexion devrait ressembler à ceci :

![Écran de consentement Se connecter avec Apple](./assets/sign-in-with-apple-consent-screen.png)

Voir la discussion des développeurs [ici](https://forums.developer.apple.com/forums/thread/132223).

## Tester le connecteur Apple \{#test-apple-connector}

C'est tout. Le connecteur Apple devrait être disponible à la fois dans les applications web et natives. N'oubliez pas de [Activer le connecteur social dans l'expérience de connexion](/connectors/social-connectors/#enable-social-sign-in).
