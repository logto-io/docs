# Connecteur YunPian SMS

Le connecteur officiel Logto pour le service SMS de YunPian.

<GuideTip />

## Commencer \{#get-started}

YunPian est un fournisseur de services de communication proposant divers services, y compris les SMS. Le connecteur YunPian SMS est un plugin fourni par l'équipe Logto pour s'intégrer au service SMS de YunPian, permettant aux utilisateurs finaux Logto de s'inscrire et de se connecter via des codes de vérification SMS.

## Configurer le service SMS dans la console YunPian \{#set-up-sms-service-in-yunpian-console}

### Créer un compte YunPian \{#create-a-yunpian-account}

Visitez le [site web de YunPian](https://www.yunpian.com/) pour enregistrer un compte et compléter la vérification d'identité.

### Obtenir la CLÉ API \{#get-api-key}

1. Connectez-vous à la console YunPian
2. Allez dans "Paramètres du compte" -> "Gestion des sous-comptes"
3. Trouvez et copiez la CLÉ API

### Configurer les modèles SMS \{#configure-sms-templates}

1. Dans la console YunPian, allez dans "SMS domestique" -> "Dépôt de signature"
2. Créez et soumettez une signature, attendez l'approbation de l'opérateur
3. Allez dans "SMS domestique" -> "Dépôt de modèle" et sélectionnez "Code de vérification"
4. Créez un modèle de code de vérification, assurez-vous qu'il inclut la variable `#code#` (vous pouvez également utiliser les "Modèles courants" pour accélérer le processus d'approbation)
5. Attendez l'approbation du modèle
6. Si vous devez envoyer des SMS internationaux, répétez les étapes ci-dessus mais sélectionnez "SMS international" -> "Dépôt de modèle"

## Configurer dans Logto \{#configure-in-logto}

1. Dans la console Logto, allez dans "Connecteurs"
2. Trouvez et cliquez sur "Service SMS YunPian"
3. Remplissez le formulaire de configuration :
   - CLÉ API : La CLÉ API obtenue auprès de YunPian
   - Modèles SMS : Configurez les modèles selon l'utilisation, assurez-vous qu'ils correspondent exactement aux modèles approuvés dans YunPian

## Remarques \{#notes}

1. Le contenu du modèle SMS doit correspondre exactement au modèle approuvé dans YunPian
2. Le placeholder de la variable du code de vérification dans les modèles YunPian est `#code#`, tandis que dans la configuration du connecteur c'est `{{code}}`
3. YunPian ajoute automatiquement la signature par défaut en fonction de la CLÉ API, il n'est pas nécessaire d'inclure la signature dans le contenu du modèle
4. Il est recommandé de tester la configuration avant une utilisation officielle
