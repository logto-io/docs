# Configurer la vérification par SMS avec SMS Aero

Le connecteur officiel Logto pour le service de messages courts SMS Aero.

<GuideTip />

## Enregistrer un compte \{#register-account}

Créez un nouveau compte sur [SMSAero](https://smsaero.ru/). (Passez à l'étape suivante si vous en avez déjà un.)

## Obtenir les identifiants du compte \{#get-account-credentials}

Nous aurons besoin des identifiants API pour faire fonctionner le connecteur. Commençons par
l'[API et SMPP](https://smsaero.ru/cabinet/settings/apikey/).

Copiez la "clé API" ou générez-en une nouvelle.

## Composer le JSON du connecteur \{#compose-the-connector-json}

Remplissez les champs _email_, _apiKey_ et _senderName_ avec votre email, clé API et nom de l'expéditeur.

Vous pouvez remplir le nom de l'expéditeur avec "SMSAero" pour utiliser le nom d'expéditeur par défaut fourni par SMSAero.

Vous pouvez ajouter plusieurs modèles de connecteurs SMS pour différents cas. Voici un exemple d'ajout d'un modèle unique :

- Remplissez le champ `content` avec des contenus de type chaîne arbitraire. N'oubliez pas de laisser l'espace réservé `{{code}}` pour
  le code de vérification aléatoire.
- Remplissez le champ `usageType` avec soit `Register`, `SignIn`, `ForgotPassword`, `Generic` pour différents cas d'utilisation.
  Afin d'activer les flux utilisateur complets, les modèles avec usageType `Register`, `SignIn`, `ForgotPassword` et `Generic` sont
  requis.

### Tester le connecteur SMSAero \{#test-smsaero-connector}

Vous pouvez entrer un numéro de téléphone et cliquer sur "Envoyer" pour voir si les paramètres fonctionnent avant "Enregistrer et Terminer".

C'est tout. N'oubliez pas de <MainSiteUrl href="/connectors/sms-connectors/#enable-phone-number-for-end-user-flows">Activer le connecteur dans l'expérience de connexion</MainSiteUrl>.

### Types de configuration \{#config-types}

| Nom        | Type        |
| ---------- | ----------- |
| email      | string      |
| apiKey     | string      |
| senderName | string      |
| templates  | Templates[] |

| Propriétés du modèle | Type        | Valeurs énumérées                                       |
| -------------------- | ----------- | ------------------------------------------------------- |
| content              | string      | N/A                                                     |
| usageType            | enum string | 'Register' \| 'SignIn' \| 'ForgotPassword' \| 'Generic' |
