# Configurer la vérification par e-mail avec le service d’e-mail Postmark

Connecteur Logto pour le service d’e-mail Postmark.

<GuideTip />

## Commencer \{#get-started}

Postmark est une plateforme d’e-mails pour les e-mails transactionnels et marketing. Nous pouvons utiliser sa fonction d’envoi d’e-mails pour envoyer un _code de vérification_.

## Créer un compte Postmark \{#register-postmark-account}

Créez un nouveau compte sur le [site web de Postmark](https://postmark.com/). Vous pouvez ignorer cette étape si vous avez déjà un compte.

## Configurer votre connecteur \{#configure-your-connector}

Renseignez le champ `serverToken` avec le Server Token que vous trouverez dans les paramètres de votre serveur sur Postmark.

Renseignez le champ `fromEmail` avec l’_adresse d’expéditeur_.

Pour activer tous les parcours utilisateur, des modèles avec usageType `Register`, `SignIn`, `ForgotPassword` et `Generic` sont requis.

Voici un exemple de JSON de modèle de connecteur Postmark.

```jsonc
[
  {
    "usageType": "Register",
    "templateAlias": "logto-register",
  },
  {
    "usageType": "SignIn",
    "templateAlias": "logto-sign-in",
  },
  {
    "usageType": "ForgotPassword",
    "templateAlias": "logto-forgot-password",
  },
  {
    "usageType": "Generic",
    "templateAlias": "logto-generic",
  },
]
```

## Tester le connecteur e-mail Postmark \{#test-postmark-email-connector}

Vous pouvez saisir une adresse e-mail et cliquer sur "Envoyer" pour vérifier si les paramètres fonctionnent avant de cliquer sur "Enregistrer et terminer".

C’est tout. N’oubliez pas d’<MainSiteUrl href="/connectors/email-connectors/#enable-email-address-for-end-user-flows">activer le connecteur dans l’inscription et la connexion</MainSiteUrl>.

## Types de configuration \{#config-types}

| Nom         | Type   |
| ----------- | ------ |
| serverToken | string |
| fromEmail   | string |

| Propriétés du modèle | Type        | Valeurs d’énumération                                |
| -------------------- | ----------- | ---------------------------------------------------- |
| usageType            | enum string | 'Register' / 'SignIn' / 'ForgotPassword' / 'Generic' |
| templateAlias        | string      | N/A                                                  |
