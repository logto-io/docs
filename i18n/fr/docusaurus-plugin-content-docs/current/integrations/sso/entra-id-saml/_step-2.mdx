Pour que l'intégration SAML SSO fonctionne, vous devrez fournir les métadonnées du fournisseur d’identité (IdP) à Logto. Revenons du côté de Logto et naviguons vers l'onglet `Connection` de votre connecteur Azure AD SSO.

Logto propose trois manières différentes de configurer les métadonnées du fournisseur d’identité (IdP). La méthode la plus simple consiste à fournir l'`URL des métadonnées` de l'application Azure AD SSO.

Copiez l'`URL des métadonnées de fédération de l'application` à partir de la section `Certificats SAML` de votre application Azure AD SSO et collez-la dans le champ `Metadata URL` dans Logto.

![Azure AD Metadata URL.webp](./assets/azure_ad_metadata_url.webp)

Logto récupérera les métadonnées à partir de l'URL et configurera automatiquement l'intégration SAML SSO.
