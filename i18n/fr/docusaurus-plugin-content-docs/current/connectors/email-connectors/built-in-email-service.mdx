Logto fournit des services e-mail intégrés pour votre commodité dans les scénarios suivants :

1. Explorer ou tester rapidement l'expérience de connexion par e-mail de Logto.
2. L'utiliser directement pour vos produits en ligne. Il est principalement destiné aux nouvelles startups qui sont à l'aise avec l'utilisation de `logto.email` comme domaine d'expéditeur.

Les caractéristiques du service e-mail Logto :

- **Gratuit à utiliser :** Il est entièrement gratuit sans aucune limite quotidienne d'envoi d'e-mails, ce qui vous permet de réduire vos coûts.
- **Sans effort :** Aucune configuration avec un fournisseur de service e-mail tiers n'est requise. Il suffit de personnaliser les informations de base de votre marque pour le modèle d'e-mail. Si vous ne disposez pas encore de vos propres informations de marque, vous pouvez choisir de commencer à l'utiliser en quelques clics.
- **Livraison assurée :** Basé sur le service e-mail de Logto, vous bénéficiez d'un service stable et d'une livraison fiable des e-mails, garantissant que les utilisateurs peuvent accéder à votre produit.

Cependant, bien que pratique, il existe certaines limitations à prendre en compte :

1. Les e-mails seront envoyés à partir de l'adresse fixe `no-reply@logto.email`.
2. Vous ne pouvez pas ajouter de lien ou tout autre contenu personnalisé dans les e-mails.

En fonction de l'évolution de vos besoins métier, vous pouvez choisir d'utiliser d'autres fournisseurs de services e-mail par la suite. Nous proposons une gamme de [connecteurs de services e-mail prêts à l'emploi](/connectors/email-connectors#popular-email-providers), et prenons également en charge [SMTP](/integrations/smtp), [HTTP](/integrations/http-email) et [WebHook](/developers/webhooks) comme déclencheurs pour l'envoi d'e-mails, vous trouverez donc toujours une solution adaptée.

:::note
Le service e-mail intégré gratuit de Logto est actuellement disponible uniquement pour les utilisateurs [Cloud](https://cloud.logto.io/). Pour les utilisateurs du service open-source, vous avez la flexibilité de configurer votre propre fournisseur de service e-mail pour la connexion par e-mail.
:::

## Étapes de configuration \{#configuration-steps}

Suivez ces étapes pour configurer le service e-mail Logto :

1. Rendez-vous sur <CloudLink to="/connectors/passwordless">Connecteur > Connecteurs e-mail et SMS</CloudLink>.
2. Pour ajouter un nouveau connecteur e-mail, cliquez sur le bouton "**Configurer**" et sélectionnez le connecteur "**Service e-mail Logto**".
3. Une fois le connecteur "Service e-mail Logto" créé avec succès, vous pouvez personnaliser les informations de base de la marque affichées dans les modèles d'e-mails.
4. Après avoir effectué ces modifications, n'oubliez pas d'envoyer un e-mail de test à votre adresse avant d'enregistrer les changements.

Options de personnalisation :

- **E-mail de l'expéditeur :** L'adresse d'expéditeur est définie sur `no-reply@logto.email` et ne peut pas être modifiée.
- **Nom de l'expéditeur :** Définissez le nom de votre marque comme nom d'expéditeur pour garantir la reconnaissance par l'utilisateur.
- **Informations sur l'entreprise :** Affichez le nom de votre entreprise, son adresse ou son code postal pour renforcer la confiance des utilisateurs et répondre aux exigences de conformité. _Notez que les URL ne sont pas autorisées._
- **Logo de l'application :** Téléchargez le logo de votre application afin que la valeur de la marque soit mise en avant dans les e-mails reçus par les utilisateurs.

## Modèles d'e-mails unifiés \{#unified-email-templates}

Le service e-mail Logto utilise des modèles d'e-mails unifiés adaptés à des scénarios d'authentification spécifiques :

| Utilisation              | Scénario                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Register                 | Les utilisateurs créent un compte avec leur e-mail et le valident en saisissant un code de vérification envoyé par Logto à leur adresse e-mail.                                                                                                                                                                                                                                                                                   |
| SignIn                   | Les utilisateurs se connectent avec leur e-mail et valident en saisissant un code de vérification au lieu d'un mot de passe.                                                                                                                                                                                                                                                                                                      |
| ForgotPassword           | Si les utilisateurs oublient leur mot de passe lors de la connexion, ils peuvent choisir de vérifier leur identité à l'aide de l'e-mail déjà vérifié avec Logto.                                                                                                                                                                                                                                                                  |
| Generic                  | Ce modèle peut être utilisé comme option de secours générale pour divers scénarios, y compris les tests de configuration de connecteur, etc.                                                                                                                                                                                                                                                                                      |
| OrganizationInvitation   | Utilisez ce modèle pour envoyer aux utilisateurs un lien d'invitation à rejoindre l'organisation.                                                                                                                                                                                                                                                                                                                                 |
| UserPermissionValidation | Lors de l'utilisation de l'application, certaines opérations à risque élevé ou à niveau de risque relativement élevé peuvent nécessiter une vérification utilisateur supplémentaire, telles que les virements bancaires, la suppression de ressources en cours d'utilisation ou l'annulation d'abonnements. Le modèle `UserPermissionValidation` permet de définir le contenu du code de vérification envoyé dans ces situations. |
| BindNewIdentifier        | Lorsqu'un utilisateur modifie son profil, il peut lier une adresse e-mail à son compte actuel. Dans ce cas, le modèle `BindNewIdentifier` permet de personnaliser le contenu de l'e-mail de vérification.                                                                                                                                                                                                                         |
| MfaVerification          | Lorsque l'authentification multi-facteurs (MFA) par e-mail est activée, ce modèle est utilisé pour envoyer des codes de vérification aux utilisateurs pendant le processus MFA.                                                                                                                                                                                                                                                   |
| BindMfa                  | Lorsque l'authentification multi-facteurs (MFA) par e-mail est activée, ce modèle est utilisé pour configurer le code de vérification par e-mail pour la MFA. Les utilisateurs reçoivent ce code lorsqu'ils lient ou configurent leur adresse e-mail comme facteur MFA pour leur compte.                                                                                                                                          |

### Localisation \{#localization}

Le service e-mail intégré de Logto prend en charge plusieurs langues. Il détecte automatiquement les paramètres d'expérience de connexion de l'utilisateur et envoie les e-mails dans la langue préférée de l'utilisateur.

Lors de l'envoi d'invitations à une organisation via Management API, vous pouvez spécifier le paramètre `locale` dans la charge utile de la requête. Par exemple :

```json
{
  "inviterId": "inviter-user-id",
  "invitee": "invitee-email-address",
  "expiresAt": "epoch-timestamp",
  "organizationId": "organization-id",
  "messagePayload": {
    "link": "https://your-app.com/invite-accept?invitation-id=abcd1234",
    // highlight-start
    "locale": "fr"
    // highlight-end
  }
}
```

## FAQ \{#faqs}

<details>
<summary>

### Est-il possible d'ajouter du CSS ou du HTML personnalisé au service e-mail Logto ? \{#is-it-possible-to-add-custom-css-or-html-to-logto-email-service}

</summary>

Le service e-mail intégré de Logto ne prend pas en charge le CSS ou le HTML personnalisé. Vous ne pouvez modifier que les éléments génériques de la marque. Cette restriction vise à maintenir la stabilité du service e-mail intégré, car tous les locataires partagent la même adresse IP et la même adresse d'expéditeur. Pour plus de détails, veuillez consulter "[Facteurs pour améliorer la délivrabilité des e-mails](https://blog.logto.io/verification-email-delivery#factors-to-improve-email-delivery)".

Pour personnaliser les modèles d'e-mails, nous recommandons d'utiliser un autre connecteur e-mail, tel que AWS Direct Mail, SendGrid, Mailgun, Postmark ou SMTP.

</details>
