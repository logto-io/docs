Logto propose quatre modèles différents pour personnaliser le contenu des SMS, qui sont classés en fonction de leur type d'utilisation : Register, SignIn, ForgotPassword et Generic. Il est fortement recommandé d'utiliser des modèles différents pour chaque cas d'usage, sinon vous pourriez atteindre la limite de débit, ce qui entraînerait une interruption temporaire de votre service.

## Types de modèles SMS et exemples \{#sms-template-types-and-examples}

Voici quelques exemples à titre de référence :

| usageType                | Scénario                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | Exemples de modèles                                                                                  |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| SignIn                   | Les utilisateurs [se connectent en utilisant leur numéro de téléphone](/end-user-flows/sign-up-and-sign-in/sign-in) et vérifient leur identité en saisissant un code de vérification SMS au lieu d'un mot de passe.                                                                                                                                                                                                                                                                                                                                                      | Code de vérification de connexion Logto : `{{code}}`. Expire dans 10 minutes.                        |
| Register                 | Les utilisateurs [créent un compte en utilisant leur numéro de téléphone](/end-user-flows/sign-up-and-sign-in/sign-up) et le vérifient en saisissant un code de vérification envoyé par Logto à leur numéro de téléphone.                                                                                                                                                                                                                                                                                                                                                | Code de vérification d'inscription Logto : `{{code}}`. Expire dans 10 minutes.                       |
| ForgotPassword           | Si les utilisateurs oublient leur mot de passe lors de la connexion, ils peuvent choisir de vérifier leur identité en utilisant d'abord leur numéro de téléphone pour [réinitialiser le mot de passe](/end-user-flows/sign-up-and-sign-in/reset-password).                                                                                                                                                                                                                                                                                                               | Code de vérification de réinitialisation du mot de passe Logto : `{{code}}`. Expire dans 10 minutes. |
| Generic                  | Ce modèle peut être utilisé comme option de secours générale pour divers scénarios, y compris la configuration de connecteur de test, [la vérification ou la liaison du numéro de téléphone après connexion](/end-user-flows/account-settings/by-management-api#email-and-phone-number-verification), etc.                                                                                                                                                                                                                                                               | Code de vérification Logto : `{{code}}`. Expire dans 10 minutes.                                     |
| OrganizationInvitation   | Utilisez ce modèle pour [envoyer aux utilisateurs un lien d'invitation](/end-user-flows/organization-experience/invite-organization-members#configure-your-email-connector) afin de rejoindre l'organisation.                                                                                                                                                                                                                                                                                                                                                            | Code de vérification d'invitation à l'organisation Logto : `{{code}}`. Expire dans 10 minutes.       |
| UserPermissionValidation | Lors de l'utilisation de l'application, certaines opérations à risque élevé ou à niveau de risque relativement élevé peuvent [nécessiter une vérification utilisateur supplémentaire](/end-user-flows/account-settings/by-account-api#verify-by-sending-a-verification-code-to-the-users-email-or-phone), telles que les virements bancaires, la suppression de ressources en cours d'utilisation ou l'annulation d'abonnements. Le modèle `UserPermissionValidation` permet de définir le contenu du SMS de vérification reçu par les utilisateurs dans ces situations. | Code de vérification Logto : `{{code}}`. Expire dans 10 minutes.                                     |
| BindNewIdentifier        | Lorsqu'un utilisateur modifie son profil, il peut [lier un numéro de téléphone à son compte actuel](/end-user-flows/account-settings/by-account-api#manage-phone). Dans ce cas, le modèle `BindNewIdentifier` permet de personnaliser le contenu du SMS de vérification.                                                                                                                                                                                                                                                                                                 | Code de vérification de liaison de compte Logto : `{{code}}`. Expire dans 10 minutes.                |
| MfaVerification          | Lorsque [SMS MFA](/end-user-flows/mfa/sms-mfa) est activé, ce modèle est utilisé pour envoyer des codes de vérification aux utilisateurs lors du processus d'authentification multi-facteurs.                                                                                                                                                                                                                                                                                                                                                                            | Code de vérification en 2 étapes Logto : `{{code}}`. Expire dans 10 minutes.                         |
| BindMfa                  | Lorsque [SMS MFA](/end-user-flows/mfa/sms-mfa) est activé, ce modèle est utilisé pour configurer le code de vérification SMS pour MFA. Les utilisateurs reçoivent ce code lorsqu'ils lient ou configurent leur numéro de téléphone comme facteur MFA pour leur compte.                                                                                                                                                                                                                                                                                                   | Code d'ajout de vérification en 2 étapes Logto : `{{code}}`. Expire dans 10 minutes.                 |

Il est important de comprendre ces paramètres :

- Le code de vérification est valable pendant 10 minutes. Nous ne prenons actuellement pas en charge la personnalisation de la durée d'expiration.
- Logto remplacera l'espace réservé `{{code}}` dans le modèle SMS par un code de vérification. Veuillez donc vous assurer que le modèle contient bien cet espace réservé.

:::note
Certains pays et régions peuvent interdire l'envoi de contenu non approuvé par SMS pour des raisons de conformité. Les modèles SMS doivent être enregistrés et approuvés par le fournisseur SMS avant de pouvoir être utilisés. Dans ce cas, le contenu peut être indexé par ID de modèle vers le modèle correspondant.
:::
