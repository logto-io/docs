# Autorisation

[L'autorisation (Authorization)](https://auth.wiki/authorization) définit quelles actions un utilisateur peut effectuer ou quelles ressources il peut accéder après avoir été authentifié, déterminant ce qu'il peut faire dans votre application.

Le service d'autorisation de Logto utilise un mécanisme **basé sur des jetons**. Bien que Logto fournisse un `jeton d’accès`, votre serveur de ressources est responsable de les valider et de les appliquer dans le cadre du contrôle d'accès.

Il existe deux types de [jetons d’accès](https://auth.wiki/access-token) :

1. **Jetons d’accès** : Utilisés pour accéder aux ressources API utilisateur / système.
2. **Jetons d’organisation** : Jetons d'accès spéciaux pour les ressources d'organisation, utilisés dans une architecture multi-locataire lorsque la fonctionnalité d'organisation est activée.

Logto offre des services d'autorisation pour divers scénarios. Voici plusieurs cas d'utilisation typiques et vous pouvez choisir celui qui convient et correspond à vos besoins avec les fonctionnalités et services de Logto.

| Scénario                                                                                                                                                                             | Comment                                                                                                                                                                                    | Flux d'autorisation                                                                                                                |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| [Protégez simplement vos ressources API, sans RBAC.](/authorization/api-resources/protect-your-api)                                                                                  | Enregistrez votre API dans Logto Console, et Logto accordera simplement des jetons d’accès                                                                                                 | Logto accorde un `jeton d’accès` (sans permissions), et vous devez valider ces jetons pour protéger vos ressources API.            |
| [Protégez vos ressources API avec RBAC](/authorization/role-based-access-control/protect-api-resources-with-rbac)                                                                    | Enregistrez votre API et appliquez RBAC pour un contrôle d'accès flexible et granulaire.                                                                                                   | Logto accorde un `jeton d’accès` avec `permission` incluse pour protéger vos ressources API.                                       |
| [Protégez vos ressources d'organisation en utilisant RBAC au niveau de l'organisation (modèle d'organisation).](/authorization/organization-template/protect-organization-resources) | Traitez chaque organisation comme une ressource, en utilisant RBAC au niveau de l'organisation (modèle d'organisation) pour le contrôle d'accès dans une architecture **multi-locataire**. | Logto accorde un `jeton d’organisation` avec `permission d’organisation` pour protéger les ressources au niveau de l'organisation. |

Vous pouvez également utiliser des outils comme [revendications de jetons personnalisées](/developers/custom-token-claims) pour inclure des revendications supplémentaires dans les jetons d’accès afin de répondre à vos exigences techniques personnalisées. Aussi un cas d'utilisation plus avancé : [Protégez vos ressources API en utilisant RBAC au niveau de l'organisation (modèle d'organisation)](/authorization/organization-template/protect-api-reources)

## Familiarisez-vous avec le contrôle d’accès basé sur les rôles

Le contrôle d’accès basé sur les rôles est utilisé dans toute l'infrastructure Logto, à la fois au niveau du système (contrôle d’accès basé sur les rôles) et au niveau de l'organisation (RBAC d'organisation) :

1. **Sujets (Subjects)** : Il peut s'agir de vrais utilisateurs (ou membres dans le contexte de l'organisation) ou d'entités non humaines, comme des applications machine à machine.
2. **Ressources API** : Les ressources au sein de votre système qui nécessitent une protection.
3. **Rôles** : Représentent des fonctions ou des responsabilités professionnelles.
4. **Permissions** : Définissent quelles actions sont autorisées sur des ressources spécifiques.

<Rbac style={{ maxWidth: '100%' }} />

Gérer ces entités et leurs relations vous permet de répondre de manière flexible à vos besoins en matière de contrôle d'accès. Dans Logto, vous pouvez utiliser RBAC à deux niveaux, niveau utilisateur / systèmes (aucune couche intermédiaire d'organisation impliquée), et organisations. Dans le contexte de l'organisation, il a un design RBAC indépendant que nous appelons "modèle d'organisation".
