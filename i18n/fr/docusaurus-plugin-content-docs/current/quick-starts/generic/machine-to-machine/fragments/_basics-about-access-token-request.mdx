L'application M2M effectue une requête `POST` vers l'endpoint de jeton pour obtenir un jeton d’accès (Access token) en ajoutant les paramètres suivants au format `application/x-www-form-urlencoded` dans le corps de la requête HTTP :

- **grant_type** : Doit être défini sur `client_credentials`
- **resource** : La ressource à laquelle vous souhaitez accéder
- **scope** : La portée (Scope) de la demande d'accès

Vous devez également inclure les identifiants de votre application M2M dans l'en-tête de la requête afin que l'endpoint de jeton puisse authentifier votre application M2M.

Cela se fait en incluant les identifiants de l'application sous la forme [Authentification Basic](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Authorization#basic_authentication) dans l'en-tête `Authorization` de la requête : encodez en base64 `{App ID}:{App Secret}` et définissez-le comme valeur après le préfixe `Basic`.
