# Configurer Logto OSS

Cette section couvre les étapes de configuration de base et les actions clés pour livrer efficacement votre produit et gérer votre flux de travail de développement pour [Logto open-source service (OSS)](https://github.com/logto-io/logto).

## Commencer avec Logto OSS \{#get-started-with-logto-oss}

Suivez le [guide de démarrage](/logto-oss/get-started-with-oss/) pour lancer Logto aujourd'hui, et plus tard, vous pourrez vous référer au [guide de déploiement](/logto-oss/deployment-and-configuration) complet pour une utilisation en production.

## Fonctionnalité prise en charge par Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS prend en charge la plupart des capacités principales du service Logto et est régulièrement mis à jour et maintenu.

Certaines fonctionnalités avancées sont actuellement exclusives à la version Logto Cloud, y compris :

| Limitations des fonctionnalités dans OSS                                            | Description                                                                                                                                        |
| ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Console Logto : plusieurs locataires](/logto-cloud/tenant-settings)                | Créez et gérez plusieurs locataires Logto dans une console.                                                                                        |
| [Console Logto : invitation de collaborateurs](/logto-cloud/tenant-member-management) | Invitez et gérez plusieurs membres d'un locataire Logto.                                                                                           |
| Console Logto : Activer MFA                                                         | Améliorez la sécurité en exigeant une authentification multi-facteurs lors de la connexion à la Console Logto.                                      |
| [Application protégée Logto](/integrate-logto/protected-app)                        | Intégration d'authentification rapide et sans SDK alimentée par Cloudflare.                                                                        |
| [Service de messagerie intégré Logto](/connectors/email-connectors/built-in-email-service) | Utilisez le service de messagerie intégré et gratuit pour la livraison des e-mails.                                                                |
| [Apportez votre UI](/customization/bring-your-ui)                                   | Personnalisez l'interface utilisateur ou les flux de l'expérience de connexion à l'aide de cette fonctionnalité (OSS peut forker le code sur GitHub pour personnaliser directement l'expérience de connexion.) |
| [SSO initié par IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)              | Activez l'authentification unique d'entreprise initiée par le fournisseur d’identité.                                                              |
| [Applications SAML](/integrate-logto/saml-app)                                      | Intégrez des applications via le protocole SAML. La version OSS est limitée à 3 applications SAML.                                                 |

Conseils : La multi-tenancy, les invitations de membres et MFA ne sont pas disponibles pour votre équipe pour se connecter à une console Logto open-source. Cependant, vous pouvez implémenter ces fonctionnalités dans votre propre produit en utilisant Logto OSS, les rendant disponibles pour vos utilisateurs finaux.

## Restez à jour avec les versions de Logto \{#stay-updated-with-logto-releases}

Pour garder votre instance Logto à jour avec les dernières fonctionnalités, assurez-vous de suivre la page [Logto GitHub Releases](https://github.com/logto-io/logto/releases), vous pouvez y trouver tous les journaux de publication.

Lisez le [guide sur la mise à niveau](/logto-oss/upgrading-oss-version) pour apprendre comment mettre à niveau Logto sans changer votre code ou votre schéma de base de données.
