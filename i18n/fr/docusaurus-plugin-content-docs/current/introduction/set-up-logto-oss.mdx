# Configurer Logto OSS

Cette section couvre les étapes de configuration de base et les actions clés pour livrer efficacement votre produit et gérer votre flux de travail de développement pour le service open-source Logto (OSS).

## Commencer avec Logto OSS

Suivez le [guide de démarrage](/logto-oss/get-started-with-oss/) pour lancer Logto aujourd'hui, et plus tard, vous pourrez vous référer au [guide de déploiement](/logto-oss/deployment-and-configuration) complet pour une utilisation en production.

## Fonctionnalité prise en charge par Logto OSS

Logto OSS prend en charge la plupart des capacités principales du service Logto et est régulièrement mis à jour et maintenu.

Certaines fonctionnalités avancées sont actuellement exclusives à la version Logto Cloud, y compris Logto Protected App, le service de messagerie gratuit, l'authentification unique (SSO) initiée par IdP, et SAML IdP.

## Restez à jour avec les versions de Logto

Pour garder votre instance Logto à jour avec les dernières fonctionnalités, assurez-vous de suivre la page [Logto GitHub Releases](https://github.com/logto-io/logto/releases), vous pouvez y trouver tous les journaux de publication.

Lisez le guide sur la mise à jour pour apprendre comment mettre à niveau Logto sans changer votre code ou le schéma de votre base de données.

## Contribuer à Logto OSS

Merci pour votre intérêt à contribuer à Logto ! Voici le [guide de contribution](/logto-oss/contribution).
