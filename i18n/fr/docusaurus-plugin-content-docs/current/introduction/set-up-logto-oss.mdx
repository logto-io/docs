# Configurer Logto OSS

Cette section couvre les étapes de configuration de base et les actions clés pour livrer efficacement votre produit et gérer votre flux de travail de développement pour [Logto open-source service (OSS)](https://github.com/logto-io/logto).

## Commencer avec Logto OSS \{#get-started-with-logto-oss}

Suivez le [guide de démarrage](/logto-oss/get-started-with-oss/) pour lancer Logto aujourd'hui, et plus tard, vous pourrez vous référer au [guide de déploiement](/logto-oss/deployment-and-configuration) complet pour une utilisation en production.

## Fonctionnalités prises en charge par Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS prend en charge la plupart des capacités principales du service Logto et est régulièrement mis à jour et maintenu.

Certaines fonctionnalités avancées sont actuellement exclusives à la version Logto Cloud, y compris [Logto Protected App](/integrate-logto/protected-app), [service de messagerie gratuit](/connectors/email-connectors/built-in-email-service), [SSO initié par IdP](/end-user-flows/enterprise-sso/idp-initiated-sso), et [SAML IdP](/integrate-logto/third-party-applications).

## Restez à jour avec les versions de Logto \{#stay-updated-with-logto-releases}

Pour garder votre instance Logto à jour avec les dernières fonctionnalités, assurez-vous de suivre la page [Logto GitHub Releases](https://github.com/logto-io/logto/releases), vous pouvez y trouver tous les journaux de version.

Lisez le [guide sur la mise à niveau](/logto-oss/upgrading-oss-version) pour apprendre comment mettre à niveau Logto sans modifier votre code ou le schéma de votre base de données.

## Contribuer à Logto OSS \{#contributing-to-logto-oss}

Merci pour votre intérêt à contribuer à Logto ! Voici le [guide de contribution](/logto-oss/contribution).
