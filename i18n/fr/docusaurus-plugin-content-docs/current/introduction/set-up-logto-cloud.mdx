# Configurer Logto Cloud

Cette section couvre les étapes de configuration de base et les actions clés pour livrer efficacement votre produit et gérer votre flux de travail de développement.

## Créer un tenant Logto \{#create-logto-tenant}

Commencez par vous inscrire en tant qu'entreprise et créez un tenant Logto. Un tenant de Logto Cloud est un environnement isolé où vous pouvez gérer les identités des utilisateurs, les applications et toutes les autres ressources Logto.

Votre premier tenant sera automatiquement configuré comme un environnement [**Développement**](/logto-cloud/tenant-settings#development). C'est idéal pour les tests et le développement, et il est gratuit à utiliser.

Consultez les paramètres du tenant et [en savoir plus](/logto-cloud) sur le service cloud Logto.

## Inviter des collaborateurs \{#invite-collaborators}

Lors de l'intégration, vous avez la possibilité d'inviter des collaborateurs pour travailler ensemble sur le développement. Si vous souhaitez en inviter d'autres ou plus tard, allez dans <CloudLink to="/tenant-settings/members">Paramètres du tenant > Membres</CloudLink>. Vous verrez vos membres actuels et pourrez en inviter d'autres. Saisissez les adresses e-mail et attribuez des rôles (vous pouvez le faire en masse), puis envoyez.

## Passer au tenant de production \{#move-to-production-tenant}

Vous pouvez passer un certain temps à développer et tester les capacités et fonctionnalités de Logto pour constituer une preuve de concept.

Une fois que votre projet a terminé les tests et est prêt pour la prochaine étape, n'oubliez pas de créer un nouveau [tenant de type production](/logto-cloud/tenant-settings#production) (nécessitant un [tenant Free ou Pro](https://logto.io/pricing)). Car le tenant de développement n'est pas destiné à un usage en production en raison de ses [limitations et contraintes](/logto-cloud/tenant-settings#development).

:::note

Assurez-vous de saisir le [domaine personnalisé](/logto-cloud/custom-domain) dès que vous créez un tenant de production, car cela aura un impact sur la configuration de l'étape suivante.

:::

Cette configuration permet à votre projet de disposer à la fois d'environnements de test de développement et de production, ce qui vous aide à gérer votre flux de travail de développement en toute fluidité.

## Migrer depuis le système existant \{#migrate-from-the-existing-system}

Si vous travaillez sur un projet existant et devez changer de fournisseur d'authentification, Logto prend en charge la migration des utilisateurs depuis d'autres plateformes. Vous pouvez migrer les [données de base](/user-management/user-data#basic-data), [données personnalisées](/user-management/user-data#custom-data), [identités sociales](/user-management/user-data#social-identities) et les hachages de mots de passe. Pour plus de détails, consultez le guide [Migrer vers Logto](../user-management/user-migration.mdx).

## Utiliser Logto MCP Server avec des outils d’IA \{#use-logto-mcp-server-with-ai-tools}

Si vous utilisez des outils de développement alimentés par l'IA comme VS Code avec Copilot, Cursor ou Claude Desktop, vous pouvez vous connecter à [Logto MCP Server](/logto-cloud/logto-mcp-server) pour interagir directement avec Logto depuis votre assistant IA. L'IA peut détecter votre framework, créer des applications et générer pour vous du code d'intégration fonctionnel.
