# Logto OSS

Logto propose des versions open-source (OSS) et [cloud](/logto-cloud). Cette section se concentre sur la solution OSS : <Url href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS est une solution d'identité gratuite qui offre aux développeurs des capacités complètes de personnalisation et un contrôle total sur leurs données. Logto OSS offre la flexibilité d'adapter la base de code pour répondre à des besoins spécifiques, c'est un choix idéal pour ceux qui privilégient la souveraineté des données et souhaitent une solution de fournisseur d'identité sur mesure pouvant évoluer avec leur application.

## Fonctionnalités prises en charge par Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS prend en charge la plupart des fonctionnalités principales du service Logto et est régulièrement mis à jour et maintenu.

Certaines fonctionnalités avancées sont actuellement exclusives à la version Logto Cloud, notamment :

| Limitations des fonctionnalités dans OSS                                              | Description                                                                                                                                                                                 |
| ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Console Logto : multi-tenants](/logto-cloud/tenant-settings)                         | Créez et gérez plusieurs tenants Logto dans une console.                                                                                                                                    |
| [Console Logto : invitation de collaborateurs](/logto-cloud/tenant-member-management) | Invitez et gérez plusieurs membres d’un tenant Logto.                                                                                                                                       |
| Console Logto : Activer MFA                                                           | Renforcez la sécurité en exigeant l’authentification multi-facteurs lors de la connexion à la Console Logto.                                                                                |
| [Logto Protected App](/integrate-logto/protected-app)                                 | Intégration d’authentification rapide et sans SDK propulsée par Cloudflare.                                                                                                                 |
| [Service e-mail intégré Logto](/connectors/email-connectors/built-in-email-service)   | Utilisez le service e-mail intégré et gratuit pour l’envoi d’e-mails.                                                                                                                       |
| [Apportez votre UI](/customization/bring-your-ui)                                     | Personnalisez l’interface ou les flux de l’expérience de connexion avec cette fonctionnalité (OSS peut forker le code sur GitHub pour personnaliser directement l’expérience de connexion). |
| [SSO initié par IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)                | Activez l’authentification unique d’entreprise initiée par le fournisseur d’identité.                                                                                                       |
| [Applications SAML](/integrate-logto/saml-app)                                        | Intégrez des applications via le protocole SAML. La version OSS est limitée à 3 applications SAML.                                                                                          |
| [Masquer la marque Logto](/customization/match-your-brand#hide-logto-branding)        | Supprimez la mention "Powered by Logto" de l’expérience de connexion.                                                                                                                       |

Conseil : La multi-tenance, les invitations de membres et la MFA ne sont pas disponibles pour que votre équipe se connecte à une console Logto open-source. Cependant, vous pouvez implémenter ces fonctionnalités dans votre propre produit en utilisant Logto OSS, les rendant ainsi disponibles pour vos utilisateurs finaux.
