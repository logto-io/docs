# Installer Logto

## Interactif \{#interactive}

Exécutez la commande ci-dessous pour procéder à une installation interactive dans votre terminal :

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npm init @logto@latest
```

  </TabItem>

</Tabs>

## Options et installation silencieuse \{#options-and-silent-installation}

Vous pouvez également ajouter l’une des options ci-dessous (toutes facultatives) pour passer certaines questions :

| Option                                         | Description                                                                                                                                                                                                                                                                                           |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--db-url`                                     | L’URL Postgres vers la base de données Logto.                                                                                                                                                                                                                                                         |
| `-p, --path`                                   | Chemin vers votre instance Logto, doit être un chemin inexistant.                                                                                                                                                                                                                                     |
| `--ss`                                         | Ignorer l’initialisation de la base de données Logto.                                                                                                                                                                                                                                                 |
| `--disable-admin-pwned-password-check, --dapc` | Initialiser le tenant admin avec la vérification de compromission de mot de passe Have I Been Pwned (HIBP) désactivée, pour les déploiements isolés ou hors ligne. Voir [Initialisation pour les déploiements isolés ou hors ligne](/logto-oss/using-cli#seed-for-air-gapped-or-offline-deployments). |

Exécutez `logto init --help` pour obtenir l’aide complète.

Voici un exemple pour effectuer une installation silencieuse (par exemple, en CI) :

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npx @logto/cli init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>

</Tabs>

Cela effectuera les actions suivantes dans l’ordre :

1. Télécharger et extraire Logto dans `./logto`
2. Tenter d’initialiser et d’injecter les données dans la base de données avec l’URL `postgresql://your-postgres-dsn:5432/logto`
3. Ajouter les connecteurs officiels Logto dans `./logto/packages/core/connectors`
