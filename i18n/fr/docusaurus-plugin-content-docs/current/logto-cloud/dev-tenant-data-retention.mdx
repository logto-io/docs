# Politique de conservation des données des tenants de développement

Logto Cloud suit un principe clair : **la production est destinée à la persistance, le développement est destiné aux tests**.

- Dans les **tenants de production** (Free, Pro et Enterprise), les objets et entités sont conservés pendant toute la durée de vie du compte, sauf si vous les supprimez ou en faites la demande autrement.
- Dans les **tenants de développement**, l'objectif est différent. Ce sont **des bacs à sable sûrs pour les tests**, axés sur la sécurité, la performance et la propreté de l'environnement plutôt que sur le stockage de données à long terme.

Pour respecter ce principe, Logto applique une politique de conservation de 90 jours dans les tenants de développement : **les utilisateurs** dans les tenants de développement sont automatiquement supprimés après 90 jours.

Dans les environnements de développement, les développeurs créent souvent de nombreux comptes de test lors de l'intégration. Ces comptes génèrent de grandes quantités de données mais apportent rarement une valeur à long terme. La politique des 90 jours garantit une utilisation efficace des ressources de la plateforme Logto, maintient les environnements de test propres et performants, empêche l'accumulation de comptes inutilisés et sépare clairement les données de test temporaires des données de production à long terme.

## Comprendre la politique des 90 jours \{#understanding-the-90-day-policy}

### Seuls les comptes utilisateurs de plus de 90 jours sont supprimés \{#only-user-accounts-older-than-90-days-are-deleted}

Supprimer des utilisateurs signifie retirer toutes les informations de profil, données d'authentification et informations associées.

### Ce qui reste intact \{#what-stays-intact}

Votre travail de développement et vos configurations restent intacts. Cela facilite la migration de votre configuration de développement et d'intégration vers les environnements de production à l'avenir. Nous proposons des options pour [convertir directement votre tenant de développement](./tenant-settings#tenant-types-dev-vs-prod) en tenant de production.

## Ce que la suppression automatique des utilisateurs signifie pour vous \{#what-automatic-user-deletion-means-for-you}

- Protection de la vie privée : Les comptes de test sont souvent partagés entre équipes pendant le développement ou laissés inutilisés pendant de longues périodes. Nous recommandons de les renouveler tous les 90 jours.
- Efficacité des ressources : La suppression automatique libère de l'espace de stockage et de la capacité de traitement, maintenant des environnements propres et efficaces.
- Séparation claire des environnements : Les tenants de développement sont des bacs à sable temporaires. Les données persistantes et à long terme appartiennent aux tenants de production.

## Bonnes pratiques pour les tenants de développement \{#best-practices-for-dev-tenants}

### Pendant le développement \{#during-development}

- Utilisez les tenants de développement Logto comme bac à sable pour vos tests.
- Prenez des notes sur les scénarios de test importants afin de pouvoir les recréer facilement.
- Planifiez soigneusement votre cycle de développement et passez en production au moment opportun.

### Pour les projets de longue durée \{#for-long-term-projects}

- Créez de nouveaux utilisateurs ou comptes de test selon les besoins au lieu de conserver les anciens.
- Utilisez des tenants de production si vous avez besoin d'environnements de préproduction avec des données persistantes.

## FAQ \{#faqs}

<details>
<summary>

### Puis-je prolonger la période de conservation de 90 jours ? \{#can-i-extend-the-90-day-retention-period}

</summary>

Non. La politique des 90 jours est fixe pour tous les tenants de développement afin de garantir cohérence et sécurité. Si vous avez des besoins particuliers, veuillez nous contacter pour obtenir des conseils.

</details>

<details>
<summary>

### Cela affecte-t-il les tenants de production ? \{#does-this-affect-production-tenants}

</summary>

Non. Toutes les données des tenants de production, y compris les utilisateurs et les configurations pour les offres Free, Pro et Enterprise, ne sont pas concernées.

</details>

<details>
<summary>

### Pour les clients Enterprise, les tenants de développement sous instances privées seront-ils supprimés ? \{#for-enterprise-customers-will-dev-tenants-under-private-instances-be-deleted}

</summary>

Non. Les tenants de développement dans les instances privées sont exclus de la politique de suppression des 90 jours.

</details>

## Résumé \{#summary}

1. Les utilisateurs dans les tenants de développement sont automatiquement supprimés après 90 jours
2. Toutes vos configurations, applications et paramètres restent permanents
3. Cette politique garantit la sécurité, la performance et des conditions de test propres
4. Les tenants de production ont des politiques de conservation différentes pour les données persistantes
5. Planifiez votre flux de développement en tenant compte de ce cycle de nettoyage automatisé

Cette politique de conservation garantit que votre environnement de développement reste un terrain de test sécurisé, efficace et fiable tout en protégeant la vie privée des utilisateurs et en maintenant la performance du système.

Pour un stockage persistant des données utilisateurs, envisagez de passer à un tenant de production où la conservation des données suit les exigences spécifiques de votre organisation.
