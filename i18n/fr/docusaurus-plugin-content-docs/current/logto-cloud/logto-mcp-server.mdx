# Logto MCP Server

Logto MCP Server est un serveur [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) distant. Ajoutez l’URL à votre application IA, connectez-vous avec votre compte Logto Cloud, et commencez à dialoguer avec Logto.

:::tip
Logto MCP Server est construit avec Logto et [MCP Auth](https://mcp-auth.dev). Si vous souhaitez créer votre propre serveur MCP distant, consultez [Construire votre propre serveur MCP distant](#build-your-own-remote-mcp-server).
:::

## Ce que vous pouvez faire \{#what-you-can-do}

- **Ajoutez l’authentification à votre projet** : L’IA détecte votre framework, crée une application Logto et génère un code d’intégration fonctionnel. Prend en charge plus de 38 frameworks, dont React, Next.js, Vue, Nuxt, SvelteKit, Express, Go, Python, iOS, Android, Flutter, et plus encore.
- **Posez des questions sur Logto** : L’IA recherche dans la documentation Logto et vous fournit des réponses avec les sources.

D’autres fonctionnalités arrivent bientôt, notamment la configuration de l’expérience de connexion, la gestion du RBAC et des permissions avec des conseils de bonnes pratiques, la configuration multi-tenancy pour la création d’applications B2B avec des organisations, et plus encore.

## Configuration dans votre application IA \{#set-up-in-your-ai-application}

### Claude Code \{#claude-code}

<Tabs>
  <TabItem value="claude-mcp-add" label="claude mcp add">

    ```bash
    claude mcp add --transport http --client-id mcp-client --callback-port 8080 logto https://mcp.logto.io
    ```

  </TabItem>
  <TabItem value="claude-mcp-add-json" label="claude mcp add-json">

    ```bash
    claude mcp add-json logto '{"type":"http","url":"https://mcp.logto.io","oauth":{"clientId":"mcp-client","callbackPort":8080}}'
    ```

  </TabItem>
</Tabs>

Vous pouvez remplacer `8080` par n’importe quel port disponible sur votre machine.

Ensuite, exécutez `/mcp` dans Claude Code et suivez le flux de connexion via le navigateur pour vous authentifier avec votre compte Logto Cloud.

### Claude Desktop \{#claude-desktop}

1. Ouvrez **Paramètres** depuis le menu principal, accédez à **Connecteurs**, puis cliquez sur **Ajouter un connecteur personnalisé**.
2. Entrez le nom `Logto MCP Server` et l’URL du serveur MCP distant : `https://mcp.logto.io`.
3. Développez **Paramètres avancés** et saisissez l’ID client OAuth : `mcp-client`. Cliquez sur **Ajouter**.
4. Dans la liste **Connecteurs**, trouvez **Logto MCP Server** et cliquez sur **Connecter**.
5. Vous serez d’abord redirigé vers la page web de Claude. Si vous n’avez pas de session active, connectez-vous d’abord à votre compte Claude.
6. Après la connexion à Claude, vous serez redirigé vers la page de connexion Logto Cloud. Connectez-vous avec votre compte Logto Cloud et autorisez la connexion.

### Cursor \{#cursor}

1. Ouvrez la palette de commandes (`Command + Shift + P`), puis sélectionnez **View: Open MCP Settings**.
2. Dans la section **Installed MCP Servers**, cliquez sur le bouton **Add Custom MCP**.
3. Dans le fichier `mcp.json` qui s’ouvre, ajoutez la configuration suivante :

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Une page de connexion Logto Cloud apparaîtra automatiquement. Connectez-vous avec votre compte Logto Cloud pour établir la connexion.

### OpenCode \{#opencode}

1. Ajoutez la configuration suivante à votre fichier `opencode.json` :

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. Exécutez la commande suivante pour vous authentifier :
   ```bash
   opencode mcp auth Logto
   ```
   Notez que `Logto` correspond au nom du serveur MCP dans votre fichier de configuration.

Une fenêtre de navigateur s’ouvrira. Connectez-vous avec votre compte Logto Cloud pour autoriser et établir la connexion.

### VS Code \{#vs-code}

1. Ouvrez la palette de commandes (`Command + Shift + P`), puis sélectionnez **MCP: Add Server...**.
2. Choisissez **HTTP** comme type de serveur.
3. Entrez l’URL : `https://mcp.logto.io`. Vous pouvez choisir de l’installer dans votre espace de travail pour une meilleure isolation et gestion.
4. Lorsque le flux OAuth démarre, saisissez l’ID de l’application : `mcp-client`. Laissez le secret de l’application vide et appuyez sur Entrée pour passer.
5. Vous serez redirigé vers la page de connexion Logto Cloud. Connectez-vous avec votre compte Logto Cloud pour établir la connexion.

### Windsurf \{#windsurf}

1. Ouvrez la palette de commandes (`Command + Shift + P`) et sélectionnez **Windsurf: MCP Marketplace**.
2. Dans la section **Installed MCPs**, cliquez sur l’icône des paramètres à droite pour créer et ouvrir le fichier `~/.codeium/windsurf/mcp_config.json`.
3. Ajoutez la [configuration](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json) suivante :

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Une page de connexion Logto Cloud apparaîtra automatiquement. Connectez-vous avec votre compte Logto Cloud pour établir la connexion.

### Autres clients MCP \{#other-mcp-clients}

Logto MCP Server fonctionne avec tout client MCP prenant en charge l’autorisation OAuth. Utilisez la configuration suivante :

- **URL du serveur MCP :** `https://mcp.logto.io`
- **ID client OAuth :** `mcp-client`
- **Ressource :** `https://mcp.logto.io/` (si votre client le requiert)
- **Portée :** `mcp:all` (si votre client le requiert)

Si vous êtes développeur d’application IA et souhaitez prendre en charge la connexion à Logto MCP Server, mais que Logto MCP Server ne prend pas encore en charge l’URI de redirection de votre application, veuillez [nous contacter](https://logto.io/contact) avec les détails de votre application.

- [Rejoignez notre Discord](https://discord.gg/fUQmaFyAam) : Partagez vos apprentissages et bonnes pratiques sur la création de serveurs MCP distants
- [MCP Auth](https://mcp-auth.dev) : Tout ce dont vous avez besoin pour ajouter une authentification prête pour la production à votre serveur MCP. Plus besoin de passer des semaines à lire des spécifications ou à tout câbler manuellement.
