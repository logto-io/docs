# Conventions du SDK de la plateforme

Logto (Logto) fournit un service d'Authentification (Authentication) web très puissant et flexible.

Dans l'utilisation pratique des services de Logto (Logto), pour plus de commodité, il est souvent nécessaire pour les développeurs d'intégrer le SDK de Logto (Logto) dans leurs propres applications clientes pour gérer le statut de connexion des utilisateurs, les Permissions, et plus encore.

Vous pouvez trouver les SDK pour tous les langages de programmation / frameworks pris en charge par Logto (Logto) [ici](/quick-starts).

Si vous n'avez pas de chance et ne trouvez pas le SDK que vous souhaitez, voici une convention que vous pouvez suivre pour implémenter le SDK pour le langage de programmation de votre choix, facilitant ainsi l'utilisation des services de Logto (Logto).

Cette convention contient trois parties principales :

- [Stratégie de conception](/developers/sdk-conventions/design-strategy)
- [Convention du SDK principal](/developers/sdk-conventions/core-sdk-convention)
- [Convention du SDK de la plateforme](/developers/sdk-conventions/platform-sdk-convention)
