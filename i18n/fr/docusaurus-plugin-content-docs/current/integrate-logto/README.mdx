# Intégrer l’authentification Logto

Logto propose des solutions complètes d’authentification pour les applications web, mobiles et desktop, prend en charge l’[authentification machine à machine (M2M)](/quick-starts/m2m) entre services, et peut servir de Fournisseur d’identité (IdP) pour des [applications tierces](/integrate-logto/third-party-applications) via des protocoles standards comme [OpenID Connect (OIDC)](https://auth.wiki/openid-connect) et [OAuth 2.0](https://auth.wiki/oauth-2.0).

Commencez votre intégration en sélectionnant la solution qui correspond le mieux à vos besoins :

## Ajouter l’authentification à vos applications \{#add-authentication-for-your-applications}

Que vous développiez des applications destinées aux utilisateurs finaux (comme des applications web, mobiles ou desktop) ou des applications machine à machine (M2M) pour la communication service à service, vous pouvez rapidement mettre en œuvre des fonctionnalités complètes d’[authentification](/end-user-flows) et de [gestion des utilisateurs](/user-management) en intégrant Logto.

Basé sur les standards OIDC, Logto permet une **connexion omni-canal** sur l’ensemble de vos applications. Lorsque vous intégrez plusieurs applications avec Logto, elles partagent le même système d’identité et les mêmes méthodes d’authentification. Cela signifie que les utilisateurs peuvent se connecter une seule fois et accéder sans interruption à toutes vos applications connectées avec une expérience d’authentification unifiée.

## Ajouter des applications SAML \{#add-saml-apps}

Logto (en tant que Fournisseur d’identité) prend en charge l’intégration avec des applications basées sur le protocole SAML (en tant que Fournisseur de service). Grâce à l’intégration d’applications SAML, vous pouvez offrir aux utilisateurs d’entreprise une expérience d’authentification unique (SSO) sécurisée et standardisée.

## Ajouter des applications OIDC/OAuth tierces \{#add-third-party-oidc-oauth-apps}

Lorsque vous souhaitez permettre à des applications tierces de s’authentifier avec votre système d’identité Logto, vous pouvez configurer Logto comme [Fournisseur d’identité (IdP)](https://auth.wiki/identity-provider).

Logto peut être utilisé comme IdP via des protocoles standards comme OAuth et OIDC, et des services externes peuvent implémenter « Se connecter avec Logto » comme « Se connecter avec Google », permettant à leurs utilisateurs de s’authentifier via votre système utilisateur Logto.

Contrairement aux applications propriétaires, les applications tierces ne peuvent demander que des permissions limitées. Lors du flux d’authentification, les utilisateurs verront un **écran de consentement (consent screen)** où ils pourront examiner et approuver les données et permissions demandées par l’application tierce avant d’accorder l’accès.

En créant des applications tierces dans Logto, vous établissez des connexions sécurisées entre votre système d’identité et des services externes tout en donnant à vos utilisateurs le contrôle sur le partage de leurs données.

## Étendre les fonctionnalités de Logto avec Management API \{#extend-logto-features-with-management-api}

La [Management API](https://openapi.logto.io/) de Logto vous permet de contrôler et d’automatiser de façon programmatique les fonctionnalités de Logto dans vos applications.

Lorsque vous souhaitez étendre les capacités de Logto pour améliorer votre produit (comme [gérer des organisations multi-locataires](/organizations), ou automatiser des workflows avec Logto), vous pouvez exploiter la Management API de Logto pour construire des fonctionnalités avancées adaptées à vos besoins.
