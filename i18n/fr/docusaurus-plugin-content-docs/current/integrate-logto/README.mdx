# Intégrer Logto

Logto fournit des solutions d'authentification complètes pour les applications web, mobiles et de bureau, prend en charge l'authentification Machine-to-Machine (M2M) entre services, et peut servir de Fournisseur d’identité (IdP) pour des applications tierces via des protocoles standard comme OIDC et OAuth 2.0.

Commencez votre intégration en sélectionnant la solution qui correspond le mieux à vos besoins :

## Ajoutez l’authentification à votre application

Que vous construisiez des applications destinées aux utilisateurs (comme des applications web, mobiles ou de bureau) ou des applications machine-to-machine (M2M) pour la communication de service à service, vous pouvez rapidement mettre en œuvre des fonctionnalités complètes d'authentification et de gestion des utilisateurs en intégrant Logto.

## Utiliser Logto comme Fournisseur d’identité

Lorsque vous souhaitez permettre à des applications tierces de s'authentifier avec votre système d'identité Logto, vous pouvez configurer Logto comme Fournisseur d’identité (IdP).

Logto peut être utilisé comme un IdP via des protocoles standard comme OAuth, OIDC, ou SAML (bientôt disponible), et les services externes peuvent implémenter "Se connecter avec Logto" tout comme "Se connecter avec Google", permettant à leurs utilisateurs de s'authentifier en utilisant votre système utilisateur Logto.

En créant des applications tierces dans Logto, vous établissez des connexions sécurisées entre votre système d'identité et les services externes.

## Étendre les fonctionnalités de Logto avec Management API

Logto Management API vous permet de contrôler et d'automatiser de manière programmatique les fonctionnalités de Logto dans vos applications.

Lorsque vous avez besoin d'étendre les capacités de Logto pour améliorer votre produit (comme gérer des organisations multi-locataires, ou automatiser des flux de travail avec Logto), vous pouvez exploiter Logto Management API pour construire des fonctionnalités avancées adaptées aux besoins de votre produit.
