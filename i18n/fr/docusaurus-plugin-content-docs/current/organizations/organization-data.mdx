# Structure des données de l'organisation

Au niveau de l'organisation, il existe plusieurs attributs dont les valeurs peuvent être créées, mises à jour, lues et supprimées (CRUD).

## ID de l'organisation \{#organization-id}

L'_ID de l'organisation_ est un identifiant unique pour chaque organisation. Il est utile pour mettre en œuvre des expériences de connexion au niveau de l'organisation et récupérer des jetons d’organisation.

## Nom \{#name}

Le _nom_ prend en charge la connexion au niveau de l'organisation et peut être intégré dans les interfaces de produits au niveau de l'organisation selon les besoins.

## Description \{#description}

Le champ _description_ vous permet d'ajouter du texte pour aider à identifier et étiqueter l'organisation.

## Logos de l'organisation \{#organization-logos}

Pour afficher dynamiquement le logo de l'organisation de votre client dans l'expérience de connexion, vous pouvez télécharger les logos de l'organisation sur la page des paramètres de l'organisation.

Voir [logos spécifiques à l'organisation](/customization/match-your-brand/#organization-specific-branding) pour plus de détails.

## Données personnalisées \{#custom-data}

Les _données personnalisées_ sont un objet JSON utilisé pour stocker des informations supplémentaires sur l'organisation. Cela donne aux développeurs la flexibilité de stocker des détails supplémentaires sur l'organisation à des fins de développement.

## MFA est-il requis \{#is-mfa-required}

_isMfaRequired_ indique si l'authentification multi-facteurs (MFA) est obligatoire pour l'organisation. Si elle est définie sur `true`, tous les membres doivent passer par la MFA lors de la connexion pour accéder à l'organisation. Ce paramètre de politique de sécurité est configuré au niveau de l'organisation.

Voir [Gérer l'organisation](/organizations/organization-management#require-mfa-for-organization-members) pour plus de détails.

## Créé à \{#created-at}

_createdAt_ est l'horodatage avec le fuseau horaire lorsque l'organisation a été créée.

## ID du locataire \{#tenant-id}

_tenantId_ identifie le locataire auquel l'organisation appartient.
