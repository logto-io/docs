# Organisations

La fonctionnalité d'organisations est particulièrement efficace dans les applications SaaS multi-locataires et business-to-business (B2B). En plus des utilisateurs individuels, vos clients peuvent également être composés d'équipes, d'organisations ou d'entreprises entières.

Avec cet élément fondamental, vous pouvez construire les fonctionnalités indispensables pour les applications multi-locataires, telles que :

- Un produit qui peut être utilisé par plusieurs organisations.
- Provisionnement des membres de l'organisation sur invitation ou sur une base juste-à-temps.
- Contrôles d'accès définis par les rôles assignés aux membres au sein d'une organisation.
- Lier les organisations avec l' expérience SSO d'entreprise (authentification unique).

Le terme "organisation" est également utilisé sous d'autres formes, telles que "espace de travail", "équipe", "entreprise", etc. Dans Logto, nous utilisons "organisation" comme terme générique pour représenter le concept de multi-locataires.

## Considérations de planification

Lors de l'utilisation des organisations, l'intégration peut impliquer différents niveaux d'effort, nous vous suggérons de les croiser pour personnaliser votre développement.

- **Expérience d'organisation** : Quelle serait l' expérience d'organisation ? Un administrateur d'organisation peut-il créer son organisation ? Les administrateurs d'organisation peuvent-ils inviter des membres ? Avez-vous l'intention de créer un tableau de bord personnalisé qui permet aux administrateurs de gérer eux-mêmes les informations d'identité des membres de leur organisation ?
- **Rôles et contrôle d'accès** : L'application a-t-elle besoin que les utilisateurs aient des rôles spécifiques assignés au sein de leurs organisations ?
- **Gérer les organisations via Logto Console et Management API :** Quelle configuration souhaitez-vous configurer directement dans Logto Console et Management API ?
