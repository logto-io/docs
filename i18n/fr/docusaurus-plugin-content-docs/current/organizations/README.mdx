# Organisations

La fonctionnalité des organisations est particulièrement efficace dans les applications SaaS multi-locataires et [applications business-to-business (B2B)](/introduction/plan-your-architecture/b2b). En plus des utilisateurs individuels, vos clients peuvent également être constitués d'équipes, d'organisations ou d'entreprises entières.

Avec cet élément fondamental, vous pouvez construire les fonctionnalités indispensables pour les [applications multi-locataires](https://auth.wiki/multi-tenancy), telles que :

- Un produit qui peut être utilisé par plusieurs organisations.
- Provisionnement des membres de l'organisation sur une base [d'invitation](/end-user-flows/organization-experience/invite-organization-members) ou [just-in-time](/organizations/just-in-time-provisioning).
- [Contrôles d'accès](/authorization/organization-template) définis par les rôles attribués aux membres au sein d'une organisation.
- Lier les organisations avec l'expérience [SSO d’entreprise (authentification unique)](/end-user-flows/enterprise-sso).

Le terme "organisation" est également utilisé sous d'autres formes, telles que "espace de travail", "équipe", "entreprise", etc. Dans Logto, nous utilisons "organisation" comme terme générique pour représenter le concept de multi-location.

## Considérations de planification

Lors de l'utilisation des organisations, l'intégration peut impliquer différents niveaux d'effort, nous vous suggérons de les croiser pour personnaliser votre développement.

- **Expérience de l'organisation** : Quelle serait l'expérience de l'organisation ? Un administrateur d'organisation peut-il créer son organisation ? Les administrateurs d'organisation peuvent-ils inviter des membres ? Avez-vous l'intention de créer un tableau de bord personnalisé permettant aux administrateurs de gérer eux-mêmes les informations liées à l'identité de leurs membres ?
- **Rôles et contrôle d'accès** : L'application a-t-elle besoin que les utilisateurs aient des rôles spécifiques attribués au sein de leurs organisations ?
- **Gérer les organisations via Logto Console et Management API :** Quelle configuration souhaitez-vous mettre en place directement dans Logto Console et Management API ?
