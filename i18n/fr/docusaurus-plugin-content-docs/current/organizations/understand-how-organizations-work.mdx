# Comprendre le fonctionnement des organisations

Pour avoir une vision globale de la compréhension, il est important de comprendre les entités impliquées dans les fonctionnalités d'organisation, car ce sont les bases pour intégrer les organisations dans votre application.

## Organisation \{#organization}

Une organisation (Organization) se compose d'un groupe de [utilisateurs (identités)](/user-management/user-data). Elle peut représenter les équipes, les clients professionnels et les entreprises partenaires qui peuvent accéder à votre application.

L'introduction d'une organisation en tant qu'entité est importante, car elle ne se contente pas de regrouper les utilisateurs, mais fournit également un contexte pour l'isolation des locataires dans les applications multi-locataires.

## Membre d'organisation \{#organization-member}

Dans Logto, un utilisateur qui possède l'appartenance à une organisation est appelé membre d'organisation (organization member, c'est-à-dire membre) dans le contexte de cette organisation.

## Application M2M d'organisation \{#organization-m2m-application}

Une application M2M d'organisation (Organization M2M application) fait référence aux [applications machine à machine](/quick-starts/m2m) qui sont associées à une organisation. À l'instar des [membres d'organisation](#organization-member), les applications d'organisation sont limitées au contexte d'une organisation.

Une fois que vous associez une application à une organisation, l'application peut se voir attribuer des [rôles d'organisation](/authorization/role-based-access-control#configure-organization-roles). Ceci est utile lorsque vous souhaitez permettre un accès non interactif aux ressources d'organisation.

:::note

Les autres types d'applications ne prennent pas en charge l'association à une organisation ni les rôles d'organisation, car ils sont conçus pour l'interaction utilisateur.

:::

Consultez cette [section](/organizations/organization-management) pour gérer les organisations, les membres et les applications.

## Modèle d'organisation \{#organization-template}

Le modèle d'organisation (Organization template) est conçu pour les besoins de contrôle d'accès dans les applications multi-locataires. Il inclut les rôles d'organisation et les permissions d'organisation. Consultez ce [modèle d'organisation](/authorization/organization-template) pour en savoir plus sur la conception du contrôle d'accès dans une organisation.
