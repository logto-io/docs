# Comprendre le fonctionnement des organisations

Pour avoir une vue d'ensemble complète, il est important de comprendre les entités impliquées dans les fonctionnalités d'organisation, et ce sont les éléments fondamentaux pour intégrer les organisations dans votre application.

## Organisation

Une organisation se compose d'un groupe d'[utilisateurs (identités)](/user-management/user-data). Elle peut représenter les équipes, les clients professionnels et les entreprises partenaires qui peuvent accéder à votre application.

L'introduction d'une organisation en tant qu'entité est importante, car elle ne se contente pas de regrouper les utilisateurs, mais elle fournit également un contexte pour l'isolation des locataires dans les applications multi-locataires.

## Membre de l'organisation

Dans Logto, un utilisateur qui a l'adhésion à une organisation est appelé membre de l'organisation (c'est-à-dire membre) dans le contexte de cette organisation.

## Application M2M de l'organisation

L'application M2M de l'organisation fait référence aux [applications machine à machine](/quick-starts/m2m) qui sont associées à une organisation. Similaire aux [membres de l'organisation](#organization-member), les applications d'organisation sont définies dans le contexte d'une organisation.

Une fois que vous associez une application à une organisation, l'application peut se voir attribuer des [rôles d'organisation](/authorization/organization-template/configure-organization-template#create-organization-role). Cela est utile lorsque vous souhaitez permettre un accès sans interaction aux ressources de l'organisation.

:::note

D'autres types d'applications ne prennent pas en charge l'association d'organisation et les rôles d'organisation, car elles sont conçues pour l'interaction utilisateur.

:::

Consultez cette [section](/organizations/organization-management) pour gérer les organisations, les membres et les applications.

## Modèle d'organisation

Le modèle d'organisation est conçu pour les besoins de contrôle d'accès dans les applications multi-locataires. Il inclut les rôles d'organisation et les permissions d'organisation. Consultez ce [modèle d'organisation](/authorization/organization-template) pour en savoir plus sur la conception du contrôle d'accès dans l'organisation.
