# Logs de auditoria

O log de auditoria do Logto permite monitorar facilmente a atividade e os eventos dos usuários. Ele fornece uma base sólida para vários cenários de negócios de gerenciamento de usuários e verificação de integridade.

## Visualizar todos os logs

Navegue até <CloudLink to="/audit-logs">Console > Logs de auditoria</CloudLink>. O Logto captura e organiza eventos de autenticação em uma tabela. Ele rastreia o nome do evento, usuário, aplicativo e carimbo de data/hora. Você pode restringir os resultados filtrando com base no nome do evento e no nome do aplicativo. Clicar em um evento específico fornecerá detalhes adicionais.

:::warning
Os logs de auditoria contêm apenas logs que ocorrem durante o processo de autenticação do usuário, logs de operações da Management API não são registrados.
:::

## Capturar atividade do usuário no nível do locatário

Os logs do Logto oferecem detalhes abrangentes, garantindo facilidade de ação e segurança do cliente. Eles capturam e registram as seguintes informações:

- Tipo de evento (a lista completa de eventos de log de auditoria pode ser encontrada [aqui](https://github.com/logto-io/logto/blob/master/packages/console/src/consts/logs.ts))
- Aplicativo envolvido
- Endereço IP
- Usuário envolvido
- ID do log
- Carimbo de data/hora
- User-agent

Ao manter esses registros de eventos, as organizações podem detectar efetivamente possíveis riscos de segurança e abordá-los prontamente para evitar acesso não autorizado ao sistema.

![Página de detalhes de sucesso do log de auditoria](./assets/audit-log-success-details-page.png)

## Realizar uma análise detalhada no nível do usuário

Os administradores podem realizar uma análise detalhada dos logs associados a usuários específicos, facilitando investigações abrangentes sobre eventos específicos. O processo de navegação é simples e amigável.

Para acessar logs específicos de usuários, siga estas etapas:

1. Navegue até <CloudLink to="/users">Console > Gerenciamento de usuários</CloudLink>.
2. Selecione o usuário desejado e vá para a página de detalhes.
3. Clique em "Logs do usuário". A tabela resultante exibirá exclusivamente eventos de log realizados e acionados por esse usuário específico.

![Logs de auditoria relacionados a um usuário específico](./assets/audit-logs-related-to-specific-user.png)
