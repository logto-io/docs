# Integrar Logto

Logto fornece soluções abrangentes de autenticação para aplicativos web, móveis e desktop, suporta autenticação Máquina para Máquina (M2M) entre serviços e pode servir como um Provedor de Identidade (IdP) para aplicativos de terceiros através de protocolos padrão como OIDC e OAuth 2.0.

Inicie sua integração selecionando a solução que melhor atende às suas necessidades:

## Adicionar autenticação ao seu aplicativo

Seja você desenvolvendo aplicativos voltados para o usuário (como aplicativos web, móveis ou desktop) ou aplicativos máquina para máquina (M2M) para comunicação serviço a serviço, você pode implementar rapidamente recursos abrangentes de autenticação e gerenciamento de usuários integrando o Logto.

## Usar Logto como um Provedor de Identidade

Quando você deseja permitir que aplicativos de terceiros autentiquem com seu sistema de identidade Logto, você pode configurar o Logto como um Provedor de Identidade (IdP).

Logto pode ser usado como um IdP através de protocolos padrão como OAuth, OIDC ou SAML (em breve), e serviços externos podem implementar "Entrar com Logto" assim como "Entrar com Google", permitindo que seus usuários autentiquem usando seu sistema de usuários Logto.

Ao criar aplicativos de terceiros no Logto, você estabelece conexões seguras entre seu sistema de identidade e serviços externos.

## Estender recursos do Logto com Management API

Logto Management API permite que você controle e automatize programaticamente a funcionalidade do Logto em seus aplicativos.

Quando você precisa estender as habilidades do Logto para aprimorar seu produto (como gerenciar organizações multi-tenant ou automatizar fluxos de trabalho com Logto), você pode aproveitar Logto Management API para construir recursos avançados que atendam às necessidades do seu produto.
