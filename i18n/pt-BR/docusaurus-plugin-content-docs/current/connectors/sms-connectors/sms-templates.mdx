O Logto fornece quatro modelos diferentes para personalizar o conteúdo de SMS, que são categorizados com base no tipo de uso: Register, SignIn, ForgotPassword e Generic. É altamente recomendado que você utilize modelos diferentes para cada caso de uso, caso contrário, pode atingir o limite de taxa, levando a uma interrupção temporária do seu serviço.

## Tipos de modelos de SMS e exemplos \{#sms-template-types-and-examples}

Aqui estão alguns exemplos apenas para referência:

| usageType                | Cenário                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            | Exemplos de modelo                                                                         |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------ |
| SignIn                   | Usuários [fazem login usando o número de telefone](/end-user-flows/sign-up-and-sign-in/sign-in) e verificam inserindo o código de verificação recebido por SMS em vez de digitar uma senha.                                                                                                                                                                                                                                                                                                                        | Código de verificação de login Logto: `{{code}}`. Expira em 10 minutos.                    |
| Register                 | Usuários [criam uma conta usando o número de telefone](/end-user-flows/sign-up-and-sign-in/sign-up) e a verificam inserindo um código de verificação enviado pelo Logto para o número de telefone.                                                                                                                                                                                                                                                                                                                 | Código de verificação de cadastro Logto: `{{code}}`. Expira em 10 minutos.                 |
| ForgotPassword           | Se os usuários esquecerem a senha durante o login, podem optar por verificar a identidade usando o número de telefone primeiro para [redefinir a senha](/end-user-flows/sign-up-and-sign-in/reset-password).                                                                                                                                                                                                                                                                                                       | Código de verificação para redefinição de senha Logto: `{{code}}`. Expira em 10 minutos.   |
| Generic                  | Este modelo pode ser usado como uma opção geral de backup para vários cenários, incluindo testar configurações de conectores, [verificar ou vincular o número de telefone após o login](/end-user-flows/account-settings/by-management-api#email-and-phone-number-verification), entre outros.                                                                                                                                                                                                                     | Código de verificação Logto: `{{code}}`. Expira em 10 minutos.                             |
| OrganizationInvitation   | Use este modelo para [enviar aos usuários um link de convite](/end-user-flows/organization-experience/invite-organization-members#configure-your-email-connector) para ingressar na organização.                                                                                                                                                                                                                                                                                                                   | Código de verificação de convite para organização Logto: `{{code}}`. Expira em 10 minutos. |
| UserPermissionValidation | Durante o uso do aplicativo, pode haver algumas operações de alto risco ou operações com nível de risco relativamente alto que [exigem verificação adicional do usuário](/end-user-flows/account-settings/by-account-api#verify-by-sending-a-verification-code-to-the-users-email-or-phone), como transferências bancárias, exclusão de recursos em uso e cancelamento de assinaturas. O modelo `UserPermissionValidation` pode ser usado para definir o conteúdo do SMS de verificação recebido nestas situações. | Código de verificação Logto: `{{code}}`. Expira em 10 minutos.                             |
| BindNewIdentifier        | Quando um usuário modifica seu perfil, ele pode [vincular um número de telefone à conta atual](/end-user-flows/account-settings/by-account-api#manage-phone). Neste caso, o modelo `BindNewIdentifier` pode ser usado para personalizar o conteúdo do SMS de verificação.                                                                                                                                                                                                                                          | Código de verificação para vinculação de conta Logto: `{{code}}`. Expira em 10 minutos.    |
| MfaVerification          | Quando [SMS MFA](/end-user-flows/mfa/sms-mfa) está habilitado, este modelo é usado para enviar códigos de verificação aos usuários durante o processo de autenticação multifatorial.                                                                                                                                                                                                                                                                                                                               | Código de verificação em 2 etapas Logto: `{{code}}`. Expira em 10 minutos.                 |
| BindMfa                  | Quando [SMS MFA](/end-user-flows/mfa/sms-mfa) está habilitado, este modelo é usado para configurar o código de verificação SMS para MFA. Os usuários recebem este código ao vincular ou configurar o número de telefone como fator MFA para a conta.                                                                                                                                                                                                                                                               | Código de verificação para adicionar 2 etapas Logto: `{{code}}`. Expira em 10 minutos.     |

É importante entender estes parâmetros:

- O código de verificação é válido por 10 minutos. Atualmente, não suportamos personalização do tempo de expiração.
- O Logto substituirá o marcador `{{code}}` no modelo de SMS por um código de verificação. Portanto, certifique-se de que o modelo tenha um marcador reservado.

:::note
Alguns países e regiões podem não permitir o envio de conteúdo não aprovado via SMS devido a requisitos de conformidade. Os modelos de SMS precisam ser registrados e aprovados pelo provedor de SMS antes de serem utilizados. Nesses casos, o conteúdo pode ser indexado pelo ID do modelo para o modelo correspondente.
:::
