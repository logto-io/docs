O Logto fornece serviços de email integrados para sua conveniência nos seguintes cenários:

1. Explorar ou testar rapidamente a experiência de login por email do Logto.
2. Usar diretamente em seus produtos online. É voltado principalmente para startups que se sentem confortáveis em usar `logto.email` como domínio de email do remetente.

As características do serviço de email do Logto:

- **Gratuito para usar:** É totalmente gratuito, sem limites diários de envio de emails, economizando seus custos.
- **Sem esforço:** Não é necessário configurar nenhum provedor de serviço de email de terceiros. Basta personalizar as informações básicas de branding para o seu modelo de email. Se você ainda não possui informações de branding, pode optar por começar a usar com poucos cliques.
- **Entrega garantida:** Com base no serviço de email do Logto, você obtém um serviço estável e entrega confiável de emails, garantindo que os usuários possam acessar seu produto.

No entanto, apesar da conveniência, existem algumas limitações a serem consideradas:

1. Os emails serão enviados a partir do endereço fixo `no-reply@logto.email`.
2. Não é possível adicionar links ou qualquer outro conteúdo personalizado aos emails.

Dependendo das necessidades do seu negócio, você pode optar por usar outros provedores de serviço de email posteriormente. Oferecemos uma variedade de [conectores de serviço de email prontos para uso](/connectors/email-connectors#popular-email-providers), e também suportamos [SMTP](/integrations/smtp), [HTTP](/integrations/http-email) e [Webhook](/developers/webhooks) para envio de emails, então você sempre encontrará uma opção que atenda às suas necessidades.

:::note
O serviço de email gratuito integrado do Logto está atualmente disponível apenas para usuários do [Cloud](https://cloud.logto.io/). Para usuários do serviço open-source, você tem a flexibilidade de configurar seu próprio provedor de serviço de email para login por email.
:::

## Etapas de configuração \{#configuration-steps}

Siga estas etapas para configurar o serviço de email do Logto:

1. Vá para <CloudLink to="/connectors/passwordless">Conector > Conectores de Email e SMS</CloudLink>.
2. Para adicionar um novo conector de Email, clique no botão "**Configurar**" e selecione o conector "**Serviço de email do Logto**".
3. Depois que o conector "Serviço de email do Logto" for criado com sucesso, você pode personalizar as informações básicas de branding exibidas nos modelos de email.
4. Após fazer essas alterações, lembre-se de enviar um email de teste para seu endereço de email antes de salvar as alterações.

Opções de personalização:

- **Email do remetente:** O email do remetente é definido como `no-reply@logto.email` e não pode ser modificado.
- **Nome do remetente:** Defina o nome da sua marca como nome do remetente para garantir o reconhecimento do usuário.
- **Informações da empresa:** Exiba o nome da sua empresa, endereço ou CEP para aumentar a confiança do usuário e atender aos requisitos de conformidade. _Observe que URLs não são permitidas._
- **Logo do aplicativo:** Faça upload do logo da sua marca para que o valor da marca do aplicativo seja exibido nos emails recebidos pelos usuários.

## Modelos de email unificados \{#unified-email-templates}

O serviço de email do Logto utiliza modelos de email unificados adaptados para cenários específicos de autenticação:

| Uso                      | Cenário                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Register                 | Usuários criam uma conta usando o email e a verificam inserindo um código de verificação enviado pelo Logto para o email.                                                                                                                                                                                                                                                                |
| SignIn                   | Usuários fazem login usando o email e verificam inserindo o código de verificação em vez de uma senha.                                                                                                                                                                                                                                                                                   |
| ForgotPassword           | Se os usuários esquecerem a senha durante o login, podem optar por verificar sua identidade usando o email já verificado com o Logto.                                                                                                                                                                                                                                                    |
| Generic                  | Este modelo pode ser usado como opção geral de backup para vários cenários, incluindo teste de configurações de conectores e assim por diante.                                                                                                                                                                                                                                           |
| OrganizationInvitation   | Use este modelo para enviar aos usuários um link de convite para ingressar na organização.                                                                                                                                                                                                                                                                                               |
| UserPermissionValidation | Durante o uso do aplicativo, pode haver operações de alto risco ou operações com nível de risco relativamente alto que exigem verificação adicional do usuário, como transferências bancárias, exclusão de recursos em uso e cancelamento de assinaturas. O modelo `UserPermissionValidation` pode ser usado para definir o conteúdo do código de verificação recebido nestas situações. |
| BindNewIdentifier        | Quando um usuário modifica seu perfil, pode vincular um endereço de email à conta atual. Nesse caso, o modelo `BindNewIdentifier` pode ser usado para personalizar o conteúdo do email de verificação.                                                                                                                                                                                   |
| MfaVerification          | Quando o MFA por email está ativado, este modelo é usado para enviar códigos de verificação aos usuários durante o processo de autenticação multifatorial.                                                                                                                                                                                                                               |
| BindMfa                  | Quando o MFA por email está ativado, este modelo é usado para configurar o código de verificação por email para MFA. Os usuários recebem este código ao vincular ou configurar o email como fator de MFA para a conta.                                                                                                                                                                   |

## Perguntas frequentes \{#faqs}

<details>
<summary>

### É possível adicionar CSS ou HTML personalizado ao serviço de email do Logto? \{#is-it-possible-to-add-custom-css-or-html-to-logto-email-service}

</summary>

O serviço de email integrado do Logto não suporta CSS ou HTML personalizado. Você só pode modificar elementos genéricos de branding. Essa restrição existe para manter a estabilidade do serviço de email integrado, já que todos os locatários compartilham o mesmo endereço IP e endereço do remetente. Para mais detalhes, consulte "[Fatores para melhorar a entrega de emails](https://blog.logto.io/verification-email-delivery#factors-to-improve-email-delivery)".

Para personalizar modelos de email, recomendamos usar outro conector de email, como AWS Direct Mail, SendGrid, Mailgun, Postmark ou SMTP.

</details>
