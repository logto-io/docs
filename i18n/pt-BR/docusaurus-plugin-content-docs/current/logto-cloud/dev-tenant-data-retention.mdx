# Política de retenção de dados de tenant de desenvolvimento

O Logto Cloud segue um princípio claro: **produção é para persistência, desenvolvimento é para testes**.

- Em **tenants de produção** (Free, Pro e Enterprise), objetos e entidades são mantidos durante todo o ciclo de vida da conta, a menos que você os remova ou solicite o contrário.
- Em **tenants de desenvolvimento**, o objetivo é diferente. Eles são **ambientes seguros para testes**, focados em segurança, desempenho e ambientes limpos, em vez de armazenar dados a longo prazo.

Para manter esse princípio, o Logto aplica uma política de retenção de 90 dias em tenants de desenvolvimento: **usuários** em tenants de desenvolvimento são automaticamente excluídos após 90 dias.

Em ambientes de desenvolvimento, desenvolvedores frequentemente criam muitas contas de teste durante a integração. Essas contas geram grandes quantidades de dados, mas raramente oferecem valor a longo prazo. A política de 90 dias garante que os recursos da plataforma Logto sejam usados de forma eficiente, mantém os ambientes de teste limpos e performáticos, evita o acúmulo de contas não utilizadas e separa claramente dados temporários de teste dos dados de produção de longo prazo.

## Entendendo a política de 90 dias \{#understanding-the-90-day-policy}

### Apenas contas de usuário com mais de 90 dias são excluídas \{#only-user-accounts-older-than-90-days-are-deleted}

Excluir usuários significa remover todas as informações de perfil, dados de autenticação e informações relacionadas.

### O que permanece intacto \{#what-stays-intact}

Seu trabalho de desenvolvimento e configurações permanecem intocados. Isso facilita a transição do seu ambiente de desenvolvimento e integração para ambientes de produção no futuro. Oferecemos opções para [converter seu tenant de desenvolvimento](./tenant-settings#tenant-types-dev-vs-prod) diretamente em um tenant de produção.

## O que a exclusão automática de usuários significa para você \{#what-automatic-user-deletion-means-for-you}

- Proteção de privacidade: Contas de teste são frequentemente compartilhadas entre equipes durante o desenvolvimento ou ficam sem uso por longos períodos. Recomendamos renová-las a cada 90 dias.
- Eficiência de recursos: A exclusão automática libera capacidade de armazenamento e processamento, mantendo os ambientes limpos e eficientes.
- Separação clara de ambientes: Tenants de desenvolvimento são sandboxes temporários. Dados persistentes e de longo prazo pertencem aos tenants de produção.

## Melhores práticas para tenants de desenvolvimento \{#best-practices-for-dev-tenants}

### Durante o desenvolvimento \{#during-development}

- Use tenants de desenvolvimento do Logto como seu sandbox para testes.
- Anote cenários de teste importantes para poder recriá-los facilmente.
- Planeje cuidadosamente o ciclo de vida do seu desenvolvimento e migre para produção quando for o momento certo.

### Para projetos de longo prazo \{#for-long-term-projects}

- Crie novos usuários ou contas de teste conforme necessário, em vez de manter antigos.
- Use tenants de produção se precisar de ambientes de staging com dados persistentes.

## Perguntas frequentes \{#faqs}

<details>
<summary>

### Posso estender o período de retenção de 90 dias? \{#can-i-extend-the-90-day-retention-period}

</summary>

Não. A política de 90 dias é fixa para todos os tenants de desenvolvimento para manter a consistência e a segurança. Se você tiver necessidades especiais, entre em contato conosco para orientação.

</details>

<details>
<summary>

### Isso afeta tenants de produção? \{#does-this-affect-production-tenants}

</summary>

Não. Todos os dados em tenants de produção, incluindo usuários e configurações dos planos Free, Pro e Enterprise, não são afetados.

</details>

<details>
<summary>

### Para clientes enterprise, tenants de desenvolvimento em instâncias privadas serão excluídos? \{#for-enterprise-customers-will-dev-tenants-under-private-instances-be-deleted}

</summary>

Não. Tenants de desenvolvimento em instâncias privadas estão excluídos da política de exclusão de 90 dias.

</details>

## Resumo \{#summary}

1. Usuários em tenants de desenvolvimento são automaticamente removidos após 90 dias
2. Todas as suas configurações, aplicativos e ajustes permanecem permanentes
3. Esta política garante segurança, desempenho e condições limpas de teste
4. Tenants de produção possuem políticas de retenção diferentes para dados persistentes
5. Planeje seu fluxo de trabalho de desenvolvimento considerando esse ciclo automático de limpeza

Esta política de retenção garante que seu ambiente de desenvolvimento permaneça um campo de testes seguro, eficiente e confiável, protegendo a privacidade dos usuários e mantendo o desempenho do sistema.

Para armazenamento persistente de dados de usuários, considere atualizar para um tenant de produção, onde a retenção de dados segue os requisitos específicos da sua organização.
