# Limites do sistema

Os limites do sistema mantêm o Logto estável e justo para todos. Eles são separados da cobrança.

**Resumindo**

- **Locatários Dev:** limites fixos (limites rígidos)
- **Locatários Pro:** limites iniciais (limites flexíveis). Podemos aumentá-los para usos legítimos.
- **Locatários Enterprise:** configuráveis, definidos em contrato

Na página de preços, **ilimitado** significa _sem limite pré-definido de cobrança sob uso justo_. Para proteger a estabilidade da plataforma, aplicamos **limites iniciais** (como limites de entidades e limites de taxa) que podem ser aumentados para cargas de trabalho legítimas.

## Tipos de limites \{#types-of-limits}

### Limites de recursos do sistema e entidades \{#system-resource-and-entity-caps}

Contagens máximas para objetos como aplicativos, papéis, organizações, conectores, webhooks e certos recursos do sistema baseados em uso, como emissão de token de acesso.

### Limite de taxa \{#rate-limit}

Proteção a nível de requisição aplicada por locatário.

Padrão: **200 requisições por 10 segundos por locatário** (token-bucket). Isso equivale a cerca de **20 RPS sustentados**, com picos curtos permitidos até a capacidade do bucket.

Se seu tráfego for legítimo e exceder isso de forma consistente, podemos avaliar uma exceção.

## O que acontece quando você atinge um limite \{#what-happens-when-you-hit-a-limit}

Quando um limite é atingido, as requisições podem falhar com um erro como:

- **HTTP 429** (limite de taxa)
- **HTTP 4xx** com uma mensagem relacionada ao limite (limite de entidade ou restrição de recurso)

Isso não exclui dados. Apenas bloqueia a operação que excedeu o limite.

## Como solicitar aumento de limite \{#request-a-limit-increase}

Se você atingir um limite e seu caso de uso for legítimo, entre em contato conosco. Para Pro e Enterprise, geralmente podemos aumentar os limites rapidamente.

Por favor, inclua:

- ID do locatário (ou domínio do locatário)
- qual limite você atingiu (nome do recurso + o número que você precisa)
- (para solicitações de RPS) tráfego esperado: RPS médio, RPS de pico, padrão de burst
- endpoints afetados (ex.: login, token, JWKS, userinfo, Management API)
- prazo (quando você precisa do aumento)

[Entre em contato](https://logto.io/contact)

## Proteção de limite a nível de locatário \{#tenant-level-rate-limit-protection}

### Locatários Dev \{#dev-tenants}

[Locatários Dev](/logto-cloud/tenant-settings#development) são destinados para desenvolvimento e testes.

Os limites dos locatários Dev são **limites rígidos**, dimensionados para cargas de trabalho de desenvolvimento. Se você precisa de um ambiente semelhante à produção, pode [converter o locatário para Pro](/logto-cloud/tenant-settings#tenant-types-dev-vs-prod).

Locatários Dev também seguem uma [política de retenção de dados](/logto-cloud/dev-tenant-data-retention) separada.
