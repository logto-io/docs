# Limite do sistema

Os limites do sistema definem o número máximo de entidades que um locatário pode criar. Eles protegem a estabilidade da plataforma e são independentes da cobrança. Alguns itens na página de preços estão marcados como _ilimitados_ ou _pay-as-you-go_. Isso significa que não há limite de cobrança, mas limites internos do sistema ainda se aplicam para garantir o uso justo. Podemos analisar aumentos caso a caso.

## Tipos de limites \{#types-of-limits}

### Limites de recursos do sistema e entidades \{#system-resource-and-entity-caps}

Contagens máximas para objetos como aplicativos, papéis (roles), organizações (organizations), conectores (connectors) e certos recursos do sistema baseados em uso, como consumo total de tokens. Esses limites variam conforme o plano.

### Limite de taxa \{#rate-limit}

Proteção a nível de requisição aplicada por locatário. A diretriz geral é **200 requisições por 10 segundos por locatário**. Se seu tráfego for legítimo e exceder esse valor de forma consistente, podemos avaliar uma exceção caso a caso.

---

Se seu caso de uso for válido e você atingir um limite, entre em contato conosco. Isso nos ajuda a ajustar os limites e apoiar cenários do mundo real.

## Proteção de limite a nível de locatário \{#tenant-level-rate-limit-protection}

### Locatários de desenvolvimento (Dev tenants) \{#dev-tenants}

[Locatários de desenvolvimento](/logto-cloud/tenant-settings#development) oferecem acesso gratuito para testes. Para evitar abusos e manter as expectativas claras, locatários Dev possuem limites fixos de entidades. Se precisar de mais, você pode solicitar um aumento ou fazer upgrade para Pro para acesso total.

Locatários Dev também seguem uma [política de retenção de dados](/logto-cloud/dev-tenant-data-retention) separada.
