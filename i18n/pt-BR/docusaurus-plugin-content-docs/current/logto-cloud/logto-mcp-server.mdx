# Logto MCP Server

O Logto MCP Server é um servidor remoto [MCP (Model Context Protocol)](https://modelcontextprotocol.io/). Adicione a URL ao seu aplicativo de IA, faça login com sua conta Logto Cloud e comece a conversar com o Logto.

:::tip
O Logto MCP Server é construído com Logto e [MCP Auth](https://mcp-auth.dev). Se você quiser construir seu próprio servidor remoto MCP, veja [Construa seu próprio servidor remoto MCP](#build-your-own-remote-mcp-server).
:::

## O que você pode fazer \{#what-you-can-do}

- **Adicionar autenticação ao seu projeto**: A IA detecta seu framework, cria um aplicativo Logto e gera código de integração funcional. Suporta mais de 38 frameworks incluindo React, Next.js, Vue, Nuxt, SvelteKit, Express, Go, Python, iOS, Android, Flutter e outros.
- **Faça perguntas sobre Logto**: A IA pesquisa na documentação do Logto e fornece respostas com fontes.

Mais funcionalidades estão chegando em breve, incluindo configuração da experiência de login, gerenciamento de RBAC e permissões com orientações de melhores práticas, configuração de multi-tenancy para construir apps B2B com organizações, e mais.

## Configuração no seu aplicativo de IA \{#set-up-in-your-ai-application}

### Claude Code \{#claude-code}

<Tabs>
  <TabItem value="claude-mcp-add" label="claude mcp add">

    ```bash
    claude mcp add --transport http --client-id mcp-client --callback-port 8080 logto https://mcp.logto.io
    ```

  </TabItem>
  <TabItem value="claude-mcp-add-json" label="claude mcp add-json">

    ```bash
    claude mcp add-json logto '{"type":"http","url":"https://mcp.logto.io","oauth":{"clientId":"mcp-client","callbackPort":8080}}'
    ```

  </TabItem>
</Tabs>

Você pode alterar `8080` para qualquer porta disponível em sua máquina.

Em seguida, execute `/mcp` no Claude Code e siga o fluxo de login pelo navegador para autenticar com sua conta Logto Cloud.

### Claude Desktop \{#claude-desktop}

1. Abra **Configurações** no menu principal, navegue até **Conectores** e clique em **Adicionar conector personalizado**.
2. Insira o nome `Logto MCP Server` e a URL do servidor remoto MCP: `https://mcp.logto.io`.
3. Expanda **Configurações avançadas** e insira o OAuth Client ID: `mcp-client`. Clique em **Adicionar**.
4. Na lista **Conectores**, encontre **Logto MCP Server** e clique em **Conectar**.
5. Você será redirecionado primeiro para a página web do Claude. Se não tiver uma sessão ativa, faça login na sua conta Claude primeiro.
6. Após fazer login no Claude, você será redirecionado para a página de login do Logto Cloud. Faça login com sua conta Logto Cloud e autorize a conexão.

### Cursor \{#cursor}

1. Abra o palette de comandos (`Command + Shift + P`) e selecione **View: Open MCP Settings**.
2. Na seção **Installed MCP Servers**, clique no botão **Add Custom MCP**.
3. No arquivo `mcp.json` que abrir, adicione a seguinte configuração:

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Uma página de login do Logto Cloud aparecerá automaticamente. Faça login com sua conta Logto Cloud para conectar.

### OpenCode \{#opencode}

1. Adicione a seguinte configuração ao seu arquivo `opencode.json`:

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. Execute o seguinte comando para autenticar:
   ```bash
   opencode mcp auth Logto
   ```
   Observe que `Logto` corresponde ao nome do servidor MCP no seu arquivo de configuração.

Uma janela do navegador será aberta. Faça login com sua conta Logto Cloud para autorizar e conectar.

### VS Code \{#vs-code}

1. Abra o palette de comandos (`Command + Shift + P`) e selecione **MCP: Add Server...**.
2. Escolha **HTTP** como o tipo de servidor.
3. Insira a URL: `https://mcp.logto.io`. Você pode optar por instalar no seu workspace para melhor isolamento e gerenciamento.
4. Quando o fluxo OAuth iniciar, insira o App ID: `mcp-client`. Deixe o App Secret em branco e pressione Enter para pular.
5. Você será redirecionado para a página de login do Logto Cloud. Faça login com sua conta Logto Cloud para conectar.

### Windsurf \{#windsurf}

1. Abra o palette de comandos (`Command + Shift + P`) e selecione **Windsurf: MCP Marketplace**.
2. Na seção **Installed MCPs**, clique no ícone de configurações à direita para criar e abrir o arquivo `~/.codeium/windsurf/mcp_config.json`.
3. Adicione a seguinte [configuração](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json):

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Uma página de login do Logto Cloud aparecerá automaticamente. Faça login com sua conta Logto Cloud para conectar.

### Outros clientes MCP \{#other-mcp-clients}

O Logto MCP Server funciona com qualquer cliente MCP que suporte autorização OAuth. Use a seguinte configuração:

- **URL do servidor MCP:** `https://mcp.logto.io`
- **OAuth client ID:** `mcp-client`
- **Resource:** `https://mcp.logto.io/` (se seu cliente exigir)
- **Scope:** `mcp:all` (se seu cliente exigir)

Se você é desenvolvedor de aplicativos de IA e deseja suportar conexão com o Logto MCP Server, mas o Logto MCP Server ainda não suporta o redirect URI do seu aplicativo, por favor, [entre em contato conosco](https://logto.io/contact) com os detalhes do seu aplicativo.
