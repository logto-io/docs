### Buscar token de acesso para o recurso de API \{#fetch-access-token-for-the-api-resource}

<FetchAccessTokenForApiResources />

Se você precisar buscar um token de acesso no componente do servidor, pode usar a função `getAccessTokenRSC`:

```tsx title="app/page.tsx"
import { getAccessTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const accessToken = await getAccessTokenRSC(logtoConfig, 'https://shopping.your-app.com/api');

  return (
    <main>
      <p>Token de acesso: {accessToken}</p>
    </main>
  );
}
```

:::tip

HTTP não permite definir cookies após o início do streaming, `getAccessTokenRSC` não pode atualizar o valor do cookie, então se o token de acesso for atualizado, ele não será persistido na sessão. Recomenda-se usar a função `getAccessToken` no lado do cliente ou em manipuladores de rotas.

:::

### Buscar tokens de organização \{#fetch-organization-tokens}

<FetchOrganizationTokenForUser />

Se você precisar buscar um token de organização no componente do servidor, pode usar a função `getOrganizationTokenRSC`:

```tsx title="app/page.tsx"
import { getOrganizationTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const token = await getOrganizationTokenRSC(logtoConfig, 'organization-id');

  return (
    <main>
      <p>Token de organização: {token}</p>
    </main>
  );
}
```

:::tip

HTTP não permite definir cookies após o início do streaming, `getOrganizationTokenRSC` não pode atualizar o valor do cookie, então se o token de acesso for atualizado, ele não será persistido na sessão. Recomenda-se usar a função `getOrganizationToken` no lado do cliente ou em manipuladores de rotas.

:::

## Redirecionamento automático para login em caso de não autorizado \{#automatic-sign-in-redirect\}

:::tip
O helper `signIn` modifica cookies para estabelecer a sessão de login, então não pode ser invocado diretamente em um React Server Component (RSC). Para acionar o login automaticamente quando um RSC detectar um usuário não autorizado, chame `signIn` em um manipulador de rota dedicado e redirecione para essa rota.
:::

```ts title="app/sign-in/route.ts"
import { signIn } from '@logto/next/server-actions';
import { logtoConfig } from '../../logto';

export async function GET() {
  await signIn(logtoConfig);
}
```

```tsx title="app/protected/page.tsx"
import { getLogtoContext } from '@logto/next/server-actions';
import { redirect } from 'next/navigation';
import { logtoConfig } from '../logto';

export default async function ProtectedPage() {
  const { isAuthenticated } = await getLogtoContext(logtoConfig);

  if (!isAuthenticated) {
    redirect('/sign-in');
  }

  return <div>Conteúdo protegido</div>;
}
```
