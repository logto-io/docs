# Gerenciar configurações de banco de dados

Logto ajuda a manter algumas configurações técnicas, como chaves OIDC e Cookie. Antes de podermos gerenciá-las via Admin Console ou Management API, você pode usar CLI para alcançar esse objetivo.

## Obter configuração por chave \{#get-config-by-key}

Uso:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config get <key> [keys...]
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config get <key> [keys...]
```

  </TabItem>

</Tabs>

Digamos que você queira obter as chaves privadas OIDC:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config get oidc.privateKeys
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config get oidc.privateKeys
```

  </TabItem>

</Tabs>

A saída será como:

```bash
oidc.privateKeys=["\n-----BEGIN PRIVATE KEY-----\nMIIJRAIBA..."]
```

Execute `logto db config get --help` para todas as chaves disponíveis, ou veja [Configuração](/concepts/core-service/configuration/#database-configs) para explicação detalhada.

## Definir configuração por chave \{#set-config-by-key}

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config set <key> [keys...]
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config set <key> [keys...]
```

  </TabItem>

</Tabs>

Digamos que você queira definir as chaves de Cookie OIDC:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config set oidc.cookieKeys "[\"key1\",\"key2\"]"
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config set oidc.cookieKeys "[\"key1\",\"key2\"]"
```

  </TabItem>

</Tabs>

Normalmente, o valor a ser definido deve ser uma string JSON válida e seguir a definição de tipo pré-definida. Execute `logto db config set --help` para todas as chaves disponíveis, ou veja [Configuração](/concepts/core-service/configuration/) para explicação detalhada.
