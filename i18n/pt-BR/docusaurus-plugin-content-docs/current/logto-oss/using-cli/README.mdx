# Logto CLI

Logto CLI oferece uma maneira conveniente de instalar, manter e atualizar sua instância Logto sem lidar manualmente com aquelas tarefas técnicas entediantes.

:::note
Atualmente, não fornecemos binários, portanto, Node.js `^18.12.0` é necessário no ambiente.
:::

### Instalar CLI globalmente

Instale via seu gerenciador de pacotes preferido para usar o comando `logto` globalmente:

<Tabs>

  <TabItem value="npm" label="npm">

```bash
npm i @logto/cli -g
```

  </TabItem>
  <TabItem value="yarn" label="Yarn">

```bash
yarn global add @logto/cli
```

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

```bash
pnpm add -g @logto/cli
```

  </TabItem>

</Tabs>

Você sempre pode adicionar `--help` para mostrar as informações de ajuda de cada comando:

```bash
logto --help
logto db seed --help
```

### CLI Local

Uma versão embutida do CLI é fornecida com cada lançamento do Logto desde a v1.0.0-beta.11. Suponha que você já tenha uma instância Logto em `~/logto`, então você pode executar os comandos abaixo:

```bash
cd ~/logto
npm run cli # Invocar o CLI local
```

Observe que se você quiser executar com opções, precisa adicionar um `--` ANTES das opções para o Logto CLI, por exemplo:

```bash
npm run cli db seed -- --db-url postgresql://your-database-url
```

### Usar CLI com `npx`

Outra maneira de usar o Logto CLI é com `npx`, que pode executar um pacote sem instalação.

```bash
npx @logto/cli
```

Isso será útil para invocações únicas, por exemplo:

```bash
npx @logto/cli db seed --db-url postgresql://your-database-url
```
