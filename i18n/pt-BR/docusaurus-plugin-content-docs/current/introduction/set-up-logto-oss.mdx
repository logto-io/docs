# Configurar Logto OSS

Esta seção abrange as etapas básicas de configuração e ações-chave para entregar efetivamente seu produto e gerenciar seu fluxo de trabalho de desenvolvimento para o [serviço de código aberto Logto (OSS)](https://github.com/logto-io/logto).

## Comece com Logto OSS

Siga o [guia de introdução](/logto-oss/get-started-with-oss/) para lançar o Logto hoje, e mais tarde você pode consultar o [guia de implantação](/logto-oss/deployment-and-configuration) completo para uso em produção.

## Funcionalidades suportadas pelo Logto OSS

O Logto OSS suporta a maioria das capacidades principais do serviço Logto e é regularmente atualizado e mantido.

Alguns recursos avançados são atualmente exclusivos da versão Logto Cloud, incluindo [Logto Protected App](/integrate-logto/protected-app), [serviço de email gratuito](/connectors/email-connectors/built-in-email-service), [SSO iniciado por IdP](/end-user-flows/enterprise-sso/idp-initiated-sso) e [SAML IdP](/integrate-logto/third-party-applications).

## Mantenha-se atualizado com os lançamentos do Logto

Para manter sua instância do Logto atualizada com os recursos mais recentes, certifique-se de seguir a página de [Lançamentos do Logto no GitHub](https://github.com/logto-io/logto/releases), onde você pode encontrar todos os registros de lançamento.

Leia o [guia sobre atualização](/logto-oss/upgrading-oss-version) para aprender como atualizar o Logto sem alterar seu código ou esquema de banco de dados.
