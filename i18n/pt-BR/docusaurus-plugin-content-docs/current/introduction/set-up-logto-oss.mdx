# Configurar Logto OSS

Esta seção abrange as etapas básicas de configuração e ações-chave para entregar efetivamente seu produto e gerenciar seu fluxo de trabalho de desenvolvimento para o [serviço open-source Logto (OSS)](https://github.com/logto-io/logto).

## Comece com Logto OSS \{#get-started-with-logto-oss}

Siga o [guia de introdução](/logto-oss/get-started-with-oss/) para lançar o Logto hoje, e mais tarde você pode consultar o [guia de implantação](/logto-oss/deployment-and-configuration) completo para uso em produção.

## Funcionalidade suportada pelo Logto OSS \{#feature-supported-by-logto-oss}

O Logto OSS suporta a maioria das capacidades principais do serviço Logto e é regularmente atualizado e mantido.

Alguns recursos avançados são atualmente exclusivos da versão Logto Cloud, incluindo:

| Limitações de recursos no OSS                                                       | Descrição                                                                                                                                          |
| ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Console Logto: múltiplos locatários](/logto-cloud/tenant-settings)                 | Criar e gerenciar múltiplos locatários Logto em um console.                                                                                        |
| [Console Logto: convite de colaborador](/logto-cloud/tenant-member-management)      | Convidar e gerenciar múltiplos membros de um locatário Logto.                                                                                      |
| Console Logto: Habilitar MFA                                                        | Aumentar a segurança exigindo autenticação multifatorial ao fazer login no Console Logto.                                                          |
| [Logto Protected App](/integrate-logto/protected-app)                               | Integração rápida e sem SDK de autenticação alimentada pelo Cloudflare.                                                                            |
| [Serviço de email embutido do Logto](/connectors/email-connectors/built-in-email-service) | Utilizar o serviço de email embutido e gratuito para entrega de emails.                                                                            |
| [Traga sua UI](/customization/bring-your-ui)                                        | Personalizar a UI ou fluxos da experiência de login usando este recurso (OSS pode bifurcar o código no GitHub para personalizar a experiência de login diretamente.) |
| [SSO iniciado pelo IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)           | Habilitar autenticação única corporativa iniciada pelo provedor de identidade.                                                                     |
| [Aplicativos SAML](/integrate-logto/saml-app)                                       | Integrar aplicativos via o protocolo SAML. A versão OSS é limitada a 3 aplicativos SAML.                                                           |

Dicas: Multi-tenancy, convites de membros e MFA não estão disponíveis para sua equipe fazer login em um console Logto open-source. No entanto, você pode implementar esses recursos em seu próprio produto usando Logto OSS, tornando-os disponíveis para seus usuários finais.

## Mantenha-se atualizado com os lançamentos do Logto \{#stay-updated-with-logto-releases}

Para manter sua instância Logto atualizada com os recursos mais recentes, certifique-se de seguir a página de [Lançamentos do Logto no GitHub](https://github.com/logto-io/logto/releases), onde você pode encontrar todos os registros de lançamento.

Leia o [guia sobre atualização](/logto-oss/upgrading-oss-version) para aprender como atualizar o Logto sem alterar seu código ou esquema de banco de dados.
