# Configurar Logto OSS

Esta seção aborda as etapas básicas de configuração e ações-chave para entregar seu produto de forma eficaz e gerenciar seu fluxo de trabalho de desenvolvimento para o [serviço Logto open-source (OSS)](https://github.com/logto-io/logto).

## Comece com Logto OSS \{#get-started-with-logto-oss}

Siga o [guia de início rápido](/logto-oss/get-started-with-oss/) para lançar o Logto hoje mesmo, e depois você pode consultar o [guia completo de implantação](/logto-oss/deployment-and-configuration) para uso em produção.

## Funcionalidades suportadas pelo Logto OSS \{#feature-supported-by-logto-oss}

O Logto OSS suporta a maioria das capacidades principais do serviço Logto e é atualizado e mantido regularmente.

Alguns recursos avançados são atualmente exclusivos da versão Logto Cloud, incluindo:

| Limitações de recursos no OSS                                                              | Descrição                                                                                                                                                           |
| ------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Console Logto: múltiplos tenants](/logto-cloud/tenant-settings)                           | Crie e gerencie múltiplos tenants Logto em um console.                                                                                                              |
| [Console Logto: convite de colaboradores](/logto-cloud/tenant-member-management)           | Convide e gerencie múltiplos membros de um tenant Logto.                                                                                                            |
| Console Logto: Ativar MFA                                                                  | Aumente a segurança exigindo autenticação multifatorial (MFA) ao fazer login no Console Logto.                                                                      |
| [Logto Protected App](/integrate-logto/protected-app)                                      | Integração rápida e sem SDK de autenticação, alimentada pelo Cloudflare.                                                                                            |
| [Serviço de email integrado do Logto](/connectors/email-connectors/built-in-email-service) | Utilize o serviço de email integrado e gratuito para envio de emails.                                                                                               |
| [Traga sua UI](/customization/bring-your-ui)                                               | Personalize a interface ou os fluxos da experiência de login usando este recurso (no OSS, é possível fazer fork do código no GitHub para personalizar diretamente). |
| [SSO iniciado pelo IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)                  | Habilite autenticação única corporativa iniciada pelo provedor de identidade (IdP).                                                                                 |
| [Apps SAML](/integrate-logto/saml-app)                                                     | Integre aplicativos via o protocolo SAML. A versão OSS é limitada a 3 apps SAML.                                                                                    |

Dicas: Multi-tenancy, convites de membros e MFA não estão disponíveis para sua equipe fazer login em um console Logto open-source. No entanto, você pode implementar esses recursos em seu próprio produto usando Logto OSS, tornando-os disponíveis para seus usuários finais.

## Mantenha-se atualizado com os lançamentos do Logto \{#stay-updated-with-logto-releases}

Para manter sua instância Logto atualizada com os recursos mais recentes, siga a página de [Releases do Logto no GitHub](https://github.com/logto-io/logto/releases), onde você encontra todos os registros de lançamentos.

Leia o [guia de atualização](/logto-oss/upgrading-oss-version) para aprender como atualizar o Logto sem alterar seu código ou o esquema do banco de dados.
