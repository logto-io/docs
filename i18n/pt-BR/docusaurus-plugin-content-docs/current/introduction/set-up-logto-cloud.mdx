# Configurar Logto Cloud

Esta seção cobre as etapas básicas de configuração e ações-chave para entregar seu produto de forma eficaz e gerenciar seu fluxo de trabalho de desenvolvimento.

## Criar tenant Logto \{#create-logto-tenant}

Primeiro, cadastre-se como empresa e crie um tenant Logto. Um tenant do Logto Cloud é um ambiente isolado onde você pode gerenciar identidades de usuários, aplicativos e todos os outros recursos do Logto.

Seu primeiro tenant será configurado automaticamente como um ambiente de [**Desenvolvimento**](/logto-cloud/tenant-settings#development). Isso é ideal para fins de teste e desenvolvimento e é gratuito para usar.

Confira as configurações do tenant e [saiba mais](/logto-cloud) sobre o serviço Logto Cloud.

## Convidar colaboradores \{#invite-collaborators}

Durante a integração, você tem a oportunidade de convidar colaboradores para trabalhar juntos no desenvolvimento. Se quiser fazer isso mais tarde ou adicionar mais pessoas, vá para <CloudLink to="/tenant-settings/members">Configurações do tenant > Membros</CloudLink>. Você verá seus membros atuais e poderá convidar mais. Insira os endereços de e-mail e atribua papéis (você pode fazer isso em lote) e então envie.

## Migrar para tenant de produção \{#move-to-production-tenant}

Você pode gastar algum tempo desenvolvendo e testando as capacidades e funcionalidades do Logto para montar uma prova de conceito.

Após o término dos testes do seu projeto e quando estiver pronto para a próxima fase, lembre-se de criar um novo [tenant do tipo produção](/logto-cloud/tenant-settings#production) (requer um [tenant Free ou Pro](https://logto.io/pricing)). Pois o tenant de desenvolvimento não é destinado ao uso em produção devido às suas [limitações e restrições](/logto-cloud/tenant-settings#development).

:::note

Certifique-se de inserir o [domínio personalizado](/logto-cloud/custom-domain) assim que criar um tenant de produção, pois isso impactará sua configuração dos próximos passos.

:::

Essa configuração permite que seu projeto tenha ambientes de teste de desenvolvimento e produção, ajudando você a gerenciar seu fluxo de trabalho de desenvolvimento de forma tranquila.

## Migrar do sistema existente \{#migrate-from-the-existing-system}

Se você está trabalhando em um projeto existente e precisa trocar de provedor de autenticação, o Logto oferece suporte à migração de usuários de outras plataformas. Você pode migrar [dados básicos](/user-management/user-data#basic-data), [dados personalizados](/user-management/user-data#custom-data), [identidades sociais](/user-management/user-data#social-identities) e hashes de senha. Para detalhes, veja o guia [Migrar para Logto](../user-management/user-migration.mdx).

## Usar Logto MCP Server com ferramentas de IA \{#use-logto-mcp-server-with-ai-tools}

Se você está usando ferramentas de desenvolvimento com IA como VS Code com Copilot, Cursor ou Claude Desktop, pode se conectar ao [Logto MCP Server](/logto-cloud/logto-mcp-server) para interagir com o Logto diretamente do seu assistente de IA. A IA pode detectar seu framework, criar aplicativos e gerar código de integração funcional para você.
