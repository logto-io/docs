# Configurar Logto Cloud

Esta seção abrange as etapas básicas de configuração e ações-chave para entregar seu produto de forma eficaz e gerenciar seu fluxo de trabalho de desenvolvimento.

## Criar locatário Logto \{#create-logto-tenant}

Primeiro, inscreva-se para um negócio e crie um locatário Logto. Um locatário do Logto Cloud é um ambiente isolado onde você pode gerenciar identidades de usuários, aplicativos e todos os outros recursos do Logto.

Seu primeiro locatário será automaticamente configurado como um ambiente de [**Desenvolvimento**](/logto-cloud/tenant-settings#development). Isso é ideal para fins de teste e desenvolvimento e é gratuito para usar.

Confira as configurações do locatário e [saiba mais](/logto-cloud) sobre o serviço Logto Cloud.

## Convidar colaboradores \{#invite-collaborators}

Durante a integração, você tem a chance de convidar colaboradores para realizar o trabalho de desenvolvimento juntos. Se desejar fazer mais ou mais tarde, vá para <CloudLink to="/tenant-settings/members">Configurações do locatário > Membros</CloudLink>. Você verá seus membros atuais e poderá convidar mais. Insira endereços de e-mail e atribua papéis (você pode fazer isso em massa) e, em seguida, envie.

## Mover para locatário de produção \{#move-to-production-tenant}

Você pode gastar algum tempo desenvolvendo e testando as capacidades e recursos do Logto para montar uma prova de conceito.

Após a conclusão dos testes do seu projeto e ele estiver pronto para a próxima fase, lembre-se de criar um novo [locatário do tipo produção](/logto-cloud/tenant-settings#production) (requer um [locatário Gratuito ou Pro](https://logto.io/pricing)). Porque o locatário de desenvolvimento não é destinado para uso em produção devido às suas [limitações e restrições](/logto-cloud/tenant-settings#development).

:::note

Certifique-se de inserir o [domínio personalizado](/logto-cloud/custom-domain) assim que criar um locatário de produção, pois isso impactará sua configuração na próxima etapa.

:::

Essa configuração permite que seu projeto tenha ambientes de teste de desenvolvimento e produção, ajudando você a gerenciar seu fluxo de trabalho de desenvolvimento de forma suave.

## Migrar do sistema existente \{#migrate-from-the-existing-system}

Se você está trabalhando em um projeto existente e precisa trocar de provedores de autenticação, o Logto suporta a migração de usuários de outras plataformas. Você pode migrar [dados básicos](/user-management/user-data#basic-data), [dados personalizados](/user-management/user-data#custom-data), [identidades sociais](/user-management/user-data#social-identities) e hashes de senha. Para detalhes, consulte o guia [Migrar para Logto](../user-management/user-migration.mdx).
