# Parâmetros de autenticação

Em um fluxo de login OIDC padrão, aplicativos clientes iniciam uma solicitação de autenticação que redireciona o usuário para a página web de experiência de login universal hospedada pelo Logto. Com base nas suas configurações de experiência de login, os usuários podem fazer login ou se registrar usando vários identificadores, métodos de verificação e conectores de SSO social ou corporativo de terceiros.

Além dos [parâmetros de autenticação OIDC](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest) padrão, nosso produto introduz vários parâmetros de autenticação personalizados que permitem adaptar a experiência de login desejada para os usuários finais.

Isso é particularmente útil quando você deseja impor fluxos de login específicos para diferentes segmentos de usuários. Incluindo, mas não se limitando aos seguintes cenários:

1. **Registro direto para novos usuários**: Para uma campanha de registro direcionada, você pode querer direcionar novos usuários diretamente para a página de registro, ignorando o formulário de login padrão, para garantir uma experiência de integração perfeita.
2. **Registro com email pré-preenchido a partir de um link de assinatura**: Se você estiver coletando endereços de email através de um formulário de assinatura auto-hospedado, esses parâmetros permitem redirecionar os novos assinantes para a página de registro com o campo de email pré-preenchido, facilitando o processo de registro.
3. **Link direto de redefinição de senha**: Para novos usuários que precisam configurar sua nova senha, envie um link direto e redirecione-os para a página de redefinição de senha com o campo de email pré-preenchido.
4. **Login social instantâneo**: Se você fornecer botões de login de mídia social diretamente no seu site, pode usar esses parâmetros para pular o formulário de login padrão e permitir que os usuários se autentiquem diretamente com seu provedor social escolhido.
5. **Login SSO direto para clientes corporativos**: Para produtos que atendem a vários clientes corporativos, esses parâmetros permitem exibir um botão de SSO dedicado que faz login dos usuários com seu provedor de SSO específico, pulando o formulário de login universal e proporcionando uma experiência perfeita para cada organização.

Continue lendo para saber mais sobre os parâmetros de autenticação para personalizar a [primeira tela](/end-user-flows/authentication-parameters/first-screen) ou habilitar uma experiência de [login direto](/end-user-flows/authentication-parameters/direct-sign-in) para seus usuários.
