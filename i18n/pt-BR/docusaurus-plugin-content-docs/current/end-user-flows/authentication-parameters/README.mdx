# Parâmetros de autenticação

Em um fluxo padrão de login OIDC, os aplicativos clientes iniciam uma solicitação de autenticação que redireciona o usuário para a página web universal de experiência de login hospedada pelo Logto. Com base nas configurações da sua experiência de login, os usuários podem fazer login ou cadastro usando vários identificadores, métodos de verificação e conectores de SSO social ou corporativo de terceiros.

O Logto também suporta parâmetros opcionais padrão do OIDC, como:

- `login_hint`: Fornece uma dica para o identificador do usuário, por exemplo, preencher previamente email / nome de usuário
- [`ui_locales`](/end-user-flows/authentication-parameters/ui-locales): Controla o idioma em tempo de execução para a interação atual, incluindo a interface de login e emails relacionados.

Além dos [parâmetros de autenticação OIDC](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest) padrão, nosso produto introduz vários parâmetros de autenticação personalizados que permitem adaptar a experiência de login desejada para os usuários finais.

Isso é especialmente útil quando você deseja impor fluxos de login específicos para diferentes segmentos de usuários. Incluindo, mas não se limitando, aos seguintes cenários:

### Cadastro direto para novos usuários \{#direct-sign-up-for-new-users}

    Para uma campanha de registro direcionada, você pode querer direcionar novos usuários diretamente para a página de cadastro, ignorando o formulário de login padrão, para garantir uma experiência de onboarding sem atritos.

    **Uso**: Parâmetro [Primeira tela](/end-user-flows/authentication-parameters/first-screen) → `first_screen=register`

### Cadastro com email preenchido a partir de um link de assinatura \{#email-prefilled-sign-up-from-a-subscription-link}

    Se você está coletando endereços de email por meio de um formulário de assinatura auto-hospedado, esses parâmetros permitem redirecionar os novos assinantes para a página de cadastro com o campo de email pré-preenchido, facilitando o processo de registro.

    **Uso**: Parâmetro [Primeira tela](/end-user-flows/authentication-parameters/first-screen) → `first_screen=identifier:sign_up&identifier=email&login_hint=foo@gmail.com`

### Link direto para redefinição de senha \{#direct-password-reset-link}

    Para novos usuários que precisam configurar sua nova senha, envie um link direto e redirecione-os para a página de redefinição de senha com o campo de email preenchido.

    **Uso**: Parâmetro [Primeira tela](/end-user-flows/authentication-parameters/first-screen) → `first_screen=reset_password`

### Login social instantâneo \{#instant-social-login}

    Se você oferece botões de login de redes sociais diretamente em seu site, pode usar esses parâmetros para pular o formulário de login padrão e permitir que os usuários se autentiquem diretamente com o provedor social escolhido.

    **Uso**: Parâmetro [Login direto](/end-user-flows/authentication-parameters/direct-sign-in#social-sign-in) → `direct_sign_in=social:google`

### Login SSO direto para clientes corporativos \{#direct-sso-sign-in-for-enterprise-customers}

    Para produtos que atendem vários clientes corporativos, esses parâmetros permitem exibir um botão SSO dedicado que autentica os usuários com seu provedor SSO específico, pulando o formulário universal de login e proporcionando uma experiência fluida para cada organização.

    **Uso**: Parâmetro [Login direto](/end-user-flows/authentication-parameters/direct-sign-in#enterprise-sso) → `direct_sign_in=sso:1234567890`

### Métodos de login personalizados exibidos para diferentes sites \{#custom-sign-in-methods-displayed-for-different-sites}

    Se seu produto atende tanto equipes internas quanto consumidores, ou diferentes organizações, você pode adaptar a experiência de login conforme necessário. Por exemplo: - Para ferramentas internas, exiba apenas login por email / nome de usuário e senha. - Para páginas voltadas ao consumidor, ofereça todas as opções de login disponíveis, incluindo login com Google.

    **Uso**: Diferentes configurações de [primeira tela](/end-user-flows/authentication-parameters/first-screen) ou [login direto](/end-user-flows/authentication-parameters/direct-sign-in) com base no tipo de site.

Continue lendo para saber mais sobre os parâmetros de autenticação para personalizar a [primeira tela](/end-user-flows/authentication-parameters/first-screen) ou habilitar uma experiência de [login direto](/end-user-flows/authentication-parameters/direct-sign-in) para seus usuários.
