# Estrutura de dados da organização

No nível da organização, existem vários atributos cujos valores podem ser criados, atualizados, lidos e excluídos (CRUD).

## ID da Organização \{#organization-id}

O _organization id_ é um identificador único para cada organização. É útil para implementar experiências de login no nível da organização e recuperar tokens de organização.

## Nome \{#name}

O _name_ suporta login no nível da organização e pode ser integrado em interfaces de produtos no nível da organização conforme necessário.

## Descrição \{#description}

O campo _description_ permite adicionar texto para ajudar a identificar e rotular a organização.

## Logos da Organização \{#organization-logos}

Para mostrar dinamicamente o logotipo da organização do seu cliente na experiência de login, você pode fazer upload dos logos da organização na página de configurações da organização.

Veja [logos específicos da organização](/customization/match-your-brand/#organization-specific-branding) para mais detalhes.

## Dados personalizados \{#custom-data}

_Dados personalizados (Custom data)_ é um objeto JSON usado para armazenar informações extras sobre a organização. Ele oferece aos desenvolvedores flexibilidade para armazenar detalhes adicionais da organização para fins de desenvolvimento.

## MFA é obrigatório \{#is-mfa-required}

_isMfaRequired_ indica se a Autenticação Multifatorial (MFA) é obrigatória para a organização. Se configurado como `true`, todos os membros devem passar pela MFA durante o login para acessar a organização. Esta configuração de política de segurança é configurada no nível da organização.

Veja [Gerenciar organização](/organizations/organization-management#require-mfa-for-organization-members) para mais detalhes.

## Criado em \{#created-at}

_createdAt_ é o carimbo de data/hora com o fuso horário quando a organização foi criada.

## ID do Inquilino \{#tenant-id}

_tenantId_ identifica o inquilino ao qual a organização pertence.
