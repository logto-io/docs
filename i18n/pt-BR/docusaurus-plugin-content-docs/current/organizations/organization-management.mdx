# Gerenciar organização

Esta seção aborda como os desenvolvedores gerenciam suas organizações via Logto Console ou Logto Management API, não como os administradores da organização gerenciam seus membros dentro do seu aplicativo. Para saber mais sobre como desenvolver a experiência da sua organização, por favor, consulte [este guia](/end-user-flows/organization-experience/organization-management).

## Gerenciar via Logto Console \{#manage-via-logto-console}

### Criar uma organização \{#create-an-organization}

Navegue até <CloudLink to="/organizations">Console > Organizations</CloudLink> e clique no botão "Create organization".

### Configurações básicas \{#basic-settings}

Você pode configurar os atributos básicos da organização, como nome, descrição, logotipo, etc.

### Exigir MFA para membros da organização \{#require-mfa-for-organization-members}

Você pode exigir que todos os membros de uma organização habilitem MFA. Esta é uma medida de segurança para garantir que todos os membros tenham uma camada extra de proteção ao acessar os recursos da organização.

Para habilitar este recurso, vá para a página de detalhes da organização e ative o interruptor "Multi-factor authentication (MFA)".

:::note

Você precisará [habilitar pelo menos um método de MFA](/end-user-flows/mfa) para que este recurso funcione corretamente.

:::

Uma vez habilitado, membros sem MFA configurado não poderão trocar [tokens de acesso da organização](/authorization/organization-template/protect-organization-resources#step-2-fetch-organization-token) até que configurem o MFA.

Por favor, note que:

- Este recurso apenas verifica se o usuário tem MFA configurado. Ele não força os usuários a usarem MFA ao trocar tokens de acesso.
- Este recurso não limita quais métodos de MFA os usuários podem usar.

### Provisionamento Just-in-Time \{#just-in-time-provisioning}

[Provisionamento Just-in-Time](https://auth.wiki/jit-provisioning) adiciona automaticamente usuários a uma organização quando eles fazem login pela primeira vez no aplicativo. No Logto, isso é suportado para [SSO corporativo (Enterprise SSO)](/end-user-flows/enterprise-sso) e provisionamento baseado em domínio de email. Quando os usuários atendem a critérios específicos, como fazer login através de um IdP corporativo específico ou usar um email com um determinado domínio, eles são automaticamente adicionados à organização.

Você também pode definir papéis padrão da organização para membros quando eles ingressam pela primeira vez na organização.

Para mais detalhes sobre o provisionamento Just-in-Time e como configurá-lo, consulte [esta seção](/organizations/just-in-time-provisioning).

### Gerenciar membros da organização \{#manage-organization-members}

Os usuários podem ter um ou mais papéis. Ao adicionar membros a uma organização, você tem a opção de atribuir papéis a vários usuários de uma vez. Se você deixar essa atribuição em branco, os usuários adicionados não receberão nenhum papel.

Na <CloudLink to="/users"> Console > User management > User details page</CloudLink>, você pode ver a quais organizações o usuário pertence e quais papéis da organização ele possui.

### Gerenciar aplicativos M2M da organização \{#manage-organization-m2m-applications}

Aplicativos máquina para máquina também podem ser adicionados a organizações. Você pode [atribuir papéis a aplicativos máquina para máquina](/organizations/understand-how-organizations-work#organization-m2m-application) da mesma forma que atribui papéis a usuários.

Na <CloudLink to="/applications"> Console > Applications > Application details page</CloudLink>, você pode ver a quais organizações o aplicativo está associado e quais papéis da organização ele possui.

## Gerenciar via Logto Management API \{#manage-via-logto-management-api}

Tudo o que você pode fazer no Logto Console também pode ser feito através do [Management API](/integrate-logto/interact-with-management-api). Isso inclui, mas não se limita a:

1. Criar, excluir ou editar uma organização.
2. Gerenciar modelo de organização: criar, excluir ou editar permissões e papéis da organização.
3. Adicionar membros a, ou remover membros de uma organização.
4. Atribuir ou remover os papéis da organização do usuário.
5. Adicionar aplicativos máquina para máquina a, ou remover aplicativos máquina para máquina de uma organização.
6. Atribuir ou remover os papéis da organização de aplicativos máquina para máquina.

Você também pode conferir esta seção para usar o Management API para habilitar mais experiência e gerenciamento em nível de organização. [Saiba mais](/end-user-flows/organization-experience)

Para uma lista completa de capacidades, por favor, consulte nossas [referências de API](https://openapi.logto.io/group/endpoint-organizations).
