# Configuración del tenant

Los nuevos usuarios que se registran en Logto Cloud son incorporados automáticamente a un tenant de entorno **Desarrollo** (Dev) gratuito. Puedes explorar todas las funciones en este tenant.

Si deseas crear un tenant separado para el entorno de **Producción** (Prod) o para un nuevo proyecto, haz clic en el nombre de tu tenant actual en la esquina superior izquierda de la barra superior. En este menú, puedes cambiar entre tenants o crear uno nuevo.

Haz clic en "Crear tenant", luego:

- Nombre para el tenant
- Selecciona una [región de datos del tenant](#tenant-region)
- Selecciona el [tipo de tenant](#tenant-types-dev-vs-prod) (entorno)

Para un tenant existente, ve a <CloudLink to="/settings">Consola > Configuración del tenant > Configuración</CloudLink>. Aquí puedes:

- Ver el ID del tenant
- Actualizar el nombre del tenant
- Ver la [región del tenant](#tenant-region). Esto no se puede cambiar después de la creación.
- Ver el [tipo de tenant](#tenant-types-dev-vs-prod). Puedes convertir un tenant Dev en un tenant Prod si es necesario.
- [Abandonar el tenant](#leave-tenant)
- [Eliminar el tenant](#delete-tenant)

## Región del tenant \{#tenant-region}

Cuando creas un tenant, puedes elegir la región donde se almacenarán los datos del tenant. No se puede cambiar después de crear el tenant. Estas son las regiones disponibles:

- Europa (Países Bajos)
- Oeste de EE. UU. (Arizona)
- Australia (Australia Este)
- Japón (Japón Este)

Normalmente, deberías elegir la región más cercana a tus clientes para minimizar la latencia y mejorar el rendimiento.

Logto aprovecha la red global edge para ofrecer el mejor rendimiento y disponibilidad para tus aplicaciones. El enrutamiento de solicitudes está optimizado para garantizar que tus usuarios siempre estén conectados con la mejor opción disponible.

:::note
¿Buscas otra región? [Ponte en contacto con nosotros](https://logto.io/contact) para:

- Solicitar una nueva región de nube pública
- Consultar sobre un despliegue de Logto Private Cloud en tu ubicación preferida
  :::

## Tipos de tenant: Dev vs. Prod \{#tenant-types-dev-vs-prod}

Hay dos tipos de tenants en Logto Cloud: Desarrollo (Dev) y Producción (Prod). Con esta diferenciación de tenants, puedes gestionar mejor tus proyectos en diferentes entornos para mayor eficiencia y, al mismo tiempo, disfrutar de todo el valor de Logto.

Puedes elegir el tipo de tenant durante la creación. Cuando estés listo para pasar a producción, hay dos opciones:

- **Crear un nuevo tenant de Producción**  
  Configura un tenant de producción desde cero. Esto es ideal si deseas mantener separados los entornos de desarrollo y producción.
- **Convertir tu tenant Dev actual en Producción**  
  Si prefieres no rehacer la configuración ni migrar usuarios, puedes actualizar tu tenant de Desarrollo existente a un tenant de Producción de pago.

  - **Convertir a un plan Pro**: Ve a <CloudLink to="/tenant-settings/settings">Consola > Configuración del tenant > Configuración</CloudLink>, luego haz clic en "Convertir" para actualizar por autoservicio. Cualquier función de pago que hayas usado en el tenant dev se trasladará al checkout de Stripe.
  - **Convertir a un plan Enterprise**: [Contáctanos](https://logto.io/contact) y te ayudaremos a completar la actualización.

  :::note
  Una vez convertido, el tenant no puede volver a un entorno Dev; por favor, confirma que estás listo antes de proceder.
  :::

### Desarrollo \{#development}

El tenant de desarrollo (tenant dev) está destinado principalmente a fines de prueba y no debe utilizarse en un entorno de producción. Estos tenants permiten el acceso a funciones premium y de pago disponibles en los planes de pago, de forma gratuita y sin requerir suscripción.

Sin embargo, existen ciertas limitaciones que se aplican a los tenants de desarrollo:

- El tenant dev eliminará automáticamente usuarios después de 90 días. Consulta la [política de retención de datos del tenant Dev](./dev-tenant-data-retention) para más detalles.
- Aparece un banner durante la experiencia de inicio de sesión, indicando que el tenant está en modo desarrollo.
- Los tenants de desarrollo pueden tener límites de cuota en funciones específicas. Estos límites se explican en la página de detalles de la función, si corresponde.

### Producción \{#production}

El tenant de producción es donde los usuarios finales acceden a la aplicación en vivo y es posible que necesites una [suscripción de pago](https://logto.io/pricing). Puedes suscribirte al plan Free o Pro para crear un tenant de producción. Si te suscribes al plan Free, solo puedes crear hasta 10 tenants.

## Habilitar MFA \{#enable-mfa}

Mejora la seguridad de tu espacio de trabajo exigiendo la Autenticación Multifactor (MFA) para todos los miembros de tu tenant Logto Pro / Enterprise.

Como el autoservicio aún no está disponible, por favor [contáctanos](https://logto.io/contact) para habilitar esta función.

## Habilitar SSO empresarial \{#enable-enterprise-sso}

Logto Cloud admite la integración de inicio de sesión único empresarial para tenants del Plan Enterprise, incluyendo proveedores como Google Workspace, Okta, Azure AD y más.

Para comenzar, por favor [contáctanos](https://logto.io/contact). Te ayudaremos a configurarlo rápidamente.

## Abandonar tenant \{#leave-tenant}

Un administrador puede [invitar miembros adicionales](/logto-cloud/tenant-member-management) a este tenant.

Si hay al menos otro **admin** (rol), o si eres un **colaborador** (rol), puedes elegir abandonar el tenant. Después de salir, todos los recursos del tenant permanecerán, pero ya no tendrás acceso a ellos.

Si eres el último administrador, debes asignar a otro colaborador como administrador antes de poder salir.

## Eliminar tenant \{#delete-tenant}

[Los administradores](/logto-cloud/tenant-member-management#invite-collaborators) pueden eliminar un tenant de Logto. Eliminar un tenant elimina permanentemente todos los datos de usuario y configuraciones asociadas. Esta acción NO SE PUEDE deshacer. Logto te pedirá que ingreses el nombre del tenant para confirmar y ayudar a prevenir eliminaciones accidentales.

Si necesitas ayuda, por favor [contáctanos](https://logto.io/contact) por correo electrónico.

## Preguntas frecuentes \{#faqs}

<details>
<summary>

### ¿Cómo migrar entre tenants de Logto o entre Cloud y OSS, o exportar todos los datos de usuario? \{#how-to-migrate-between-logto-tenants-or-between-cloud-and-oss-or-export-all-user-data}

</summary>

Actualmente puedes convertir tu tenant de **Desarrollo** en un tenant de **Producción** de pago por ti mismo. [Más información](#tenant-types-dev-vs-prod)

Sin embargo, la migración por autoservicio (todas las configuraciones y datos de usuario) entre Logto Cloud y la versión OSS no está soportada. Si necesitas este servicio, por favor [contacta al equipo de Logto](https://logto.io/contact) para discutir tus opciones.

Si planeas dejar de usar Logto Cloud para un proyecto, Logto puede ayudarte a exportar todos los datos de usuario. Por favor, [contáctanos](https://logto.io/contact).

</details>
