# Límites del sistema

Los límites del sistema mantienen Logto estable y justo para todos. Son independientes de la facturación.

**TL;DR**

- **Tenants de desarrollo (Dev tenants):** límites fijos (límites duros)
- **Tenants Pro:** límites iniciales (límites blandos). Podemos aumentarlos para usos legítimos.
- **Tenants Enterprise:** configurables, definidos en el contrato

En la página de precios, **ilimitado** significa _sin límite de facturación preestablecido bajo uso justo_. Para proteger la estabilidad de la plataforma, aplicamos **límites iniciales** (como límites de entidades y límites de tasa) que pueden aumentarse para cargas de trabajo legítimas.

## Tipos de límites \{#types-of-limits}

### Límites de recursos del sistema y entidades \{#system-resource-and-entity-caps}

Recuentos máximos para objetos como aplicaciones, roles, organizaciones, conectores, webhooks y ciertos recursos del sistema basados en uso, como la emisión de tokens de acceso (access token).

### Límite de tasa \{#rate-limit}

Protección a nivel de solicitud aplicada por tenant.

Por defecto: **200 solicitudes cada 10 segundos por tenant** (token-bucket). Esto equivale aproximadamente a **20 RPS sostenidos**, permitiendo ráfagas cortas hasta la capacidad del bucket.

Si tu tráfico es legítimo y supera consistentemente este límite, podemos evaluar una excepción.

## ¿Qué sucede cuando alcanzas un límite? \{#what-happens-when-you-hit-a-limit}

Cuando se activa un límite, las solicitudes pueden fallar con un error como:

- **HTTP 429** (límite de tasa)
- **HTTP 4xx** con un mensaje relacionado con el límite (límite de entidad o límite de función)

Esto no elimina datos. Solo bloquea la operación que excedió el límite.

## Cómo solicitar un aumento de límite \{#request-a-limit-increase}

Si alcanzas un límite y tu caso de uso es real, contáctanos. Para Pro y Enterprise, normalmente podemos aumentar los límites rápidamente.

Por favor incluye:

- ID del tenant (o dominio del tenant)
- qué límite alcanzaste (nombre de la función + el número que necesitas)
- (para solicitudes de RPS) tráfico esperado: RPS promedio, RPS máximo, patrón de ráfaga
- endpoints afectados (por ejemplo, sign-in, token, JWKS, userinfo, Management API)
- marco de tiempo (cuándo necesitas el aumento)

[Contáctanos](https://logto.io/contact)

Los límites de tenants de desarrollo son **límites duros**, dimensionados para cargas de trabajo de desarrollo. Si necesitas un entorno similar a producción, puedes [convertir el tenant a Pro](/logto-cloud/tenant-settings#tenant-types-dev-vs-prod).
