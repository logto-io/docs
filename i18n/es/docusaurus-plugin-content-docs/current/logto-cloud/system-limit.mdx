# Límite del sistema

Los límites del sistema definen el número máximo de entidades que un inquilino puede crear. Protegen la estabilidad de la plataforma y son independientes de la facturación. Algunos elementos en la página de precios están marcados como _ilimitados_ o _pago por uso_. Esto significa que no hay un tope de facturación, pero aún se aplican límites internos del sistema para garantizar un uso justo. Podemos revisar aumentos caso por caso.

## Tipos de límites \{#types-of-limits}

### Límites de recursos del sistema y entidades \{#system-resource-and-entity-caps}

Recuentos máximos para objetos como aplicaciones, roles, organizaciones, conectores y ciertos recursos del sistema basados en el uso, como el consumo total de tokens. Estos límites varían según el plan.

### Límite de velocidad \{#rate-limit}

Protección a nivel de solicitud aplicada por inquilino. La pauta general es **200 solicitudes cada 10 segundos por inquilino**. Si tu tráfico es legítimo y supera este límite de forma constante, podemos evaluar una excepción caso por caso.

---

Si tu caso de uso es válido y alcanzas un límite, contáctanos. Esto nos ayuda a ajustar los límites y apoyar escenarios del mundo real.

## Protección de límite a nivel de inquilino \{#tenant-level-rate-limit-protection}

### Inquilinos de desarrollo \{#dev-tenants}

[Los inquilinos de desarrollo](/logto-cloud/tenant-settings#development) ofrecen acceso gratuito para pruebas. Para prevenir abusos y mantener expectativas claras, los inquilinos de desarrollo tienen límites fijos de entidades. Si necesitas más, puedes solicitar un aumento o actualizar a Pro para acceso completo.

Los inquilinos de desarrollo también siguen una [política de retención de datos](/logto-cloud/dev-tenant-data-retention) separada.
