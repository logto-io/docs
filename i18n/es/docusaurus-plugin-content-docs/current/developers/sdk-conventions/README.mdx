# Convenciones del SDK de la plataforma

Logto proporciona un servicio de autenticación web muy poderoso y flexible.

En el uso práctico de los servicios de Logto, por conveniencia, a menudo es necesario que los desarrolladores integren el SDK de Logto en sus propias aplicaciones cliente para gestionar el estado de inicio de sesión del usuario, permisos y más.

Puedes encontrar SDKs para todos los lenguajes de programación / frameworks soportados por Logto [aquí](/quick-starts).

Si tienes mala suerte y no encuentras el SDK que deseas, aquí tienes una convención que puedes seguir para implementar el SDK para tu lenguaje de programación deseado, facilitando el uso de los servicios de Logto.

Esta convención contiene tres partes principales:

- [Estrategia de diseño](/developers/sdk-conventions/design-strategy)
- [Convención del SDK principal](/developers/sdk-conventions/core-sdk-convention)
- [Convención del SDK de la plataforma](/developers/sdk-conventions/platform-sdk-convention)
