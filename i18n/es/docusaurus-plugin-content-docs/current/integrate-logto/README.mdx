# Integra la autenticación de Logto

Logto proporciona soluciones integrales de autenticación para aplicaciones web, móviles y de escritorio, admite la autenticación [máquina a máquina (M2M)](/quick-starts/m2m) entre servicios y puede funcionar como Proveedor de Identidad (IdP) para [aplicaciones de terceros](/integrate-logto/third-party-applications) a través de protocolos estándar como [OpenID Connect (OIDC)](https://auth.wiki/openid-connect) y [OAuth 2.0](https://auth.wiki/oauth-2.0).

Comienza tu integración seleccionando la solución que mejor se adapte a tus necesidades:

## Añade autenticación a tus aplicaciones \{#add-authentication-for-your-applications}

Ya sea que estés desarrollando aplicaciones orientadas al usuario (como aplicaciones web, móviles o de escritorio) o aplicaciones máquina a máquina (M2M) para la comunicación entre servicios, puedes implementar rápidamente funciones completas de [autenticación](/end-user-flows) y [gestión de usuarios](/user-management) integrando Logto.

Construido sobre los estándares OIDC, Logto permite el **inicio de sesión omni** en todas tus aplicaciones. Cuando integras varias aplicaciones con Logto, comparten el mismo sistema de identidad y métodos de autenticación. Esto significa que los usuarios pueden iniciar sesión una vez y acceder sin problemas a todas tus aplicaciones conectadas con una experiencia de autenticación unificada.

## Añade aplicaciones OIDC/OAuth de terceros \{#add-third-party-oidc-oauth-apps}

Cuando quieras permitir que aplicaciones de terceros se autentiquen con tu sistema de identidad Logto, puedes configurar Logto como un [Proveedor de Identidad (IdP)](https://auth.wiki/identity-provider).

Logto puede usarse como IdP a través de protocolos estándar como OAuth y OIDC, y los servicios externos pueden implementar "Iniciar sesión con Logto" igual que "Iniciar sesión con Google", permitiendo que sus usuarios se autentiquen usando tu sistema de usuarios de Logto.

A diferencia de las aplicaciones de primera parte, las aplicaciones de terceros solo pueden solicitar permisos limitados. Durante el flujo de autenticación, los usuarios verán una **pantalla de consentimiento (consent screen)** donde podrán revisar y aprobar qué datos y permisos solicita la aplicación de terceros antes de conceder acceso.

Al crear aplicaciones de terceros en Logto, estableces conexiones seguras entre tu sistema de identidad y servicios externos, mientras das a tus usuarios control sobre el intercambio de sus datos.

## Amplía las funciones de Logto con Management API \{#extend-logto-features-with-management-api}

La [Management API](https://openapi.logto.io/) de Logto te permite controlar y automatizar programáticamente la funcionalidad de Logto en tus aplicaciones.

Cuando necesites ampliar las capacidades de Logto para mejorar tu producto (como [gestionar organizaciones multi-inquilino](/organizations), o automatizar flujos de trabajo con Logto), puedes aprovechar la Management API de Logto para construir funciones avanzadas que se adapten a las necesidades de tu producto.
