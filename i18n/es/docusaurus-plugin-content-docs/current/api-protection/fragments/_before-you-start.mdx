## Antes de comenzar \{#before-you-start}

Tus aplicaciones cliente necesitan obtener tokens de acceso (Access tokens) de Logto. Si aún no has configurado la integración del cliente, consulta nuestras [Guías rápidas](/quick-starts) para React, Vue, Angular u otros frameworks de cliente, o revisa nuestra [Guía de máquina a máquina](/quick-starts/m2m) para el acceso de servidor a servidor.

Esta guía se centra en la **validación del lado del servidor** de esos tokens en tu aplicación {getFrameworkName(props.framework)}.

### Lo que aprenderás \{#what-you-will-learn}

- **Validación de JWT:** Aprende a validar tokens de acceso (Access tokens) y extraer información de autenticación (Authentication)
- **Implementación de middleware:** Crea middleware reutilizable para la protección de API
- **Modelos de permisos:** Comprende e implementa diferentes patrones de autorización (Authorization):
  - Recursos de API globales para endpoints a nivel de aplicación
  - Permisos de organización para el control de funciones específicas de inquilinos
  - Recursos de API a nivel de organización para el acceso a datos multi-inquilino
- **Integración de RBAC:** Aplica permisos y alcances (Scopes) basados en roles (Roles) en tus endpoints de API

### Requisitos previos \{#prerequisites}

- Última versión estable de {getFrameworkLanguageName(props.framework)} instalada
- Conocimientos básicos de {getFrameworkName(props.framework)} y desarrollo de API web
- Una aplicación Logto configurada (consulta las [Guías rápidas](/quick-starts) si es necesario)
