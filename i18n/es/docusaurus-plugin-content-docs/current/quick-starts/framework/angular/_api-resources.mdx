### Configurar `angular-auth-oidc-client` para el recurso de API \{#configure-angular-auth-oidc-client-for-api-resource}

<ApiResourcesDescription />

<ConfigApiResources
  configResourcesCode={<ConfigResourcesCode />}
  configResourcesWithScopesCode={<ConfigResourcesWithScopesCode />}
/>

Ahora, el token de acceso estará en el formato JSON Web Token (JWT) en lugar de una cadena aleatoria (token opaco).

:::warning
Tanto `autoUserInfo` como `renewUserInfoAfterTokenRenew` estarán deshabilitados cuando se establezca `resource`. Esto se debe a que el token de acceso se solicitará para el recurso de API específico y no para el endpoint de información del usuario.
:::

Actualmente, solo los SDKs oficiales de Logto admiten la capacidad de solicitar tanto información del usuario como tokens de acceso a recursos de API. Si necesitas solicitar ambos, no dudes en contactarnos.
