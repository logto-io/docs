# Configurar Logto OSS

Esta sección cubre los pasos básicos de configuración y acciones clave para entregar eficazmente tu producto y gestionar tu flujo de trabajo de desarrollo para [Logto open-source service (OSS)](https://github.com/logto-io/logto).

## Comenzar con Logto OSS

Sigue la [guía de inicio](/logto-oss/get-started-with-oss/) para lanzar Logto hoy, y más adelante puedes consultar la [guía de despliegue](/logto-oss/deployment-and-configuration) completa para su uso en producción.

## Funciones compatibles con Logto OSS

Logto OSS admite la mayoría de las capacidades centrales del servicio Logto y se actualiza y mantiene regularmente.

Algunas funciones avanzadas son actualmente exclusivas de la versión Logto Cloud, incluyendo [Logto Protected App](/integrate-logto/protected-app), [servicio de correo electrónico gratuito](/connectors/email-connectors/built-in-email-service), [SSO iniciado por IdP](/end-user-flows/enterprise-sso/idp-initiated-sso) y [SAML IdP](/integrate-logto/third-party-applications).

## Mantente actualizado con las versiones de Logto

Para mantener tu instancia de Logto actualizada con las últimas funciones, asegúrate de seguir la página de [Logto GitHub Releases](https://github.com/logto-io/logto/releases), donde puedes encontrar todos los registros de versiones.

Lee la [guía sobre cómo actualizar](/logto-oss/upgrading-oss-version) para aprender cómo actualizar Logto sin cambiar tu código o esquema de base de datos.
