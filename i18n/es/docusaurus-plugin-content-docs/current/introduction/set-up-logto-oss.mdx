# Configura Logto OSS

Esta sección cubre los pasos básicos de configuración y las acciones clave para entregar tu producto de manera efectiva y gestionar tu flujo de trabajo de desarrollo para el [servicio open-source de Logto (OSS)](https://github.com/logto-io/logto).

## Comienza con Logto OSS \{#get-started-with-logto-oss}

Sigue la [guía de inicio](/logto-oss/get-started-with-oss/) para lanzar Logto hoy mismo, y más adelante puedes consultar la [guía completa de despliegue](/logto-oss/deployment-and-configuration) para su uso en producción.

## Funcionalidades soportadas por Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS admite la mayoría de las capacidades principales del servicio Logto y se actualiza y mantiene regularmente.

Algunas funciones avanzadas son actualmente exclusivas de la versión Logto Cloud, incluyendo:

| Limitaciones de funcionalidades en OSS                                                                   | Descripción                                                                                                                                                                                          |
| -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Consola Logto: múltiples inquilinos](/logto-cloud/tenant-settings)                                      | Crea y gestiona múltiples inquilinos de Logto en una consola.                                                                                                                                        |
| [Consola Logto: invitación de colaboradores](/logto-cloud/tenant-member-management)                      | Invita y gestiona múltiples miembros de un inquilino Logto.                                                                                                                                          |
| Consola Logto: Habilitar MFA                                                                             | Mejora la seguridad requiriendo autenticación multifactor (MFA) al iniciar sesión en la Consola Logto.                                                                                               |
| [Logto Protected App](/integrate-logto/protected-app)                                                    | Integración rápida y sin SDK de autenticación impulsada por Cloudflare.                                                                                                                              |
| [Servicio de correo electrónico integrado de Logto](/connectors/email-connectors/built-in-email-service) | Utiliza el servicio de correo electrónico integrado y gratuito para el envío de emails.                                                                                                              |
| [Trae tu UI](/customization/bring-your-ui)                                                               | Personaliza la UI o los flujos de la experiencia de inicio de sesión usando esta función (OSS puede bifurcar el código en GitHub para personalizar la experiencia de inicio de sesión directamente). |
| [SSO iniciado por IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)                                 | Habilita el inicio de sesión único empresarial iniciado por el proveedor de identidad (IdP).                                                                                                         |
| [Aplicaciones SAML](/integrate-logto/saml-app)                                                           | Integra aplicaciones mediante el protocolo SAML. La versión OSS está limitada a 3 aplicaciones SAML.                                                                                                 |

Consejo: La multi-tenencia, las invitaciones de miembros y MFA no están disponibles para que tu equipo inicie sesión en una consola Logto open-source. Sin embargo, puedes implementar estas funciones en tu propio producto usando Logto OSS, poniéndolas a disposición de tus usuarios finales.

## Mantente actualizado con los lanzamientos de Logto \{#stay-updated-with-logto-releases}

Para mantener tu instancia de Logto actualizada con las últimas funciones, asegúrate de seguir la página de [Lanzamientos de Logto en GitHub](https://github.com/logto-io/logto/releases), donde puedes encontrar todos los registros de lanzamientos.

Lee la [guía sobre actualización](/logto-oss/upgrading-oss-version) para aprender cómo actualizar Logto sin cambiar tu código ni el esquema de tu base de datos.
