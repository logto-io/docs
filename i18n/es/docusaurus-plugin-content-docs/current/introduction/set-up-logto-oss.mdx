# Configurar Logto OSS

Esta sección cubre los pasos básicos de configuración y acciones clave para entregar eficazmente tu producto y gestionar tu flujo de trabajo de desarrollo para [Logto servicio de código abierto (OSS)](https://github.com/logto-io/logto).

## Comenzar con Logto OSS \{#get-started-with-logto-oss}

Sigue la [guía de inicio](/logto-oss/get-started-with-oss/) para lanzar Logto hoy, y más adelante puedes consultar la [guía de implementación](/logto-oss/deployment-and-configuration) completa para uso en producción.

## Funcionalidades soportadas por Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS admite la mayoría de las capacidades centrales del servicio Logto y se actualiza y mantiene regularmente.

Algunas características avanzadas son actualmente exclusivas de la versión Logto Cloud, incluyendo [Logto Protected App](/integrate-logto/protected-app), [servicio de correo electrónico gratuito](/connectors/email-connectors/built-in-email-service), [SSO iniciado por IdP](/end-user-flows/enterprise-sso/idp-initiated-sso), y [SAML IdP](/integrate-logto/third-party-applications).

## Mantente actualizado con los lanzamientos de Logto \{#stay-updated-with-logto-releases}

Para mantener tu instancia de Logto actualizada con las últimas características, asegúrate de seguir la página de [Lanzamientos de Logto en GitHub](https://github.com/logto-io/logto/releases), donde puedes encontrar todos los registros de lanzamientos.

Lee la [guía sobre cómo actualizar](/logto-oss/upgrading-oss-version) para aprender cómo actualizar Logto sin cambiar tu código o esquema de base de datos.
