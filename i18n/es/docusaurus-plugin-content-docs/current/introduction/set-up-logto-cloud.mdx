# Configura Logto Cloud

Esta sección cubre los pasos básicos de configuración y las acciones clave para entregar tu producto de manera efectiva y gestionar tu flujo de trabajo de desarrollo.

## Crea un tenant de Logto \{#create-logto-tenant}

Primero, regístrate para una empresa y crea un tenant de Logto. Un tenant de Logto Cloud es un entorno aislado donde puedes gestionar identidades de usuarios, aplicaciones y todos los demás recursos de Logto.

Tu primer tenant se configurará automáticamente como un entorno de [**Desarrollo**](/logto-cloud/tenant-settings#development). Esto es ideal para pruebas y propósitos de desarrollo y es gratuito.

Consulta la configuración del tenant y [aprende más](/logto-cloud) sobre el servicio Logto Cloud.

## Invita colaboradores \{#invite-collaborators}

Durante la incorporación, tienes la oportunidad de invitar colaboradores para trabajar juntos en el desarrollo. Si deseas hacer más o hacerlo más tarde, ve a <CloudLink to="/tenant-settings/members">Configuración del tenant > Miembros</CloudLink>. Verás tus miembros actuales y podrás invitar a más. Ingresa las direcciones de correo electrónico y asigna roles (puedes hacerlo en lote) y luego envía.

## Pasa a un tenant de producción \{#move-to-production-tenant}

Puedes dedicar un tiempo a desarrollar y probar las capacidades y características de Logto para armar una prueba de concepto.

Después de que tu proyecto complete las pruebas y esté listo para la siguiente fase, recuerda crear un nuevo [tenant de tipo producción](/logto-cloud/tenant-settings#production) (requiere un [tenant Free o Pro](https://logto.io/pricing)). Porque el tenant de desarrollo no está destinado para uso en producción debido a sus [limitaciones y restricciones](/logto-cloud/tenant-settings#development).

:::note

Asegúrate de ingresar el [dominio personalizado](/logto-cloud/custom-domain) tan pronto como crees un tenant de producción, ya que impactará la configuración de los siguientes pasos.

:::

Esta configuración permite que tu proyecto tenga tanto entornos de pruebas de desarrollo como de producción, ayudándote a gestionar tu flujo de trabajo de desarrollo sin problemas.

## Migra desde el sistema existente \{#migrate-from-the-existing-system}

Si estás trabajando en un proyecto existente y necesitas cambiar de proveedor de autenticación, Logto admite la migración de usuarios desde otras plataformas. Puedes migrar [datos básicos](/user-management/user-data#basic-data), [datos personalizados](/user-management/user-data#custom-data), [identidades sociales](/user-management/user-data#social-identities) y hashes de contraseñas. Para más detalles, consulta la guía [Migrar a Logto](../user-management/user-migration.mdx).

## Usa Logto MCP Server con herramientas de IA \{#use-logto-mcp-server-with-ai-tools}

Si utilizas herramientas de desarrollo potenciadas por IA como VS Code con Copilot, Cursor o Claude Desktop, puedes conectarte a [Logto MCP Server](/logto-cloud/logto-mcp-server) para interactuar con Logto directamente desde tu asistente de IA. La IA puede detectar tu framework, crear aplicaciones y generar código de integración funcional para ti.
