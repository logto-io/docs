# Configura Logto Cloud

Esta sección cubre los pasos básicos de configuración y acciones clave para entregar efectivamente tu producto y gestionar tu flujo de trabajo de desarrollo.

## Crear un tenant de Logto \{#create-logto-tenant}

Primero, regístrate para un negocio y crea un tenant de Logto. Un tenant de Logto Cloud es un entorno aislado donde puedes gestionar identidades de usuarios, aplicaciones y todos los demás recursos de Logto.

Tu primer tenant se configurará automáticamente como un entorno de [**Desarrollo**](/logto-cloud/tenant-settings#development). Esto es ideal para propósitos de prueba y desarrollo y es gratuito.

Consulta la configuración del tenant y [aprende más](/logto-cloud) sobre el servicio en la nube de Logto.

## Invitar a colaboradores \{#invite-collaborators}

Durante la incorporación, tienes la oportunidad de invitar a colaboradores para trabajar juntos en el desarrollo. Si deseas hacer más o más tarde, ve a <CloudLink to="/tenant-settings/members">Configuración del tenant > Miembros</CloudLink>. Verás a tus miembros actuales y podrás invitar a más. Ingresa direcciones de correo electrónico y asigna roles (puedes hacerlo en bloque) y luego envía.

## Moverse al tenant de producción \{#move-to-production-tenant}

Puedes pasar algún tiempo desarrollando y probando las capacidades y características de Logto para armar una prueba de concepto.

Después de que tu proyecto complete las pruebas y esté listo para la siguiente fase, recuerda crear un nuevo [tenant de tipo producción](/logto-cloud/tenant-settings#production) (requiriendo un [tenant Free o Pro](https://logto.io/pricing)). Porque el tenant de desarrollo no está destinado para uso en producción debido a sus [limitaciones y restricciones](/logto-cloud/tenant-settings#development).

:::note

Asegúrate de ingresar el [dominio personalizado](/logto-cloud/custom-domain) tan pronto como crees un tenant de producción, ya que impactará en la configuración de tu siguiente paso.

:::

Esta configuración permite que tu proyecto tenga tanto entornos de prueba de desarrollo como de producción, ayudándote a gestionar tu flujo de trabajo de desarrollo sin problemas.

## Migrar desde el sistema existente \{#migrate-from-the-existing-system}

Si estás trabajando en un proyecto existente y necesitas cambiar de proveedores de autenticación, Logto admite la migración de usuarios desde otras plataformas. Puedes migrar [datos básicos](/user-management/user-data#basic-data), [datos personalizados](/user-management/user-data#custom-data), [identidades sociales](/user-management/user-data#social-identities) y hashes de contraseñas. Para más detalles, consulta la guía [Migrar a Logto](../user-management/user-migration.mdx).
