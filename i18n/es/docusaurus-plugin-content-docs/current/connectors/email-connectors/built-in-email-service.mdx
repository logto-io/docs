Logto proporciona servicios de correo electrónico integrados para tu comodidad en los siguientes escenarios:

1. Explora o prueba rápidamente la experiencia de inicio de sesión por correo electrónico de Logto.
2. Úsalo directamente para tus productos en línea. Está pensado principalmente para nuevas startups que se sienten cómodas usando `logto.email` como su dominio de correo electrónico remitente.

Las características del servicio de correo electrónico de Logto:

- **Gratis:** Es completamente gratuito y sin límites diarios de uso de correo electrónico, lo que te ahorra costes.
- **Sin esfuerzo:** No se requiere configuración con ningún proveedor de servicios de correo electrónico de terceros. Simplemente personaliza la información básica de marca para tu plantilla de correo electrónico. Si aún no tienes tu propia información de marca, puedes optar por empezar a usarlo con unos pocos clics.
- **Entrega garantizada:** Basado en el servicio de correo electrónico de Logto, puedes obtener un servicio estable y una entrega de correos fiable, asegurando que los usuarios puedan acceder a tu producto.

Sin embargo, aunque es conveniente, hay algunas limitaciones que debes tener en cuenta:

1. Los correos electrónicos se enviarán desde la dirección fija `no-reply@logto.email`.
2. No puedes añadir enlaces ni ningún otro contenido personalizado a los correos electrónicos.

Dependiendo de las necesidades cambiantes de tu negocio, puedes optar por usar otros proveedores de servicios de correo electrónico más adelante. Ofrecemos una variedad de [conectores de servicios de correo electrónico listos para usar](/connectors/email-connectors#popular-email-providers), y también soportamos [SMTP](/integrations/smtp), [HTTP](/integrations/http-email) y [WebHook](/developers/webhooks) como desencadenadores para el envío de correos electrónicos, por lo que siempre encontrarás una opción que se adapte a ti.

:::note
El servicio de correo electrónico gratuito integrado de Logto está disponible actualmente solo para usuarios de [Cloud](https://cloud.logto.io/). Para los usuarios del servicio de código abierto, tienes la flexibilidad de configurar tu propio proveedor de servicios de correo electrónico para el inicio de sesión por correo electrónico.
:::

## Pasos de configuración \{#configuration-steps}

Sigue estos pasos para configurar el servicio de correo electrónico de Logto:

1. Ve a <CloudLink to="/connectors/passwordless">Conector > Conectores de correo electrónico y SMS</CloudLink>.
2. Para añadir un nuevo conector de correo electrónico, haz clic en el botón "**Configurar**" y selecciona el conector "**Servicio de correo electrónico de Logto**".
3. Una vez que el conector "Servicio de correo electrónico de Logto" se haya creado correctamente, puedes personalizar la información básica de marca que se muestra en las plantillas de correo electrónico.
4. Después de realizar estos cambios, recuerda enviar una plantilla de correo electrónico de prueba a tu dirección de correo antes de guardar los cambios.

Opciones de personalización:

- **Correo electrónico del remitente:** El correo del remitente está configurado como `no-reply@logto.email` y no puede ser modificado.
- **Nombre del remitente:** Establece el nombre de tu marca como nombre del remitente para asegurar el reconocimiento por parte del usuario.
- **Información de la empresa:** Muestra el nombre de tu empresa, dirección o código postal para aumentar la confianza del usuario y cumplir con los requisitos de cumplimiento. _Ten en cuenta que no se permiten URLs._
- **Logo de la app:** Sube el logo de tu marca para que el valor de tu marca se muestre en los correos recibidos por los usuarios.

## Plantillas de correo electrónico unificadas \{#unified-email-templates}

El servicio de correo electrónico de Logto utiliza plantillas de correo electrónico unificadas adaptadas para escenarios específicos de autenticación:

| Uso                               | Escenario                                                                                                                                                                                                                                                                                                                                                                                                                             |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Registro                          | Los usuarios crean una cuenta usando su correo electrónico y la verifican introduciendo un código de verificación enviado por Logto a su correo.                                                                                                                                                                                                                                                                                      |
| Inicio de sesión                  | Los usuarios inician sesión usando su correo electrónico y verifican introduciendo un código de verificación en lugar de una contraseña.                                                                                                                                                                                                                                                                                              |
| Olvidé mi contraseña              | Si los usuarios olvidan su contraseña durante el inicio de sesión, pueden optar por verificar su identidad usando el correo electrónico que ya han verificado con Logto.                                                                                                                                                                                                                                                              |
| Genérico                          | Esta plantilla puede usarse como opción de respaldo general para varios escenarios, incluyendo pruebas de configuración de conectores, etc.                                                                                                                                                                                                                                                                                           |
| Invitación a organización         | Usa esta plantilla para enviar a los usuarios un enlace de invitación para unirse a la organización.                                                                                                                                                                                                                                                                                                                                  |
| Validación de permisos de usuario | Durante el uso de la app, puede haber algunas operaciones de alto riesgo o con un nivel de riesgo relativamente alto que requieran verificación adicional del usuario, como transferencias bancarias, eliminación de recursos en uso y cancelación de membresías. La plantilla `UserPermissionValidation` puede usarse para definir el contenido del código de verificación por correo que los usuarios reciben en estas situaciones. |
| Vincular nuevo identificador      | Cuando un usuario modifica su perfil, puede vincular una dirección de correo electrónico a su cuenta actual. En este caso, la plantilla `BindNewIdentifier` puede usarse para personalizar el contenido del correo de verificación.                                                                                                                                                                                                   |
| Verificación MFA                  | Cuando el MFA por correo electrónico está habilitado, esta plantilla se usa para enviar códigos de verificación a los usuarios durante el proceso de autenticación multifactor.                                                                                                                                                                                                                                                       |
| Vincular MFA                      | Cuando el MFA por correo electrónico está habilitado, esta plantilla se usa para configurar el código de verificación por correo para MFA. Los usuarios reciben este código de verificación cuando vinculan o configuran su dirección de correo como factor MFA para su cuenta.                                                                                                                                                       |

## Preguntas frecuentes \{#faqs}

<details>
<summary>

### ¿Es posible añadir CSS o HTML personalizado al servicio de correo electrónico de Logto? \{#is-it-possible-to-add-custom-css-or-html-to-logto-email-service}

</summary>

El servicio de correo electrónico integrado de Logto no admite CSS ni HTML personalizado. Solo puedes modificar los elementos genéricos de marca. Esta restricción existe para mantener la estabilidad del servicio de correo electrónico integrado, ya que todos los inquilinos comparten la misma dirección IP y dirección de remitente. Para más detalles, consulta "[Factores para mejorar la entrega de correos electrónicos](https://blog.logto.io/verification-email-delivery#factors-to-improve-email-delivery)".

Para personalizar las plantillas de correo electrónico, recomendamos usar otro conector de correo, como AWS Direct Mail, SendGrid, Mailgun, Postmark o SMTP.

</details>
