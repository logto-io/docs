# Configura la verificación por correo electrónico con HTTP (Set up email verification via HTTP call)

El conector oficial de Logto para correo electrónico HTTP.

<GuideTip />

## Comenzar \{#get-started}

El conector de correo electrónico HTTP te permite enviar correos electrónicos mediante una llamada HTTP. Para usar el conector de correo electrónico HTTP, necesitarás tener tu propio servicio de correo electrónico que exponga una API HTTP para enviar correos electrónicos. Logto llamará a esta API cuando necesite enviar un correo electrónico. Por ejemplo, cuando un usuario se registra, Logto llamará a la API HTTP para enviar un correo de verificación.

## Configura el conector de correo electrónico HTTP \{#set-up-http-email-connector}

Para usar el conector de correo electrónico HTTP, necesitas configurar un endpoint HTTP que Logto pueda llamar. Y un token de autorización opcional para el endpoint.

## Carga útil (Payload) \{#payload}

El conector de correo electrónico HTTP envía una carga útil al endpoint al activar un correo electrónico. Incluye el correo electrónico del destinatario (`to`), el [tipo de uso del correo electrónico](/connectors/email-connectors/email-templates#email-template-types) que indica el flujo de autenticación (`type`), un `payload` que contiene las [variables](/connectors/email-connectors/email-templates#email-template-variables) disponibles (por ejemplo, código de verificación, enlace, información de la aplicación, información de la organización, información del usuario) y la [configuración regional](/connectors/email-connectors/email-templates#email-template-localization) del usuario, y un campo opcional `ip` que contiene la dirección IP del cliente (si está disponible).

Ejemplo:

```json
{
  "to": "foo@logto.io",
  "type": "SignIn",
  "payload": {
    "code": "123456",
    "locale": "de"
  },
  "ip": "192.168.1.1"
}
```

El campo `ip` puede usarse para limitación de tasa, detección de fraude o registro mejorado.

Puedes usar esta información para personalizar las plantillas de correo electrónico en tu proveedor de entrega de correos.

Para ver todos los tipos disponibles, consulta [Plantilla de correo electrónico](/connectors/email-connectors/email-templates#email-template-types). La definición completa del tipo `SendMessageData` está en [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts).
