# Configura la verificación por SMS con HTTP SMS

El conector oficial de Logto para HTTP SMS.

<GuideTip />

## Comenzar \{#get-started}

El conector HTTP SMS te permite enviar mensajes SMS mediante una llamada HTTP. Para usar el conector HTTP SMS, necesitarás tener tu propio servicio de SMS que exponga una API HTTP para enviar mensajes SMS. Logto llamará a esta API cuando necesite enviar un SMS. Por ejemplo, cuando un usuario se registre, Logto llamará a la API HTTP para enviar un SMS de verificación.

## Configura el conector HTTP SMS \{#set-up-http-sms-connector}

Para usar el conector HTTP SMS, necesitas configurar un endpoint HTTP que Logto pueda llamar, y un token de autorización opcional para el endpoint.

:::tip

Ten en cuenta que, para evitar errores en el flujo de autenticación, el `endpoint` configurado debe devolver una respuesta 2xx después de recibir el webhook para informar a Logto que ha recibido la notificación para enviar el SMS.

Mientras tanto, en este escenario, necesitas monitorear el servicio de SMS para asegurar la entrega exitosa del SMS. Alternativamente, puedes agregar monitoreo a tu API de envío de SMS para detectar rápidamente fallos en la entrega de SMS.

:::

## Payload \{#payload}

El conector HTTP SMS enviará el siguiente payload al endpoint cuando necesite enviar un SMS:

```json
{
  "to": "+1234567890",
  "type": "SignIn",
  "payload": {
    "code": "123456"
  }
}
```

Puedes encontrar todos los tipos en [plantillas de SMS](/connectors/sms-connectors/sms-templates), y la definición completa del tipo `SendMessageData` en [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts).
