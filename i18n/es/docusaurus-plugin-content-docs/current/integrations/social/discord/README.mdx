# Configura el inicio de sesión social con Discord (OAuth 2.0)

El conector de Discord proporciona una manera para que tu aplicación use Discord como un sistema de autorización.

<GuideTip />

## Registra una aplicación de desarrollador \{#register-a-developer-application}

- Visita el [Portal de Desarrolladores de Discord](https://discord.com/developers/applications) e inicia sesión con tu cuenta de Discord.
- Haz clic en el botón **New Application** para crear una aplicación, elige un nombre para ella (Ej: LogtoAuth), marca la casilla y haz clic en **Create**.
- Ve a la página de **OAuth2** y haz clic en **Reset Secret**.
- Toma nota de los campos **CLIENT ID** y **CLIENT SECRET**.
- Añade los redireccionamientos válidos (Ej: **`http://auth.mycompany.io/callback/${connector_id}`**). El `connector_id` se puede encontrar en la barra superior de la página de detalles del conector en la Consola de Administración de Logto.

## Configura Logto \{#configure-logto}

### Tipos de configuración \{#config-types}

| Nombre       | Tipo   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |

#### clientId \{#clientid}

`clientId` es el campo `CLIENT ID` que guardamos anteriormente.
(Puedes encontrarlo en la página de OAuth2 en el Portal de Desarrolladores de Discord.)

#### clientSecret \{#clientsecret}

`clientSecret` es el `CLIENT SECRET` que guardamos anteriormente.
(Si lo has perdido, necesitas hacer clic en **Reset Secret**)

#### scope \{#scope}

`scope` son los permisos otorgados por la autorización del usuario. El valor predeterminado es `identify email`.
Puedes ver la lista completa de alcances [aquí](https://discord.com/developers/docs/topics/oauth2#shared-resources-oauth2-scopes).
