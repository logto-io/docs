# Logto OSS

Logto ofrece versiones tanto de código abierto (OSS) como [cloud](/logto-cloud). Esta sección se centrará en la solución OSS: <Url href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS es una solución de identidad gratuita que proporciona a los desarrolladores capacidades de personalización total y control completo sobre sus datos. Logto OSS ofrece la flexibilidad de adaptar la base de código para satisfacer necesidades específicas; es ideal para quienes priorizan la soberanía de los datos y desean una solución de proveedor de identidad personalizada que pueda crecer junto con su aplicación.

## Funcionalidades soportadas por Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS admite la mayoría de las capacidades principales del servicio Logto y se actualiza y mantiene regularmente.

Algunas funciones avanzadas son actualmente exclusivas de la versión Logto Cloud, incluyendo:

| Limitaciones de funciones en OSS                                                                         | Descripción                                                                                                                                                                                                |
| -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Consola Logto: múltiples tenants](/logto-cloud/tenant-settings)                                         | Crea y gestiona múltiples tenants de Logto en una consola.                                                                                                                                                 |
| [Consola Logto: invitación de colaboradores](/logto-cloud/tenant-member-management)                      | Invita y gestiona múltiples miembros de un tenant de Logto.                                                                                                                                                |
| Consola Logto: Habilitar MFA                                                                             | Mejora la seguridad requiriendo autenticación multifactor (MFA) al iniciar sesión en la Consola Logto.                                                                                                     |
| [Logto Protected App](/integrate-logto/protected-app)                                                    | Integración rápida y sin SDK de autenticación impulsada por Cloudflare.                                                                                                                                    |
| [Servicio de correo electrónico integrado de Logto](/connectors/email-connectors/built-in-email-service) | Utiliza el servicio de correo electrónico integrado y gratuito para el envío de emails.                                                                                                                    |
| [Trae tu UI](/customization/bring-your-ui)                                                               | Personaliza la interfaz o los flujos de la experiencia de inicio de sesión usando esta función (OSS puede bifurcar el código en GitHub para personalizar la experiencia de inicio de sesión directamente). |
| [SSO iniciado por IdP](/end-user-flows/enterprise-sso/idp-initiated-sso)                                 | Habilita el inicio de sesión único empresarial iniciado por el proveedor de identidad (IdP).                                                                                                               |
| [Aplicaciones SAML](/integrate-logto/saml-app)                                                           | Integra aplicaciones mediante el protocolo SAML. La versión OSS está limitada a 3 aplicaciones SAML.                                                                                                       |

Consejos: La multi-tenencia, invitaciones de miembros y MFA no están disponibles para que tu equipo inicie sesión en una consola Logto de código abierto. Sin embargo, puedes implementar estas funciones en tu propio producto usando Logto OSS, poniéndolas a disposición de tus usuarios finales.
