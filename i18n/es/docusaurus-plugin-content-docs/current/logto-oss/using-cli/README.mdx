# Logto CLI

Logto CLI proporciona una manera conveniente de instalar, mantener y actualizar tu instancia de Logto sin manejar manualmente esas cosas técnicas aburridas.

:::note
Actualmente no distribuimos binarios, por lo que se requiere Node.js `^18.12.0` en el entorno.
:::

### Instalar CLI globalmente

Instala a través de tu gestor de paquetes preferido para usar el comando `logto` globalmente:

<Tabs>

  <TabItem value="npm" label="npm">

```bash
npm i @logto/cli -g
```

  </TabItem>
  <TabItem value="yarn" label="Yarn">

```bash
yarn global add @logto/cli
```

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

```bash
pnpm add -g @logto/cli
```

  </TabItem>

</Tabs>

Siempre puedes añadir `--help` para mostrar la información de ayuda de cada comando:

```bash
logto --help
logto db seed --help
```

### CLI local

Una versión incorporada de CLI se incluye con cada lanzamiento de Logto desde la v1.0.0-beta.11. Supongamos que ya tienes una instancia de Logto en `~/logto`, entonces puedes ejecutar los siguientes comandos:

```bash
cd ~/logto
npm run cli # Invocar el CLI local
```

Ten en cuenta que si deseas ejecutar con opciones, necesitas anteponer un `--` ANTES de las opciones para Logto CLI, por ejemplo:

```bash
npm run cli db seed -- --db-url postgresql://your-database-url
```

### Usar CLI con `npx`

Otra forma de usar Logto CLI es `npx`, que puede ejecutar un paquete sin instalación.

```bash
npx @logto/cli
```

Esto será útil para invocaciones puntuales, por ejemplo:

```bash
npx @logto/cli db seed --db-url postgresql://your-database-url
```
