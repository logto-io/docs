# Социальные коннекторы

Упростите процесс регистрации пользователей и увеличьте коэффициент конверсии, включив [социальный вход](/end-user-flows/sign-up-and-sign-in/social-sign-in) с Logto. Пользователи могут быстро и безопасно войти, используя свои существующие учетные записи в социальных сетях, исключая необходимость создания пароля или сложного процесса регистрации. Logto предлагает различные предустановленные социальные коннекторы и поддерживает пользовательские интеграции для максимальной гибкости.

Если наши стандартные коннекторы не соответствуют вашим конкретным требованиям, не стесняйтесь [связаться с нами](https://logto.productlane.com/roadmap). Для пользователей Logto OSS вы можете [реализовать ваш коннектор (OSS)](/logto-oss/develop-your-connector/implement-connectors#build-a-social-connector), если это требование срочное. Мы всегда приветствуем [вклад](/logto-oss/contribution); ваши усилия могут помочь другим членам сообщества с такими же потребностями.

## Шаги настройки \{#configuration-steps}

1. Перейдите в <CloudLink to="/connectors/social">Консоль > Коннекторы > Социальные коннекторы</CloudLink>.
2. Нажмите "Добавить социальный коннектор" и выберите нужный тип.
3. Следуйте руководству README и заполните необходимые поля и настройки.
4. Нажмите "Сохранить и завершить", чтобы закончить.
5. Проверьте коннектор, инициировав социальный вход.

Обратите внимание на следующие настройки:

- **Имя провайдера идентификации**: Каждый социальный коннектор имеет уникальное имя провайдера идентификации (IdP) для различения идентичностей пользователей. В то время как общие коннекторы используют фиксированное имя IdP, пользовательские коннекторы требуют уникального значения. Узнайте больше о [именах IdP](/connectors/connector-data-structure#target-identity-provider-name) для получения дополнительной информации.
- **Синхронизация профилей пользователей**: Выберите, когда синхронизировать [информацию профиля пользователя](/user-management/user-data#social-identities) (например, аватар, имя пользователя). По умолчанию синхронизация происходит "только при регистрации". "Синхронизация при каждом входе" является альтернативой, но может перезаписать пользовательские данные.

## Включение социального входа \{#enable-social-sign-in}

После успешного создания социального коннектора вы можете включить его в качестве кнопки социального входа (например, Продолжить с Google) в Опыт входа.

1. Перейдите в <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Консоль > Опыт входа > Регистрация и вход</CloudLink>.
2. (Необязательно) Выберите "Не применимо" для идентификатора регистрации, если вам нужен только социальный вход.
3. Добавьте настроенные социальные коннекторы в раздел "Социальный вход".
4. Переставьте коннекторы по мере необходимости.
5. Нажмите "Сохранить изменения" и протестируйте "Живой просмотр".

Обратитесь к [Социальный вход](/end-user-flows/sign-up-and-sign-in/social-sign-in), чтобы узнать подробности.
