# SMS коннекторы

Настройка SMS коннектора позволяет отправлять [одноразовые пароли (OTP)](https://auth.wiki/otp) на номер телефона пользователя. Этот механизм аутентификации без пароля может быть использован в различных сценариях, включая [регистрацию](/end-user-flows/sign-up-and-sign-in/sign-up), [вход](/end-user-flows/sign-up-and-sign-in/sign-in), [восстановление пароля](/end-user-flows/sign-up-and-sign-in/reset-password), [процессы связывания аккаунтов](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking), [приглашения участников](/end-user-flows/organization-experience/invite-organization-members) и [проверку личности пользователя](/end-user-flows/security-verification). Это упрощает аутентификацию пользователей и повышает безопасность, минимизируя риск утечек, связанных с паролями.

## Шаги настройки \{#configuration-steps}

Следуйте приведенным ниже шагам, чтобы добавить SMS коннектор:

1. Перейдите в <CloudLink to="/connectors/passwordless">Консоль > Коннектор > Коннекторы Email и SMS</CloudLink>.
2. Чтобы добавить новый SMS коннектор, нажмите кнопку "Настроить" и выберите желаемого SMS провайдера.
3. Ознакомьтесь с документацией README для выбранного провайдера.
4. Заполните обязательные поля в разделе "Конфигурация параметров".
5. Настройте шаблон SMS с помощью JSON редактора.
6. Проверьте вашу конфигурацию, отправив код подтверждения на ваш телефон, используя шаблон "Generic".
7. Нажмите "Сохранить и завершить", чтобы закончить.

## Изменение SMS коннектора \{#change-sms-connector}

Чтобы изменить SMS коннектор:

- Перейдите на страницу деталей коннектора и нажмите "Изменить коннектор" в правом верхнем углу
- Или удалите существующий коннектор и создайте новый

Примечание: Вы можете активировать ТОЛЬКО ОДИН _SMS коннектор_.

Когда у вас уже есть SMS коннектор, вы можете выбрать только "Изменить SMS коннектор". При выполнении этой операции в консоли, она вступает в силу сразу после сохранения, поэтому обязательно протестируйте вашу конфигурацию перед сохранением.

Чтобы обеспечить стабильность вашего сервиса, пользователи Logto Cloud могут создать [dev tenant](/logto-cloud/tenant-settings#development) для среды разработки. После тестирования конфигурации в среде разработки, "мигрируйте" конфигурацию в [production](/logto-cloud/tenant-settings#production), а затем сохраните настройку.

## Включение регистрации или входа по номеру телефона \{#enable-phone-number-sign-up-or-sign-in}

После успешного добавления SMS коннектора и завершения конфигурации, вы можете включить настройки входа и регистрации, которые зависят от телефона + кода подтверждения, в соответствии с вашими бизнес-потребностями.

1. **Перейдите в**: <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Консоль > Опыт входа > Регистрация и вход</CloudLink>
2. [**Установите методы регистрации**](/end-user-flows/sign-up-and-sign-in/sign-up): Выберите "Номер телефона" или "Адрес электронной почты или номер телефона" в качестве идентификатора регистрации. Примечание: Верификация номера телефона в настоящее время обязательна при регистрации.
3. [**Установите методы входа**](/end-user-flows/sign-up-and-sign-in/sign-in): Выбранный идентификатор регистрации автоматически становится методом входа. Нажмите "Добавить другой" для дополнительных опций входа.
4. **Сохраните изменения и протестируйте:** Нажмите "Сохранить изменения", чтобы внедрить новый опыт входа. Не забудьте использовать "[Живой предпросмотр](/customization/live-preview)", чтобы протестировать функциональность.

:::warning
Поскольку наши конфигурации вступают в силу немедленно, удаление существующего SMS коннектора может повлиять на нормальные процессы входа и регистрации конечных пользователей.

Например, если идентификатор входа включает комбинацию "номер телефона + код подтверждения", удаление существующего SMS коннектора предотвратит получение пользователями кодов подтверждения по электронной почте и приведет к ошибке в процессе входа.

Пожалуйста, будьте внимательны к потенциальным рискам при работе с коннекторами.
:::
