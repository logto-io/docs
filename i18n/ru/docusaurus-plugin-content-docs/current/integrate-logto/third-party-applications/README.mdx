# Сторонние приложения

Интеграция сторонних приложений Logto позволяет использовать Logto в качестве [провайдера идентификации (IdP)](https://auth.wiki/identity-provider) для внешних приложений.

Провайдер идентификации (IdP) — это сервис, который проверяет идентичность пользователей и управляет их учетными данными для входа. После подтверждения идентичности пользователя IdP генерирует токены аутентификации или утверждения и позволяет пользователю получить доступ к различным приложениям или сервисам без необходимости повторного входа.

В отличие от приложений, которые вы создали в руководстве [Интеграция Logto в ваше приложение](/integrate-logto/integrate-logto-into-your-application), которые разработаны и полностью контролируются вами, сторонние приложения — это независимые сервисы, разработанные внешними разработчиками или бизнес-партнерами.

Этот подход к интеграции хорошо подходит для общих бизнес-сценариев. Вы можете позволить пользователям получать доступ к приложениям партнеров, используя свои учетные записи Logto, так же как корпоративные пользователи входят в Slack с помощью Google Workspace. Вы также можете создать открытую платформу, где сторонние приложения могут добавить функцию "Войти с Logto", аналогичную "Войти с Google".

## Создание стороннего приложения в Logto \{#create-an-third-party-application-in-logto}

1. Перейдите в <CloudLink to="/applications">Консоль > Приложения</CloudLink>
2. Выберите "Стороннее приложение" в качестве типа приложения и выберите один из следующих протоколов интеграции:
   - OIDC / OAuth
3. Введите имя и описание для вашего приложения и нажмите кнопку «Создать». Будет создано новое стороннее приложение.

Все созданные сторонние приложения будут каталогизированы на странице Приложения во вкладке "Сторонние приложения". Это расположение помогает отличать их от ваших собственных приложений, облегчая управление всеми вашими приложениями в одном месте.
