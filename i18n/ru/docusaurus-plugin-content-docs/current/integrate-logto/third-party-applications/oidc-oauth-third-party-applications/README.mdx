# OIDC / OAuth сторонние приложения

Logto — это сервис идентификации, построенный на протоколе [OpenID Connect (OIDC)](https://auth.wiki/openid-connect), предоставляющий возможности как [аутентификации](https://auth.wiki/authentication), так и [авторизации](https://auth.wiki/authorization). Это делает интеграцию стороннего OIDC приложения такой же простой, как и традиционного веб-приложения.

Поскольку OIDC строится на основе [OAuth 2.0](https://auth.wiki/oauth-2.0), добавляя слой аутентификации, вы также можете интегрировать стороннее приложение, используя протокол OAuth.

## Настройка конфигураций OIDC \{#set-up-the-oidc-configurations}

:::note
Перед настройкой конфигураций OIDC, пожалуйста, убедитесь, что вы [создали стороннее OIDC приложение](/quick-starts/third-party-oidc).
:::

1. Укажите [**redirect URI**](/integrate-logto/application-data-structure#redirect-uris) вашего стороннего OIDC приложения. Это URL, на который стороннее приложение будет перенаправлять пользователей после их аутентификации в Logto.
   Обычно вы можете найти эту информацию на странице настроек подключения IdP стороннего приложения.

2. Получите [**client ID**](/integrate-logto/application-data-structure#application-id) и [**client secret**](/integrate-logto/application-data-structure#application-secret) со страницы деталей приложения Logto и введите их на странице настроек подключения IdP вашего поставщика услуг.

3. Получите [**authorization endpoint**](/integrate-logto/application-data-structure#authorization-endpoint) и [**token endpoint**](/integrate-logto/application-data-structure#token-endpoint) со страницы деталей приложения Logto и предоставьте их вашему поставщику услуг.
   Если ваш поставщик услуг поддерживает OIDC discovery, вы можете просто скопировать **discovery endpoint** со страницы деталей приложения Logto и предоставить его вашему поставщику услуг. Поставщик услуг сможет автоматически получить всю актуальную информацию об аутентификации OIDC с discovery endpoint.
   В противном случае нажмите кнопку **Show endpoint details**, чтобы просмотреть все точки аутентификации OIDC.

## Экран согласия для сторонних OIDC приложений \{#consent-screen-for-oidc-third-party-applications}

По соображениям безопасности все сторонние OIDC приложения будут перенаправлены на [экран согласия](/end-user-flows/consent-screen) для авторизации пользователя после их аутентификации в Logto.

Все запрашиваемые сторонними приложениями [разрешения профиля пользователя](/integrate-logto/third-party-applications/oidc-oauth-third-party-applications/permission-management#user-permissions-user-profile-scopes), [области действия ресурсов API](/integrate-logto/third-party-applications/oidc-oauth-third-party-applications/permission-management#api-resource-permissions-api-resource-scopes), [разрешения организации](/integrate-logto/third-party-applications/oidc-oauth-third-party-applications/permission-management#organization-permissions-organization-scopes) и информация о членстве в организации будут отображены на экране согласия.

Эти запрашиваемые разрешения будут предоставлены сторонним приложениям только после того, как пользователь нажмет кнопку "Authorize".
