# SAML Приложение

Logto поддерживает интеграцию в качестве [Провайдера идентификации (IdP)](https://auth.wiki/identity-provider) с приложениями на основе протокола SAML [Поставщик услуг, SP](https://auth.wiki/service-provider). Через интеграцию SAML приложения вы можете предоставить корпоративным пользователям безопасный, стандартизированный опыт Единого входа (SSO).

## Введение \{#introduction}

SAML (Security Assertion Markup Language) — это основанный на XML открытый стандарт для обмена данными аутентификации и авторизации между сторонами. В интеграции SAML:

- **Logto как IdP**: Выступает в качестве центрального органа аутентификации, управляя идентификацией пользователей и выдавая утверждения SAML
- **Ваше приложение как SP**: Полагается на Logto для аутентификации пользователей и использует утверждения SAML для контроля доступа

### Как работает аутентификация SAML \{#how-saml-authentication-works}

Аутентификация SAML в Logto в основном следует потоку, инициированному SP, где процесс аутентификации начинается с вашего приложения (Поставщик услуг). Вот краткий обзор:

1. Пользователь пытается получить доступ к вашему приложению
2. Ваше приложение генерирует запрос SAML и перенаправляет пользователя в Logto
3. Пользователь проходит аутентификацию в Logto
4. Logto генерирует ответ SAML, содержащий информацию о пользователе
5. Ваше приложение проверяет ответ и предоставляет доступ

Для более подробного объяснения потоков аутентификации SAML и сравнения с другими протоколами, ознакомьтесь с нашим [руководством по потокам аутентификации](/integrate-logto/integrate-logto-into-your-application/understand-authentication-flow.mdx#saml-authentication-flow).

### Преимущества интеграции SAML \{#benefits-of-saml-integration}

- **Повышенная безопасность**: Шифрованная связь и цифровые подписи обеспечивают безопасный обмен данными
- **Упрощенный пользовательский опыт**: Пользователям нужно войти только один раз для доступа к нескольким приложениям
- **Снижение административной нагрузки**: Централизованное управление пользователями и контроль доступа
- **Готовность к корпоративному использованию**: Широко используется организациями для безопасной федерации идентификаций

## Основные функции \{#key-features}

- **Стандартизированная интеграция**: Полная поддержка протокола SAML 2.0, обеспечивающая совместимость с различными поставщиками услуг
- **Гибкое отображение атрибутов**: Поддержка пользовательского отображения атрибутов пользователей для удовлетворения различных требований к данным приложений
- **Безопасность и надежность**: Поддержка подписания и шифрования для защиты процесса аутентификации
- **Автоматическая настройка**: Поддержка быстрой настройки интеграции SAML через URL метаданных или файл

## Сценарии использования \{#use-cases}

Интеграция SAML приложения подходит для следующих сценариев:

- Корпоративные системы приложений, требующие Единого входа (SSO)
- Интеграция с сторонними сервисами, поддерживающими протокол SAML
- Требования к высокой безопасности и стандартизированным процессам аутентификации

## Создание SAML приложения в Logto \{#create-an-saml-application-in-logto}

1. Перейдите в <CloudLink to="/applications">Консоль > Приложения</CloudLink>
2. Выберите "Мои приложения" как тип приложения и выберите следующий протокол интеграции "SAML"
3. Введите имя и описание для вашего приложения и нажмите кнопку «Создать». Будет создано новое SAML приложение.

## Руководство по настройке \{#configuration-guide}

Чтобы начать использовать интеграцию SAML приложения, вам нужно выполнить следующие шаги:

1. [Настройка SAML приложения](/integrate-logto/saml-app/setup.mdx): Настройте основные параметры интеграции SAML
2. [Настройка отображения атрибутов](/integrate-logto/saml-app/attribute-mapping.mdx): Определите, как отображать атрибуты пользователей Logto в утверждениях SAML

После завершения настройки ваше приложение сможет безопасно аутентифицироваться и обмениваться данными с Logto через протокол SAML.
