# Организации

Функция организаций особенно эффективна в многопользовательских SaaS и [приложениях бизнес-бизнес (B2B)](/introduction/plan-your-architecture/b2b). В дополнение к индивидуальным пользователям, ваши клиенты также могут состоять из команд, организаций или целых компаний.

С этим основным элементом вы можете создать необходимые функции для [многопользовательских приложений](https://auth.wiki/multi-tenancy), такие как:

- Продукт, который может использоваться несколькими организациями.
- Предоставление членов организации на основе [приглашения](/end-user-flows/organization-experience/invite-organization-members) или [в режиме реального времени](/organizations/just-in-time-provisioning).
- [Контроль доступа](/authorization/organization-template), определяемый ролями, назначенными членам внутри организации.
- Связывание организаций с опытом [корпоративного единого входа (SSO)](/end-user-flows/enterprise-sso).

Термин "организация" также используется в других формах, таких как "рабочее пространство", "команда", "компания" и т. д. В Logto мы используем "организация" как общий термин для представления концепции многопользовательства.

## Соображения по планированию \{#planning-considerations}

При использовании организаций интеграция может потребовать различных уровней усилий, мы предлагаем вам сопоставить их, чтобы настроить вашу разработку.

- **Опыт организации**: Каков будет опыт организации? Может ли администратор организации создать свою организацию? Могут ли администраторы организаций приглашать членов? Планируете ли вы создать пользовательскую панель, которая позволит администраторам самостоятельно управлять информацией о членах их организации, связанной с идентификацией?
- **Роли и контроль доступа**: Нужно ли приложению, чтобы пользователи имели определенные роли в своих организациях?
- **Управление организациями через Logto Console и Management API:** Какую конфигурацию вы хотите настроить непосредственно в Logto Console и Management API?

Для получения дополнительных ресурсов по планированию изучите эти два раздела для идей и вдохновения.
