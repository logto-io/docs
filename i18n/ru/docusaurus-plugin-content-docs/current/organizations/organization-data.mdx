# Структура данных организации

На уровне организации есть несколько атрибутов, значения которых могут быть созданы, обновлены, прочитаны и удалены (CRUD).

## Идентификатор организации \{#organization-id}

_organization id_ — это уникальный идентификатор для каждой организации. Он полезен для реализации опыта входа на уровне организации и получения токенов организации.

## Имя \{#name}

_name_ поддерживает вход на уровне организации и может быть интегрировано в интерфейсы продуктов на уровне организации по мере необходимости.

## Описание \{#description}

Поле _description_ позволяет добавлять текст для помощи в идентификации и маркировке организации.

## Логотипы организации \{#organization-logos}

Чтобы динамически отображать логотип организации вашего клиента в процессе входа, вы можете загрузить логотипы организации на страницу настроек организации.

См. [логотипы для конкретной организации](/customization/match-your-brand/#organization-specific-branding) для получения более подробной информации.

## Пользовательские данные \{#custom-data}

_Custom data_ — это объект JSON, используемый для хранения дополнительной информации об организации. Это дает разработчикам гибкость в хранении дополнительных деталей об организации для целей разработки.

## Требуется ли MFA \{#is-mfa-required}

_isMfaRequired_ указывает, обязательна ли многофакторная аутентификация (MFA) для организации. Если установлено значение `true`, все участники должны пройти MFA во время входа, чтобы получить доступ к организации. Эта настройка политики безопасности настраивается на уровне организации.

См. [Управление организацией](/organizations/organization-management#require-mfa-for-organization-members) для получения более подробной информации.

## Дата создания \{#created-at}

_createdAt_ — это временная метка с часовым поясом, когда организация была создана.

## Идентификатор арендатора \{#tenant-id}

_tenantId_ идентифицирует арендатора, к которому принадлежит организация.
