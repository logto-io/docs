# Понимание работы организаций

Для получения целостного представления важно понять сущности, участвующие в функциях организации, и это основа для интеграции организаций в ваше приложение.

## Организация \{#organization}

Организация состоит из группы [пользователей (идентичностей)](/user-management/user-data). Она может представлять команды, бизнес-клиентов и партнерские компании, которые могут получить доступ к вашему приложению.

Введение организации как сущности важно, так как она не только группирует пользователей, но и предоставляет контекст для изоляции арендаторов в многопользовательских приложениях.

## Член организации \{#organization-member}

В Logto пользователь, который является членом организации, называется членом организации (т.е. членом) в контексте этой организации.

## Организационное M2M приложение \{#organization-m2m-application}

Организационное M2M приложение относится к [приложениям машина-машина](/quick-starts/m2m), которые связаны с организацией. Подобно [членам организации](#organization-member), организационные приложения ограничены в контексте организации.

Как только вы связываете приложение с организацией, приложению могут быть назначены [роли организации](/authorization/organization-template/configure-organization-template#create-organization-role). Это полезно, когда вы хотите предоставить доступ к ресурсам организации без взаимодействия.

:::note

Другие типы приложений не поддерживают ассоциацию с организацией и роли организации, так как они предназначены для взаимодействия с пользователем.

:::

Обратитесь к этому [разделу](/organizations/organization-management) для управления организациями, членами и приложениями.

## Шаблон организации \{#organization-template}

Шаблон организации разработан для нужд управления доступом в многопользовательских приложениях. Он включает в себя роли организации и разрешения организации. Ознакомьтесь с этим [шаблоном организации](/authorization/organization-template), чтобы узнать больше о дизайне управления доступом в организации.
