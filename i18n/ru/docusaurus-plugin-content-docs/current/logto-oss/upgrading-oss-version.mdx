# Обновление вашей версии OSS

Logto следует [semver](https://semver.org/), поэтому, если мы не упоминаем об этом в журнале изменений, вы можете смело обновлять Logto без изменения вашего кода или схемы базы данных. Вы можете найти список релизов в [Github Release](https://github.com/logto-io/logto/releases).

## Шаг 1: Обновление до новой версии \{#step-1-update-to-the-new-version}

Чтобы обновиться, следуйте соответствующему методу в зависимости от вашего развертывания:

- **Docker Image**: Загрузите последнюю версию образа.
- **NPM Package**: Обновитесь до последней версии.
- **Tar File**: Скачайте последний файл `.tar` со страницы релиза.

## Шаг 2: Применение изменений в базе данных \{#step-2-apply-database-alterations}

Используйте [Logto CLI](/logto-oss/using-cli/database-alteration/) для выполнения необходимых изменений в базе данных. Для беспрепятственного обновления нет необходимости останавливать производственное приложение. Logto гарантирует, что новая схема базы данных останется совместимой с существующим кодом, поэтому производство будет продолжать работать без перебоев.

## Шаг 3: Перезапуск приложения \{#step-3-restart-the-application}

После обновления кода и применения изменений в базе данных перезапустите ваше приложение, чтобы применить изменения. Убедитесь, что новая версия работает гладко и все компоненты функционируют как ожидалось.

**Совет**: Чтобы достичь обновления без простоя, рассмотрите возможность использования стратегии «swap», где вы постепенно переводите трафик со старой инстанции на новую. Таким образом, пользователи не испытывают никаких сбоев во время обновления.

Ваше обновление завершено!
