# Logto CLI

Logto CLI предоставляет удобный способ установки, поддержки и обновления вашей инстанции Logto без необходимости вручную заниматься скучными техническими задачами.

:::note
В настоящее время мы не поставляем бинарные файлы, поэтому в среде требуется Node.js `^18.12.0`.
:::

### Установка CLI глобально \{#install-cli-globally}

Установите через предпочитаемый менеджер пакетов, чтобы использовать команду `logto` глобально:

<Tabs>

  <TabItem value="npm" label="npm">

```bash
npm i @logto/cli -g
```

  </TabItem>
  <TabItem value="yarn" label="Yarn">

```bash
yarn global add @logto/cli
```

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

```bash
pnpm add -g @logto/cli
```

  </TabItem>

</Tabs>

Вы всегда можете добавить `--help`, чтобы показать справочную информацию по каждой команде:

```bash
logto --help
logto db seed --help
```

### Локальный CLI \{#local-cli}

Встроенная версия CLI поставляется с каждым выпуском Logto начиная с v1.0.0-beta.11. Предположим, у вас уже есть инстанция Logto в `~/logto`, тогда вы можете выполнить следующие команды:

```bash
cd ~/logto
npm run cli # Вызов локального CLI
```

Обратите внимание, если вы хотите выполнить с опциями, вам нужно добавить `--` ПЕРЕД опциями для Logto CLI, например:

```bash
npm run cli db seed -- --db-url postgresql://your-database-url
```

### Использование CLI через `npx` \{#use-cli-by-npx}

Другой способ использования Logto CLI — это `npx`, который может выполнять пакет без установки.

```bash
npx @logto/cli
```

Это будет полезно для одноразовых вызовов, например:

```bash
npx @logto/cli db seed --db-url postgresql://your-database-url
```
