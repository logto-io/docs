# Структура файлов коннектора

Прилагается дерево файлов, демонстрирующее, как организованы файлы коннекторов в проекте на TypeScript.

Файлы, название которых заканчивается на `^`, являются необязательными, остальные — обязательными.

```
ConnectorRootDir/
├── package.json
├── README.md
├── logo.svg
├── logo-dark.svg^
└── src/
    ├── index.ts
    ├── index.test.ts
    ├── types.ts
    ├── constant.ts
    ├── mock.ts^
    ├── utils.ts^
    ├── utils.test.ts^
    └── ...
```

Основываясь на структуре файлов коннектора, давайте рассмотрим каждый файл и поймем, как они работают вместе как единая система.

### package.json \{#packagejson}

`package.json` содержит конфигурации этого репозитория коннектора. Он будет объединен с общим `package.json`, который содержит общие зависимости и атрибуты.

### README.md \{#readmemd}

`README.md` — это подробное руководство по настройке конфигурации коннектора.

### logo.svg \{#logosvg}

`logo.svg` — это файл, который хранит логотип коннектора в формате векторного изображения.

### logo-dark.svg \{#logo-darksvg}

Файл `logo-dark.svg` содержит векторное изображение логотипа коннектора для темного режима.

:::note
Смотрите [логотип коннектора](/connectors/connector-data-structure/#logo), чтобы узнать больше о взаимосвязи между `logo.svg` и `logo-dark.svg`.
:::

### index.ts \{#indexts}

`index.ts` — это файл для реализации класса коннектора.

### index.test.ts \{#indextestts}

`index.test.ts` — это файл, содержащий соответствующие модульные тесты (UTs) для реализаций в `index.ts`.

### types.ts \{#typests}

`types.ts` — это место, где вы должны определять типы переменных.

### constant.ts \{#constantts}

`constant.ts` — это место, где вы должны размещать константы, связанные с коннектором, включая конечные точки, метаданные коннектора и т. д.

### mock.ts \{#mockts}

`mock.ts` — это файл, в который вы можете помещать макетные значения для тестирования.

### utils.ts \{#utilsts}

`utils.ts` — это файл, где разработчики размещают вспомогательные функции, а `utils.test.ts` должен содержать соответствующие UTs.
