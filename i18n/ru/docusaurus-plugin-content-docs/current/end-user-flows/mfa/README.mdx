# Многофакторная аутентификация

## Что такое MFA? \{#what-is-mfa}

[Многофакторная аутентификация (MFA)](https://auth.wiki/mfa) — это метод безопасности, который добавляет дополнительный уровень защиты во время процесса входа. Он требует от пользователей предоставления нескольких учетных данных для установления их цифровой идентичности.

Существует два основных типа аутентификации:

- **SFA/1FA (Однофакторная аутентификация)**: Это начальный метод входа, обычно требующий [имя пользователя / электронную почту / телефон и пароль](/end-user-flows/sign-up-and-sign-in/sign-in) или [социальный вход](/end-user-flows/sign-up-and-sign-in/social-sign-in).
- **MFA/2FA (Многофакторная аутентификация / Двухфакторная аутентификация)**: MFA требует как минимум два разных метода проверки для доступа к вашей учетной записи, эффективно укрепляя вашу защиту от несанкционированного доступа.

Факторы аутентификации — это меры, которые проверяют вашу личность. Существует множество факторов, классифицированных по атрибутам, из которых можно выбрать:

| Типы        | Что это значит | Факторы проверки (поддерживаемые Logto)                                                 |
| ----------- | -------------- | --------------------------------------------------------------------------------------- |
| Знание      | Что вы знаете  | Пароль, код проверки по электронной почте и резервные коды                              |
| Владение    | Что у вас есть | Код проверки по SMS, OTP приложения-аутентификатора, аппаратный OTP (ключ безопасности) |
| Присущность | Что вы есть    | Биометрия, такая как отпечатки пальцев, Face ID                                         |

В потоке MFA второй шаг аутентификации должен использовать другой тип атрибута (Знание / Владение / Присущность), чем первый. Например, использование "Пароль (Знание)" в качестве первого фактора и "OTP приложения-аутентификатора (Владение)" в качестве второго фактора может эффективно снизить различные векторы атак.

## Зачем нам нужна MFA? \{#why-do-we-need-an-mfa}

MFA является важной мерой безопасности, особенно для B2B и B2E сервисов. Она широко используется в современном цифровом ландшафте по нескольким причинам:

- **Взлом аккаунтов**: Несанкционированный доступ к аккаунтам остается значительной угрозой безопасности. Однако MFA предлагает надежную защиту, эффективно блокируя 99,9% взломов аккаунтов, особенно тех, которые происходят из-за утечек паролей. Это является экономически эффективным улучшением безопасности, дополненным стратегиями, такими как вход без пароля, надежные политики паролей, менеджеры паролей и защитные меры против атак.
- **Принятие SaaS**: Многие предприятия все чаще внедряют MFA для защиты своих сотрудников и обеспечения безопасности конфиденциальных данных и активов компании. Согласно опросу LastPass, 57% глобальных компаний теперь используют MFA, что отражает увеличение на 12% по сравнению с предыдущим годом.
- **Соответствие нормативным требованиям**: MFA помогает организациям поддерживать соответствие требованиям по защите данных, таким как GDPR и NIST, обеспечивая безопасность пользовательских данных.

## Поддержка Logto \{#logto-support}

Logto упрощает процесс активации MFA с помощью переключателя в один клик, устраняя необходимость в сложных настройках. Начните с нашего быстрого руководства по [включению факторов проверки](/end-user-flows/mfa/configure-mfa).

**Поддерживаемые факторы MFA**:

- [OTP приложения-аутентификатора](/end-user-flows/mfa/authenticator-app-otp): Используйте одноразовый пароль на основе времени (TOTP), сгенерированный приложением-аутентификатором, таким как Google Authenticator или Authy.
- [Ключи доступа (WebAuthn)](/end-user-flows/mfa/webauthn): Используйте ключ безопасности или биометрическую аутентификацию для входа без пароля.
- [Резервные коды](/end-user-flows/mfa/backup-codes): Генерируйте одноразовые резервные коды для экстренного доступа.
