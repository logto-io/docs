# Параметры аутентификации

В стандартном потоке входа OIDC клиентские приложения инициируют запрос аутентификации, который перенаправляет пользователя на размещенную Logto универсальную веб-страницу опыта входа. В зависимости от настроек вашего опыта входа, пользователи могут войти или зарегистрироваться, используя различные идентификаторы, методы проверки и сторонние социальные или корпоративные SSO коннекторы.

В дополнение к стандартным [параметрам аутентификации OIDC](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest), наш продукт вводит несколько пользовательских параметров аутентификации, которые позволяют настроить желаемый опыт входа для конечных пользователей.

Это особенно полезно, когда вы хотите внедрить определенные потоки входа для различных сегментов пользователей. Включая, но не ограничиваясь следующими сценариями:

1. **Прямая регистрация для новых пользователей**: Для целевой кампании по регистрации вы можете направить новых пользователей прямо на страницу регистрации, минуя форму входа по умолчанию, чтобы обеспечить беспрепятственный процесс адаптации.
2. **Регистрация с предварительно заполненным email из ссылки подписки**: Если вы собираете адреса электронной почты через размещенную форму подписки, эти параметры позволяют перенаправить новых подписчиков на страницу регистрации с предварительно заполненным полем email, что облегчает им завершение процесса регистрации.
3. **Прямая ссылка на сброс пароля**: Для новых пользователей, которым нужно установить новый пароль, отправьте им прямую ссылку и перенаправьте их на страницу сброса пароля с предварительно заполненным полем email.
4. **Мгновенный социальный вход**: Если вы предоставляете кнопки входа через социальные сети непосредственно на вашем сайте, вы можете использовать эти параметры, чтобы пропустить форму входа по умолчанию и позволить пользователям аутентифицироваться непосредственно через выбранного социального провайдера.
5. **Прямой вход через SSO для корпоративных клиентов**: Для продуктов, обслуживающих нескольких корпоративных клиентов, эти параметры позволяют отображать специальную кнопку SSO, которая позволяет пользователям входить через их конкретного SSO провайдера, минуя универсальную форму входа и обеспечивая беспрепятственный опыт для каждой организации.

Продолжайте чтение, чтобы узнать больше о параметрах аутентификации для настройки [первого экрана](/end-user-flows/authentication-parameters/first-screen) или включения [прямого входа](/end-user-flows/authentication-parameters/direct-sign-in) для ваших пользователей.
