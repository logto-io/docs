# Параметры первого экрана

Набор пользовательских параметров аутентификации, которые позволяют настроить желаемый опыт первого экрана для конечных пользователей.

- `first_screen`: Указывает первый экран, который увидит пользователь.
- `identifier`: Указывает типы идентификаторов, которые форма входа или регистрации будет принимать.
- `login_hint`: Заполняет поле идентификатора адресом электронной почты или именем пользователя. (Это стандартный параметр OIDC)

## first_screen \{#first_screen}

Параметр `first_screen` является ключевым параметром, который определяет первый экран, который пользователи увидят при перенаправлении на страницу входа Logto. По умолчанию будет отображаться универсальная форма входа. Используйте этот параметр для настройки первого экрана в зависимости от требований вашего приложения. Поддерживаемые значения:

- `sign_in`: Отображает форму входа. (По умолчанию)
- `register`: Отображает форму регистрации.
- `reset_password`: Отображает форму сброса пароля.
- `single_sign_on`: Отображает форму входа корпоративного SSO. (Будет запрошен адрес электронной почты для определения доступных провайдеров SSO)
- `identifier:sign-in`: Отображает форму входа с определенным идентификатором. Тип идентификатора можно указать с помощью параметра `identifier`. Это полезно, когда у вас включены несколько методов входа с идентификатором.
- `identifier:register`: Отображает форму регистрации с определенным идентификатором. Тип идентификатора можно указать с помощью параметра `identifier`. Это полезно, когда у вас включены несколько методов регистрации с идентификатором.

![Параметр первого экрана](./assets/first-screen-parameter.png)

Например, отправка пользователей непосредственно на форму входа корпоративного SSO:

```sh
curl --location \
--request GET 'https://<your-tenant>.logto.app/oidc/auth?client_id=<client_id>&...&first_screen=single_sign_on'
```

## identifier \{#identifier}

Параметр `identifier` используется для указания типов идентификаторов, которые форма входа или регистрации будет принимать. Этот параметр применим только тогда, когда параметр `first_screen` установлен на `identifier:sign-in`, `identifier:register` или `reset_password`. Поддерживаемые значения: `username`, `email` и `phone`. Разделяйте несколько значений пробелом, чтобы разрешить несколько типов идентификаторов.

Например, отправка пользователей непосредственно на страницу регистрации по электронной почте или номеру телефона:

```sh
curl --location \
--request GET 'https://<your-tenant>.logto.app/oidc/auth?client_id=<client_id>&...&first_screen=identifier:register&identifier=email phone'
```

Все типы идентификаторов, указанные в этом параметре, должны быть включены в ваших настройках входа или регистрации в Logto Console.

Любые неподдерживаемые или отключенные типы идентификаторов будут проигнорированы. Если все указанные идентификаторы не поддерживаются, будет использована конфигурация по умолчанию для опыта входа.

## login_hint \{#login_hint}

Параметр `login_hint`, определенный в стандарте [OpenID Connect](https://openid.net/specs/openid-connect-core-1_0.html#AuthorizationEndpoint), используется для предварительного заполнения формы входа идентификатором пользователя (например, электронной почтой, номером телефона или именем пользователя). В Logto он может быть комбинирован с другими параметрами экрана входа для улучшения пользовательского опыта. Этот параметр особенно полезен, если у вас есть пользовательская форма предварительной аутентификации, которая заранее собирает идентификатор пользователя, позволяя ему пропустить повторный ввод во время входа.

Например, предварительное заполнение собранного адреса электронной почты в форме входа:

```sh
curl --location \
--request GET 'https://<your-tenant>.logto.app/oidc/auth?client_id=<client_id>&...&first_screen=identifier:sign_in&identifier=email&login_hint=example@logto.io
```

## Поддержка SDK \{#sdk-support}

В поддерживаемых Logto SDK вы можете установить параметры при вызове метода `signIn`:

```javascript
logtoClient.signIn({
  redirectUri: 'https://your-app.com/callback',
  firstScreen: 'identifier:register',
  identifier: ['email', 'phone'],
  loginHint: 'example@logto.io',
});
```

:::note
Мы постепенно добавляем поддержку параметров `first_screen`, `identifier` и `login_hint` во все Logto SDK. Если вы не видите их в вашем SDK, пожалуйста, откройте проблему или свяжитесь с нами.

Для пользователей [Logto OSS](/logto-oss) эти параметры поддерживаются с версии 1.15.0. Если вы используете более старую версию, пожалуйста, [обновитесь](/logto-oss/upgrading-oss-version) до последней версии.
:::
