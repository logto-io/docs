# Корпоративная идентификация SSO

## Связывание учетных записей корпоративного SSO \{#enterprise-sso-account-linking}

**Новые пользователи входят с корпоративным SSO**

Когда новый пользователь регистрируется с новой корпоративной SSO идентификацией, Logto автоматически создаст новую учетную запись пользователя, связанную с корпоративной идентификацией. `основной email`, `имя` и `аватар` будут автоматически заполнены данными, предоставленными IdP. Другие дополнительные данные профиля пользователя будут сохранены в профиле идентификации SSO пользователя.

:::note
Ситуация с связыванием профилей может быть иной, если [отображение атрибутов SAML](/integrations/saml-sso#step-3-configure-user-attributes-mapping) настроено неправильно или если провайдер идентификации не предоставляет email пользователя.
:::

**Существующие пользователи входят с корпоративным SSO**

Если рабочий адрес электронной почты, связанный с корпоративной SSO идентификацией, совпадает с существующей учетной записью пользователя в Logto, Logto автоматически свяжет корпоративную SSO идентификацию с существующей учетной записью пользователя.

:::note
Как только домен электронной почты будет связан с корпоративным SSO коннектором, все существующие пользователи с указанным доменом электронной почты будут ограничены в возможности входа с использованием корпоративного SSO коннектора. Их предыдущие методы входа будут заблокированы. Например, email / пароль, код проверки email и методы социального входа.
:::

## Многофакторная аутентификация (MFA) с корпоративным SSO \{#multi-factor-authentication-mfa-with-enterprise-sso}

При использовании корпоративного SSO требования MFA обычно управляются IdP. В Logto все аутентифицированные идентификации от IdP считаются доверенными, поэтому [проверка MFA](/end-user-flows/mfa) обходится для пользователей, входящих через корпоративное SSO, чтобы улучшить пользовательский опыт. Важно убедиться, что защита MFA включена на стороне [IdP](/end-user-flows/enterprise-sso#key-components-of-enterprise-sso).

## Удаление корпоративного коннектора \{#deleting-an-enterprise-connector}

Когда вы удаляете корпоративный коннектор из Logto:

- **Учетные записи пользователей остаются**: Учетные записи пользователей не удаляются; только их связь с корпоративным провайдером идентификации удаляется.
- **Следующий вход пользователей**: При следующей попытке входа пользователи будут предложены использовать альтернативный метод, такой как стандартный метод входа, настроенный в Logto (например, email и пароль). Если они ранее не устанавливали пароль, им будет предложено создать его в этот момент.
- **Удаление профиля идентификации SSO пользователя**: Идентификация SSO пользователя, а также связанные данные профиля будут удалены из Logto.
