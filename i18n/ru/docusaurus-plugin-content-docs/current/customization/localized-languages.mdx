# Локализованные языки

Logto поддерживает широкий спектр предопределенных языков и предлагает 113 дополнительных языковых тегов. Этот мощный инструмент позволяет вам настраивать опыт входа, создавая и управляя своими собственными языковыми опциями и переводами.

## Шаги настройки в Logto Console \{#customization-steps-in-logto-console}

Легко настраивайте языковые настройки в Logto Console без программирования.

1. **Перейдите в**: <CloudLink to="/sign-in-experience/content">Console > Sign-in experience > Content > Languages</CloudLink>.
2. **Управление языком**: Нажмите кнопку «Управление языком», чтобы получить доступ к вашей языковой библиотеке.
   - **Редактирование существующих языков:** Настройте переводы предоставленных Logto языков. Эти языки нельзя удалить, но ваши изменения заменят значения по умолчанию.
   - **Добавление нового языка**: Нажмите кнопку «Добавить язык», выберите языковой тег, предоставьте свои переводы, затем сохраните изменения, чтобы добавить новый язык.
3. **Включить автоопределение**: При включении автоматически отображает страницу входа на предпочитаемом языке пользователя на основе настроек его устройства.
4. **Установить язык по умолчанию:** Вы можете выбрать язык по умолчанию из вашей языковой библиотеки. Он будет использоваться, когда обнаруженный язык пользователя не охвачен в текущей языковой библиотеке.

Вот некоторые ключевые термины, которые нужно понимать при управлении языками:

| Определение                 | Описание                                                                                                                                                                                                     |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Языковой тег                | Языковой тег идентифицирует язык контента. Языковой тег состоит из кода языка (например, en, fr, zh) и кода страны / региона (например, US, UK, KR), разделенных дефисами. Языковой тег выглядит так: en-US. |
| Язык, предоставленный Logto | Язык, предоставленный Logto, является официальным языком Logto и хранится в оригинальной кодовой базе Logto.                                                                                                 |
| Добавленный язык            | Добавленный язык — это язык, добавленный пользователями.                                                                                                                                                     |
| Исходные значения Logto     | Исходные значения Logto — это значения, предоставленные Logto, которые не были настроены пользователями.                                                                                                     |
| Пользовательские значения   | Пользовательские значения — это значения, которые уже были настроены пользователями. Исходные значения Logto будут перезаписаны.                                                                             |

## Настройка с использованием Management API \{#customization-using-management-api}

Вы можете использовать Management API [PUT /api/custom-phrases/\{languageTag\}](https://openapi.logto.io/operation/operation-replacecustomphrase) для настройки языковых переводов. Тело запроса API является частичным объектом локали, например:

```json
{
  "input": { "username": "Username", "password": "Password" },
  "secondary": {
    "social_bind_with": "Уже есть аккаунт? Войдите, чтобы связать {{methods, list(type: disjunction;)}} с вашей социальной идентичностью."
  },
  "action": { "sign_in": "Sign in" },
  "error": {
    "general_required": "{{types, list(type: disjunction;)}} требуется"
  },
  "list": { "or": "или" },
  "user_scopes": {
    "descriptions": { "custom_data": "Ваши пользовательские данные" }
  }
}
```

Обратитесь к [исходному коду](https://github.com/logto-io/logto/blob/master/packages/phrases-experience/src/locales/en/index.ts), чтобы увидеть все настраиваемые содержимое.

Вы также можете использовать [PATCH /api/sign-in-exp](https://openapi.logto.io/group/endpoint-sign-in-experience) API для управления [политиками определения языка](https://openapi.logto.io/operation/operation-getsigninexp#operation-getsigninexp-200-body-application-json-languageinfo).

## Примеры использования \{#use-cases}

Как добавленный язык отображается конечным клиентам?

Предположим, у вас есть веб-сайт, где английский является языком по умолчанию, и автоопределение включено. Пользователь из Японии заходит на ваш сайт и решает создать аккаунт. Если он / она использует японский как язык приложения, но Logto еще не поддерживает этот язык, экран входа будет отображаться на английском.

Опыт входа Logto i18n делает возможным использование настраиваемого языка.

Нажмите на языковой тег `ja`, чтобы добавить свой собственный японский перевод.

Таким образом, пользователь, заходящий на ваш сайт из Японии, сможет читать контент на японском, который вы только что перевели с английского.

## Часто задаваемые вопросы \{#faqs}

<details>
  <summary>

### Что если добавленный мной язык станет языком, предоставленным Logto? \{#what-if-the-language-i-added-becomes-logto-provided-language}

</summary>

Рядом с языковым тегом слева появится тег, предоставленный Logto, и добавленный вами язык больше не будет удаляемым. Измененные значения продолжают функционировать и заменяют оригинальные значения Logto. Удалите значения, предоставленные пользователем, чтобы использовать значения, предоставленные конфигурацией Logto по умолчанию.

</details>

<details>
  <summary>

### Что если я добавил только несколько пользовательских значений? \{#what-if-i-only-added-a-few-custom-values}

</summary>

То, что видят конечные пользователи, является результатом слияния двух колонок.
Предположим, что вы хотите внести изменения только в подмножество оригинальных копий контента, предоставленных Logto. Единственное отличие между вашим экраном регистрации и тем, который предоставлен Logto, будут ключи, которые вы отредактировали. Остальной контент останется неизменным.

</details>
