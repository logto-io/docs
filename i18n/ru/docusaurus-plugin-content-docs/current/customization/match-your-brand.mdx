# Соответствие вашему бренду

## Универсальный опыт входа \{#omni-sign-in-experience}

Настройте внешний вид вашей страницы входа, перейдя в <CloudLink to="/sign-in-experience/branding">Консоль > Опыт входа > Брендинг</CloudLink>. Этот раздел позволяет легко настроить ключевые элементы брендинга.

**Цвет бренда**

Определите основной цвет, используемый в процессе входа, включая основные кнопки, ссылки и другие компоненты. Замените стандартный фиолетовый цвет Logto на цвет вашего бренда. Для темного режима можно указать отдельный цвет бренда.

**Логотип компании**

Логотип будет отображаться на главной странице входа, странице регистрации, странице загрузки и других интерфейсах с нашим расширением.

- Существуют некоторые ограничения для изображений: они должны быть менее 500KB и в формате SVG, PNG, JPG, JPEG или ICO.
- Если оставить поле логотипа пустым, логотип не будет отображаться в интерфейсе.
- Также можно загрузить версию логотипа для темного режима.

**Фавикон**

Фавикон — это маленькая иконка, представляющая веб-сайт, и отображается на вкладке браузера, в закладках и других областях интерфейса браузера.

- Изображение должно быть менее 500KB и в формате SVG, PNG, JPG, JPEG или ICO. Рекомендуется загружать квадратное изображение для обеспечения хорошего эффекта отображения.
- Также можно загрузить версию фавикона для темного режима.
- Кроме того, теперь вместо пользовательского заголовка используется заголовок браузера для различных потоков (Вход / Регистрация / Забыли пароль и т. д.).

**Темный режим**

Включите темный режим, чтобы автоматически настроить внешний вид страницы входа в зависимости от предпочтений системы пользователя.

## Брендинг для конкретного приложения \{#app-specific-branding}

Если вашему бизнесу с несколькими приложениями требуется опыт входа на уровне приложения, вы можете настроить это на странице сведений о приложении. Перейдите в <CloudLink to="/applications">Консоль > Приложения</CloudLink>.

Включив "опыт входа на уровне приложения", вы можете настроить конкретный брендинг и цвета для каждого приложения. Когда вход инициируется из приложения с включенным брендингом на уровне приложения, опыт входа будет настроен в соответствии с настройками брендинга на уровне приложения; в противном случае он будет по умолчанию использовать настройки универсального опыта входа.

Доступны настройки как для светлого, так и для темного режима для брендинга на уровне приложения. Настройки темного режима будут действовать только тогда, когда темный режим включен в настройках [универсального опыта входа](#omni-sign-in-experience).

## Брендинг для конкретной организации \{#organization-specific-branding}

Чтобы динамически отображать логотип вашей клиентской организации в процессе входа, вы можете загрузить логотипы организаций на странице настроек организации. Перейдите в <CloudLink to="/organizations">Консоль > Организации</CloudLink>.

Затем, при инициировании входа, вы можете передать ID организации в параметре `organization_id`, чтобы указать Logto, какой логотип организации отображать. Например, чтобы показать логотип организации для ID организации `123456`:

- Если вы используете Logto SDK, вы можете передать параметр `organization_id` в объекте `extraParams` метода `signIn`.
- Если вы используете OIDC клиент, вы можете передать параметр `organization_id` при запросе [authorization endpoint](https://openid.net/specs/openid-connect-core-1_0.html#AuthorizationEndpoint).

Доступны настройки как для светлого, так и для темного режима для брендинга на уровне организации. Настройки темного режима будут действовать только тогда, когда темный режим включен в настройках [универсального опыта входа](#omni-sign-in-experience).

Вот пример того, как передать параметр `organization_id` в методе `signIn` с использованием [Logto browser SDK](/quick-starts/vanilla-js/#implement-sign-in-and-sign-out):

**index.ts**

```tsx
logtoClient.signIn({
  // ...другие параметры
  redirectUri: 'https://your-redirect-uri',
  extraParams: {
    organization_id: '123456',
  },
});
```

:::note

Мы постепенно внедряем функцию `extraParams` во все Logto SDK. Если вы еще не видите ее в вашем SDK, пожалуйста, свяжитесь с нами.

:::
