# Принесите свой интерфейс

Logto позволяет вам использовать свой собственный интерфейс для полной замены встроенного интерфейса опыта входа, в дополнение к предустановленным опциям кастомизации интерфейса. Эта функция позволяет вам загрузить zip-файл, содержащий ваши пользовательские UI-ресурсы (HTML, CSS, JavaScript, изображения и т. д.), разместить его на серверах [Logto Cloud](https://cloud.logto.io) и использовать его в качестве [опыта входа](/end-user-flows#authentication-flows) для пользователей вашего арендатора.

:::note

Эта функция требует, чтобы ваш пользовательский интерфейс был совместим с одностраничными приложениями (SPA), и рекомендуется всегда использовать производственную сборку для оптимальной производительности.

:::

## Изучите функцию пользовательского интерфейса в Logto Console \{#explore-the-custom-ui-feature-in-logto-console}

### Начните с официального проекта \{#get-started-with-official-project}

Самый быстрый способ начать с пользовательского интерфейса входа — это клонировать [проект опыта Logto](https://github.com/logto-io/logto/tree/master/packages/experience). Это встроенный интерфейс опыта входа Logto, который охватывает все функции и лучшие практики Logto. Вы можете настроить его в соответствии с вашими потребностями.

После проверки кода просто выполните следующую команду для сборки проекта:

```bash
pnpm install && pnpm build
```

:::note

Весь проект Logto является [pnpm монорепозиторием](https://pnpm.io/workspaces). Если вы хотите запустить пакет опыта отдельно (без pnpm), замените `workspace:` _зависимости в package.json на конкретные номера версий (например, `^1.0.0`)._

:::

После завершения сборки вы найдете все скомпилированные ресурсы в каталоге `dist`. Затем создайте ZIP-архив каталога `dist` - этот архив будет использоваться для загрузки в Logto Cloud позже.

### Требования к пользовательским UI-ресурсам \{#custom-ui-assets-requirements}

Перед загрузкой ваших пользовательских UI-ресурсов убедитесь, что они соответствуют следующим требованиям:

- Загружаемые ресурсы должны быть упакованы в один zip-файл.
- Zip-файл должен содержать файл `index.html` в корневом каталоге.
- Zip-файл не должен превышать 20 МБ.
- Zip-файл не должен содержать файлы, превышающие 10 МБ.
- Zip-файл не должен содержать более 200 файлов в общей сложности.

### Загрузите ваши пользовательские UI-ресурсы \{#upload-your-custom-ui-assets}

:::note

Будьте осторожны при использовании этой функции в производственной среде, так как она немедленно повлияет на опыт входа ваших пользователей.

:::

1. Перейдите в <CloudLink to="/sign-in-experience/branding">Консоль > Опыт входа > Брендинг > Принесите свой интерфейс</CloudLink>.
2. Нажмите, чтобы выбрать или перетащить ваш zip-файл, созданный на предыдущем шаге, и процесс загрузки начнется автоматически.
3. После завершения загрузки сохраните изменения, и ваш пользовательский интерфейс будет немедленно обслуживаться.
4. Окно "Предварительный просмотр входа" будет отключено, когда вы используете свой пользовательский интерфейс входа. Однако вы все равно можете нажать кнопку "[Живой предварительный просмотр](/customization/live-preview)", чтобы протестировать вашу пользовательскую страницу входа в новой открытой вкладке браузера.

## Разработка вашего пользовательского интерфейса \{#develop-your-custom-ui}

### Взаимодействие с Experience API \{#interact-with-experience-api}

Ваш пользовательский интерфейс должен взаимодействовать с [Experience API](https://openapi.logto.io/group/endpoint-experience) для выполнения различных действий, таких как вход, регистрация, сброс пароля, привязка социальных аккаунтов, включение MFA и многое другое. Чтобы лучше понять пользовательские потоки и детали реализации, мы рекомендуем ознакомиться с нашим [Experience API design RFC](https://github.com/logto-io/rfcs/blob/master/draft/0004-experience-api.md), который предоставляет полные технические спецификации и примеры.

Вам также необходимо получить доступ к другим настройкам опыта входа, таким как цвета брендинга, логотип компании, фавикон, политика паролей, локализованные языковые фразы и даже пользовательский CSS, через [endpoint опыта входа API](https://openapi.logto.io/operation/operation-getsigninexperienceconfig).

### Примеры проектов \{#sample-projects}

Мы предоставляем коллекцию примеров проектов, чтобы помочь вам быстро понять и начать реализовывать пользовательский интерфейс. Пожалуйста, ознакомьтесь с репозиторием GitHub [Logto Experience project](https://github.com/logto-io/experience-samples) для получения подробной информации.

Кроме того, мы продолжим работать над предоставлением более упрощенных и основанных на сценариях примеров проектов, чтобы охватить наиболее распространенные случаи использования. Следите за нашими будущими обновлениями!

### Локальная разработка и отладка \{#local-development-and-debugging}

Для локальной разработки и отладки мы предоставляем инструмент Logto Tunnel CLI для:

- Прокси-запросов Experience API с вашего локального компьютера на конечную точку Logto Cloud.
- Тестирования вашей реализации пользовательского интерфейса локально.

Это позволяет пользователям тестировать и отлаживать пользовательский интерфейс локально перед загрузкой в Logto Cloud.

Пожалуйста, обратитесь к [Отладка и тестирование вашего пользовательского интерфейса локально](/customization/bring-your-ui/debug-and-test-your-custom-ui-locally/) для получения более подробной информации.

## Загрузите ваши пользовательские UI-ресурсы с помощью CLI \{#upload-your-custom-ui-assets-using-the-cli}

В дополнение к загрузке через Консоль, вы также можете использовать Logto CLI для загрузки пользовательских UI-ресурсов. Это особенно полезно для автоматизированных рабочих процессов развертывания.

Пожалуйста, обратитесь к [Загрузка пользовательских UI-ресурсов с помощью CLI](/customization/bring-your-ui/upload-custom-ui-assets-using-cli/) для получения более подробной информации.

## Восстановление встроенного опыта входа Logto \{#restore-to-logto-built-in-sign-in-experience}

Если вы хотите восстановить встроенный опыт входа Logto, просто нажмите кнопку удаления на карточке "Принесите свой интерфейс". После сохранения изменений интерфейс опыта входа будет возвращен к стандартному интерфейсу Logto.
