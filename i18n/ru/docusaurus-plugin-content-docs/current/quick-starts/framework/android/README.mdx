В Android перенаправление URI следует шаблону: `$(LOGTO_REDIRECT_SCHEME)://$(YOUR_APP_PACKAGE)/callback`:

- `LOGTO_REDIRECT_SCHEME` должен быть пользовательской схемой в формате обратного домена.
- `YOUR_APP_PACKAGE` — это имя пакета вашего приложения.

Предполагая, что вы используете `io.logto.android` как пользовательскую `LOGTO_REDIRECT_SCHEME`, а `io.logto.sample` — это имя пакета вашего приложения, URI перенаправления должен быть `io.logto.android://io.logto.sample/callback`.

## Получение информации о пользователе \{#get-user-information}

### Отображение информации о пользователе \{#display-user-information}

Чтобы отобразить информацию о пользователе, вы можете использовать метод `logtoClient.getIdTokenClaims()`. Например, вы можете получить информацию о пользователе в ViewModel и затем отобразить её в вашей активности:

```kotlin title="LogtoModelView.kt"
class LogtoViewModel(application: Application) : AndroidViewModel(application) {
    // ...другие коды

    // Добавьте живые данные для наблюдения за утверждениями ID токена
    private val _idTokenClaims = MutableLiveData<IdTokenClaims>()
    val idTokenClaims: LiveData<IdTokenClaims>
        get() = _idTokenClaims

    fun getIdTokenClaims() {
        logtoClient.getIdTokenClaims { logtoException, idTokenClaims ->
            logtoException?.let { _logtoException.postValue(it) } ?: _idTokenClaims.postValue(idTokenClaims)
        }
    }
}
```

```kotlin title="MainActivity.kt"
//...с другими импортами
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        //...другие коды

        // Предположим, у вас есть текстовое представление с id `user_info_text_view` в вашем макете
        val userInfoResponseTextView: TextView = findViewById(R.id.user_info_text_view)
        logtoViewModel.userInfoResponse.observe(this) { userInfoResponse ->
            userInfoResponseTextView.text = if (userInfoResponse !== null) {
                val json = Gson().toJson(userInfoResponse, UserInfoResponse::class.java)
                JSONObject(json).toString(2)
            } else {
                ""
            }
        }
    }
}
```

### Запрос дополнительных утверждений \{#request-additional-claims}

Чтобы запросить дополнительные области действия, вы можете передать их в объект `LogtoConfig`. Например:

```kotlin title="LogtoViewModel.kt"
private val logtoConfig = LogtoConfig(
    // ...другие настройки
    // highlight-start
    scopes = listOf("email", "phone"), // или `listOf(UserScope.EMAIL, UserScope.PHONE)`
    // highlight-end
)
```

Затем вы можете получить доступ к дополнительным утверждениям в возвращаемом значении `logtoClient.getIdTokenClaims()`:

```kotlin
logtoClient.getIdTokenClaims { logtoException, idTokenClaims ->
    println("IdTokenClaims:$idTokenClaims")
}
// Теперь вы можете получить доступ к дополнительным утверждениям `claims.email`, `claims.phone` и т.д.
```
