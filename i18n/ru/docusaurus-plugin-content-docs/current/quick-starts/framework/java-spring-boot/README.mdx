# Добавление аутентификации в ваше приложение на Java Spring Boot

Это руководство покажет вам, как интегрировать Logto в ваше приложение на Java Spring Boot.

:::tip

- Вы можете найти пример кода для этого руководства в нашем репозитории [spring-boot-sample](https://github.com/logto-io/spring-boot-sample) на GitHub.
- Для интеграции Logto с вашим приложением на Java Spring Boot не требуется официальный SDK. Мы будем использовать библиотеки [Spring Security](https://spring.io/projects/spring-security) и [Spring Security OAuth2](https://spring.io/guides/tutorials/spring-boot-oauth2) для обработки OIDC аутентификационного потока с Logto.

:::

## Предварительные требования \{#prerequisites}

- Аккаунт [Logto Cloud](https://cloud.logto.io) или [самостоятельно размещенный Logto](/introduction/set-up-logto-oss).
- Наш пример кода был создан с использованием [securing web starter](https://spring.io/guides/gs/securing-web) для Spring Boot. Следуйте инструкциям по созданию нового веб-приложения, если у вас его нет.
- В этом руководстве мы будем использовать библиотеки [Spring Security](https://spring.io/projects/spring-security) и [Spring Security OAuth2](https://spring.io/guides/tutorials/spring-boot-oauth2) для обработки OIDC аутентификационного потока с Logto. Пожалуйста, ознакомьтесь с официальной документацией, чтобы понять концепции.

## Настройка вашего приложения на Java Spring Boot \{#configure-your-java-spring-boot-application}

### Добавление зависимостей \{#adding-dependencies}

Для пользователей [gradle](https://spring.io/guides/gs/gradle) добавьте следующие зависимости в ваш файл `build.gradle`:

```groovy title="build.gradle"
dependencies {
  implementation 'org.springframework.boot:spring-boot-starter-thymeleaf'
	implementation 'org.springframework.boot:spring-boot-starter-web'
  implementation 'org.springframework.boot:spring-boot-starter-security'
	implementation 'org.springframework.boot:spring-boot-starter-oauth2-client'
}
```

Для пользователей [maven](https://spring.io/guides/gs/maven) добавьте следующие зависимости в ваш файл `pom.xml`:

```xml title="pom.xml"
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-thymeleaf</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>
```

### Конфигурация клиента OAuth2 \{#oauth2-client-configuration}

Зарегистрируйте новое приложение `Java Spring Boot` в Logto Console и получите учетные данные клиента и конфигурации IdP для вашего веб-приложения.

Добавьте следующую конфигурацию в ваш файл `application.properties`:

```properties title="application.properties"
spring.security.oauth2.client.registration.logto.client-name=logto
spring.security.oauth2.client.registration.logto.client-id={{YOUR_CLIENT_ID}}
spring.security.oauth2.client.registration.logto.client-secret={{YOUR_CLIENT_ID}}
spring.security.oauth2.client.registration.logto.redirect-uri={baseUrl}/login/oauth2/code/{registrationId}
spring.security.oauth2.client.registration.logto.authorization-grant-type=authorization_code
spring.security.oauth2.client.registration.logto.scope=openid,profile,offline_access
spring.security.oauth2.client.registration.logto.provider=logto

spring.security.oauth2.client.provider.logto.issuer-uri={{LOGTO_ENDPOINT}}/oidc
spring.security.oauth2.client.provider.logto.authorization-uri={{LOGTO_ENDPOINT}}/oidc/auth
spring.security.oauth2.client.provider.logto.jwk-set-uri={{LOGTO_ENDPOINT}}/oidc/jwks
```

### Реализация WebSecurityConfig \{#implement-the-websecurityconfig}

#### Создайте новый класс `WebSecurityConfig` в вашем проекте \{#create-a-new-class-websecurityconfig-in-your-project}

Класс `WebSecurityConfig` будет использоваться для настройки параметров безопасности вашего приложения. Это ключевой класс, который будет обрабатывать аутентификацию и авторизацию. Пожалуйста, ознакомьтесь с [документацией Spring Security](https://spring.io/guides/topicals/spring-security-architecture) для получения более подробной информации.

```java title="WebSecurityConfig.java"
package com.example.securingweb;

import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;

@Configuration
@EnableWebSecurity

public class WebSecurityConfig {
 // ...
}
```

#### Создайте bean `idTokenDecoderFactory` \{#create-a-idtokendecoderfactory-bean}

Это необходимо, потому что Logto использует `ES384` в качестве алгоритма по умолчанию, и нам нужно переопределить стандартный `OidcIdTokenDecoderFactory`, чтобы использовать тот же алгоритм.

```java title="WebSecurityConfig.java"
import org.springframework.context.annotation.Bean;
import org.springframework.security.oauth2.client.oidc.authentication.OidcIdTokenDecoderFactory;
import org.springframework.security.oauth2.client.registration.ClientRegistration;
import org.springframework.security.oauth2.jose.jws.SignatureAlgorithm;
import org.springframework.security.oauth2.jwt.JwtDecoderFactory;

public class WebSecurityConfig {
  // ...

  @Bean
  public JwtDecoderFactory<ClientRegistration> idTokenDecoderFactory() {
    OidcIdTokenDecoderFactory idTokenDecoderFactory = new OidcIdTokenDecoderFactory();
    idTokenDecoderFactory.setJwsAlgorithmResolver(clientRegistration -> SignatureAlgorithm.ES384);
    return idTokenDecoderFactory;
  }
}
```

#### Создайте класс LoginSuccessHandler для обработки события успешного входа \{#create-a-loginsuccesshandler-class-to-handle-the-login-success-event}

Мы будем перенаправлять пользователя на страницу `/user` после успешного входа.

```java title="CustomSuccessHandler.java"
package com.example.securingweb;

import java.io.IOException;

import org.springframework.security.core.Authentication;
import org.springframework.security.web.authentication.AuthenticationSuccessHandler;

import jakarta.servlet.ServletException;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

public class CustomSuccessHandler implements AuthenticationSuccessHandler {
  @Override
  public void onAuthenticationSuccess(HttpServletRequest request, HttpServletResponse response,
      Authentication authentication) throws IOException, ServletException {
    response.sendRedirect("/user");
  }
}
```

#### Создайте класс LogoutSuccessHandler для обработки события успешного выхода \{#create-a-logoutsuccesshandler-class-to-handle-the-logout-success-event}

Очистите сессию и перенаправьте пользователя на главную страницу.

```java title="CustomLogoutHandler.java"
package com.example.securingweb;

import java.io.IOException;

import org.springframework.security.core.Authentication;
import org.springframework.security.web.authentication.logout.LogoutSuccessHandler;

import jakarta.servlet.ServletException;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;

public class CustomLogoutHandler implements LogoutSuccessHandler {
  @Override
  public void onLogoutSuccess(HttpServletRequest request, HttpServletResponse response, Authentication authentication)
      throws IOException, ServletException {
    HttpSession session = request.getSession();

    if (session != null) {
      session.invalidate();
    }

    response.sendRedirect("/home");
  }
}
```

#### Обновите класс `WebSecurityConfig` с `securityFilterChain` \{#update-the-websecurityconfig-class-with-a-securityfilterchain}

[securityFilterChain](https://docs.spring.io/spring-security/site/docs/current/api/org/springframework/security/web/DefaultSecurityFilterChain.html) — это цепочка фильтров, которые отвечают за обработку входящих запросов и ответов.

Мы настроим `securityFilterChain`, чтобы разрешить доступ к главной странице и требовать аутентификацию для всех остальных запросов. Используйте `CustomSuccessHandler` и `CustomLogoutHandler` для обработки событий входа и выхода.

```java title="WebSecurityConfig.java"
import org.springframework.context.annotation.Bean;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.web.DefaultSecurityFilterChain;

public class WebSecurityConfig {
  // ...

  @Bean
  public DefaultSecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
      .authorizeRequests(authorizeRequests ->
        authorizeRequests
          .antMatchers("/", "/home").permitAll() // Разрешить доступ к главной странице
          .anyRequest().authenticated() // Все остальные запросы требуют аутентификации
      )
      .oauth2Login(oauth2Login ->
        oauth2Login
          .successHandler(new CustomSuccessHandler())
      )
      .logout(logout ->
        logout
          .logoutSuccessHandler(new CustomLogoutHandler())
      );
    return http.build();
  }
}
```

### Создайте главную страницу \{#create-a-home-page}

(Вы можете пропустить этот шаг, если у вас уже есть главная страница в вашем проекте)

```java title="HomeController.java"
package com.example.securingweb;

import java.security.Principal;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class HomeController {
  @GetMapping({ "/", "/home" })
  public String home(Principal principal) {
    return principal != null ? "redirect:/user" : "home";
  }
}
```

Этот контроллер перенаправит пользователя на страницу пользователя, если пользователь аутентифицирован, в противном случае он покажет главную страницу. Добавьте ссылку для входа на главную страницу.

```html title="resources/templates/home.html"
<body>
  <h1>Добро пожаловать!</h1>

  <p><a th:href="@{/oauth2/authorization/logto}">Войти с Logto</a></p>
</body>
```

### Создайте страницу пользователя \{#create-a-user-page}

Создайте новый контроллер для обработки страницы пользователя:

```java title="UserController.java"
package com.example.securingweb;

import java.security.Principal;
import java.util.Map;

import org.springframework.security.oauth2.client.authentication.OAuth2AuthenticationToken;
import org.springframework.security.oauth2.core.user.OAuth2User;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
@RequestMapping("/user")
public class UserController {

  @GetMapping
  public String user(Model model, Principal principal) {
    if (principal instanceof OAuth2AuthenticationToken) {
      OAuth2AuthenticationToken token = (OAuth2AuthenticationToken) principal;
      OAuth2User oauth2User = token.getPrincipal();
      Map<String, Object> attributes = oauth2User.getAttributes();

      model.addAttribute("username", attributes.get("username"));
      model.addAttribute("email", attributes.get("email"));
      model.addAttribute("sub", attributes.get("sub"));
    }

    return "user";
  }
}
```

После аутентификации пользователя мы получим данные `OAuth2User` из объекта аутентифицированного субъекта. Пожалуйста, обратитесь к [OAuth2AuthenticationToken](https://docs.spring.io/spring-security/site/docs/current/api/org/springframework/security/oauth2/client/authentication/OAuth2AuthenticationToken.html) и [OAuth2User](https://docs.spring.io/spring-security/site/docs/current/api/org/springframework/security/oauth2/core/user/OAuth2User.html) для получения более подробной информации.

Прочитайте данные пользователя и передайте их в шаблон `user.html`.

```html title="resources/templates/user.html"
<body>
  <h1>Детали пользователя</h1>
  <div>
    <p>
    <div><strong>имя:</strong> <span th:text="${username}"></span></div>
    <div><strong>электронная почта:</strong> <span th:text="${email}"></span></div>
    <div><strong>идентификатор:</strong> <span th:text="${sub}"></span></div>
    </p>
  </div>

  <form th:action="@{/logout}" method="post">
    <input type="submit" value="Выйти" />
  </form>
</body>
```

#### Запрос дополнительных утверждений \{#request-additional-claims}

<FindUserInfoMissing method="principal (OAuth2AuthenticationToken)" />

<ScopesAndClaimsIntroduction />

Чтобы получить дополнительную информацию о пользователе, вы можете добавить дополнительные области в файл `application.properties`. Например, чтобы запросить области `email`, `phone` и `urn:logto:scope:organizations`, добавьте следующую строку в файл `application.properties`:

```properties title="application.properties"
  spring.security.oauth2.client.registration.logto.scope=openid,profile,offline_access,email,phone,urn:logto:scope:organizations
```

Затем вы можете получить доступ к дополнительным утверждениям в объекте `OAuth2User`.

### Запуск и тестирование приложения \{#run-and-test-the-application}

Запустите приложение и перейдите на `http://localhost:8080`.

- Вы увидите главную страницу со ссылкой для входа.
- Нажмите на ссылку, чтобы войти с Logto.
- После успешной аутентификации вы будете перенаправлены на страницу пользователя с вашими данными.
- Нажмите на кнопку выхода, чтобы выйти. Вы будете перенаправлены обратно на главную страницу.
