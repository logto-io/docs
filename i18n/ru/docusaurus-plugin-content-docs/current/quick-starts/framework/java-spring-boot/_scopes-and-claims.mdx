Logto использует [конвенции областей действия и утверждений](https://openid.net/specs/openid-connect-core-1_0.html#Claims) OIDC для определения областей действия и утверждений для получения информации о пользователе из ID токена и OIDC <a href="https://openid.net/specs/openid-connect-core-1_0.html#UserInfo">userinfo endpoint</a>. Оба термина "область действия" и "утверждение" взяты из спецификаций OAuth 2.0 и OpenID Connect (OIDC).

Вкратце, когда вы запрашиваете область действия, вы получаете соответствующие утверждения в информации о пользователе. Например, если вы запрашиваете область действия `email`, вы получите данные `email` и `email_verified` пользователя.

Добавьте дополнительные области действия и утверждения в файл `application.properties`, чтобы запросить больше информации о пользователе. Например, чтобы запросить область действия `urn:logto:scope:organizations`, добавьте следующую строку в файл `application.properties`:

```properties title="application.properties"
spring.security.oauth2.client.registration.logto.scope=openid,profile,offline_access,urn:logto:scope:organizations
```

Утверждения организации пользователя будут включены в токен авторизации.
