### Отображение информации о пользователе \{#display-user-information}

Чтобы отобразить информацию о пользователе, вы можете внедрить объект `locals.user` в макет, сделав его доступным для всех страниц:

```ts title="+layout.server.ts"
import type { LayoutServerLoad } from './$types';

export const load: LayoutServerLoad = async ({ locals }) => {
  return { user: locals.user };
};
```

В вашем Svelte компоненте:

```html title="+page.svelte"
<script>
  /** @type {import('./$types').PageData} */
  export let data;
</script>

{#if data.user}
<ul>
  {#each Object.entries(data.user) as [key, value]}
  <li>{key}: {value}</li>
  {/each}
</ul>
{/if}
```

### Утверждения, требующие сетевых запросов \{#claims-that-need-network-requests}

Чтобы избежать перегрузки объекта пользователя, некоторые утверждения требуют выполнения сетевых запросов для их получения. Например, утверждение custom_data не включается в объект пользователя, даже если оно запрашивается в областях действия. Для доступа к этим утверждениям вы можете использовать метод `client.fetchUserInfo()`:

По умолчанию объект `locals.user` является удобством ручного вызова метода `getIdTokenClaims`. Если вы хотите использовать результат метода `fetchUserInfo`, вы можете сделать это, установив опцию `fetchUserInfo` в `true` для хука:

```ts title="hooks.server.ts"
import { handleLogto } from '@logto/sveltekit';

export const handle = handleLogto(
  {
    /* Logto config */
  },
  {
    /* Cookie config */
  },
  {
    fetchUserInfo: true,
  }
);
```
