### Настройка провайдера Auth.js \{#set-up-authjs-provider}

<GetAppSecret />

Измените конфигурацию маршрута API Auth.js, добавьте Logto в качестве провайдера OIDC:

<Tabs>

<TabItem value="v5" label="Auth.js v5">

```ts title="./app/api/auth/[...nextauth]/route.ts"
import { handlers } from '@/auth';
export const { GET, POST } = handlers;
```

```ts title="./auth.ts"
import NextAuth from 'next-auth';

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [
    {
      id: 'logto',
      name: 'Logto',
      type: 'oidc',
      // Вы можете получить значение эмитента на странице сведений о приложении Logto,
      // в поле "Эндпоинт эмитента"
      issuer: 'https://xxxx.logto.app/oidc',
      clientId: '<logto-app-id>',
      clientSecret: '<logto-app-secret>',
      authorization: {
        params: { scope: 'openid offline_access profile email' },
      },
      profile(profile) {
        // Вы можете настроить отображение профиля пользователя здесь
        return {
          id: profile.sub,
          name: profile.name ?? profile.username,
          email: profile.email,
          image: profile.picture,
        };
      },
    },
  ],
});
```

1. Замените URL `issuer` на "Эндпоинт эмитента" вашего приложения Logto.
2. Замените `clientId` и `clientSecret` на ID и секрет вашего приложения Logto.
3. Настройте функцию `profile` для отображения профиля пользователя в объект пользователя Next Auth, пример отображения показан в примере.

Затем вы также можете добавить необязательное Middleware для поддержания сессии:

```ts title="./middleware.ts"
export { auth as middleware } from '@/auth';
```

<TabItem value="v4" label="Next Auth v4">

```ts title="app/api/auth/[...nextauth]/route.ts"
import NextAuth from 'next-auth';

const handler = NextAuth({
  providers: [
    {
      id: 'logto',
      name: 'Logto',
      type: 'oauth',
      // Вы можете получить URL well-known на странице сведений о приложении Logto,
      // в поле "Эндпоинт конфигурации провайдера OpenID"
      wellKnown: 'https://xxxx.logto.app/oidc/.well-known/openid-configuration',
      authorization: { params: { scope: 'openid offline_access profile email' } },
      clientId: '<logto-app-id>',
      clientSecret: '<logto-app-secret>',
      client: {
        id_token_signed_response_alg: 'ES384',
      },
      profile(profile) {
        // Вы можете настроить отображение профиля пользователя здесь
        return {
          id: profile.sub,
          name: profile.name ?? profile.username,
          email: profile.email,
          image: profile.picture,
        };
      },
    },
  ],
});

export { handler as GET, handler as POST };
```

1. Замените URL `wellKnown` на "Эндпоинт конфигурации провайдера OpenID" вашего приложения Logto.
2. Замените `clientId` и `clientSecret` на ID и секрет вашего приложения Logto.
3. Настройте функцию `profile` для отображения профиля пользователя в объект пользователя Next Auth, пример отображения показан в примере.
4. Не забудьте установить `id_token_signed_response_alg` на `ES384`.

### Реализация входа и выхода \{#implement-sign-in-and-sign-out}

#### Реализация кнопки входа и выхода \{#implement-sign-in-and-sign-out-button}

```tsx title="app/components/sign-in.tsx"
import { signIn } from '@/auth';

export default function SignIn() {
  return (
    <form
      action={async () => {
        'use server';
        await signIn('logto');
      }}
    >
      <button type="submit">Sign In</button>
    </form>
  );
}
```

```tsx title="app/components/sign-out.tsx"
import { signOut } from '@/auth';

export function SignOut() {
  return (
    <form
      action={async () => {
        'use server';
        await signOut();
      }}
    >
      <button type="submit">Sign Out</button>
    </form>
  );
}
```

#### Показать кнопку входа и выхода на странице \{#show-sign-in-and-sign-out-button-in-the-page}

```tsx title="app/page.tsx"
import SignIn from './components/sign-in';
import SignOut from './components/sign-out';
import { auth } from '@/auth';

export default function Home() {
  const session = await auth();

  return <div>{session?.user ? <SignOut /> : <SignIn />}</div>;
}
```

Выше приведен простой пример, вы можете ознакомиться с [документацией Auth.js](https://authjs.dev/getting-started/session-management/login) для получения более подробной информации.
