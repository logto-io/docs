### Отображение информации о пользователе \{#display-user-information}

Когда пользователь вошел в систему, возвращаемое значение `auth()` будет объектом, содержащим информацию о пользователе. Вы можете отобразить эту информацию в вашем приложении:

```tsx title="app/page.tsx"
import { auth } from '@/auth';

export default async function Home() {
  const session = await auth();

  return (
    <main>
      {session?.user && (
        <div>
          <h2>Утверждения:</h2>
          <table>
            <thead>
              <tr>
                <th>Имя</th>
                <th>Значение</th>
              </tr>
            </thead>
            <tbody>
              {Object.entries(session.user).map(([key, value]) => (
                <tr key={key}>
                  <td>{key}</td>
                  <td>{String(value)}</td>
                </tr>
              ))}
            </tbody>
          </table>
        </div>
      )}
    </main>
  );
}
```

```ts title="./auth.ts"
import NextAuth from 'next-auth';

  providers: [

      id: 'logto',,
      // ...
      authorization: {
        params: {
          // highlight-next-line
          scope: 'openid offline_access profile email',
        },
      },
      // ...
    },
  ],
});
```

### Утверждения, требующие сетевых запросов \{#claims-that-need-network-requests}

Чтобы избежать перегрузки ID токена, некоторые утверждения требуют сетевых запросов для получения. Например, утверждение `custom_data` не включено в объект пользователя, даже если оно запрошено в областях действия. Чтобы получить доступ к этим утверждениям, вам нужно сделать сетевой запрос для получения информации о пользователе.

#### Получение токена доступа \{#get-access-token}

Обновите конфигурацию `NextAuth`, чтобы мы могли получить токен доступа:

```ts title="./auth.ts"
export const { handlers, signIn, signOut, auth } = NextAuth({
  // ...
  callbacks: {
    async jwt({ token, account }) {
      if (account) {
        token.accessToken = account.access_token;
      }
      return token;
    },
    async session({ session, token }) {
      // Внедрите токен доступа в объект сессии
      session.accessToken = token.accessToken;
      return session;
    },
  },
});
```

#### Получение информации о пользователе \{#fetch-user-info}

Теперь получите доступ к конечной точке OIDC user info с токеном доступа:

```ts title="./app/page.tsx"
// ...

export default async function Home() {
  const session = await auth();
  // Замените URL на ваш конечный пункт Logto, должен заканчиваться на `/oidc/me`
  const response = await fetch('https://xxx.logto.app/oidc/me', {
    headers: {
      Authorization: `Bearer ${session?.accessToken}`,
    },
  });
  const user = await response.json();
  console.log(user);

  // ...
}
```

Выше приведен простой пример. Не забудьте обработать случаи ошибок.

#### Обновление токена доступа \{#access-token-refresh}

Токен доступа действителен в течение короткого периода времени. По умолчанию, Next.js будет получать его только при создании сессии. Чтобы реализовать автоматическое обновление токена доступа, смотрите [Обновление токена](https://next-auth.js.org/v3/tutorials/refresh-token-rotation).
