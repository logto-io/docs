### Настройка `angular-auth-oidc-client` для ресурса API \{#configure-angular-auth-oidc-client-for-api-resource}

<ApiResourcesDescription />

<ConfigApiResources
  configResourcesCode={<ConfigResourcesCode />}
  configResourcesWithScopesCode={<ConfigResourcesWithScopesCode />}
/>

Теперь токен доступа будет в формате JSON Web Token (JWT) вместо случайной строки (непрозрачный токен).

:::warning
Оба параметра `autoUserInfo` и `renewUserInfoAfterTokenRenew` будут отключены, когда задан `resource`. Это связано с тем, что токен доступа будет запрашиваться для конкретного ресурса API, а не для конечной точки информации о пользователе.
:::

В настоящее время только официальные SDK Logto поддерживают возможность запроса как информации о пользователе, так и токенов доступа к ресурсам API. Если вам нужно запросить оба, пожалуйста, не стесняйтесь обращаться к нам.
