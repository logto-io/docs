После успешного входа пользователя, Logto выдаст [ID токен](https://openid.net/specs/openid-connect-core-1_0.html#IDToken), который содержит утверждения информации о пользователе. ID токен является JSON Web Token (JWT).

Важно отметить, что утверждения информации о пользователе, которые могут быть получены, зависят от
областей действия, использованных пользователем во время входа, и учитывая производительность и размер данных, ID токен
может не содержать всех утверждений пользователя, некоторые утверждения пользователя доступны только в [userinfo endpoint](https://openid.net/specs/openid-connect-core-1_0.html#UserInfo) (см.
связанный список ниже).

Утилита `buildAngularAuthConfig()` включит `autoUserInfo` и `renewUserInfoAfterTokenRenew`, если в конфигурации не указан `resource`. Это означает, что Logto автоматически получит информацию о пользователе после входа пользователя и обновит информацию о пользователе после обновления токена.

:::info
Чтобы узнать больше о настройке библиотеки `angular-auth-oidc-client`, смотрите [официальную документацию](https://angular-auth-oidc-client.com/).
:::

### Отображение информации о пользователе \{#display-user-information}

`OidcSecurityService` предоставляет удобный способ подписаться на состояние аутентификации, а также информацию о пользователе:

```ts title="app/app.component.ts"
import { OidcSecurityService } from 'angular-auth-oidc-client';
import { decodeIdToken, type IdTokenClaims } from '@logto/js';

export class AppComponent implements OnInit {
  isAuthenticated = false;
  idTokenClaims?: IdTokenClaims;
  accessToken?: string;

  constructor(public oidcSecurityService: OidcSecurityService) {}

  ngOnInit() {
    this.oidcSecurityService.checkAuth().subscribe(({ isAuthenticated, idToken, accessToken }) => {
      console.log('app authenticated', isAuthenticated, idToken);
      this.isAuthenticated = isAuthenticated;
      this.idTokenClaims = decodeIdToken(idToken);
      this.accessToken = accessToken;
    });
  }

  // ...другие методы
}
```

И используйте это в шаблоне:

```html title="app/app.component.html"
<button *ngIf="!isAuthenticated" (click)="signIn()">Sign in</button>
<ng-container *ngIf="isAuthenticated">
  <pre>{{ idTokenClaims | json }}</pre>
  <p>Access token: {{ accessToken }}</p>
  <!-- ... -->
  <button (click)="signOut()">Sign out</button>
</ng-container>
```

Чтобы запросить дополнительные области действия, вы можете настроить конфигурации провайдера аутентификации:

```tsx title="app/app.config.ts"
import { UserScope, buildAngularAuthConfig } from '@logto/js';

export const appConfig: ApplicationConfig = {
  providers: [
    provideHttpClient(withFetch()),
    provideAuth({
      config: buildAngularAuthConfig({
        // ...другие конфигурации
        // highlight-start
        scopes: [
          UserScope.Email,
          UserScope.Phone,
          UserScope.CustomData,
          UserScope.Identities,
          UserScope.Organizations,
        ],
        // highlight-end
      }),
    }),
    // ...другие провайдеры
  ],
};
```

Затем вы можете получить доступ к дополнительным утверждениям в возвращаемом значении `idToken`.
