### Подготовка конфигураций \{#prepare-configs}

Подготовьте конфигурацию для клиента Logto:

```ts title="app/logto.ts"
import { LogtoNextConfig } from '@logto/next';

export const logtoConfig: LogtoNextConfig = {
  appId: '<your-application-id>',
  appSecret: '<your-app-secret-copied-from-console>',
  endpoint: '<your-logto-endpoint>', // Например, http://localhost:3001
  baseUrl: '<your-nextjs-app-base-url>', // Например, http://localhost:3000
  cookieSecret: 'complex_password_at_least_32_characters_long',
  cookieSecure: process.env.NODE_ENV === 'production',
};
```

### Обработка обратного вызова \{#handle-callback}

После входа пользователя Logto перенаправит его обратно на URI перенаправления, настроенный выше. Однако, чтобы ваше приложение работало правильно, необходимо выполнить некоторые действия.

Мы предоставляем вспомогательную функцию `handleSignIn` для обработки обратного вызова входа:

```tsx title="app/callback/route.ts"
import { handleSignIn } from '@logto/next/server-actions';
import { redirect } from 'next/navigation';
import { NextRequest } from 'next/server';
import { logtoConfig } from '../logto';

export async function GET(request: NextRequest) {
  const searchParams = request.nextUrl.searchParams;
  await handleSignIn(logtoConfig, searchParams);

  redirect('/');
}
```

### Реализация входа и выхода \{#implement-sign-in-and-sign-out}

#### Реализация кнопки входа и выхода \{#implement-sign-in-and-sign-out-button}

В Next.js App Router события обрабатываются в клиентских компонентах, поэтому сначала нужно создать два компонента: `SignIn` и `SignOut`.

```tsx title="app/sign-in.tsx"
'use client';

type Props = {
  onSignIn: () => Promise<void>;
};

const SignIn = ({ onSignIn }: Props) => {
  return (
    <button
      onClick={() => {
        onSignIn();
      }}
    >
      Sign In
    </button>
  );
};

export default SignIn;
```

```tsx title="app/sign-out.tsx"
'use client';

type Props = {
  onSignOut: () => Promise<void>;
};

const SignOut = ({ onSignOut }: Props) => {
  return (
    <button
      onClick={() => {
        onSignOut();
      }}
    >
      Sign Out
    </button>
  );
};

export default SignOut;
```

Не забудьте добавить `'use client'` в начало файла, чтобы указать, что эти компоненты являются клиентскими.

#### Добавление кнопок на главную страницу \{#add-buttons-to-home-page}

<ServerActionsTip />

Теперь давайте добавим кнопки входа и выхода на вашу главную страницу. Нам нужно вызывать серверные действия в SDK, когда это необходимо. Чтобы помочь в этом, используйте `getLogtoContext` для получения статуса аутентификации.

```tsx title="app/page.tsx"
import { getLogtoContext, signIn, signOut } from '@logto/next/server-actions';
import SignIn from './sign-in';
import SignOut from './sign-out';
import { logtoConfig } from './logto';

const Home = () => {
  const { isAuthenticated, claims } = await getLogtoContext(logtoConfig);

  return (
    <nav>
      {isAuthenticated ? (
        <p>
          Здравствуйте, {claims?.sub},
          <SignOut
            onSignOut={async () => {
              'use server';

              await signOut(logtoConfig);
            }}
          />
        </p>
      ) : (
        <p>
          <SignIn
            onSignIn={async () => {
              'use server';

              await signIn(logtoConfig);
            }}
          />
        </p>
      )}
    </nav>
  );
};

export default Home;
```
