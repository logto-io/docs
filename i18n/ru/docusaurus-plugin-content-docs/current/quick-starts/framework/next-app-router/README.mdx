### Получение токена доступа для ресурса API \{#fetch-access-token-for-the-api-resource}

<FetchAccessTokenForApiResources />

Если вам нужно получить токен доступа в серверном компоненте, вы можете использовать функцию `getAccessTokenRSC`:

```tsx title="app/page.tsx"
import { getAccessTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const accessToken = await getAccessTokenRSC(logtoConfig, 'https://shopping.your-app.com/api');

  return (
    <main>
      <p>Токен доступа: {accessToken}</p>
    </main>
  );
}
```

:::tip

HTTP не позволяет устанавливать cookies после начала потоковой передачи, `getAccessTokenRSC` не может обновить значение cookie, поэтому если токен доступа обновляется, он не будет сохранен в сессии. Рекомендуется использовать функцию `getAccessToken` на стороне клиента или в обработчиках маршрутов.

:::

### Получение токенов организации \{#fetch-organization-tokens}

<FetchOrganizationTokenForUser />

Если вам нужно получить токен организации в серверном компоненте, вы можете использовать функцию `getOrganizationTokenRSC`:

```tsx title="app/page.tsx"
import { getOrganizationTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const token = await getOrganizationTokenRSC(logtoConfig, 'organization-id');

  return (
    <main>
      <p>Токен организации: {token}</p>
    </main>
  );
}
```

:::tip

HTTP не позволяет устанавливать cookies после начала потоковой передачи, `getOrganizationTokenRSC` не может обновить значение cookie, поэтому если токен доступа обновляется, он не будет сохранен в сессии. Рекомендуется использовать функцию `getOrganizationToken` на стороне клиента или в обработчиках маршрутов.

:::
