### Отображение информации о пользователе \{#display-user-information}

Когда пользователь вошел в систему, возвращаемое значение `getLogtoContext()` будет объектом, содержащим информацию о пользователе. Вы можете отобразить эту информацию в вашем приложении:

```tsx title="app/page.tsx"
import { getLogtoContext } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const { claims } = await getLogtoContext(logtoConfig);

  return (
    <main>
      {claims && (
        <div>
          <h2>Утверждения:</h2>
          <table>
            <thead>
              <tr>
                <th>Имя</th>
                <th>Значение</th>
              </tr>
            </thead>
            <tbody>
              {Object.entries(claims).map(([key, value]) => (
                <tr key={key}>
                  <td>{key}</td>
                  <td>{String(value)}</td>
                </tr>
              ))}
            </tbody>
          </table>
        </div>
      )}
    </main>
  );
}
```

### Получение информации о пользователе в обработчиках маршрутов API \{#get-user-information-in-api-route-handlers}

Вы также можете получить информацию о пользователе в [обработчиках маршрутов](https://nextjs.org/docs/app/building-your-application/routing/route-handlers) API:

```tsx title="app/api/profile/route.ts"
import { getLogtoContext } from '@logto/next/server-actions';
import { logtoConfig } from '../../logto';

export const dynamic = 'force-dynamic';

export async function GET() {
  const { claims } = await getLogtoContext(logtoConfig);

  return Response.json({ claims });
}
```

Чтобы запросить дополнительные области действия, вы можете настроить параметры при инициализации клиента Logto:

```ts title="app/logto.ts"
import { UserScope, LogtoNextConfig } from '@logto/next';

export const logtoConfig: LogtoNextConfig = {
  // highlight-next-line
  scopes: [UserScope.Email, UserScope.Phone], // Добавьте больше областей действия, если необходимо
  // ...другие настройки
});
```

Затем вы можете получить доступ к дополнительным утверждениям в ответе контекста:

```ts title="app/page.tsx"
export default async function Home() {
  const { claims: { email } = {}, } = await getLogtoContext(logtoConfig);

  return (
    <div>
      {email && <p>Email: {email}</p>}
    </div>
  );
};

export default Home;
```
