## Получение информации о пользователе \{#get-user-information}

### Отображение информации о пользователе \{#display-user-information}

Чтобы отобразить информацию о пользователе, вы можете использовать метод `client.getIdTokenClaims()`. Например, в приложении SwiftUI:

```swift title="ContentView.swift"
struct ContentView: View {
  @State var isAuthenticated: Bool
  @State var name: String?

  init() {
    isAuthenticated = client.isAuthenticated
    name = try? client.getIdTokenClaims().name
  }

  var body: some View {
    VStack {
      if isAuthenticated {
        Text("Добро пожаловать, \(name)")
      } else {
        Text("Пожалуйста, войдите")
      }
    }
  }
}
```

### Запрос дополнительных утверждений \{#request-additional-claims}

Чтобы запросить дополнительные области действия, вы можете передать их в объект `LogtoConfig`. Например:

```swift title="ContentView.swift"
let config = try? LogtoConfig(
  endpoint: "<your-logto-endpoint>", // Например, http://localhost:3001
  appId: "<your-app-id>",
  // highlight-start
  scopes: [
    UserScope.Email.rawValue,
    UserScope.Phone.rawValue,
  ]
  // highlight-end
)
```

Затем вы можете получить доступ к дополнительным утверждениям в возвращаемом значении `client.getIdTokenClaims()`:

```swift
let claims = try? client.getIdTokenClaims()
// Теперь вы можете получить доступ к дополнительным утверждениям `claims.email`, `claims.phone` и т.д.
```

### Присоединение токена доступа к заголовкам запроса \{#attach-access-token-to-request-headers}

Поместите токен в поле `Authorization` HTTP-заголовков в формате Bearer (`Bearer YOUR_TOKEN`), и вы готовы к работе.

:::note
Процесс интеграции Bearer Token может варьироваться в зависимости от используемого вами фреймворка или запроса. Выберите свой способ применения заголовка запроса `Authorization`.
:::

```swift
await LogtoRequest.get(
  useSession: session,
  endpoint: userInfoEndpoint,
  // highlight-next-line
  headers: ["Authorization": "Bearer \(accessToken)"]
)
```
