## Получение информации о пользователе \{#get-user-information}

### Отображение информации о пользователе \{#display-user-information}

Чтобы отобразить информацию о пользователе, вы можете использовать метод `getIdTokenClaims` или метод `fetchUserInfo` для получения информации о пользователе. В то время как `getIdTokenClaims` возвращает информацию о пользователе, содержащуюся в ID токене, `fetchUserInfo` извлекает информацию о пользователе из конечной точки userinfo.

Как вы можете видеть, мы используем декоратор `@authenticated`, чтобы предоставить контекст информации о пользователе API приложения Flask.

<CodeBlock title="authenticated.py" language="python">
  {`from functools import wraps
from flask import g, jsonify, redirect
from samples.client import client
def authenticated(shouldRedirect: bool = False, fetchUserInfo: bool = False):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            if client.isAuthenticated() is False:
                if shouldRedirect:
                    return redirect("/sign-in")
                return jsonify({"error": "Not authenticated"}), 401
            # Сохранение информации о пользователе в контексте приложения Flask
            g.user = (
                await client.fetchUserInfo()
                if fetchUserInfo
                else client.getIdTokenClaims()
            )
            return await func(*args, **kwargs)
        return wrapper
    return decorator
`}
</CodeBlock>

Например, чтобы отобразить информацию о пользователе в API, вы можете использовать следующий код:

<CodeBlock title="flask.py" language="python">
  {`@app.route("/protected/userinfo")
@authenticated(shouldRedirect=True, fetchUserInfo=True)
async def protectedUserinfo():
    try:
        return (
            "<h2>User info</h2>"
            + g.user.model_dump_json(indent=2, exclude_unset=True).replace("\\n", "<br>")
            + navigationHtml
        )
    except LogtoException as e:
        return "<h2>Error</h2>" + str(e) + "<br>" + navigationHtml
`}
</CodeBlock>

Наши модели данных основаны на [pydantic](https://docs.pydantic.dev/), поэтому вы можете использовать `model_dump_json` для преобразования модели данных в JSON.

Добавление `exclude_unset=True` исключит неустановленные поля из вывода JSON, что делает вывод более точным.

Например, если мы не запрашивали область `email` при входе, поле `email` будет исключено из вывода JSON. Однако, если мы запрашивали область `email`, но у пользователя нет адреса электронной почты, поле `email` будет включено в вывод JSON со значением `null`.

Чтобы узнать больше об областях и утверждениях, смотрите [Получение информации о пользователе](#get-user-information).

Чтобы запросить дополнительные области, вы можете передать области в объект `LogtoConfig`. Например:

```python title="client.py"
# highlight-next-line
from logto import UserInfoScope

client = LogtoClient(
    LogtoConfig(
        # ...другие настройки
        # highlight-start
        scopes = [
            UserInfoScope.email,
            UserInfoScope.phone,
        ],
        # highlight-end
    ),
    storage=SessionStorage(),
)
```

Затем вы можете получить доступ к дополнительным утверждениям в возвращаемом значении `client.getIdTokenClaims()`:

```python
idTokenClaims = await client.getIdTokenClaims();
```
