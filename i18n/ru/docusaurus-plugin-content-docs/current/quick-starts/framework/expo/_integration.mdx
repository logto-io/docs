### Инициализация провайдера Logto \{#init-logto-provider}

Импортируйте и используйте `LogtoProvider`, чтобы предоставить контекст Logto:

```tsx title="App.tsx"
import { LogtoProvider, LogtoConfig } from '@logto/rn';

const config: LogtoConfig = {
  endpoint: '<your-logto-endpoint>',
  appId: '<your-application-id>',
};

const App = () => (
  <LogtoProvider config={config}>
    <YourAppContent />
  </LogtoProvider>
);
```

### Реализация входа и выхода \{#implement-sign-in-and-sign-out}

<SignInFlowSummary />

Перейдите на страницу деталей приложения в Logto Console. Добавьте нативный URI перенаправления (например, `io.logto://callback`), затем нажмите "Сохранить".

- Для iOS схема URI перенаправления не имеет значения, так как класс `ASWebAuthenticationSession` будет отслеживать URI перенаправления независимо от того, зарегистрирован он или нет.
- Для Android схема URI перенаправления должна быть указана в файле `app.json` Expo, например:

  ```json title="app.json"
  {
    "expo": {
      "scheme": "io.logto"
    }
  }
  ```

Теперь вернитесь к вашему приложению, вы можете использовать хук `useLogto` для входа и выхода:

```tsx title="App.tsx"
import { useLogto } from '@logto/rn';
import { Button } from 'react-native';

const Content = () => {
  const { signIn, signOut, isAuthenticated } = useLogto();

  return (
    <div>
      {isAuthenticated ? (
        <Button title="Sign out" onPress={async () => signOut()} />
      ) : (
        // Замените URI перенаправления на ваш собственный
        <Button title="Sign in" onPress={async () => signIn('io.logto://callback')} />
      )}
    </div>
  );
};
```
