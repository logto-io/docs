# Управление пользователями

## Управление через Logto Console \{#manage-via-logto-console}

### Просмотр и поиск пользователей \{#browse-and-search-users}

Чтобы получить доступ к функциональности управления пользователями в Logto Console, перейдите в <CloudLink to="/users">Console > User management</CloudLink>. Там вы увидите таблицу со всеми пользователями.

Таблица состоит из трех столбцов:

- **Пользователь**: Отображает информацию о пользователе, такую как аватар, полное имя, имя пользователя, номер телефона и электронную почту.
- **Из приложения**: Отображает название приложения, с которым пользователь изначально зарегистрировался.
- **Последний вход**: Отображает временную метку последнего входа пользователя.

Поддерживается сопоставление ключевых слов для [`name`](/user-management/user-data#name), [`id`](/user-management/user-data#id), [`username`](/user-management/user-data#username), [`primary-phone`](/user-management/user-data#primary_phone), [`primary-email`](/user-management/user-data#primary_email).

### Добавление пользователей \{#add-users}

С помощью Console разработчики могут создавать новые учетные записи для конечных пользователей. Для этого нажмите кнопку "Add user" в верхнем правом углу экрана.

После создания нового пользователя вы можете скопировать их **введенные идентификаторы (адрес электронной почты / номер телефона / имя пользователя)** и **начальный пароль**, чтобы отправить их пользователю. Это позволит им войти в систему и начать ее использовать.

Начальный пароль будет отображаться только один раз, но вы можете [сбросить пароль](/end-user-flows/sign-up-and-sign-in/reset-password) позже.

### Просмотр и обновление профиля пользователя \{#view-and-update-the-user-profile}

Чтобы просмотреть детали пользователя, просто нажмите на соответствующую строку в таблице пользователей. Это перенесет вас на страницу "**User Details**", где вы найдете информацию о профиле пользователя, включая:

- **Данные, связанные с аутентификацией**: адрес электронной почты, номер телефона, имя пользователя, наличие пароля, социальные подключения, корпоративные SSO подключения, многофакторная аутентификация и персональный токен доступа.
- **Данные профиля пользователя**: имя, URL аватара, пользовательские данные и дополнительные стандартные утверждения OpenID Connect, которые не включены.

Раздел "Social connections" отображает список социальных коннекторов, которые пользователь связал со своей учетной записью. Например, если пользователь вошел в систему, используя свою учетную запись Facebook, вы увидите элемент "Facebook" в списке. Вы можете обратиться к документации по [социальным идентификациям](/user-management/user-data/#social-identities) для получения дополнительной информации.

Обратите внимание, что полное имя, URL изображения аватара и пользовательские данные можно редактировать, социальные подключения можно удалять, а остальные свойства доступны только для чтения.

:::warning

Важно убедиться, что у пользователя есть альтернативный метод входа перед удалением социального подключения, такой как другое социальное подключение, номер телефона, электронная почта или имя пользователя с паролем. Если у пользователя нет другого метода входа, он не сможет снова получить доступ к своей учетной записи после удаления социального подключения.

:::

### Просмотр активности пользователя \{#view-user-activities}

Чтобы просмотреть недавнюю активность пользователя, перейдите на вкладку "User logs" на странице "User details". Здесь вы найдете таблицу, отображающую недавнюю активность пользователя, включая выполненное действие, результат действия, связанное приложение и время, когда пользователь действовал.

Нажмите на строку таблицы, чтобы увидеть больше деталей в журнале пользователя, например, IP-адрес, пользовательский агент, необработанные данные и т. д.

### Приостановка пользователя \{#suspend-user}

На странице "User details" нажмите "Три точки" -> кнопку "Suspend user".

После приостановки пользователь не сможет войти в ваше приложение и не сможет получить новый токен доступа после истечения срока действия текущего. Кроме того, любые API-запросы, сделанные этим пользователем, будут неудачными.

Если вы хотите реактивировать этого пользователя, вы можете сделать это, нажав "Три точки" -> кнопку "Reactivate user".

### Удаление пользователя \{#delete-user}

На странице "User details" нажмите "Три точки" -> кнопку "Delete". Удаление пользователя не может быть отменено.

### Сброс пароля пользователя \{#reset-user-password}

На странице "User details" нажмите "Три точки" -> кнопку "Reset password".

После сброса пароля скопируйте и отправьте его конечному пользователю. После закрытия модального окна "Reset password" вы больше не сможете просмотреть пароль. Если вы забудете его сохранить, вы можете сбросить его снова.

### Управление ролями пользователей \{#manage-roles-of-users}

На вкладке "Roles" страницы деталей пользователя вы можете легко назначать или удалять роли для достижения желаемого результата. Ознакомьтесь с [RBAC - Настройка ролей](/authorization/role-based-access-control/configure-roles), чтобы узнать больше.

### Просмотр организаций, к которым принадлежит пользователь \{#view-the-organizations-the-user-belongs-to}

Logto поддерживает [организации](/organizations/organization-management) и может управлять их членами. Вы можете легко просмотреть детали пользователя и увидеть, к какой организации он принадлежит.

## Управление через Logto Management API \{#manage-via-logto-management-api}

[Management API](/concepts/core-service/#management-api) — это набор API, предоставляющих доступ к бэкенд-сервису Logto. Как упоминалось ранее, пользовательский API является критически важным компонентом этого сервиса и может поддерживать широкий спектр сценариев.

Пользовательские [RESTful](https://en.wikipedia.org/wiki/Representational_state_transfer) API размещены по адресу `/api/users`, за исключением активности пользователей, т.е. журналов пользователей `/api/logs?userId=:userId`.

Вы можете управлять пользователями через Management API в нескольких случаях использования. Например, [расширенный поиск пользователей](/user-management/advanced-user-search), [массовое создание учетных записей](https://openapi.logto.io/operation/operation-createuser), [регистрация только по приглашению](/end-user-flows/sign-up-and-sign-in/disable-user-registration) и т. д.

## Часто задаваемые вопросы \{#faqs}

<details>

<summary>

### Как ограничить доступ к определенным приложениям для конкретных пользователей? \{#how-to-restrict-access-to-certain-application-for-specific-users}

</summary>

Из-за природы [Omni-sign-in](https://logto.io/products/omni-sign-in) Logto не предназначен для ограничения доступа пользователей к определенным приложениям до аутентификации.
Однако вы все равно можете разработать специфичные для приложения роли пользователей и разрешения для защиты ваших ресурсов API и проверять разрешения на доступ к API после успешного входа пользователя.
Обратитесь к Авторизация: [Управление доступом на основе ролей](/authorization/role-based-access-control) для получения дополнительной информации.

</details>
