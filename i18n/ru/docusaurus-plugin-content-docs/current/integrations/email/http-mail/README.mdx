# Настройка проверки электронной почты через HTTP вызов

Официальный коннектор Logto для HTTP email.

<GuideTip />

## Начало работы \{#get-started}

Коннектор HTTP email позволяет отправлять электронные письма через HTTP вызов. Чтобы использовать коннектор HTTP email, вам потребуется собственный почтовый сервис, который предоставляет HTTP API для отправки писем. Logto будет вызывать это API, когда потребуется отправить письмо. Например, когда пользователь регистрируется, Logto вызовет HTTP API для отправки проверочного письма.

## Настройка коннектора HTTP email \{#set-up-http-email-connector}

Чтобы использовать коннектор HTTP email, вам нужно настроить HTTP endpoint, который Logto сможет вызвать. И необязательный токен авторизации для этого endpoint.

## Полезная нагрузка \{#payload}

Коннектор HTTP email отправит следующую полезную нагрузку на endpoint, когда потребуется отправить письмо:

```json
{
  "to": "foo@logto.io",
  "type": "SignIn",
  "payload": {
    "code": "123456"
  }
}
```

Вы можете найти все типы в [Шаблоне электронной почты](/connectors/email-connectors/built-in-email-service/#unified-email-templates), а полное определение типа `SendMessageData` в [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts).
