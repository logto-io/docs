1. Укажите `App integration name`. Он будет использоваться в качестве идентификатора вашего OIDC приложения.
2. Добавьте новый `Sign-in redirect URIs`, используя URL обратного вызова коннектора Logto SSO.

Это URI, на который Okta перенаправит браузер пользователя после успешной аутентификации. После успешной аутентификации с IdP, IdP перенаправляет браузер пользователя обратно на этот указанный URI вместе с кодом авторизации. Logto завершит процесс аутентификации на основе кода авторизации, полученного с этого URI.

![Настройки приложения Okta.webp](./assets/okta_application_settings.webp)

3. Назначьте пользователей для приложения.

На основе настроек `Assignments` вы можете выбрать назначение приложения всем пользователям или конкретным пользователям / группам.

![Назначение пользователей Okta.webp](./assets/okta_assign_users.webp)

Нажмите кнопку `Save`, чтобы сохранить настройки приложения.
