Инициируйте интеграцию SAML SSO, создав приложение на стороне провайдера идентификации (IdP). Получите следующие конфигурации от Logto, представляющие вашего поставщика услуг (SP):

- **URI аудитории (SP Entity ID)**: Он представляет собой глобально уникальный идентификатор для вашего сервиса Logto, функционирующий как EntityId для SP во время запросов аутентификации к IdP. Этот идентификатор является ключевым для безопасного обмена SAML утверждениями и другими данными, связанными с аутентификацией, между IdP и Logto.
- **ACS URL**: URL сервиса потребления утверждений (ACS) — это место, куда SAML утверждение отправляется с POST запросом. Этот URL используется IdP для отправки SAML утверждения в Logto. Он действует как URL обратного вызова, где Logto ожидает получить и обработать SAML ответ, содержащий информацию о личности пользователя.

Заполните конфигурации URI аудитории и ACS URL в вашем SAML приложении IdP и продолжайте получать следующие конфигурации от вашего IdP.
