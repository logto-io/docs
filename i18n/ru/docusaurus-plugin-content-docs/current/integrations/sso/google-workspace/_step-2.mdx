Для создания новых учетных данных OIDC вам необходимо настроить экран согласия для вашего приложения.

1. Перейдите на страницу [OAuth consent screen](https://console.cloud.google.com/apis/credentials/consent) и выберите тип пользователя `Internal`. Это сделает OAuth-приложение доступным только для пользователей внутри вашей организации.

![Тип пользователя экрана согласия Google Workspace.webp](./assets/google_workspace_consent_screen_user_type.webp)

2. Заполните настройки `Экран согласия`, следуя инструкциям на странице. Вам необходимо предоставить следующую минимальную информацию:

- **Имя приложения**: Имя вашего приложения. Оно будет отображаться на экране согласия.
- **Поддержка по электронной почте**: Электронная почта поддержки вашего приложения. Она будет отображаться на экране согласия.

![Настройки экрана согласия Google Workspace.webp](./assets/google_workspace_consent_screen_settings.webp)

3. Установите `Области действия` для вашего приложения. Чтобы правильно получить информацию о личности пользователя и адрес электронной почты от IdP, коннекторы Logto SSO должны предоставить следующие области действия от IdP:

![Области действия экрана согласия Google Workspace.webp](./assets/google_workspace_consent_screen_scopes.webp)

- **openid**: Эта область действия необходима для аутентификации OIDC. Она используется для получения ID токена и доступа к endpoint userInfo IdP.
- **profile**: Эта область действия необходима для доступа к основной информации профиля пользователя.
- **email**: Эта область действия необходима для доступа к адресу электронной почты пользователя.

Нажмите кнопку `Сохранить`, чтобы сохранить настройки экрана согласия.
