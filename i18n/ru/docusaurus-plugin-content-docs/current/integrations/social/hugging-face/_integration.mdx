## Вход с аккаунтом Hugging Face \{#sign-in-with-hugging-face-account}

Перейдите на [веб-сайт Hugging Face](https://huggingface.co/) и войдите с вашим аккаунтом Hugging Face. Вы можете зарегистрировать новый аккаунт, если у вас его нет.

## Создание OAuth приложения в Hugging Face \{#create-an-oauth-app-in-the-hugging-face}

Следуйте руководству [Создание OAuth приложения](https://huggingface.co/docs/hub/en/oauth#creating-an-oauth-app) и зарегистрируйте новое приложение.

В процессе создания вам потребуется предоставить следующую информацию:

- **Имя приложения**: Название вашего приложения.
- **URL домашней страницы**: URL домашней страницы или целевой страницы вашего приложения.
- **URL логотипа**: URL логотипа вашего приложения.
- **Области действия**: Области действия, разрешенные для OAuth приложения. Для коннектора Hugging Face обычно используется `profile` для получения информации о профиле пользователя и `email` для получения адреса электронной почты пользователя. Убедитесь, что эти области действия разрешены в вашем OAuth приложении Hugging Face, если вы хотите их использовать.
- **Redirect URI**: URL для перенаправления пользователя после аутентификации. Вы можете найти Redirect URI в Logto Admin Console, когда создаете коннектор Hugging Face или на странице деталей созданного коннектора Hugging Face.

## Управление OAuth приложениями Hugging Face \{#managing-hugging-face-oauth-apps}

Перейдите на страницу [Подключенные приложения](https://huggingface.co/settings/connected-applications), где вы можете добавлять, редактировать или удалять существующие OAuth приложения.
Вы также можете найти `Client ID` и сгенерировать `App secrets` на соответствующих страницах настроек OAuth приложения.

## Настройка вашего коннектора \{#configure-your-connector}

Заполните поля `clientId` и `clientSecret` с _Client ID_ и _App Secret_, которые вы получили на страницах деталей OAuth приложения, упомянутых в предыдущем разделе.

`scope` — это список [поддерживаемых областей действия Hugging Face](https://huggingface.co/docs/hub/en/oauth#currently-supported-scopes), разделенный пробелами. Если не указано, область действия по умолчанию будет `profile`. Для коннектора Hugging Face области действия, которые вы можете использовать, это `profile` и `email`. Область действия `profile` необходима для получения информации о профиле пользователя, а область действия `email` необходима для получения адреса электронной почты пользователя. Убедитесь, что вы разрешили эти области действия в вашем OAuth приложении Hugging Face (настроено в разделе [Создание OAuth приложения в Hugging Face](#create-an-oauth-app-in-the-hugging-face)).

### Типы конфигурации \{#config-types}

| Имя          | Тип    |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
