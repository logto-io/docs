## Поддержка только корейского языка на сайте разработчиков \{#developer-site-only-korean-support-now}

В настоящее время сайт `Naver Developers` поддерживает только корейский язык. Пожалуйста, рассмотрите возможность использования переводчика.

## Для продакшена \{#for-the-production}

- Для продакшена вам необходимо получить одобрение от команды Naver.
  В противном случае только зарегистрированные пользователи смогут войти.
  - Вы можете добавить тестировщика из меню `맴버관리(Member Manage)`.
- Чтобы получить одобрение, пожалуйста, проверьте `애플리케이션 개발 상태(Application Development Status)`
  в `API 설정(API Setting)` в настройках вашего проекта приложения.

## Настройка проекта в Naver Developers \{#set-up-a-project-in-the-naver-developers}

- Посетите [Naver Developers](https://developers.naver.com/main/) и войдите в систему с вашей учетной записью Naver.
- Нажмите **Application -> 어플리케이션 등록** в меню, чтобы создать новый проект.
- Следуйте инструкциям ниже для создания приложения.

### Имя приложения (어플리케이션 이름) \{#application-name-어플리케이션-이름}

- Введите имя вашего приложения в `어플리케이션 이름` (Это имя отображается, когда пользователь входит в систему.)

### Использование API (사용 API) \{#api-usage-사용-api}

- Выберите `네이버 로그인(Naver Login)` для `사용 API(API Usage)`
- Отметьте `이메일 주소(Email Address), 별명(Nickname), 프로필 사진(Profile Image)` как `필수(Neccessary)` в `권한(Role)` (Вы можете отметить `추가(Add)` как необязательные эти опции, но вы не сможете получить информацию от пользователя.)

### Среда службы Open API для входа (로그인 오픈 API 서비스 환경) \{#sign-in-open-api-service-environment-로그인-오픈-api-서비스-환경}

- Для `로그인 오픈 API 서비스 환경(Sign in Open API Service Environment)`, добавьте две среды `PC웹(PC Web)` и `모바일웹(Mobile Web)`.

#### PC Web (PC 웹) \{#pc-web-pc-웹}

- Для `서비스 URL(Service URL)`, введите `http(s)://YOUR_URL` (например, https://logto.io)
- Для `네이버 로그인(Naver Login) Callback URL`, введите `http(s)://YOUR_URL/callback/${connector_id}` (например, https://logto.io/callback/${connector_id})

#### Mobile Web (Mobile 웹) \{#mobile-web-mobile-웹}

- Для `서비스 URL(Service URL)`, введите `http(s)://YOUR_URL` (например, https://logto.io)
- Для `네이버 로그인(Naver Login) Callback URL`, введите `http(s)://YOUR_URL/callback/${connector_id}` (например, https://logto.io/callback/${connector_id})

:::caution

`connector_id` можно найти на верхней панели страницы деталей коннектора в Logto Admin Console.

:::

## Настройка Logto \{#configure-logto}

### Типы конфигурации \{#config-types}

| Имя          | Тип    |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |

#### clientId \{#clientid}

`clientId` — это `Client ID` вашего проекта.
(Вы можете найти его в `애플리케이션 정보(Application Info)` вашего проекта на Naver developers.)

#### clientSecret \{#clientseceret}

`clientSecret` — это `Client Secret` вашего проекта.
(Вы можете найти его в `애플리케이션 정보(Application Info)` вашего проекта на Naver developers.)
