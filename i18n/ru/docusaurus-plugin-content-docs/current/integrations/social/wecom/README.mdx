# Настройка социального входа через WeCom (OAuth 2.0)

Пользовательский коннектор для социального входа через WeCom (возможно, называется WXwork).

<GuideTip />

## Начало работы \{#get-started}

Войдите в [WeCom WebUI](https://work.weixin.qq.com/) с учетной записью администратора или нажмите **Управление предприятием(管理企业)** в приложении WeCom.

На вкладке "Управление приложениями" прокрутите страницу вниз и нажмите "Создать приложение".

Заполните соответствующую информацию в соответствии с вашим приложением. Затем создайте.

Теперь у нас есть Agent ID (НЕ APPID) и Secret.

### Информация о сайте \{#website-info}

Настройте необходимые параметры на этой странице. Это будет выглядеть так:

- Разрешенные пользователи: _кто может видеть это приложение_
- Главная страница приложения: _Главная страница вашего приложения. Например, `logto.io/demo-app`_

**Важно**
На этой странице есть три элемента, касающиеся "Developer API(开发者接口)".

1. Веб-авторизация и JS-SDK;
2. Вход через авторизацию WeCom;
3. Доверенный IP предприятия;

Заполните их в соответствии с руководством WeCom.

### Corp ID \{#corp-id}

Если вы знакомы с разработкой WeChat, вы можете заметить, что использование Corp ID такое же, как и APP ID.

Вы можете найти Corp ID внизу страницы вкладки "Мое предприятие(我的企业)". Это выглядит как **ww\*\*\*\*** .

## Настройка коннектора \{#configure-the-connector}

Итак, у нас есть Agent ID, Secret и Corp ID.

Давайте заполним форму для коннектора.

Вы можете оставить поле `Scope` пустым, так как оно является необязательным. В качестве альтернативы, вы можете заполнить `snsapi_base` или `snsapi_privateinfo`.

**Сохраните и готово**
