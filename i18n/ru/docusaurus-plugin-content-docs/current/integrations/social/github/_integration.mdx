### Вход с аккаунтом GitHub \{#sign-in-with-github-account}

Перейдите на [веб-сайт GitHub](https://github.com/) и войдите в систему с вашим аккаунтом GitHub. Вы можете зарегистрировать новый аккаунт, если у вас его нет.

### Создание и настройка OAuth приложения \{#create-and-configure-oauth-app}

Следуйте руководству по [созданию OAuth приложения](https://docs.github.com/en/developers/apps/building-oauth-apps/creating-an-oauth-app) и зарегистрируйте новое приложение.

Назовите ваше новое OAuth приложение в поле **Application name** и заполните **Homepage URL** приложения. Вы можете оставить поле **Application description** пустым и настроить **Authorization callback URL** как `${your_logto_origin}/callback/${connector_id}`. `connector_id` можно найти на верхней панели страницы деталей коннектора в Logto Admin Console.

:::note

Если вы столкнулись с сообщением об ошибке "The redirect_uri MUST match the registered callback URL for this application." при входе в систему, попробуйте выровнять URL обратного вызова авторизации вашего GitHub OAuth приложения и URL перенаправления вашего Logto приложения (включая протокол), чтобы решить проблему.

:::

Мы рекомендуем не отмечать галочку перед **Enable Device Flow**, иначе пользователи, входящие в систему с GitHub на мобильных устройствах, должны будут подтвердить начальное действие входа в приложении GitHub. Многие пользователи GitHub не устанавливают мобильное приложение GitHub на свои телефоны, что может заблокировать процесс входа. Пожалуйста, проигнорируйте наше предложение, если вы ожидаете, что конечные пользователи подтвердят свой процесс входа. См. подробности о [device flow](https://docs.github.com/en/developers/apps/building-oauth-apps/authorizing-oauth-apps#device-flow).

### Управление OAuth приложениями \{#managing-oauth-apps}

Перейдите на [страницу OAuth приложений](https://github.com/settings/developers), где вы можете добавлять, редактировать или удалять существующие OAuth приложения. Вы также можете найти `Client ID` и сгенерировать `Client secrets` на страницах деталей OAuth приложения.

### Настройка вашего коннектора \{#configure-your-connector}

Заполните поля `clientId` и `clientSecret` значениями _Client ID_ и _Client Secret_, которые вы получили на страницах деталей OAuth приложения, упомянутых в предыдущем разделе.

`scope` — это список [областей](https://docs.github.com/en/apps/oauth-apps/building-oauth-apps/scopes-for-oauth-apps), разделенных пробелами. Если не указано, область по умолчанию будет `read:user`.

### Типы конфигурации \{#config-types}

| Имя          | Тип    |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
