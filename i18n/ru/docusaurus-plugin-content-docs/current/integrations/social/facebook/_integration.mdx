### Зарегистрируйте учетную запись разработчика Facebook \{#register-a-facebook-developer-account}

[Зарегистрируйтесь как разработчик Facebook](https://developers.facebook.com/docs/development/register/), если у вас еще нет учетной записи.

### Настройте приложение Facebook \{#set-up-a-facebook-app}

1. Посетите страницу [Apps](https://developers.facebook.com/apps).
2. Нажмите на ваше существующее приложение или [создайте новое](https://developers.facebook.com/docs/development/create-an-app), если необходимо.
   - Выбранный [тип приложения](https://developers.facebook.com/docs/development/create-an-app/app-dashboard/app-types) зависит от вас, но он должен иметь продукт _Facebook Login_.
3. На странице панели управления приложением прокрутите до раздела "Add a product" и нажмите кнопку "Set up" на карточке "Facebook Login".
4. Пропустите страницу Facebook Login Quickstart и нажмите на боковую панель -> "Products" -> "Facebook Login" -> "Settings".
5. На странице настроек Facebook Login заполните поле "Valid OAuth Redirect URIs" значением `${your_logto_origin}/callback/${connector_id}`. `connector_id` можно найти на верхней панели страницы деталей коннектора в Logto Admin Console. Например:
   - `https://logto.dev/callback/${connector_id}` для продакшена
   - `https://localhost:3001/callback/${connector_id}` для тестирования в локальной среде
6. Нажмите кнопку "Save changes" в правом нижнем углу.

## Составьте JSON коннектора \{#compose-the-connector-json}

1. На странице панели управления приложением Facebook нажмите на боковую панель -> "Settings" -> "Basic".
2. Вы увидите "App ID" и "App secret" на панели.
3. Нажмите кнопку "Show" рядом с полем ввода App secret, чтобы скопировать его содержимое.
4. Заполните настройки коннектора Logto:
   - Заполните поле `clientId` строкой из _App ID_.
   - Заполните поле `clientSecret` строкой из _App secret_.
   - Заполните поле `scope` списком [разрешений](https://developers.facebook.com/docs/permissions/reference), разделенных запятыми или пробелами, в виде строки. Если вы не укажете область действия, область действия по умолчанию будет `email,public_profile`.

## Протестируйте вход с тестовыми пользователями Facebook \{#test-sign-in-with-facebooks-test-users}

Вы можете использовать учетные записи тестовых, разработчиков и администраторов для тестирования входа с соответствующим приложением как в режиме разработки, так и в режиме [live](https://developers.facebook.com/docs/development/build-and-test/app-modes).

Вы также можете [перевести приложение в режим live](https://developers.facebook.com/docs/development/build-and-test/app-modes#live-mode), чтобы любой пользователь Facebook мог войти в приложение.

- На странице панели управления приложением нажмите на боковую панель -> "Roles" -> "Test Users".
- Нажмите кнопку "Create test users", чтобы создать тестового пользователя.
- Нажмите кнопку "Options" для существующего тестового пользователя, и вы увидите больше операций, например, "Change name and password".

## Опубликуйте настройки входа через Facebook \{#publish-facebook-sign-in-settings}

Обычно только тестовые, администраторы и разработчики могут войти в соответствующее приложение в [режиме разработки](https://developers.facebook.com/docs/development/build-and-test/app-modes#development-mode).

Чтобы разрешить обычным пользователям Facebook входить в приложение в производственной среде, возможно, вам нужно переключить ваше приложение Facebook в _[режим live](https://developers.facebook.com/docs/development/build-and-test/app-modes#live-mode)_, в зависимости от типа приложения.
Например, чистое приложение _business type_ не имеет кнопки переключения "live", но это не помешает вашему использованию.

1. На странице панели управления приложением Facebook нажмите на боковую панель -> "Settings" -> "Basic".
2. Заполните поля "Privacy Policy URL" и "User data deletion" на панели, если это требуется.
3. Нажмите кнопку "Save changes" в правом нижнем углу.
4. Нажмите кнопку переключения "Live" на верхней панели приложения.
