# Logto OSS

Logto にはオープンソース (OSS) 版と [クラウド](/logto-cloud) 版の両方があります。このセクションでは OSS ソリューションに焦点を当てます： <Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS は、開発者に完全なカスタマイズ性とデータの完全なコントロールを提供する、無料で利用できるアイデンティティソリューションです。Logto OSS は、コードベースを特定のニーズに合わせて柔軟に適応できるため、データ主権を重視し、アプリケーションとともに成長できるカスタマイズ可能なアイデンティティプロバイダーソリューションを求める方に最適です。

## Logto OSS でサポートされている機能 \{#feature-supported-by-logto-oss}

Logto OSS は Logto サービスのほとんどのコア機能をサポートしており、定期的にアップデート・メンテナンスされています。

一部の高度な機能は現在 Logto Cloud 版のみで利用可能です。主なものは以下の通りです：

| OSS での機能制限                                                                    | 説明                                                                                                             |
| ----------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| [Logto コンソール：複数テナント](/logto-cloud/tenant-settings)                      | コンソールで複数の Logto テナントを作成・管理できます。                                                          |
| [Logto コンソール：コラボレーター招待](/logto-cloud/tenant-member-management)       | 1 つの Logto テナントに複数メンバーを招待・管理できます。                                                        |
| Logto コンソール：MFA 有効化                                                        | Logto コンソールへのサインイン時に多要素認証 (MFA) を必須にしてセキュリティを強化。                              |
| [Logto Protected App](/integrate-logto/protected-app)                               | Cloudflare によるクイックかつ SDK 不要の認証統合。                                                               |
| [Logto 組み込みメールサービス](/connectors/email-connectors/built-in-email-service) | 組み込みかつ無料のメールサービスでメール配信が可能。                                                             |
| [Bring your UI](/customization/bring-your-ui)                                       | サインイン体験の UI やフローをカスタマイズ可能（OSS では GitHub 上のコードをフォークして直接カスタマイズ可能）。 |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)               | アイデンティティプロバイダー (IdP) 主導のエンタープライズシングルサインオン (SSO) を有効化。                     |
| [SAML アプリ](/integrate-logto/saml-app)                                            | SAML プロトコル経由でアプリケーションを統合。OSS 版は 3 つまでの SAML アプリに制限。                             |

ヒント：マルチテナンシー、メンバー招待、MFA はオープンソースの Logto コンソールへのサインインには利用できません。ただし、Logto OSS を使って自社製品にこれらの機能を実装し、エンドユーザーに提供することは可能です。
