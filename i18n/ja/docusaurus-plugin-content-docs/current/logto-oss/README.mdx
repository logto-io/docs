# Logto OSS

Logto はオープンソース (OSS) バージョンと [クラウド](/logto-cloud) バージョンの両方を提供しています。このセクションでは、OSS ソリューションに焦点を当てます：<Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS は、開発者に完全なカスタマイズ機能とデータに対する完全なコントロールを提供する無料のアイデンティティソリューションです。Logto OSS は、コードベースを特定のニーズに合わせて適応させる柔軟性を提供し、データ主権を重視し、アプリケーションと共に成長できるカスタマイズされたアイデンティティプロバイダーソリューションを求める人々に最適です。

## Logto OSS がサポートする機能 \{#feature-supported-by-logto-oss}

Logto OSS は、Logto サービスのほとんどのコア機能をサポートしており、定期的に更新およびメンテナンスされています。

一部の高度な機能は現在、Logto Cloud バージョンにのみ限定されています。これには以下が含まれます：

| OSS の機能制限                                                          | 説明                                                                                                                                        |
| ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto コンソール：複数のテナント](/logto-cloud/tenant-settings)                     | コンソールで複数の Logto テナントを作成および管理します。                                                                                             |
| [Logto コンソール：コラボレーター招待](/logto-cloud/tenant-member-management)     | 1 つの Logto テナントの複数のメンバーを招待および管理します。                                                                                            |
| Logto コンソール：MFA の有効化                                                           | Logto コンソールにサインインする際に多要素認証を要求することでセキュリティを強化します。                                                    |
| [Logto Protected App](/integrate-logto/protected-app)                               | Cloudflare によって強化された SDK を使用しない迅速な認証統合。                                                                                |
| [Logto 組み込みメールサービス](/connectors/email-connectors/built-in-email-service) | メール配信のための組み込みの無料メールサービスを利用します。                                                                                    |
| [UI の持ち込み](/customization/bring-your-ui)                                       | この機能を使用してサインイン体験の UI やフローをカスタマイズします（OSS は GitHub でコードをフォークしてサインイン体験を直接カスタマイズできます）。 |
| [IdP 主導の SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)               | アイデンティティプロバイダー主導のエンタープライズシングルサインオンを有効にします。                                                                                      |
| [SAML アプリ](/integrate-logto/saml-app)                                              | SAML プロトコルを介してアプリケーションを統合します。OSS バージョンは 3 つの SAML アプリに制限されています。                                                               |

ヒント：マルチテナンシー、メンバー招待、および MFA は、オープンソースの Logto コンソールにチームがサインインするためには利用できません。ただし、Logto OSS を使用してこれらの機能を独自の製品に実装し、エンドユーザーに提供することができます。
