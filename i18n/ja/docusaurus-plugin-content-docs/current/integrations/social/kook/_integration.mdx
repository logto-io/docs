## KOOK Developer Center でアプリケーションを作成する \{#create-an-application-in-the-kook-developer-center}

ヒント：すでに完了しているセクションはスキップできます。

:::note
OAuth2 アプリケーションを作成する前に、KOOK OAuth2 権限を申請する必要があります。詳細は OAuth2 ページの情報を確認するか、KOOK 公式開発者サーバーで KOOK のスタッフにお問い合わせください。
:::

1. [KOOK Developer Center](https://developer.kookapp.cn/app/index) にアクセスし、KOOK アカウントでサインインします。アカウントをお持ちでない場合は新規登録も可能です。
2. **Create Application** ボタンをクリックしてアプリケーションを作成し、名前（例：LogtoAuth）を入力して **Confirm** をクリックします。
3. 作成したアプリケーションのアイコンをクリックし、**OAuth2** ページに移動します。
4. **Callback URIs** 欄に有効なリダイレクト（例：`${your_logto_origin}/callback/${connector_id}`）を追加します。リダイレクト URI は logto 管理コンソールのコネクター詳細ページで確認できます。
5. **OAuth2 link builder** で **get_user_info** スコープにチェックを入れるのを忘れずに。

## KOOK コネクターを設定する \{#configure-your-kook-connector}

前のセクションで取得した OAuth2 ページの Client ID と Client Secret を clientId および clientSecret フィールドに入力します。

### 設定タイプ \{#config-types}

| Name         | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |

## KOOK コネクターをテストする \{#test-kook-connector}

これで完了です。KOOK コネクターが利用可能になっているはずです。[サインイン体験でコネクターを有効化する](https://docs.logto.io/docs/recipes/configure-connectors/social-connector/enable-social-sign-in/) のもお忘れなく。
