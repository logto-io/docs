## Patreon アカウントでサインイン \{#sign-in-with-patreon-account}

[Patreon のウェブサイト](https://www.patreon.com/) にアクセスし、Patreon アカウントでサインインします。アカウントをお持ちでない場合は、新規登録も可能です。

## OAuth アプリの作成と設定 \{#create-and-configure-oauth-app}

[Patreon OAuth アプリの作成](https://www.patreon.com/portal/registration/register-clients) ガイドに従い、新しいアプリケーションを登録します。

**App Name** に新しい OAuth アプリケーションの名前を入力し、**App URL** にアプリの URL を入力します。**App Description** フィールドは空白のままでも構いません。**Redirect URIs** には `${your_logto_origin}/callback/${connector_id}` を設定します。`connector_id` は Logto 管理コンソールのコネクター詳細ページ上部で確認できます。

:::note
ログイン時に「The redirect_uri MUST match the registered callback URL for this application.」というエラーメッセージが表示された場合は、Patreon OAuth アプリの Redirect URI と Logto アプリのリダイレクト URL（プロトコルを含む）を一致させて問題を解決してください。
:::

## OAuth アプリの管理 \{#managing-oauth-apps}

Patreon の [Clients & API Keys ページ](https://www.patreon.com/portal/registration/register-clients) にアクセスすると、既存の OAuth アプリの追加・編集・削除ができます。また、OAuth アプリの詳細ページで `Client ID` の確認や `Client secrets` の生成も可能です。

## コネクターの設定 \{#configure-your-connector}

前のセクションで取得した _Client ID_ と _Client Secret_ を `clientId` および `clientSecret` フィールドに入力します。

`scope` は [スコープ](https://docs.patreon.com/#scopes) のスペース区切りリストです。指定しない場合、デフォルトで `identity identity[email]` となります。

### 設定タイプ \{#config-types}

| Name         | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
