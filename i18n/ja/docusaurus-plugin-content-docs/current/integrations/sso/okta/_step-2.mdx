1. `App integration name` を提供します。これは、あなたの OIDC アプリケーションの識別子として使用されます。
2. Logto SSO コネクターのコールバック URL を使用して、新しい `Sign-in redirect URIs` を追加します。

これは、Okta が認証 (Authentication) に成功した後にユーザーのブラウザをリダイレクトする URI です。ユーザーが IdP での認証 (Authentication) に成功すると、IdP は認可 (Authorization) コードと共にユーザーのブラウザをこの指定された URI にリダイレクトします。Logto は、この URI から受け取った認可 (Authorization) コードに基づいて認証 (Authentication) プロセスを完了します。

![Okta application settings.webp](./assets/okta_application_settings.webp)

3. アプリケーションにユーザーを割り当てます。

`Assignments` 設定に基づいて、アプリケーションをすべてのユーザーまたは特定のユーザー / グループに割り当てることを選択できます。

![Okta assign users.webp](./assets/okta_assign_users.webp)

`Save` ボタンをクリックして、アプリケーション設定を保存します。
