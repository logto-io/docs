## SMTP コネクターの設定 \{#set-up-smtp-connector}

SMTP は特定のメールサービスプロバイダーに限定されない送信プロトコルであり、すべてのプロバイダーで動作します。

以下のプロバイダーに従ってメールを送信するための SMTP コネクターの使用方法についてのガイドを提供しています：

- _Gmail_ は世界中で最も人気のあるメールサービスベンダーです。
- _Aliyun direct mail_ と _SendGrid mail_。Logto チームが対応するコネクターを提供しているため、これらのメールサービスプロバイダーに馴染みのある方もいるかもしれません。一般的なアイデアを持っていることでしょう。

以下の例を参考にして、他のすべてのメールベンダーのセットアップを理解できることを願っています :rocket:

### Gmail の使用設定 \{#set-up-for-gmail-use}

[Gmail](https://mail.google.com/) で新しい Gmail アカウントを取得するか、既存のアカウントを使用できます。

[Gmail の公式投稿](https://support.google.com/a/answer/176600) では、SMTP コネクターを介して Gmail を操作するために必要なプロパティの値を決定する方法が示されています。

### SendGrid SMTP API との統合 \{#integrate-with-sendgrid-smtp-api}

最初に、すでに SendGrid アカウントを持っていると仮定します。持っていない場合は、[SendGrid のウェブサイト](https://app.sendgrid.com/) で新しいアカウントを作成してください。

["Integrating with the SMTP API"](https://docs.sendgrid.com/for-developers/sending-email/integrating-with-the-smtp-api) のステップバイステップガイドを見つけることができます。

開発者は ["Sender Management"](https://mc.sendgrid.com/senders) で _sender_ の詳細にアクセスできます。

### Aliyun direct mail アカウントの設定 \{#configure-with-aliyun-direct-mail-account}

[Aliyun のウェブサイト](https://cn.aliyun.com/) にサインインします。アカウントを持っていない場合は、新しいアカウントを登録してください。

[SMTP を使用してメールを送信するガイド](https://www.alibabacloud.com/help/en/directmail/latest/send-emails-using-smtp) に従い、必要な設定と情報を取得するための「タスク」を完了してください。

[SMTP サービスアドレスページ](https://www.alibabacloud.com/help/en/directmail/latest/smtp-service-address) にアクセスして、適切な SMTP サービスアドレスホストとポート番号を選択できます。

「送信者アドレス」を確認するには、[DirectMail コンソール](https://dm.console.aliyun.com/) の左側のナビゲーションペインで入り口を見つけることができます。ここで `Sender address` と `SMTP Password` を確認することができます。

:::note
前述のケースでは、シンプルに保つために 1 つのサンプルテンプレートのみが提供されています。他のユースケースのために、より多くのテンプレートを追加する必要があります。
"\<" と ">" で囲まれた値は、Gmail、SendGrid または Aliyun アカウント設定に従って変更し、"\<" と ">" のない他のフィールドはそのままにしておくことを選択してください。
テンプレートのコンテンツに `{{code}}` をプレースホルダーとして追加し、メール送信時にランダムな認証コードを表示します。
:::

### SMTP コネクターのテスト \{#test-smtp-connector}

メールアドレスを入力し、「送信」をクリックして、設定が機能するかどうかを「保存して完了」する前に確認できます。

以上です。「サインイン体験でコネクターを有効にする」ことを忘れないでください。(/connectors/email-connectors/#enable-email-sign-up-or-sign-in)

### 設定タイプ \{#config-types}

| Name      | Type       |
| --------- | ---------- |
| host      | string     |
| port      | string     |
| fromEmail | string     |
| templates | Template[] |

| Template Properties | Type        | Enum values                                             |
| ------------------- | ----------- | ------------------------------------------------------- |
| subject             | string      | N/A                                                     |
| content             | string      | N/A                                                     |
| usageType           | enum string | 'Register' \| 'SignIn' \| 'ForgotPassword' \| 'Generic' |
| contentType         | enum string | 'text/plain' \| 'text/html'                             |

**ユーザー名とパスワード認証オプション**

| Name | Type                   | Enum values |
| ---- | ---------------------- | ----------- |
| user | string                 | N/A         |
| pass | string                 | N/A         |
| type | enum string (OPTIONAL) | 'login'     |

[OAuth2 認証オプション](https://nodemailer.com/smtp/oauth2/) やその他の高度な設定も構成できます。詳細は [こちら](https://nodemailer.com/smtp/) を参照してください。

テキストボックスにすべての設定可能なパラメーターを含む設定例を示しました。独自の設定を行う際に役立ててください。（設定はあなたの責任で行ってください。一部の値はデモンストレーション目的であり、あなたのユースケースに適合しない場合があります。）
