# SMTP を使用してメール認証を設定する

SMTP のための公式 Logto コネクター。

<GuideTip />

## はじめに \{#get-started}

SMTP (Simple Mail Transfer Protocol) は、電子メール送信のためのインターネット標準通信プロトコルです。メールサーバーやその他のメッセージ転送エージェントは、SMTP を使用してメッセージを送受信します。

## SMTP コネクターを設定する \{#set-up-smtp-connector}

SMTP は特定のメールサービスプロバイダーに限定されない送信プロトコルであり、すべてのプロバイダーで動作します。

現在、以下のプロバイダーに従ってメールを送信するための SMTP コネクターの使用方法に関するガイドを提供しています：

- _Gmail_ は世界で最も人気のあるメールサービスベンダーです。
- _Aliyun direct mail_ と _SendGrid mail_。Logto チームが対応するコネクターを提供しているため、これらのメールサービスプロバイダーに慣れている方もいるかもしれません。一般的なアイデアを持っている可能性があります。

以下の例を参考にして、他のすべてのメールベンダーのセットアップを理解できることを願っています :rocket:

### Gmail の使用設定 \{#set-up-for-gmail-use}

[Gmail](https://mail.google.com/) で新しい Gmail アカウントを取得するか、既存のアカウントを使用できます。

[Gmail 公式投稿](https://support.google.com/a/answer/176600) では、SMTP コネクターを介して Gmail を操作するために必要なプロパティの値を決定する方法が示されています。

### SendGrid SMTP API との統合 \{#integrate-with-sendgrid-smtp-api}

最初に、すでに SendGrid アカウントを持っていると仮定します。持っていない場合は、[SendGrid ウェブサイト](https://app.sendgrid.com/) で新しいアカウントを作成してください。

["SMTP API との統合"](https://docs.sendgrid.com/for-developers/sending-email/integrating-with-the-smtp-api) に関するステップバイステップガイドを見つけることができます。

開発者は ["Sender Management"](https://mc.sendgrid.com/senders) で _sender_ の詳細にアクセスできます。

### Aliyun direct mail アカウントの設定 \{#configure-with-aliyun-direct-mail-account}

[Aliyun ウェブサイト](https://cn.aliyun.com/) にサインインします。アカウントを持っていない場合は、新しいアカウントを登録してください。

[SMTP を使用してメールを送信するガイド](https://www.alibabacloud.com/help/en/directmail/latest/send-emails-using-smtp) に従い、必要な設定と情報を取得するための「タスク」を完了してください。

[SMTP サービスアドレスページ](https://www.alibabacloud.com/help/en/directmail/latest/smtp-service-address) にアクセスして、適切な SMTP サービスアドレスホストとポート番号を選択できます。

「送信者アドレス」を確認するには、[DirectMail コンソール](https://dm.console.aliyun.com/) の左側のナビゲーションペインにある入口を見つけることができます。ここで `Sender address` と `SMTP Password` を確認することができます。

> ℹ️ **注意**
>
> 前述のケースでは、シンプルさを保つために 1 つのサンプルテンプレートのみが提供されています。他のユースケースのために、より多くのテンプレートを追加する必要があります。
> Gmail、SendGrid または Aliyun アカウント設定に応じて "\<" と ">" で囲まれた値を変更し、"\<" と ">" のない他のフィールドを保持することを選択してください。
> テンプレートのコンテンツに `{{code}}` をプレースホルダーとして追加し、メール送信時にランダムな認証コードを表示します。

### SMTP コネクターをテストする \{#test-smtp-connector}

メールアドレスを入力し、「送信」をクリックして、設定が「保存して完了」する前に機能するかどうかを確認できます。

以上です。[サインイン体験でコネクターを有効にする](/connectors/email-connectors/#enable-email-sign-up-or-sign-in) のを忘れないでください。

### 設定タイプ \{#config-types}

| 名前      | タイプ     |
| --------- | ---------- |
| host      | string     |
| port      | string     |
| fromEmail | string     |
| templates | Template[] |

| テンプレートプロパティ | タイプ      | 列挙値                                                  |
| ---------------------- | ----------- | ------------------------------------------------------- |
| subject                | string      | N/A                                                     |
| content                | string      | N/A                                                     |
| usageType              | enum string | 'Register' \| 'SignIn' \| 'ForgotPassword' \| 'Generic' |
| contentType            | enum string | 'text/plain' \| 'text/html'                             |

**ユーザー名とパスワードの認証オプション**

| 名前 | タイプ                 | 列挙値  |
| ---- | ---------------------- | ------- |
| user | string                 | N/A     |
| pass | string                 | N/A     |
| type | enum string (OPTIONAL) | 'login' |

[OAuth2 認証オプション](https://nodemailer.com/smtp/oauth2/) やその他の高度な設定も構成できます。詳細は [こちら](https://nodemailer.com/smtp/) を参照してください。

テキストボックス内にすべての設定可能なパラメータを含む設定例を示しました。独自の設定を行う際に役立ててください。（設定はあなたの責任で行ってください。一部の値はデモンストレーション目的であり、あなたのユースケースに適合しない場合があります。）
