# API 保護

[ロールベースのアクセス制御 (RBAC)](/authorization/role-based-access-control) と [JSON Web Token (JWT)](https://auth.wiki/jwt) の検証を実装し、API エンドポイントを不正アクセスから保護する方法を学びましょう。

ガイドでは、ミドルウェアのセットアップ、権限モデル、複数のプログラミング言語やフレームワークでの実践例を紹介しています。

## はじめに \{#get-started}

### API セキュリティが初めての方へ \{#new-to-api-security}

API の認証 (Authentication) と認可 (Authorization) が初めての場合は、まず以下の基本から始めましょう：

- **[認証 (Authentication) と認可 (Authorization)](/concepts/authn-vs-authz)**：主な違いを理解する
- **[ロールベースのアクセス制御 (RBAC)](/authorization/role-based-access-control)**：権限とロールの仕組みを学ぶ
- **[JSON Web Token (JWT)](https://auth.wiki/jwt)**：JSON Web Token がどのように API を保護するかを知る

### 言語やフレームワークを選択 \{#choose-your-language-or-framework}

API 保護の実装を始める準備ができたら、サイドバーから技術スタックを選択し、コード例やベストプラクティス付きのステップバイステップ統合ガイドを参照してください。
