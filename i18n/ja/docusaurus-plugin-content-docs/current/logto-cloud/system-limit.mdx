# システム制限

システム制限は、テナントが作成できるエンティティの最大数を定義します。これらはプラットフォームの安定性を保護するためのもので、課金とは独立しています。料金ページの一部の項目には _無制限_ や _従量課金_ と記載されていますが、これは課金上の上限がないことを意味し、内部的なシステム上限は公平な利用のために引き続き適用されます。増加が必要な場合は個別に審査可能です。

## 制限の種類 \{#types-of-limits}

### システムリソースおよびエンティティ上限 \{#system-resource-and-entity-caps}

アプリケーション、ロール、組織 (Organizations)、コネクター (Connectors)、およびトークン消費量などの一部の利用ベースのシステムリソースの最大数です。これらの制限はプランによって異なります。

### レート制限 \{#rate-limit}

テナントごとに適用されるリクエストレベルの保護です。一般的なガイドラインは **テナントごとに 10 秒あたり 200 リクエスト** です。トラフィックが正当で継続的にこの制限を超える場合は、個別に審査可能です。

---

利用ケースが妥当で制限に達した場合は、お問い合わせください。これにより、制限の調整や実際のシナリオへの対応が可能になります。

## テナントレベルの制限保護 \{#tenant-level-rate-limit-protection}

### 開発テナント \{#dev-tenants}

[開発テナント](/logto-cloud/tenant-settings#development) は、テスト用に無料で利用できます。悪用防止と明確な期待値のため、開発テナントには固定のエンティティ上限があります。さらに必要な場合は、増加申請または Pro へのアップグレードでフルアクセスが可能です。

開発テナントには、別途 [データ保持ポリシー](/logto-cloud/dev-tenant-data-retention) も適用されます。

### エンタープライズテナント \{#enterprise-tenants}

エンタープライズの制限や機能は完全にカスタマイズ可能で、契約で定義されます。詳細は [お問い合わせ](https://logto.io/contact) ください。
