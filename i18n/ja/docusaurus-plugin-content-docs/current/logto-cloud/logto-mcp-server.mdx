# Logto MCP Server

Logto MCP Server はリモート [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) サーバーです。AI アプリケーションに URL を追加し、Logto Cloud アカウントでサインインして、Logto との対話を始めましょう。

:::tip
Logto MCP Server は Logto と [MCP Auth](https://mcp-auth.dev) で構築されています。独自のリモート MCP サーバーを構築したい場合は、[独自のリモート MCP サーバーを構築する](#build-your-own-remote-mcp-server) をご覧ください。
:::

## できること \{#what-you-can-do}

- **プロジェクトへ認証 (Authentication) 機能の追加**：AI がフレームワークを自動検出し、Logto アプリケーションを作成し、動作する統合コードを生成します。React、Next.js、Vue、Nuxt、SvelteKit、Express、Go、Python、iOS、Android、Flutter など 38 以上のフレームワークに対応しています。
- **Logto に関する質問**：AI が Logto ドキュメントを検索し、出典付きで回答します。

今後、サインイン体験の設定、RBAC および権限管理のベストプラクティスガイド、多テナンシーによる B2B アプリ構築のための組織設定など、さらに多くの機能が追加される予定です。

## AI アプリケーションでのセットアップ \{#set-up-in-your-ai-application}

### Claude Code \{#claude-code}

<Tabs>
  <TabItem value="claude-mcp-add" label="claude mcp add">

    ```bash
    claude mcp add --transport http --client-id mcp-client --callback-port 8080 logto https://mcp.logto.io
    ```

  </TabItem>
  <TabItem value="claude-mcp-add-json" label="claude mcp add-json">

    ```bash
    claude mcp add-json logto '{"type":"http","url":"https://mcp.logto.io","oauth":{"clientId":"mcp-client","callbackPort":8080}}'
    ```

  </TabItem>
</Tabs>

`8080` はマシン上の利用可能な任意のポートに変更できます。

その後、Claude Code で `/mcp` を実行し、ブラウザのログインフローに従って Logto Cloud アカウントで認証 (Authentication) してください。

### Claude Desktop \{#claude-desktop}

1. メインメニューから **Settings** を開き、**Connectors** に移動して **Add custom connector** をクリックします。
2. 名前に `Logto MCP Server`、リモート MCP サーバーの URL に `https://mcp.logto.io` を入力します。
3. **Advanced settings** を展開し、OAuth Client ID に `mcp-client` を入力して **Add** をクリックします。
4. **Connectors** リストで **Logto MCP Server** を見つけて **Connect** をクリックします。
5. 最初に Claude のウェブページにリダイレクトされます。アクティブなセッションがない場合は、先に Claude アカウントでサインインしてください。
6. Claude へのサインイン後、Logto Cloud のサインインページにリダイレクトされます。Logto Cloud アカウントでサインインし、接続を許可してください。

### Cursor \{#cursor}

1. コマンドパレット（`Command + Shift + P`）を開き、**View: Open MCP Settings** を選択します。
2. **Installed MCP Servers** セクションで **Add Custom MCP** ボタンをクリックします。
3. 開いた `mcp.json` ファイルに次の設定を追加します：

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud のサインインページが自動的に表示されます。Logto Cloud アカウントでサインインして接続してください。

### OpenCode \{#opencode}

1. `opencode.json` ファイルに次の設定を追加します：

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. 次のコマンドを実行して認証 (Authentication) します：
   ```bash
   opencode mcp auth Logto
   ```
   `Logto` は設定ファイル内の MCP サーバー名に対応しています。

ブラウザウィンドウが開きます。Logto Cloud アカウントでサインインして認可・接続してください。

### VS Code \{#vs-code}

1. コマンドパレット（`Command + Shift + P`）を開き、**MCP: Add Server...** を選択します。
2. サーバータイプとして **HTTP** を選択します。
3. URL に `https://mcp.logto.io` を入力します。ワークスペースにインストールして分離・管理を強化することもできます。
4. OAuth フローが開始されたら、App ID に `mcp-client` を入力します。App Secret は空のまま Enter でスキップしてください。
5. Logto Cloud のサインインページにリダイレクトされます。Logto Cloud アカウントでサインインして接続してください。

### Windsurf \{#windsurf}

1. コマンドパレット（`Command + Shift + P`）を開き、**Windsurf: MCP Marketplace** を選択します。
2. **Installed MCPs** セクションで右側の設定アイコンをクリックし、`~/.codeium/windsurf/mcp_config.json` ファイルを作成・開きます。
3. 次の [設定](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json) を追加します：

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud のサインインページが自動的に表示されます。Logto Cloud アカウントでサインインして接続してください。

### その他の MCP クライアント \{#other-mcp-clients}

Logto MCP Server は、OAuth 認可 (Authorization) をサポートする任意の MCP クライアントで動作します。次の設定を使用してください：

- **MCP サーバー URL:** `https://mcp.logto.io`
- **OAuth クライアント ID:** `mcp-client`
- **リソース:** `https://mcp.logto.io/`（クライアントが必要とする場合）
- **スコープ:** `mcp:all`（クライアントが必要とする場合）

AI アプリケーション開発者で Logto MCP Server への接続をサポートしたいが、Logto MCP Server がまだアプリケーションのリダイレクト URI をサポートしていない場合は、[こちらからご連絡ください](https://logto.io/contact)（アプリケーションの詳細を添えて）。

## 独自のリモート MCP サーバーを構築する \{#build-your-own-remote-mcp-server}

Logto MCP Server は Logto と [MCP Auth](https://mcp-auth.dev) で構築されています。認証 (Authentication) 機能付きの独自リモート MCP サーバーを構築したい場合は、次のリソースをご覧ください：

- [Remote MCP server in action: a new entry point for SaaS products in the AI era](https://blog.logto.io/remote-mcp-server-in-action)
- [Discord に参加](https://discord.gg/fUQmaFyAam)：リモート MCP サーバー構築に関する知見やベストプラクティスを共有
- [MCP Auth](https://mcp-auth.dev)：MCP サーバーに本番運用レベルの認証 (Authentication) を追加するために必要なすべて。仕様書を何週間も読む必要はありません。
