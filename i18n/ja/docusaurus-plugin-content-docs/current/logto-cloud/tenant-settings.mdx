# テナント設定

## テナントリージョン

テナントを作成する際に、テナントデータが保存されるリージョンを選択できます。テナントが作成された後は変更できません。利用可能なリージョンは次のとおりです：

- EU (オランダ)
- US (アメリカ西部)
- AU (オーストラリア)

通常、遅延を最小限に抑え、パフォーマンスを向上させるために、顧客に最も近いリージョンを選択するべきです。

Logto はグローバルエッジネットワークを活用して、アプリケーションに最高のパフォーマンスと可用性を提供します。リクエストルーティングは最適化されており、ユーザーが常に最高のパフォーマンスオプションに接続されるようにしています。

:::note
他のリージョンをお探しですか？ [こちらからお問い合わせください](https://logto.io/contact)。
:::

## テナントタイプ: 開発 vs. 本番

Logto Cloud には、開発と本番の 2 種類のテナントがあります。このテナントの区別により、異なる環境でプロジェクトを効率的に管理しながら、Logto の完全な価値を享受できます。

テナントタイプは作成時にのみ指定でき、一度作成されると編集できないことに注意してください。

## 開発

開発テナント（dev テナント）は主にテスト目的で使用され、本番環境では利用しないでください。これらのテナントでは、有料プランで利用可能なプレミアムおよび有料機能に無料でアクセスでき、サブスクリプションは不要です。

ただし、開発テナントには特定の制限があります：

- Dev テナントは 90 日を超えるユーザーと組織を自動的に削除します。
- サインイン体験中に、テナントが開発モードであることを示すバナーが表示されます。
- 開発テナントには特定の機能にクォータ制限がある場合があります。これらの制限は、該当する場合、機能詳細ページで説明されています。
- Logto は開発テナントのクォータ制限を更新することがあり、事前に通知するよう努めます。

## 本番

本番テナントは、エンドユーザーがライブアプリにアクセスする場所であり、[有料サブスクリプション](https://logto.io/pricing)が必要な場合があります。無料プランまたはプロプランに加入して本番テナントを作成できます。無料プランに加入した場合、最大 10 個のテナントのみ作成できます。
