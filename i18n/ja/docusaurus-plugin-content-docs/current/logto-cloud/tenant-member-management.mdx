# テナントメンバー管理

## コラボレーターを招待する

<CloudLink to="/tenant-settings/members">コンソール > テナント設定 > メンバー</CloudLink> に移動します。現在のメンバーが表示され、さらに招待することができます。メールアドレスを入力し、ロールを割り当て（まとめて行うことができます）、送信します。

Logto では、Logto Cloud のテナントにチームメンバーを 2 つの異なるロールで招待できます：

- **Admin:** すべてのテナントリソースへのフルアクセスと管理。
- **Collaborator:** アイデンティティ関連のリソースを管理できますが、制限があります：
  - 請求情報へのアクセス不可
  - 他のメンバーを招待または管理できない
  - テナント名を変更できない
  - テナントを削除できない

### メンバーの制限

| テナントタイプ     | メンバー制限                                      |
| ------------------ | ------------------------------------------------- |
| 開発テナント       | 最大 3 人のメンバー（詳細はお問い合わせください） |
| 無料の本番テナント | 1 人のメンバー（管理者のみ）                      |
| プロの本番テナント | 3 人のメンバーから始まり、追加メンバーごとに $8   |

### 招待の管理

招待タブには、過去の招待とそのステータスが表示されます

| ステータス   | 説明                                           | 利用可能なアクション |
| ------------ | ---------------------------------------------- | -------------------- |
| 保留中       | 招待が送信されたが、まだ受け入れられていない。 | 再送信               |
| 受け入れ済み | ユーザーは現在テナントメンバーです。           | 削除                 |
| 期限切れ     | 招待が 7 日の制限を超えた。                    | 削除                 |
| 取り消し済み | 管理者が招待をキャンセルした。                 | 削除                 |

すでに保留中の招待がある人やメンバーを招待することはできません。

### テナントメンバーの管理（管理者のみ）

管理者として、チームを管理するためのいくつかの方法があります：

- **メンバーを招待または削除する:** 必要に応じて新しいチームメンバーを追加したり、アクセスを削除したりします。
- **メンバーのロールを変更する:** 必要なアクセスレベルに応じて「admin」または「collaborator」ロールを割り当てます。
- **テナントの所有権を移転する:** 必要に応じて、テナントの所有権を他のメンバーに移転できます。最初に他のメンバーに管理者ロールを付与し、その後、通常のメンバーとして残るか、テナントを完全に離れるかを選択できます。
- **複数の管理者を追加する:** 他のチームメンバーと管理責任を共有し、協力的な管理を行います。
