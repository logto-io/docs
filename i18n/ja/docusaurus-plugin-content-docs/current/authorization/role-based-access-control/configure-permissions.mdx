# 権限 (Permissions) を設定する

## Logto コンソールを使用して API 権限 (Permissions) を設定する

権限 (Permissions) は **API リソース** レベルで設定されます。<CloudLink to="/api-resources">コンソール > API リソース</CloudLink> に移動し、管理したい API リソースをクリックして、「権限 (Permissions)」タブに進みます。

権限 (Permissions) の名前と詳細を提供することが重要です。これにより、権限 (Permissions) を識別し管理しやすくなります。

## Logto Management API を使用して API 権限 (Permissions) を設定する

Logto Management API を使用して管理します。関連するエンドポイントに呼び出しを行います。この [リファレンス](https://openapi.logto.io/operation/operation-listresourcescopes) を確認してください。

| method | path                                                                                                                 | description                                                                               |
| ------ | -------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| GET    | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-listresourcescopes)              | API リソースに定義されたスコープを取得します。                                            |
| POST   | [/api/resources/\{resourceId\}/scopes](https://openapi.logto.io/operation/operation-createresourcescope)             | API リソースの新しいスコープを作成します。                                                |
| DELETE | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-deleteresourcescope) | 指定されたリソースから API リソーススコープを削除します。                                 |
| PATCH  | [/api/resources/\{resourceId\}/scopes/\{scopeId\}](https://openapi.logto.io/operation/operation-updateresourcescope) | 指定されたリソースの API リソーススコープを更新します。このメソッドは部分更新を行います。 |
