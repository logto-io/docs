# 利用規約とプライバシー

プロダクトのオープン性と透明性を保つために、サインインおよびサインアップページに利用規約とプライバシーポリシーへのリンクを追加しましょう。これにより、業界固有のコンプライアンス要件を満たすことができます。

## 利用規約とプライバシーの設定 \{#configure-terms--privacy}

1. <CloudLink to="/sign-in-experience/content">
     コンソール > サインイン & アカウント > コンテンツ
   </CloudLink>
   に移動します。
2. 「利用規約」と「プライバシーポリシー」の外部リンクを追加します。
3. 特定の地域や業界のコンプライアンス要件に応じて、「利用規約への同意」ポリシーを設定します。

:::note
値が空の場合、Logto のサインインフロー内で利用規約やプライバシー同意を管理する必要はありません。ユーザーがアプリケーションにサインインした後に対応できます。
:::

## 利用規約への同意に関するユーザー体験 \{#user-experience-for-agreeing-to-terms}

Logto では、さまざまなコンプライアンス要件に応じて、利用規約への同意に関する複数の事前構築フローを提供しています：

### オプション 1：サインイン / サインアップ継続時に自動的に同意 \{#option-1-agree-to-terms-automatically-on-continue-sign-insign-up}

このポリシーは、ユーザーがサインインまたはサインアップを継続する際に自動的に利用規約へ同意することで、最もスムーズなユーザー体験を提供します。
ユーザーに明示的な同意を求めるプロンプトは表示されません。

<img
  src="/img/assets/auto-agree-to-terms.png"
  alt="サインイン / サインアップ継続時に自動的に同意"
/>

### オプション 2：サインアップ時のみ同意 \{#opetion-2-agree-to-terms-on-sign-up-only}

このポリシーでは、新規ユーザーが **利用規約とプライバシーポリシー** に手動で同意する必要があります。EU の [GDPR](https://gdpr-info.eu/art-4-gdpr/) やカリフォルニア州の [CCPA](https://oag.ca.gov/privacy/ccpa) の下では、企業は個人情報を収集する前にユーザーからインフォームドコンセントを取得する必要があります。

ユーザーが初めてソーシャル登録（例：Google や GitHub）を利用する際、Logto にリダイレクトされた後に **利用規約とプライバシーポリシー** への同意を求めるポップアップが表示されます。この同意は、2 回目以降のソーシャルログインでは不要です。

<img src="/img/assets/sign-up-only-agree-to-terms.png" alt="サインアップ時のみ同意" />

### オプション 3：サインインおよびサインアップ時に同意 \{#option-3-agree-to-terms-on-sign-in-and-sign-up}

このポリシーでは、ユーザーがサインインまたはサインアップするたびに **利用規約とプライバシーポリシー** への同意が必要です。毎回同意が必要なアプリケーションに適しています。ユーザーは、先に進む前にチェックボックスをオンにして同意する必要があります。この方法は、中国の PIPL やブラジルの LGPD に準拠しています。

<img src="/img/assets/sign-in-agree-to-terms.png" alt="サインインおよびサインアップ時に同意" />
