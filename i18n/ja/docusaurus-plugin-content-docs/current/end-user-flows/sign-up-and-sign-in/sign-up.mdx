# メール / 電話番号 / ユーザー名でのサインアップ

ユーザー登録は、ユーザーがアプリケーションと関わるための最初のステップです。Logto は、ユーザー名とパスワード、メールまたは電話番号の確認、[ソーシャルサインアップ](/end-user-flows/sign-up-and-sign-in/social-sign-in)、および [エンタープライズ SSO](/end-user-flows/enterprise-sso) を含むさまざまなサインアップ方法をサポートしています。アプリケーションの要件に最も適したサインアップ方法を設定できます。

<CloudLink to="/sign-in-experience/sign-up-and-sign-in">
  Console > サインイン体験 > サインアップとサインイン
</CloudLink>
にアクセスして、識別子サインアップフローの設定を開始してください。

<img src="/img/assets/sign-in-experience-sign-up.png" alt="サインアップ設定" />

## サインアップ識別子の設定 \{#set-up-the-sign-up-identifier}

Logto で新しいユーザーアカウントを正常に作成するには、ユーザーが Logto のシステム内で一意に識別される **識別子** を提供する必要があります。最初のステップとして、サインアッププロセス中にユーザーが提供しなければならない識別子を選択します。利用可能なオプションは次のとおりです：

- **ユーザー名**：アプリケーションにサインインするために使用できる一意の [ユーザー名](/user-management/user-data#username)。
- **メールアドレス**：アプリケーションにサインインするために使用できる有効な [メールアドレス](/user-management/user-data#primary_email)。
- **電話番号**：アプリケーションにサインインするために使用できる有効な [電話番号](/user-management/user-data#primary_phone)。
- **メールアドレスまたは電話番号**：有効なメールアドレスまたは電話番号のいずれかでサインアップを許可します。

サインアッププロセス中に収集されたすべての識別子は、同じテナント内のユーザー間で一意でなければなりません。これらは [ユーザープロファイル](/user-management/user-data#user-profile) に保存され、Logto と統合されたアプリケーションにサインインするために使用できます。

識別子が選択されていない場合、これは [ソーシャル](/end-user-flows/sign-up-and-sign-in/social-sign-in) のみまたは [エンタープライズ SSO](/end-user-flows/enterprise-sso) のみのサインアップ方法に適用されます。

サインアップ時にユーザーが最初に提供する識別子を優先するために、サインアップ識別子の順序を調整できます。この順序はサインアッププロセスに反映され、最初の識別子が初期登録画面に表示され、残りは後続のステップで収集されます。

## サインアップ確認設定の設定 \{#set-up-the-sign-up-verification-settings}

ユーザーのサインアップおよび将来のサインインプロセスのセキュリティを確保するために、サインアッププロセス中に収集する識別子の確認設定を構成する必要があります。利用可能な設定は次のとおりです：

- **パスワードを作成する**：サインアップ時に、サインイン体験設定で構成されたパスワードポリシーに準拠したパスワードをユーザーに作成させます。このパスワードは、ユーザーの識別子と共に、アプリケーションにサインインするための資格情報として機能します。サインアップ識別子として **ユーザー名** を設定した場合、この要件は自動的に有効になります。**ユーザー名** はパスワードと共に使用することで、ユーザーのアイデンティティを効果的に確認できます。[パスワードポリシー](/security/password-policy) は、セキュリティ要件に合わせてカスタマイズできます。
- **サインアップ時に確認**：サインアップ時にメールアドレスまたは電話番号を確認するようユーザーに要求します。現在、Logto は確認済みのメールと電話番号のみを識別子として受け入れます。この設定は、サインアップ識別子として **メールアドレス** または **電話番号** が使用される場合に自動的に有効になります。ユーザーは、サインアッププロセス中にメールまたは電話番号に送信された確認コードを入力して所有権を確認する必要があります。

| 識別子               | ユーザーパスワードの作成 | サインアップ時に確認 |
| -------------------- | ------------------------ | -------------------- |
| ユーザー名           | 任意                     | 該当なし             |
| メールアドレス       | 任意                     | 必須                 |
| 電話番号             | 任意                     | 必須                 |
| メールまたは電話番号 | 任意                     | 必須                 |

## ソーシャルまたはエンタープライズ SSO でのサインアップ \{#sign-up-with-social-or-enterprise-sso}

これらの従来の識別子サインアップ方法に加えて、Logto はソーシャルおよびエンタープライズ SSO アイデンティティプロバイダーを使用したパスワードレスサインアップもサポートしており、オンボーディングプロセスをよりシームレスでユーザーフレンドリーにします。

Logto で [ソーシャルコネクター](/connectors/social-connectors) または [エンタープライズ SSO コネクター](/connectors/enterprise-connectors) が設定され有効になっている場合、ユーザーはコネクターによって提供される既存のソーシャルまたはエンタープライズアイデンティティを使用して簡単にサインアップできます。ソーシャルおよびエンタープライズ SSO サインアップ方法では、パスワードの作成やメールアドレスまたは電話番号の確認などの追加ステップを回避できます。Logto は、確認済みのソーシャルまたはエンタープライズアイデンティティを通じてユーザーの情報を自動的に同期し、ユーザープロファイルに保存します。

ソーシャルおよびエンタープライズ SSO コネクターを使用したサインアップフローについて詳しくは、[ソーシャルサインイン](/end-user-flows/sign-up-and-sign-in/social-sign-in/) および [エンタープライズ SSO](/end-user-flows/enterprise-sso/) セクションを確認してください。

:::note
注意: カスタムサインアップフローについては、[Bring your UI](/customization/bring-your-ui/) 機能を参照してください。
:::

## よくある質問 \{#faqs}

<details>
  <summary>

### サインアップ時に追加のユーザー情報を収集する \{#collect-additional-user-information-on-sign-up}

</summary>

サインアップ時に追加のユーザープロファイル情報（例：フルネーム、誕生日、会社名）を収集するには、現在次の 2 つのオプションがあります：

- **セルフホスト型オンボーディングフロー**：サインアップが成功した後、カスタマイズ可能なデータ収集のためにユーザーを独自のオンボーディングフローにリダイレクトします。
- **独自の UI を持ち込む**：ユーザープロファイル設定 API を使用して、[独自の UI を持ち込む](/customization/bring-your-ui) 機能を使用して独自のサインアップフローをカスタマイズします。

</details>

<details>
  <summary>

### 管理者が作成したユーザー / 招待されたユーザー \{#admin-created-users--invited-users}

</summary>

[招待のみのサインアップフローを実装する方法](/end-user-flows/sign-up-and-sign-in/disable-user-registration/#implement-an-invitation-only-sign-up-flow) を学びます。

</details>

<details>
  <summary>

### ウェブサイトに埋め込まれたサインアップフォーム \{#embedded-sign-up-forms-on-your-website}

</summary>
  
Logto は現在、サインインおよびサインアップのためのヘッドレス API をサポートしていません。独自のサインアップフォームを Logto にアップロードするには [Bring your UI](/customization/bring-your-ui/) 機能を使用するか、サインインパラメーターを使用してウェブサイトから Logto にユーザー情報を入力できます。ユーザー識別子の入力について詳しくは、[認証パラメーター](/end-user-flows/authentication-parameters/) を参照してください。

</details>

<details>
  <summary>

### 新しいユーザーへのウェルカムメールの送信 \{#sending-welcome-emails-to-new-users}

</summary>

`User.Created` Webhook イベントを購読して、新しいユーザーにウェルカムメールを送信します。[Webhook イベント](/developers/webhooks/webhooks-events/#data-mutation-hook-events) について詳しく学びます。

</details>

<details>
  <summary>

### サインアップ時のメール確認をスキップする \{#skip-email-verification-on-sign-up}

</summary>

現在、Logto は確認済みのメールと電話番号のみを識別子としてサポートしています。確認プロセスは、ユーザーの識別子のセキュリティと所有権を確保するために必要です。未確認のメールまたは電話番号のサポートは、私たちの [ロードマップ](https://logto.productlane.com/roadmap) にあります。更新情報をお待ちください！

</details>
