# メール / 電話番号 / ユーザー名でのサインアップ

ユーザー登録は、ユーザーがアプリケーションと関わるための最初のステップです。Logto は、ユーザー名とパスワード、メールまたは電話番号の認証 (Authentication)、[ソーシャルサインアップ](/end-user-flows/sign-up-and-sign-in/social-sign-in)、および [エンタープライズシングルサインオン (SSO)](/end-user-flows/enterprise-sso) を含むさまざまなサインアップ方法をサポートしています。アプリケーションの要件に最も適したサインアップ方法を設定できます。

<CloudLink to="/sign-in-experience/sign-up-and-sign-in">
  Console > Sign-in experience > Sign-up and sign-in
</CloudLink>
にアクセスして、識別子サインアップフローの設定を開始してください。

<img src="/img/assets/sign-in-experience-sign-up.png" alt="サインアップ設定" />

## サインアップ識別子の設定 \{#set-up-the-sign-up-identifier}

Logto で新しいユーザーアカウントを正常に作成するには、ユーザーが Logto のシステム内で一意に識別される **識別子** を提供する必要があります。最初のステップとして、サインアッププロセス中にユーザーから収集する必要がある **識別子** を設定します。利用可能なオプションは次のとおりです：

- **ユーザー名**：アプリケーションにサインインするために使用できる一意の [ユーザー名](/user-management/user-data#username)。
- **メールアドレス**：アプリケーションにサインインするために使用できる有効な [メールアドレス](/user-management/user-data#primary_email)。
- **電話番号**：アプリケーションにサインインするために使用できる有効な [電話番号](/user-management/user-data#primary_phone)。
- **メールアドレスまたは電話番号**：有効なメールアドレスまたは電話番号のいずれかでサインアップを許可します。
- **該当なし**：サインアップ識別子が有効になっていません。これは、[ソーシャル](/end-user-flows/sign-up-and-sign-in/social-sign-in) のみまたは [エンタープライズシングルサインオン (SSO)](/end-user-flows/enterprise-sso) のみのサインアップ方法に適用されます。

サインアッププロセス中に収集されたすべての識別子は、同じテナント内のユーザー間で一意である必要があります。これらは [ユーザープロファイル](/user-management/user-data#user-profile) に保存され、Logto と統合されたアプリケーションにサインインするために使用できます。

## サインアップ認証 (Authentication) 設定の設定 \{#set-up-the-sign-up-verification-settings}

ユーザーのサインアップと将来のサインインプロセスのセキュリティを確保するために、サインアッププロセス中に収集する識別子の認証 (Authentication) 設定を構成する必要があります。利用可能な設定は次のとおりです：

- **パスワードを作成する**：サインアップ時に、サインイン体験設定で構成されたパスワードポリシーに準拠したパスワードを作成するようユーザーに要求します。このパスワードは、ユーザーの識別子と共に、アプリケーションにサインインするための資格情報として機能します。サインアップ識別子として **ユーザー名** を設定した場合、この要件は自動的に有効になります。**ユーザー名** はパスワードと共にのみ使用でき、ユーザーのアイデンティティを効果的に確認します。[パスワードポリシー](/end-user-flows/sign-up-and-sign-in/reset-password#set-up-password-policy) は、セキュリティ要件に合わせてカスタマイズできます。
- **サインアップ時に認証 (Authentication)**：サインアップ時にメールアドレスまたは電話番号を認証 (Authentication) するようユーザーに要求します。現在、Logto は認証 (Authentication) 済みのメールと電話番号のみを識別子として受け入れます。この設定は、サインアップ識別子として **メールアドレス** または **電話番号** が使用されると自動的に有効になります。ユーザーは、サインアッププロセス中にメールまたは電話番号に送信された認証 (Authentication) コードを入力して所有権を確認する必要があります。

| 識別子               | ユーザーパスワードの作成 | サインアップ時に認証 (Authentication) |
| -------------------- | ------------------------ | ------------------------------------- |
| ユーザー名           | 必須                     | 該当なし                              |
| メールアドレス       | 任意                     | 必須                                  |
| 電話番号             | 任意                     | 必須                                  |
| メールまたは電話番号 | 任意                     | 必須                                  |
| 該当なし             | 該当なし                 | 該当なし                              |

## ソーシャルまたはエンタープライズシングルサインオン (SSO) でのサインアップ \{#type-4-email-address-with-verification-username-and-password-creation}

これらの従来の識別子サインアップ方法に加えて、Logto はソーシャルおよびエンタープライズシングルサインオン (SSO) アイデンティティプロバイダーを使用したパスワードレスサインアップもサポートしており、オンボーディングプロセスをよりシームレスでユーザーフレンドリーにします。

Logto で [ソーシャルコネクター](/connectors/social-connectors) または [エンタープライズシングルサインオン (SSO) コネクター](/connectors/enterprise-connectors) が設定され有効になっている場合、ユーザーはコネクターによって提供される既存のソーシャルまたはエンタープライズアイデンティティを使用して簡単にサインアップできます。ソーシャルおよびエンタープライズシングルサインオン (SSO) サインアップ方法では、パスワードの作成やメールアドレスまたは電話番号の認証 (Authentication) などの追加のステップを省略できます。Logto は、ユーザーの情報を認証 (Authentication) 済みのソーシャルまたはエンタープライズアイデンティティを通じて自動的に同期し、ユーザープロファイルに保存します。

ソーシャルおよびエンタープライズシングルサインオン (SSO) コネクターを使用したサインアップフローについて詳しくは、[ソーシャルサインイン](/end-user-flows/sign-up-and-sign-in/social-sign-in/) および [エンタープライズシングルサインオン (SSO)](/end-user-flows/enterprise-sso/) セクションを確認してください。

:::note
注意: カスタムサインアップフローについては、[Bring your UI](/customization/bring-your-ui/) 機能を参照してください。
:::

## よくある質問 \{#sign-up-with-social-or-enterprise-sso}

<details>
  <summary>

### サインアップ時に追加のユーザー情報を収集する \{#faqs}

</summary>
  
サインアップ時に追加のユーザープロファイル情報（例：フルネーム、誕生日、会社名）を収集するには、次の 2 つのオプションがあります：

- **セルフホスト型オンボーディングフロー**：サインアップが成功した後、カスタマイズ可能なデータ収集のためにユーザーを独自のオンボーディングフローにリダイレクトします。
- **独自の UI を持ち込む**：ユーザープロファイル設定 API を使用して、独自のサインアップフローをカスタマイズします（近日公開予定）。

</details>

<details>
  <summary>

### ウェブサイトに埋め込まれたサインアップフォーム \{#admin-created-users--invited-users}

</summary>
  
Logto は現在、サインインおよびサインアップのためのヘッドレス API をサポートしていません。独自のサインアップフォームを Logto にアップロードするには [Bring your UI](/customization/bring-your-ui/) 機能を使用するか、サインインパラメーターを使用してウェブサイトから Logto にユーザー情報を入力します。ユーザー識別子の入力について詳しくは、[認証 (Authentication) パラメーター](/end-user-flows/authentication-parameters/) を参照してください。

</details>

<details>
  <summary>

### 新しいユーザーへのウェルカムメールの送信 \{#embedded-sign-up-forms-on-your-website}

</summary>

`User.Created` Webhook イベントを購読して、新しいユーザーにウェルカムメールを送信します。[Webhook イベント](/developers/webhooks/webhooks-events/#data-mutation-hook-events) について詳しく学びます。

</details>

<details>
  <summary>

### サインアップ時のメール認証 (Authentication) をスキップする \{#sending-welcome-emails-to-new-users}

</summary>

現在、Logto は認証 (Authentication) 済みのメールと電話番号のみを識別子としてサポートしています。認証 (Authentication) プロセスは、ユーザーの識別子のセキュリティと所有権を確保するために必要です。
未認証 (Authentication) のメールまたは電話番号のサポートは、私たちの [ロードマップ](https://logto.productlane.com/roadmap) にあります。更新情報をお待ちください！

</details>
