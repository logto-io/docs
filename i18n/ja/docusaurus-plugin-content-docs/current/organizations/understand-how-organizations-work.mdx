# 組織の仕組みを理解する

全体的な理解を得るためには、組織機能に関与するエンティティを理解することが重要です。これらは、アプリに組織を統合するための基本です。

## 組織 (Organization) \{#organization}

組織 (Organization) は、[ユーザー (アイデンティティ)](/user-management/user-data) のグループで構成されます。これは、あなたのアプリケーションにアクセスできるチーム、ビジネス顧客、パートナー企業を表すことができます。

エンティティとしての組織の導入は重要です。なぜなら、ユーザーをグループ化するだけでなく、マルチテナントアプリにおけるテナント分離のコンテキストも提供するからです。

## 組織メンバー \{#organization-member}

Logto では、組織のメンバーシップを持つユーザーは、その組織のコンテキスト内で組織メンバー（つまりメンバー）と呼ばれます。

## 組織 M2M アプリケーション \{#organization-m2m-application}

組織 M2M アプリケーションは、組織に関連付けられた [マシン間通信 (M2M) アプリケーション](/quick-starts/m2m) を指します。[組織メンバー](#organization-member) と同様に、組織アプリケーションは組織のコンテキスト内でスコープされます。

アプリケーションを組織に関連付けると、そのアプリケーションに [組織ロール](/authorization/organization-template/configure-organization-template#create-organization-role) を割り当てることができます。これは、組織リソースへの非インタラクションアクセスを許可したい場合に便利です。

:::note

他のタイプのアプリケーションは、ユーザーインタラクション用に設計されているため、組織の関連付けや組織ロールをサポートしていません。

:::

組織、メンバー、およびアプリケーションを管理するには、この [セクション](/organizations/organization-management) を参照してください。

## 組織テンプレート \{#organization-template}

組織テンプレートは、マルチテナントアプリのアクセス制御ニーズのために設計されています。これには、組織ロールと組織権限が含まれます。組織におけるアクセス制御設計について詳しく知るには、この [組織テンプレート](/authorization/organization-template) を確認してください。
