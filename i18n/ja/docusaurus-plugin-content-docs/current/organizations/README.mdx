# 組織 (Organizations)

組織 (Organizations) 機能は、マルチテナント SaaS や [ビジネス向け (B2B) アプリ](/introduction/plan-your-architecture/b2b) で特に効果的です。個人ユーザーに加えて、クライアントはチーム、組織、または企業全体で構成される場合もあります。

この基本要素を活用することで、[マルチテナンシーアプリ](https://auth.wiki/multi-tenancy) に必須の機能を構築できます。例えば：

- 複数の組織で利用できるプロダクト。
- [招待](/end-user-flows/organization-experience/invite-organization-members) または [ジャストインタイム](/organizations/just-in-time-provisioning) ベースでの組織メンバーの提供。
- 組織内のメンバーに割り当てられたロールによって定義される [アクセス制御](/authorization/organization-template)。
- 組織と [エンタープライズシングルサインオン (SSO)](/end-user-flows/enterprise-sso) 体験の連携。

「組織 (Organization)」という用語は、「ワークスペース」「チーム」「会社」など、他の形でも使われます。Logto では、マルチテナンシーの概念を表す一般的な用語として「組織 (Organization)」を使用しています。

## 計画時の考慮事項 \{#planning-considerations}

組織 (Organizations) を利用する際、統合にはさまざまなレベルの工数がかかる場合があります。開発をカスタマイズする際は、以下を参考にしてください。

- **組織体験**：どのような組織体験を提供しますか？組織管理者が自分で組織を作成できますか？組織管理者はメンバーを招待できますか？管理者が組織メンバーのアイデンティティ関連情報を自己管理できるカスタムダッシュボードを構築する予定ですか？
- **ロールとアクセス制御**：アプリケーションで、ユーザーに組織内で特定のロールを割り当てる必要がありますか？
- **Logto Console および Management API での組織管理**：Logto Console および Management API で直接設定したい構成は何ですか？
