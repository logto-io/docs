# 組織データ構造

組織レベルでは、値を作成、更新、読み取り、削除 (CRUD) できるいくつかの属性があります。

## 組織 ID \{#organization-id}

_組織 ID_ は、各組織の一意の識別子です。組織レベルのサインイン体験を実装し、組織トークンを取得するのに役立ちます。

## 名前 \{#name}

_名前_ は、組織レベルのサインインをサポートし、必要に応じて組織レベルの製品インターフェースに統合できます。

## 説明 \{#description}

_説明_ フィールドでは、組織を識別しラベル付けするためのテキストを追加できます。

## 組織ロゴ \{#organization-logos}

サインイン体験でクライアントの組織ロゴを動的に表示するには、組織設定ページに組織ロゴをアップロードできます。

詳細については、[組織固有のロゴ](/customization/match-your-brand/#organization-specific-branding) を参照してください。

## カスタムデータ \{#custom-data}

_カスタムデータ_ は、組織に関する追加情報を保存するために使用される JSON オブジェクトです。開発目的で追加の組織詳細を保存するための柔軟性を開発者に提供します。

## MFA が必要かどうか \{#is-mfa-required}

_isMfaRequired_ は、組織に対して多要素認証 (MFA) が必須かどうかを示します。`true` に設定されている場合、すべてのメンバーはサインイン時に MFA をバイパスして組織にアクセスする必要があります。このセキュリティポリシー設定は、組織レベルで構成されます。

詳細については、[組織の管理](/organizations/organization-management#require-mfa-for-organization-members) を参照してください。

## 作成日時 \{#created-at}

_createdAt_ は、組織が作成されたときのタイムゾーン付きのタイムスタンプです。

## テナント ID \{#tenant-id}

_tenantId_ は、組織が属するテナントを識別します。
