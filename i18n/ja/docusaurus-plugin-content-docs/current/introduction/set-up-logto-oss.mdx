# Logto OSS をセットアップする

このセクションでは、[Logto オープンソースサービス (OSS)](https://github.com/logto-io/logto) の製品を効果的に提供し、開発ワークフローを管理するための基本的なセットアップ手順と主要なアクションについて説明します。

## Logto OSS を始める

[開始ガイド](/logto-oss/get-started-with-oss/) に従って、今日 Logto を起動し、後で本番環境での使用のために完全な [デプロイメントガイド](/logto-oss/deployment-and-configuration) を参照できます。

## Logto OSS がサポートする機能

Logto OSS は、Logto サービスのほとんどのコア機能をサポートしており、定期的に更新および保守されています。

一部の高度な機能は現在 Logto Cloud バージョンにのみ限定されており、[Logto Protected App](/integrate-logto/protected-app)、[無料メールサービス](/connectors/email-connectors/built-in-email-service)、[IdP 主導の SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)、および [SAML IdP](/integrate-logto/third-party-applications) などがあります。

## Logto リリースで最新情報を入手する

Logto インスタンスを最新の機能で最新の状態に保つために、[Logto GitHub Releases](https://github.com/logto-io/logto/releases) ページをフォローしてください。すべてのリリースログをそこに見つけることができます。

コードやデータベーススキーマを変更せずに Logto をアップグレードする方法については、[アップグレードガイド](/logto-oss/upgrading-oss-version) をお読みください。
