# Logto OSS のセットアップ

このセクションでは、[Logto オープンソースサービス (OSS)](https://github.com/logto-io/logto) を効果的に導入し、製品を提供・開発ワークフローを管理するための基本的なセットアップ手順と主要なアクションについて説明します。

## Logto OSS のはじめかた \{#get-started-with-logto-oss}

[はじめかたガイド](/logto-oss/get-started-with-oss/) に従って、今すぐ Logto を起動できます。後で本番運用向けの [デプロイメントガイド](/logto-oss/deployment-and-configuration) も参照してください。

## Logto OSS でサポートされている機能 \{#feature-supported-by-logto-oss}

Logto OSS は Logto サービスのほとんどのコア機能をサポートしており、定期的にアップデート・メンテナンスされています。

一部の高度な機能は現在 Logto Cloud バージョン限定となっています。主なものは以下の通りです：

| OSS での機能制限                                                                    | 説明                                                                                                                           |
| ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| [Logto コンソール：マルチテナント](/logto-cloud/tenant-settings)                    | コンソールで複数の Logto テナントを作成・管理できます。                                                                        |
| [Logto コンソール：コラボレーター招待](/logto-cloud/tenant-member-management)       | 1 つの Logto テナントに複数メンバーを招待・管理できます。                                                                      |
| Logto コンソール：MFA 有効化                                                        | Logto コンソールへのサインイン時に多要素認証 (MFA) を必須にしてセキュリティを強化します。                                      |
| [Logto Protected App](/integrate-logto/protected-app)                               | Cloudflare による迅速かつ SDK 不要の認証統合。                                                                                 |
| [Logto 組み込みメールサービス](/connectors/email-connectors/built-in-email-service) | 組み込みかつ無料のメールサービスを利用してメール配信が可能です。                                                               |
| [Bring your UI](/customization/bring-your-ui)                                       | この機能でサインイン体験の UI やフローをカスタマイズできます（OSS では GitHub 上でコードをフォークして直接カスタマイズ可能）。 |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)               | アイデンティティプロバイダー (IdP) 主導のエンタープライズシングルサインオン (SSO) を有効化。                                   |
| [SAML アプリ](/integrate-logto/saml-app)                                            | SAML プロトコル経由でアプリケーションを統合。OSS 版では SAML アプリは 3 つまでに制限されています。                             |
| [Logto ブランディング非表示](/customization/match-your-brand#hide-logto-branding)   | サインイン体験から「Powered by Logto」マークを削除できます。                                                                   |

ヒント：マルチテナンシー、メンバー招待、MFA は、オープンソース Logto コンソールへのチームサインインでは利用できません。ただし、Logto OSS を使って自社製品にこれらの機能を実装し、エンドユーザー向けに提供することは可能です。

## Logto のリリース情報をチェック \{#stay-updated-with-logto-releases}

Logto インスタンスを最新機能で保つために、[Logto GitHub Releases](https://github.com/logto-io/logto/releases) ページをフォローしてください。すべてのリリースログが掲載されています。

[アップグレードガイド](/logto-oss/upgrading-oss-version) を読んで、コードやデータベーススキーマを変更せずに Logto をアップグレードする方法を学べます。
