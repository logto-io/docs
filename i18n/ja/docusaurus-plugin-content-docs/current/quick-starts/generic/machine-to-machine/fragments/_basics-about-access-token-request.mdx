M2M アプリは、`POST` リクエストをトークンエンドポイントに送信し、HTTP リクエストのエンティティボディに `application/x-www-form-urlencoded` 形式で次のパラメーターを追加してアクセス トークン (アクセス トークン) を取得します：

- **grant_type**：`client_credentials` に設定する必要があります
- **resource**：アクセスしたいリソース
- **scope**：アクセスリクエストのスコープ

また、トークンエンドポイントが M2M アプリを認証 (Authentication) できるように、リクエストヘッダーに M2M アプリの認証情報も含める必要があります。

これは、リクエストの `Authorization` ヘッダーに [Basic 認証 (Authentication)](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Authorization#basic_authentication) 形式でアプリの認証情報を含めることで実現します：`{App ID}:{App Secret}` を base64 エンコードし、`Basic` プレフィックスの後に値として設定します。
