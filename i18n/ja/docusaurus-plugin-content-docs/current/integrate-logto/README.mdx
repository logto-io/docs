# Logto 認証 (Authentication) の統合

Logto は、Web、モバイル、デスクトップアプリケーション向けの包括的な認証 (Authentication) ソリューションを提供し、[マシン間通信 (M2M)](/quick-starts/m2m) 認証 (Authentication) をサービス間でサポートします。また、[OpenID Connect (OIDC)](https://auth.wiki/openid-connect) や [OAuth 2.0](https://auth.wiki/oauth-2.0) などの標準プロトコルを通じて [サードパーティアプリケーション](/integrate-logto/third-party-applications) のためのアイデンティティプロバイダー (IdP) としても機能します。

ニーズに最適なソリューションを選択して、統合を始めましょう：

## アプリケーションへ認証 (Authentication) 機能の追加 \{#add-authentication-for-your-applications}

ユーザー向けアプリケーション（Web、モバイル、デスクトップアプリなど）や、サービス間通信のためのマシン間通信 (M2M) アプリケーションを構築する場合でも、Logto を統合することで、包括的な [認証 (Authentication)](/end-user-flows) および [ユーザー管理](/user-management) 機能をすばやく実装できます。

OIDC 標準に基づき、Logto はすべてのアプリケーションで **Omni サインイン** を実現します。複数のアプリケーションを Logto と統合すると、同じアイデンティティシステムと認証 (Authentication) 方法を共有できます。つまり、ユーザーは一度サインインするだけで、すべての連携アプリケーションにシームレスにアクセスでき、統一された認証 (Authentication) 体験を得られます。

## SAML アプリの追加 \{#add-saml-apps}

Logto（アイデンティティプロバイダー (IdP) として）は、SAML プロトコルベースのアプリケーション（サービスプロバイダーとして）との統合をサポートしています。SAML アプリ統合を通じて、エンタープライズユーザーに安全で標準化されたシングルサインオン (SSO) 体験を提供できます。

## サードパーティ OIDC / OAuth アプリの追加 \{#add-third-party-oidc-oauth-apps}

サードパーティアプリケーションが Logto のアイデンティティシステムで認証 (Authentication) できるようにしたい場合は、Logto を [アイデンティティプロバイダー (IdP)](https://auth.wiki/identity-provider) として設定できます。

Logto は OAuth や OIDC などの標準プロトコルを通じて IdP として利用でき、外部サービスは「Logto でサインイン」を「Google でサインイン」と同様に実装できるため、ユーザーは Logto のユーザーシステムを使って認証 (Authentication) できます。

ファーストパーティアプリケーションとは異なり、サードパーティアプリは限定された権限のみリクエストできます。認証 (Authentication) フロー中、ユーザーは **同意画面** で、サードパーティアプリがリクエストしているデータや権限を確認し、承認することができます。

Logto でサードパーティアプリケーションを作成することで、アイデンティティシステムと外部サービス間の安全な接続を確立し、ユーザーにデータ共有のコントロールを提供できます。

## Management API で Logto 機能を拡張 \{#extend-logto-features-with-management-api}

Logto [Management API](https://openapi.logto.io/) を利用すると、アプリケーション内で Logto の機能をプログラムによって制御・自動化できます。

Logto の機能を拡張してプロダクトを強化したい場合（例：[マルチテナント組織の管理](/organizations) や Logto を使ったワークフローの自動化など）、Logto Management API を活用して、プロダクトニーズに合わせた高度な機能を構築できます。
