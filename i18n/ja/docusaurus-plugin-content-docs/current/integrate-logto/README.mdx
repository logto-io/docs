# Logto を統合する

Logto は、Web、モバイル、デスクトップアプリケーション向けに包括的な認証 (Authentication) ソリューションを提供し、サービス間の [マシン間通信 (M2M)](/quick-starts/m2m) 認証 (Authentication) をサポートし、[サードパーティアプリケーション](/integrate-logto/third-party-applications) に対して [OpenID Connect (OIDC)](https://auth.wiki/openid-connect) や [OAuth 2.0](https://auth.wiki/oauth-2.0) などの標準プロトコルを通じてアイデンティティプロバイダー (IdP) として機能できます。

ニーズに最も合ったソリューションを選択して統合を開始してください：

## アプリケーションに認証 (Authentication) を追加する

ユーザー向けアプリケーション（Web、モバイル、デスクトップアプリなど）やサービス間通信のためのマシン間通信 (M2M) アプリケーションを構築している場合でも、Logto を統合することで、包括的な [認証 (Authentication)](/end-user-flows) と [ユーザー管理](/user-management) 機能を迅速に実装できます。

## Logto をアイデンティティプロバイダーとして使用する

サードパーティアプリケーションが Logto のアイデンティティシステムで認証 (Authentication) できるようにしたい場合、Logto を [アイデンティティプロバイダー (IdP)](https://auth.wiki/identity-provider) として構成できます。

Logto は、OAuth、OIDC、または SAML（近日公開予定）などの標準プロトコルを通じて IdP として使用でき、外部サービスは「Logto でサインイン」を「Google でサインイン」と同様に実装し、ユーザーが Logto ユーザーシステムを使用して認証 (Authentication) できるようにします。

Logto でサードパーティアプリケーションを作成することで、アイデンティティシステムと外部サービス間の安全な接続を確立します。

## Management API で Logto の機能を拡張する

Logto [Management API](https://openapi.logto.io/) は、アプリケーション内で Logto の機能をプログラム的に制御および自動化することを可能にします。

Logto の能力を拡張して製品を強化する必要がある場合（例えば、[マルチテナント組織の管理](/organizations) や Logto を使用したワークフローの自動化など）、Logto Management API を活用して製品のニーズに合わせた高度な機能を構築できます。
