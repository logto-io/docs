# 原生 JS：集成 `@logto/browser`

:::tip
Browser SDK 是 [框架无关](https://stackoverflow.com/questions/64725017/what-does-it-mean-by-framework-agnostic) 的，也就是说只要你写一层适配框架的 SDK 包装，即可在任意一种前端框架中使用。这也是 `@logto/react` 和 `@logto/vue` 做的事。
:::

## 初始化 LogtoClient

引入并通过配置初始化 `LogtoClient`：

```tsx
import LogtoClient from '@logto/browser';

const logtoClient = new LogtoClient({
  endpoint: '<your-logto-endpoint>', // 例如 http://localhost:3001
  appId: '<your-application-id>',
});
```

:::note
在接下来的代码片段中，我们假设你的应用运行在 `http://localhost:3000`。
:::

### 实现登录按钮

<SignInNote calling=".signIn()" />

返回你的 IDE 或编辑器，使用如下代码来实现一个登录按钮：

```html
<button onclick="logtoClient.signIn('http://localhost:3000/callback')">登录</button>
```

### 处理重定向

马上就要大功告成！在上一步，我们将 `http://localhost:3000/callback` 用作 Redirect URI，现在我们需要对其妥善处理。在你的 `/callback` 路由下插入如下代码：

```ts
try {
  await logtoClient.handleSignInCallback();
  console.log(logtoClient.isAuthenticated); // true
} catch {
  // 处理错误
}
```

现在可以测试登录流程了。

## 为后端 API 请求授权

<GetAuthorizationToken />

在启动时将这些资源的 **API Identifier** 作为配置传给 SDK。

```js
import LogtoClient from '@logto/browser';

const logtoClient = new LogtoClient({
  endpoint: '<your-logto-endpoint>', // E.g. http://localhost:3001
  appId: '<your-application-id>',
  resources: ['<your-api-resource>'],
});
```

在发出 API 请求之前，你需要从 Logto 处获取授权令牌：

```js
const token = await logtoClient.getAccessToken('<your-target-api-resource>');
```
