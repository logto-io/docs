# 📱 应用

## 简介

_应用_ 是用于标识你接入 Logto 中的应用程序，你将通过它来使用 OIDC 服务、配置登录体验、查看审计日志等。

如果你想要在你的应用程序中集成 Logto，详见[在你的应用中集成 Logto](../../recipes/integrate-logto)。

## 属性

### 应用 ID

_应用 ID_ 是一个自动生成的用来在 Logto 中标识你应用的唯一 key，在 OAuth 2.0 中用作 [client id](https://www.oauth.com/oauth2-servers/client-registration/client-id-secret/)。

### 应用类型

_应用_ 应为以下的几种类型之一：

<ApplicationTypesIntro />

### 应用名称

_应用名称_ 是在控制台中显示的可读的应用名称。

### 描述

_描述_ 是在控制台应用详情页中显示的关于应用的简短的描述。

### Authorization Endpoint

_Authorization Endpoint_ 是一个 OIDC 术语，该 Endpoint 用来进行用户的身份验证。

在 Logto 中，_Authorization Endpoint_ 将为用户唤起授权流程。

详见 [Authorization Endpoint](https://openid.net/specs/openid-connect-core-1_0.html#AuthorizationEndpoint) 以了解更多信息。

### Redirect URIs

允许用户在 Logto 登录后将重定向回应用程序的 URI 列表。

它将用于验证授权请求中使用的 Redirect URI，因此登录时每个要使用的 Redirect URI 都应预先添加到此允许列表中。

详见 [Redirection Endpoint](https://datatracker.ietf.org/doc/html/rfc6749#section-3.1.2) 以了解更多信息。

### Post Sign Out Redirect URI

允许用户在 Logto 退出登录后将重定向回应用程序的 URI 列表。

它将用于验证退出登录请求中使用的重定向 URI，因此退出登录时每个要使用的重定向 URI 都应预先添加到此允许列表中。

详见 [RP-Initiated Logout](https://openid.net/specs/openid-connect-rpinitiated-1_0.html#RPLogout) 以了解更多信息。

### CORS Allowed Origins

被允许向 Logto 发起请求的 Origins。

所有需要被允许的用于用户向 Logto 服务发起请求的 Origins 都应被添加到此列表中。

通常，_CORS Allowed Origins_ 应该包含你的应用程序所部署的地址。

### Token Endpoint

_Token Endpoint_ 是一个 OIDC 术语, 客户端通过访问该 Enpoint 来进行授权认证以获得访问令牌，或刷新已有的访问令牌。

详见 [Token Endpoint](https://openid.net/specs/openid-connect-core-1_0.html#TokenEndpoint) 以了解更多信息。
