# Zustimmungsbildschirm-Branding

Es ist wichtig sicherzustellen, dass die Branding-Informationen und der Link zu den Bedingungen des Drittanbieters korrekt angezeigt werden, wenn die Benutzer zum Zustimmungsbildschirm der Drittanbieteranwendung weitergeleitet werden.

Logto ermöglicht es dir, die Branding-Informationen deiner Drittanbieteranwendungen anzupassen, einschließlich des Anwendungsnamens, Logos und des Links zu den Bedingungen.

## Branding-Informationen anpassen

1. Gehe zur <CloudLink to="/applications/third-party-applications">Konsole > Anwendung > Drittanbieter-Apps</CloudLink> und öffne die Detailseite für eine spezifische OIDC-Drittanbieteranwendung.

2. Navigiere zum **Branding**-Tab.

3. Konfiguriere die Anzeigeinformationen für den Zustimmungsbildschirm:

- **Anzeigename**: Der Name der Drittanbieteranwendung, der auf dem Zustimmungsbildschirm angezeigt wird. Er repräsentiert den Namen der Drittanbieteranwendung, die Zugriff auf die Informationen deiner Benutzer anfordert. **Anwendungsname** wird verwendet, wenn dieses Feld leer gelassen wird.
- **App-Logo (Hell)**: Das Logo der Drittanbieteranwendung, das auf dem Zustimmungsbildschirm angezeigt wird. Es repräsentiert die Marke der Drittanbieteranwendung, die Zugriff auf die Informationen deiner Benutzer anfordert. Sowohl das Logo der Drittanbieteranwendung als auch dein universelles Anmeldeerfahrungslogo werden auf dem Zustimmungsbildschirm angezeigt, wenn beide bereitgestellt werden.
- **App-Logo (Dunkel)**: Nur verfügbar, wenn die Dunkelmodus-Anmeldeerfahrung aktiviert ist. Verwalte die Dunkelmodus-Einstellungen auf der <CloudLink to="/sign-in-experience/branding">Konsole > Anmeldeerfahrung > Branding</CloudLink>-Seite.
- **Nutzungsbedingungen URL**: Der Link zu den Bedingungen der Drittanbieteranwendung, der auf dem Zustimmungsbildschirm angezeigt wird.
- **Datenschutzrichtlinie URL**: Der Link zur Datenschutzrichtlinie der Drittanbieteranwendung, der auf dem Zustimmungsbildschirm angezeigt wird.
