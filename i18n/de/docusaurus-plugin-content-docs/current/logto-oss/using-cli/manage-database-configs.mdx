# Datenbankkonfigurationen verwalten

Logto hilft dabei, einige technische Konfigurationen wie OIDC- und Cookie-Schlüssel zu verwalten. Bevor wir sie über das Admin Console oder die Management API verwalten können, kannst du CLI verwenden, um das Ziel zu erreichen.

## Konfiguration nach Schlüssel abrufen \{#get-config-by-key}

Verwendung:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config get <key> [keys...]
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config get <key> [keys...]
```

  </TabItem>

</Tabs>

Angenommen, du möchtest die OIDC-Privatschlüssel abrufen:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config get oidc.privateKeys
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config get oidc.privateKeys
```

  </TabItem>

</Tabs>

Die Ausgabe wird wie folgt aussehen:

```bash
oidc.privateKeys=["\n-----BEGIN PRIVATE KEY-----\nMIIJRAIBA..."]
```

Führe `logto db config get --help` aus, um alle verfügbaren Schlüssel zu sehen, oder siehe [Konfiguration](/concepts/core-service/configuration/#database-configs) für eine detaillierte Erklärung.

## Konfiguration nach Schlüssel setzen \{#set-config-by-key}

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config set <key> [keys...]
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config set <key> [keys...]
```

  </TabItem>

</Tabs>

Angenommen, du möchtest die OIDC-Cookie-Schlüssel setzen:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto db config set oidc.cookieKeys "[\"key1\",\"key2\"]"
```

  </TabItem>
  <TabItem value="npx" label="npx">

```bash
npx @logto/cli db config set oidc.cookieKeys "[\"key1\",\"key2\"]"
```

  </TabItem>

</Tabs>

Normalerweise sollte der zu setzende Wert ein gültiger JSON-String sein und der vordefinierten Typdefinition folgen. Führe `logto db config set --help` aus, um alle verfügbaren Schlüssel zu sehen, oder siehe [Konfiguration](/concepts/core-service/configuration/) für eine detaillierte Erklärung.
