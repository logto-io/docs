# Logto CLI

Logto CLI bietet eine bequeme Möglichkeit, deine Logto-Instanz zu installieren, zu warten und zu aktualisieren, ohne diese langweiligen technischen Dinge manuell zu erledigen.

:::note
Derzeit liefern wir keine Binärdateien aus, daher wird Node.js `^18.12.0` in der Umgebung benötigt.
:::

### CLI global installieren

Installiere über deinen bevorzugten Paketmanager, um den `logto`-Befehl global zu verwenden:

<Tabs>

  <TabItem value="npm" label="npm">

```bash
npm i @logto/cli -g
```

  </TabItem>
  <TabItem value="yarn" label="Yarn">

```bash
yarn global add @logto/cli
```

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

```bash
pnpm add -g @logto/cli
```

  </TabItem>

</Tabs>

Du kannst immer `--help` anhängen, um die Hilfeinformationen zu jedem Befehl anzuzeigen:

```bash
logto --help
logto db seed --help
```

### Lokale CLI

Eine eingebaute Version der CLI wird mit jeder Logto-Version seit v1.0.0-beta.11 ausgeliefert. Angenommen, du hast bereits eine Logto-Instanz in `~/logto`, dann kannst du die folgenden Befehle ausführen:

```bash
cd ~/logto
npm run cli # Lokale CLI aufrufen
```

Beachte, dass du, wenn du mit Optionen ausführen möchtest, ein `--` VOR den Optionen für Logto CLI voranstellen musst, z.B.:

```bash
npm run cli db seed -- --db-url postgresql://your-database-url
```

### CLI mit `npx` verwenden

Eine weitere Möglichkeit, Logto CLI zu verwenden, ist `npx`, das ein Paket ohne Installation ausführen kann.

```bash
npx @logto/cli
```

Dies ist hilfreich für einmalige Aufrufe, z.B.:

```bash
npx @logto/cli db seed --db-url postgresql://your-database-url
```
