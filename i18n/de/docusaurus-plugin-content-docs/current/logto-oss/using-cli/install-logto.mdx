# Logto installieren

## Interaktiv \{#interactive}

Führe den folgenden Befehl aus, um eine interaktive Installation in deinem Terminal zu starten:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npm init @logto@latest
```

  </TabItem>

</Tabs>

## Optionen und stille Installation \{#options-and-silent-installation}

Alternativ kannst du eine der folgenden Optionen (alle optional) hinzufügen, um einige Fragen zu überspringen:

| Option                                         | Beschreibung                                                                                                                                                                                                                                                      |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--db-url`                                     | Die Postgres-URL zur Logto-Datenbank.                                                                                                                                                                                                                             |
| `-p, --path`                                   | Pfad zu deiner Logto-Instanz, muss ein nicht existierender Pfad sein.                                                                                                                                                                                             |
| `--ss`                                         | Überspringt das Seeding der Logto-Datenbank.                                                                                                                                                                                                                      |
| `--disable-admin-pwned-password-check, --dapc` | Seedet den Admin-Tenant mit deaktivierter Have I Been Pwned (HIBP) Passwortüberprüfung, für air-gapped oder Offline-Bereitstellungen. Siehe [Seed für air-gapped oder Offline-Bereitstellungen](/logto-oss/using-cli#seed-for-air-gapped-or-offline-deployments). |

Führe `logto init --help` aus, um die vollständige Hilfe anzuzeigen.

Hier ist ein Beispiel für eine stille Installation (z. B. in CI):

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npx @logto/cli init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>

</Tabs>

Dabei werden der Reihe nach folgende Schritte ausgeführt:

1. Logto wird heruntergeladen und in `./logto` entpackt
2. Es wird versucht, die Datenbank mit der URL `postgresql://your-postgres-dsn:5432/logto` zu initialisieren und zu seed-en
3. Die offiziellen Logto Connectors werden zu `./logto/packages/core/connectors` hinzugefügt
