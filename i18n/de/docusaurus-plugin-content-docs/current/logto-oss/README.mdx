# Logto OSS

Logto bietet sowohl Open-Source- (OSS) als auch [Cloud](/logto-cloud)-Versionen an. Dieser Abschnitt konzentriert sich auf die OSS-Lösung: <Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS ist eine kostenlose Identitätslösung, die Entwicklern vollständige Anpassungsmöglichkeiten und die volle Kontrolle über ihre Daten bietet. Logto OSS bietet die Flexibilität, den Code an spezifische Anforderungen anzupassen – ideal für alle, die Wert auf Datensouveränität legen und eine maßgeschneiderte Identitätsanbieter-Lösung suchen, die mit ihrer Anwendung mitwachsen kann.

## Von Logto OSS unterstützte Funktionen \{#feature-supported-by-logto-oss}

Logto OSS unterstützt die meisten Kernfunktionen des Logto-Dienstes und wird regelmäßig aktualisiert und gepflegt.

Einige erweiterte Funktionen sind derzeit exklusiv in der Logto Cloud-Version verfügbar, darunter:

| Einschränkungen der Funktionen in OSS                                                   | Beschreibung                                                                                                                                        |
| --------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto-Konsole: mehrere Tenants](/logto-cloud/tenant-settings)                          | Erstelle und verwalte mehrere Logto-Tenants in einer Konsole.                                                                                       |
| [Logto-Konsole: Kollaboratoren einladen](/logto-cloud/tenant-member-management)         | Lade mehrere Mitglieder zu einem Logto-Tenant ein und verwalte sie.                                                                                 |
| Logto-Konsole: MFA aktivieren                                                           | Erhöhe die Sicherheit, indem du Multi-Faktor-Authentifizierung beim Anmelden in der Logto-Konsole verlangst.                                        |
| [Logto Protected App](/integrate-logto/protected-app)                                   | Schnelle und SDK-freie Authentifizierungsintegration, bereitgestellt durch Cloudflare.                                                              |
| [Logto integrierter E-Mail-Dienst](/connectors/email-connectors/built-in-email-service) | Nutze den integrierten und kostenlosen E-Mail-Dienst für den E-Mail-Versand.                                                                        |
| [Bring your UI](/customization/bring-your-ui)                                           | Passe das UI oder die Abläufe der Anmeldeerfahrung mit dieser Funktion an (OSS kann den Code auf GitHub forken, um die Anmeldung direkt zu ändern). |
| [IdP-initiiertes SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)                 | Aktiviere vom Identitätsanbieter (IdP) initiiertes Enterprise Single Sign-On.                                                                       |
| [SAML-Apps](/integrate-logto/saml-app)                                                  | Integriere Anwendungen über das SAML-Protokoll. Die OSS-Version ist auf 3 SAML-Apps beschränkt.                                                     |

Hinweis: Multi-Tenancy, Mitglieder-Einladungen und MFA stehen deinem Team nicht zur Anmeldung an einer Open-Source-Logto-Konsole zur Verfügung. Du kannst diese Funktionen jedoch in deinem eigenen Produkt mit Logto OSS implementieren und so deinen Endnutzern bereitstellen.
