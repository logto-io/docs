# Organisationen

Die Organisationsfunktion ist besonders effektiv in Multi-Tenant-SaaS und [Business-to-Business (B2B) Apps](/introduction/plan-your-architecture/b2b). Zusätzlich zu einzelnen Benutzern können deine Kunden auch aus Teams, Organisationen oder ganzen Unternehmen bestehen.

Mit diesem grundlegenden Element kannst du die unverzichtbaren Funktionen für [Multi-Tenancy-Apps](https://auth.wiki/multi-tenancy) erstellen, wie zum Beispiel:

- Ein Produkt, das von mehreren Organisationen genutzt werden kann.
- Bereitstellung von Organisationsmitgliedern auf Basis von [Einladungen](/end-user-flows/organization-experience/invite-organization-members) oder [Just-in-Time](/organizations/just-in-time-provisioning).
- [Zugangskontrollen](/authorization/organization-template), die durch Rollen definiert werden, die Mitgliedern innerhalb einer Organisation zugewiesen sind.
- Verknüpfe Organisationen mit der [Enterprise SSO (Single Sign-On)](/end-user-flows/enterprise-sso) Erfahrung.

Der Begriff "Organisation" wird auch in anderen Formen verwendet, wie "Arbeitsbereich", "Team", "Unternehmen" usw. In Logto verwenden wir "Organisation" als generischen Begriff, um das Konzept der Multi-Tenancy darzustellen.

## Planungserwägungen \{#planning-considerations}

Beim Einsatz von Organisationen kann die Integration unterschiedliche Anstrengungen erfordern. Wir empfehlen, diese zu vergleichen, um deine Entwicklung anzupassen.

- **Organisationserfahrung**: Wie würde die Organisationserfahrung aussehen? Kann ein Organisationsadministrator seine Organisation erstellen? Können Organisationsadministratoren Mitglieder einladen? Möchtest du ein benutzerdefiniertes Dashboard erstellen, das Administratoren ermöglicht, die identitätsbezogenen Informationen ihrer Organisationsmitglieder selbst zu verwalten?
- **Rollen und Zugangskontrolle**: Benötigt die Anwendung, dass Benutzer spezifische Rollen innerhalb ihrer Organisationen zugewiesen bekommen?
- **Verwalte Organisationen über Logto Console und Management API:** Welche Konfiguration möchtest du direkt in der Logto Console und Management API einrichten?
