# Logto OSS einrichten

Dieser Abschnitt behandelt grundlegende Einrichtungsschritte und wichtige Aktionen, um dein Produkt effektiv bereitzustellen und deinen Entwicklungsworkflow für den [Logto Open-Source-Service (OSS)](https://github.com/logto-io/logto) zu verwalten.

## Einstieg mit Logto OSS \{#get-started-with-logto-oss}

Folge der [Einstiegsanleitung](/logto-oss/get-started-with-oss/), um Logto noch heute zu starten. Später kannst du für den produktiven Einsatz auf die vollständige [Bereitstellungsanleitung](/logto-oss/deployment-and-configuration) zurückgreifen.

## Von Logto OSS unterstützte Funktionen \{#feature-supported-by-logto-oss}

Logto OSS unterstützt die meisten Kernfunktionen des Logto-Dienstes und wird regelmäßig aktualisiert und gepflegt.

Einige erweiterte Funktionen sind derzeit exklusiv in der Logto Cloud-Version verfügbar, darunter:

| Funktionsbeschränkungen in OSS                                                          | Beschreibung                                                                                                                                         |
| --------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto-Konsole: mehrere Mandanten](/logto-cloud/tenant-settings)                        | Erstelle und verwalte mehrere Logto-Mandanten in einer Konsole.                                                                                      |
| [Logto-Konsole: Einladung von Mitarbeitern](/logto-cloud/tenant-member-management)      | Lade mehrere Mitglieder zu einem Logto-Mandanten ein und verwalte sie.                                                                               |
| Logto-Konsole: MFA aktivieren                                                           | Erhöhe die Sicherheit, indem du Multi-Faktor-Authentifizierung (MFA) beim Anmelden an der Logto-Konsole verlangst.                                   |
| [Logto Protected App](/integrate-logto/protected-app)                                   | Schnelle und SDK-freie Authentifizierungsintegration, bereitgestellt durch Cloudflare.                                                               |
| [Logto integrierter E-Mail-Dienst](/connectors/email-connectors/built-in-email-service) | Nutze den integrierten und kostenlosen E-Mail-Dienst für den E-Mail-Versand.                                                                         |
| [Bring your UI](/customization/bring-your-ui)                                           | Passe das UI oder die Abläufe der Anmeldeerfahrung mit dieser Funktion an (OSS kann den Code auf GitHub forken, um die Anmeldung direkt anzupassen). |
| [IdP-initiiertes SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)                 | Aktiviere vom Identitätsanbieter (IdP) initiiertes Enterprise Single Sign-On (SSO).                                                                  |
| [SAML-Apps](/integrate-logto/saml-app)                                                  | Integriere Anwendungen über das SAML-Protokoll. Die OSS-Version ist auf 3 SAML-Apps beschränkt.                                                      |
| [Logto-Branding ausblenden](/customization/match-your-brand#hide-logto-branding)        | Entferne das "Powered by Logto"-Zeichen aus der Anmeldeerfahrung.                                                                                    |

Hinweis: Multi-Tenancy, Mitarbeitereinladungen und MFA stehen deinem Team nicht zur Verfügung, um sich an einer Open-Source-Logto-Konsole anzumelden. Du kannst diese Funktionen jedoch in deinem eigenen Produkt mit Logto OSS implementieren und sie so deinen Endbenutzern bereitstellen.

## Bleibe mit Logto-Releases auf dem Laufenden \{#stay-updated-with-logto-releases}

Um deine Logto-Instanz mit den neuesten Funktionen aktuell zu halten, folge der [Logto GitHub Releases](https://github.com/logto-io/logto/releases) Seite. Dort findest du alle Release-Logs.

Lies die [Anleitung zum Upgrade](/logto-oss/upgrading-oss-version), um zu erfahren, wie du Logto aktualisierst, ohne deinen Code oder das Datenbankschema zu ändern.
