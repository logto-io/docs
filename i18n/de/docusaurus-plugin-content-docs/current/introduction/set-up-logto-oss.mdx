# Logto OSS einrichten

Dieser Abschnitt behandelt grundlegende Einrichtungsschritte und wichtige Aktionen, um Ihr Produkt effektiv bereitzustellen und Ihren Entwicklungsworkflow für den [Logto Open-Source-Dienst (OSS)](https://github.com/logto-io/logto) zu verwalten.

## Erste Schritte mit Logto OSS \{#get-started-with-logto-oss}

Folgen Sie der [Anleitung für den Einstieg](/logto-oss/get-started-with-oss/), um Logto heute zu starten, und später können Sie auf die vollständige [Bereitstellungsanleitung](/logto-oss/deployment-and-configuration) für den Produktionseinsatz verweisen.

## Von Logto OSS unterstützte Funktionen \{#feature-supported-by-logto-oss}

Logto OSS unterstützt die meisten Kernfunktionen des Logto-Dienstes und wird regelmäßig aktualisiert und gewartet.

Einige erweiterte Funktionen sind derzeit exklusiv für die Logto Cloud-Version verfügbar, einschließlich [Logto Protected App](/integrate-logto/protected-app), [kostenloser E-Mail-Dienst](/connectors/email-connectors/built-in-email-service), [IdP-initiiertes SSO](/end-user-flows/enterprise-sso/idp-initiated-sso) und [SAML IdP](/integrate-logto/third-party-applications).

## Bleiben Sie mit Logto-Veröffentlichungen auf dem Laufenden \{#stay-updated-with-logto-releases}

Um Ihre Logto-Instanz mit den neuesten Funktionen auf dem neuesten Stand zu halten, folgen Sie der [Logto GitHub Releases](https://github.com/logto-io/logto/releases) Seite, dort finden Sie alle Veröffentlichungsprotokolle.

Lesen Sie die [Anleitung zum Upgrade](/logto-oss/upgrading-oss-version), um zu erfahren, wie Sie Logto aktualisieren können, ohne Ihren Code oder Ihr Datenbankschema zu ändern.
