# Logto Cloud einrichten

Dieser Abschnitt behandelt grundlegende Einrichtungsschritte und wichtige Aktionen, um Ihr Produkt effektiv bereitzustellen und Ihren Entwicklungsworkflow zu verwalten.

## Logto-Mandant erstellen

Melden Sie sich zunächst für ein Geschäftskonto an und erstellen Sie einen Logto-Mandanten. Ein Mandant von Logto Cloud ist eine isolierte Umgebung, in der Sie Benutzeridentitäten, Anwendungen und alle anderen Logto-Ressourcen verwalten können.

Ihr erster Mandant wird automatisch als **Entwicklungs**umgebung eingerichtet. Diese ist ideal für Test- und Entwicklungszwecke und kann kostenlos genutzt werden.

Überprüfen Sie die Mandanteneinstellungen und [erfahren Sie mehr](/logto-cloud) über den Logto-Cloud-Dienst.

## Mitarbeiter einladen

Während des Onboardings haben Sie die Möglichkeit, Mitarbeiter einzuladen, um gemeinsam an der Entwicklung zu arbeiten. Wenn Sie mehr oder später einladen möchten, gehen Sie zu <CloudLink to="/tenant-settings/members">Mandanteneinstellungen > Mitglieder</CloudLink>. Dort sehen Sie Ihre aktuellen Mitglieder und können weitere einladen. Geben Sie E-Mail-Adressen ein, weisen Sie Rollen zu (dies können Sie in großen Mengen tun) und senden Sie dann die Einladung.

## Zum Produktionsmandanten wechseln

Sie können einige Zeit damit verbringen, die Fähigkeiten und Funktionen von Logto zu entwickeln und zu testen, um ein Proof of Concept zu erstellen.

Nachdem Ihr Projekt die Tests abgeschlossen hat und bereit für die nächste Phase ist, denken Sie daran, einen neuen Produktionsmandanten zu erstellen (erfordert einen Free- oder Pro-Mandanten). Da der Entwicklungsmandant aufgrund seiner Einschränkungen und Begrenzungen nicht für den Produktionseinsatz vorgesehen ist.

:::note

Stellen Sie sicher, dass Sie die benutzerdefinierte Domain eingeben, sobald Sie einen Produktionsmandanten erstellen, da dies Ihre nächste Konfigurationsschritte beeinflussen wird.

:::

Diese Einrichtung ermöglicht es Ihrem Projekt, sowohl Entwicklungs- als auch Produktionsumgebungen zu haben, was Ihnen hilft, Ihren Entwicklungsworkflow reibungslos zu verwalten.

## Vom bestehenden System migrieren

Wenn Sie an einem bestehenden Projekt arbeiten und den Authentifizierungsanbieter wechseln müssen, unterstützt Logto die Benutzermigration von anderen Plattformen. Sie können Basisdaten, benutzerdefinierte Daten, soziale Identitäten und Passwort-Hashes migrieren. Einzelheiten finden Sie im Leitfaden [Zu Logto migrieren](../user-management/user-migration.mdx).
