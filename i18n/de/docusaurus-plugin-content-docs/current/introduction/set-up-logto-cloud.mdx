# Logto Cloud einrichten

In diesem Abschnitt werden grundlegende Einrichtungsschritte und wichtige Aktionen behandelt, um Ihr Produkt effektiv bereitzustellen und Ihren Entwicklungsworkflow zu verwalten.

## Logto-Mandant erstellen \{#create-logto-tenant}

Melden Sie sich zunächst für ein Geschäftskonto an und erstellen Sie einen Logto-Mandanten. Ein Mandant von Logto Cloud ist eine isolierte Umgebung, in der Sie Benutzeridentitäten, Anwendungen und alle anderen Logto-Ressourcen verwalten können.

Ihr erster Mandant wird automatisch als [**Entwicklungs**](/logto-cloud/tenant-settings#development)umgebung eingerichtet. Dies ist ideal für Test- und Entwicklungszwecke und kann kostenlos genutzt werden.

Überprüfen Sie die Mandanteneinstellungen und [erfahren Sie mehr](/logto-cloud) über den Logto-Cloud-Dienst.

## Mitarbeiter einladen \{#invite-collaborators}

Während des Onboardings haben Sie die Möglichkeit, Mitarbeiter einzuladen, um gemeinsam an der Entwicklung zu arbeiten. Wenn Sie mehr oder später einladen möchten, gehen Sie zu <CloudLink to="/tenant-settings/members">Mandanteneinstellungen > Mitglieder</CloudLink>. Dort sehen Sie Ihre aktuellen Mitglieder und können weitere einladen. Geben Sie E-Mail-Adressen ein und weisen Sie Rollen zu (dies können Sie in großen Mengen tun) und senden Sie dann die Einladung.

## Zum Produktionsmandanten wechseln \{#move-to-production-tenant}

Sie können einige Zeit damit verbringen, die Fähigkeiten und Funktionen von Logto zu entwickeln und zu testen, um einen Proof of Concept zu erstellen.

Nachdem Ihr Projekt die Tests abgeschlossen hat und bereit für die nächste Phase ist, denken Sie daran, einen neuen [Produktions-Mandanten](/logto-cloud/tenant-settings#production) zu erstellen (erfordert einen [Free- oder Pro-Mandanten](https://logto.io/pricing)). Da der Entwicklungsmandant aufgrund seiner [Einschränkungen und Beschränkungen](/logto-cloud/tenant-settings#development) nicht für den Produktionseinsatz vorgesehen ist.

:::note

Stellen Sie sicher, dass Sie die [benutzerdefinierte Domain](/logto-cloud/custom-domain) eingeben, sobald Sie einen Produktionsmandanten erstellen, da dies Ihre nächste Konfigurationsschritte beeinflussen wird.

:::

Diese Einrichtung ermöglicht es Ihrem Projekt, sowohl Entwicklungs- als auch Produktionsumgebungen zu haben, was Ihnen hilft, Ihren Entwicklungsworkflow reibungslos zu verwalten.

## Vom bestehenden System migrieren \{#migrate-from-the-existing-system}

Wenn Sie an einem bestehenden Projekt arbeiten und den Authentifizierungsanbieter wechseln müssen, unterstützt Logto die Benutzermigration von anderen Plattformen. Sie können [Basisdaten](/user-management/user-data#basic-data), [benutzerdefinierte Daten](/user-management/user-data#custom-data), [soziale Identitäten](/user-management/user-data#social-identities) und Passwort-Hashes migrieren. Weitere Details finden Sie im Leitfaden [Zu Logto migrieren](../user-management/user-migration.mdx).
