Logto bietet vier verschiedene Vorlagen zur Anpassung von SMS-Inhalten an, die basierend auf ihrem Verwendungstyp kategorisiert sind: Registrierung, Anmeldung, Passwort vergessen und Allgemein. Es wird dringend empfohlen, für verschiedene Anwendungsfälle unterschiedliche Vorlagen zu verwenden, da sonst das Ratenlimit erreicht werden könnte, was zu einem vorübergehenden Ausfall deines Dienstes führen kann.

## SMS-Vorlagentypen und Beispiele

Hier sind einige Beispiele nur zur Referenz:

| Verwendung         | Szenario                                                                                                                                                | Vorlagenbeispiel                                                                                     |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| Registrierung      | Benutzer erstellen ein Konto mit ihrer E-Mail und verifizieren es, indem sie einen von Logto an ihre E-Mail gesendeten Bestätigungscode eingeben.       | Dein Logto-Anmeldebestätigungscode ist `{{code}}`. Der Code bleibt 10 Minuten lang aktiv.            |
| Anmeldung          | Benutzer melden sich mit ihrer E-Mail an und verifizieren sich, indem sie einen Bestätigungscode eingeben, anstatt ein Passwort einzugeben.             | Dein Logto-Anmeldebestätigungscode ist `{{code}}`. Der Code bleibt 10 Minuten lang aktiv.            |
| Passwort vergessen | Wenn Benutzer ihr Passwort beim Anmelden vergessen, können sie ihre Identität mit der E-Mail verifizieren, die sie bereits bei Logto verifiziert haben. | Dein Logto-Passwortänderungs-Bestätigungscode ist `{{code}}`. Der Code bleibt 10 Minuten lang aktiv. |
| Allgemein          | Diese Vorlage kann als allgemeine Backup-Option für verschiedene Szenarien verwendet werden, einschließlich der Konfiguration von Connector-Tests usw.  | Dein Logto-Bestätigungscode ist `{{code}}`. Der Code bleibt 10 Minuten lang aktiv.                   |

Es ist wichtig, diese Parameter zu verstehen:

- Der Bestätigungscode ist 10 Minuten lang gültig. Wir unterstützen derzeit keine Anpassung der Ablaufzeit.
- Logto wird den Platzhalter `{{code}}` in der SMS-Vorlage durch einen Bestätigungscode ersetzen. Stelle daher sicher, dass die Vorlage einen reservierten Platzhalter hat.

:::note
Einige Länder und Regionen erlauben möglicherweise nicht das Versenden von nicht genehmigten Inhalten per SMS aufgrund von Compliance-Anforderungen. SMS-Vorlagen müssen beim SMS-Anbieter registriert und genehmigt werden, bevor sie verwendet werden können. In solchen Fällen könnte der Inhalt durch die Vorlagen-ID der entsprechenden Vorlage zugeordnet werden.
:::
