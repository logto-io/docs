## Mit Patreon-Konto anmelden \{#sign-in-with-patreon-account}

Gehe auf die [Patreon-Website](https://www.patreon.com/) und melde dich mit deinem Patreon-Konto an. Du kannst ein neues Konto registrieren, falls du noch keines hast.

## OAuth-App erstellen und konfigurieren \{#create-and-configure-oauth-app}

Folge der Anleitung zum [Erstellen einer Patreon OAuth-App](https://www.patreon.com/portal/registration/register-clients) und registriere eine neue Anwendung.

Vergib im Feld **App Name** einen Namen für deine neue OAuth-Anwendung und trage die **App URL** der App ein. Das Feld **App Description** kannst du leer lassen und die **Redirect URIs** wie folgt anpassen: `${your_logto_origin}/callback/${connector_id}`. Die `connector_id` findest du in der oberen Leiste der Logto Admin Console auf der Detailseite des Connectors.

:::note
Wenn beim Anmelden die Fehlermeldung „The redirect_uri MUST match the registered callback URL for this application.“ erscheint, versuche, die Redirect URI deiner Patreon OAuth-App und die Redirect-URL deiner Logto-App (einschließlich des Protokolls) abzugleichen, um das Problem zu beheben.
:::

## OAuth-Apps verwalten \{#managing-oauth-apps}

Gehe auf die [Clients & API Keys-Seite](https://www.patreon.com/portal/registration/register-clients) bei Patreon. Dort kannst du bestehende OAuth-Apps hinzufügen, bearbeiten oder löschen. Außerdem findest du dort die `Client ID` und kannst `Client secrets` auf den Detailseiten der OAuth-App generieren.

## Deinen Connector konfigurieren \{#configure-your-connector}

Trage in die Felder `clientId` und `clientSecret` die _Client ID_ und das _Client Secret_ ein, die du auf den Detailseiten der OAuth-App im vorherigen Abschnitt erhalten hast.

`scope` ist eine durch Leerzeichen getrennte Liste von [Berechtigungen (Scopes)](https://docs.patreon.com/#scopes). Wenn kein Wert angegeben wird, ist der Standardwert für scope `identity identity[email]`.

### Konfigurationstypen \{#config-types}

| Name         | Typ    |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
