# Soziale Anmeldung mit WeCom (OAuth 2.0) einrichten

Der benutzerdefinierte Connector für die WeCom (vielleicht WXwork genannt) soziale Anmeldung.

<GuideTip />

## Erste Schritte

Melde dich mit einem Administratorkonto bei [WeCom WebUI](https://work.weixin.qq.com/) an oder klicke auf **Manage the enterprise(管理企业)** in der WeCom-App.

Im Tab "App Management" scrolle die Seite nach unten und klicke auf "Create an app".

Fülle die entsprechenden Informationen gemäß deiner App aus. Dann erstelle.

Jetzt haben wir die Agent ID (NICHT APPID) und das Secret.

### Website-Informationen

Stelle die Dinge ein, die du auf dieser Seite benötigst. Es könnte so aussehen:

- Erlaubte Benutzer: _wer diese App sehen kann_
- App-Startseite: _Deine App-Startseite. Z.B., `logto.io/demo-app`_

**Wichtig**
Es gibt drei Punkte auf dieser Seite bezüglich der "Developer API(开发者接口)".

1. Web Authorization und JS-SDK;
2. Anmeldung über Autorisierung durch WeCom;
3. Enterprise Trusted IP;

Fülle sie gemäß der Anleitung von WeCom aus.

### Corp ID

Wenn du mit der WeChat-Entwicklung vertraut bist, wirst du bemerken, dass die Verwendung der Corp ID der APP ID entspricht.

Du findest die Corp ID am unteren Rand der Seite "My Enterprise(我的企业)". Sie sieht aus wie **ww\*\*\*\*** .

## Den Connector konfigurieren

Also haben wir die Agent ID, das Secret und die Corp ID.

Lass uns das Formular für den Connector ausfüllen.

Du kannst das Feld `Scope` leer lassen, da es optional ist. Alternativ kannst du `snsapi_base` oder `snsapi_privateinfo` eintragen.

**Speichern und fertig**
