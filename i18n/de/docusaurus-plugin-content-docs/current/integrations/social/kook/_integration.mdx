## Anwendung im KOOK Developer Center erstellen \{#create-an-application-in-the-kook-developer-center}

Tipp: Du kannst einige Abschnitte überspringen, wenn du sie bereits abgeschlossen hast.

:::note
Du musst die KOOK OAuth2-Berechtigung beantragen, bevor du eine OAuth2-Anwendung erstellst. Bitte informiere dich auf der OAuth2-Seite oder frage das KOOK-Team im offiziellen KOOK-Entwicklerserver nach weiteren Informationen.
:::

1. Gehe zum [KOOK Developer Center](https://developer.kookapp.cn/app/index) und melde dich mit deinem KOOK-Konto an. Du kannst ein neues Konto registrieren, falls du noch keines hast.
2. Klicke auf die Schaltfläche **Anwendung erstellen** (Create Application), um eine Anwendung zu erstellen, wähle einen Namen dafür (z. B. LogtoAuth) und klicke auf **Bestätigen** (Confirm).
3. Klicke auf das von dir erstellte Anwendungssymbol und gehe zur Seite **OAuth2**.
4. Füge die gültigen Redirects (z. B. `${your_logto_origin}/callback/${connector_id}`) in die Felder **Callback-URIs** ein. Du findest die Redirect-URI auf der Detailseite des Connectors in der Logto Admin-Konsole.
5. Vergiss nicht, den **get_user_info**-Scope im **OAuth2-Link-Builder** auszuwählen.

## Deinen KOOK-Connector konfigurieren \{#configure-your-kook-connector}

Trage die Felder clientId und clientSecret mit der Client ID und dem Client Secret aus der im vorherigen Abschnitt genannten OAuth2-Seite ein.

### Konfigurationstypen \{#config-types}

| Name         | Typ    |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
