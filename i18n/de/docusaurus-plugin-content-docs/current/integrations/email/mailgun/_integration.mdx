## Voraussetzungen \{#prerequisites}

- Ein [Mailgun](https://www.mailgun.com/) Konto
- Ein API-Schlüssel aus deinem Mailgun-Konto, benötigt die Berechtigung zum Versenden von Nachrichten (E-Mails). Siehe [Where Can I Find My API Key and SMTP Credentials?](https://help.mailgun.com/hc/en-us/articles/203380100-Where-Can-I-Find-My-API-Key-and-SMTP-Credentials-) für weitere Informationen.

## Grundkonfiguration \{#basic-configuration}

- Fülle das Feld `endpoint` aus, wenn du einen anderen Mailgun API-Endpunkt verwendest, zum Beispiel sollte die EU-Region `https://api.eu.mailgun.net` sein. Der Standardwert ist `https://api.mailgun.net`.
- Fülle das Feld `domain` mit der Domain aus, die du in deinem Mailgun-Konto registriert hast. Dieser Wert ist im Bereich **Domains** des Mailgun-Dashboards zu finden. Die Domain sollte im Format `example.com` angegeben werden, ohne das Präfix `https://` oder `http://`.
- Fülle das Feld `apiKey` mit dem API-Schlüssel aus, den du in deinem Mailgun-Konto generiert hast.
- Fülle das Feld `from` mit der E-Mail-Adresse aus, von der du E-Mails versenden möchtest. Diese E-Mail-Adresse muss in deinem Mailgun-Konto registriert sein. Die E-Mail-Adresse sollte im Format `Sender Name \<sender@example.com>` angegeben werden.

## Zustellungen \{#deliveries}

### Config-Objekt \{#config-object}

Der Abschnitt "Zustellungen" ermöglicht es dir, den Inhalt der zu versendenden E-Mails in verschiedenen Szenarien zu konfigurieren. Es handelt sich um eine JSON-Key-Value-Map, wobei der Schlüssel der Verwendungstyp ist und der Wert ein Objekt mit der Inhaltskonfiguration für die zu versendende E-Mail.

```json
{
  "<usage-type>": {
    // ...
  }
}
```

### Verwendungstypen \{#usage-types}

Die folgenden Verwendungstypen werden unterstützt:

- `Register`: Die E-Mail, die versendet wird, wenn sich ein Benutzer registriert.
- `SignIn`: Die E-Mail, die versendet wird, wenn sich ein Benutzer anmeldet.
- `ForgotPassword`: Die E-Mail, die versendet wird, wenn ein Benutzer sein Passwort zurücksetzt.
- `Generic`: Die E-Mail, die versendet wird, wenn ein Benutzer eine generische Aktion durchführt, zum Beispiel das Testen des E-Mail-Connectors.

:::note
Wenn der Verwendungstyp nicht in der Zustellungs-Konfiguration angegeben ist, wird die generische E-Mail versendet. Wenn die generische E-Mail nicht angegeben ist, gibt der Connector einen Fehler zurück.
:::

### Inhaltskonfiguration \{#content-config}

Der Connector unterstützt sowohl direkten HTML-Inhalt als auch Mailgun-Vorlagen. Du kannst für jeden Verwendungstyp eine der beiden Optionen nutzen.

Sowohl im Betreff als auch im Inhalt kannst du den Platzhalter `{{code}}` verwenden, um den Verifizierungscode einzufügen.

Um direkten HTML-Inhalt zu verwenden, fülle die folgenden Felder aus:

- `subject`: Der Betreff der zu versendenden E-Mail.
- `replyTo`: Die E-Mail-Adresse, die als Antwortadresse verwendet werden soll.
- `html`: (Erforderlich) Der HTML-Inhalt der zu versendenden E-Mail.
- `text`: Die Nur-Text-Version der zu versendenden E-Mail.

Um eine Mailgun-Vorlage zu verwenden, fülle die folgenden Felder aus:

- `subject`: Der Betreff der zu versendenden E-Mail.
- `replyTo`: Die E-Mail-Adresse, die als Antwortadresse verwendet werden soll.
- `template`: (Erforderlich) Der Name der zu verwendenden Mailgun-Vorlage.
- `variables`: Die Variablen, die an die Mailgun-Vorlage übergeben werden. Sollte eine JSON-Key-Value-Map sein, da sie vor dem Senden an Mailgun als String serialisiert wird. Beachte, dass die Variable `code` nicht enthalten sein muss, da sie automatisch vom Connector hinzugefügt wird.

### Beispiel \{#example}

Nachfolgend ein Beispiel für die Zustellungs-Konfiguration:

```json
{
  "Register": {
    "subject": "{{code}} ist dein Verifizierungscode",
    "replyTo": "Foo <foo@bar.com>",
    "html": "<h1>Willkommen bei Logto</h1><p>Dein Verifizierungscode ist {{code}}.</p>",
    "text": "Willkommen bei Logto. Dein Verifizierungscode ist {{code}}."
  },
  "SignIn": {
    "subject": "Willkommen zurück bei Logto",
    "replyTo": "Foo <foo@bar.com>",
    "template": "logto-sign-in",
    "variables": {
      "bar": "baz"
    }
  }
}
```
