# E-Mail-Verifizierung mit Postmark einrichten

Logto Connector für den Postmark E-Mail-Dienst.

<GuideTip />

## Erste Schritte \{#get-started}

Postmark ist eine Plattform für transaktionale und Marketing-E-Mails. Wir können ihre E-Mail-Versandfunktion nutzen, um einen _Verifizierungscode_ zu versenden.

## Postmark-Konto registrieren \{#register-postmark-account}

Erstelle ein neues Konto auf der [Postmark-Website](https://postmark.com/). Du kannst diesen Schritt überspringen, wenn du bereits ein Konto hast.

## Deinen Connector konfigurieren \{#configure-your-connector}

Trage das Feld `serverToken` mit dem Server Token aus den Einstellungen deines Servers in Postmark aus.

Trage das Feld `fromEmail` mit der _Absenderadresse_ ein.

Um vollständige Benutzerflüsse zu ermöglichen, sind Vorlagen mit usageType `Register`, `SignIn`, `ForgotPassword` und `Generic` erforderlich.

Hier ist ein Beispiel für das Postmark Connector Template JSON.

```jsonc
[
  {
    "usageType": "Register",
    "templateAlias": "logto-register",
  },
  {
    "usageType": "SignIn",
    "templateAlias": "logto-sign-in",
  },
  {
    "usageType": "ForgotPassword",
    "templateAlias": "logto-forgot-password",
  },
  {
    "usageType": "Generic",
    "templateAlias": "logto-generic",
  },
]
```

## Postmark E-Mail-Connector testen \{#test-postmark-email-connector}

Du kannst eine E-Mail-Adresse eingeben und auf „Senden“ klicken, um zu prüfen, ob die Einstellungen funktionieren, bevor du auf „Speichern und Fertig“ klickst.

Das war's. Vergiss nicht, den <MainSiteUrl href="/connectors/email-connectors/#enable-email-address-for-end-user-flows">Connector für Anmeldung und Registrierung zu aktivieren</MainSiteUrl>.

## Konfigurationstypen \{#config-types}

| Name        | Typ    |
| ----------- | ------ |
| serverToken | string |
| fromEmail   | string |

| Template-Eigenschaften | Typ         | Enum-Werte                                           |
| ---------------------- | ----------- | ---------------------------------------------------- |
| usageType              | enum string | 'Register' / 'SignIn' / 'ForgotPassword' / 'Generic' |
| templateAlias          | string      | N/A                                                  |
