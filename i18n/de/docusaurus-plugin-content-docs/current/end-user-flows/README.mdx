# Endbenutzer-Flows

Endbenutzer-Flows decken alle Verifizierungsprozesse für Benutzerinteraktionen ab, kategorisiert wie folgt:

- **Authentifizierungs-Flows**: Beinhaltet Benutzeranmeldung, Registrierung und Passwortzurücksetzung. Logto bietet sofort einsatzbereite Erfahrungen, die in <CloudLink to="/sign-in-experience">Konsole > Anmeldeerfahrung</CloudLink> konfiguriert sind, und du kannst [dein UI einbringen](/customization/bring-your-ui) über [Experience APIs](https://openapi.logto.io/group/endpoint-experience).
- **Konto-Flows**: Beinhaltet Kontoeinstellungen, Benutzerprofil und Sicherheitsüberprüfung. Verwende [Account APIs](https://openapi.logto.io/group/endpoint-account-center) oder [Management APIs](https://openapi.logto.io/operation/operation-getuser) zur Implementierung.
- **Organisations-Flows**: Für Multi-Tenancy-Produkte, wie B2B- oder SaaS-Dienste. Beinhaltet Organisationserstellung, Mitglieder-Einladung und Verwaltung. Verwende [Management APIs](https://openapi.logto.io/operation/operation-listorganizations), um dies in deine Apps zu integrieren.
