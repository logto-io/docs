# Endbenutzer-Flows

Endbenutzer-Flows umfassen alle Verifizierungsprozesse für Benutzerinteraktionen und werden wie folgt kategorisiert:

- **Authentifizierungs-Flows (Authentication flows)**: Beinhaltet Benutzeranmeldung, Registrierung und Zurücksetzen des Passworts. Logto bietet sofort einsatzbereite Erlebnisse, die in <CloudLink to="/sign-in-experience">Konsole > Anmeldeerfahrung</CloudLink> konfiguriert werden, und du kannst [dein eigenes UI bereitstellen](/customization/bring-your-ui) über [Experience APIs](https://openapi.logto.io/group/endpoint-experience).
- **Konto-Flows (Account flows)**: Beinhaltet Kontoeinstellungen, Benutzerprofil und Sicherheitsüberprüfung. Nutze [Account APIs](https://openapi.logto.io/group/endpoint-account-center) oder [Management APIs](https://openapi.logto.io/operation/operation-getuser) zur Implementierung.
- **Organisations-Flows (Organization flows)**: Für Multi-Tenancy-Produkte wie B2B- oder SaaS-Dienste. Beinhaltet Organisationserstellung, Mitglieder-Einladung und Verwaltung. Nutze [Management APIs](https://openapi.logto.io/operation/operation-listorganizations), um dies in deine Apps zu integrieren.
