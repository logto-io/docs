# Sicherheitsüberprüfung

Wenn authentifizierte Benutzer versuchen, sensible oder risikoreiche Aktionen durchzuführen – wie das Ändern ihres Passworts, das Tätigen einer Zahlung oder den Zugriff auf sensible Informationen wie Gehaltsabrechnungen oder Bankkontodetails – sind zusätzliche Sicherheitsmaßnahmen unerlässlich. Dieser Ablauf ist besonders wichtig in sensiblen Anwendungen wie dem Bankwesen, dem Gesundheitswesen und staatlichen Diensten.

Dieser Prozess, bekannt als _Sicherheitsüberprüfung_, erfordert, dass Benutzer ihre Identität erneut verifizieren, um zu bestätigen, dass sie der autorisierte Kontoinhaber sind. Durch die Implementierung der Sicherheitsüberprüfung verstärkt das System den Schutz vor unbefugtem Zugriff, fügt eine kritische Sicherheitsebene für risikoreiche Aktionen hinzu und hilft, sensible Daten zu schützen.

Erfahre, wie du den Sicherheitsüberprüfungsprozess mit Logto [Management APIs](/end-user-flows/security-verification/by-management-api) zu deiner Anwendung hinzufügen kannst.
