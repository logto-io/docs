# Organisation erstellen

Stellen Sie sich vor, Sie bauen eine [Multi-Tenant-App](https://auth.wiki/multi-tenancy) (z. B. eine Multi-Tenant-SaaS-App), die zahlreichen Kunden dient, und jeder Kunde besitzt einen eigenen dedizierten Mandanten. Wenn ein neuer Kunde hinzukommt, erstellt er ein neues Konto sowie einen neuen Mandanten für sein eigenes Geschäft.

Genauso wie Sie Ihr [Logto Cloud](https://cloud.logto.io/)-Konto registriert haben und Ihren eigenen Logto-Mandanten besitzen. Sie können dieselbe Architektur auch in Ihrer App implementieren, indem Sie die Logto-Funktion "[Organisation](/organizations)" verwenden.

## Erstellen Sie Ihre Organisationen

Es gibt zwei Möglichkeiten, Organisationen für Ihre App zu erstellen.

### Erstellen über die Logto-Konsole

Erstellen Sie Ihre Organisationen manuell über die Logto Console UI. Gehen Sie zu <CloudLink to="/organizations">Konsole > Organisationen</CloudLink>, um Organisationen zu erstellen, Mitglieder und Rollen zuzuweisen und die Benutzeroberfläche der Organisation-Anmeldeerfahrung anzupassen.

Erstellen Sie eine "[Organisation-Vorlage](/authorization/organization-template)", wenn Sie ähnliche Organisationen im Batch erstellen möchten, die dieselben [Rolle](/authorization/organization-template/configure-organization-template#create-organization-role) und [Berechtigung](/authorization/organization-template/configure-organization-template#create-organization-permission) Einstellungen teilen.

### Erstellen über die Logto Management API

Das Klicken von Schaltflächen in der Console UI funktioniert gut, aber wir möchten wirklich, dass die Endbenutzer sich selbst bedienen und Organisationen erstellen sowie die Organisationen selbst in **IHRER App** verwalten. Daher müssen Sie diese Funktionen in Ihrer App implementieren, mit Hilfe der Logto Management API.

:::note

Wenn Sie mit der Logto Management API nicht vertraut sind, stellen Sie bitte sicher, dass Sie diese Dokumente zuerst lesen.

<Url href="/concepts/core-service/#management-api">Management API</Url>
<Url href="/integrate-logto/interact-with-management-api">Interagieren mit der Management API</Url>

:::

Angenommen, Sie haben Ihren API-Backend-Server bereits über den Maschine-zu-Maschine (M2M) Mechanismus mit dem Logto Management API-Endpunkt verbunden und das M2M-Zugangstoken erworben.

Erstellen Sie eine Organisation mit der Management API ([API-Referenzen](https://openapi.logto.io/operation/operation-createorganization)):

```bash
curl \
 -X POST https://[tenant_id].logto.app/api/organizations \
 -H "Authorization: Bearer $M2M_ACCESS_TOKEN" \
 -H "Content-Type: application/json" \
 -d '{"tenantId":"string","name":"string","description":"string","customData":{},"isMfaRequired":false,"branding":{"logoUrl":"string","darkLogoUrl":"string","favicon":"string","darkFavicon":"string"},"createdAt":1234567890}'
```

Antwortbeispiel (201)

```json
{
  "tenantId": "string",
  "id": "string",
  "name": "string",
  "description": "string",
  "customData": {},
  "isMfaRequired": false,
  "branding": {
    "logoUrl": "string",
    "darkLogoUrl": "string",
    "favicon": "string",
    "darkFavicon": "string"
  },
  "createdAt": 1234567890
}
```

Als Nächstes können Sie Ihre eigene API für Ihre App implementieren. Wenn Ihre Benutzer die Aktion "Organisation erstellen" in Ihrer App ausführen, können Sie die Anfrage validieren, indem Sie ihre Berechtigungen überprüfen und dann die Logto Management API aufrufen, um den Rest der Arbeit zu erledigen.

## Validierung des Organisationstokens in Benutzeranfragen

In Ihrer App, wenn Benutzer Aktionen im Kontext einer Organisation ausführen, müssen sie das Organisationstoken anstelle des regulären Zugangstokens verwenden. Das Organisationstoken ist eine spezielle Art von [JWT](https://auth.wiki/jwt), das Organisationsberechtigungen enthält. Und genau wie bei jedem JWT [Zugangstoken](https://auth.wiki/access-token) können Sie die Token-Ansprüche dekodieren und den "Berechtigung" Anspruch zur Berechtigungsprüfung überprüfen.
