# Logto MCP Server

Logto MCP Server ist ein entfernter [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) Server. Füge die URL zu deiner KI-Anwendung hinzu, melde dich mit deinem Logto Cloud-Konto an und beginne, mit Logto zu interagieren.

:::tip
Logto MCP Server ist mit Logto und [MCP Auth](https://mcp-auth.dev) gebaut. Wenn du deinen eigenen entfernten MCP-Server bauen möchtest, siehe [Baue deinen eigenen entfernten MCP-Server](#build-your-own-remote-mcp-server).
:::

## Was du tun kannst \{#what-you-can-do}

- **Authentifizierung zu deinem Projekt hinzufügen**: Die KI erkennt dein Framework, erstellt eine Logto-Anwendung und generiert funktionierenden Integrationscode. Unterstützt über 38 Frameworks, darunter React, Next.js, Vue, Nuxt, SvelteKit, Express, Go, Python, iOS, Android, Flutter und mehr.
- **Fragen zu Logto stellen**: Die KI durchsucht die Logto-Dokumentation und gibt dir Antworten mit Quellenangabe.

Weitere Funktionen folgen in Kürze, darunter Konfiguration der Anmeldeerfahrung, RBAC und Berechtigungsmanagement mit Best-Practice-Anleitungen, Multi-Tenancy-Einrichtung für den Bau von B2B-Apps mit Organisationen und mehr.

## Einrichtung in deiner KI-Anwendung \{#set-up-in-your-ai-application}

### Claude Code \{#claude-code}

<Tabs>
  <TabItem value="claude-mcp-add" label="claude mcp add">

    ```bash
    claude mcp add --transport http --client-id mcp-client --callback-port 8080 logto https://mcp.logto.io
    ```

  </TabItem>
  <TabItem value="claude-mcp-add-json" label="claude mcp add-json">

    ```bash
    claude mcp add-json logto '{"type":"http","url":"https://mcp.logto.io","oauth":{"clientId":"mcp-client","callbackPort":8080}}'
    ```

  </TabItem>
</Tabs>

Du kannst `8080` auf jeden verfügbaren Port auf deinem Rechner ändern.

Führe dann `/mcp` in Claude Code aus und folge dem Browser-Login-Flow, um dich mit deinem Logto Cloud-Konto zu authentifizieren.

### Claude Desktop \{#claude-desktop}

1. Öffne **Einstellungen** im Hauptmenü, navigiere zu **Connectors** und klicke auf **Add custom connector**.
2. Gib den Namen `Logto MCP Server` und die entfernte MCP-Server-URL ein: `https://mcp.logto.io`.
3. Klappe **Erweiterte Einstellungen** aus und gib die OAuth Client ID ein: `mcp-client`. Klicke auf **Add**.
4. Finde in der **Connectors**-Liste **Logto MCP Server** und klicke auf **Connect**.
5. Du wirst zuerst auf die Claude-Webseite weitergeleitet. Wenn du keine aktive Sitzung hast, melde dich zuerst bei deinem Claude-Konto an.
6. Nach der Anmeldung bei Claude wirst du auf die Logto Cloud-Anmeldeseite weitergeleitet. Melde dich mit deinem Logto Cloud-Konto an und autorisiere die Verbindung.

### Cursor \{#cursor}

1. Öffne die Befehlspalette (`Command + Shift + P`) und wähle **View: Open MCP Settings**.
2. Klicke im Abschnitt **Installed MCP Servers** auf die Schaltfläche **Add Custom MCP**.
3. Füge in der sich öffnenden Datei `mcp.json` die folgende Konfiguration hinzu:

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Eine Logto Cloud-Anmeldeseite erscheint automatisch. Melde dich mit deinem Logto Cloud-Konto an, um die Verbindung herzustellen.

### OpenCode \{#opencode}

1. Füge die folgende Konfiguration zu deiner `opencode.json`-Datei hinzu:

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. Führe den folgenden Befehl zur Authentifizierung aus:
   ```bash
   opencode mcp auth Logto
   ```
   Beachte, dass `Logto` dem MCP-Servernamen in deiner Konfigurationsdatei entspricht.

Ein Browserfenster öffnet sich. Melde dich mit deinem Logto Cloud-Konto an, um zu autorisieren und die Verbindung herzustellen.

### VS Code \{#vs-code}

1. Öffne die Befehlspalette (`Command + Shift + P`) und wähle **MCP: Add Server...**.
2. Wähle **HTTP** als Servertyp.
3. Gib die URL ein: `https://mcp.logto.io`. Du kannst wählen, sie in deinem Workspace zu installieren, um bessere Isolierung und Verwaltung zu erreichen.
4. Wenn der OAuth-Flow startet, gib die App-ID ein: `mcp-client`. Lasse das App Secret leer und drücke Enter, um zu überspringen.
5. Du wirst auf die Logto Cloud-Anmeldeseite weitergeleitet. Melde dich mit deinem Logto Cloud-Konto an, um die Verbindung herzustellen.

### Windsurf \{#windsurf}

1. Öffne die Befehlspalette (`Command + Shift + P`) und wähle **Windsurf: MCP Marketplace**.
2. Klicke im Abschnitt **Installed MCPs** rechts auf das Einstellungssymbol, um die Datei `~/.codeium/windsurf/mcp_config.json` zu erstellen und zu öffnen.
3. Füge die folgende [Konfiguration](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json) hinzu:

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Eine Logto Cloud-Anmeldeseite erscheint automatisch. Melde dich mit deinem Logto Cloud-Konto an, um die Verbindung herzustellen.

### Andere MCP-Clients \{#other-mcp-clients}

Logto MCP Server funktioniert mit jedem MCP-Client, der OAuth-Autorisierung unterstützt. Verwende die folgende Konfiguration:

- **MCP-Server-URL:** `https://mcp.logto.io`
- **OAuth-Client-ID:** `mcp-client`
- **Ressource:** `https://mcp.logto.io/` (falls dein Client dies benötigt)
- **Scope:** `mcp:all` (falls dein Client dies benötigt)

Wenn du KI-Anwendungsentwickler bist und das Verbinden mit Logto MCP Server unterstützen möchtest, aber Logto MCP Server deine Anwendungs-Redirect-URI noch nicht unterstützt, [kontaktiere uns](https://logto.io/contact) mit den Details deiner Anwendung.

Logto MCP Server ist mit Logto und [MCP Auth](https://mcp-auth.dev) gebaut. Wenn du deinen eigenen entfernten MCP-Server mit Authentifizierung bauen möchtest, schaue dir die folgenden Ressourcen an:

- [Remote MCP server in action: a new entry point for SaaS products in the AI era](https://blog.logto.io/remote-mcp-server-in-action)
- [Tritt unserem Discord bei](https://discord.gg/fUQmaFyAam): Teile Erfahrungen und Best Practices zum Bau von entfernten MCP-Servern
- [MCP Auth](https://mcp-auth.dev): Alles, was du brauchst, um produktionsreife Authentifizierung zu deinem MCP-Server hinzuzufügen. Keine wochenlange Spezifikationslektüre oder Verkabelung nötig.
