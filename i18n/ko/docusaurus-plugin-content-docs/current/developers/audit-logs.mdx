# 감사 로그

Logto의 감사 로그는 사용자 활동과 이벤트를 쉽게 모니터링할 수 있게 해줍니다. 이는 다양한 사용자 관리 및 상태 점검 비즈니스 시나리오에 강력한 기반을 제공합니다.

## 모든 로그 보기 \{#view-all-logs}

<CloudLink to="/audit-logs">콘솔 > 감사 로그</CloudLink>로 이동하세요. Logto는 인증 (Authentication)
이벤트를 테이블로 캡처하고 정리합니다. 이벤트 이름, 사용자, 애플리케이션 및 타임스탬프를 추적합니다.
이벤트 이름과 애플리케이션 이름을 기준으로 결과를 필터링하여 좁힐 수 있습니다. 특정 이벤트를
클릭하면 추가 세부 정보를 제공합니다.

:::warning
감사 로그에는 사용자 인증 (Authentication) 과정에서 발생하는 로그만 포함되며, Management API 작업의 로그는 기록되지 않습니다.
:::

## 테넌트 수준에서 사용자 활동 캡처 \{#capture-user-activity-at-the-tenant-level}

Logto의 로그는 포괄적인 세부 정보를 제공하여 조치의 용이성과 고객 안전을 보장합니다. 다음 정보를 캡처하고 기록합니다:

- 이벤트 유형 (감사 로그 이벤트의 전체 목록은 [여기](https://github.com/logto-io/logto/blob/master/packages/console/src/consts/logs.ts)에서 확인할 수 있습니다)
- 관련 애플리케이션
- IP 주소
- 관련 사용자
- 로그 ID
- 타임스탬프
- 사용자 에이전트

이러한 이벤트 기록을 유지함으로써 조직은 잠재적인 보안 위험을 효과적으로 감지하고, 시스템의 무단 접근을 방지하기 위해 신속하게 대응할 수 있습니다.

![감사 로그 성공 세부 정보 페이지](./assets/audit-log-success-details-page.png)

## 사용자 수준에서 상세 분석 수행 \{#perform-a-detailed-analysis-at-the-user-level}

관리자는 특정 사용자와 관련된 로그를 상세히 분석하여 특정 이벤트에 대한 포괄적인 조사를 수행할 수 있습니다. 탐색 과정은 간단하고 사용자 친화적입니다.

사용자별 로그에 접근하려면 다음 단계를 따르세요:

1. <CloudLink to="/users">콘솔 > 사용자 관리</CloudLink>로 이동합니다.
2. 원하는 사용자를 선택하고 세부 정보 페이지로 이동합니다.
3. "사용자 로그"를 클릭합니다. 결과 테이블은 해당 사용자가 수행하고 트리거한 로그 이벤트만 표시합니다.

![특정 사용자와 관련된 감사 로그](./assets/audit-logs-related-to-specific-user.png)
