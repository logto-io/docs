# 이메일 / 전화번호 / 사용자 이름 회원가입

사용자 등록은 사용자가 애플리케이션과 상호작용을 시작하는 첫 번째 단계입니다. Logto는 사용자 이름 비밀번호, 이메일 또는 전화번호 인증, [소셜 회원가입](/end-user-flows/sign-up-and-sign-in/social-sign-in), [엔터프라이즈 SSO](/end-user-flows/enterprise-sso) 등 다양한 회원가입 방법을 지원합니다. 애플리케이션의 요구 사항에 가장 적합한 회원가입 방법을 설정할 수 있습니다.

<CloudLink to="/sign-in-experience/sign-up-and-sign-in">
  Console > Sign-in experience > Sign-up and sign-in
</CloudLink>
을 방문하여 식별자 회원가입 흐름을 설정하세요.

<img src="/img/assets/sign-in-experience-sign-up.png" alt="회원가입 설정" />

## 회원가입 식별자 설정 \{#set-up-the-sign-up-identifier}

Logto에서 새로운 사용자 계정을 성공적으로 생성하려면, 사용자는 Logto 시스템 내에서 고유하게 식별할 수 있는 **식별자**를 제공해야 합니다. 첫 번째 단계로, 회원가입 과정에서 사용자로부터 수집해야 하는 **식별자**를 설정하세요. 가능한 옵션은 다음과 같습니다:

- **사용자 이름**: 애플리케이션에 로그인할 수 있는 고유한 [사용자 이름](/user-management/user-data#username).
- **이메일 주소**: 애플리케이션에 로그인할 수 있는 유효한 [이메일 주소](/user-management/user-data#primary_email).
- **전화번호**: 애플리케이션에 로그인할 수 있는 유효한 [전화번호](/user-management/user-data#primary_phone).
- **이메일 주소 또는 전화번호**: 유효한 이메일 주소 또는 전화번호로 회원가입을 허용합니다.
- **해당 없음**: 회원가입 식별자가 활성화되지 않습니다. 이는 [소셜](/end-user-flows/sign-up-and-sign-in/social-sign-in) 전용 또는 [엔터프라이즈 SSO](/end-user-flows/enterprise-sso) 전용 회원가입 방법에 적용됩니다.

회원가입 과정에서 수집된 모든 식별자는 동일한 테넌트 내의 사용자 간에 고유해야 합니다. 이들은 [사용자 프로필](/user-management/user-data#user-profile)에 저장되며 Logto와 통합된 애플리케이션에 로그인하는 데 사용할 수 있습니다.

## 회원가입 인증 설정 \{#set-up-the-sign-up-verification-settings}

사용자 회원가입 및 향후 로그인 과정의 보안을 보장하기 위해, 회원가입 과정에서 수집한 식별자에 대한 인증 설정도 구성해야 합니다. 가능한 설정은 다음과 같습니다:

- **비밀번호 생성:** 사용자에게 회원가입 시 비밀번호를 생성하도록 요구하며, 이는 로그인 경험 설정에서 구성된 비밀번호 정책을 준수해야 합니다. 이 비밀번호는 사용자의 식별자와 함께 애플리케이션에 로그인하는 자격 증명으로 사용됩니다. 회원가입 식별자로 **사용자 이름**을 설정한 경우, 이 요구 사항은 자동으로 활성화됩니다. **사용자 이름**은 비밀번호와 함께 사용해야만 사용자의 아이덴티티를 효과적으로 확인할 수 있습니다. [비밀번호 정책](/end-user-flows/sign-up-and-sign-in/reset-password#set-up-password-policy)은 보안 요구 사항에 맞게 사용자 정의할 수 있습니다.
- **회원가입 시 인증**: 회원가입 시 이메일 주소 또는 전화번호를 인증하도록 요구합니다. 현재 Logto는 인증된 이메일과 전화번호만 식별자로 허용합니다. 이 설정은 회원가입 식별자로 **이메일 주소** 또는 **전화번호**를 사용할 때 자동으로 활성화됩니다. 사용자는 회원가입 과정에서 이메일 또는 전화번호로 전송된 인증 코드를 입력하여 소유권을 확인해야 합니다.

| 식별자               | 사용자 비밀번호 생성 | 회원가입 시 인증 |
| -------------------- | -------------------- | ---------------- |
| 사용자 이름          | 필수                 | 해당 없음        |
| 이메일 주소          | 선택 사항            | 필수             |
| 전화번호             | 선택 사항            | 필수             |
| 이메일 또는 전화번호 | 선택 사항            | 필수             |
| 해당 없음            | 해당 없음            | 해당 없음        |

## 소셜 또는 엔터프라이즈 SSO로 회원가입 \{#type-4-email-address-with-verification-username-and-password-creation}

이러한 전통적인 식별자 회원가입 방법 외에도, Logto는 소셜 및 엔터프라이즈 SSO 아이덴티티 제공자를 통한 비밀번호 없는 회원가입을 지원하여 온보딩 과정을 더욱 원활하고 사용자 친화적으로 만듭니다.

Logto에서 [소셜 커넥터](/connectors/social-connectors) 또는 [엔터프라이즈 SSO 커넥터](/connectors/enterprise-connectors)가 구성되고 활성화되면, 사용자는 커넥터가 제공하는 기존 소셜 또는 엔터프라이즈 아이덴티티를 사용하여 쉽게 회원가입할 수 있습니다. 소셜 및 엔터프라이즈 SSO 회원가입 방법은 비밀번호 생성이나 이메일 주소 또는 전화번호 인증과 같은 추가 단계를 건너뛸 수 있도록 합니다. Logto는 사용자의 인증된 소셜 또는 엔터프라이즈 아이덴티티를 통해 사용자의 정보를 자동으로 동기화하고 이를 사용자 프로필에 저장합니다.

소셜 및 엔터프라이즈 SSO 커넥터를 통한 회원가입 흐름에 대해 더 알고 싶다면 [소셜 로그인](/end-user-flows/sign-up-and-sign-in/social-sign-in/) 및 [엔터프라이즈 SSO](/end-user-flows/enterprise-sso/) 섹션을 확인하세요.

:::note
참고: 사용자 정의 회원가입 흐름에 대해서는 [UI 가져오기](/customization/bring-your-ui/) 기능을 참조하세요.
:::

## 자주 묻는 질문 \{#sign-up-with-social-or-enterprise-sso}

<details>
  <summary>

### 회원가입 시 추가 사용자 정보 수집 \{#faqs}

</summary>
  
회원가입 시 추가 사용자 프로필 정보 (예: 전체 이름, 생일, 회사 이름)를 수집하려면 두 가지 옵션이 있습니다:

- **셀프 호스팅 온보딩 흐름**: 성공적인 회원가입 후 사용자에게 맞춤형 데이터 수집을 위한 자체 온보딩 흐름으로 리디렉션합니다.
- **UI 가져오기**: 사용자 프로필 설정 API (곧 출시 예정)를 사용하여 UI 가져오기 기능을 통해 자체 회원가입 흐름을 사용자 정의합니다.

</details>

<details>
  <summary>

### 관리자 생성 사용자 / 초대된 사용자 \{#collect-additional-user-information-on-sign-up}

</summary>

[초대 전용 회원가입 흐름](/end-user-flows/sign-up-and-sign-in/disable-user-registration/#implement-an-invitation-only-sign-up-flow)을 구현하는 방법을 알아보세요.

</details>

<details>
  <summary>

### 웹사이트에 내장된 회원가입 양식 \{#admin-created-users--invited-users}

</summary>
  
Logto는 현재 로그인 및 회원가입을 위한 헤드리스 API를 지원하지 않습니다. [UI 가져오기](/customization/bring-your-ui/) 기능을 사용하여 자체 회원가입 양식을 Logto에 업로드하거나 로그인 매개변수를 사용하여 웹사이트에서 Logto로 사용자 정보를 채울 수 있습니다. 사용자 식별자 채우기에 대한 자세한 내용은 [인증 매개변수](/end-user-flows/authentication-parameters/)를 참조하세요.

</details>

<details>
  <summary>

### 신규 사용자에게 환영 이메일 보내기 \{#embedded-sign-up-forms-on-your-website}

</summary>

신규 사용자에게 환영 이메일을 보내기 위해 `User.Created` Webhook 이벤트를 구독하세요. [Webhook 이벤트](/developers/webhooks/webhooks-events/#data-mutation-hook-events)에 대해 더 알아보세요.

</details>

<details>
  <summary>

### 회원가입 시 이메일 인증 건너뛰기 \{#sending-welcome-emails-to-new-users}

</summary>

현재 Logto는 인증된 이메일과 전화번호만 식별자로 지원합니다. 인증 과정은 사용자의 식별자 보안 및 소유권을 보장하기 위해 필요합니다.
인증되지 않은 이메일 또는 전화번호에 대한 지원은 우리의 [로드맵](https://logto.productlane.com/roadmap)에 있습니다. 업데이트를 기대해 주세요!

</details>
