# 인증기 앱 OTP

## 개념 \{#concepts}

인증기 앱(Authenticator app), 또는 소프트웨어 토큰(Software Token)은 가장 널리 사용되는 [다단계 인증 (MFA)](https://auth.wiki/mfa) 방식 중 하나입니다. 이 방식은 온라인 서비스 인증의 보안을 강화하기 위해 임시 [일회용 비밀번호 (OTP)](https://auth.wiki/otp)를 생성합니다. 물리적 하드웨어 토큰과 달리, 소프트웨어 토큰은 일반적으로 사용자가 스마트폰이나 컴퓨터 브라우저 등 자신의 기기에 설치하는 애플리케이션 또는 플러그인입니다. 소프트웨어 토큰은 인증기의 기능과 개별 사용자 설정에 따라 단일 기기에서 로컬로 동작하거나 여러 기기에서 동기화될 수 있습니다.

대표적인 소프트웨어 토큰 예시로는 Google Authenticator, Microsoft Authenticator, Duo, 1Password, Authy 등이 있습니다.

## 인증(Authentication) 플로우 \{#authentication-flows}

### 인증기 앱 OTP 설정 플로우 \{#authentication-app-otp-setup-flows}

1. **QR 코드 또는 시크릿 키**: 사용자는 서비스로부터 QR 코드 또는 시크릿 키를 받습니다.
2. **계정 추가**: 인증기 앱을 사용하여 QR 코드를 스캔하거나 시크릿 키를 수동으로 입력해 계정을 추가합니다.
3. **동적 일회용 비밀번호**: 인증기 앱은 추가된 계정에 대해 1~2분마다 갱신되는 6자리 코드를 표시합니다.
4. **MFA 설정 완료**: 사용자는 이 코드를 유효 시간 내에 MFA 설정 페이지에 입력하여 인증기 앱 OTP의 MFA 설정을 완료합니다.

### 인증기 앱 OTP 검증 플로우 \{#authentication-app-otp-verification-flows}

1. **로그인 시도**: 로그인 과정에서 사용자는 MFA 입력을 요청받습니다.
2. **OTP 확인**: 인증기 앱을 열어 해당 계정의 OTP를 확인합니다.
3. **OTP 입력**: 앱에 표시된 OTP를 유효 시간 내에 2단계 인증 페이지에 입력합니다.
4. **인증(Authentication)**: 시스템이 OTP를 검증하여 성공 시 접근 권한을 부여합니다.
