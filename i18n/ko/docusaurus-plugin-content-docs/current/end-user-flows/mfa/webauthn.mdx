# 패스키 (Passkeys, WebAuthn)

[패스키 (Passkey)](https://auth.wiki/passkey)는 기존 비밀번호보다 더 안전하고 사용자 친화적인 대안을 제공합니다. 공개키 암호화 방식을 사용하여, 패스키는 사용자의 디바이스, 서비스 도메인, 그리고 사용자 ID를 연결함으로써 피싱 및 비밀번호 공격을 효과적으로 방지합니다. 다양한 디바이스나 브라우저와 호환되며, 사용자가 생체 인식 및 하드웨어 보안 기능을 활용하여 편리하게 인증 (Authentication)할 수 있습니다. [WebAuthn](https://auth.wiki/webauthn)은 웹사이트가 패스키를 구현할 수 있도록 API를 제공합니다.

Logto는 이제 다단계 인증 (MFA)을 위해 패스키(WebAuthn)를 지원합니다. 패스키 로그인 기능은 곧 제공될 예정입니다. 업데이트를 기대해 주세요.

## 개념 (Concepts) \{#concepts}

고객들은 항상 WebAuthn보다는 패스키(Passkeys)를 더 잘 알고 있습니다. 그렇다면 이 둘의 관계는 무엇이며, 어떻게 사용해야 할까요? 다음 개념들을 살펴봅시다:

- **패스키 (Passkeys)**: 패스키는 비밀번호를 대체하는 FIDO 기반의 피싱 저항 인증 정보입니다. 비대칭 공개키 암호화를 활용하여 보안을 강화합니다. USB나 Bluetooth 장치와 같은 하드웨어 토큰 또는 보안 키가 될 수 있습니다. "패스키"는 사용자에게 표시되는 인증 (Authentication) 방식이므로, 제품 클라이언트 내에서 사용해야 합니다.
- **WebAuthn**: W3C와 FIDO Alliance가 개발한 JavaScript API로, 웹 애플리케이션 인증 (Authentication)에 FIDO2 표준을 적용할 수 있게 해줍니다. 패스키는 WebAuthn이 지원하는 인증 (Authentication) 방식 중 하나입니다. Logto 콘솔에서는 이 통합을 전문적으로 "WebAuthn"이라고 부릅니다.

WebAuthn은 사용자가 선택할 수 있는 다양한 인증기(Authenticator)를 제공합니다. 로컬 및 클라우드 용도로 두 가지 유형이 있습니다:

- **플랫폼 인증기 (Platform authenticator, 내부 인증기)**: 단일 및 특정 디바이스 OS(컴퓨터, 노트북, 휴대폰, 태블릿 등)에 연결되어 있습니다. 사용자가 로그인하는 디바이스에서만 동작하며, 생체 인식이나 디바이스 비밀번호 등으로 인가 (Authorization)를 수행하므로 빠른 인증 (Authentication)이 가능합니다. 예: macOS 또는 iOS에서 Touch ID, Face ID, 또는 디바이스 비밀번호로 인증되는 iCloud Keychain; Windows Hello의 얼굴 인식, 지문, PIN 등.
- **로밍 인증기 (Roaming authenticator, 외부 인증기, 크로스 플랫폼 인증기)**: 별도의 휴대용 디바이스나 소프트웨어 애플리케이션(예: 하드웨어 보안 키, 스마트폰 등)입니다. USB 연결 또는 NFC, Bluetooth를 켜서 디바이스를 연결해야 합니다. 로밍 인증기는 단일 디바이스나 브라우저에 국한되지 않아 더 큰 유연성을 제공합니다.

WebAuthn의 원리와 프로세스에 대해 더 깊이 알고 싶다면, 우리의 블로그 글 [WebAuthn과 패스키 101](https://blog.logto.io/web-authn-and-passkey-101/) 및 [WebAuthn 통합 전 알아야 할 것들](https://blog.logto.io/webauthn-base-knowledge/)을 참고하세요.

## 제한 사항에 주의하세요 \{#pay-attention-to-limitations}

WebAuthn을 구현할 때 몇 가지 제한 사항을 반드시 인지해야 합니다:

1. **플랫폼 및 브라우저 제한**: Logto는 현재 네이티브 애플리케이션에 대한 WebAuthn 지원을 제공하지 않습니다. 또한, WebAuthn 인증기의 사용 가능 여부는 브라우저 및 디바이스의 기능에 따라 다릅니다 ([목록 확인하기](https://caniuse.com/?search=webauthn)). 따라서 WebAuthn은 다단계 인증 (MFA)을 구현할 때 항상 유일한 옵션이 아니며, 그렇지 않으면 어떤 브라우저와 디바이스가 제품에 접근할 수 있는지 제어해야 합니다.
2. **도메인 제한**: 도메인이 변경되면 기존 WebAuthn 계정을 통한 사용자 인증 (Authentication)이 어려워질 수 있습니다. 패스키는 현재 웹 페이지의 특정 도메인에 바인딩되어 있으며, 서로 다른 도메인 간에는 사용할 수 없습니다.
3. **디바이스 제한**: 디바이스를 분실하면, 특히 "이 디바이스" 플랫폼 인증기에 의존하는 경우 계정 접근이 불가능해질 수 있습니다. 인증 (Authentication) 접근성을 높이기 위해, 사용자에게 둘 이상의 인증 (Authentication) 요소를 제공하는 것이 좋습니다.
