# 인증 (Authentication) 파라미터

표준 OIDC 로그인 플로우에서, 클라이언트 애플리케이션은 인증 요청을 시작하여 사용자를 Logto가 호스팅하는 범용 로그인 경험 웹 페이지로 리디렉션합니다. 로그인 경험 설정에 따라, 사용자는 다양한 식별자, 인증 방법, 그리고 서드파티 소셜 또는 엔터프라이즈 SSO 커넥터를 사용하여 로그인 또는 회원가입할 수 있습니다.

Logto는 다음과 같은 선택적 표준 OIDC 파라미터도 지원합니다:

- `login_hint`: 사용자 식별자에 대한 힌트 제공, 예: 이메일/사용자명 미리 입력
- [`ui_locales`](/end-user-flows/authentication-parameters/ui-locales): 현재 상호작용(로그인 UI 및 관련 이메일 포함)에 대한 런타임 언어 제어

표준 [OIDC 인증 파라미터](https://openid.net/specs/openid-connect-core-1_0.html#AuthRequest) 외에도, Logto는 최종 사용자를 위한 원하는 로그인 경험을 맞춤화할 수 있도록 여러 커스텀 인증 파라미터를 제공합니다.

이는 서로 다른 사용자 그룹에 대해 특정 로그인 플로우를 강제하고자 할 때 특히 유용합니다. 다음과 같은 시나리오에 적용할 수 있습니다:

### 신규 사용자를 위한 직접 회원가입 \{#direct-sign-up-for-new-users}

타겟 등록 캠페인에서는 신규 사용자를 기본 로그인 폼을 건너뛰고 바로 회원가입 페이지로 안내하여 원활한 온보딩 경험을 제공할 수 있습니다.

**사용**: [첫 화면](/end-user-flows/authentication-parameters/first-screen) 파라미터 → `first_screen=register`

### 홈페이지에서 이메일 미리 입력된 회원가입 \{#email-prefilled-sign-up-from-homepage}

홈페이지의 히어로 섹션에 이메일 회원가입 입력 필드를 눈에 띄게 배치하여 빠른 회원가입을 유도하는 경우가 많습니다. 사용자가 이메일을 입력하고 "지금 시작하기" 버튼을 클릭하면, 이 파라미터를 통해 이메일 필드가 미리 채워진 회원가입 페이지로 리디렉션하여 회원가입 과정을 간소화할 수 있습니다.

이 방식은 구독 폼이나 기타 이메일 수집 시나리오 등 회원가입 플로우의 마찰을 줄이고자 할 때도 유용합니다.

**사용**: [첫 화면](/end-user-flows/authentication-parameters/first-screen) 파라미터 → `first_screen=identifier:sign_up&identifier=email&login_hint=foo@gmail.com`

### 직접 비밀번호 재설정 링크 \{#direct-password-reset-link}

신규 사용자가 새 비밀번호를 설정해야 하는 경우, 직접 링크를 보내 이메일 필드가 미리 입력된 비밀번호 재설정 페이지로 리디렉션할 수 있습니다.

**사용**: [첫 화면](/end-user-flows/authentication-parameters/first-screen) 파라미터 → `first_screen=reset_password`

### 즉시 소셜 로그인 \{#instant-social-login}

사이트에 소셜 미디어 로그인 버튼을 직접 제공하는 경우, 이 파라미터를 사용하여 기본 로그인 폼을 건너뛰고 사용자가 선택한 소셜 제공자로 바로 인증할 수 있습니다.

**사용**: [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in#social-sign-in) 파라미터 → `direct_sign_in=social:google`

### 엔터프라이즈 고객을 위한 직접 SSO 로그인 \{#direct-sso-sign-in-for-enterprise-customers}

여러 엔터프라이즈 고객을 대상으로 하는 제품의 경우, 이 파라미터를 통해 각 조직에 맞는 SSO 버튼을 표시하여 범용 로그인 폼을 건너뛰고 해당 SSO 제공자로 바로 로그인할 수 있습니다.

**사용**: [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in#enterprise-sso) 파라미터 → `direct_sign_in=sso:1234567890`

### 웹사이트 또는 다이얼로그에 내장된 회원가입 폼 \{#embedded-sign-up-form-on-website-or-in-a-dialog}

원활한 사용자 경험을 위해, 완전한 인증 폼을 웹사이트나 모달 다이얼로그에 직접 내장하여 사용자가 적절한 시점에 자연스럽게 회원가입 또는 로그인을 할 수 있도록 안내할 수 있습니다.

- 사용자가 소셜 로그인 버튼(예: "Google로 계속하기")을 클릭하면, [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in) 파라미터를 사용하여 즉시 소셜 제공자로 리디렉션하세요.
- 사용자가 이메일을 입력하고 회원가입 버튼을 클릭하면, [첫 화면](/end-user-flows/authentication-parameters/first-screen) 파라미터를 사용하여 이메일이 미리 입력된 회원가입 페이지로 리디렉션하세요.

위에서 언급한 인증 파라미터를 조합하면, Logto의 안전한 인증 인프라를 활용하면서도 웹사이트의 컨텍스트와 브랜딩을 유지하여 매끄럽고 전문적인 사용자 경험을 만들 수 있습니다.

### 사이트별로 다른 커스텀 로그인 방식 표시 \{#custom-sign-in-methods-displayed-for-different-sites}

제품이 내부 팀과 소비자, 또는 여러 조직을 모두 대상으로 하는 경우, 로그인 경험을 각각 맞춤화할 수 있습니다. 예를 들어:

- 내부 도구에는 이메일/사용자명과 비밀번호 로그인만 표시
- 소비자 대상 페이지에는 Google 로그인을 포함한 모든 로그인 옵션 제공

**사용**: 웹사이트 유형에 따라 서로 다른 [첫 화면](/end-user-flows/authentication-parameters/first-screen) 또는 [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in) 설정 사용

아래 내용을 계속 읽고, [첫 화면](/end-user-flows/authentication-parameters/first-screen) 맞춤화 또는 [직접 로그인](/end-user-flows/authentication-parameters/direct-sign-in) 경험 활성화를 위한 인증 파라미터에 대해 더 알아보세요.
