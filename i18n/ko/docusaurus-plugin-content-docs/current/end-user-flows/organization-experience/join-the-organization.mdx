# 조직에 가입하기

## 사용 위치 \{#where-to-use-it}

조직 목록은 일반적으로 사용자 온보딩 플로우 중에 나타납니다. 예를 들어, 관리자가 워크스페이스에 초대할 때 볼 수 있습니다.

제품 관점에서, 다음 두 가지 주요 위치에 표시될 수 있습니다:

- 로그인 또는 회원가입 중의 **조직 찾기**

## Logto Management API를 사용하여 사용자가 조직에 가입하도록 하기 \{#use-the-logto-management-api-to-let-users-join-an-organization}

이전 섹션에서는 Logto Management API를 사용하여 조직 초대를 생성하고 전송하는 방법을 다루었습니다.
다음으로, 초대받은 사용자가 애플리케이션에 도착했을 때 초대 링크를 처리하는 랜딩 페이지를 구현합니다.

- `GET /api/organization-invitations` 및 `GET /api/organization-invitations/{id}`: 모든 초대 또는 특정 ID의 초대를 가져옵니다.
  랜딩 페이지에서 이 API들을 사용하여 사용자가 받은 모든 초대 목록을 표시하거나 특정 초대의 세부 정보를 보여줄 수 있습니다.
- `PUT /api/organization-invitations/{id}/status`: 초대의 상태를 업데이트하여 초대를 수락하거나 거절합니다.
  이 API를 사용하여 사용자의 응답을 처리하세요.
