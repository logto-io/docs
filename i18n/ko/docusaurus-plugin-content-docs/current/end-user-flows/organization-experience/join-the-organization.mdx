# 조직에 가입하기

## 어디에 사용하나요 \{#where-to-use-it}

조직 목록과 가입 플로우는 일반적으로 사용자 온보딩 과정에서 나타납니다.
예를 들어, 관리자가 누군가를 워크스페이스에 초대했지만, 사용자가 이메일 초대를 건너뛰고 앱에서 직접 로그인 또는 회원가입을 하는 경우입니다.

제품 내에서 이 플로우로 진입할 수 있는 진입점을 추가할 수 있습니다.
주요 위치는 두 가지입니다:

- 로그인 또는 회원가입 중에 표시되는 **조직 찾기**

## Logto Management API를 사용하여 사용자가 조직에 가입하도록 하기 \{#use-the-logto-management-api-to-let-users-join-an-organization}

이전 섹션에서는 Logto Management API를 사용하여 조직 초대를 생성하고 발송하는 방법을 다뤘습니다.
다음으로, 초대받은 사용자가 애플리케이션에 도착했을 때 초대 링크를 처리하는 랜딩 페이지를 구현합니다.

- `GET /api/organization-invitations` 및 `GET /api/organization-invitations/{id}`: 모든 초대 또는 특정 ID의 초대를 조회합니다.
  랜딩 페이지에서 이 API들을 사용하여 사용자가 받은 모든 초대를 나열하거나 특정 초대의 세부 정보를 보여줄 수 있습니다.
- `PUT /api/organization-invitations/{id}/status`: 초대의 상태를 업데이트하여 초대를 수락하거나 거절합니다.
  이 API를 사용하여 사용자의 응답을 처리하세요.
