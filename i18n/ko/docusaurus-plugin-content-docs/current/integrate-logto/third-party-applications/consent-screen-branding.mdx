# 맞춤형 동의 화면 브랜딩

사용자가 타사 애플리케이션의 동의 화면으로 리디렉션될 때, 타사의 브랜딩 정보와 약관 링크가 올바르게 표시되는 것이 중요합니다.

Logto에서는 타사 애플리케이션의 브랜딩 정보를 맞춤화할 수 있습니다. 여기에는 애플리케이션 이름, 로고, 약관 링크가 포함됩니다.

## 브랜딩 정보 맞춤화하기 \{#customize-the-branding-information}

사용자에게 일관되고 안전한 인증 (Authentication) 경험을 제공하기 위해 타사 애플리케이션의 브랜딩 정보를 잘 설정하세요.

1. <CloudLink to="/applications/third-party-applications">콘솔 > 애플리케이션 > 타사 앱</CloudLink>
   으로 이동하여 특정 OIDC 타사 애플리케이션의 상세 페이지를 엽니다.

2. **브랜딩** 탭으로 이동합니다.

3. 동의 화면에 표시할 정보를 설정합니다:

- **표시 이름**: 동의 화면에 표시될 타사 애플리케이션의 이름입니다. 이 이름은 사용자의 정보 접근을 요청하는 타사 애플리케이션의 이름을 나타냅니다. 이 필드를 비워두면 **애플리케이션 이름**이 사용됩니다.
- **앱 로고 (라이트)**: 동의 화면에 표시될 타사 애플리케이션의 로고입니다. 이 로고는 사용자의 정보 접근을 요청하는 타사 애플리케이션의 브랜드를 나타냅니다. 타사 애플리케이션의 로고와 범용 로그인 경험 로고가 모두 제공된 경우, 두 로고가 동의 화면에 함께 표시됩니다.
- **앱 로고 (다크)**: 다크 모드 로그인 경험이 활성화된 경우에만 사용 가능합니다. 다크 모드 설정은 <CloudLink to="/sign-in-experience/branding">콘솔 > 로그인 & 계정 > 브랜딩</CloudLink> 페이지에서 관리할 수 있습니다.
- **이용 약관 URL**: 동의 화면에 표시될 타사 애플리케이션의 약관 링크입니다.
- **개인정보처리방침 URL**: 동의 화면에 표시될 타사 애플리케이션의 개인정보처리방침 링크입니다.
