# Logto 인증 (Authentication) 통합하기

Logto는 웹, 모바일, 데스크톱 애플리케이션을 위한 종합적인 인증 (Authentication) 솔루션을 제공하며, 서비스 간 [기계 간 (M2M)](/quick-starts/m2m) 인증도 지원합니다. 또한 [OpenID Connect(OIDC)](https://auth.wiki/openid-connect) 및 [OAuth 2.0](https://auth.wiki/oauth-2.0)과 같은 표준 프로토콜을 통해 [서드파티 애플리케이션](/integrate-logto/third-party-applications)에 대한 아이덴티티 제공자 (IdP) 역할도 할 수 있습니다.

필요에 가장 적합한 솔루션을 선택하여 통합을 시작하세요:

## 애플리케이션에 인증 (Authentication) 추가하기 \{#add-authentication-for-your-applications}

사용자 대상 애플리케이션(웹, 모바일, 데스크톱 앱 등) 또는 서비스 간 통신을 위한 기계 간 (M2M) 애플리케이션을 구축하든, Logto를 통합하여 종합적인 [인증 (Authentication)](/end-user-flows) 및 [사용자 관리](/user-management) 기능을 빠르게 구현할 수 있습니다.

선호하는 프레임워크 또는 프로그래밍 언어에 대한 통합 가이드를 찾아보세요:

## SAML 앱 추가하기 \{#add-saml-apps}

Logto(아이덴티티 제공자 (IdP)로서)는 SAML 프로토콜 기반 애플리케이션(서비스 제공자)과의 통합을 지원합니다. SAML 앱 통합을 통해 엔터프라이즈 사용자에게 안전하고 표준화된 싱글 사인온 (SSO) 경험을 제공할 수 있습니다.

## OIDC/OAuth 앱 추가하기 \{#add-oidc-oauth-apps}

서드파티 애플리케이션이 Logto 아이덴티티 시스템으로 인증 (Authentication)할 수 있도록 하려면, Logto를 [아이덴티티 제공자 (IdP)](https://auth.wiki/identity-provider)로 구성할 수 있습니다.

Logto는 OAuth, OIDC와 같은 표준 프로토콜을 통해 IdP로 사용할 수 있으며, 외부 서비스는 "Logto로 로그인"을 "Google로 로그인"처럼 구현하여, 해당 서비스의 사용자가 Logto 사용자 시스템을 통해 인증할 수 있습니다.

Logto에서 서드파티 애플리케이션을 생성하면, 아이덴티티 시스템과 외부 서비스 간의 안전한 연결이 구축됩니다.

## Management API로 Logto 기능 확장하기 \{#extend-logto-features-with-management-api}

Logto [Management API](https://openapi.logto.io/)를 사용하면 애플리케이션에서 Logto 기능을 프로그래밍 방식으로 제어하고 자동화할 수 있습니다.

Logto의 기능을 확장하여 제품을 강화해야 할 때(예: [다중 테넌트 조직 관리](/organizations) 또는 Logto와 함께 워크플로우 자동화 등), Logto Management API를 활용하여 제품 요구에 맞는 고급 기능을 구축할 수 있습니다.
