:::note
Logto Android SDK의 최소 지원 Android API 레벨은 레벨 24입니다.
:::

Logto Android SDK는 두 가지 주요 버전으로 제공됩니다:

- **v3 (beta)**: [Chrome Custom Tabs](https://developer.android.com/develop/ui/views/layout/webapps/overview-of-android-custom-tabs) (시스템 브라우저)에서 로그인 환경을 열어 패스키 로그인을 지원하고 브라우저 세션을 공유합니다. v3은 <MainSiteUrl href="/integrations/wechat-native">WeChat (Native)</MainSiteUrl> 및 <MainSiteUrl href="/integrations/alipay-native">Alipay (Native)</MainSiteUrl> 커넥터 지원을 제거합니다. 대신 브라우저를 통해 작동하는 <MainSiteUrl href="/integrations/wechat-web">WeChat (Web)</MainSiteUrl> 및 <MainSiteUrl href="/integrations/alipay-web">Alipay (Web)</MainSiteUrl>을 사용할 수 있습니다. 네이티브 커넥터에 의존하는 경우 v2를 유지하세요.
- **v2**: 네이티브 소셜 커넥터에 필요한 임베디드 WebView에서 로그인 환경을 엽니다. 단, <MainSiteUrl href="/end-user-flows/sign-up-and-sign-in/passkey-sign-in">패스키 로그인</MainSiteUrl>은 지원하지 않습니다 (WebView는 패스키의 기반 표준인 WebAuthn을 지원하지 않습니다).

이 가이드는 두 버전을 모두 다룹니다. 아래 탭에서 버전을 선택하면, 선택이 이 가이드 전반에 걸쳐 동기화됩니다.

Logto Android SDK를 설치하기 전에, Gradle 프로젝트 빌드 파일의 저장소 구성에 `mavenCentral()`이 추가되어 있는지 확인하세요:

```kotlin title="settings.gradle.kts"
dependencyResolutionManagement {
  repositories {
    mavenCentral()
  }
}
```

SDK가 인터넷에 접근해야 하므로, `AndroidManifest.xml` 파일에 다음 권한을 추가해야 합니다:

```xml title="AndroidManifest.xml"
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
  xmlns:tools="http://schemas.android.com/tools">

  <!-- 인터넷 권한 추가 -->
  <uses-permission android:name="android.permission.INTERNET" />

  <!-- 기타 구성... -->
</manifest>
```
