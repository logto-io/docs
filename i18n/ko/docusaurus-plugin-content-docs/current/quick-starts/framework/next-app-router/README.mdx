### API 리소스를 위한 액세스 토큰 (Access token) 가져오기 \{#fetch-access-token-for-the-api-resource}

<FetchAccessTokenForApiResources />

서버 컴포넌트에서 액세스 토큰 (Access token)이 필요하다면, `getAccessTokenRSC` 함수를 사용할 수 있습니다:

```tsx title="app/page.tsx"
import { getAccessTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const accessToken = await getAccessTokenRSC(logtoConfig, 'https://shopping.your-app.com/api');

  return (
    <main>
      <p>액세스 토큰 (Access token): {accessToken}</p>
    </main>
  );
}
```

:::tip

HTTP는 스트리밍이 시작된 후 쿠키 설정을 허용하지 않으므로, `getAccessTokenRSC`는 쿠키 값을 업데이트할 수 없습니다. 따라서 액세스 토큰 (Access token)이 갱신되더라도 세션에 저장되지 않습니다. 클라이언트 사이드 또는 라우트 핸들러에서 `getAccessToken` 함수를 사용하는 것을 권장합니다.

:::

### 조직 토큰 (Organization token) 가져오기 \{#fetch-organization-tokens}

<FetchOrganizationTokenForUser />

서버 컴포넌트에서 조직 토큰 (Organization token)이 필요하다면, `getOrganizationTokenRSC` 함수를 사용할 수 있습니다:

```tsx title="app/page.tsx"
import { getOrganizationTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const token = await getOrganizationTokenRSC(logtoConfig, 'organization-id');

  return (
    <main>
      <p>조직 토큰 (Organization token): {token}</p>
    </main>
  );
}
```

:::tip

HTTP는 스트리밍이 시작된 후 쿠키 설정을 허용하지 않으므로, `getOrganizationTokenRSC`는 쿠키 값을 업데이트할 수 없습니다. 따라서 액세스 토큰 (Access token)이 갱신되더라도 세션에 저장되지 않습니다. 클라이언트 사이드 또는 라우트 핸들러에서 `getOrganizationToken` 함수를 사용하는 것을 권장합니다.

:::

## 인가되지 않은 경우 자동 로그인 리디렉션 \{#automatic-sign-in-redirect\}

:::tip
`signIn` 헬퍼는 로그인 세션을 설정하기 위해 쿠키를 수정하므로, React Server Component (RSC)에서 직접 호출할 수 없습니다. RSC에서 인가되지 않은 사용자를 감지했을 때 자동으로 로그인을 트리거하려면, 별도의 라우트 핸들러에서 `signIn`을 호출하고 해당 라우트로 리디렉션하세요.
:::

```ts title="app/sign-in/route.ts"
import { signIn } from '@logto/next/server-actions';
import { logtoConfig } from '../../logto';

export async function GET() {
  await signIn(logtoConfig);
}
```

```tsx title="app/protected/page.tsx"
import { getLogtoContext } from '@logto/next/server-actions';
import { redirect } from 'next/navigation';
import { logtoConfig } from '../logto';

export default async function ProtectedPage() {
  const { isAuthenticated } = await getLogtoContext(logtoConfig);

  if (!isAuthenticated) {
    redirect('/sign-in');
  }

  return <div>보호된 콘텐츠</div>;
}
```
