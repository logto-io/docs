### API 리소스의 액세스 토큰 (Access token) 가져오기 \{#fetch-access-token-for-the-api-resource}

<FetchAccessTokenForApiResources />

서버 컴포넌트에서 액세스 토큰 (Access token)을 가져와야 하는 경우, `getAccessTokenRSC` 함수를 사용할 수 있습니다:

```tsx title="app/page.tsx"
import { getAccessTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const accessToken = await getAccessTokenRSC(logtoConfig, 'https://shopping.your-app.com/api');

  return (
    <main>
      <p>Access token: {accessToken}</p>
    </main>
  );
}
```

:::tip[RSC 토큰 캐싱 제한 사항]

React Server Components는 쿠키를 쓸 수 없습니다 (Next.js의 제한). `getAccessTokenRSC`는 리프레시 토큰 (Refresh token)을 사용해 만료된 토큰을 갱신할 수 있지만, 새 액세스 토큰 (Access token)은 세션 쿠키에 저장되지 않습니다. 즉, 캐시된 토큰이 만료된 경우 모든 RSC 요청마다 토큰 갱신이 발생할 수 있습니다.

**해결 방법:**

1. **Server Actions와 함께 클라이언트 컴포넌트 사용** - 클라이언트 컴포넌트에서 Server Actions를 통해 `getAccessToken`을 호출하면 쿠키를 갱신할 수 있습니다.
2. **외부 세션 스토리지 사용** - Redis/KV 스토리지와 함께 `sessionWrapper`를 구성하세요. 쿠키에는 고정된 세션 ID만 저장되고, 토큰 데이터는 외부 스토리지에 저장되어 RSC에서도 갱신된 토큰을 유지할 수 있습니다. 아래 [외부 세션 스토리지 사용](#use-external-session-storage)을 참고하세요.

:::

### 조직 토큰 (Organization token) 가져오기 \{#fetch-organization-tokens}

<FetchOrganizationTokenForUser />

서버 컴포넌트에서 조직 토큰 (Organization token)을 가져와야 하는 경우, `getOrganizationTokenRSC` 함수를 사용할 수 있습니다:

```tsx title="app/page.tsx"
import { getOrganizationTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const token = await getOrganizationTokenRSC(logtoConfig, 'organization-id');

  return (
    <main>
      <p>Organization token: {token}</p>
    </main>
  );
}
```

:::tip[RSC 토큰 캐싱 제한 사항]

`getAccessTokenRSC`와 마찬가지로, 갱신된 조직 토큰 (Organization token)은 RSC에 저장되지 않습니다. [위의 해결 방법](#fetch-access-token-for-the-api-resource)을 참고하세요.

:::

## 인증되지 않은 경우 자동 로그인 리디렉션 \{#automatic-sign-in-redirect\}

:::tip
`signIn` 헬퍼는 로그인 세션을 설정하기 위해 쿠키를 수정하므로, React Server Component (RSC)에서 직접 호출할 수 없습니다. RSC에서 인증되지 않은 사용자를 감지했을 때 자동으로 로그인하도록 하려면, 별도의 라우트 핸들러에서 `signIn`을 호출하고 해당 라우트로 리디렉션하세요.
:::

```ts title="app/sign-in/route.ts"
import { signIn } from '@logto/next/server-actions';
import { logtoConfig } from '../../logto';

export async function GET() {
  await signIn(logtoConfig);
}
```

```tsx title="app/protected/page.tsx"
import { getLogtoContext } from '@logto/next/server-actions';
import { redirect } from 'next/navigation';
import { logtoConfig } from '../logto';

export default async function ProtectedPage() {
  const { isAuthenticated } = await getLogtoContext(logtoConfig);

  if (!isAuthenticated) {
    redirect('/sign-in');
  }

  return <div>Protected content</div>;
}
```
