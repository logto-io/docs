## 시작하기 전에 \{#before-you-start}

클라이언트 애플리케이션은 Logto에서 액세스 토큰 (Access token)을 받아야 합니다. 아직 클라이언트 통합을 설정하지 않았다면, React, Vue, Angular 또는 기타 클라이언트 프레임워크를 위한 [빠른 시작](/quick-starts)이나 서버 간 접근을 위한 [기계 간 (M2M) 가이드](/quick-starts/m2m)를 확인하세요.

이 가이드는 {getFrameworkName(props.framework)} 애플리케이션에서 이러한 토큰의 **서버 측 검증**에 중점을 둡니다.

### 학습 내용 \{#what-you-will-learn}

- **JWT 검증:** 액세스 토큰 (Access token)을 검증하고 인증 (Authentication) 정보를 추출하는 방법 학습
- **미들웨어 구현:** API 보호를 위한 재사용 가능한 미들웨어 생성
- **권한 모델:** 다양한 인가 (Authorization) 패턴 이해 및 구현:
  - 애플리케이션 전체 엔드포인트를 위한 글로벌 API 리소스
  - 테넌트별 기능 제어를 위한 조직 권한
  - 다중 테넌트 데이터 접근을 위한 조직 수준 API 리소스
- **RBAC 통합:** API 엔드포인트에서 역할 기반 권한 (Role) 및 스코프 (Scope) 적용

- Logto 애플리케이션 구성 완료 ([빠른 시작](/quick-starts) 참고)
