# SMS 커넥터

SMS 커넥터를 구성하면 사용자의 휴대폰 번호로 [일회용 비밀번호 (OTPs)](https://auth.wiki/otp)를 전송할 수 있습니다. 이 비밀번호 없는 인증 (Authentication) 메커니즘은 [회원가입](/end-user-flows/sign-up-and-sign-in/sign-up), [로그인](/end-user-flows/sign-up-and-sign-in/sign-in), [비밀번호 찾기](/end-user-flows/sign-up-and-sign-in/reset-password), [계정 연결 프로세스](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking), [멤버 초대](/end-user-flows/organization-experience/invite-organization-members), [사용자 아이덴티티 검증](/end-user-flows/security-verification) 등 다양한 시나리오에서 활용할 수 있습니다. 이 방식은 사용자 인증 (Authentication) 과정을 간소화하고, 비밀번호 관련 침해 위험을 최소화하여 보안을 강화합니다.

## 구성 단계 \{#configuration-steps}

SMS 커넥터를 추가하려면 아래 단계를 따르세요:

1. <CloudLink to="/connectors/passwordless">콘솔 > 커넥터 > 이메일 및 SMS 커넥터</CloudLink>로
   이동합니다.
2. 새 SMS 커넥터를 추가하려면 "설정" 버튼을 클릭하고 원하는 SMS 제공업체를 선택하세요.
3. 선택한 제공업체의 README 문서를 확인하세요.
4. "파라미터 구성" 섹션의 필수 항목을 입력하세요.
5. JSON 에디터를 사용하여 SMS 템플릿을 커스터마이즈하세요.
6. "일반" 템플릿을 사용해 휴대폰으로 인증 코드를 전송하여 구성을 테스트하세요.
7. "저장 및 완료"를 클릭하여 마무리하세요.

## SMS 커넥터 변경하기 \{#change-sms-connector}

다른 SMS 커넥터로 변경하려면:

- 커넥터 상세 페이지에서 우측 상단의 "커넥터 변경"을 클릭하세요.
- 또는 기존 커넥터를 삭제한 후 새 커넥터를 생성하세요.

참고: *SMS 커넥터*는 단 하나만 활성화할 수 있습니다.

이미 SMS 커넥터가 있는 경우 "SMS 커넥터 변경"만 선택할 수 있습니다. 콘솔에서 이 작업을 수행하면 저장 즉시 적용되므로, 저장 전에 반드시 구성을 테스트하세요.

서비스의 안정성을 위해, Cloud 사용자는 개발 환경용 [dev 테넌트](/logto-cloud/tenant-settings#development)를 생성할 수 있습니다. 개발 환경에서 구성을 테스트한 후, [운영 환경](/logto-cloud/tenant-settings#production)으로 "마이그레이션"하여 설정을 저장하세요.

## 엔드유저 플로우에서 휴대폰 번호 활성화하기 \{#enable-phone-number-for-end-user-flows}

SMS 커넥터를 추가 및 구성한 후, 휴대폰 기반 인증 (Authentication) 플로우를 활성화할 수 있습니다:

- [**회원가입 방식 설정**](/end-user-flows/sign-up-and-sign-in/sign-up): <CloudLink to="/sign-in-experience/sign-up-and-sign-in">로그인 경험 > 회원가입 및 로그인</CloudLink>으로 이동하세요. 회원가입 섹션에서 "휴대폰 번호" 또는 "이메일 주소 또는 휴대폰 번호"를 식별자로 추가하세요. 참고: 회원가입 시 휴대폰 번호 인증은 필수입니다.
- [**로그인 방식 설정**](/end-user-flows/sign-up-and-sign-in/sign-in): 로그인 섹션에서 "휴대폰 번호"를 로그인 방식으로 추가하세요. 인증 (Authentication) 방식으로 비밀번호 또는 SMS 인증 코드를 선택할 수 있습니다. 추가 옵션이 필요하면 "다른 방식 추가"를 클릭하세요.
- [**비밀번호 찾기 설정**](/end-user-flows/sign-up-and-sign-in/reset-password): 로그인 섹션에서 비밀번호 로그인이 활성화된 경우 "비밀번호 찾기" 설정이 표시됩니다. "휴대폰 인증 코드"를 활성화하여 SMS로 비밀번호 재설정을 허용하세요.
- [**MFA용 SMS 설정**](/end-user-flows/mfa/sms-mfa): <CloudLink to="/mfa">로그인 경험 > 다단계 인증 (MFA)</CloudLink>으로 이동하세요. 보안 강화를 위해 "SMS 인증 코드"를 2차 인증 수단으로 활성화하세요.
- [**계정 설정 구성**](/end-user-flows/account-settings/by-account-api#manage-phone): [Account API](/end-user-flows/account-settings/by-account-api)를 사용하여 계정 센터를 커스터마이즈하고 사용자가 휴대폰 번호를 관리할 수 있도록 하세요.

:::warning
구성이 즉시 적용되기 때문에, 기존 SMS 커넥터를 삭제하면 엔드유저의 정상적인 로그인 및 회원가입 과정에 영향을 줄 수 있습니다.

예를 들어, 로그인 식별자에 "휴대폰 번호 + 인증 코드" 조합이 포함된 경우, 기존 SMS 커넥터를 삭제하면 사용자가 이메일로 인증 코드를 받을 수 없게 되어 로그인 플로우에서 오류가 발생합니다.

커넥터 작업 시 잠재적 위험을 반드시 인지하세요.
:::

## 자주 묻는 질문 \{#faqs}

<details>
<summary>

### Logto에 내가 원하는 SMS 커넥터가 없습니다. 어떻게 해야 하나요? \{#logto-doesnt-have-the-sms-connector-i-want-to-use-what-should-i-do}

</summary>

더 많은 커넥터를 계속 개발 중입니다. 추가 옵션이 필요하다면 Discord에서 요구 사항을 알려주시고, [GitHub](https://github.com/logto-io/logto/issues)에 기능 요청을 남겨주세요. 추가 지원이 필요하면 [이메일로 문의](mailto:contact@logto.io)하실 수도 있습니다.

오픈소스 Logto 사용자라면, 손쉽게 확장 가능한 커넥터 생성 방식을 제공하므로, 특정 시나리오에 맞게 [직접 커넥터를 커스터마이즈](/logto-oss/develop-your-connector)할 수 있습니다. 언제든지 Logto에 pull request를 제출해 주시면, 커뮤니티의 다른 분들도 여러분의 작업 혜택을 누릴 수 있습니다.

</details>
