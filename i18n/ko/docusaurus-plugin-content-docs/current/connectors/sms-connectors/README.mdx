# SMS 커넥터

SMS 커넥터를 구성하면 사용자의 전화번호로 [일회용 비밀번호 (OTPs)](https://auth.wiki/otp)를 보낼 수 있습니다. 이 비밀번호 없는 인증 메커니즘은 [회원가입](/end-user-flows/sign-up-and-sign-in/sign-up), [로그인](/end-user-flows/sign-up-and-sign-in/sign-in), [비밀번호 찾기](/end-user-flows/sign-up-and-sign-in/reset-password), [계정 연결 프로세스](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking), [회원 초대](/end-user-flows/organization-experience/invite-organization-members) 및 [사용자 아이덴티티 검증](/end-user-flows/security-verification) 등 다양한 시나리오에서 활용될 수 있습니다. 이는 사용자 인증을 간소화하고 비밀번호 관련 침해의 위험을 최소화하여 보안을 강화합니다.

## 구성 단계 \{#configuration-steps}

SMS 커넥터를 추가하려면 아래 단계를 따르세요:

1. <CloudLink to="/connectors/passwordless">콘솔 > 커넥터 > 이메일 및 SMS 커넥터</CloudLink>로
   이동합니다.
2. 새 SMS 커넥터를 추가하려면 "설정" 버튼을 클릭하고 원하는 SMS 제공업체를 선택합니다.
3. 선택한 제공업체의 README 문서를 검토합니다.
4. "매개변수 구성" 섹션에서 필요한 필드를 완료합니다.
5. JSON 편집기를 사용하여 SMS 템플릿을 사용자 정의합니다.
6. "일반" 템플릿을 사용하여 휴대폰으로 인증 코드를 보내 구성 테스트를 수행합니다.
7. "저장 및 완료"를 클릭하여 마무리합니다.

## SMS 커넥터 변경하기 \{#change-sms-connector}

다른 SMS 커넥터로 변경하려면:

- 커넥터 세부 정보 페이지에 액세스하여 오른쪽 상단의 "커넥터 변경"을 클릭합니다.
- 또는 기존 커넥터를 삭제하고 새로 생성합니다.

참고: *SMS 커넥터*는 하나만 활성화할 수 있습니다.

이미 SMS 커넥터가 있는 경우 "SMS 커넥터 변경"만 선택할 수 있습니다. 콘솔에서 이 작업을 수행할 때 저장하면 즉시 적용되므로 저장하기 전에 구성을 테스트해야 합니다.

서비스의 안정성을 보장하기 위해, Cloud 사용자는 개발 환경을 위한 [개발 테넌트](/logto-cloud/tenant-settings#development)를 생성할 수 있습니다. 개발 환경에서 구성을 테스트한 후, 구성을 [프로덕션](/logto-cloud/tenant-settings#production)으로 "이전"한 다음 설정을 저장합니다.

## 전화번호 회원가입 또는 로그인 활성화하기 \{#enable-phone-number-for-end-user-flows}

SMS 커넥터를 성공적으로 추가하고 구성을 완료하면, 비즈니스 요구에 따라 전화번호 + 인증 코드에 의존하는 로그인 및 회원가입 구성을 활성화할 수 있습니다.

1. **이동 경로**: <CloudLink to="/sign-in-experience/sign-up-and-sign-in">콘솔 > 로그인 경험 > 회원가입 및 로그인</CloudLink>
2. [**회원가입 방법 설정**](/end-user-flows/sign-up-and-sign-in/sign-up): 회원가입 식별자로 "전화번호" 또는 "이메일 주소 또는 전화번호"를 선택합니다. 참고: 회원가입 시 전화번호 인증은 현재 필수입니다.
3. [**로그인 방법 설정**](/end-user-flows/sign-up-and-sign-in/sign-in): 선택한 회원가입 식별자는 자동으로 로그인 방법이 됩니다. 추가 로그인 옵션을 위해 "다른 추가"를 클릭합니다.
4. **변경 사항 저장 및 테스트:** "변경 사항 저장"을 클릭하여 새로운 로그인 경험을 구현합니다. "[라이브 미리보기](/customization/live-preview)"를 사용하여 기능을 테스트하는 것을 잊지 마세요.

:::warning
구성이 즉시 적용되기 때문에, 기존 SMS 커넥터를 삭제하면 최종 사용자의 정상적인 로그인 및 등록 프로세스에 영향을 미칠 수 있습니다.

예를 들어, 로그인 식별자가 "전화번호 + 인증 코드" 조합을 포함하는 경우, 기존 SMS 커넥터를 삭제하면 사용자가 이메일을 통해 인증 코드를 받을 수 없게 되어 로그인 흐름에서 오류가 발생합니다.

커넥터 작업 시 잠재적인 위험을 인지하세요.
:::

## 자주 묻는 질문 \{#faqs}

<details>
<summary>

### Logto에 내가 사용하고 싶은 SMS 커넥터가 없습니다. 어떻게 해야 하나요? \{#logto-doesnt-have-the-sms-connector-i-want-to-use-what-should-i-do}

</summary>

우리는 더 많은 커넥터를 개발 중입니다. 추가 옵션이 필요하면 Discord에서 요구 사항을 알려주시고 [GitHub](https://github.com/logto-io/logto/issues)에서 기능 요청을 제출하세요. 추가 지원이 필요하면 [이메일로 문의](mailto:contact@logto.io)할 수도 있습니다.

오픈 소스 Logto 사용자에게는 특정 시나리오에 따라 [자신의 커넥터를 사용자 정의](/logto-oss/develop-your-connector)할 수 있는 확장 가능한 커넥터 생성 방법을 제공합니다. 커뮤니티의 다른 사람들이 여러분의 작업으로부터 혜택을 받을 수 있도록 Logto에 풀 리퀘스트를 제출하는 것을 항상 환영합니다.

</details>
