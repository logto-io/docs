# 테넌트 설정

## 테넌트 지역 \{#tenant-region}

테넌트를 생성할 때, 테넌트 데이터가 저장될 지역을 선택할 수 있습니다. 테넌트가 생성된 후에는 변경할 수 없습니다. 사용 가능한 지역은 다음과 같습니다:

- 유럽 (네덜란드)
- 미국 서부 (애리조나)
- 호주 (오스트레일리아 동부)
- 일본 (일본 동부)

일반적으로 고객과 가장 가까운 지역을 선택하여 지연 시간을 최소화하고 성능을 향상시키는 것이 좋습니다.

Logto는 글로벌 엣지 네트워크를 활용하여 애플리케이션에 최고의 성능과 가용성을 제공합니다. 요청 라우팅이 최적화되어 사용자가 항상 최상의 성능을 가진 옵션에 연결될 수 있도록 보장합니다.

:::note
다른 지역이 필요하신가요? [문의하기](https://logto.io/contact) 를 통해 다음을 요청하세요:

- 새로운 퍼블릭 클라우드 지역 요청
- 원하는 위치에서 Logto 프라이빗 클라우드 배포 문의
  :::

## 테넌트 유형: 개발(Dev) vs. 운영(Prod) \{#tenant-types-dev-vs-prod}

Logto Cloud에는 두 가지 유형의 테넌트가 있습니다: 개발(Development)과 운영(Production). 이 테넌트 구분을 통해 다양한 환경에서 프로젝트를 효율적으로 관리할 수 있으며, 동시에 Logto의 모든 가치를 누릴 수 있습니다.

테넌트 생성 시 유형을 선택할 수 있습니다. 운영 환경에서 실제 서비스를 시작할 준비가 되면 두 가지 옵션이 있습니다:

- **새로운 운영(Production) 테넌트 생성**  
  새로운 운영 테넌트를 새로 설정하고 처음부터 구성하세요. 개발 환경과 운영 환경을 분리하고 싶을 때 이상적입니다.
- **현재 개발(Dev) 테넌트를 운영(Production)으로 전환**  
  구성을 다시 하거나 사용자를 마이그레이션하지 않으려면, 기존 개발 테넌트를 Pro 요금제(월 $16부터) 구독을 통해 유료 운영 테넌트로 업그레이드할 수 있습니다.
  - 개발 테넌트에서 사용한 모든 유료 기능은 Stripe 결제에 반영됩니다.
  - **한 번 전환하면 테넌트를 다시 개발 환경으로 되돌릴 수 없으니, 준비가 되었는지 반드시 확인하세요.**

### 개발(Development) \{#development}

개발 테넌트(Dev 테넌트)는 주로 테스트 용도로 사용되며, 운영 환경에서는 사용하지 않아야 합니다. 이 테넌트에서는 유료 요금제에서 제공하는 프리미엄 및 유료 기능을 무료로, 구독 없이 사용할 수 있습니다.

하지만 개발 테넌트에는 다음과 같은 제한이 있습니다:

- 90일이 지난 사용자 및 조직은 자동으로 삭제됩니다.
- 로그인 경험 중에 테넌트가 개발 모드임을 알리는 배너가 표시됩니다.
- 개발 테넌트는 특정 기능에 대해 할당량 제한이 있을 수 있습니다. 해당 제한은 기능 상세 페이지에서 확인할 수 있습니다.
- Logto는 개발 테넌트의 할당량 제한을 업데이트할 수 있으며, 사전에 최대한 안내해 드릴 예정입니다.

| ------------------------------- | ----------- |

| **사용자 인증(Authentication)** |             |
| 소셜 커넥터                     | 100         |

| **사용자 관리**                 |             |

| **조직**                        |             |

### 운영(Production) \{#production}

운영 테넌트는 최종 사용자가 실제 앱에 접근하는 환경이며, [유료 구독](https://logto.io/pricing)이 필요할 수 있습니다. Free 요금제 또는 Pro 요금제에 가입하여 운영 테넌트를 생성할 수 있습니다. Free 요금제 구독 시 최대 10개의 테넌트만 생성할 수 있습니다.

## MFA 활성화 \{#enable-mfa}

Logto Pro/Enterprise 테넌트의 모든 멤버에게 다단계 인증(MFA)을 요구하여 워크스페이스 보안을 강화하세요.

셀프 서비스가 아직 제공되지 않으므로, 이 기능을 활성화하려면 [문의하기](https://logto.io/contact) 를 이용해 주세요.

## 엔터프라이즈 SSO 활성화 \{#enable-enterprise-sso}

Logto Cloud는 Google Workspace, Okta, Azure AD 등과 같은 공급자를 포함하여 유료 테넌트에 대해 엔터프라이즈 싱글 사인온(SSO) 통합을 지원합니다.

시작하려면 [문의하기](https://logto.io/contact) 를 이용해 주세요. 빠르게 설정할 수 있도록 도와드리겠습니다.
