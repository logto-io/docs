# 시스템 한도

시스템 한도는 테넌트가 생성할 수 있는 엔티티의 최대 수를 정의합니다. 이는 플랫폼의 안정성을 보호하며, 과금과는 독립적입니다. 가격 페이지의 일부 항목에는 _무제한_ 또는 *사용량 기반 결제*로 표시되어 있습니다. 이는 과금 한도가 없음을 의미하지만, 공정한 사용을 보장하기 위해 내부 시스템 한도는 여전히 적용됩니다. 한도 상향이 필요한 경우 개별적으로 검토할 수 있습니다.

## 한도의 종류 \{#types-of-limits}

### 시스템 리소스 및 엔티티 한도 \{#system-resource-and-entity-caps}

애플리케이션, 역할, 조직, 커넥터, 그리고 총 토큰 소비량과 같은 일부 사용량 기반 시스템 리소스 등 객체의 최대 개수입니다. 이 한도는 요금제에 따라 다릅니다.

### 속도 제한 (Rate limit) \{#rate-limit}

테넌트별로 적용되는 요청 수준 보호입니다. 일반적인 기준은 **테넌트당 10초에 200건의 요청**입니다. 트래픽이 정상적이고 지속적으로 이 한도를 초과하는 경우, 개별적으로 예외 적용을 평가할 수 있습니다.

---

사용 사례가 유효하고 한도에 도달했다면 저희에게 연락해 주세요. 이를 통해 한도를 조정하고 실제 시나리오를 지원할 수 있습니다.

## 테넌트 수준 한도 보호 \{#tenant-level-rate-limit-protection}

### 개발(Dev) 테넌트 \{#dev-tenants}

[개발(Dev) 테넌트](/logto-cloud/tenant-settings#development)는 테스트를 위해 무료로 제공됩니다. 오용을 방지하고 명확한 기대치를 제공하기 위해, Dev 테넌트에는 고정된 엔티티 한도가 있습니다. 더 많은 한도가 필요하다면 한도 상향을 요청하거나 Pro로 업그레이드하여 전체 접근 권한을 얻을 수 있습니다.

Dev 테넌트는 별도의 [데이터 보존 정책](/logto-cloud/dev-tenant-data-retention)도 따릅니다.

| **기능**                                   | **엔티티 한도** |
| ------------------------------------------ | --------------- |
| **액세스 토큰 (Access token) 한도 (월간)** | 월 50,000개     |
| **애플리케이션**                           |                 |
| 전체 애플리케이션                          | 100             |
| 기계 간(M2M) 애플리케이션                  | 100             |
| 서드파티 애플리케이션                      | 100             |
| **API 리소스**                             |                 |
| 리소스 개수                                | 100             |
| **사용자 인증 (Authentication)**           |                 |
| 소셜 커넥터                                | 100             |
| 엔터프라이즈 SSO                           | 100             |
| **사용자 관리**                            |                 |
| 사용자 역할                                | 100             |
| 기계 간 역할                               | 100             |
| 역할당 권한                                | 100             |
| **조직**                                   |                 |
| 조직 개수                                  | 5,000           |
| 조직당 사용자                              | 100,000         |
| 조직 사용자 역할                           | 1,000           |
| 조직 기계 간 역할                          | 100             |
