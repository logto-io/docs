# Logto 프라이빗 클라우드 (Private Cloud)

더 강력한 격리, 컴플라이언스 준수, 인프라 제어를 위해 전용 Azure 환경에 Logto를 배포하세요.

## 개요 (Overview) \{#overview}

Logto 프라이빗 클라우드 (Private Cloud)는 공유 퍼블릭 클라우드 환경보다 더 높은 인프라 격리와 제어가 필요한 고객을 위한 전용 배포 옵션입니다.

이는 보안, 컴플라이언스, 조달, 또는 배포 요구 사항이 더 엄격한 조직을 위해 설계되었으며, Logto Cloud와 동일한 핵심 제품 경험을 제공합니다.

배포 옵션의 차이점은 다음과 같습니다:

- **Logto Cloud**: 관리형 퍼블릭 클라우드, 가장 빠른 프로덕션 진입 경로
- **Logto 프라이빗 클라우드 (Private Cloud)**: 더 엄격한 인프라 또는 컴플라이언스 요구 사항을 가진 고객을 위한 전용 배포
- **Logto OSS**: 직접 호스팅 및 직접 관리

## 프라이빗 클라우드를 선택해야 할 때 (When to choose Private Cloud) \{#when-to-choose-private-cloud}

다음 중 하나 이상의 요구 사항이 있는 경우 Logto 프라이빗 클라우드 (Private Cloud)가 적합합니다:

- 더 강력한 환경 격리
- Logto 팀의 지원 및 협업이 포함된 SLA 기반 가동 시간 보장
- 공유 퍼블릭 클라우드로는 충족할 수 없는 특정 컴플라이언스 요구 사항
- Logto Cloud에서 지원하지 않는 특정 지역 또는 인프라에 배포해야 하는 경우
- 퍼블릭 클라우드 환경에서 접근할 수 없는 내부 리소스와의 연결

이러한 요구 사항이 없다면, [Logto Cloud](https://cloud.logto.io/?sign_up=1)가 더 간단하게 도입할 수 있고 운영이 쉬우므로 일반적으로 더 나은 선택입니다.

### 배포 범위 (Deployment scope) \{#deployment-scope}

Logto 프라이빗 클라우드 (Private Cloud)는 현재 **Azure**에서만 제공됩니다.

지원되는 지역은 해당 Azure 지역이 Logto에 필요한 서비스를 제공하는지에 따라 달라집니다. 특정 지역을 염두에 두고 있다면, 저희에게 연락해 주시면 귀하의 요구 사항에 따라 가능 여부를 확인해 드릴 수 있습니다.

### 보안 및 컴플라이언스 (Security and compliance) \{#security-and-compliance}

프라이빗 클라우드 (Private Cloud)는 공유 퍼블릭 클라우드 배포보다 더 명확한 격리 경계를 가진 전용 환경을 제공합니다.

저희는 보안 및 컴플라이언스 논의를 실질적이고 요구 사항 중심으로 진행합니다. 보안 검토, 컴플라이언스 논의, 또는 배포별 설명이 필요하다면 언제든지 연락해 주시면 직접 자세히 안내해 드릴 수 있습니다.
