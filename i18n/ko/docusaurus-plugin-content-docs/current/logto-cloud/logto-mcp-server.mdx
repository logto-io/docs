# Logto MCP Server

Logto MCP Server는 원격 [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) 서버입니다. AI 애플리케이션에 URL을 추가하고, Logto Cloud 계정으로 로그인하여 Logto와 대화를 시작하세요.

:::tip
Logto MCP Server는 Logto와 [MCP Auth](https://mcp-auth.dev)로 구축되었습니다. 직접 원격 MCP 서버를 구축하고 싶다면 [직접 원격 MCP 서버 구축하기](#build-your-own-remote-mcp-server)를 참고하세요.
:::

## 할 수 있는 일 \{#what-you-can-do}

- **프로젝트에 인증 (Authentication) 추가하기**: AI가 프레임워크를 감지하고, Logto 애플리케이션을 생성하며, 동작하는 통합 코드를 생성합니다. React, Next.js, Vue, Nuxt, SvelteKit, Express, Go, Python, iOS, Android, Flutter 등 38개 이상의 프레임워크를 지원합니다.
- **Logto에 대해 질문하기**: AI가 Logto 문서를 검색하여 출처와 함께 답변을 제공합니다.

곧 로그인 경험 구성, 역할 기반 접근 제어 (RBAC) 및 권한 관리, 조직을 통한 B2B 앱 구축을 위한 다중 테넌시 설정 등 더 많은 기능이 추가될 예정입니다.

## AI 애플리케이션에서 설정하기 \{#set-up-in-your-ai-application}

### Claude Code \{#claude-code}

<Tabs>
  <TabItem value="claude-mcp-add" label="claude mcp add">

    ```bash
    claude mcp add --transport http --client-id mcp-client --callback-port 8080 logto https://mcp.logto.io
    ```

  </TabItem>
  <TabItem value="claude-mcp-add-json" label="claude mcp add-json">

    ```bash
    claude mcp add-json logto '{"type":"http","url":"https://mcp.logto.io","oauth":{"clientId":"mcp-client","callbackPort":8080}}'
    ```

  </TabItem>
</Tabs>

`8080`은 사용 가능한 다른 포트로 변경할 수 있습니다.

그런 다음, Claude Code에서 `/mcp`를 실행하고 브라우저 로그인 플로우를 따라 Logto Cloud 계정으로 인증 (Authentication)하세요.

### Claude Desktop \{#claude-desktop}

1. 메인 메뉴에서 **Settings**를 열고, **Connectors**로 이동한 후 **Add custom connector**를 클릭하세요.
2. 이름에 `Logto MCP Server`를 입력하고 원격 MCP 서버 URL: `https://mcp.logto.io`를 입력하세요.
3. **Advanced settings**를 확장하고 OAuth Client ID에 `mcp-client`를 입력한 뒤 **Add**를 클릭하세요.
4. **Connectors** 목록에서 **Logto MCP Server**를 찾아 **Connect**를 클릭하세요.
5. 먼저 Claude 웹 페이지로 리디렉션됩니다. 활성 세션이 없다면 먼저 Claude 계정에 로그인하세요.
6. Claude에 로그인한 후 Logto Cloud 로그인 페이지로 리디렉션됩니다. Logto Cloud 계정으로 로그인하고 연결을 승인하세요.

### Cursor \{#cursor}

1. 커맨드 팔레트(`Command + Shift + P`)를 열고 **View: Open MCP Settings**를 선택하세요.
2. **Installed MCP Servers** 섹션에서 **Add Custom MCP** 버튼을 클릭하세요.
3. 열리는 `mcp.json` 파일에 다음 구성을 추가하세요:

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud 로그인 페이지가 자동으로 나타납니다. Logto Cloud 계정으로 로그인하여 연결하세요.

### OpenCode \{#opencode}

1. `opencode.json` 파일에 다음 구성을 추가하세요:

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. 다음 명령어를 실행하여 인증 (Authentication)하세요:
   ```bash
   opencode mcp auth Logto
   ```
   여기서 `Logto`는 구성 파일의 MCP 서버 이름과 일치해야 합니다.

브라우저 창이 열립니다. Logto Cloud 계정으로 로그인하여 승인 및 연결하세요.

### VS Code \{#vs-code}

1. 커맨드 팔레트(`Command + Shift + P`)를 열고 **MCP: Add Server...**를 선택하세요.
2. 서버 유형으로 **HTTP**를 선택하세요.
3. URL에 `https://mcp.logto.io`를 입력하세요. 더 나은 격리 및 관리를 위해 워크스페이스에 설치할 수 있습니다.
4. OAuth 플로우가 시작되면 App ID에 `mcp-client`를 입력하세요. App Secret은 비워두고 Enter를 눌러 건너뜁니다.
5. Logto Cloud 로그인 페이지로 리디렉션됩니다. Logto Cloud 계정으로 로그인하여 연결하세요.

### Windsurf \{#windsurf}

1. 커맨드 팔레트(`Command + Shift + P`)를 열고 **Windsurf: MCP Marketplace**를 선택하세요.
2. **Installed MCPs** 섹션에서 오른쪽의 설정 아이콘을 클릭하여 `~/.codeium/windsurf/mcp_config.json` 파일을 생성 및 엽니다.
3. 다음 [구성](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json)을 추가하세요:

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud 로그인 페이지가 자동으로 나타납니다. Logto Cloud 계정으로 로그인하여 연결하세요.

### 기타 MCP 클라이언트 \{#other-mcp-clients}

Logto MCP Server는 OAuth 인가 (Authorization)를 지원하는 모든 MCP 클라이언트와 호환됩니다. 다음 구성을 사용하세요:

- **MCP 서버 URL:** `https://mcp.logto.io`
- **OAuth 클라이언트 ID:** `mcp-client`
- **리소스:** `https://mcp.logto.io/` (클라이언트에서 필요할 경우)
- **스코프:** `mcp:all` (클라이언트에서 필요할 경우)

AI 애플리케이션 개발자이고 Logto MCP Server와의 연결을 지원하고 싶지만, Logto MCP Server가 아직 애플리케이션의 리디렉션 URI를 지원하지 않는 경우, [문의하기](https://logto.io/contact)에서 애플리케이션 정보를 보내주세요.

## 직접 원격 MCP 서버 구축하기 \{#build-your-own-remote-mcp-server}

Logto MCP Server는 Logto와 [MCP Auth](https://mcp-auth.dev)로 구축되었습니다. 인증 (Authentication)이 포함된 직접 원격 MCP 서버를 구축하고 싶다면 다음 리소스를 참고하세요:

- [Remote MCP server in action: a new entry point for SaaS products in the AI era](https://blog.logto.io/remote-mcp-server-in-action)
- [Discord에 참여하기](https://discord.gg/fUQmaFyAam): 원격 MCP 서버 구축에 대한 경험과 모범 사례를 공유하세요
- [MCP Auth](https://mcp-auth.dev): MCP 서버에 프로덕션 수준의 인증 (Authentication)을 추가하는 데 필요한 모든 것. 사양을 읽거나 직접 연결하는 데 몇 주를 소비할 필요가 없습니다.
