# 결제 및 가격 정책

Logto에서는 [Pro 플랜](https://logto.io/pricing)이 완전히 셀프 서비스로 제공되며, 투명성을 위해 설계되어 결제 내역을 쉽게 이해하고 관리할 수 있습니다.

## 구조 이해하기 \{#understand-the-structure}

기본 구조는 다음과 같습니다:

```math
\begin{align*}
\text{다음 결제 금액} &= \$16 \text{(기본 가격)} \\
&+ \text{애드온 비용 (미청구 사용분)} \\
&+ \text{애드온 비용 (다음 사이클 청구)}
\end{align*}
```

콘솔의 테넌트 설정 섹션에서 "다음 결제 금액" 영역을 확인할 수 있으며, 여기서 다가오는 청구 내역을 모니터링하고 관리할 수 있습니다.

## 항목별 상세 내역 \{#item-breakdown}

애드온 구조에 포함되는 기능은 일부에 한정되어 있다는 점을 참고하세요.

동시에, Logto는 사용량 기반 결제 및 일할 계산 방식의 가격 정책도 제공하여, 서비스를 더욱 편리하게 활용할 수 있도록 합니다.

|                         | 포함된 할당량                 | 추가 비용            |
| ----------------------- | ----------------------------- | -------------------- |
| **토큰 (Tokens)**       | 100K                          | $0.08 / 100개당 월별 |
| **기계 간 (M2M) 앱**    | 1                             | $8 **_개당_** 월별   |
| **API 리소스**          | 3                             | $4 **_개당_** 월별   |
| **엔터프라이즈 SSO**    | 0                             | $48 **_개당_** 월별  |
| **다단계 인증 (MFA)**   | _해당 없음_                   | $48 / 월별           |
| **조직 (Organization)** | 무제한 조직 및 모든 조직 기능 | $48 / 월별           |
| **고급 보안 번들**      | 모든 고급 보안 기능           | $48 / 월별           |
| **테넌트 멤버**         | 3                             | $8 **_명당_** 월별   |

이러한 요소들을 고려하면, 알고리즘은 다음과 같습니다.

```math
\begin{align*}
\text{다음 결제 금액} &= \$16 \\
&+ \text{애드온 단가} \times \\
\quad & (\text{추가된 수량} \times \text{생성 시 남은 일할 시간} \\
\quad &- \text{삭제된 수량} \times \text{삭제 시 남은 일할 시간}) \\
\quad &+ \text{애드온 단가} \times \text{남은 수량}
\end{align*}
```

1. **애드온 단가**: 해당 애드온의 단가
2. **추가된 수량**: **_현재_** 결제 주기 내에 추가된 애드온 수량
3. **삭제된 수량**: **_현재_** 결제 주기 내에 삭제된 애드온 수량
4. **생성 시 남은 일할 시간**: 리소스를 생성할 때 **_현재_** 주기에 남은 일할 시간
5. **삭제 시 남은 일할 시간**: 리소스를 삭제할 때 **_현재_** 주기에 남은 일할 시간
6. **남은 수량**: **_다음_** 주기의 일관된 결제를 위해 필요한 수량

두 가지 예시를 통해 작동 방식을 살펴보겠습니다.

### 예시 1: 변경 없이 일관된 결제 \{#example-1-consistent-billing-with-no-changes}

항목을 추가하거나 삭제하지 않는다면 결제 금액은 일관적으로 유지됩니다.

예를 들어, Pro 플랜을 사용 중이며 SSO 서비스를 2개 사용하고 있다면, 변경이 없는 한 결제 금액은 변하지 않습니다.

```math
\text{다음 결제 금액} = \$ 16 + \$ 48 \times 2 = \$ 112
```

### 예시 2: 애드온 추가 또는 변경 시 결제 금액 변동 \{#example-2-adding-add-ons-or-making-changes-will-affect-your-upcoming-bills}

상황에 따라 다가오는 결제 금액이 달라질 수 있습니다. 알고리즘은 동일하게 적용됩니다.

**사례 1: Pro 플랜에 애드온 없이 가입 후, 엔터프라이즈 SSO만 10일간 테스트한 경우**

매월 5일에 구독이 갱신되고, 20일에 엔터프라이즈 SSO를 추가해 그 달에 15일간 사용했다고 가정합니다. 30일에 엔터프라이즈 SSO를 삭제하면 실제로 10일만 사용한 셈입니다.

```math
\$16 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$32
```

**사례 2: Pro 플랜에 애드온 없이 가입 후, API 리소스를 추가/삭제하여 최종적으로 5개를 사용한 경우**

이 경우, 결제 주기 중에 애드온 기능을 업데이트했으므로, 변경 사항이 다음 결제에 반영됩니다. 변경 후 첫 달은 약간 더 높을 수 있습니다. 결제 금액에는 $16 기본 가격, 미청구 사용분에 대한 애드온 비용, 다음 주기 전체 청구액이 포함됩니다.

결제 주기는 매월 1일 시작, 처음 3개 API 리소스는 무료입니다. 이번 달에는 두 가지 작업을 했습니다:

1. 5일에 리소스 4개 추가
2. 15일에 그 중 2개 삭제

결과적으로, 다음 주기에는 2개의 API 리소스가 일관적으로 남게 됩니다.

```math
\$16 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$50.67
```

### 애드온 사용 시간은 어떻게 산정되나요? \{#how-is-the-add-on-usage-time-determined}

Logto는 사용량 기반 결제 모델을 사용합니다. 리소스를 자주 추가/삭제할 경우 어떻게 되는지 궁금할 수 있습니다. 애드온은 실제로 활성화된 시간만큼만 사용한 것으로 간주됩니다. 걱정하지 마세요—Stripe로 구동되는 시스템이 초 단위까지 사용량을 계산하여, 각 리소스가 실제로 사용된 시간만큼 정확하게 청구합니다.

Pro 플랜 구독을 해지하면, 미사용 애드온 비용을 환불해드리며, 해당 금액은 최종 결제 금액에서 차감됩니다.

## 업그레이드 또는 다운그레이드 \{#upgrade-or-downgrade}

### 무료 플랜에서 Pro 플랜으로 업그레이드 \{#free-plan-upgrade-to-pro}

업그레이드 및 다운그레이드는 프로덕션 테넌트에만 적용됩니다. 무료 플랜에서 업그레이드를 고려해야 하는 상황은 다음과 같습니다:

1. MAU 또는 토큰 사용량이 할당량을 초과할 경우, 적절한 시점에 이메일 알림을 보내드리며, 업그레이드 필요성을 안내합니다.
2. 프리미엄 플랜에서만 제공되는 기능을 사용하고자 할 경우.

### Pro 플랜에서 엔터프라이즈로 업그레이드 \{#pro-plan-upgrade-to-enterprise}

Pro 플랜이 요구 사항을 충족하지 못하고, 엔터프라이즈급 지원이 필요하다면 [문의](https://logto.io/contact)하여 맞춤형 엔터프라이즈 플랜을 상담해 주세요.

### Pro 플랜에서 무료 플랜으로 다운그레이드 \{#pro-plan-downgrade-to-free-plan}

Pro 플랜 사용을 중단하고 무료 플랜으로 전환할 수 있습니다. 단, 무료 플랜의 할당량에 맞게 사용량을 조정해야 합니다.

## 향후 가격 정책 변경 \{#future-pricing-plan-change}

안정적이고 예측 가능한 결제 경험을 제공하기 위해, 가격 정책이 변경될 경우 현재 플랜을 그대로 유지(그랜드파더링)해 드립니다.

## 자주 묻는 질문 \{#faqs}

<details>

<summary>

### 토큰 소비와 MAU는 어떤 관계인가요? \{#what-is-the-connection-between-token-consumption-and-mau}

</summary>

Logto는 토큰 기반 인증 (Authentication)을 사용합니다. 한 명의 사용자가 한 번 로그인할 때 ID 토큰, 액세스 토큰, 리프레시 토큰 등 2~3개의 토큰이 소비될 수 있습니다. 기계 간 (M2M) 또는 조직 (Organization) 기능을 사용할 경우, M2M 토큰이나 조직 토큰 등 추가 토큰이 사용될 수 있습니다. 토큰 소비량은 사용자가 시스템과 상호작용하는 방식에 따라 달라집니다. 이를 추적할 수 있도록 <CloudLink to="/dashboard">콘솔 > 대시보드</CloudLink>에서 실시간 토큰 사용량 모니터링을 제공합니다. 예를 들어, 무료 플랜에서는 100,000개의 토큰이 무료로 제공되며, 약 30,000~50,000 MAU(월간 활성 사용자)를 지원할 수 있습니다.

</details>
