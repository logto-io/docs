# 결제 및 가격 정책

Logto에서는 [Pro 플랜](https://logto.io/pricing)이 완전한 셀프 서비스로 제공되며, 투명성을 위해 설계되어 있어 결제 내역을 쉽게 이해하고 관리할 수 있습니다.

## 구조 이해하기 \{#understand-the-structure}

기본 구조는 다음과 같습니다:

```math
\begin{align*}
\text{다음 결제 금액} &= \$24 \text{(기본 요금)} \\
&+ \text{애드온 비용 (미청구 사용분)} \\
&+ \text{애드온 비용 (다음 사이클 청구분)}
\end{align*}
```

콘솔의 테넌트 설정 섹션에서 "다음 결제 금액" 영역을 확인할 수 있으며, 다가오는 청구 내역을 모니터링하고 관리할 수 있습니다.

## 항목별 상세 내역 \{#item-breakdown}

애드온 구조에 포함되는 기능은 일부에 한정되어 있습니다.

동시에, Logto는 사용량 기반 결제(pay-as-you-go) 및 일할 계산(pro-rated) 가격 정책을 제공하여, 서비스를 편리하게 최대한 활용할 수 있도록 합니다.

|                               | 포함된 할당량                 | 추가 비용        |
| ----------------------------- | ----------------------------- | ---------------- |
| **토큰**                      | 50K                           | 100개당 월 $0.08 |
| **기계 간(M2M) 앱**           | 1                             | 개당 월 $8       |
| **API 리소스**                | 3                             | 개당 월 $4       |
| **엔터프라이즈 SSO**          | 0                             | 개당 월 $48      |
| **다단계 인증(MFA)**          | 모든 인증 요소                | 월 $48           |
| **조직**                      | 무제한 조직 및 모든 조직 기능 | 월 $48           |
| **역할 기반 접근 제어(RBAC)** | 무제한 역할 및 권한           | 월 $32           |
| **SAML 앱**                   | 0                             | 개당 월 $96      |
| **서드파티 앱**               | 0                             | 개당 월 $8       |
| **고급 보안 번들**            | 모든 고급 보안 기능           | 월 $48           |
| **테넌트 멤버**               | 1                             | 개당 월 $8       |

이러한 요소들을 고려하면, 알고리즘은 다음과 같습니다.

```math
\begin{align*}
\text{다음 결제 금액} &= \$24 \\
&+ \text{애드온 단가} \times \\
\quad & (\text{추가된 수량} \times \text{생성 시 남은 일할 시간} \\
\quad &- \text{삭제된 수량} \times \text{삭제 시 남은 일할 시간}) \\
\quad &+ \text{애드온 단가} \times \text{남은 수량}
\end{align*}
```

1. **애드온 단가**: 해당 애드온의 단가
2. **추가된 수량**: **_현재_** 결제 주기 내에 추가된 애드온 수량
3. **삭제된 수량**: **_현재_** 결제 주기 내에 삭제된 애드온 수량
4. **생성 시 남은 일할 시간**: 리소스를 생성할 때 **_현재_** 주기에 남은 일할 시간
5. **삭제 시 남은 일할 시간**: 리소스를 삭제할 때 **_현재_** 주기에 남은 일할 시간
6. **남은 수량**: **_다음_** 주기에서 일관된 결제를 위해 필요한 수량

두 가지 예시를 통해 작동 방식을 살펴보겠습니다.

### 예시 1: 변경 없이 일관된 결제 \{#example-1-consistent-billing-with-no-changes}

항목을 추가하거나 삭제하지 않는다면 결제 금액은 변하지 않습니다.

예를 들어, Pro 플랜을 사용 중이고 SSO 서비스를 2개 사용하고 있다면, 변경이 없는 한 결제 금액은 일정하게 유지됩니다.

```math
\text{다음 결제 금액} = \$ 24 + \$ 48 \times 2 = \$ 120
```

### 예시 2: 애드온 추가 또는 변경 시 결제 금액 변동 \{#example-2-adding-add-ons-or-making-changes-will-affect-your-upcoming-bills}

상황에 따라 다음 결제 금액이 달라질 수 있습니다. 알고리즘은 동일하게 적용됩니다.

**사례 1: Pro 플랜에 애드온 없이 가입 후, 엔터프라이즈 SSO를 10일간만 테스트한 경우**

매월 5일에 구독이 갱신되고, 20일에 엔터프라이즈 SSO를 추가하여 해당 달에 15일간 사용했다고 가정합니다. 30일에 SSO를 삭제하면 실제로 10일만 사용한 셈입니다.

```math
\$24 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$40
```

**사례 2: Pro 플랜에 애드온 없이 가입 후, API 리소스를 추가/삭제하여 최종적으로 5개를 사용한 경우**

이 경우, 결제 주기 중에 애드온 기능을 변경했으므로, 변경 내역이 다음 결제에 반영됩니다. 변경 후 첫 달은 약간 더 높을 수 있습니다. 결제 금액에는 $24 기본 요금, 미청구 사용분 애드온 비용, 다음 주기 전체 청구분이 포함됩니다.

결제 주기는 매월 1일 시작, 처음 3개 API 리소스는 무료입니다. 이번 달에는 다음 두 가지 작업을 했습니다:

1. 5일에 4개 리소스 추가
2. 15일에 그 중 2개 리소스 삭제

결과적으로, 다음 주기에는 2개 API 리소스가 일관되게 남게 됩니다.

```math
\$24 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$58.67
```

### 애드온 사용 시간은 어떻게 산정되나요? \{#how-is-the-add-on-usage-time-determined}

Logto는 사용량 기반(pay-as-you-go) 모델을 사용합니다. 리소스를 자주 추가/삭제할 경우 어떻게 되는지 궁금할 수 있습니다. 애드온은 실제 활성화된 시간만큼만 사용한 것으로 간주됩니다. Stripe 기반 시스템이 초 단위까지 사용량을 계산하여, 각 리소스가 실제로 사용된 시간만큼만 정확하게 청구합니다.

Pro 플랜 구독을 해지하면, 미사용 애드온 비용은 환불되며, 해당 금액이 최종 결제 금액에서 차감됩니다.

## 업그레이드 또는 다운그레이드 \{#upgrade-or-downgrade}

### 무료 플랜에서 Pro로 업그레이드 \{#free-plan-upgrade-to-pro}

다음과 같은 경우 무료 플랜에서 업그레이드를 고려해야 할 수 있습니다:

1. MAU 또는 토큰 사용량이 할당량을 초과한 경우
2. Pro 플랜에서만 제공되는 기능을 사용하고 싶은 경우

### 개발 테넌트에서 Pro로 업그레이드 \{#dev-tenant-upgrade-to-pro}

현재 개발 테넌트를 사용 중이고, 이를 프로덕션 Pro 플랜으로 전환하고 싶다면 쉽게 전환할 수 있습니다:

1. 작업 보존: 모든 설정, 애플리케이션, 환경설정이 Pro 플랜으로 그대로 이전됩니다.
2. 즉시 사용: 전환 즉시 모든 Pro 플랜 기능과 더 높은 할당량을 사용할 수 있습니다.

개발 테넌트를 Pro 플랜으로 전환하려면 <CloudLink to="/tenant-settings/settings">콘솔 > 테넌트 설정 > 설정</CloudLink>에서 "전환"을 클릭하세요.

### Pro 플랜에서 엔터프라이즈로 업그레이드 \{#pro-plan-upgrade-to-enterprise}

Pro 플랜이 요구 사항을 충족하지 못하고, 엔터프라이즈급 지원이 필요하다면 [문의](https://logto.io/contact)하여 맞춤형 엔터프라이즈 플랜을 상담해 주세요.

### Pro 플랜에서 무료 플랜으로 다운그레이드 \{#pro-plan-downgrade-to-free-plan}

Pro 플랜 사용을 중단하려면 무료 플랜으로 전환할 수 있습니다. 단, 무료 플랜 할당량에 맞게 사용량을 조정해야 합니다.

## 향후 가격 정책 변경 \{#future-pricing-plan-change}

안정적이고 예측 가능한 결제 경험을 위해, Logto는 가격 정책 변경 시 항상 사전에 안내합니다. 플랜 및 사용량에 따라 1. 기존 플랜을 유지(Grandfather)하거나, 2. 새로운 플랜으로 이전할 수 있는 유예 기간을 제공합니다.

Logto는 공정하고 신중하게 조정하며, 최종 해석 권한을 보유합니다.

## 엔터프라이즈 고객 가격 정책 참고사항 \{#enterprise-customer-pricing-considerations}

엔터프라이즈 고객의 경우, 가격은 계약에 따라 결정됩니다. 1. 계약 외 셀프 서비스 추가분은 Pro 플랜의 애드온 및 사용량 기반 구조를 따릅니다. 2. 계약 할당량 초과분은 계약서에 명시된 애드온 단가로 청구됩니다. 3. 계약에 포함되지 않은 신규 기능은 최신 Pro 플랜 가격 정책을 따릅니다.

엔터프라이즈 가격 정책의 자세한 조건은 [엔터프라이즈 영업팀에 문의](https://logto.io/contact)해 주세요.

<summary>

### 토큰 소비와 MAU의 관계는 무엇인가요? \{#what-is-the-connection-between-token-consumption-and-mau}

</summary>

Logto는 OAuth 2.0 및 OpenID Connect 표준을 따르는 **토큰 기반 인증 (Authentication)**을 채택하고 있습니다. 각 인증 (Authentication) 또는 인가 (Authorization) 요청 시, Logto는 사용자의 인증 (Authentication) 또는 인가 (Authorization) 상태를 나타내는 토큰(액세스 토큰, ID 토큰, 리프레시 토큰 등)을 발급합니다.

결제 기준으로는 **액세스 토큰만 집계**되며 (ID 토큰 및 리프레시 토큰은 제외됩니다).

**액세스 토큰**은 다음과 같은 상황에서 발급 및 집계됩니다:

- **사용자 로그인**: 사용자가 성공적으로 로그인하면 불투명 토큰(액세스 토큰)이 발급됩니다. 이 토큰은 UserInfo 엔드포인트 또는 Logto Account API를 호출하여 계정 관리를 할 수 있습니다.
- **백엔드 API 인가 (RBAC)**: 역할 기반 접근 제어 (RBAC)가 활성화되어 백엔드 API를 보호할 때, 사용자의 최초 인증 (Authentication) 이후 각 리소스별 인가 (Authorization) 요청마다 JWT 액세스 토큰이 발급됩니다. [RBAC](https://docs.logto.io/authorization/role-based-access-control) 및 API [보호](https://docs.logto.io/authorization/global-api-resources) 참고
- **조직 인가 (Authorization)**: 조직 기능이 활성화된 경우, Logto는 조직별 인가 (Authorization) 요청마다 조직 토큰(조직 액세스 토큰)을 발급하여 조직 단위 접근 제어를 제공합니다. 조직별 [인가 (Authorization)](https://docs.logto.io/authorization/organization-permissions) 및 API [보호](https://docs.logto.io/authorization/organization-level-api-resources) 참고
- **기계 간(M2M) 인가 (Authorization)**: 각 `client_credentials` 인가 (Authorization) 요청(예: Logto Management API 접근 시)마다 액세스 토큰이 발급됩니다.
- **토큰 교환**: 개인 액세스 토큰 교환 또는 사용자 가장 토큰 교환 시 새로운 액세스 토큰이 발급됩니다. [개인 액세스 토큰](https://docs.logto.io/user-management/personal-access-token) 및 [사용자 가장](https://docs.logto.io/developers/user-impersonation) 참고
- **리프레시 토큰 사용**: `offline_access`가 활성화된 경우, 유효한 리프레시 토큰을 교환할 때마다 새로운 액세스 토큰이 발급됩니다.

**토큰 vs. MAU 및 토큰 기반 결제의 장점**

MAU 기반 결제는 월별 고유 사용자 수에 따라 과금되며, 활동 여부와 무관합니다. 토큰 기반 결제는 실제 사용량, 즉 **액세스 토큰**이 발급될 때마다 과금됩니다.

토큰 기반 결제는 진정한 사용량 기반(pay-as-you-go) 가격 정책입니다. MAU 기반 모델은 사용 여부와 관계없이 한 달 전체를 과금하지만, 토큰 기반은 실제 인증 (Authentication) 및 인가 (Authorization) 활동에 대해서만 비용이 발생합니다. 이 방식은 **저빈도 사용자**가 간헐적으로 제품을 사용할 때 비용 부담이 적어, 고정 MAU 과금 대비 비용 효율적입니다. 즉, **Logto는 경쟁사 대비 더 경제적**이며, 다양한 사용자 참여 패턴을 가진 애플리케이션에 특히 유리합니다.

예를 들어, 한 번의 로그인은 액세스 토큰 1개를 소비할 수 있습니다. M2M 또는 조직 기능을 사용할 경우 추가 토큰이 생성될 수 있습니다. Logto는 <CloudLink to="/tenant-settings/subscription">콘솔 > 설정 > 플랜 및 결제</CloudLink>에서 **실시간 토큰 사용량 모니터링**을 제공하여, 프로젝트별 MAU와 토큰의 관계를 쉽게 파악할 수 있습니다.
