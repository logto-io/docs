# Logto Cloud 설정하기

이 섹션에서는 제품을 효과적으로 제공하고 개발 워크플로우를 관리하기 위한 기본 설정 단계와 주요 작업을 다룹니다.

## Logto 테넌트 생성하기 \{#create-logto-tenant}

먼저, 비즈니스에 가입하고 Logto 테넌트를 생성하세요. Logto Cloud의 테넌트는 사용자 아이덴티티, 애플리케이션, 그리고 모든 Logto 리소스를 관리할 수 있는 격리된 환경입니다.

첫 번째 테넌트는 자동으로 [**개발**](/logto-cloud/tenant-settings#development) 환경으로 설정됩니다. 이는 테스트 및 개발 목적에 이상적이며 무료로 사용할 수 있습니다.

테넌트 설정을 확인하고 Logto 클라우드 서비스에 대해 [더 알아보세요](/logto-cloud).

## 협업자 초대하기 \{#invite-collaborators}

온보딩 과정에서 협업자를 초대하여 함께 개발 작업을 할 수 있습니다. 더 많은 협업자를 초대하거나 나중에 추가하고 싶다면, <CloudLink to="/tenant-settings/members">테넌트 설정 > 멤버</CloudLink>로 이동하세요. 현재 멤버를 확인하고 추가로 초대할 수 있습니다. 이메일 주소를 입력하고 역할을 지정한 후(여러 명을 한 번에 지정할 수 있습니다) 초대장을 보내세요.

## 프로덕션 테넌트로 이동하기 \{#move-to-production-tenant}

Logto의 기능과 특징을 개발 및 테스트하여 개념 증명을 준비하는 데 시간을 보낼 수 있습니다.

프로젝트의 테스트가 완료되고 다음 단계로 진행할 준비가 되면, 새로운 [프로덕션 타입 테넌트](/logto-cloud/tenant-settings#production)를 생성하는 것을 잊지 마세요 ([Free 또는 Pro 테넌트](https://logto.io/pricing)가 필요합니다). 개발 테넌트는 [제한 사항](/logto-cloud/tenant-settings#development) 때문에 프로덕션 용도로 사용하기에 적합하지 않습니다.

:::note

프로덕션 테넌트를 생성하자마자 [커스텀 도메인](/logto-cloud/custom-domain)을 입력하세요. 이는 다음 단계의 설정에 영향을 미칩니다.

:::

이 설정을 통해 프로젝트는 개발 테스트 환경과 프로덕션 환경을 모두 가질 수 있어 개발 워크플로우를 원활하게 관리할 수 있습니다.

## 기존 시스템에서 마이그레이션하기 \{#migrate-from-the-existing-system}

기존 프로젝트에서 인증 제공자를 변경해야 하는 경우, Logto는 다른 플랫폼에서 사용자 마이그레이션을 지원합니다. [기본 데이터](/user-management/user-data#basic-data), [커스텀 데이터](/user-management/user-data#custom-data), [소셜 아이덴티티](/user-management/user-data#social-identities), 그리고 비밀번호 해시를 마이그레이션할 수 있습니다. 자세한 내용은 [Logto로 마이그레이션하기](../user-management/user-migration.mdx) 가이드를 참고하세요.

## AI 도구와 함께 Logto MCP Server 사용하기 \{#use-logto-mcp-server-with-ai-tools}

VS Code의 Copilot, Cursor, Claude Desktop 등 AI 기반 개발 도구를 사용하는 경우, [Logto MCP Server](/logto-cloud/logto-mcp-server)에 연결하여 AI 어시스턴트에서 직접 Logto와 상호작용할 수 있습니다. AI가 프레임워크를 감지하고, 애플리케이션을 생성하며, 동작하는 통합 코드를 자동으로 생성해줍니다.
