# Logto OSS 설정하기

이 섹션에서는 [Logto 오픈소스 서비스 (OSS)](https://github.com/logto-io/logto) 를 효과적으로 제공하고 개발 워크플로우를 관리하기 위한 기본 설정 단계와 주요 작업을 다룹니다.

## Logto OSS 시작하기 \{#get-started-with-logto-oss}

[시작 가이드](/logto-oss/get-started-with-oss/)를 따라 오늘 바로 Logto를 시작하세요. 이후 프로덕션 환경에서는 전체 [배포 가이드](/logto-oss/deployment-and-configuration)를 참고할 수 있습니다.

## Logto OSS에서 지원하는 기능 \{#feature-supported-by-logto-oss}

Logto OSS는 Logto 서비스의 대부분의 핵심 기능을 지원하며, 정기적으로 업데이트 및 유지 관리됩니다.

일부 고급 기능은 현재 Logto Cloud 버전에서만 제공됩니다. 예를 들면:

| OSS에서의 기능 제한                                                             | 설명                                                                                                                                                           |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto 콘솔: 다중 테넌트](/logto-cloud/tenant-settings)                         | 콘솔에서 여러 Logto 테넌트를 생성 및 관리할 수 있습니다.                                                                                                       |
| [Logto 콘솔: 협업자 초대](/logto-cloud/tenant-member-management)                | 하나의 Logto 테넌트에 여러 멤버를 초대하고 관리할 수 있습니다.                                                                                                 |
| Logto 콘솔: MFA 활성화                                                          | Logto 콘솔에 로그인할 때 다단계 인증 (MFA)을 요구하여 보안을 강화합니다.                                                                                       |
| [Logto Protected App](/integrate-logto/protected-app)                           | Cloudflare 기반의 빠르고 SDK가 필요 없는 인증 통합.                                                                                                            |
| [Logto 내장 이메일 서비스](/connectors/email-connectors/built-in-email-service) | 이메일 발송을 위한 내장 및 무료 이메일 서비스를 활용할 수 있습니다.                                                                                            |
| [UI 직접 구현](/customization/bring-your-ui)                                    | 이 기능을 사용하여 로그인 경험의 UI 또는 플로우를 커스터마이즈할 수 있습니다 (OSS는 GitHub에서 코드를 포크하여 직접 로그인 경험을 커스터마이즈할 수 있습니다). |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)           | 아이덴티티 제공자 (IdP) 주도의 엔터프라이즈 싱글 사인온 (SSO)을 활성화할 수 있습니다.                                                                          |
| [SAML 앱](/integrate-logto/saml-app)                                            | SAML 프로토콜을 통해 애플리케이션을 통합할 수 있습니다. OSS 버전은 SAML 앱 3개로 제한됩니다.                                                                   |

팁: 다중 테넌시, 멤버 초대, MFA는 오픈소스 Logto 콘솔에 팀이 로그인할 때 사용할 수 없습니다. 하지만 Logto OSS를 사용하여 여러분의 제품에 이러한 기능을 구현하면, 최종 사용자에게는 제공할 수 있습니다.

## Logto 릴리즈 최신 정보 받기 \{#stay-updated-with-logto-releases}

Logto 인스턴스를 최신 기능으로 유지하려면 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 페이지를 팔로우하세요. 모든 릴리즈 로그를 그곳에서 확인할 수 있습니다.

[업그레이드 가이드](/logto-oss/upgrading-oss-version)를 읽고, 코드나 데이터베이스 스키마를 변경하지 않고 Logto를 업그레이드하는 방법을 알아보세요.

## Logto OSS에 기여하기 \{#contributing-to-logto-oss}

Logto에 기여해 주셔서 감사합니다! [기여 가이드라인](/logto-oss/contribution)을 참고하세요.
