### Slack 앱 설정 \{#set-up-slack-app}

[Slack API: Applications](https://api.slack.com/apps)로 이동하여 Slack 계정으로 로그인하세요. 계정이 없다면, 등록할 수 있습니다.

그런 다음, 앱을 생성하세요.

**1단계:** `Client ID`와 `Client Secret` 찾기.

**"Basic Information"** 섹션에서 `Client ID`와 `Client Secret`을 찾을 수 있습니다.

**2단계:** 리디렉션 URL 설정.

**"OAuth & Permissions"** 섹션으로 이동하면 **"Redirect URLs"** 양식을 찾을 수 있습니다.

우리의 경우, 이는 `${your_logto_endpoint}/callback/${connector_id}`가 됩니다. 예를 들어, `https://foo.logto.app/callback/${connector_id}`입니다. `connector_id`는 Logto 관리자 콘솔 커넥터 세부 정보 페이지의 상단 바에서 찾을 수 있습니다.

자세한 내용은 [Slack API 문서](https://api.slack.com/authentication/sign-in-with-slack)를 참조하세요.

### 커넥터 구성 \{#configure-your-connector}

Logto 커넥터 구성에서, 앱의 "Keys and tokens" 페이지의 "OAuth 2.0 Client ID and Client Secret" 섹션에서 얻은 값으로 다음 필드를 채우세요:

- **clientId:** 앱의 Client ID.
- **clientSecret:** 앱의 Client Secret.

`scope`는 OpenID 스코프의 공백으로 구분된 목록입니다. 제공되지 않으면 기본 스코프는 `openid profile`입니다.

### 구성 유형 \{#config-types}

| 이름         | 유형   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
