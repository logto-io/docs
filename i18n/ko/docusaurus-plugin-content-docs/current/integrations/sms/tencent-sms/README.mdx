# Tencent Short Message Service로 SMS 인증을 설정하세요

Tencent Short Message Service에 대한 공식 Logto 커넥터입니다.

<GuideTip />

## 시작하기 \{#get-started}

Tencent Cloud는 아시아의 주요 클라우드 서비스 제공업체로, SMS (단문 메시지 서비스)를 포함한 다양한 서비스를 제공합니다.

Tencent Short Message Service 커넥터는 최종 사용자가 SMS 인증 코드를 사용하여 등록하거나 로그인할 수 있도록 돕는 공식 Logto 통합입니다.

## Tencent Cloud에서 단문 메시지 서비스를 설정하세요 \{#set-up-a-short-message-service-in-tencent-cloud}

### Tencent Cloud 계정 생성하기 \{#create-an-tencent-cloud-account}

[Tencent Cloud 웹사이트](https://cloud.tencent.com/)로 이동하여 계정이 없으면 등록하세요.

### Tencent Cloud SMS 활성화 및 구성하기 \{#enable-and-configure-tencent-cloud-sms}

1. [Tencent Cloud 웹사이트](https://cloud.tencent.com/)에 계정으로 로그인하고 [SMS 서비스 콘솔 페이지](https://cloud.tencent.com/product/sms)로 이동하세요.
2. SMS 서비스 페이지에서 "무료 체험" (免费试用) 버튼을 클릭하고 설정 단계를 따르세요.
3. "SMS 서비스 활성화 약관" (短信开通服务条款)에 동의하고 "서비스 활성화" (开通服务)를 클릭하여 진행하세요.
4. "[SMS 콘솔 개요](https://console.cloud.tencent.com/smsv2)" (短信服务控制台概览)에서 필요에 따라 "국내 메시지"(国内消息) 또는 "국제/홍콩, 마카오, 대만 메시지"(国际/港澳台消息)를 선택하세요.
5. SMS 메시지에 대한 서명과 템플릿을 추가하세요. 참고:
   - "서명" (签名)을 추가할 때, "적용 가능한 시나리오" (模版类型) 필드에서 "인증 코드" (验证码)를 선택하세요.
   - 템플릿을 추가할 때, 템플릿 유형으로 "인증 코드"를 선택하세요.
   - 템플릿 내용에 자리 표시자 `{1}`을 포함하세요. 이는 무작위로 생성된 코드로 대체됩니다.
   - 여러 자리 표시자가 있는 템플릿은 지원되지 않습니다. 단일 자리 표시자 템플릿을 사용하거나 생성하세요.
6. 서명과 템플릿을 승인받기 위해 제출하세요. 제출 후:
   - 적용될 때까지 기다리세요.
   - "[SMS 콘솔 개요](https://console.cloud.tencent.com/smsv2)" (短信服务控制台概览)에서 테스트 SMS를 보낼 수 있습니다. 승인된 서명과 템플릿은 직접 테스트에 사용할 수 있으며, 아직 검토 중인 경우 제공된 테스트 템플릿을 사용할 수 있습니다.
   - 테스트를 위해 계정에 충분한 잔액이 있는지 확인하고, SMS를 성공적으로 수신하기 위해 테스트 전화번호를 사전 등록하세요.

자세한 안내는 SMS 콘솔 개요의 "초보자 설정 안내" (新手配置指引) 탭을 클릭하세요.

## 커넥터 JSON 작성하기 \{#compose-the-connector-json}

1. "[SMS 콘솔 개요](https://console.cloud.tencent.com/smsv2)"로 이동하여 오른쪽 상단의 아바타 위에 마우스를 올리세요. "접근 관리 > 접근 키 > API 키"로 이동하여 보안 인증을 완료한 후 `AccessKey ID`와 `AccessKey Secret`을 생성하세요. 이를 잘 보관하세요.
2. "국내 메시지" (国内消息) 또는 "국제/홍콩, 마카오, 대만 메시지" (国际/港澳台消息) 탭에서 "서명 이름" (签名名称)과 "템플릿 코드" (模版 CODCE)를 가져오세요.
   - 테스트 템플릿의 경우, "테스트 전용 템플릿" (测试专用签名模版) 아래의 "빠른 시작" (快速开始) 탭을 확인하세요.

- 다음 필드로 Tencent Cloud SMS 커넥터를 구성하세요:
  - `accessKeyId`와 `accessKeySecret`: 1단계에서 얻은 자격 증명을 사용하세요.
  - `signName`: 2단계에서 얻은 서명 이름. 이는 모든 템플릿에 적용됩니다.
  - 필요에 따라 다양한 시나리오에 대한 여러 템플릿을 추가하세요. 단일 템플릿의 예:
    - `templateCode`: 2단계에서 얻은 "템플릿 코드".
    - `usageType`: Logto 전용 속성으로, `Register`, `SignIn`, `ForgotPassword`, `Generic` 중 하나를 지정하여 다양한 시나리오에 사용하세요.
    - 모든 네 가지 시나리오에 대해 템플릿이 구성되어 흐름을 완료할 수 있도록 하세요.
  - `sdkAppId`: "[애플리케이션 관리](https://console.cloud.tencent.com/smsv2/app-manage)" (应用管理)에서 찾을 수 있습니다.
  - `region`: "[전송 문서](https://cloud.tencent.com/document/api/382/52071#.E5.9C.B0.E5.9F.9F.E5.88.97.E8.A1.A8)" (发送文档)에서 지원되는 지역을 지정하세요.

### Tencent Cloud SMS 커넥터 테스트하기 \{#test-tencent-cloud-sms-connector}

전화번호를 입력하고 "보내기"를 클릭하여 설정이 작동하는지 "저장 및 완료" 전에 확인할 수 있습니다.

이제 완료되었습니다. [로그인 경험에서 커넥터 활성화](/connectors/sms-connectors#enable-phone-number-sign-up-or-sign-in)를 잊지 마세요.

### 구성 유형 \{#config-types}

| 이름            | 유형       |
| --------------- | ---------- |
| accessKeyId     | string     |
| accessKeySecret | string     |
| signName        | string     |
| region          | string     |
| sdkAppId        | string     |
| templates       | Template[] |

| 템플릿 속성  | 유형        | 열거형 값                                               |
| ------------ | ----------- | ------------------------------------------------------- |
| templateCode | string      | N/A                                                     |
| usageType    | enum string | 'Register' \| 'SignIn' \| 'ForgotPassword' \| 'Generic' |
