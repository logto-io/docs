# HTTP 호출로 이메일 인증을 설정하세요

HTTP 이메일을 위한 공식 Logto 커넥터입니다.

<GuideTip />

## 시작하기 \{#get-started}

HTTP 이메일 커넥터를 사용하면 HTTP 호출을 통해 이메일을 전송할 수 있습니다. HTTP 이메일 커넥터를 사용하려면 이메일 전송을 위한 HTTP API를 제공하는 자체 이메일 서비스가 필요합니다. Logto는 이메일을 전송해야 할 때 이 API를 호출합니다. 예를 들어, 사용자가 회원가입을 하면 Logto가 HTTP API를 호출하여 인증 이메일을 전송합니다.

## HTTP 이메일 커넥터 설정하기 \{#set-up-http-email-connector}

HTTP 이메일 커넥터를 사용하려면 Logto가 호출할 수 있는 HTTP 엔드포인트를 설정해야 합니다. 그리고 엔드포인트에 대한 선택적 인가 토큰을 설정할 수 있습니다.

## 페이로드 \{#payload}

HTTP 이메일 커넥터는 이메일을 트리거할 때 엔드포인트로 페이로드를 전송합니다. 이 페이로드에는 수신자의 이메일(`to`), 인증 (Authentication) 플로우를 나타내는 이메일 [usageType](/connectors/email-connectors/email-templates#email-template-types) (`type`), 사용 가능한 [변수](/connectors/email-connectors/email-templates#email-template-variables) (예: 인증 코드, 링크, 애플리케이션 정보, 조직 정보, 사용자 정보 등)을 포함하는 `payload`, 사용자의 [locale](/connectors/email-connectors/email-templates#email-template-localization), 그리고 선택적으로 클라이언트의 IP 주소가 포함된 `ip` 필드(가능한 경우)가 포함됩니다.

예시:

```json
{
  "to": "foo@logto.io",
  "type": "SignIn",
  "payload": {
    "code": "123456",
    "locale": "de"
  },
  "ip": "192.168.1.1"
}
```

`ip` 필드는 속도 제한, 사기 탐지 또는 향상된 로깅에 사용할 수 있습니다.

이 정보를 활용하여 이메일 발송 서비스에서 이메일 템플릿을 맞춤화할 수 있습니다.

사용 가능한 모든 타입은 [이메일 템플릿](/connectors/email-connectors/email-templates#email-template-types)에서 확인할 수 있습니다. 전체 `SendMessageData` 타입 정의는 [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts)에 있습니다.
