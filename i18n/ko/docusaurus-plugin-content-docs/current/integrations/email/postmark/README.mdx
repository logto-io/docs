# Postmark 이메일 서비스를 사용하여 이메일 인증 설정하기

Logto 커넥터는 Postmark 이메일 서비스를 지원합니다.

<GuideTip />

## 시작하기 \{#get-started}

Postmark는 트랜잭션 및 마케팅 이메일을 위한 메일 플랫폼입니다. 우리는 이 플랫폼의 이메일 발송 기능을 사용하여 *인증 코드*를 보낼 수 있습니다.

## Postmark 계정 등록하기 \{#register-postmark-account}

[Postmark 웹사이트](https://postmark.com/)에서 새 계정을 만드세요. 이미 계정이 있다면 이 단계를 건너뛸 수 있습니다.

## 커넥터 구성하기 \{#configure-your-connector}

Postmark의 서버 설정에서 찾을 수 있는 Server Token으로 `serverToken` 필드를 채우세요.

발신자의 *From Address*로 `fromEmail` 필드를 채우세요.

전체 사용자 흐름을 활성화하려면 `Register`, `SignIn`, `ForgotPassword` 및 `Generic` 사용 유형의 템플릿이 필요합니다.

다음은 Postmark 커넥터 템플릿 JSON의 예입니다.

```jsonc
[
  {
    "usageType": "Register",
    "templateAlias": "logto-register",
  },
  {
    "usageType": "SignIn",
    "templateAlias": "logto-sign-in",
  },
  {
    "usageType": "ForgotPassword",
    "templateAlias": "logto-forgot-password",
  },
  {
    "usageType": "Generic",
    "templateAlias": "logto-generic",
  },
]
```

## Postmark 이메일 커넥터 테스트하기 \{#test-postmark-email-connector}

이메일 주소를 입력하고 "Send"를 클릭하여 설정이 작동하는지 "Save and Done" 전에 확인할 수 있습니다.

이제 끝났습니다. <MainSiteUrl href="/connectors/email-connectors/#enable-email-address-for-end-user-flows">로그인 경험에서 커넥터 활성화하기</MainSiteUrl>를 잊지 마세요.

## 구성 유형 \{#config-types}

| 이름        | 유형   |
| ----------- | ------ |
| serverToken | string |
| fromEmail   | string |

| 템플릿 속성   | 유형        | 열거형 값                                               |
| ------------- | ----------- | ------------------------------------------------------- |
| usageType     | enum string | 'Register' \| 'SignIn' \| 'ForgotPassword' \| 'Generic' |
| templateAlias | string      | N/A                                                     |
