## Aliyun DirectMail 콘솔에서 이메일 서비스 설정하기 \{#set-up-an-email-service-in-aliyun-directmail-console}

### Aliyun 계정 생성하기 \{#create-an-aliyun-account}

[Aliyun](https://aliyun.com/)으로 이동하여 Aliyun 계정이 없다면 계정을 생성하세요.

### Aliyun Direct Mail 활성화 및 구성하기 \{#enable-and-configure-aliyun-direct-mail}

[DM 서비스 콘솔 페이지](https://www.aliyun.com/product/directmail)로 이동하여 로그인하세요. 페이지 왼쪽 상단의 "신청하여 활성화" (申请开通) 버튼을 클릭하여 Direct Mail 서비스를 활성화하고 구성 과정을 시작하세요.

[DM 관리자 콘솔 페이지](https://dm.console.aliyun.com/)에서 시작하여 다음을 수행해야 합니다:

1. 사이드바에서 "발신 도메인" (发信域名)으로 이동하여 지침에 따라 "새 도메인" (新建域名)을 추가하세요.
2. 각각 "발신 주소" (发信地址)와 "이메일 태그" (邮件标签)를 사용자 정의하세요.

설정을 완료한 후, 테스트할 수 있는 두 가지 방법이 있습니다:

- [DirectMail 개요 페이지](https://dm.console.aliyun.com/)로 이동하여 페이지 하단의 "작업 가이드" (操作引导)를 찾아 "이메일 보내기" (发送邮件)를 클릭하세요. 다양한 테스트 방법을 찾을 수 있습니다.
- 사이드바에서 "이메일 보내기" (发送邮件) -> "이메일 작업" (发送邮件) 경로를 따라가 테스트 작업을 생성하세요.

## 커넥터 JSON 작성하기 \{#compose-the-connector-json}

1. [DM 관리자 콘솔 페이지](https://dm.console.aliyun.com/)에서 오른쪽 상단의 아바타 위로 마우스를 올리고 "AccessKey 관리" (AccessKey 管理)로 이동하여 "AccessKey 생성" (创建 AccessKey)을 클릭하세요. 보안 인증을 완료한 후 "AccessKey ID"와 "AccessKey Secret" 쌍을 받게 됩니다. 이를 잘 보관하세요.
2. [DM 관리자 콘솔 페이지](https://dm.console.aliyun.com/)에서 방금 방문한 "발신 주소" (发信地址) 또는 "이메일 태그" (邮件标签) 탭으로 이동하여 _발신 주소_ 또는 *이메일 태그*를 쉽게 찾을 수 있습니다.
3. Aliyun DM 커넥터 설정을 작성하세요:
   - 1단계에서 얻은 액세스 키 쌍으로 `accessKeyId`와 `accessKeySecret` 필드를 채우세요.
   - 2단계에서 찾은 "발신 주소"와 "이메일 태그"로 `accountName`과 `fromAlias` 필드를 채우세요. 모든 템플릿은 이 서명 이름을 공유합니다. (`fromAlias`는 선택 사항이므로 비워둘 수 있습니다.)
   - 다양한 경우에 대해 여러 DM 커넥터 템플릿을 추가할 수 있습니다. 단일 템플릿을 추가하는 예시는 다음과 같습니다:
     - 발송 이메일의 제목으로 사용할 `subject` 필드를 채우세요.
     - 임의의 문자열 유형 콘텐츠로 `content` 필드를 채우세요. 무작위 인증 코드를 위한 `{{code}}` 자리 표시자를 잊지 마세요.
     - `usageType` 필드를 `Register`, `SignIn`, `ForgotPassword` 또는 `Generic` 중 하나로 채워 다양한 사용 사례에 맞추세요. (`usageType`은 적절한 사용 사례를 식별하기 위한 Logto 속성입니다.) 전체 사용자 흐름을 활성화하려면 `Register`, `SignIn`, `ForgotPassword` 및 `Generic` 사용 유형의 템플릿이 필요합니다.

### Aliyun DM 커넥터 테스트하기 \{#test-aliyun-dm-connector}

이메일 주소를 입력하고 "보내기"를 클릭하여 설정이 작동하는지 확인한 후 "저장 및 완료"를 클릭하세요.

이제 끝났습니다. [로그인 경험에서 커넥터 활성화](/connectors/email-connectors/#enable-email-sign-up-or-sign-in)를 잊지 마세요.

### 구성 유형 \{#config-types}

| 이름            | 유형               |
| --------------- | ------------------ |
| accessKeyId     | string             |
| accessKeySecret | string             |
| accountName     | string             |
| fromAlias       | string (선택 사항) |
| templates       | Template[]         |

| 템플릿 속성 | 유형        | 열거형 값                                               |
| ----------- | ----------- | ------------------------------------------------------- |
| subject     | string      | N/A                                                     |
| content     | string      | N/A                                                     |
| usageType   | enum string | 'Register' \| 'SignIn' \| 'ForgotPassword' \| 'Generic' |
