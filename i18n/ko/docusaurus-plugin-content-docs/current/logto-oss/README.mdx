# Logto OSS

Logto는 오픈 소스 (OSS)와 [클라우드](/logto-cloud) 버전을 모두 제공합니다. 이 섹션에서는 OSS 솔루션에 중점을 둡니다: <Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS는 개발자에게 완전한 커스터마이징 기능과 데이터에 대한 완전한 제어권을 제공하는 무료 아이덴티티 솔루션입니다. Logto OSS는 코드베이스를 특정 요구에 맞게 유연하게 조정할 수 있어, 데이터 주권을 중시하고 애플리케이션과 함께 성장할 수 있는 맞춤형 아이덴티티 제공자 솔루션을 원하는 분들에게 완벽한 선택입니다.

## Logto OSS에서 지원하는 기능 \{#feature-supported-by-logto-oss}

Logto OSS는 Logto 서비스의 대부분의 핵심 기능을 지원하며, 정기적으로 업데이트 및 유지 관리됩니다.

일부 고급 기능은 현재 Logto Cloud 버전에서만 제공됩니다. 예를 들면:

| OSS의 기능 제한                                                                 | 설명                                                                                                                                                           |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto 콘솔: 다중 테넌트](/logto-cloud/tenant-settings)                         | 콘솔에서 여러 Logto 테넌트를 생성 및 관리할 수 있습니다.                                                                                                       |
| [Logto 콘솔: 협업자 초대](/logto-cloud/tenant-member-management)                | 하나의 Logto 테넌트에 여러 멤버를 초대 및 관리할 수 있습니다.                                                                                                  |
| Logto 콘솔: MFA 활성화                                                          | Logto 콘솔 로그인 시 다단계 인증 (MFA)을 요구하여 보안을 강화할 수 있습니다.                                                                                   |
| [Logto Protected App](/integrate-logto/protected-app)                           | Cloudflare 기반의 빠르고 SDK 없는 인증 통합.                                                                                                                   |
| [Logto 내장 이메일 서비스](/connectors/email-connectors/built-in-email-service) | 내장된 무료 이메일 서비스를 활용하여 이메일을 발송할 수 있습니다.                                                                                              |
| [UI 직접 제공](/customization/bring-your-ui)                                    | 이 기능을 사용하여 로그인 경험의 UI 또는 플로우를 커스터마이즈할 수 있습니다 (OSS는 GitHub에서 코드를 포크하여 직접 로그인 경험을 커스터마이즈할 수 있습니다). |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)           | 아이덴티티 제공자 (IdP) 주도의 엔터프라이즈 싱글 사인온 (SSO)을 활성화할 수 있습니다.                                                                          |
| [SAML 앱](/integrate-logto/saml-app)                                            | SAML 프로토콜을 통해 애플리케이션을 통합할 수 있습니다. OSS 버전은 최대 3개의 SAML 앱으로 제한됩니다.                                                          |

팁: 다중 테넌시, 멤버 초대, MFA는 오픈 소스 Logto 콘솔에 팀이 로그인할 때 사용할 수 없습니다. 하지만 Logto OSS를 사용하여 여러분의 제품에 이러한 기능을 직접 구현하고, 최종 사용자에게 제공할 수 있습니다.
