# Logto 설치하기

## 인터랙티브 설치 \{#interactive}

터미널에서 아래 명령어를 실행하여 인터랙티브 설치를 진행하세요:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npm init @logto@latest
```

  </TabItem>

</Tabs>

## 옵션 및 무인 설치 \{#options-and-silent-installation}

또는 아래의 옵션들(모두 선택 사항)을 추가하여 일부 질문을 건너뛸 수 있습니다:

| Option                                         | 설명                                                                                                                                                                                                                                                                          |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--db-url`                                     | Logto 데이터베이스에 사용할 Postgres URL입니다.                                                                                                                                                                                                                               |
| `-p, --path`                                   | Logto 인스턴스의 경로로, 존재하지 않는 경로여야 합니다.                                                                                                                                                                                                                       |
| `--ss`                                         | Logto 데이터베이스 시딩을 건너뜁니다.                                                                                                                                                                                                                                         |
| `--disable-admin-pwned-password-check, --dapc` | 관리자 테넌트의 Have I Been Pwned (HIBP) 비밀번호 유출 검사 기능을 비활성화하여 시드합니다. 에어갭 또는 오프라인 배포 환경에 적합합니다. 자세한 내용은 [에어갭 또는 오프라인 배포를 위한 시드](/logto-oss/using-cli#seed-for-air-gapped-or-offline-deployments)를 참고하세요. |

전체 도움말은 `logto init --help` 명령어를 실행하세요.

다음은 무인 설치(예: CI 환경) 예시입니다:

<Tabs groupId="cmd">

  <TabItem value="cli" label="CLI">

```bash
logto init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>
  <TabItem value="npm" label="npm">

```bash
npx @logto/cli init -p ./logto --db-url postgresql://your-postgres-dsn:5432/logto --oc
```

  </TabItem>

</Tabs>

아래 작업이 순서대로 진행됩니다:

1. Logto를 `./logto` 경로에 다운로드 및 압축 해제
2. URL `postgresql://your-postgres-dsn:5432/logto`로 데이터베이스를 초기화 및 시딩 시도
3. Logto 공식 커넥터를 `./logto/packages/core/connectors`에 추가
