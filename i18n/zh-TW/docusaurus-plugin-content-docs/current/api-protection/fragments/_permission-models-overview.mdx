## 權限 (Permission) 模型總覽 \{#permission-models-overview}

在實作保護機制前，請先選擇最適合你應用程式架構的權限模型。這與 Logto 的三大[授權 (Authorization) 情境](/authorization#authorization-scenarios)相符：

<Tabs groupId="permission-models">
  <TabItem value="global-api-resources" label="全域 API 資源 (Global API resources)">

    <img src={globalApiResources} alt="全域 API 資源 RBAC" style={{ maxWidth: '100%' }} />

    - **適用情境：** 保護整個應用程式共用的 API 資源（非組織專屬）
    - **權杖類型：** 具有全域受眾 (global audience) 的存取權杖 (Access token)
    - **範例：** 公開 API、核心產品服務、管理端點
    - **最適用於：** 所有客戶共用 API 的 SaaS 產品、無租戶隔離的微服務架構
    - **深入瞭解：** [保護全域 API 資源](/authorization/global-api-resources)

  </TabItem>
  <TabItem value="organization-permissions" label="組織（非 API）權限 (Organization (non-API) permissions)">

    <img
      src={organizationPermissions}
      alt="組織權限 RBAC"
      style={{ maxWidth: '800px', width: '100%' }}
    />

    - **適用情境：** 控制組織專屬的操作、UI 功能或商業邏輯（非 API）
    - **權杖類型：** 具有組織專屬受眾 (organization-specific audience) 的組織權杖 (Organization token)
    - **範例：** 功能開關、儀表板權限、成員邀請控制
    - **最適用於：** 具有組織專屬功能與流程的多租戶 SaaS
    - **深入瞭解：** [保護組織（非 API）權限](/authorization/organization-permissions)

  </TabItem>
  <TabItem value="organization-level-api-resources" label="組織層級 API 資源 (Organization-level API resources)">

    <img
      src={organizationLevelApiResources}
      alt="組織層級 API 資源 RBAC"
      style={{ width: '100%' }}
    />

    - **適用情境：** 保護僅在特定組織情境下可存取的 API 資源
    - **權杖類型：** 具有 API 資源受眾 (API resource audience) + 組織情境 (organization context) 的組織權杖 (Organization token)
    - **範例：** 多租戶 API、組織範圍資料端點、租戶專屬微服務
    - **最適用於：** API 資料以組織為範圍的多租戶 SaaS
    - **深入瞭解：** [保護組織層級 API 資源](/authorization/organization-level-api-resources)

  </TabItem>
</Tabs>

<br />

**💡 請在繼續前選擇你的模型** —— 本指南後續內容將以你選擇的方式為參考。
