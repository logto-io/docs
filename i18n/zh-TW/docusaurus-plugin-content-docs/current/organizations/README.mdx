# 組織 (Organizations)

組織功能在多租戶 SaaS 和 [企業對企業 (B2B) 應用程式](/introduction/plan-your-architecture/b2b) 中特別有效。除了個別使用者外，你的客戶還可以由團隊、組織或整個公司組成。

透過這個基本元素，你可以為 [多租戶應用程式](https://auth.wiki/multi-tenancy) 建立必備功能，例如：

- 一個可供多個組織使用的產品。
- 基於 [邀請](/end-user-flows/organization-experience/invite-organization-members) 或 [即時](/organizations/just-in-time-provisioning) 的組織成員佈建。
- 由組織內成員分配的角色定義的 [存取控制](/authorization/organization-template)。
- 將組織與 [企業級單一登入 (Enterprise SSO)](/end-user-flows/enterprise-sso) 體驗連結。

「組織」一詞也以其他形式使用，如「工作空間」、「團隊」、「公司」等。在 Logto 中，我們使用「組織」作為表示多租戶概念的通用術語。

## 規劃考量 \{#planning-considerations}

使用組織時，整合可能涉及不同程度的努力，我們建議你交叉參考這些以自訂你的開發。

- **組織體驗**：組織體驗會是什麼樣子？組織管理員可以創建自己的組織嗎？組織管理員可以邀請成員嗎？你是否打算建立自訂儀表板，允許管理員自行管理其組織成員的身分相關資訊？
- **角色與存取控制**：應用程式是否需要使用者在其組織內分配特定角色？
- **透過 Logto Console 和 Management API 管理組織**：你希望直接在 Logto Console 和 Management API 中設置哪些配置？
