# 瞭解組織的運作方式

為了全面瞭解，了解組織功能中涉及的實體非常重要，這些是將組織整合到應用程式中的基礎。

## 組織 (Organization) \{#organization}

組織由一群[使用者（身分）](/user-management/user-data)組成。它可以代表可以存取你的應用程式的團隊、商業客戶和合作夥伴公司。

引入組織作為實體非常重要，因為它不僅可以將使用者分組，還提供了多租戶應用程式中租戶隔離的上下文。

## 組織成員 (Organization member) \{#organization-member}

在 Logto 中，擁有組織成員資格的使用者在該組織的上下文中被稱為組織成員（即成員）。

## 組織 M2M 應用程式 \{#organization-m2m-application}

組織 M2M 應用程式指的是與組織相關聯的[機器對機器應用程式](/quick-starts/m2m)。類似於[組織成員](#organization-member)，組織應用程式在組織的上下文中具有範圍。

一旦將應用程式與組織關聯，該應用程式可以被分配[組織角色](/authorization/organization-template/configure-organization-template#create-organization-role)。這在你希望允許非互動存取組織資源時非常有用。

:::note

其他類型的應用程式不支援組織關聯和組織角色，因為它們是為使用者互動而設計的。

:::

參考此[部分](/organizations/organization-management)來管理組織、成員和應用程式。

## 組織範本 (Organization template) \{#organization-template}

組織範本是為多租戶應用程式中的存取控制需求而設計的。它包括組織角色和組織權限。查看此[組織範本](/authorization/organization-template)以瞭解更多關於組織中存取控制設計的資訊。
