# 瞭解組織 (Organization) 的運作方式

為了全面理解，認識組織功能所涉及的實體至關重要，這些也是將組織整合進你的應用程式的基礎。

## 組織 (Organization) \{#organization}

組織 (Organization) 由一群 [使用者（身分，identities）](/user-management/user-data) 組成。它可以代表團隊、企業客戶或合作夥伴公司，這些對象都能存取你的應用程式。

將組織作為一個實體引入非常重要，因為它不僅能群組使用者，還為多租戶應用程式提供租戶隔離的情境。

## 組織成員 (Organization member) \{#organization-member}

在 Logto 中，擁有某組織成員資格的使用者，在該組織情境下被稱為組織成員（即 member）。

## 組織 M2M 應用程式 (Organization M2M application) \{#organization-m2m-application}

組織 M2M 應用程式指的是與組織關聯的 [機器對機器應用程式 (M2M applications)](/quick-starts/m2m)。與 [組織成員 (organization members)](#organization-member) 類似，組織應用程式的作用範圍限定於特定組織情境內。

一旦你將應用程式與組織關聯，即可為該應用程式指派 [組織角色 (organization roles)](/authorization/role-based-access-control#configure-organization-roles)。這在你希望允許非互動式存取組織資源時非常有用。

:::note

其他類型的應用程式不支援組織關聯與組織角色，因為它們設計用於使用者互動。

:::

請參考此 [章節](/organizations/organization-management) 來管理組織、成員與應用程式。

## 組織範本 (Organization template) \{#organization-template}

組織範本 (Organization template) 是為多租戶應用程式的存取控制需求而設計。它包含組織角色與組織權限。請參閱這份 [組織範本](/authorization/organization-template) 以深入瞭解組織中的存取控制設計。
