# 審計日誌

Logto 的審計日誌讓你輕鬆監控使用者活動和事件。它為各種使用者管理和健康檢查業務場景提供了堅實的基礎。

## 查看所有日誌 \{#view-all-logs}

導航至 <CloudLink to="/audit-logs">Console > Audit logs</CloudLink>。Logto 將驗證 (Authentication) 事件捕捉並組織成表格，記錄事件名稱、使用者、應用程式和時間戳。你可以根據事件名稱和應用程式名稱進行篩選以縮小結果範圍。點擊特定事件將提供更多詳細資訊。

:::warning
審計日誌僅包含使用者驗證 (Authentication) 過程中發生的日誌，Management API 操作的日誌不會被記錄。
:::

## 在租戶層級捕捉使用者活動 \{#capture-user-activity-at-the-tenant-level}

Logto 的日誌提供全面的細節，確保行動便利和客戶安全。它們捕捉並記錄以下資訊：

- 事件類型（完整的審計日誌事件列表可在[此處](https://github.com/logto-io/logto/blob/master/packages/console/src/consts/logs.ts)找到）
- 涉及的應用程式
- IP 位址
- 涉及的使用者
- 日誌 ID
- 時間戳
- 使用者代理

透過維護這些事件記錄，組織可以有效地檢測可能的安全風險，並及時處理以防止未經授權的系統存取。

## 在使用者層級進行詳細分析 \{#perform-a-detailed-analysis-at-the-user-level}

管理員可以對特定使用者相關的日誌進行詳細分析，便於對特定事件進行全面調查。導航過程簡單且使用者友好。

要存取使用者特定的日誌，請按照以下步驟操作：

1. 導航至 <CloudLink to="/users">Console > User management</CloudLink>。
2. 選擇所需的使用者並進入詳細頁面。
3. 點擊「User logs」。結果表格將僅顯示該特定使用者執行和觸發的日誌事件。
