# 驗證器應用程式 OTP

## 概念 \{#concepts}

驗證器應用程式（Authenticator app），又稱為軟體權杖（Software Token），是最廣泛採用的 [多重要素驗證 (MFA, Multi-factor authentication)](https://auth.wiki/mfa) 方法之一。它會產生臨時的 [一次性密碼 (OTP, One-time password)](https://auth.wiki/otp)，以提升線上服務驗證 (Authentication) 的安全性。與實體硬體權杖不同，軟體權杖通常是使用者安裝於裝置（如智慧型手機或電腦瀏覽器）上的應用程式或外掛。軟體權杖可根據驗證器的功能與個人設定，在單一裝置本地運作，或於多裝置間同步。

常見的軟體權杖範例包括 Google Authenticator、Microsoft Authenticator、Duo、1Password、Authy 等。

## 驗證流程 \{#authentication-flows}

### 驗證器應用程式 OTP 設定流程 \{#authentication-app-otp-setup-flows}

1. **QR Code 或密鑰**：使用者從你的服務取得 QR Code 或密鑰。
2. **新增帳號**：使用驗證器應用程式，掃描 QR Code 或手動輸入密鑰以新增帳號。
3. **動態一次性密碼**：驗證器應用程式會顯示一組六位數密碼，並於每 1-2 分鐘為該帳號自動更新。
4. **完成 MFA 設定**：使用者在 MFA 設定頁於有效時間內輸入此密碼，即完成驗證器應用程式 OTP 的 MFA 設定。

### 驗證器應用程式 OTP 驗證流程 \{#authentication-app-otp-verification-flows}

1. **登入嘗試**：登入時，系統會提示使用者進行 MFA。
2. **取得 OTP**：開啟驗證器應用程式，取得對應帳號的 OTP。
3. **輸入 OTP**：使用者於 2 步驟驗證頁，在有效時間內輸入應用程式顯示的 OTP。
4. **驗證 (Authentication)**：系統驗證 OTP，驗證成功後授予存取權限。
