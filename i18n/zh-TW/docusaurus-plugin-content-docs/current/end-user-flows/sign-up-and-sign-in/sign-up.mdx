# 電子郵件 / 電話號碼 / 使用者名稱註冊

使用者註冊是使用者與你的應用程式互動的第一步。Logto 支援多種註冊方法，包括使用者名稱密碼、電子郵件或電話號碼驗證、[社交註冊](/end-user-flows/sign-up-and-sign-in/social-sign-in) 和 [企業級單一登入 (SSO)](/end-user-flows/enterprise-sso)。你可以設定最符合應用程式需求的註冊方法。

訪問 <CloudLink to="/sign-in-experience/sign-up-and-sign-in">控制台 > 登入體驗 > 註冊和登入</CloudLink> 開始配置識別碼註冊流程。

<img src="/img/assets/sign-in-experience-sign-up.png" alt="註冊設定" />

## 設定註冊識別碼 \{#set-up-the-sign-up-identifier}

要在 Logto 中成功創建新使用者帳戶，使用者必須提供一個在 Logto 系統中唯一識別他們的 **識別碼**。第一步是設定在註冊過程中需要從使用者收集的 **識別碼**。可用的選項有：

- **使用者名稱**：使用者可用來登入應用程式的唯一 [使用者名稱](/user-management/user-data#username)。
- **電子郵件地址**：使用者可用來登入應用程式的有效 [電子郵件地址](/user-management/user-data#primary_email)。
- **電話號碼**：使用者可用來登入應用程式的有效 [電話號碼](/user-management/user-data#primary_phone)。
- **電子郵件地址或電話號碼**：允許使用者使用有效的電子郵件地址或電話號碼註冊。
- **不適用**：未啟用註冊識別碼。這適用於僅 [社交](/end-user-flows/sign-up-and-sign-in/social-sign-in) 或僅 [企業級 SSO](/end-user-flows/enterprise-sso) 的註冊方法。

在註冊過程中收集的所有識別碼必須在同一租戶下的使用者中唯一。它們將存儲在 [使用者的個人資料](/user-management/user-data#user-profile) 中，並可用於登入與 Logto 整合的應用程式。

## 設定註冊驗證設定 \{#set-up-the-sign-up-verification-settings}

為確保使用者註冊和未來登入過程的安全性，你還需要配置在註冊過程中收集的識別碼的驗證設定。可用的設定有：

- **創建你的密碼**：要求使用者在註冊時創建符合登入體驗設定中密碼政策的密碼。此密碼與使用者的識別碼一起作為他們登入應用程式的憑證。如果你將 **使用者名稱** 設為註冊識別碼，則此要求會自動啟用，因為 **使用者名稱** 只能與密碼一起使用以有效驗證使用者的身分。可以自訂 [密碼政策](/end-user-flows/sign-up-and-sign-in/reset-password#set-up-password-policy) 以滿足你的安全需求。
- **註冊時驗證**：要求使用者在註冊時驗證他們的電子郵件地址或電話號碼。目前，Logto 只接受已驗證的電子郵件和電話號碼作為識別碼。當使用 **電子郵件地址** 或 **電話號碼** 作為註冊識別碼時，此設定會自動啟用。使用者必須在註冊過程中輸入發送到其電子郵件或電話號碼的驗證碼以確認擁有權。

| 識別碼             | 創建使用者密碼 | 註冊時驗證 |
| ------------------ | -------------- | ---------- |
| 使用者名稱         | 必需           | 不適用     |
| 電子郵件地址       | 可選           | 必需       |
| 電話號碼           | 可選           | 必需       |
| 電子郵件或電話號碼 | 可選           | 必需       |
| 不適用             | 不適用         | 不適用     |

## 使用社交或企業級 SSO 註冊 \{#type-4-email-address-with-verification-username-and-password-creation}

除了這些傳統的識別碼註冊方法，Logto 還支援使用社交和企業級 SSO 身分提供者的無密碼註冊，使上手過程更加順暢和使用者友好。

一旦在 Logto 中配置並啟用了 [社交連接器](/connectors/social-connectors) 或 [企業級 SSO 連接器](/connectors/enterprise-connectors)，使用者可以輕鬆使用連接器提供的現有社交或企業身分註冊。社交和企業級 SSO 註冊方法允許使用者跳過創建密碼或驗證電子郵件地址或電話號碼等額外步驟。Logto 將自動通過他們已驗證的社交或企業身分同步使用者資訊並將其存儲在使用者的個人資料中。

查看 [社交登入](/end-user-flows/sign-up-and-sign-in/social-sign-in/) 和 [企業級 SSO](/end-user-flows/enterprise-sso/) 部分以了解更多關於社交和企業級 SSO 連接器的註冊流程。

:::note
注意：如需自訂註冊流程，請參考 [自帶 UI](/customization/bring-your-ui/) 功能。
:::

## 常見問題 \{#sign-up-with-social-or-enterprise-sso}

<details>
  <summary>

### 在註冊時收集額外的使用者資訊 \{#faqs}

</summary>
  
要在註冊時收集額外的使用者個人資料資訊（例如，全名、生日、公司名稱），你有兩個選擇：

- **自託管上手流程**：在成功註冊後將使用者重定向到你自己的上手流程以進行可自訂的資料收集。
- **自帶 UI**：使用我們的自帶 UI 功能與使用者個人資料設定 API（即將推出）自訂你的註冊流程。

</details>

<details>
  <summary>

### 管理員創建的使用者 / 被邀請的使用者 \{#collect-additional-user-information-on-sign-up}

</summary>

了解如何實現 [僅限邀請註冊流程](/end-user-flows/sign-up-and-sign-in/disable-user-registration/#implement-an-invitation-only-sign-up-flow)。

</details>

<details>
  <summary>

### 嵌入式註冊表單在你的網站上 \{#admin-created-users--invited-users}

</summary>
  
Logto 目前不支援無頭 API 用於登入和註冊。你可以使用 [自帶 UI](/customization/bring-your-ui/) 功能將你自己的註冊表單上傳到 Logto，或使用登入參數將使用者資訊從你的网站填入 Logto。了解更多關於使用者識別碼填入的資訊，請參閱 [驗證參數](/end-user-flows/authentication-parameters/)。

</details>

<details>
  <summary>

### 向新使用者發送歡迎電子郵件 \{#embedded-sign-up-forms-on-your-website}

</summary>

訂閱 `User.Created` Webhook 事件以觸發向新使用者發送歡迎電子郵件。了解更多關於 [Webhook 事件](/developers/webhooks/webhooks-events/#data-mutation-hook-events)。

</details>

<details>
  <summary>

### 跳過註冊時的電子郵件驗證 \{#sending-welcome-emails-to-new-users}

</summary>

目前，Logto 只支援已驗證的電子郵件和電話號碼作為識別碼。驗證過程是確保使用者識別碼的安全性和擁有權所必需的。
對未驗證的電子郵件或電話號碼的支援在我們的 [路線圖](https://logto.productlane.com/roadmap) 上。請關注更新！

</details>
