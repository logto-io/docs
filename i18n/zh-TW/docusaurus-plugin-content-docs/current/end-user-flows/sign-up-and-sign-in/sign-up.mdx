# 電子郵件 / 電話 / 使用者名稱註冊

使用者註冊是使用者參與你的應用程式的第一步。Logto 支援多種註冊方式，包括使用者名稱密碼、電子郵件或電話號碼驗證、[社交註冊](/end-user-flows/sign-up-and-sign-in/social-sign-in) 以及 [企業級單一登入 (Enterprise SSO)](/end-user-flows/enterprise-sso)。你可以根據應用程式需求設定最合適的註冊方式。

前往 <CloudLink to="/sign-in-experience/sign-up-and-sign-in">控制台 > 登入體驗 > 註冊與登入</CloudLink> 開始設定識別碼註冊流程。

<img src="/img/assets/sign-in-experience-sign-up.png" alt="註冊設定" />

## 設定註冊識別碼 \{#set-up-the-sign-up-identifier}

為了在 Logto 成功建立新使用者帳號，使用者必須提供至少一個能在 Logto 系統中唯一識別他們的 **識別碼**。第一步，請選擇使用者在註冊時必須提供的識別碼。可用選項如下：

- **使用者名稱 (Username)**：使用者可用於登入應用程式的唯一 [使用者名稱](/user-management/user-data#username)。
- **電子郵件地址 (Email address)**：使用者可用於登入應用程式的有效 [電子郵件地址](/user-management/user-data#primary_email)。
- **電話號碼 (Phone number)**：使用者可用於登入應用程式的有效 [電話號碼](/user-management/user-data#primary_phone)。
- **電子郵件地址或電話號碼 (Email address or phone number)**：允許使用者以有效電子郵件地址或電話號碼註冊。

所有在註冊過程中收集的識別碼，在同一租戶下必須是唯一的。這些識別碼會儲存在 [使用者個人檔案](/user-management/user-data#user-profile) 中，並可用於登入與 Logto 整合的應用程式。

若未選擇任何識別碼，則適用於僅限 [社交](/end-user-flows/sign-up-and-sign-in/social-sign-in) 或 [企業級單一登入 (Enterprise SSO)](/end-user-flows/enterprise-sso) 的註冊方式。

你可以調整註冊識別碼的順序，以決定使用者註冊時優先填寫哪一項。此順序會反映在註冊流程中，第一個識別碼會顯示在初始註冊畫面，其餘則於後續步驟收集。

## 設定註冊驗證設定 \{#set-up-the-sign-up-verification-settings}

為確保使用者註冊及未來登入流程的安全性，你還需為註冊時收集的識別碼設定驗證機制。可用設定如下：

- **建立密碼 (Create your password)：** 要求使用者在註冊時建立一組符合登入體驗設定中密碼政策的密碼。此密碼將與使用者識別碼一同作為登入應用程式的憑證。若你將 **使用者名稱 (Username)** 設為註冊識別碼，則此需求會自動啟用，因為 **使用者名稱** 只能搭配密碼有效驗證使用者身分。[密碼政策](/security/password-policy) 可自訂以符合你的安全需求。
- **註冊時驗證 (Verify at sign-up)：** 要求使用者在註冊時驗證其電子郵件地址或電話號碼。目前 Logto 僅接受已驗證的電子郵件與電話號碼作為識別碼。當 **電子郵件地址 (Email address)** 或 **電話號碼 (Phone number)** 作為註冊識別碼時，此設定會自動啟用。使用者必須在註冊過程中輸入發送至其電子郵件或電話號碼的驗證碼以確認擁有權。

| 識別碼                                     | 建立使用者密碼 | 註冊時驗證 |
| ------------------------------------------ | -------------- | ---------- |
| 使用者名稱 (Username)                      | 選用           | 不適用     |
| 電子郵件地址 (Email address)               | 選用           | 必要       |
| 電話號碼 (Phone number)                    | 選用           | 必要       |
| 電子郵件或電話號碼 (Email or phone number) | 選用           | 必要       |

## 使用社交或企業級單一登入 (SSO) 註冊 \{#sign-up-with-social-or-enterprise-sso}

除了上述傳統識別碼註冊方式外，Logto 也支援透過社交與企業級單一登入 (SSO) 身分提供者進行無密碼註冊，讓導入流程更順暢、友善。

一旦在 Logto 設定並啟用 [社交連接器](/connectors/social-connectors) 或 [企業級單一登入 (Enterprise SSO) 連接器](/connectors/enterprise-connectors)，使用者即可直接使用連接器提供的現有社交或企業身分註冊。社交與企業級單一登入 (SSO) 註冊方式可讓使用者省略建立密碼或驗證電子郵件 / 電話號碼等額外步驟。Logto 會自動透過已驗證的社交或企業身分同步使用者資訊並儲存於使用者個人檔案中。

請參閱 [社交登入](/end-user-flows/sign-up-and-sign-in/social-sign-in/) 與 [企業級單一登入 (Enterprise SSO)](/end-user-flows/enterprise-sso/) 章節，瞭解更多社交與企業級單一登入 (SSO) 連接器的註冊流程。

:::note
注意：如需自訂註冊流程，請參考 [自帶 UI (Bring your UI)](/customization/bring-your-ui/) 功能。
:::

## 常見問題 \{#faqs}

<details>
  <summary>

### 註冊時收集額外使用者資訊 \{#collect-additional-user-information-on-sign-up}

</summary>

若需在註冊時收集額外使用者個人資料（如姓名、生日、公司名稱等），你有兩種選擇：

- **自架引導流程 (Self-hosted onboarding flows)**：註冊成功後將使用者導向你自訂的引導流程，自由收集所需資料。
- **自帶 UI (Bring your own UI)**：利用我們的 [自帶 UI](/customization/bring-your-ui) 功能，搭配使用者個人檔案設定 API，自訂註冊流程。

</details>

<details>
  <summary>

### 管理員建立 / 邀請使用者 \{#admin-created-users--invited-users}

</summary>

瞭解如何實作 [僅限邀請註冊流程](/end-user-flows/sign-up-and-sign-in/disable-user-registration/#implement-an-invitation-only-sign-up-flow)。

</details>

<details>
  <summary>

### 網站內嵌註冊表單 \{#embedded-sign-up-forms-on-your-website}

</summary>
  
Logto 目前尚未支援無頭 API 進行登入與註冊。你可以使用 [自帶 UI (Bring your UI)](/customization/bring-your-ui/) 功能將自訂註冊表單上傳至 Logto，或利用登入參數將網站上的使用者資訊帶入 Logto。詳情請參閱 [驗證 (Authentication) 參數](/end-user-flows/authentication-parameters/)。

</details>

<details>
  <summary>

### 向新使用者發送歡迎郵件 \{#sending-welcome-emails-to-new-users}

</summary>

訂閱 `User.Created` webhook 事件，即可向新使用者發送歡迎郵件。詳情請參閱 [Webhook 事件](/developers/webhooks/webhooks-events/#data-mutation-hook-events)。

</details>

<details>
  <summary>

### 註冊時跳過電子郵件驗證 \{#skip-email-verification-on-sign-up}

</summary>

目前 Logto 僅支援已驗證的電子郵件與電話號碼作為識別碼。驗證流程是為了確保使用者識別碼的安全性與擁有權。
支援未驗證電子郵件或電話號碼已在我們的 [產品藍圖](https://logto.productlane.com/roadmap) 上，敬請期待後續更新！

</details>
