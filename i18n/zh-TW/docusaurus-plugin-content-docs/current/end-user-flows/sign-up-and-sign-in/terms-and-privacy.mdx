# 條款與隱私

為確保你的產品保持開放與透明，請在登入與註冊頁面新增「使用條款」與「隱私政策」的連結。這能協助你符合特定產業的合規要求。

## 設定條款與隱私 \{#configure-terms--privacy}

1. 前往 <CloudLink to="/sign-in-experience/content">主控台 > 登入與帳號 > 內容</CloudLink>。
2. 新增「使用條款」與「隱私政策」的外部連結。
3. 根據特定地區或產業的合規需求，設定「同意條款」的政策。

:::note
若值為空，則無需在 Logto 的登入流程中管理條款與隱私協議。你可以在使用者登入應用程式後再處理這些事項。
:::

## 同意條款的使用者體驗 \{#user-experience-for-agreeing-to-terms}

Logto 提供多種預設流程，協助你根據不同合規需求處理同意條款的使用者體驗：

### 選項 1：在繼續登入／註冊時自動同意條款 \{#option-1-agree-to-terms-automatically-on-continue-sign-insign-up}

此政策提供最順暢的使用者體驗，當使用者繼續登入或註冊流程時，系統會自動同意條款。
使用者不會被明確要求同意條款。

<img src="/img/assets/auto-agree-to-terms.png" alt="自動同意條款" />

### 選項 2：僅在註冊時同意條款 \{#opetion-2-agree-to-terms-on-sign-up-only}

此政策要求新使用者手動同意**條款與隱私政策**。根據歐盟 [GDPR](https://gdpr-info.eu/art-4-gdpr/) 與加州 [CCPA](https://oag.ca.gov/privacy/ccpa) 的規定，企業必須在蒐集個人資訊前取得使用者的知情同意。

當使用者首次以社交註冊（如 Google 或 GitHub）時，重新導向至 Logto 後會跳出視窗，提示同意**條款與隱私政策**。後續社交登入則不再需要此同意。

<img src="/img/assets/sign-up-only-agree-to-terms.png" alt="僅在註冊時同意條款" />

### 選項 3：每次登入與註冊時同意條款 \{#option-3-agree-to-terms-on-sign-in-and-sign-up}

此政策要求使用者每次登入或註冊時都需同意**條款與隱私政策**。適用於要求每次登入都需同意條款的應用程式。使用者必須勾選同意條款後才能繼續。此做法符合中國 PIPL 與巴西 LGPD 的規範。

<img src="/img/assets/sign-in-agree-to-terms.png" alt="每次登入與註冊時同意條款" />
