# 將 Logto 整合進你的應用程式

依照下列步驟，無論是面向使用者的應用程式還是機器對機器服務，都能輕鬆為你的應用程式新增驗證 (Authentication) 功能：

1. 前往 <CloudLink to="/applications">控制台 > 應用程式</CloudLink>

2. 點擊「建立應用程式」以新增應用程式

3. 選擇你的 [應用程式框架](/quick-starts) 開始。如果找不到你的框架，請點擊應用程式建立頁面右下角的「不選擇框架建立應用程式」按鈕，透過選擇 [應用程式類型](/integrate-logto/application-data-structure/#application-types) 建立應用程式，或依照我們的 [SDK 規範](/developers/sdk-conventions) 提交功能需求或貢獻 SDK。

4. 選擇框架後，會看到該框架 SDK 的快速入門指南。請依照步驟設定並整合你的應用程式。如果你需要更深入理解整合過程中的相關概念，可參考 [理解 Logto 驗證流程](/integrate-logto/integrate-logto-into-your-application/understand-authentication-flow/)。

:::note
控制台中的指南僅適用於使用我們 SDK 快速開始 Logto。完整整合指南（包含進階 SDK 用法）請參閱 [快速入門](/quick-starts) 區段。

快速入門指南主要示範如何實作登入。如果你需要直接導向註冊、忘記密碼，或特定驗證方式（如電子郵件註冊或社交登入），請參閱 [驗證參數](/end-user-flows/authentication-parameters) 文件。
:::

5. 完成後，你可以進一步探索 Logto 更多功能：

   <Url href="/end-user-flows">終端使用者流程</Url>
   <Url href="/authorization">授權 (Authorization)</Url>
   <Url href="/organizations">組織 (Organizations)</Url>

<details>
  <summary>
    ### 我的後端服務如何驗證使用者權杖並管理使用者？
  </summary>
若要在你的後端 API（如 Python、Node.js、Go、Java、PHP 等）安全驗證存取權杖 (Access tokens)，並以程式方式管理使用者，請參考指南：[如何在 API 服務或後端驗證存取權杖](/authorization/validate-access-tokens)。

本文件涵蓋：

- 如何在每次 API 呼叫時檢查 bearer 權杖的有效性
- Logto 與多個前端應用程式及後端服務整合的最佳實踐

</details>
