# 自訂使用者授權頁面品牌 (Custom consent screen branding)

當使用者被導向第三方應用程式的使用者授權頁面 (Consent screen) 時，確保第三方的品牌資訊與條款連結正確顯示給使用者非常重要。

Logto 允許你自訂第三方應用程式的品牌資訊，包括應用程式名稱、Logo 及條款連結。

## 自訂品牌資訊 \{#customize-the-branding-information}

請妥善設定第三方應用程式的品牌資訊，確保為使用者帶來一致且安全的驗證 (Authentication) 體驗。

1. 前往 <CloudLink to="/applications/third-party-applications">控制台 > 應用程式 > 第三方應用程式</CloudLink>，並開啟特定 OIDC 第三方應用程式的詳細頁面。

2. 切換至 **品牌設定 (Branding)** 分頁。

3. 設定使用者授權頁面 (Consent screen) 的顯示資訊：

- **顯示名稱 (Display name)**：將顯示於使用者授權頁面的第三方應用程式名稱，代表該應用程式正在請求存取你使用者資訊。若此欄位留空，將使用 **應用程式名稱 (Application name)**。
- **應用程式 Logo（淺色）(App logo (Light))**：將顯示於使用者授權頁面的第三方應用程式 Logo，代表該應用程式的品牌。若同時提供第三方應用程式 Logo 與你的通用登入體驗 Logo，兩者都會顯示於授權頁面。
- **應用程式 Logo（深色）(App logo (Dark))**：僅於啟用深色模式登入體驗時可用。可於 <CloudLink to="/sign-in-experience/branding">控制台 > 登入體驗 > 品牌設定</CloudLink> 頁面管理深色模式設定。
- **使用條款網址 (Terms of use URL)**：將顯示於使用者授權頁面的第三方應用程式條款連結。
- **隱私權政策網址 (Privacy policy URL)**：將顯示於使用者授權頁面的第三方應用程式隱私權政策連結。
