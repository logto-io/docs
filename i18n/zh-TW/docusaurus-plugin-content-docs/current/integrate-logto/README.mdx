# 整合 Logto 驗證 (Authentication)

Logto 為網頁、行動裝置與桌面應用程式提供完整的驗證 (Authentication) 解決方案，支援服務間 [機器對機器 (M2M)](/quick-starts/m2m) 驗證，並可透過 [OpenID Connect (OIDC)](https://auth.wiki/openid-connect) 和 [OAuth 2.0](https://auth.wiki/oauth-2.0) 等標準協議，作為 [第三方應用程式](/integrate-logto/third-party-applications) 的身分提供者 (IdP)。

請根據你的需求選擇最合適的整合方案開始：

## 為你的應用程式新增驗證 (Authentication) \{#add-authentication-for-your-applications}

無論你正在打造面向使用者的應用程式（如網頁、行動或桌面應用程式），還是用於服務間通訊的機器對機器 (M2M) 應用程式，都能透過整合 Logto 快速實現完整的 [驗證 (Authentication)](/end-user-flows) 與 [使用者管理](/user-management) 功能。

## 新增 OIDC/OAuth 應用程式 \{#add-oidc-oauth-apps}

當你希望讓第三方應用程式能以你的 Logto 身分系統進行驗證時，可以將 Logto 設定為 [身分提供者 (IdP)](https://auth.wiki/identity-provider)。

Logto 可透過 OAuth、OIDC 等標準協議作為 IdP，外部服務可像「使用 Google 登入」一樣實作「使用 Logto 登入」，讓他們的使用者能以你的 Logto 使用者系統進行驗證。

在 Logto 中建立第三方應用程式，即可在你的身分系統與外部服務間建立安全連結。

## 透過 Management API 擴充 Logto 功能 \{#extend-logto-features-with-management-api}

Logto [Management API](https://openapi.logto.io/) 讓你能以程式方式控制並自動化 Logto 在應用程式中的功能。

當你需要擴充 Logto 能力以強化你的產品（例如 [管理多租戶組織](/organizations) 或自動化 Logto 工作流程），可以利用 Logto Management API 建構進階功能，滿足你的產品需求。
