# 設定 SAML Assertion 屬性

SAML 屬性是 SAML Assertion 的關鍵組成部分，承載著已驗證使用者的特定資訊。這些屬性可以包含使用者識別碼、角色 (Roles)、權限 (Permissions) 以及服務提供者 (SP, Service Provider) 進行授權 (Authorization) 與個人化所需的其他相關使用者資料。

## 理解 SAML 屬性 \{#understanding-saml-attributes}

在 SAML 驗證 (Authentication) 中：

- 屬性是包含使用者資訊的名稱 / 值配對
- 它們會包含在從身分提供者 (Logto) 傳送給服務提供者的 SAML Assertion 中
- 它們協助服務提供者根據使用者資訊做出存取與個人化決策

## Logto 中的屬性對應 \{#attribute-mapping-in-logto}

屬性對應 (Attribute mapping) 讓你定義 Logto 的使用者資訊如何對應到 SAML Assertion 中的特定屬性。這樣可確保你的服務提供者收到格式正確且屬性名稱符合預期的使用者資訊。

當你設定屬性對應時：

1. 你可以指定哪些 Logto 使用者屬性要包含在 SAML Assertion 中
2. 你可以自訂服務提供者所需的屬性名稱
3. 在驗證 (Authentication) 過程中，對應的屬性會自動包含在 SAML Assertion 內

### 常見屬性 \{#common-attributes}

常用的 SAML 屬性包括：

- Sub（使用者 ID）
- Email
- 組織 (Organizations)
- Name
- Preferred username

[檢視 Logto 可用的所有使用者資料](/user-management/user-data)

妥善設定屬性對應後，可確保服務提供者收到所有必要的使用者資訊，為你的使用者提供適當的存取權限與個人化體驗。

你可以透過屬性對應設定，將所有 Logto 可用的使用者資訊對應到 SP 預期的屬性上。
