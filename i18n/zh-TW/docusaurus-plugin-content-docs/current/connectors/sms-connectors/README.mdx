# SMS 連接器 (SMS connectors)

設定 SMS 連接器可以將[一次性密碼 (OTPs, one-time passwords)](https://auth.wiki/otp) 發送到使用者的手機號碼。這種無密碼驗證 (Authentication) 機制可應用於多種情境，包括[註冊](/end-user-flows/sign-up-and-sign-in/sign-up)、[登入](/end-user-flows/sign-up-and-sign-in/sign-in)、[忘記密碼](/end-user-flows/sign-up-and-sign-in/reset-password)、[帳號連結流程](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking)、[成員邀請](/end-user-flows/organization-experience/invite-organization-members)以及[驗證使用者身分](/end-user-flows/security-verification)。這能簡化使用者驗證 (Authentication) 流程，並透過降低密碼相關風險來提升安全性。

## 設定步驟 \{#configuration-steps}

請依下列步驟新增 SMS 連接器：

1. 前往 <CloudLink to="/connectors/passwordless">Console &gt; Connector &gt; Email and SMS connectors</CloudLink>。
2. 點擊「Set up」按鈕並選擇你想要的 SMS 供應商以新增連接器。
3. 閱讀你選擇供應商的 README 文件。
4. 在「Parameter Configuration」區段填寫必要欄位。
5. 使用 JSON 編輯器自訂 SMS 範本。
6. 透過「Generic」範本發送驗證碼到你的手機，測試設定是否正確。
7. 點擊「Save and Done」完成設定。

## 更換 SMS 連接器 \{#change-sms-connector}

若要更換其他 SMS 連接器：

- 進入連接器詳細頁，點擊右上角的「Change connector」
- 或刪除現有連接器後再新增新的連接器

注意：你一次只能啟用「一個」_SMS 連接器_。

當你已經有一個 SMS 連接器時，只能選擇「更換 SMS 連接器」。在 Console 執行此操作時，儲存後會立即生效，請務必先測試設定再儲存。

為確保服務穩定，Cloud 用戶可建立[開發租戶 (dev tenant)](/logto-cloud/tenant-settings#development) 作為開發環境。測試無誤後，將設定「遷移」到[正式環境 (production)](/logto-cloud/tenant-settings#production) 並儲存。

## 啟用終端用戶流程的手機號碼 \{#enable-phone-number-for-end-user-flows}

新增並設定 SMS 連接器後，你可以啟用基於手機號碼的驗證 (Authentication) 流程：

- [**設定註冊方式**](/end-user-flows/sign-up-and-sign-in/sign-up)：前往 <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Sign-in & account &gt; Sign-up and sign-in</CloudLink>。在註冊區段，新增「手機號碼」或「電子郵件地址或手機號碼」作為識別子。注意：註冊時必須驗證手機號碼。
- [**設定登入方式**](/end-user-flows/sign-up-and-sign-in/sign-in)：在登入區段，新增「手機號碼」作為登入方式。可選擇密碼或 SMS 驗證碼進行驗證 (Authentication)。點擊「Add another」可新增其他選項。
- [**設定忘記密碼**](/end-user-flows/sign-up-and-sign-in/reset-password)：在登入區段，若啟用密碼登入，將看到「Forgot password」設定。啟用「Phone verification code」即可讓使用者透過 SMS 重設密碼。
- [**設定 SMS 作為 MFA**](/end-user-flows/mfa/sms-mfa)：前往 <CloudLink to="/mfa">Console &gt; Multi-factor auth</CloudLink>。啟用「SMS verification code」作為第二驗證因素，提升安全性。
- [**設定帳號管理**](/end-user-flows/account-settings/by-account-api#manage-phone)：使用 [Account API](/end-user-flows/account-settings/by-account-api) 自訂你的帳號中心，讓使用者管理手機號碼。

:::warning
由於我們的設定會立即生效，刪除現有 SMS 連接器可能會影響終端使用者的正常登入與註冊流程。

例如，若登入識別子包含「手機號碼 + 驗證碼」組合，刪除現有 SMS 連接器後，使用者將無法透過電子郵件收到驗證碼，登入流程也會出現錯誤。

請操作連接器時務必注意潛在風險。
:::
