# SMS 連接器 (SMS connectors)

設定 SMS 連接器可讓你將 [一次性密碼 (OTP, One-Time Passwords)](https://auth.wiki/otp) 發送到使用者的手機號碼。這種無密碼驗證 (Authentication) 機制可應用於多種情境，包括 [註冊](/end-user-flows/sign-up-and-sign-in/sign-up)、[登入](/end-user-flows/sign-up-and-sign-in/sign-in)、[忘記密碼](/end-user-flows/sign-up-and-sign-in/reset-password)、[帳號連結流程](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking)、[成員邀請](/end-user-flows/organization-experience/invite-organization-members) 以及 [驗證使用者身分](/end-user-flows/security-verification)。這能簡化使用者驗證流程並提升安全性，降低密碼洩漏的風險。

## 設定步驟 \{#configuration-steps}

請依下列步驟新增 SMS 連接器：

1. 前往 <CloudLink to="/connectors/passwordless">控制台 > 連接器 > Email 與 SMS 連接器</CloudLink>。
2. 點擊「設定」按鈕並選擇你想要的 SMS 供應商以新增連接器。
3. 閱讀你選擇供應商的 README 文件。
4. 在「參數設定」區域填寫必要欄位。
5. 使用 JSON 編輯器自訂 SMS 範本。
6. 透過「通用」範本發送驗證碼到你的手機，測試設定是否正確。
7. 點擊「儲存並完成」結束設定。

## 更換 SMS 連接器 \{#change-sms-connector}

若要更換其他 SMS 連接器：

- 進入連接器詳情頁，點擊右上角「更換連接器」
- 或刪除現有連接器後新增新的連接器

注意：你一次只能啟用「一個」_SMS 連接器_。

當你已有 SMS 連接器時，只能選擇「更換 SMS 連接器」。在控制台執行此操作時，儲存後會立即生效，請務必先測試設定再儲存。

為確保服務穩定，Cloud 用戶可建立 [開發租戶 (dev tenant)](/logto-cloud/tenant-settings#development) 作為開發環境。測試無誤後，將設定「遷移」到 [正式環境 (production)](/logto-cloud/tenant-settings#production) 並儲存。

## 啟用手機號碼於終端用戶流程 \{#enable-phone-number-for-end-user-flows}

新增並設定 SMS 連接器後，你可以啟用基於手機號碼的驗證流程：

- [**設定註冊方式**](/end-user-flows/sign-up-and-sign-in/sign-up)：前往 <CloudLink to="/sign-in-experience/sign-up-and-sign-in">登入體驗 > 註冊與登入</CloudLink>，於註冊區段新增「手機號碼」或「電子郵件或手機號碼」作為識別項。注意：註冊時必須驗證手機號碼。
- [**設定登入方式**](/end-user-flows/sign-up-and-sign-in/sign-in)：於登入區段新增「手機號碼」作為登入方式。可選擇密碼或 SMS 驗證碼進行驗證 (Authentication)。點擊「新增其他」以增加更多選項。
- [**設定忘記密碼**](/end-user-flows/sign-up-and-sign-in/reset-password)：於登入區段，若啟用密碼登入，將看到「忘記密碼」設定。啟用「手機驗證碼」即可讓使用者透過 SMS 重設密碼。
- [**設定 SMS 作為 MFA**](/end-user-flows/mfa/sms-mfa)：前往 <CloudLink to="/mfa">登入體驗 > 多重要素驗證 (MFA)</CloudLink>。啟用「SMS 驗證碼」作為第二驗證因素以提升安全性。
- [**設定帳號管理**](/end-user-flows/account-settings/by-account-api#manage-phone)：使用 [Account API](/end-user-flows/account-settings/by-account-api) 自訂帳號中心，讓使用者管理手機號碼。

:::warning
由於我們的設定會立即生效，刪除現有 SMS 連接器可能影響終端用戶的正常登入與註冊流程。

例如，若登入識別項包含「手機號碼 + 驗證碼」組合，刪除現有 SMS 連接器後，使用者將無法收到驗證碼，登入流程會出現錯誤。

請操作連接器時務必注意潛在風險。
:::
