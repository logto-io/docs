Logto 提供內建電子郵件服務，方便你在以下情境中使用：

1. 快速探索或測試 Logto 的電子郵件登入體驗。
2. 直接用於你的線上產品。這主要適用於願意使用 `logto.email` 作為發件人電子郵件域的新創公司。

Logto 電子郵件服務的特點：

- **免費使用**：完全免費，無每日電子郵件使用限制，節省你的成本。
- **輕鬆無憂**：無需配置任何第三方電子郵件服務提供商。只需自訂電子郵件模板的基本品牌資訊。如果你尚未有自己的品牌資訊，可以選擇幾個點擊即可開始使用。
- **確保送達**：基於 Logto 的電子郵件服務，你可以獲得穩定的服務和可靠的電子郵件送達，確保使用者能夠存取你的產品。

然而，儘管方便，仍需注意一些限制：

1. 電子郵件將從固定地址 `no-reply@logto.email` 發送。
2. 你無法在電子郵件中新增連結或其他自訂內容。

根據不斷變化的業務需求，你可以選擇稍後使用其他電子郵件服務提供商。我們提供一系列 [即用型電子郵件服務連接器](/connectors/email-connectors#popular-email-providers)，並支援 [SMTP](/integrations/smtp)、[HTTP](/integrations/http-email) 和 [Webhook](/developers/webhooks) 觸發器來發送電子郵件，因此你總能找到適合的方式。

:::note
Logto 內建免費電子郵件服務目前僅適用於 [Cloud](https://cloud.logto.io/) 使用者。對於開源服務的使用者，你可以靈活配置自己的電子郵件服務提供商以進行電子郵件登入。
:::

## 配置步驟 \{#configuration-steps}

按照以下步驟配置 Logto 電子郵件服務：

1. 前往 <CloudLink to="/connectors/passwordless">Connector > Email and SMS connectors</CloudLink>。
2. 要新增一個 Email 連接器，點擊 "**Set up**" 按鈕並選擇 "**Logto email service**" 連接器。
3. 當 "Logto email service" 連接器成功建立後，你可以自訂電子郵件模板中顯示的基本品牌資訊。
4. 完成這些更改後，記得在儲存更改前將測試電子郵件模板發送到你的電子郵件地址。

自訂選項：

- **From email**：發件人電子郵件設為 `no-reply@logto.email`，無法修改。
- **Sender name**：設置你的品牌名稱作為發件人名稱，以確保使用者識別。
- **Company information**：顯示你的公司名稱、地址或郵遞區號，以增強使用者信任並滿足合規要求。_注意：不允許使用 URL。_
- **App logo**：上傳你的應用程式品牌標誌，以便在使用者收到的電子郵件中展示應用程式的品牌價值。

## 統一電子郵件模板 \{#unified-email-templates}

Logto 電子郵件服務使用針對特定驗證場景量身定制的統一電子郵件模板：

| 用途           | 場景                                                                                                                                                                                                                          |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 註冊           | 使用者使用其電子郵件創建帳戶，並通過輸入 Logto 發送到其電子郵件的驗證碼進行驗證。                                                                                                                                             |
| 登入           | 使用者使用其電子郵件登入，並通過輸入驗證碼而非密碼進行驗證。                                                                                                                                                                  |
| 忘記密碼       | 如果使用者在登入時忘記密碼，他們可以選擇使用已經在 Logto 驗證過的電子郵件來驗證其身分。                                                                                                                                       |
| 通用           | 此模板可作為各種場景的通用備用選項，包括測試連接器配置等。                                                                                                                                                                    |
| 組織邀請       | 使用此模板向使用者發送邀請連結以加入組織。                                                                                                                                                                                    |
| 使用者權限驗證 | 在應用程式使用過程中，可能會有一些高風險操作或風險級別較高的操作需要額外的使用者驗證，例如銀行轉帳、刪除正在使用的資源和取消會員資格。`UserPermissionValidation` 模板可用於定義使用者在這些情況下收到的電子郵件驗證碼的內容。 |
| 綁定新識別符   | 當使用者修改其個人資料時，他們可能會將電子郵件地址綁定到其當前帳戶。在這種情況下，`BindNewIdentifier` 模板可用於自訂驗證電子郵件的內容。                                                                                      |

## 常見問題 \{#faqs}

<details>
<summary>

### 是否可以在 Logto 電子郵件服務中新增自訂 CSS 或 HTML？ \{#is-it-possible-to-add-custom-css-or-html-to-logto-email-service}

</summary>

Logto 的內建電子郵件服務不支援自訂 CSS 或 HTML。你只能修改通用品牌元素。此限制是為了維持內建電子郵件服務的穩定性，因為所有租戶共用相同的 IP 地址和發件人地址。更多詳情，請參閱「[提高電子郵件送達率的因素](https://blog.logto.io/verification-email-delivery#factors-to-improve-email-delivery)」。

若要自訂電子郵件模板，我們建議使用其他電子郵件連接器，例如 AWS Direct Mail、SendGrid、Mailgun、Postmark 或 SMTP。

</details>
