Logto 為你在以下情境下提供內建電子郵件服務：

1. 快速體驗或測試 Logto 的電子郵件登入體驗。
2. 直接用於你的線上產品。這主要適合願意以 `logto.email` 作為寄件人郵件網域的新創團隊。

Logto 電子郵件服務的特點：

- **免費使用**：完全免費，且無每日郵件發送數量限制，幫你節省成本。
- **無需設定**：無需與任何第三方電子郵件服務供應商進行設定。只需自訂電子郵件模板的基本品牌資訊。如果你尚未有自己的品牌資訊，也可以直接點幾下就開始使用。
- **穩定送達**：基於 Logto 的郵件服務，你能獲得穩定的服務與可靠的郵件送達，確保使用者能順利存取你的產品。

然而，雖然方便，仍有一些限制需注意：

1. 郵件將從固定地址 `no-reply@logto.email` 發送。
2. 你無法在郵件中新增連結或任何其他自訂內容。

隨著業務需求變化，你可以隨時選擇其他電子郵件服務供應商。我們提供多種[現成的電子郵件服務連接器](/connectors/email-connectors#popular-email-providers)，同時也支援 [SMTP](/integrations/smtp)、[HTTP](/integrations/http-email) 及 [Webhook](/developers/webhooks) 觸發郵件發送，總有一種方式適合你。

:::note
Logto 內建免費電子郵件服務目前僅提供給 [Cloud](https://cloud.logto.io/) 用戶。開源版用戶則可自行設定郵件服務供應商以支援郵件登入。
:::

## 設定步驟 \{#configuration-steps}

請依下列步驟設定 Logto 電子郵件服務：

1. 前往 <CloudLink to="/connectors/passwordless">連接器 > 電子郵件與簡訊連接器</CloudLink>。
2. 若要新增 Email 連接器，請點擊「**設定**」按鈕並選擇「**Logto 電子郵件服務**」連接器。
3. 當「Logto 電子郵件服務」連接器建立成功後，你可以自訂電子郵件模板中顯示的基本品牌資訊。
4. 完成上述設定後，請記得在儲存變更前，先發送測試郵件模板到你的電子郵件地址。

可自訂項目：

- **寄件人郵件 (From email)**：寄件人郵件固定為 `no-reply@logto.email`，不可修改。
- **寄件人名稱 (Sender name)**：設定你的品牌名稱作為寄件人名稱，提升使用者辨識度。
- **公司資訊 (Company information)**：顯示你的公司名稱、地址或郵遞區號，提升使用者信任並符合合規要求。_請注意，不允許填寫網址。_
- **應用程式 Logo (App logo)**：上傳你的應用程式品牌 Logo，讓品牌價值在使用者收到的郵件中展現。

## 統一郵件模板 \{#unified-email-templates}

Logto 電子郵件服務根據不同驗證 (Authentication) 情境，採用統一郵件模板：

| 用途                     | 情境說明                                                                                                                                             |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| Register                 | 使用者以電子郵件註冊帳號，並輸入 Logto 發送到郵箱的驗證碼完成驗證。                                                                                  |
| SignIn                   | 使用者以電子郵件登入，並以輸入驗證碼（非密碼）完成驗證。                                                                                             |
| ForgotPassword           | 使用者登入時忘記密碼，可選擇以已驗證的電子郵件進行身分驗證。                                                                                         |
| Generic                  | 此模板可作為多種情境的備用選項，包括測試連接器設定等。                                                                                               |
| OrganizationInvitation   | 使用此模板向使用者發送邀請連結，邀請其加入組織 (Organization)。                                                                                      |
| UserPermissionValidation | 應用程式使用過程中，部分高風險操作（如銀行轉帳、刪除使用中資源、取消會員等）需額外驗證。可用 `UserPermissionValidation` 模板自訂這類郵件驗證碼內容。 |
| BindNewIdentifier        | 使用者修改個人資料時，可能需將電子郵件綁定至現有帳號，此時可用 `BindNewIdentifier` 模板自訂驗證郵件內容。                                            |
| MfaVerification          | 啟用電子郵件 MFA（多重要素驗證）時，該模板用於發送驗證碼給使用者。                                                                                   |
| BindMfa                  | 啟用電子郵件 MFA 時，該模板用於設定 MFA 郵件驗證碼。使用者將在綁定或設定電子郵件作為 MFA 因子時收到此驗證碼。                                        |

## 常見問題 \{#faqs}

<details>
<summary>

### 可以在 Logto 電子郵件服務中加入自訂 CSS 或 HTML 嗎？ \{#is-it-possible-to-add-custom-css-or-html-to-logto-email-service}

</summary>

Logto 內建電子郵件服務不支援自訂 CSS 或 HTML，你僅能修改通用品牌元素。此限制是為了維護內建郵件服務的穩定性，因所有租戶共用同一 IP 與寄件人地址。詳情請參考「[提升郵件送達率的因素](https://blog.logto.io/verification-email-delivery#factors-to-improve-email-delivery)」。

若需自訂郵件模板，建議改用其他郵件連接器，如 AWS Direct Mail、SendGrid、Mailgun、Postmark 或 SMTP。

</details>
