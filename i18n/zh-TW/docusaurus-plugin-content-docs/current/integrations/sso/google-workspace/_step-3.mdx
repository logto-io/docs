導航至 [Credentials](https://console.cloud.google.com/apis/credentials) 頁面，然後點擊 `Create Credentials` 按鈕。從下拉選單中選擇 `OAuth client ID` 選項，為你的應用程式創建新的 OAuth 憑證。

<img
  src="/img/assets/google_workspace_create_credentials.webp"
  alt="Google Workspace create credentials"
/>

繼續設置 OAuth 憑證，填寫以下資訊：

<img
  src="/img/assets/google_workspace_credentials_config.webp"
  alt="Google Workspace credentials config"
/>

1. 選擇 `Web application` 作為應用程式類型。
2. 填寫客戶端應用程式的 `Name`，例如 `Logto SSO Connector`。這將幫助你在未來識別這些憑證。
3. 在 `Authorized redirect URIs` 中填入 Logto 回呼 URI。這是 Google 在成功驗證後將使用者的瀏覽器重定向的 URI。當使用者成功通過身分提供者 (IdP) 驗證後，IdP 會將使用者的瀏覽器重定向回此指定的 URI，並附帶授權碼。Logto 將根據從此 URI 接收到的授權碼完成驗證過程。
4. 在 `Authorized JavaScript origins` 中填入 Logto 回呼 URI 的來源。這確保只有你的 Logto 應用程式可以向 Google OAuth 伺服器發送請求。
5. 點擊 `Create` 按鈕以創建 OAuth 憑證。
