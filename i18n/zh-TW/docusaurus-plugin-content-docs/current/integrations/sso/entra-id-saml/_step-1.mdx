開始 Azure AD 單一登入 (SSO) 整合，首先在 Azure AD 端創建一個 SSO 應用程式。

1. 前往 [Azure 入口網站](https://portal.azure.com/) 並以管理員身分登入。
2. 選擇 `Microsoft Entra ID` 服務。
3. 使用側邊選單導航至 `企業應用程式 (Enterprise applications)`。點擊 `新增應用程式 (New application)`，然後選擇 `創建你自己的應用程式 (Create your own application)`。

<img src={azureAdCreateApplicationImage} alt="Azure AD create application" />

1. 輸入應用程式名稱並選擇 `整合任何在圖庫中找不到的其他應用程式 (Integrate any other application you don't find in the gallery) (非圖庫, Non-gallery)`。
2. 選擇 `設定單一登入 (Setup single sign-on)` > `SAML`。

<img src={azureAdSetUpSingleSignOnImage} alt="Azure AD set up SSO" />

1. 按照指示進行，第一步需要使用 Logto 提供的以下資訊填寫基本 SAML 配置。

<img src={azureAdSpConfigImage} alt="Azure AD SP config" />

- **受眾 URI (Audience URI, SP Entity ID)**：它作為你的 Logto 服務的全局唯一識別符，作為 SP 在向 IdP 發送驗證請求時的 EntityId。此識別符對於 IdP 與 Logto 之間安全交換 SAML 聲明和其他驗證相關數據至關重要。
- **ACS URL**：Assertion Consumer Service (ACS) URL 是以 POST 請求發送 SAML 聲明的位置。此 URL 由 IdP 用於將 SAML 聲明發送至 Logto。它充當回調 URL，Logto 期望在此接收並處理包含使用者身分資訊的 SAML 回應。

點擊 `儲存 (Save)` 以繼續。
