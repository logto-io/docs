# 使用 Postmark 郵件服務設定電子郵件驗證 (Set up email verification with Postmark email service)

Logto 的 Postmark 郵件服務連接器。

<GuideTip />

## 開始使用 \{#get-started}

Postmark 是一個用於交易型與行銷郵件的郵件平台。我們可以利用其郵件發送功能來發送 _驗證碼_。

## 註冊 Postmark 帳號 \{#register-postmark-account}

請至 [Postmark 官網](https://postmark.com/) 建立新帳號。如果你已有帳號可略過此步驟。

## 設定你的連接器 \{#configure-your-connector}

將 `serverToken` 欄位填入你在 Postmark 伺服器設定中找到的 Server Token。

將 `fromEmail` 欄位填入寄件者的 _From Address_。

為了啟用完整的使用者流程，必須建立 `Register`、`SignIn`、`ForgotPassword` 與 `Generic` 這些 usageType 的範本。

以下是 Postmark 連接器範本 JSON 範例。

```jsonc
[
  {
    "usageType": "Register",
    "templateAlias": "logto-register",
  },
  {
    "usageType": "SignIn",
    "templateAlias": "logto-sign-in",
  },
  {
    "usageType": "ForgotPassword",
    "templateAlias": "logto-forgot-password",
  },
  {
    "usageType": "Generic",
    "templateAlias": "logto-generic",
  },
]
```

## 測試 Postmark 郵件連接器 \{#test-postmark-email-connector}

你可以輸入一個電子郵件地址並點擊「發送」來測試設定是否可用，再點選「儲存並完成」。

就是這麼簡單。別忘了 <MainSiteUrl href="/connectors/email-connectors/#enable-email-address-for-end-user-flows">在註冊與登入流程中啟用連接器</MainSiteUrl>。

## 設定類型 \{#config-types}

| 名稱        | 類型   |
| ----------- | ------ |
| serverToken | string |
| fromEmail   | string |

| 範本屬性      | 類型        | 列舉值                                               |
| ------------- | ----------- | ---------------------------------------------------- |
| usageType     | enum string | 'Register' / 'SignIn' / 'ForgotPassword' / 'Generic' |
| templateAlias | string      | N/A                                                  |
