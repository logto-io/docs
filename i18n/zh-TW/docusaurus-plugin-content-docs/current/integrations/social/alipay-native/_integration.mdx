## 建立並配置 Alipay 應用程式 \{#create-and-configure-alipay-app}

1. 使用剛註冊的帳號登入 [Alipay 控制台](https://open.alipay.com/)。
2. 在「我的應用」(我的应用) 面板中，進入「網頁 & 移動應用」(网页&移动应用) 標籤。
3. 點擊「立即創建」(立即创建) 按鈕開始配置你的應用程式。
4. 在「應用名稱」(应用名称) 中命名你的應用程式，遵循命名規範並上傳你的「應用圖標」(应用图标)，確保選擇「移動應用」(移动应用) 作為「應用類型」(应用类型)。對於建立 iOS 應用程式，需要一個唯一的「Bundle ID」。此外，Android 應用程式需要「應用簽名」(应用签名) 和「應用包名」(应用包名)。
5. 完成應用程式創建後，進入概覽頁面，點擊「添加能力」(添加能力) 以添加「第三方應用授權」(第三方应用授权)、「獲取會員信息」(获取会员信息) 和「App 支付寶登入」(App 支付宝登录)，然後啟用 Alipay 登入。
6. 前往 [Alipay 客戶中心](https://b.alipay.com/index2.htm)，使用 Alipay 開發者帳號登入。點擊頂部欄的「帳號中心」(账号中心)，進入「APPID 綁定」(APPID 绑定)，入口在側邊欄底部。輸入你在步驟 4 創建的移動應用程式的 APPID，然後「添加綁定」(添加绑定)。
7. 點擊「App 支付寶登入」的「簽署」按鈕，按照指南完成簽署流程。完成此步驟後，你應該會看到在步驟 5 中添加的能力生效。
8. 回到 Alipay 開放平台控制台頁面，你可以在「開發信息」(开发信息) 部分找到「接口加簽方式（密鑰/證書）」(接口加签方式（密钥/证书）)。點擊「設置」(设置) 按鈕，你會進入設置簽署方式的頁面。「公鑰」(公钥) 是首選的簽署模式，並在文本輸入框中填入你生成的公鑰文件的內容。
9. 點擊 Alipay 控制台頁面底部的「設置」(设置) 按鈕設置「授權回調地址」(授权回调地址)。`${your_logto_origin}/callback/${connector_id}` 是 Logto 核心使用的預設回調 URI。`connector_id` 可以在 Logto 管理控制台連接器詳細頁面的頂部欄找到。
10. 完成所有這些步驟後，回到 Alipay 控制台頁面的右上角，點擊「提交審核」(提交审核)。一旦審核通過，你就可以順利進行 Alipay 登入流程。

:::note

你可以在本地機器上使用 _openssl_ 生成密鑰對，通過在終端執行以下代碼片段。

```bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -outform PEM -pubout -out public.pem
```

在 Alipay 應用程式設置網站填寫公鑰時，需要移除 `public.pem` 的標頭和尾部，刪除所有換行符，然後將其餘內容粘貼到「公鑰」文本輸入框中。
:::

## 設定 Logto Alipay Native 連接器設置 \{#set-up-the-logto-alipay-native-connector-settings}

1. 在 [Alipay 控制台工作區](https://open.alipay.com/dev/workspace) 進入「我的應用」(我的应用) 面板，點擊「網頁 & 移動應用」(网页&移动应用) 標籤，你可以找到所有應用程式的 APPID。
2. 在前一部分的步驟 7 中，你已經生成了一對密鑰，包括私鑰和公鑰。
3. 填寫 Logto 連接器設置：
   - 在 `appId` 欄位中填入你從步驟 1 獲得的 APPID。
   - 在 `privateKey` 欄位中填入步驟 2 中提到的私鑰文件的內容。請確保用 '\n' 替換所有換行符。你不需要移除私鑰文件中的標頭和尾部。
   - 由於我們在「建立並配置 Alipay 應用程式」步驟 7 中選擇了「公鑰」簽署模式，請在 `signType` 欄位中填入 'RSA2'。

### 配置類型 \{#config-types}

| 名稱       | 類型        | 枚舉值          |
| ---------- | ----------- | --------------- |
| appId      | string      | N/A             |
| privateKey | string      | N/A             |
| signType   | enum string | 'RSA' \| 'RSA2' |

## 在你的應用中啟用 Alipay 原生登入 \{#enable-alipay-native-sign-in-in-your-app}

### iOS \{#ios}

我們假設你已經在應用中整合了 [Logto iOS SDK](/quick-starts/swift)。在這種情況下，事情非常簡單，你甚至不需要閱讀 Alipay SDK 文檔：

**1. 將 `LogtoSocialPluginAlipay` 添加到你的 Xcode 專案**

添加框架：

<img src="/img/assets/add-framework.png" alt="Add framework" />

:::note

該插件包含 Alipay「極簡版 SDK」(极简版 SDK)。一旦導入插件，你可以直接使用 `import AFServiceSDK`。

:::

**2. 將插件添加到你的 `LogtoClient` 初始化選項**

```swift
let logtoClient = LogtoClient(
  useConfig: config,
  socialPlugins: [LogtoSocialPluginAlipay(callbackScheme: "your-scheme")]
)
```

其中 `callbackScheme` 是一個可以導航到你的應用的 [自定義 URL Schemes](https://developer.apple.com/documentation/xcode/defining-a-custom-url-scheme-for-your-app) 之一。

### Android \{#android}

我們假設你已經在應用中整合了 [Logto Android SDK](/quick-starts/android)。在這種情況下，事情非常簡單，你甚至不需要閱讀 Alipay SDK 文檔：

**1. 下載 Alipay「極簡版 SDK」並將其添加到你的專案**

從 [Logto 第三方社交 SDKs](https://github.com/logto-io/social-sdks/blob/master/alipay/android/alipaySdk-15.7.9-20200727142846.aar) 下載 Alipay「極簡版 SDK」(极简版 SDK) 到你的專案的 `app/libs` 資料夾：

```bash
project-path/app/libs/alipaySdk-15.7.9-20200727142846.aar
```

**2. 將 Alipay「極簡版 SDK」作為依賴項添加**

打開你的 `build.gradle` 文件：

```bash
project-path/app/build.gradle
```

添加依賴項：

```kotlin
dependencies {
  // ...
  implementation(files("./libs/alipaySdk-15.7.9-20200727142846.aar"))  // kotlin-script
  // or
  implementation files('./libs/alipaySdk-15.7.9-20200727142846.aar')  // groovy-script
}
```
