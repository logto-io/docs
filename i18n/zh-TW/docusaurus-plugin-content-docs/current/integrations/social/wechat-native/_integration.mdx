## 在微信開放平台創建移動應用程式 \{#create-a-mobile-app-in-the-wechat-open-platform}

:::tip

如果你已經完成某些部分，可以跳過這些部分。

:::

### 創建帳戶 \{#create-an-account}

打開 https://open.weixin.qq.com/，點擊右上角的「註冊」按鈕，然後完成註冊流程。

### 創建移動應用程式 \{#create-a-mobile-app}

使用你剛創建的帳戶登入。在「移動應用」(移动应用) 標籤中，點擊大綠色按鈕「創建移動應用」(创建移动应用)。

<img src="/img/assets/app-tabs.png" alt="App tabs" />

讓我們在申請表中填寫所需的信息。

<img src="/img/assets/create-mobile-app.png" alt="Create mobile app" />

#### 基本信息 \{#basic-info}

大多數都很簡單明瞭，我們這裡有幾個提示：

- 如果你只是想測試微信登入，且應用程式不在 App Store 上，在「應用是否上架」部分選擇「否」以跳過「應用下載鏈接」。
- 「應用操作流程圖」看起來有點棘手。根據我們的經驗，你需要準備一個簡單的流程圖和幾個應用程式截圖，以提高通過審核的可能性。

點擊「下一步」繼續。

#### 平台信息 \{#platform-info}

你可以配置 iOS 和 Android 平台之一或兩者，以將 Logto 與微信原生登入集成。

**iOS 應用程式**

勾選「iOS 應用」(iOS 应用)，然後根據你的應用程式勾選目標設備類型。

<img src="/img/assets/platform.png" alt="App platform" />

如果你選擇了「否」作為 App Store 上架狀態，可以跳過填寫「AppStore 下載地址」。

填寫 _Bundle ID_、_測試版 Bundle ID_ 和 _Universal Links_（實際上，只需要一個鏈接 😂）。

:::note

_Bundle ID_ 和 _測試版 Bundle ID_ 可以是相同的值。

:::

:::tip

微信要求使用 universal link 進行原生登入。如果你尚未設置或不知道它，請參考 [Apple 官方文檔](https://developer.apple.com/ios/universal-links/)。

:::

**Android 應用程式**

勾選「Android 應用」(Android 应用)。

<img src="/img/assets/platform-android-app.png" alt="Android app platform" />

填寫 _應用簽名_ (应用签名) 和 _應用包名_ (应用包名)。

:::note

你需要簽署你的應用程式以獲取簽名。更多信息請參考 [簽署你的應用程式](https://developer.android.com/studio/publish/app-signing)。

:::

完成簽署後，你可以執行 `signingReport` 任務以獲取簽名。

```bash
./gradlew your-android-project:signingReport
```

對應構建變體報告的 `MD5` 值將是 _應用簽名_ (应用签名)，但記得移除所有分號並將其轉為小寫。

例如：`1A:2B:3C:4D` -> `1a2b3c4d`。

#### 等待審核結果 \{#waiting-for-the-review-result}

完成平台信息後，點擊「提交審核」繼續。通常，審核速度很快，會在 1-2 天內結束。

我們懷疑每次提交的審核員是隨機分配的，因為標準是浮動的。你可能第一次被拒絕，但不要放棄！說明你的現狀並詢問審核員如何修改。

## 在你的應用程式中啟用微信原生登入 \{#enable-wechat-native-sign-in-in-your-app}

### iOS \{#ios}

我們假設你已在應用程式中集成了 [Logto iOS SDK](/quick-starts/swift)。在這種情況下，事情相當簡單，你甚至不需要閱讀微信 SDK 文檔：

**1. 在你的 Xcode 專案中配置 universal link 和 URL scheme**

在 Xcode 專案 -> 簽署與功能標籤中，添加「關聯域」功能和你之前配置的 universal link。

<img src="/img/assets/universal-link.png" alt="Universal link" />

然後進入「信息」標籤，添加一個 [自定義 URL scheme](https://developer.apple.com/documentation/xcode/defining-a-custom-url-scheme-for-your-app) 與微信 App ID。

<img src="/img/assets/custom-url-scheme.png" alt="Custom URL scheme" />

最後打開你的 `Info.plist`，在 `LSApplicationQueriesSchemes` 下添加 `weixinULAPI` 和 `weixin`。

<img src="/img/assets/plist.png" alt="Plist" />

:::note

我們知道這些操作不是很合理，但這是我們找到的最小可行解決方案。更多信息請參考 [神奇的官方指南](https://developers.weixin.qq.com/doc/oplatform/en/Mobile_App/Access_Guide/iOS.html)。

:::

**2. 將 `LogtoSocialPluginWechat` 添加到你的 Xcode 專案中**

添加框架：

<img src="/img/assets/add-framework.png" alt="Add framework" />

並在你的 Build Settings > Linking > Other Linker Flags 中添加 `-ObjC`：

<img src="/img/assets/linker-flags.png" alt="Linker flags" />

:::note

該插件包含 WeChat Open SDK 1.9.2。導入插件後，你可以直接使用 `import WechatOpenSDK`。

:::

**3. 將插件添加到你的 `LogtoClient` 初始化選項中**

```swift
let logtoClient = LogtoClient(
  useConfig: config,
  socialPlugins: [LogtoSocialPluginWechat()]
)
```

**4. 正確處理 `onOpenURL`**

:::note

函數 `LogtoClient.handle(url:)` 將處理你啟用的所有原生連接器。你只需調用一次即可。

:::

```swift
// SwiftUI
YourRootView()
  .onOpenURL { url in
      LogtoClient.handle(url: url)
  }

// 或 AppDelegate
func application(_ app: UIApplication, open url: URL, options: /*...*/) -> Bool {
  LogtoClient.handle(url: url)
}
```

### Android \{#android}

我們假設你已在應用程式中集成了 [Logto Android SDK](/quick-starts/android)。在這種情況下，事情相當簡單，你甚至不需要閱讀微信 SDK 文檔：

**1. 將 `Wechat Open SDK` 添加到你的專案中**

確保 `mavenCentral()` 存儲庫在你的 Gradle 專案存儲庫中：

```kotlin
repositories {
  // ...
  mavenCentral()
}
```

將 Wechat Open SDK 添加到你的依賴項中：

```kotlin
dependencies {
  // ...
  api("com.tencent.mm.opensdk:wechat-sdk-android:6.8.0")  // kotlin-script
  // 或
  api 'com.tencent.mm.opensdk:wechat-sdk-android:6.8.0'   // groovy-script
}
```

**2. 將 `WXEntryActivity` 引入你的專案**

在你的包根目錄下創建一個 `wxapi` 包，並在 `wxapi` 包中添加 `WXEntryActivity`（以 `com.sample.app` 為例）：

```kotlin
// WXEntryActivity.kt
package com.sample.app.wxapi

import io.logto.sdk.android.auth.social.wechat.WechatSocialResultActivity

class WXEntryActivity: WechatSocialResultActivity()
```

```java
// WXEntryActivity.java
package com.sample.app.wxapi

import io.logto.sdk.android.auth.social.wechat.WechatSocialResultActivity

public class WXEntryActivity extends WechatSocialResultActivity {}
```

專案中 `WXEntryActivity` 的最終位置應如下所示（以 Kotlin 為例）：

```bash
src/main/kotlin/com/sample/app/wxapi/WXEntryActivity.kt
```

**3. 修改 `AndroidManifest.xml`**

將以下行添加到你的 `AndroidManifest.xml`：

```xml
\<?xml version="1.0" encoding="utf-8"?>
\<manifest xmlns:android="http://schemas.android.com/apk/res/android"
  package="com.sample.app">

  \<application>
    \<!-- 要添加的行 -->
    \<activity android:name=".wxapi.WXEntryActivity" android:exported="true"/>
  \</application>

\</manifest>
```
