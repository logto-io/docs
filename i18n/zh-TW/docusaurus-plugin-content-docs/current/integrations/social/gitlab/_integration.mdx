## 使用 GitLab 帳號登入 \{#sign-in-with-gitlab-account}

前往 [GitLab 網站](https://gitlab.com/)，並使用你的 GitLab 帳號登入。如果你還沒有帳號，可以註冊一個新帳號。

## 建立並設定 OAuth 應用程式 \{#create-and-configure-oauth-app}

依照 [建立 GitLab OAuth App](https://docs.gitlab.com/ee/integration/oauth_provider.html) 指南，註冊一個新的應用程式。

在 **Name** 欄位為你的 OAuth 應用程式命名，並填寫應用程式的 **Redirect URI**。請將 **Redirect URIs** 設定為 `${your_logto_origin}/callback/${connector_id}`。`connector_id` 可以在 Logto 管理主控台連接器詳細頁面的頂部欄位找到。

在 scopes 欄位，選擇 `openid`。你也可以啟用 `profile` 和 `email`。`profile` 權限範圍 (Scope) 用於取得使用者的個人資訊，`email` 權限範圍 (Scope) 用於取得使用者的電子郵件地址。如果你想使用這些權限範圍 (Scopes)，請確保已在 GitLab OAuth 應用程式中允許這些權限範圍 (Scopes)。這些權限範圍 (Scopes) 之後在設定連接器時也會用到。

:::note

- 如果你使用自訂網域，請將自訂網域與預設 Logto 網域都加入 Redirect URIs，以確保 OAuth 流程在兩個網域下都能正常運作。
- 如果登入時遇到「The redirect_uri MUST match the registered callback URL for this application.」錯誤訊息，請嘗試讓 GitLab OAuth App 的 Redirect URI 與 Logto App 的 redirect URL（包含協定）完全一致，以解決此問題。
  :::

## 管理 OAuth 應用程式 \{#managing-oauth-apps}

前往 GitLab 的 [Applications 頁面](https://gitlab.com/-/profile/applications)，你可以在這裡新增、編輯或刪除現有的 OAuth 應用程式。你也可以在 OAuth 應用程式詳細頁面找到 `Application ID` 並產生 `Secret`。

## 設定你的連接器 (Connector) \{#configure-your-connector}

將你在前述 OAuth 應用程式詳細頁面取得的 _Application ID_ 與 _Secret_ 分別填入 `clientId` 與 `clientSecret` 欄位。

`scope` 是以空格分隔的 [權限範圍 (Scopes)](https://docs.gitlab.com/ee/integration/oauth_provider.html#authorized-applications) 列表。如果未提供，預設為 `openid`。對於 GitLab 連接器，你可能會用到的權限範圍 (Scopes) 包含 `openid`、`profile` 和 `email`。`profile` 權限範圍 (Scope) 用於取得使用者的個人資訊，`email` 權限範圍 (Scope) 用於取得使用者的電子郵件地址。請確保你已在 GitLab OAuth 應用程式中允許這些權限範圍 (Scopes)（於 [建立並設定 OAuth 應用程式](#create-and-configure-oauth-app) 章節設定）。

### 設定類型 \{#config-types}

| Name         | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
