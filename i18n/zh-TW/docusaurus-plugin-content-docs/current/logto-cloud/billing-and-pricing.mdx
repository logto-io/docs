# 計費與價格 (Billing and pricing)

在 Logto，我們的 [Pro 方案](https://logto.io/pricing) 完全自助，設計上強調透明，讓你可以輕鬆理解並管理你的帳單。

## 瞭解結構 \{#understand-the-structure}

基本結構如下：

```math
\begin{align*}
\text{你的下期帳單} &= \$16 \text{（基本價格）} \\
&+ \text{附加功能費用（未結算用量）} \\
&+ \text{附加功能費用（下期週期收費）}
\end{align*}
```

在管理主控台的租戶設定區，你會看到「你的下期帳單」區塊，可以隨時監控與管理即將產生的費用。

## 項目明細 \{#item-breakdown}

請注意，只有少數功能屬於附加功能（add-on）結構。

同時，Logto 也提供隨用隨付與按比例計費模式，確保你能方便地充分利用我們的服務。

|                                                | 包含額度               | 超出額度費用              |
| ---------------------------------------------- | ---------------------- | ------------------------- |
| **權杖 (Tokens)**                              | 100K                   | $0.08 / 月 / 100 超出部分 |
| **機器對機器應用程式**                         | 1                      | 每月每個 $8               |
| **API 資源 (API resources)**                   | 3                      | 每月每個 $4               |
| **企業級單一登入 (Enterprise SSO)**            | 0                      | 每月每個 $48              |
| **多重要素驗證 (Multi-factor authentication)** | _N/A_                  | 每月 $48                  |
| **組織 (Organization)**                        | 無限組織與所有組織功能 | 每月 $48                  |
| **進階安全組合 (Advanced security bundle)**    | 所有進階安全功能       | 每月 $48                  |
| **租戶成員 (Tenant members)**                  | 3                      | 每月每個 $8               |

考慮上述因素後，計算公式如下：

```math
\begin{align*}
\text{你的下期帳單} &= \$16 \\
&+ \text{附加功能單價} \times \\
\quad & (\text{新增數量} \times \text{建立時剩餘比例時間} \\
\quad &- \text{移除數量} \times \text{刪除時剩餘比例時間}) \\
\quad &+ \text{附加功能單價} \times \text{下期持續數量}
\end{align*}
```

1. **附加功能單價**：此附加功能的單價
2. **新增數量**：本期帳單週期內新增的附加功能數量
3. **移除數量**：本期帳單週期內移除的附加功能數量
4. **建立時剩餘比例時間**：你建立資源時，本期週期剩餘的比例時間
5. **刪除時剩餘比例時間**：你刪除資源時，本期週期剩餘的比例時間
6. **下期持續數量**：下期帳單週期需持續計費的數量

我們來看兩個例子說明運作方式。

### 範例 1：無變動時的持續計費 \{#example-1-consistent-billing-with-no-changes}

如果你沒有新增或移除項目，帳單會保持一致。

例如，你使用 Pro 方案並啟用 2 個 SSO 服務，只要不做任何變動，帳單會維持不變。

```math
\text{你的下期帳單} = \$ 16 + \$ 48 \times 2 = \$ 112
```

### 範例 2：新增附加功能或變動會影響下期帳單 \{#example-2-adding-add-ons-or-making-changes-will-affect-your-upcoming-bills}

根據不同情境，下期帳單會有所不同，但計算方式不變。

**情境 1：你訂閱 Pro 方案但未加購附加功能，僅測試企業級單一登入 (Enterprise SSO) 10 天。**

假設你的訂閱每月 5 日續約，20 日新增一個 Enterprise SSO，本月共用 15 天。若 30 日刪除，代表實際只用 10 天。

```math
\$16 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$32
```

**情境 2：你訂閱 Pro 方案未加購附加功能，新增並移除 API 資源後，最終共用 5 個 API 資源**

本期帳單週期內你調整了附加功能，這些變動會反映在下期帳單。變動後的第一個月可能略高。帳單會包含 $16 基本費、未結算用量的附加功能費用，以及下期週期的完整收費。

你的帳單週期從每月 1 日開始，已用完 3 個免費 API 資源。本月你做了兩個動作：

1. 5 日新增 4 個資源。
2. 15 日刪除其中 2 個資源。

因此，下期帳單週期你會持續有 2 個 API 資源。

```math
\$16 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$50.67
```

### 附加功能用量時間如何計算？ \{#how-is-the-add-on-usage-time-determined}

我們採用隨用隨付模式。如果你經常新增或移除資源，附加功能僅在啟用期間計費。別擔心——我們的系統（由 Stripe 支援）會精確計算到秒，確保每項資源的實際使用時間都能準確計費。

若你取消 Pro 方案，未用完的附加功能費用會退還，並從最終帳單中扣除。

## 升級或降級 \{#upgrade-or-downgrade}

### 免費方案升級至 Pro \{#free-plan-upgrade-to-pro}

升級與降級僅適用於生產租戶。以下情境建議你從免費方案升級：

1. 若你的 MAU 或權杖 (Token) 用量超過額度，我們會適時發送電子郵件提醒你升級。
2. 若你想使用僅限進階方案的功能。

### Pro 方案升級至企業級方案 \{#pro-plan-upgrade-to-enterprise}

若 Pro 方案無法滿足需求，並希望獲得完整企業級支援，請[聯絡我們](https://logto.io/contact)討論專屬企業方案。

### Pro 方案降級至免費方案 \{#pro-plan-downgrade-to-free-plan}

若你選擇停止使用 Pro 方案，可切換回免費方案，但需調整用量以符合免費方案額度。

## 未來價格方案調整 \{#future-pricing-plan-change}

為提供穩定且可預期的計費體驗，當價格調整時，我們會保留你目前的方案（grandfathering）。

## 常見問題 \{#faqs}

<details>

<summary>

### 權杖 (Token) 消耗與 MAU 有什麼關聯？ \{#what-is-the-connection-between-token-consumption-and-mau}

</summary>

Logto 採用權杖 (Token) 驗證 (Authentication)。單一使用者登入一次可能會消耗兩到三個權杖：ID 權杖 (ID token)、存取權杖 (Access token) 和重新整理權杖 (Refresh token)。若你使用 M2M 或組織 (Organization) 等功能，還可能涉及 M2M 權杖或組織權杖。權杖消耗取決於使用者與系統互動方式。為協助你追蹤，我們在 <CloudLink to="/dashboard">主控台 > 儀表板</CloudLink> 提供即時權杖用量監控。例如：免費方案提供 10 萬個權杖，約可支援 3 萬至 5 萬 MAU（每月活躍使用者）。

</details>
