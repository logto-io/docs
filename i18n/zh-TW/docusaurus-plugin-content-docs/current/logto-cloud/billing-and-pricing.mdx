# 計費與價格 (Billing and pricing)

在 Logto，我們的 [Pro 方案](https://logto.io/pricing) 完全自助，設計上強調透明，讓你能輕鬆瞭解並管理你的帳單。

## 瞭解結構 \{#understand-the-structure}

基本結構如下：

```math
\begin{align*}
\text{你的下期帳單} &= \$24 \text{（基本價格）} \\
&+ \text{附加功能費用（未結算用量）} \\
&+ \text{附加功能費用（下期週期收費）}
\end{align*}
```

在管理主控台的租戶設定區，你會看到「你的下期帳單」區塊，可以即時監控與管理即將產生的費用。

<img src="/img/assets/next-billing-info.png" alt="下期帳單資訊" />

## 項目明細 \{#item-breakdown}

請注意，只有部分功能屬於附加功能（add-on）結構。

同時，Logto 也提供隨用隨付（pay-as-you-go）與按比例計價（pro-rated）的價格模式，確保你能方便地充分利用我們的服務。

|                                                | 包含額度               | 超出額度費用              |
| ---------------------------------------------- | ---------------------- | ------------------------- |
| **權杖 (Tokens)**                              | 50K                    | $0.08 / 月 / 100 超出額度 |
| **機器對機器應用程式**                         | 1                      | $8 **_每個_** / 月        |
| **API 資源 (API resources)**                   | 3                      | $4 **_每個_** / 月        |
| **企業級單一登入 (Enterprise SSO)**            | 0                      | $48 **_每個_** / 月       |
| **多重要素驗證 (Multi-factor authentication)** | 所有驗證因子           | $48 / 月                  |
| **組織 (Organization)**                        | 無限組織與所有組織功能 | $48 / 月                  |
| **角色型存取控制 (Role-based access control)** | 無限角色與權限         | $32 / 月                  |
| **SAML 應用程式**                              | 0                      | $96 **_每個_** / 月       |
| **第三方應用程式**                             | 0                      | $8 **_每個_** / 月        |
| **進階安全組合**                               | 所有進階安全功能       | $48 / 月                  |
| **租戶成員**                                   | 1                      | $8 **_每個_** / 月        |

考慮這些因素後，計算公式如下：

```math
\begin{align*}
\text{你的下期帳單} &= \$24 \\
&+ \text{附加功能單價} \times \\
\quad & (\text{新增數量} \times \text{建立時剩餘比例時間} \\
\quad &- \text{移除數量} \times \text{刪除時剩餘比例時間}) \\
\quad &+ \text{附加功能單價} \times \text{下期週期持續數量}
\end{align*}
```

1. **附加功能單價**：此附加功能的單價
2. **新增數量**：本計費週期內新增的附加功能數量
3. **移除數量**：本計費週期內移除的附加功能數量
4. **建立時剩餘比例時間**：你建立資源時，本週期剩餘的比例時間
5. **刪除時剩餘比例時間**：你刪除資源時，本週期剩餘的比例時間
6. **下期週期持續數量**：下期週期持續計費所需的數量

我們來看兩個例子說明運作方式。

### 範例 1：無變動時的持續計費 \{#example-1-consistent-billing-with-no-changes}

如果你沒有新增或移除項目，帳單將保持一致。

例如，你在 Pro 方案下使用 2 個 SSO 服務，只要不做任何變動，帳單金額將維持不變。

```math
\text{你的下期帳單} = \$ 24 + \$ 48 \times 2 = \$ 120
```

### 範例 2：新增附加功能或變動會影響下期帳單 \{#example-2-adding-add-ons-or-making-changes-will-affect-your-upcoming-bills}

根據不同情境，這會讓你的下期帳單略有不同，但計算公式仍然適用。

**情境 1：你訂閱 Pro 方案但未加購任何附加功能，僅測試企業級單一登入 (Enterprise SSO) 10 天。**

若你的訂閱於每月 5 日續約，並於 20 日新增一個 Enterprise SSO，則本月使用 15 天。若於 30 日刪除，代表僅使用 10 天。

```math
\$24 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$40
```

**情境 2：你訂閱 Pro 方案但未加購任何附加功能，新增並移除 API 資源後，最終使用 5 個 API 資源**

這種情況下，你在本計費週期內調整了附加功能，變動將反映在下期帳單。變動後的第一個月可能略高。帳單將包含 $24 基本費、未結算用量的附加功能費用，以及下期週期的全額收費。

你的計費週期從每月 1 日開始，已用完前 3 個免費 API 資源。本月你進行了兩個操作：

1. 5 日新增 4 個資源。
2. 15 日刪除其中 2 個資源。

因此，下期週期你將持續有 2 個 API 資源。

```math
\$24 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$58.67
```

### 附加功能用量時間如何計算？ \{#how-is-the-add-on-usage-time-determined}

我們採用隨用隨付模式，你可能會好奇頻繁新增或移除資源時如何計算。附加功能僅在啟用期間計費。別擔心，我們的系統（由 Stripe 支援）會精確計算到秒，確保每項資源的實際使用時間都能準確計費。

若你取消 Pro 方案，未用完的附加功能費用將退還，並從最終帳單中扣除。

## 升級或降級 \{#upgrade-or-downgrade}

### 免費方案升級至 Pro \{#free-plan-upgrade-to-pro}

以下情境建議你考慮從免費方案升級：

1. 你的 MAU 或權杖 (Token) 用量超過額度。
2. 你需要使用僅限 Pro 方案的功能。

### 開發租戶升級至 Pro \{#dev-tenant-upgrade-to-pro}

若你目前使用開發租戶，想轉換為正式 Pro 方案，可輕鬆完成：

1. 保留你的成果：所有設定、應用程式與配置都會完整保留至 Pro 方案。
2. 立即啟用：轉換後立即享有所有 Pro 方案功能與更高額度。

要將開發租戶轉為 Pro 方案，請前往 <CloudLink to="/tenant-settings/settings">主控台 > 租戶設定 > 設定</CloudLink> 並點擊「轉換」。

### Pro 方案升級至企業方案 \{#pro-plan-upgrade-to-enterprise}

若 Pro 方案無法滿足需求，並希望獲得完整企業級支援，請 [聯絡我們](https://logto.io/contact) 以討論專屬企業方案。

### Pro 方案降級至免費方案 \{#pro-plan-downgrade-to-free-plan}

若你選擇停止使用 Pro 方案，可切換回免費方案，但需調整用量以符合免費方案額度。

## 未來價格方案調整 \{#future-pricing-plan-change}

為確保穩定且可預期的計費體驗，Logto 會在任何價格調整前提前通知。根據你的方案與用量，我們可能：1. 保留你現有方案，或 2. 提供寬限期讓你遷移至新方案。

Logto 將以公平且審慎的方式進行調整，並保留最終解釋權。

## 企業客戶價格考量 \{#enterprise-customer-pricing-considerations}

企業客戶的價格依合約而定。1. 合約外自助新增項目，依 Pro 方案附加功能與隨用隨付結構計價。2. 超出合約額度部分，依合約指定的附加功能單價計費。3. 合約未涵蓋的新功能，將依最新 Pro 方案價格計算。

如需詳細企業價格條款，請 [聯絡我們的企業銷售團隊](https://logto.io/contact)。

### 權杖 (Token) 用量與 MAU 有何關聯？ \{#what-is-the-connection-between-token-consumption-and-mau}

</summary>

Logto 採用 **權杖 (Token) 驗證 (Authentication)**，遵循 OAuth 2.0 與 OpenID Connect 標準。每次驗證 (Authentication) 或授權 (Authorization) 請求都會由 Logto 發出權杖，代表使用者的驗證 (Authentication) 或授權 (Authorization) 狀態，例如存取權杖 (Access token)、ID 權杖 (ID token) 與重新整理權杖 (Refresh token)。

計費時，**僅計算存取權杖 (Access tokens)**（ID 權杖 (ID tokens) 與重新整理權杖 (Refresh tokens) 不計入）。

**存取權杖 (Access token)** 會在下列情境發出並計入：

- **使用者登入**：使用者成功登入後，會發出一個不透明權杖 (Opaque token)。此權杖可用於呼叫 UserInfo 端點或 Logto Account API 進行帳號管理。
- **後端 API 授權 (RBAC)**：啟用基於角色的存取控制 (RBAC, Role-based access control) 保護後端 API 時，每次資源授權請求（在使用者初次驗證 (Authentication) 後）都會發出一個 JWT 存取權杖 (Access token)。詳見 [RBAC](https://docs.logto.io/authorization/role-based-access-control) 與 API [保護](https://docs.logto.io/authorization/global-api-resources)。
- **組織授權 (Organization authorization)**：啟用組織功能時，Logto 會發出組織權杖 (Organization token) 以提供組織層級存取控制。每次組織授權請求都會產生一個獨立的組織權杖 (Organization token)。詳見組織層級 [授權 (Authorization)](https://docs.logto.io/authorization/organization-permissions) 與 API [保護](https://docs.logto.io/authorization/organization-level-api-resources)。
- **機器對機器 (M2M) 授權 (Authorization)**：每次 `client_credentials` 授權請求（如存取 Logto Management API）都會發出一個存取權杖 (Access token)。
- **權杖交換**：個人權杖交換或使用者模擬 (User impersonation) 權杖交換時，會發出新的存取權杖 (Access token)。詳見 [個人權杖 (Personal access token)](https://docs.logto.io/user-management/personal-access-token) 與 [使用者模擬 (User impersonation)](https://docs.logto.io/developers/user-impersonation)。
- **重新整理權杖 (Refresh token) 使用**：若啟用 `offline_access`，每次有效重新整理權杖 (Refresh token) 兌換時，會發出新的存取權杖 (Access token)。

**權杖 (Token) vs. MAU 以及權杖 (Token) 計費的優勢**

MAU 計費模式按每月唯一使用者收費，不論活躍度。權杖 (Token) 計費則依實際用量收費：每次發出 **存取權杖 (Access token)** 都會計費。

權杖 (Token) 計費是真正的隨用隨付。不同於 MAU 模式按月收費，不論使用頻率，你只需為實際驗證 (Authentication) 與授權 (Authorization) 行為付費。這對低頻使用者特別友善，偶爾登入的用戶產生的權杖 (Token) 較少，費用也更低，遠比 MAU 固定收費模式划算。這讓 **Logto 更具成本效益**，尤其適合用戶活躍度差異大的應用。

例如，一次登入可能只消耗 1 個存取權杖 (Access token)。使用 M2M 或組織等功能會產生更多權杖 (Token)。為方便追蹤，Logto 在 <CloudLink to="/tenant-settings/subscription">主控台 > 設定 > 方案與計費</CloudLink> 提供 **即時權杖 (Token) 用量監控**，幫助你更好理解 MAU 與權杖 (Token) 在專案中的關係。
