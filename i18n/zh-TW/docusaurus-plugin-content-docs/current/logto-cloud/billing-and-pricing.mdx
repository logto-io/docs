# 計費與價格 (Billing and pricing)

在 Logto，我們的 [Pro 方案](https://logto.io/pricing) 完全自助，設計上強調透明，讓你能輕鬆理解並管理你的帳單。

## 瞭解結構 \{#understand-the-structure}

基本結構如下：

```math
\begin{align*}
\text{你的下期帳單} &= \$24 \text{（基本價格）} \\
&+ \text{附加項目費用（未結算用量）} \\
&+ \text{附加項目費用（下期週期收費）}
\end{align*}
```

在主控台的租戶設定區，你會看到「你的下期帳單」區塊，可以即時監控與管理即將產生的費用。

<img src="/img/assets/next-billing-info.png" alt="下期帳單資訊" />

## 項目明細 \{#item-breakdown}

請注意，只有少數功能屬於附加項目結構。

同時，Logto 也提供隨用隨付與按比例計價模式，確保你能方便地充分利用我們的服務。

|                                                | 包含額度                             | 超出額度費用                   |
| ---------------------------------------------- | ------------------------------------ | ------------------------------ |
| **權杖 (Tokens)**                              | 50K                                  | $0.08 / 100 / 月               |
| **機器對機器應用程式 (M2M apps)**              | 1                                    | 每月 $8 **_每個_**             |
| **API 資源 (API resources)**                   | 3                                    | 每月 $4 **_每個_**             |
| **企業級單一登入 (Enterprise SSO)**            | 0                                    | 每月 $48 **_每個_**            |
| **多重要素驗證 (Multi-factor authentication)** | 所有驗證因子                         | 每月 $48                       |
| **組織 (Organization)**                        | 無限組織與所有組織功能（含組織範本） | 每月 $48                       |
| **角色型存取控制 (Role-based access control)** | 無限角色與權限                       | 每月 $32                       |
| **SAML 應用程式 (SAML app)**                   | 0                                    | 每月 $96 **_每個_**            |
| **第三方應用程式 (Third party app)**           | 0                                    | 每月 $8 **_每個_**             |
| **進階安全組合 (Advanced security bundle)**    | 所有進階安全功能                     | 每月 $48                       |
| **多個自訂網域 (Multiple custom domains)**     | 1                                    | 每月 $48（最多 **10 個網域**） |
| **租戶成員 (Tenant members)**                  | 1                                    | 每月 $8 **_每個_**             |

考慮這些因素後，計算方式如下：

```math
\begin{align*}
\text{你的下期帳單} &= \$24 \\
&+ \text{附加項目單價} \times \\
\quad & (\text{新增數量} \times \text{建立時剩餘比例時間} \\
\quad &- \text{移除數量} \times \text{刪除時剩餘比例時間}) \\
\quad &+ \text{附加項目單價} \times \text{下期剩餘數量}
\end{align*}
```

1. **附加項目單價**：此附加項目的單價
2. **新增數量**：本期帳單週期內新增的附加項目數量
3. **移除數量**：本期帳單週期內移除的附加項目數量
4. **建立時剩餘比例時間**：建立資源時本期週期剩餘的比例時間
5. **刪除時剩餘比例時間**：刪除資源時本期週期剩餘的比例時間
6. **下期剩餘數量**：下期帳單週期需持續計費的數量

讓我們用兩個例子說明運作方式。

### 範例 1：無變動時的穩定帳單 \{#example-1-consistent-billing-with-no-changes}

若你沒有新增或移除項目，帳單將保持穩定。

例如，你使用 Pro 方案並啟用 2 個 SSO 服務，只要不變動，帳單將維持不變。

```math
\text{你的下期帳單} = \$ 24 + \$ 48 \times 2 = \$ 120
```

### 範例 2：新增附加項目或變動會影響帳單 \{#example-2-adding-add-ons-or-making-changes-will-affect-your-upcoming-bills}

根據不同情境，帳單會略有不同，但計算邏輯一致。

**情境 1：你訂閱 Pro 方案但未加購附加項目，僅測試企業級單一登入 (Enterprise SSO) 10 天。**

若你的訂閱每月 5 號續約，並於 20 號新增一個 Enterprise SSO，則本月使用 15 天。若 30 號刪除，實際僅用 10 天。

```math
\$24 + \$48 \times (1 \times 15 / 30) - \$48 \times (1 \times 5 / 30) = \$40
```

**情境 2：你訂閱 Pro 方案未加購附加項目，新增與移除 API 資源後，最終使用 5 個 API 資源**

本期帳單週期內你調整了附加項目，變動將反映於下期帳單。變動後的首月可能略高。帳單將包含 $24 基本費、未結算用量的附加項目費用，以及下期週期的完整收費。

你的帳單週期為每月 1 號起，已用完 3 個免費 API 資源。本月你進行了兩次操作：

1. 5 號新增 4 個資源。
2. 15 號刪除其中 2 個資源。

因此，下期帳單週期將持續有 2 個 API 資源。

```math
\$24 + \$8 \times (4 \times 25 - 2 \times 15) / 30 + 2 \times \$8 = \$58.67
```

### 附加項目用量時間如何計算？\{#how-is-the-add-on-usage-time-determined}

我們採用隨用隨付模式。你可能會好奇，若頻繁新增或移除資源會如何計算？附加項目僅在啟用期間計費。別擔心，我們的系統（由 Stripe 支援）會精確計算到秒，確保每項資源的實際使用時間都正確計費。

若你取消 Pro 方案，未用完的附加項目費用將退還，並從最終帳單中扣除。

## 升級或降級 \{#upgrade-or-downgrade}

### 免費方案升級至 Pro \{#free-plan-upgrade-to-pro}

以下情境建議你考慮從免費方案升級：

1. 你的 MAU 或權杖 (Token) 用量超過額度。
2. 你需要使用僅限 Pro 方案的功能。

### 開發租戶升級至 Pro \{#dev-tenant-upgrade-to-pro}

若你目前使用開發租戶，想轉為正式 Pro 方案，可輕鬆完成：

1. 保留你的成果：所有設定、應用程式與配置都會完整保留。
2. 立即啟用：轉換後立即享有 Pro 方案所有功能與更高額度。

要將開發租戶轉為 Pro 方案，請前往 <CloudLink to="/tenant-settings/settings">主控台 > 租戶設定 > 設定</CloudLink> 並點擊「轉換」。

### Pro 方案升級至企業方案 \{#pro-plan-upgrade-to-enterprise}

若 Pro 方案無法滿足需求，需企業級完整支援，請 [聯絡我們](https://logto.io/contact) 洽詢專屬企業方案。

### Pro 方案降級至免費方案 \{#pro-plan-downgrade-to-free-plan}

若你選擇停止使用 Pro 方案，可切換回免費方案，但需將用量調整至免費額度內。

## 未來價格方案調整 \{#future-pricing-plan-change}

為確保穩定且可預期的計費體驗，Logto 會在任何價格調整前提前通知。根據你的方案與用量，我們可能：1. 保留你現有方案（grandfather），或 2. 提供寬限期讓你遷移至新方案。

Logto 會以公平且審慎的方式進行調整，並保留最終解釋權。

## 企業客戶價格考量 \{#enterprise-customer-pricing-considerations}

企業客戶的價格以合約為準。1. 合約外自助新增項目依 Pro 方案附加項目與隨用隨付結構計價。2. 超出合約額度部分，依合約指定附加項目單價計費。3. 合約未涵蓋的新功能，將依最新 Pro 方案價格計算。

如需詳細企業價格條款，請 [聯絡我們的企業銷售團隊](https://logto.io/contact)。

<summary>

### 權杖 (Token) 用量與 MAU 有何關聯？\{#what-is-the-connection-between-token-consumption-and-mau}

</summary>

Logto 採用符合 OAuth 2.0 與 OpenID Connect 標準的 **權杖型驗證 (token-based authentication)**。每次驗證 (Authentication) 或授權 (Authorization) 請求，Logto 都會簽發權杖，代表使用者的驗證或授權狀態，例如存取權杖 (Access token)、ID 權杖 (ID token) 與重新整理權杖 (Refresh token)。

在計費上，**僅計算存取權杖 (Access token)**（ID 權杖與重新整理權杖不計入）。

**存取權杖 (Access token)** 在下列情境會被簽發並計入：

- **使用者登入**：使用者成功登入後，會簽發一個不透明權杖 (Opaque token)。此權杖可用於呼叫 UserInfo 端點或 Logto Account API 進行帳號管理。
- **後端 API 授權（RBAC）**：啟用角色型存取控制 (RBAC) 保護後端 API 時，每次資源專屬授權請求（在使用者初次驗證後）都會簽發一個 JWT 存取權杖。詳見 [RBAC](https://docs.logto.io/authorization/role-based-access-control) 與 API [保護](https://docs.logto.io/authorization/global-api-resources)。
- **組織授權**：啟用組織功能後，Logto 會簽發組織權杖 (Organization token) 以實現組織層級存取控制。每次組織專屬授權請求都會產生一個組織權杖。詳見組織層級 [授權](https://docs.logto.io/authorization/organization-permissions) 與 API [保護](https://docs.logto.io/authorization/organization-level-api-resources)。
- **機器對機器 (M2M) 授權**：每次 `client_credentials` 授權請求（如存取 Logto Management API）都會簽發一個存取權杖。
- **權杖交換**：個人權杖交換或使用者模擬權杖交換時會簽發新的存取權杖。詳見 [個人權杖 (personal access token)](https://docs.logto.io/user-management/personal-access-token) 與 [使用者模擬 (user impersonation)](https://docs.logto.io/developers/user-impersonation)。
- **重新整理權杖 (Refresh token) 使用**：若啟用 `offline_access`，每次有效重新整理權杖兌換後會簽發新的存取權杖。

**權杖 (Token) vs. MAU 以及權杖計費的優勢**

MAU 計費模式按每月唯一使用者收費，無論其實際活動。權杖計費則依實際用量收費：每次簽發 **存取權杖 (Access token)** 都會計費。

權杖計費是真正的隨用隨付。不同於 MAU 模式按月收費，不論使用頻率，你只需為實際驗證 (Authentication) 與授權 (Authorization) 行為付費。這對低頻使用者特別友善，偶爾登入的使用者產生的權杖較少，總費用也較低，讓 **Logto 更具成本效益**，尤其適合用戶活躍度不均的應用。

例如，一次登入可能只消耗 1 個存取權杖。使用 M2M 或組織等功能會產生更多權杖。為方便追蹤，Logto 在 <CloudLink to="/tenant-settings/subscription">主控台 > 設定 > 方案與計費</CloudLink> 提供 **即時權杖用量監控**，協助你更好理解 MAU 與權杖在專案中的關係。
