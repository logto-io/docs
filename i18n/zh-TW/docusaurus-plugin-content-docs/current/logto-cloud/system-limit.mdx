# 系統限制

系統限制可確保 Logto 對所有人都穩定且公平。這些限制與計費無關。

**重點整理**

- **Dev 租戶**：固定上限（硬性限制）
- **Pro 租戶**：起始上限（軟性限制），如有正當需求可提高
- **Enterprise 租戶**：可自訂，於合約中約定

在定價頁面上，**無上限** 意指 _在合理使用下無預設計費上限_。為保護平台穩定性，我們會設置**初始防護措施**（如實體上限與速率限制），如有正當需求可提高。

## 限制類型 \{#types-of-limits}

### 系統資源與實體上限 \{#system-resource-and-entity-caps}

對應用程式、角色、組織、連接器、Webhook 及某些依用量計算的系統資源（如存取權杖 (Access token) 發行數量）等物件的最大數量限制。

### 速率限制 \{#rate-limit}

每個租戶層級的請求保護。

預設：**每個租戶每 10 秒 200 次請求**（token-bucket）。約等於**每秒 20 次持續請求**，允許短暫突發至桶容量。

若你的流量為正當用途且持續超過此限制，我們可評估例外處理。

## 達到限制時會發生什麼 \{#what-happens-when-you-hit-a-limit}

當觸發防護措施時，請求可能會因以下錯誤而失敗：

- **HTTP 429**（速率限制）
- **HTTP 4xx** 並帶有與限制相關的訊息（實體上限或功能防護）

這不會刪除資料，只會阻擋超出限制的操作。

## 如何申請提高限制 \{#request-a-limit-increase}

若你達到限制且用例屬實，請聯絡我們。對於 Pro 與 Enterprise，我們通常能快速提高限制。

請提供：

- 租戶 ID（或租戶網域）
- 達到的限制（功能名稱 + 你需要的數字）
- （若為 RPS 需求）預期流量：平均 RPS、峰值 RPS、突發模式
- 受影響的端點（如 sign-in、token、JWKS、userinfo、Management API）
- 時間需求（何時需要提高）

[聯絡我們](https://logto.io/contact)
