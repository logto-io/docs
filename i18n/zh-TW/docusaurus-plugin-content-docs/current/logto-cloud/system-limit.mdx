# 系統限制 (System limit)

系統限制 (System limits) 定義了每個租戶 (tenant) 可建立的實體最大數量。這些限制用於保護平台穩定性，並且與計費無關。部分定價頁面上的項目標示為 _無上限 (unlimited)_ 或 _按用量計費 (pay-as-you-go)_，這表示沒有計費上限，但為確保公平使用，內部系統上限仍然適用。如有需要，我們可逐案評估是否提升上限。

## 限制類型 \{#types-of-limits}

### 系統資源與實體上限 \{#system-resource-and-entity-caps}

針對應用程式、角色 (Roles)、組織 (Organizations)、連接器 (Connectors) 等物件，以及某些依用量計算的系統資源（如權杖總消耗量）的最大數量。這些限制會依方案而異。

### 請求速率限制 (Rate limit) \{#rate-limit}

每個租戶層級的請求保護。一般指引為**每個租戶每 10 秒 200 次請求**。若你的流量屬於正常業務且持續超過此上限，我們可逐案評估是否放寬。

---

如果你的使用情境合理且遇到限制，請聯絡我們。這有助於我們調整限制並支援真實世界的需求。

## 租戶層級限制保護 \{#tenant-level-rate-limit-protection}

### 開發租戶 (Dev tenants) \{#dev-tenants}

[開發租戶 (Dev tenants)](/logto-cloud/tenant-settings#development) 提供免費測試環境。為防止濫用並明確預期，開發租戶有固定的實體上限。如需更高上限，可申請提升或升級至 Pro 方案以獲得完整存取權。

開發租戶亦遵循獨立的 [資料保留政策 (data retention policy)](/logto-cloud/dev-tenant-data-retention)。
