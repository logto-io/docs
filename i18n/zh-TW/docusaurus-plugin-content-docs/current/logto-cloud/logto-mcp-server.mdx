# Logto MCP Server

Logto MCP Server 是一個遠端 [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) 伺服器。將其 URL 加入你的 AI 應用程式，使用 Logto Cloud 帳號登入，即可開始與 Logto 對話。

:::tip
Logto MCP Server 是以 Logto 與 [MCP Auth](https://mcp-auth.dev) 建構而成。如果你想自行打造遠端 MCP 伺服器，請參閱[打造你自己的遠端 MCP 伺服器](#build-your-own-remote-mcp-server)。
:::

## 你可以做什麼 \{#what-you-can-do}

- **為你的專案新增驗證 (Authentication)**：AI 會偵測你的框架，自動建立 Logto 應用程式並產生可用的整合程式碼。支援超過 38 種框架，包括 React、Next.js、Vue、Nuxt、SvelteKit、Express、Go、Python、iOS、Android、Flutter 等。
- **詢問 Logto 相關問題**：AI 會搜尋 Logto 文件並提供帶有來源的答案。

更多功能即將推出，包括登入體驗設定、RBAC 與權限管理最佳實踐指引、多租戶組織架構（B2B 應用）設定等。

## 在你的 AI 應用程式中設定 \{#set-up-in-your-ai-application}

### Claude Desktop \{#claude-desktop}

1. 從主選單開啟 **Settings**，前往 **Connectors**，點擊 **Add custom connector**。
2. 輸入名稱 `Logto MCP Server`，並填入遠端 MCP 伺服器 URL：`https://mcp.logto.io`。
3. 展開 **Advanced settings**，輸入 OAuth Client ID：`mcp-client`。點擊 **Add**。
4. 在 **Connectors** 清單中找到 **Logto MCP Server** 並點擊 **Connect**。
5. 你會被導向 Logto Cloud 登入頁面。登入並授權以完成連線。

### Cursor \{#cursor}

1. 開啟指令選單（`Command + Shift + P`），選擇 **View: Open MCP Settings**。
2. 在 **Installed MCP Servers** 區塊，點擊 **Add Custom MCP** 按鈕。
3. 在開啟的 `mcp.json` 檔案中，加入以下設定：

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud 登入頁面會自動顯示。請使用 Logto Cloud 帳號登入以完成連線。

### OpenCode \{#opencode}

1. 在你的 `opencode.json` 檔案中加入以下設定：

   ```json
   {
     "$schema": "https://opencode.ai/config.json",
     "mcp": {
       "Logto": {
         "enabled": true,
         "type": "remote",
         "url": "https://mcp.logto.io",
         "oauth": {
           "clientId": "mcp-client"
         }
       }
     }
   }
   ```

2. 執行以下指令進行驗證：
   ```bash
   opencode mcp auth Logto
   ```
   注意 `Logto` 對應到你設定檔中的 MCP 伺服器名稱。

瀏覽器視窗會自動開啟。請使用 Logto Cloud 帳號登入以授權並完成連線。

### VS Code \{#vs-code}

1. 開啟指令選單（`Command + Shift + P`），選擇 **MCP: Add Server...**。
2. 選擇 **HTTP** 作為伺服器類型。
3. 輸入 URL：`https://mcp.logto.io`。你可以選擇安裝到工作區以獲得更好的隔離與管理。
4. 當 OAuth 流程啟動時，輸入 App ID：`mcp-client`。App Secret 保持空白並直接按 Enter 跳過。
5. 你會被導向 Logto Cloud 登入頁面。請使用 Logto Cloud 帳號登入以完成連線。

### Windsurf \{#windsurf}

1. 開啟指令選單（`Command + Shift + P`），選擇 **Windsurf: MCP Marketplace**。
2. 在 **Installed MCPs** 區塊，點擊右側設定圖示以建立並開啟 `~/.codeium/windsurf/mcp_config.json` 檔案。
3. 加入以下[設定](https://docs.windsurf.com/windsurf/cascade/mcp#mcp_config-json)：

   ```json
   {
     "mcpServers": {
       "Logto": {
         "command": "npx",
         "args": [
           "mcp-remote",
           "https://mcp.logto.io",
           "--allow-http",
           "--static-oauth-client-info",
           "{ \"client_id\": \"mcp-client\", \"scope\": \"mcp:all\" }",
           "--resource",
           "https://mcp.logto.io/"
         ]
       }
     }
   }
   ```

Logto Cloud 登入頁面會自動顯示。請使用 Logto Cloud 帳號登入以完成連線。

### 其他 MCP 用戶端 \{#other-mcp-clients}

Logto MCP Server 可搭配任何支援 OAuth 授權的 MCP 用戶端。請使用以下設定：

- **MCP 伺服器 URL：** `https://mcp.logto.io`
- **OAuth client ID：** `mcp-client`
- **Resource：** `https://mcp.logto.io/`（如你的用戶端需要）
- **Scope：** `mcp:all`（如你的用戶端需要）

如果你是 AI 應用程式開發者，想支援連接 Logto MCP Server，但 Logto MCP Server 尚未支援你的應用程式 redirect URI，請[聯絡我們](https://logto.io/contact)並提供你的應用程式資訊。

## 打造你自己的遠端 MCP 伺服器 \{#build-your-own-remote-mcp-server}

Logto MCP Server 是以 Logto 與 [MCP Auth](https://mcp-auth.dev) 建構而成。如果你想打造具備驗證 (Authentication) 的遠端 MCP 伺服器，請參考以下資源：

- [Remote MCP server in action: a new entry point for SaaS products in the AI era](https://blog.logto.io/remote-mcp-server-in-action)
- [加入我們的 Discord](https://discord.gg/fUQmaFyAam)：分享打造遠端 MCP 伺服器的經驗與最佳實踐
- [MCP Auth](https://mcp-auth.dev)：一站式為你的 MCP 伺服器加入生產等級驗證 (Authentication)，無需花費數週閱讀規範或手動串接。
