# 開發租戶資料保留政策 (Dev tenant data retention policy)

Logto Cloud 遵循明確原則：**生產環境用於持久保存，開發環境用於測試**。

- 在**生產租戶**（Free、Pro 與 Enterprise）中，物件與實體會在帳號存續期間完整保留，除非你自行移除或另有要求。
- 在**開發租戶**中，目標則不同。這些租戶是**安全的測試沙盒**，重點在於安全性、效能與乾淨的環境，而非長期儲存資料。

為了維護此原則，Logto 在開發租戶中實施 90 天保留政策：**開發租戶中的使用者**會在 90 天後自動刪除。

在開發環境中，開發者整合時常會建立許多測試帳號。這些帳號產生大量資料，但很少有長期價值。90 天政策確保 Logto 平台資源有效利用，讓測試環境保持乾淨與高效，避免未使用帳號堆積，並明確區分暫時測試資料與長期生產資料。

## 了解 90 天政策 \{#understanding-the-90-day-policy}

### 只會刪除超過 90 天的使用者帳號 \{#only-user-accounts-older-than-90-days-are-deleted}

刪除使用者即移除所有個人資料、驗證資料及相關資訊。

### 保持不變的內容 \{#what-stays-intact}

你的開發工作與設定不受影響。這讓你未來能輕鬆將開發與整合設定遷移到生產環境。我們提供直接[將開發租戶轉換為生產租戶](./tenant-settings#tenant-types-dev-vs-prod)的選項。

## 自動刪除使用者對你的意義 \{#what-automatic-user-deletion-means-for-you}

- 隱私保護：測試帳號在開發期間常被團隊共用或長期閒置。我們建議每 90 天重新整理一次。
- 資源效率：自動刪除釋放儲存與運算資源，讓環境保持乾淨與高效。
- 環境明確分離：開發租戶是暫時沙盒，長期持久資料應存於生產租戶。

## 開發租戶最佳實踐 \{#best-practices-for-dev-tenants}

### 開發期間 \{#during-development}

- 將 Logto 開發租戶作為你的測試沙盒。
- 記錄重要測試情境，方便日後重建。
- 仔細規劃開發生命週期，適時遷移至生產環境。

### 長期專案 \{#for-long-term-projects}

- 需要時建立新測試使用者或帳號，避免保留舊帳號。
- 若需具持久資料的預備環境，請使用生產租戶。

### 我可以延長 90 天保留期限嗎？ \{#can-i-extend-the-90-day-retention-period}

</summary>

無法。90 天政策適用於所有開發租戶，以維持一致性與安全性。如有特殊需求，請聯繫我們取得協助。

</details>

<details>
<summary>

### 這會影響生產租戶嗎？ \{#does-this-affect-production-tenants}

</summary>

不會。所有生產租戶（包含 Free、Pro 與 Enterprise 方案）的資料（包含使用者與設定）皆不受影響。

</details>

<details>
<summary>

### 企業客戶的私有實例下開發租戶會被刪除嗎？ \{#for-enterprise-customers-will-dev-tenants-under-private-instances-be-deleted}

不會。私有實例中的開發租戶不受 90 天刪除政策影響。

</details>

## 摘要 \{#summary}

1. 開發租戶中的使用者會在 90 天後自動清除
2. 你的所有設定、應用程式與配置皆會永久保留
3. 此政策確保安全、效能與乾淨的測試條件
4. 生產租戶對持久資料有不同的保留政策
5. 請依據此自動清理週期規劃你的開發流程

此保留政策確保你的開發環境始終是安全、高效且可靠的測試場域，同時保護使用者隱私並維持系統效能。

如需持久使用者資料儲存，請考慮升級為生產租戶，讓資料保留符合你組織的特定需求。
