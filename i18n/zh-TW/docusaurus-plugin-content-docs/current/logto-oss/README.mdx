# Logto OSS

Logto 提供開源（OSS）與 [雲端](/logto-cloud) 版本。本節將聚焦於 OSS 解決方案：<Url type="inline" href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS 是免費使用的身分解決方案，讓開發者擁有完整自訂能力與資料完全掌控權。Logto OSS 提供靈活性，可根據特定需求調整程式碼基礎，非常適合重視資料主權並希望擁有可隨應用成長的專屬身分提供者解決方案的你。

## Logto OSS 支援的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支援 Logto 服務的大多數核心功能，並持續定期更新與維護。

部分進階功能目前僅限 Logto Cloud 版本，包括：

| OSS 功能限制                                                                  | 說明                                                                         |
| ----------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| [Logto 控制台：多租戶](/logto-cloud/tenant-settings)                          | 在控制台中建立與管理多個 Logto 租戶。                                        |
| [Logto 控制台：協作者邀請](/logto-cloud/tenant-member-management)             | 邀請並管理單一 Logto 租戶的多位成員。                                        |
| Logto 控制台：啟用 MFA                                                        | 登入 Logto 控制台時強制多重要素驗證 (MFA)，提升安全性。                      |
| [Logto Protected App](/integrate-logto/protected-app)                         | 由 Cloudflare 驅動的快速、無需 SDK 的驗證 (Authentication) 整合。            |
| [Logto 內建電子郵件服務](/connectors/email-connectors/built-in-email-service) | 使用內建且免費的電子郵件服務發送郵件。                                       |
| [自帶 UI](/customization/bring-your-ui)                                       | 使用此功能自訂登入體驗的 UI 或流程（OSS 可直接在 GitHub 分支自訂登入體驗）。 |
| [IdP 發起 SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)              | 啟用身分提供者 (IdP) 發起的企業級單一登入 (SSO)。                            |
| [SAML 應用程式](/integrate-logto/saml-app)                                    | 透過 SAML 協定整合應用程式。OSS 版本僅限 3 個 SAML 應用程式。                |

提示：多租戶、成員邀請與 MFA 無法用於你的團隊登入開源 Logto 控制台。但你可在自己的產品中利用 Logto OSS 實作這些功能，讓終端使用者使用。
