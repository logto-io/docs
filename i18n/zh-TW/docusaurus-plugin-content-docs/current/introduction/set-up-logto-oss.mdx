# 設定 Logto OSS

本節涵蓋 [Logto 開源服務 (OSS)](https://github.com/logto-io/logto) 的基本設定步驟與關鍵操作，協助你有效交付產品並管理開發流程。

## 開始使用 Logto OSS \{#get-started-with-logto-oss}

請依照 [快速開始指南](/logto-oss/get-started-with-oss/) 立即啟動 Logto，日後可參考完整的 [部署指南](/logto-oss/deployment-and-configuration) 進行正式環境部署。

## Logto OSS 支援的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支援 Logto 服務的大多數核心功能，並持續定期更新與維護。

部分進階功能目前僅限於 Logto Cloud 版本，包括：

| OSS 功能限制                                                                  | 說明                                                                                  |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| [Logto 控制台：多租戶](/logto-cloud/tenant-settings)                          | 在控制台中建立並管理多個 Logto 租戶。                                                 |
| [Logto 控制台：協作者邀請](/logto-cloud/tenant-member-management)             | 邀請並管理單一 Logto 租戶的多位成員。                                                 |
| Logto 控制台：啟用 MFA                                                        | 登入 Logto 控制台時強制多重要素驗證 (MFA, Multi-factor authentication) 以提升安全性。 |
| [Logto Protected App](/integrate-logto/protected-app)                         | 由 Cloudflare 提供的快速、無需 SDK 的驗證 (Authentication) 整合。                     |
| [Logto 內建電子郵件服務](/connectors/email-connectors/built-in-email-service) | 使用內建且免費的電子郵件服務進行郵件發送。                                            |
| [自訂 UI](/customization/bring-your-ui)                                       | 使用此功能自訂登入體驗的 UI 或流程（OSS 可直接在 GitHub 分支自訂登入體驗程式碼）。    |
| [IdP 發起的單一登入 (SSO)](/end-user-flows/enterprise-sso/idp-initiated-sso)  | 啟用身分提供者 (IdP, Identity provider) 發起的企業級單一登入 (SSO, Single Sign-On)。  |
| [SAML 應用程式](/integrate-logto/saml-app)                                    | 透過 SAML 協定整合應用程式。OSS 版本僅限 3 個 SAML 應用程式。                         |

提示：多租戶、成員邀請與 MFA 無法用於你的團隊登入開源版 Logto 控制台。但你可以在自己的產品中利用 Logto OSS 實作這些功能，讓最終使用者享有這些能力。

## 追蹤 Logto 發布動態 \{#stay-updated-with-logto-releases}

為確保你的 Logto 實例隨時擁有最新功能，請關注 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 頁面，所有版本日誌都可在此查閱。

閱讀 [升級指南](/logto-oss/upgrading-oss-version) 以瞭解如何在不更動程式碼或資料庫結構的情況下升級 Logto。
