# 設定 Logto Cloud

本節涵蓋基本設定步驟與關鍵操作，協助你有效交付產品並管理開發流程。

## 建立 Logto 租戶 (tenant) \{#create-logto-tenant}

首先，註冊一個商業帳號並建立 Logto 租戶 (tenant)。Logto Cloud 的租戶是一個隔離的環境，你可以在其中管理使用者身分、應用程式及所有其他 Logto 資源。

你的第一個租戶會自動設為[**開發 (Development)**](/logto-cloud/tenant-settings#development) 環境。這非常適合測試與開發用途，且可免費使用。

請查看租戶設定，並[深入瞭解](/logto-cloud) Logto 雲端服務。

## 邀請協作者 \{#invite-collaborators}

在導入流程中，你有機會邀請協作者共同進行開發。如果你想稍後再邀請或邀請更多人，請前往 <CloudLink to="/tenant-settings/members">租戶設定 > 成員</CloudLink>。你會看到目前的成員，並可邀請更多人。輸入電子郵件地址並指派角色（可批次操作），然後發送邀請。

## 移轉至正式環境租戶 \{#move-to-production-tenant}

你可能會花些時間開發與測試 Logto 的功能，組成概念驗證（POC）。

當你的專案完成測試並準備進入下一階段時，請記得建立新的[正式型租戶 (production-type tenant)](/logto-cloud/tenant-settings#production)（需擁有 [Free 或 Pro 租戶](https://logto.io/pricing)）。因為開發租戶 (dev tenant) 並非設計用於正式環境，且有[限制與約束](/logto-cloud/tenant-settings#development)。

:::note

建立正式租戶後，請務必盡快設定[自訂網域 (custom domain)](/logto-cloud/custom-domain)，這將影響你後續的設定流程。

:::

這樣的設定讓你的專案同時擁有開發測試與正式環境，有助於順利管理開發流程。

## 從現有系統遷移 \{#migrate-from-the-existing-system}

如果你正在維護現有專案並需要切換驗證 (Authentication) 提供者，Logto 支援從其他平台遷移使用者。你可以遷移[基本資料](/user-management/user-data#basic-data)、[自訂資料](/user-management/user-data#custom-data)、[社交身分](/user-management/user-data#social-identities)及密碼雜湊。詳情請參閱 [遷移至 Logto](../user-management/user-migration.mdx) 指南。

## 搭配 AI 工具使用 Logto MCP Server \{#use-logto-mcp-server-with-ai-tools}

如果你使用 VS Code 搭配 Copilot、Cursor 或 Claude Desktop 等 AI 開發工具，可以連接 [Logto MCP Server](/logto-cloud/logto-mcp-server)，直接從 AI 助理與 Logto 互動。AI 能自動偵測你的框架、建立應用程式，並為你產生可用的整合程式碼。
