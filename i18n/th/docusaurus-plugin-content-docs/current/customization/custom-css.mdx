# CSS แบบกำหนดเอง

หาก [การปรับแต่ง UI อย่างละเอียด](/customization/match-your-brand) ของเราไม่ตรงกับแบรนด์ของคุณทั้งหมด คุณสามารถใช้ CSS แบบกำหนดเองเพื่อปรับแต่งประสบการณ์การลงชื่อเข้าใช้ของคุณได้ เพียงเพิ่มโค้ด CSS ของคุณลงในตัวแก้ไขของเราและดูผลลัพธ์ที่เปลี่ยนแปลงได้ทันที

## CSS แบบกำหนดเองใน Logto Console \{#custom-css-in-logto-console}

1. ไปที่ <CloudLink to="/sign-in-experience/branding">Console > ประสบการณ์การลงชื่อเข้าใช้ > การตั้งค่าแบรนด์ > CSS แบบกำหนดเอง</CloudLink>
2. แก้ไขโค้ด CSS ในตัวแก้ไขทางซ้าย ผลลัพธ์จะปรากฏทันทีในหน้าต่างแสดงตัวอย่างทางขวา หมายเหตุ: ตัวแก้ไขโค้ดรองรับเฉพาะโค้ด CSS เท่านั้น ยังไม่รองรับ HTML หรือ Javascript
3. บันทึกการเปลี่ยนแปลง
4. อย่าลืมคลิกปุ่ม “[แสดงตัวอย่างสด](/customization/live-preview)” เพื่อตรวจสอบการเปลี่ยนแปลงในทุกหน้า

หากต้องการดูการออกแบบอินเทอร์เฟซแบบแยกส่วนของ Logto คุณสามารถไปที่ [Logto/packages/experience/src](https://github.com/logto-io/logto/tree/master/packages/experience/src) บน GitHub หรือเปิดเครื่องมือ "Inspect Elements" ของเบราว์เซอร์ในหน้าตัวอย่างสดของ Logto

## ตัวอย่าง \{#examples}

ตัวอย่างเช่น หากคุณต้องการให้หน้าลงชื่อเข้าใช้ของคุณมีบรรยากาศแบบ _Night City_ สำหรับมุมมองเว็บ ลองใช้ CSS นี้:

:::note

เนื่องจาก Logto ใช้ CSS Modules คุณอาจเห็นค่าแฮชใน property `class` ของ element ใน DOM (เช่น `<div>` ที่มี `vUugRG_container`) หากต้องการ override เหล่านี้ คุณสามารถใช้ selector `$=` ใน CSS เพื่อเลือก element ที่ลงท้ายด้วยค่าที่ระบุ เช่น `div[class$=container]`

:::

## CSS แบบกำหนดเองผ่าน Management API \{#custom-css-by-management-api}

คุณยังสามารถใช้ [Management API](https://openapi.logto.io/operation/operation-uploadcustomuiassets) `PATCH /api/sign-in-exp` พร้อม body `{ "customCss": "arbitrary string" }` เพื่อกำหนด CSS สำหรับประสบการณ์การลงชื่อเข้าใช้แบบกำหนดเองได้ คุณจะเห็นค่าของ `customCss` ถูกแนบหลัง `<title>` ของหน้า หาก style มีลำดับความสำคัญสูงกว่า ก็จะ override ได้
