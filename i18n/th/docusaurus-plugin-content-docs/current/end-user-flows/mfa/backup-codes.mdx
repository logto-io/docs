# รหัสสำรอง (Backup codes)

## แนวคิด \{#concepts}

รหัสสำรอง (Backup codes) หรือที่เรียกว่า รหัสกู้คืน (Recovery code) คือรหัสใช้ครั้งเดียวสำหรับการยืนยันตัวตนหลายปัจจัย (MFA) ทำหน้าที่เป็นตัวสำรองในกรณีที่ปัจจัยหลักในการยืนยันตัวตนของผู้ใช้ (เช่น แอปยืนยันตัวตน หรือโทเค็นฮาร์ดแวร์) ไม่สามารถใช้งานได้

หากสูญเสียรหัสเหล่านี้ อาจทำให้การกู้คืนบัญชีเป็นเรื่องยาก ดังนั้นจึงแนะนำให้ตั้งค่าปัจจัยหลักเพิ่มเติมก่อนเปิดใช้งานรหัสสำรอง (Backup Codes) และให้ความสำคัญกับปัจจัยหลักนั้น

Logto จะสร้างรหัสสำรอง (Backup Codes) จำนวน 10 รหัสโดยอัตโนมัติให้กับผู้ใช้เมื่อมีการตั้งค่าปัจจัยเพิ่มเติมแต่ละรายการ โดยแต่ละรหัสสามารถใช้ได้เพียงครั้งเดียว ผู้ใช้ควรสร้างรหัสชุดใหม่ในหน้าการตั้งค่าบัญชีผู้ใช้ (เข้าถึงได้ผ่าน [Management API](/integrate-logto/interact-with-management-api/)) ก่อนที่จะใช้รหัสชุดเดิมหมด
