# Passkeys (WebAuthn)

[Passkey](https://auth.wiki/passkey) มอบทางเลือกที่ปลอดภัยและเป็นมิตรกับผู้ใช้มากกว่ารหัสผ่านแบบเดิม ด้วยการใช้การเข้ารหัสแบบกุญแจสาธารณะ (public-key cryptography) passkey จะเพิ่มความปลอดภัยโดยเชื่อมโยงอุปกรณ์ของผู้ใช้ โดเมนของบริการ และรหัสผู้ใช้ (user ID) ซึ่งช่วยป้องกันการโจมตีแบบฟิชชิ่งและรหัสผ่าน Passkey สามารถใช้งานร่วมกับอุปกรณ์หรือเบราว์เซอร์ต่าง ๆ และอนุญาตให้ผู้ใช้ใช้ไบโอเมตริกซ์และฟีเจอร์ความปลอดภัยของฮาร์ดแวร์เพื่อความสะดวกในการยืนยันตัวตน [WebAuthn](https://auth.wiki/webauthn) ให้ API สำหรับเว็บไซต์ในการใช้งาน passkey

ขณะนี้ Logto รองรับ passkey (WebAuthn) สำหรับการยืนยันตัวตนหลายปัจจัย (MFA) ฟีเจอร์ลงชื่อเข้าใช้ด้วย passkey กำลังจะมาเร็ว ๆ นี้ โปรดติดตามข่าวสารอัปเดต

## แนวคิด \{#concepts}

ลูกค้ามักจะรู้จัก Passkeys มากกว่า WebAuthn แล้วความสัมพันธ์ระหว่างสองสิ่งนี้คืออะไร และควรใช้อย่างไร? มาสำรวจแนวคิดเหล่านี้กัน:

- **Passkeys**: Passkey คือข้อมูลรับรอง (credential) ที่ใช้มาตรฐาน FIDO และต้านทานฟิชชิ่ง เพื่อทดแทนรหัสผ่าน โดยใช้การเข้ารหัสแบบกุญแจสาธารณะ-ส่วนตัว (asymmetric public-key cryptography) เพื่อความปลอดภัยที่สูงขึ้น อาจเป็นฮาร์ดแวร์โทเค็นหรือกุญแจความปลอดภัย เช่น อุปกรณ์ USB หรือ Bluetooth เนื่องจาก “Passkeys” เป็นวิธีการยืนยันตัวตนที่แสดงต่อผู้ใช้ จึงควรใช้คำนี้ในฝั่งไคลเอนต์ของผลิตภัณฑ์คุณ
- **WebAuthn**: เป็น JavaScript API ที่พัฒนาโดย W3C และ FIDO Alliance เพื่อให้เว็บแอปพลิเคชันสามารถยืนยันตัวตนตามมาตรฐาน FIDO2 ได้ Passkeys เป็นหนึ่งในวิธีการยืนยันตัวตนที่ WebAuthn รองรับ ใน Logto Console เราเรียกการเชื่อมต่อนี้อย่างเป็นทางการว่า “WebAuthn”

WebAuthn มีตัวเลือกตัวตรวจสอบความถูกต้อง (authenticator) หลากหลายให้ผู้ใช้เลือก โดยแบ่งเป็น 2 ประเภทสำหรับการใช้งานในเครื่องและบนคลาวด์:

- **Platform authenticator (ตัวตรวจสอบความถูกต้องภายในเครื่อง)**: ผูกกับระบบปฏิบัติการของอุปกรณ์เดียว เช่น คอมพิวเตอร์ แล็ปท็อป โทรศัพท์ หรือแท็บเล็ตที่ผู้ใช้ลงชื่อเข้าใช้ ใช้งานได้เฉพาะบนอุปกรณ์นั้น ๆ โดยใช้ไบโอเมตริกซ์หรือรหัสผ่านอุปกรณ์ เช่น iCloud Keychain ที่ยืนยันด้วย Touch ID, Face ID หรือรหัสผ่านอุปกรณ์บน macOS หรือ iOS; Windows Hello ที่ยืนยันด้วยการจดจำใบหน้า ลายนิ้วมือ หรือ PIN
- **Roaming authenticator (ตัวตรวจสอบความถูกต้องแบบพกพา, ข้ามแพลตฟอร์ม)**: เป็นอุปกรณ์หรือแอปพลิเคชันแยกต่างหาก เช่น กุญแจความปลอดภัยฮาร์ดแวร์ หรือสมาร์ทโฟน ต้องเชื่อมต่ออุปกรณ์ด้วย USB หรือเปิด NFC / Bluetooth ตัวตรวจสอบความถูกต้องแบบพกพาไม่จำกัดเฉพาะอุปกรณ์หรือเบราว์เซอร์เดียว ให้ความยืดหยุ่นมากกว่า

หากต้องการเจาะลึกหลักการและกระบวนการของ WebAuthn สามารถอ่านได้ที่บล็อกของเรา: [WebAuthn and Passkeys 101](https://blog.logto.io/web-authn-and-passkey-101/) และ [สิ่งที่ควรรู้ก่อนเชื่อมต่อ WebAuthn](https://blog.logto.io/webauthn-base-knowledge/)

## ให้ความสำคัญกับข้อจำกัด \{#pay-attention-to-limitations}

ควรตระหนักถึงข้อจำกัดบางประการเมื่อใช้งาน WebAuthn:

1. **ข้อจำกัดของแพลตฟอร์มและเบราว์เซอร์**: ปัจจุบัน Logto ยังไม่รองรับ WebAuthn สำหรับแอปพลิเคชันแบบเนทีฟ นอกจากนี้ ความพร้อมใช้งานของตัวตรวจสอบความถูกต้อง WebAuthn ขึ้นอยู่กับเบราว์เซอร์และอุปกรณ์ ([ตรวจสอบรายการ](https://caniuse.com/?search=webauthn)) ดังนั้น WebAuthn จึงไม่ใช่ตัวเลือกเดียวสำหรับการใช้งานการยืนยันตัวตนหลายปัจจัย (MFA) เสมอไป มิฉะนั้นคุณสามารถควบคุมได้ว่าเบราว์เซอร์และอุปกรณ์ใดเข้าถึงผลิตภัณฑ์ของคุณได้บ้าง
2. **ข้อจำกัดของโดเมน**: การเปลี่ยนโดเมนอาจทำให้ผู้ใช้ไม่สามารถยืนยันตัวตนผ่านบัญชี WebAuthn เดิมได้ Passkeys จะผูกกับโดเมนของหน้าเว็บปัจจุบันโดยเฉพาะ และไม่สามารถใช้ข้ามโดเมนได้
3. **ข้อจำกัดของอุปกรณ์**: หากผู้ใช้ทำอุปกรณ์หาย อาจสูญเสียการเข้าถึงบัญชี โดยเฉพาะผู้ที่ใช้ Platform Authenticator แบบ “อุปกรณ์นี้” เพียงอย่างเดียว เพื่อเพิ่มความปลอดภัยในการเข้าถึง ควรให้ผู้ใช้มีปัจจัยการยืนยันตัวตนมากกว่าหนึ่งอย่าง
