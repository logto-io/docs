# การปกป้อง API

เรียนรู้วิธีการนำ [การควบคุมการเข้าถึงตามบทบาท (RBAC)](/authorization/role-based-access-control) และ [JSON Web Token (JWT)](https://auth.wiki/jwt) มาใช้เพื่อตรวจสอบและปกป้องจุดเชื่อมต่อ API ของคุณจากการเข้าถึงโดยไม่ได้รับอนุญาต

คู่มือของเราครอบคลุมการตั้งค่า middleware, โมเดลสิทธิ์ (permission models) และตัวอย่างการใช้งานจริงในหลายภาษาโปรแกรมและเฟรมเวิร์ก

## เริ่มต้นใช้งาน \{#get-started}

### เพิ่งเริ่มต้นกับความปลอดภัยของ API? \{#new-to-api-security}

หากคุณเพิ่งเริ่มต้นกับการยืนยันตัวตน (Authentication) และการอนุญาต (Authorization) ของ API ให้เริ่มจากพื้นฐานเหล่านี้:

- **[การยืนยันตัวตน (Authentication) กับ การอนุญาต (Authorization)](/concepts/authn-vs-authz)**: เข้าใจความแตกต่างที่สำคัญ
- **[การควบคุมการเข้าถึงตามบทบาท (RBAC)](/authorization/role-based-access-control)**: เรียนรู้การทำงานของสิทธิ์และบทบาท
- **[JSON Web Token (JWT)](https://auth.wiki/jwt)**: ค้นพบวิธีที่ JSON Web Token ช่วยรักษาความปลอดภัยให้กับ API ของคุณ

### เลือกภาษาโปรแกรมหรือเฟรมเวิร์กของคุณ \{#choose-your-language-or-framework}

พร้อมจะเริ่มปกป้อง API แล้วหรือยัง? เลือกเทคโนโลยีที่คุณใช้จากแถบด้านข้างเพื่อดูคู่มือการเชื่อมต่อแบบทีละขั้นตอน พร้อมตัวอย่างโค้ดและแนวทางปฏิบัติที่ดีที่สุด
