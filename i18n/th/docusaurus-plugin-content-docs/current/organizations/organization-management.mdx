# จัดการองค์กร

ส่วนนี้ครอบคลุมวิธีที่นักพัฒนาจัดการองค์กรของตนผ่าน Logto Console หรือ Logto Management API ไม่ใช่วิธีที่ผู้ดูแลองค์กรจัดการสมาชิกด้วยตนเองภายในแอปของคุณ หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับการพัฒนา “ประสบการณ์องค์กร” โปรดดู [คู่มือนี้](/end-user-flows/organization-experience/organization-management)

## จัดการผ่าน Logto Console \{#manage-via-logto-console}

### สร้างองค์กร \{#create-an-organization}

ไปที่ <CloudLink to="/organizations">Console > องค์กร</CloudLink> แล้วคลิกปุ่ม “สร้างองค์กร”

### การตั้งค่าพื้นฐาน \{#basic-settings}

คุณสามารถกำหนดคุณสมบัติพื้นฐานขององค์กร เช่น ชื่อ คำอธิบาย โลโก้ ฯลฯ

### บังคับใช้ MFA สำหรับสมาชิกองค์กร \{#require-mfa-for-organization-members}

คุณสามารถกำหนดให้สมาชิกทุกคนในองค์กรต้องเปิดใช้งาน MFA เพื่อเป็นมาตรการรักษาความปลอดภัย เพื่อให้แน่ใจว่าสมาชิกทุกคนมีการป้องกันเพิ่มเติมเมื่อเข้าถึงทรัพยากรขององค์กร

ในการเปิดใช้งานฟีเจอร์นี้ ให้ไปที่หน้ารายละเอียดขององค์กรแล้วเปิดสวิตช์ “การยืนยันตัวตนหลายปัจจัย (MFA)”

:::note

คุณต้อง [เปิดใช้งานวิธี MFA อย่างน้อยหนึ่งวิธี](/end-user-flows/mfa) เพื่อให้ฟีเจอร์นี้ทำงานได้อย่างถูกต้อง

:::

เมื่อเปิดใช้งานแล้ว สมาชิกที่ยังไม่ได้ตั้งค่า MFA จะไม่สามารถแลกเปลี่ยนโทเค็นองค์กรได้จนกว่าจะตั้งค่า MFA ดู [การอนุญาต (Authorization)](/authorization) สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับช่วงเวลาที่มีการแลกเปลี่ยนโทเค็นองค์กร

โปรดทราบว่า:

- ฟีเจอร์นี้จะตรวจสอบเพียงว่าผู้ใช้ตั้งค่า MFA แล้วหรือไม่ ไม่ได้บังคับให้ใช้ MFA ขณะแลกเปลี่ยนโทเค็นการเข้าถึง
- ฟีเจอร์นี้ไม่จำกัดว่าผู้ใช้จะใช้วิธี MFA แบบใด

### Just-in-Time provisioning \{#just-in-time-provisioning}

[Just-in-Time provisioning](https://auth.wiki/jit-provisioning) จะเพิ่มผู้ใช้เข้าสู่องค์กรโดยอัตโนมัติเมื่อพวกเขาลงชื่อเข้าใช้แอปเป็นครั้งแรก ใน Logto ฟีเจอร์นี้รองรับสำหรับ [Enterprise SSO](/end-user-flows/enterprise-sso) และการจัดเตรียมตามโดเมนอีเมล เมื่อผู้ใช้ตรงตามเงื่อนไขที่กำหนด เช่น ลงชื่อเข้าใช้ผ่าน IdP องค์กรที่ระบุ หรือใช้อีเมลที่มีโดเมนเฉพาะ จะถูกเพิ่มเข้าสู่องค์กรโดยอัตโนมัติ

คุณยังสามารถกำหนดบทบาทองค์กรเริ่มต้นให้กับสมาชิกเมื่อเข้าร่วมองค์กรครั้งแรกได้ด้วย

สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับ Just-in-Time provisioning และวิธีตั้งค่า โปรดดู [ส่วนนี้](/organizations/just-in-time-provisioning)

### จัดการสมาชิกองค์กร \{#manage-organization-members}

ผู้ใช้สามารถมีบทบาทได้มากกว่าหนึ่งบทบาท เมื่อเพิ่มสมาชิกเข้าสู่องค์กร คุณสามารถกำหนดบทบาทให้ผู้ใช้หลายคนพร้อมกันได้ หากปล่อยว่างไว้ ผู้ใช้ที่เพิ่มเข้ามาจะไม่ได้รับบทบาทใด ๆ

ใน <CloudLink to="/users">Console > การจัดการผู้ใช้ > หน้ารายละเอียดผู้ใช้</CloudLink> คุณสามารถดูได้ว่าผู้ใช้สังกัดองค์กรใดและมีบทบาทองค์กรอะไรบ้าง

### จัดการแอป M2M ขององค์กร \{#manage-organization-m2m-applications}

แอปพลิเคชัน Machine-to-machine ก็สามารถถูกเพิ่มเข้าสู่องค์กรได้เช่นกัน คุณสามารถ [กำหนดบทบาทให้กับแอป Machine-to-machine](/organizations/understand-how-organizations-work#organization-m2m-application) ได้เหมือนกับที่กำหนดให้กับผู้ใช้

ใน <CloudLink to="/applications">Console > แอปพลิเคชัน > หน้ารายละเอียดแอปพลิเคชัน</CloudLink> คุณสามารถดูได้ว่าแอปพลิเคชันนั้นเชื่อมโยงกับองค์กรใดและมีบทบาทองค์กรอะไรบ้าง

## จัดการผ่าน Logto Management API \{#manage-via-logto-management-api}

ทุกอย่างที่คุณทำได้ใน Logto Console สามารถทำผ่าน [Management API](/integrate-logto/interact-with-management-api) ได้เช่นกัน ซึ่งรวมถึงแต่ไม่จำกัดเพียง:

1. สร้าง ลบ หรือแก้ไของค์กร
2. จัดการเทมเพลตองค์กร: สร้าง ลบ หรือแก้ไขสิทธิ์และบทบาทขององค์กร
3. เพิ่มหรือถอดสมาชิกออกจากองค์กร
4. กำหนดหรือถอดบทบาทองค์กรของผู้ใช้ออก
5. เพิ่มหรือถอดแอป Machine-to-machine ออกจากองค์กร
6. กำหนดหรือถอดบทบาทองค์กรของแอป Machine-to-machine

คุณยังสามารถดูส่วนนี้สำหรับการใช้ Management API เพื่อเปิดใช้งานประสบการณ์และการจัดการในระดับองค์กรเพิ่มเติม [เรียนรู้เพิ่มเติม](/end-user-flows/organization-experience)

สำหรับรายการความสามารถทั้งหมด โปรดดู [API references](https://openapi.logto.io/group/endpoint-organizations) ของเรา
