# เข้าใจการทำงานขององค์กร

เพื่อให้เข้าใจภาพรวมอย่างครบถ้วน สิ่งสำคัญคือต้องเข้าใจเอนทิตีที่เกี่ยวข้องกับฟีเจอร์องค์กร ซึ่งเป็นพื้นฐานสำคัญในการผสานองค์กรเข้ากับแอปของคุณ

## องค์กร (Organization) \{#organization}

องค์กรประกอบด้วยกลุ่มของ [ผู้ใช้ (identities)](/user-management/user-data) ซึ่งสามารถแทนทีม ลูกค้าธุรกิจ หรือบริษัทคู่ค้าที่สามารถเข้าถึงแอปพลิเคชันของคุณได้

การมีองค์กรเป็นเอนทิตีมีความสำคัญ เพราะไม่เพียงแต่ใช้รวมกลุ่มผู้ใช้เท่านั้น แต่ยังให้บริบทสำหรับการแยกผู้เช่า (tenant isolation) ในแอปแบบหลายผู้เช่า (multi-tenant) อีกด้วย

## สมาชิกองค์กร (Organization member) \{#organization-member}

ใน Logto ผู้ใช้ที่เป็นสมาชิกขององค์กรจะถูกเรียกว่า “สมาชิกองค์กร” (member) ภายใต้บริบทขององค์กรนั้น

## แอปพลิเคชัน M2M ขององค์กร (Organization M2M application) \{#organization-m2m-application}

แอปพลิเคชัน M2M ขององค์กร หมายถึง [แอปพลิเคชันเครื่องต่อเครื่อง](/quick-starts/m2m) ที่เชื่อมโยงกับองค์กร เช่นเดียวกับ [สมาชิกองค์กร](#organization-member) แอปพลิเคชันขององค์กรจะถูกจำกัดขอบเขตอยู่ในบริบทขององค์กร

เมื่อคุณเชื่อมโยงแอปพลิเคชันกับองค์กรแล้ว แอปพลิเคชันนั้นสามารถถูกกำหนด [บทบาทองค์กร](/authorization/role-based-access-control#configure-organization-roles) ได้ ซึ่งมีประโยชน์เมื่อคุณต้องการให้เข้าถึงทรัพยากรขององค์กรแบบไม่ต้องโต้ตอบ

:::note

แอปพลิเคชันประเภทอื่นไม่รองรับการเชื่อมโยงกับองค์กรและบทบาทองค์กร เนื่องจากออกแบบมาเพื่อการโต้ตอบกับผู้ใช้

:::

ดู [ส่วนนี้](/organizations/organization-management) เพื่อจัดการองค์กร สมาชิก และแอปพลิเคชัน

## แม่แบบองค์กร (Organization template) \{#organization-template}

แม่แบบองค์กรถูกออกแบบมาเพื่อตอบโจทย์การควบคุมการเข้าถึงในแอปแบบหลายผู้เช่า โดยประกอบด้วยบทบาทองค์กรและสิทธิ์ขององค์กร ดู [แม่แบบองค์กรนี้](/authorization/organization-template) เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับการออกแบบการควบคุมการเข้าถึงในองค์กร
