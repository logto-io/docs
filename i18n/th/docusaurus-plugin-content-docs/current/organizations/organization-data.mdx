# โครงสร้างข้อมูลองค์กร

ในระดับองค์กร จะมีแอตทริบิวต์หลายรายการที่สามารถสร้าง แก้ไข อ่าน และลบ (CRUD) ค่าได้

## รหัสองค์กร (Organization ID) \{#organization-id}

_รหัสองค์กร (organization id)_ คือ ตัวระบุที่ไม่ซ้ำกันสำหรับแต่ละองค์กร ใช้สำหรับประสบการณ์การลงชื่อเข้าใช้ในระดับองค์กร และการดึงโทเค็นองค์กร (organization tokens)

## ชื่อ (Name) \{#name}

_ชื่อ (name)_ รองรับการลงชื่อเข้าใช้ในระดับองค์กร และสามารถนำไปใช้ในอินเทอร์เฟซผลิตภัณฑ์ระดับองค์กรได้ตามต้องการ

## คำอธิบาย (Description) \{#description}

ฟิลด์ _คำอธิบาย (description)_ ช่วยให้คุณเพิ่มข้อความเพื่อระบุและติดป้ายกำกับองค์กรได้

## โลโก้องค์กร (Organization logos) \{#organization-logos}

เพื่อแสดงโลโก้องค์กรของลูกค้าแบบไดนามิกในประสบการณ์การลงชื่อเข้าใช้ คุณสามารถอัปโหลดโลโก้องค์กรได้ที่หน้าการตั้งค่าองค์กร

ดูรายละเอียดเพิ่มเติมที่ [โลโก้เฉพาะองค์กร](/customization/match-your-brand/#organization-specific-branding)

## ข้อมูลกำหนดเอง (Custom data) \{#custom-data}

_ข้อมูลกำหนดเอง (custom data)_ คืออ็อบเจกต์ JSON ที่ใช้เก็บข้อมูลเพิ่มเติมเกี่ยวกับองค์กร ช่วยให้นักพัฒนามีความยืดหยุ่นในการจัดเก็บรายละเอียดเพิ่มเติมขององค์กรเพื่อวัตถุประสงค์ในการพัฒนา

## ต้องใช้ MFA หรือไม่ (Is MFA required) \{#is-mfa-required}

_isMfaRequired_ ระบุว่าองค์กรนี้ต้องใช้การยืนยันตัวตนหลายปัจจัย (Multi-Factor Authentication; MFA) หรือไม่ หากตั้งค่าเป็น `true` สมาชิกทุกคนต้องผ่าน MFA ระหว่างการลงชื่อเข้าใช้เพื่อเข้าถึงองค์กร การตั้งค่านโยบายความปลอดภัยนี้กำหนดในระดับองค์กร

ดูรายละเอียดเพิ่มเติมที่ [จัดการองค์กร](/organizations/organization-management#require-mfa-for-organization-members)

## วันที่สร้าง (Created at) \{#created-at}

_createdAt_ คือเวลาพร้อมเขตเวลาที่องค์กรถูกสร้างขึ้น

## รหัสผู้เช่า (Tenant ID) \{#tenant-id}

_tenantId_ ใช้ระบุผู้เช่าที่องค์กรนั้นสังกัดอยู่
