# นโยบายการเก็บข้อมูลของ Dev tenant (Dev tenant data retention policy)

Logto Cloud ยึดหลักการที่ชัดเจน: **production สำหรับการเก็บถาวร, development สำหรับการทดสอบ**

- ใน **production tenant** (Free, Pro, และ Enterprise) วัตถุและเอนทิตีจะถูกเก็บไว้ตลอดอายุบัญชี เว้นแต่คุณจะลบหรือร้องขอเป็นอย่างอื่น
- ใน **dev tenant** เป้าหมายจะแตกต่างกัน Dev tenant คือ **sandbox ที่ปลอดภัยสำหรับการทดสอบ** โดยเน้นที่ความปลอดภัย ประสิทธิภาพ และสภาพแวดล้อมที่สะอาด มากกว่าการเก็บข้อมูลระยะยาว

เพื่อรักษาหลักการนี้ Logto จึงใช้ “นโยบายการเก็บข้อมูล 90 วัน” ใน dev tenant: **ผู้ใช้** ใน dev tenant จะถูกลบโดยอัตโนมัติหลังจาก 90 วัน

ในสภาพแวดล้อม dev นักพัฒนามักสร้างบัญชีทดสอบจำนวนมากระหว่างการเชื่อมต่อระบบ บัญชีเหล่านี้สร้างข้อมูลจำนวนมากแต่แทบไม่มีคุณค่าในระยะยาว นโยบาย 90 วันนี้ช่วยให้ทรัพยากรของแพลตฟอร์ม Logto ถูกใช้อย่างมีประสิทธิภาพ รักษาสภาพแวดล้อมทดสอบให้สะอาดและมีประสิทธิภาพ ป้องกันบัญชีที่ไม่ได้ใช้งานสะสม และแยกข้อมูลทดสอบชั่วคราวออกจากข้อมูล production ระยะยาวอย่างชัดเจน

## ทำความเข้าใจกับนโยบาย 90 วัน \{#understanding-the-90-day-policy}

### เฉพาะบัญชีผู้ใช้ที่มีอายุมากกว่า 90 วันเท่านั้นที่จะถูกลบ \{#only-user-accounts-older-than-90-days-are-deleted}

การลบผู้ใช้หมายถึงการลบข้อมูลโปรไฟล์ ข้อมูลการยืนยันตัวตน (authentication data) และข้อมูลที่เกี่ยวข้องทั้งหมด

### สิ่งที่ยังคงอยู่ \{#what-stays-intact}

งานพัฒนาและการตั้งค่าของคุณจะไม่ถูกแตะต้อง ทำให้คุณสามารถนำการตั้งค่า dev และ integration ไปใช้ใน production ได้ง่ายในอนาคต เรามีตัวเลือกให้ [แปลง dev tenant ของคุณเป็น production tenant](./tenant-settings#tenant-types-dev-vs-prod) ได้โดยตรง

## การลบผู้ใช้อัตโนมัติหมายถึงอะไรสำหรับคุณ \{#what-automatic-user-deletion-means-for-you}

- การปกป้องความเป็นส่วนตัว: บัญชีทดสอบมักถูกแชร์ข้ามทีมระหว่างการพัฒนาหรือถูกทิ้งไว้นานโดยไม่ได้ใช้งาน เราแนะนำให้รีเฟรชบัญชีเหล่านี้ทุก 90 วัน
- ประสิทธิภาพของทรัพยากร: การลบอัตโนมัติช่วยเพิ่มพื้นที่จัดเก็บและประสิทธิภาพการประมวลผล ทำให้สภาพแวดล้อมสะอาดและมีประสิทธิภาพ
- การแยกสภาพแวดล้อมอย่างชัดเจน: Dev tenant คือ sandbox ชั่วคราว ข้อมูลถาวรควรอยู่ใน production tenant

## แนวทางปฏิบัติที่ดีที่สุดสำหรับ dev tenant \{#best-practices-for-dev-tenants}

### ระหว่างการพัฒนา \{#during-development}

- ใช้ Logto dev tenant เป็น sandbox สำหรับการทดสอบของคุณ
- จดบันทึกกรณีทดสอบสำคัญไว้เพื่อให้สามารถสร้างใหม่ได้ง่าย
- วางแผนวงจรการพัฒนาอย่างรอบคอบและย้ายไป production เมื่อถึงเวลาที่เหมาะสม

### สำหรับโครงการระยะยาว \{#for-long-term-projects}

- สร้างผู้ใช้หรือบัญชีทดสอบใหม่ตามต้องการ แทนที่จะเก็บบัญชีเก่าไว้
- ใช้ production tenant หากคุณต้องการ staging environment ที่เก็บข้อมูลถาวร

## สรุป \{#summary}

1. ผู้ใช้ใน dev tenant จะถูกลบโดยอัตโนมัติหลัง 90 วัน
2. การตั้งค่า แอป และการกำหนดค่าทั้งหมดของคุณจะยังคงอยู่ถาวร
3. นโยบายนี้ช่วยให้มั่นใจในความปลอดภัย ประสิทธิภาพ และสภาพแวดล้อมทดสอบที่สะอาด
4. Production tenant มีนโยบายการเก็บข้อมูลที่แตกต่างกันสำหรับข้อมูลถาวร
5. วางแผน workflow การพัฒนาของคุณให้สอดคล้องกับรอบการลบอัตโนมัตินี้

นโยบายการเก็บข้อมูลนี้ช่วยให้สภาพแวดล้อมการพัฒนาของคุณเป็นพื้นที่ทดสอบที่ปลอดภัย มีประสิทธิภาพ และเชื่อถือได้ พร้อมทั้งปกป้องความเป็นส่วนตัวของผู้ใช้และรักษาประสิทธิภาพของระบบ

สำหรับการเก็บข้อมูลผู้ใช้ถาวร แนะนำให้อัปเกรดเป็น production tenant ซึ่งการเก็บข้อมูลจะเป็นไปตามข้อกำหนดขององค์กรของคุณ
