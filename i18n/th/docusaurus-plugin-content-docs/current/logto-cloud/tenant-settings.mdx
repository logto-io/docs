# การตั้งค่าเทนแนนต์

ผู้ใช้ใหม่ที่ลงทะเบียน Logto Cloud จะได้รับการเริ่มต้นใช้งานโดยอัตโนมัติในเทนแนนต์สภาพแวดล้อม **Development** (Dev) ฟรี คุณสามารถสำรวจฟีเจอร์ทั้งหมดในเทนแนนต์นี้ได้

หากคุณต้องการสร้างเทนแนนต์แยกสำหรับสภาพแวดล้อม **Production** (Prod) หรือโปรเจกต์ใหม่ ให้คลิกชื่อเทนแนนต์ปัจจุบันของคุณที่มุมซ้ายบนของแถบด้านบน ในเมนูนี้ คุณสามารถสลับระหว่างเทนแนนต์หรือสร้างเทนแนนต์ใหม่ได้

คลิก "Create tenant" จากนั้น:

- ตั้งชื่อเทนแนนต์
- เลือก [ภูมิภาคข้อมูลของเทนแนนต์](#tenant-region)
- เลือก [ประเภทของเทนแนนต์](#tenant-types-dev-vs-prod) (สภาพแวดล้อม)

สำหรับเทนแนนต์ที่มีอยู่แล้ว ไปที่ <CloudLink to="/settings">Console > การตั้งค่าเทนแนนต์ > การตั้งค่า</CloudLink> ที่นี่คุณสามารถ:

- ดูรหัสเทนแนนต์ (tenant ID)
- อัปเดตชื่อเทนแนนต์
- ดู [ภูมิภาคของเทนแนนต์](#tenant-region) (ไม่สามารถเปลี่ยนแปลงได้หลังจากสร้าง)
- ดู [ประเภทของเทนแนนต์](#tenant-types-dev-vs-prod) (คุณสามารถแปลง Dev tenant เป็น Prod tenant ได้หากต้องการ)
- [ออกจากเทนแนนต์](#leave-tenant)
- [ลบเทนแนนต์](#delete-tenant)

## ภูมิภาคของเทนแนนต์ \{#tenant-region}

เมื่อคุณสร้างเทนแนนต์ คุณสามารถเลือกภูมิภาคที่จัดเก็บข้อมูลของเทนแนนต์ได้ โดยไม่สามารถเปลี่ยนแปลงได้หลังจากสร้างเทนแนนต์แล้ว ภูมิภาคที่มีให้เลือก ได้แก่

- ยุโรป (เนเธอร์แลนด์)
- สหรัฐอเมริกาตะวันตก (แอริโซนา)
- ออสเตรเลีย (ออสเตรเลียตะวันออก)
- ญี่ปุ่น (ญี่ปุ่นตะวันออก)

โดยปกติ คุณควรเลือกภูมิภาคที่ใกล้กับลูกค้าของคุณมากที่สุด เพื่อลดความหน่วงและเพิ่มประสิทธิภาพ

Logto ใช้ประโยชน์จากเครือข่าย edge ทั่วโลกเพื่อมอบประสิทธิภาพและความพร้อมใช้งานที่ดีที่สุดสำหรับแอปพลิเคชันของคุณ การกำหนดเส้นทางคำขอได้รับการปรับให้เหมาะสมเพื่อให้แน่ใจว่าผู้ใช้ของคุณเชื่อมต่อกับตัวเลือกที่มีประสิทธิภาพสูงสุดเสมอ

:::note
กำลังมองหาภูมิภาคอื่น? [ติดต่อเรา](https://logto.io/contact) เพื่อ:

- ขอภูมิภาค public cloud ใหม่
- สอบถามเกี่ยวกับการติดตั้ง Logto Private Cloud ในสถานที่ที่คุณต้องการ
  :::

## ประเภทของเทนแนนต์: Dev กับ Prod \{#tenant-types-dev-vs-prod}

ใน Logto Cloud มีเทนแนนต์สองประเภท: Development (Dev) และ Production (Prod) ด้วยการแยกประเภทเทนแนนต์นี้ คุณสามารถจัดการโปรเจกต์ของคุณในแต่ละสภาพแวดล้อมได้อย่างมีประสิทธิภาพ และในขณะเดียวกันก็ได้รับประโยชน์จาก Logto อย่างเต็มที่

คุณสามารถเลือกประเภทของเทนแนนต์ได้ขณะสร้าง เมื่อคุณพร้อมใช้งานจริงสำหรับ production มีสองทางเลือก:

- **สร้าง Production tenant ใหม่**  
  ตั้งค่าเทนแนนต์ production ใหม่และกำหนดค่าตั้งแต่ต้น เหมาะสำหรับกรณีที่คุณต้องการแยกสภาพแวดล้อม development และ production ออกจากกัน
- **แปลง Dev tenant ปัจจุบันของคุณเป็น Production**  
  หากคุณไม่ต้องการตั้งค่าหรือย้ายผู้ใช้ใหม่ คุณสามารถอัปเกรด Dev tenant ที่มีอยู่เป็น Production tenant แบบชำระเงินได้

  - **แปลงเป็น Pro plan**: ไปที่ <CloudLink to="/tenant-settings/settings">Console > การตั้งค่าเทนแนนต์ > การตั้งค่า</CloudLink> แล้วคลิก "Convert" เพื่ออัปเกรดด้วยตนเอง ฟีเจอร์ที่ต้องชำระเงินที่คุณใช้ใน dev tenant จะถูกรวมไปยัง Stripe checkout
  - **แปลงเป็น Enterprise plan**: [ติดต่อเรา](https://logto.io/contact) แล้วเราจะช่วยคุณดำเนินการอัปเกรด

  :::note
  เมื่อแปลงแล้ว จะไม่สามารถย้อนกลับเป็น Dev environment ได้ กรุณาตรวจสอบความพร้อมก่อนดำเนินการ
  :::

### Development \{#development}

เทนแนนต์ development (dev tenant) มีไว้สำหรับการทดสอบเป็นหลัก และไม่ควรใช้ในสภาพแวดล้อม production เทนแนนต์เหล่านี้สามารถเข้าถึงฟีเจอร์พรีเมียมและฟีเจอร์ที่ต้องชำระเงินในแผนแบบชำระเงินได้ฟรีโดยไม่ต้องสมัครสมาชิก

อย่างไรก็ตาม มีข้อจำกัดบางประการสำหรับ dev tenant:

- Dev tenant จะลบผู้ใช้อัตโนมัติหลังจาก 90 วัน ดู [นโยบายการเก็บข้อมูล dev tenant](./dev-tenant-data-retention) สำหรับรายละเอียด
- มีแบนเนอร์ปรากฏระหว่างประสบการณ์การลงชื่อเข้าใช้ เพื่อแสดงว่าเทนแนนต์อยู่ในโหมด development
- Dev tenant อาจมีข้อจำกัดโควต้าสำหรับฟีเจอร์บางอย่าง โดยจะอธิบายไว้ในหน้ารายละเอียดฟีเจอร์นั้น ๆ หากมี
- Logto อาจอัปเดตข้อจำกัดโควต้าของ dev tenant และเราจะพยายามแจ้งให้คุณทราบล่วงหน้า

### Production \{#production}

Production tenant คือที่ที่ผู้ใช้ปลายทางเข้าถึงแอปจริง และคุณอาจต้องมี [การสมัครสมาชิกแบบชำระเงิน](https://logto.io/pricing) คุณสามารถสมัคร Free plan หรือ Pro plan เพื่อสร้าง production tenant ได้ หากคุณสมัคร Free plan จะสามารถสร้างเทนแนนต์ได้สูงสุด 10 เท่านั้น

## เปิดใช้งาน MFA \{#enable-mfa}

เพิ่มความปลอดภัยให้ workspace ของคุณด้วยการบังคับใช้ Multi-Factor Authentication (MFA) สำหรับสมาชิกทุกคนใน Logto Pro/Enterprise tenant ของคุณ

เนื่องจากยังไม่สามารถเปิดใช้งานด้วยตนเองได้ โปรด [ติดต่อเรา](https://logto.io/contact) เพื่อเปิดใช้งานฟีเจอร์นี้

## เปิดใช้งาน Enterprise SSO \{#enable-enterprise-sso}

Logto Cloud รองรับการเชื่อมต่อ Single Sign-On สำหรับ Enterprise Plan tenant รวมถึงผู้ให้บริการอย่าง Google Workspace, Okta, Azure AD และอื่น ๆ

หากต้องการเริ่มต้น โปรด [ติดต่อเรา](https://logto.io/contact) เราจะช่วยคุณตั้งค่าอย่างรวดเร็ว

## ออกจากเทนแนนต์ \{#leave-tenant}

แอดมินสามารถ [เชิญสมาชิกเพิ่มเติม](/logto-cloud/tenant-member-management) มายังเทนแนนต์นี้ได้

หากมี **แอดมิน** (บทบาท) อย่างน้อยหนึ่งคน หรือคุณเป็น **collaborator** (บทบาท) คุณสามารถเลือกออกจากเทนแนนต์ได้ หลังจากออกแล้ว ทรัพยากรทั้งหมดในเทนแนนต์จะยังคงอยู่ แต่คุณจะไม่สามารถเข้าถึงได้อีก

หากคุณเป็นแอดมินคนสุดท้าย คุณต้องแต่งตั้ง collaborator คนอื่นเป็นแอดมินก่อนจึงจะออกได้

## ลบเทนแนนต์ \{#delete-tenant}

[แอดมิน](/logto-cloud/tenant-member-management#invite-collaborators) สามารถลบ Logto tenant ได้ การลบเทนแนนต์จะลบข้อมูลผู้ใช้และการตั้งค่าทั้งหมดที่เกี่ยวข้องอย่างถาวร การดำเนินการนี้ไม่สามารถย้อนกลับได้ Logto จะขอให้คุณกรอกชื่อเทนแนนต์เพื่อยืนยันและช่วยป้องกันการลบโดยไม่ตั้งใจ

หากต้องการความช่วยเหลือ โปรด [ติดต่อเรา](https://logto.io/contact) ทางอีเมล

หากคุณวางแผนจะหยุดใช้ Logto Cloud สำหรับโปรเจกต์ Logto สามารถช่วยคุณส่งออกข้อมูลผู้ใช้ทั้งหมดได้ โปรด [ติดต่อเรา](https://logto.io/contact)

</details>
