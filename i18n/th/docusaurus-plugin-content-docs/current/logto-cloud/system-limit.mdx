# ขีดจำกัดของระบบ

ขีดจำกัดของระบบกำหนดจำนวนสูงสุดของเอนทิตีที่ผู้เช่าสามารถสร้างได้ เพื่อปกป้องเสถียรภาพของแพลตฟอร์ม และไม่ขึ้นกับการคิดค่าบริการ รายการบางอย่างในหน้าราคาอาจระบุว่า _ไม่จำกัด_ หรือ _จ่ายตามการใช้งาน_ ซึ่งหมายความว่าไม่มีขีดจำกัดด้านการเรียกเก็บเงิน แต่ยังมีขีดจำกัดภายในระบบเพื่อให้เกิดความเป็นธรรมในการใช้งาน หากต้องการเพิ่มขีดจำกัด สามารถพิจารณาเป็นกรณีไป

## ประเภทของขีดจำกัด \{#types-of-limits}

### ขีดจำกัดของทรัพยากรระบบและเอนทิตี \{#system-resource-and-entity-caps}

จำนวนสูงสุดของอ็อบเจ็กต์ เช่น แอปพลิเคชัน, บทบาท (Roles), องค์กร (Organizations), ตัวเชื่อมต่อ (Connectors) และทรัพยากรระบบที่อิงตามการใช้งานบางประเภท เช่น การใช้โทเค็นรวม ขีดจำกัดเหล่านี้แตกต่างกันตามแผนบริการ

### ขีดจำกัดอัตรา (Rate limit) \{#rate-limit}

การป้องกันในระดับคำขอต่อผู้เช่าแต่ละราย โดยแนวทางทั่วไปคือ **200 คำขอ ต่อ 10 วินาที ต่อผู้เช่า** หากทราฟฟิกของคุณถูกต้องและเกินขีดจำกัดนี้อย่างต่อเนื่อง สามารถขอพิจารณาเป็นกรณีไป

---

หากกรณีการใช้งานของคุณเหมาะสมและคุณชนขีดจำกัด กรุณาติดต่อเรา เพื่อช่วยปรับขีดจำกัดและสนับสนุนกรณีการใช้งานจริง

## การป้องกันขีดจำกัดในระดับผู้เช่า \{#tenant-level-rate-limit-protection}

### ผู้เช่า Dev \{#dev-tenants}

[ผู้เช่า Dev](/logto-cloud/tenant-settings#development) ให้สิทธิ์การเข้าถึงฟรีสำหรับการทดสอบ เพื่อป้องกันการใช้งานที่ไม่เหมาะสมและสร้างความคาดหวังที่ชัดเจน ผู้เช่า Dev จะมีขีดจำกัดเอนทิตีที่แน่นอน หากต้องการมากกว่านี้ สามารถขอเพิ่มขีดจำกัดหรืออัปเกรดเป็น Pro เพื่อเข้าถึงแบบเต็มรูปแบบ
