# ตั้งค่า Logto OSS

ส่วนนี้ครอบคลุมขั้นตอนการตั้งค่าพื้นฐานและการดำเนินการสำคัญเพื่อส่งมอบผลิตภัณฑ์ของคุณอย่างมีประสิทธิภาพและจัดการเวิร์กโฟลว์การพัฒนาสำหรับ [Logto open-source service (OSS)](https://github.com/logto-io/logto)

## เริ่มต้นกับ Logto OSS \{#get-started-with-logto-oss}

ทำตาม [คู่มือเริ่มต้น](/logto-oss/get-started-with-oss/) เพื่อเปิดใช้งาน Logto ได้ทันที และสามารถดู [คู่มือการปรับใช้ฉบับเต็ม](/logto-oss/deployment-and-configuration) สำหรับการใช้งานจริงในภายหลัง

## ฟีเจอร์ที่รองรับโดย Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS รองรับความสามารถหลักส่วนใหญ่ของบริการ Logto และมีการอัปเดตและดูแลอย่างสม่ำเสมอ

ฟีเจอร์ขั้นสูงบางอย่างมีเฉพาะในเวอร์ชัน Logto Cloud เท่านั้น เช่น:

| ข้อจำกัดของฟีเจอร์ใน OSS                                                                         | คำอธิบาย                                                                                                                                       |
| ------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto console: หลายผู้เช่า (multiple tenants)](/logto-cloud/tenant-settings)                    | สร้างและจัดการผู้เช่า Logto หลายรายในคอนโซลเดียว                                                                                               |
| [Logto console: เชิญผู้ร่วมงาน (collaborator invitation)](/logto-cloud/tenant-member-management) | เชิญและจัดการสมาชิกหลายคนในผู้เช่า Logto เดียวกัน                                                                                              |
| Logto console: เปิดใช้งาน MFA                                                                    | เพิ่มความปลอดภัยโดยกำหนดให้มีการยืนยันตัวตนหลายปัจจัย (MFA) เมื่อเข้าสู่ Logto Console                                                         |
| [Logto Protected App](/integrate-logto/protected-app)                                            | การผสานการยืนยันตัวตนแบบรวดเร็วโดยไม่ต้องใช้ SDK ขับเคลื่อนด้วย Cloudflare                                                                     |
| [บริการอีเมลในตัวของ Logto](/connectors/email-connectors/built-in-email-service)                 | ใช้บริการอีเมลในตัวและฟรีสำหรับการส่งอีเมล                                                                                                     |
| [Bring your UI](/customization/bring-your-ui)                                                    | ปรับแต่ง UI หรือโฟลว์ของประสบการณ์การลงชื่อเข้าใช้ด้วยฟีเจอร์นี้ (OSS สามารถ fork โค้ดบน GitHub เพื่อปรับแต่งประสบการณ์การลงชื่อเข้าใช้โดยตรง) |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)                            | เปิดใช้งานการลงชื่อเข้าใช้ครั้งเดียว (SSO) สำหรับองค์กรที่เริ่มต้นโดยผู้ให้บริการข้อมูลระบุตัวตน (IdP)                                         |
| [SAML apps](/integrate-logto/saml-app)                                                           | ผสานแอปพลิเคชันผ่านโปรโตคอล SAML โดยเวอร์ชัน OSS จำกัดที่ 3 แอป SAML                                                                           |
| [ซ่อนแบรนด์ Logto](/customization/match-your-brand#hide-logto-branding)                          | ลบเครื่องหมาย "Powered by Logto" ออกจากประสบการณ์การลงชื่อเข้าใช้                                                                              |

เคล็ดลับ: ฟีเจอร์ Multi-tenancy, การเชิญสมาชิก, และ MFA จะไม่สามารถใช้ได้สำหรับทีมของคุณในการลงชื่อเข้าใช้ Logto console แบบโอเพ่นซอร์ส อย่างไรก็ตาม คุณสามารถนำฟีเจอร์เหล่านี้ไปใช้ในผลิตภัณฑ์ของคุณเองด้วย Logto OSS เพื่อให้ผู้ใช้ปลายทางของคุณใช้งานได้

## ติดตามอัปเดต Logto เวอร์ชันล่าสุด \{#stay-updated-with-logto-releases}

เพื่อให้ Logto ของคุณทันสมัยอยู่เสมอด้วยฟีเจอร์ล่าสุด อย่าลืมติดตามหน้า [Logto GitHub Releases](https://github.com/logto-io/logto/releases) ซึ่งคุณจะพบบันทึกการอัปเดตทั้งหมดที่นั่น

อ่าน [คู่มือการอัปเกรด](/logto-oss/upgrading-oss-version) เพื่อเรียนรู้วิธีอัปเกรด Logto โดยไม่ต้องเปลี่ยนโค้ดหรือโครงสร้างฐานข้อมูลของคุณ
