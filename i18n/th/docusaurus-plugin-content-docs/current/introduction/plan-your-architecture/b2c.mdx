# สถาปัตยกรรมแบบผู้เช่าเดียวสำหรับบริการ B2C

## สถาปัตยกรรม \{#architecture}

ในแอปพลิเคชันสำหรับผู้บริโภค ผู้ใช้จะเป็นเจ้าของบัญชีของตนเองอย่างสมบูรณ์และควบคุมตัวตนและการยืนยันตัวตนของตนเอง โดยไม่มีการเกี่ยวข้องจากบุคคลที่สามหรือ “ชั้นกลาง” เช่น ธุรกิจหรือองค์กร นี่คือความแตกต่างสำคัญระหว่างสถาปัตยกรรมเอกลักษณ์แบบ B2C และ B2B

## เชื่อมโยงความต้องการของคุณกับชุดเครื่องมือสนับสนุนของ Logto \{#connect-your-requirements-to-logtos-support-toolkit}

สถาปัตยกรรมนี้ประกอบด้วยสองฝ่ายหลักที่เกี่ยวข้องกับสถานการณ์การจัดการ ขึ้นอยู่กับความต้องการและเป้าหมายเฉพาะของคุณ อาจมีฝ่ายเหล่านี้ทั้งหมดหรือบางส่วนเข้ามาเกี่ยวข้อง

เราได้สรุปกรณีการใช้งานที่พบบ่อย โดยเน้นเป้าหมายหลักของแต่ละงานจัดการเอกลักษณ์ผู้ใช้ พร้อมผลิตภัณฑ์และ API ที่เกี่ยวข้องของเรา คุณสามารถจับคู่ความต้องการของคุณกับบริการของเราเพื่อเริ่มต้นได้อย่างรวดเร็ว

| ผู้ใช้                    | เป้าหมาย                                                               | ผลิตภัณฑ์และ API ของ Logto                                                                                                                                       |
| ------------------------- | ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| นักพัฒนา                  | จัดการและปกป้องระบบเอกลักษณ์ผู้ใช้ และทำงานโดยตรงกับฐานข้อมูลเอกลักษณ์ | <ul><li>[Logto Console](https://cloud.logto.io)</li><li>[Logto Management API](https://openapi.logto.io/)</li></ul>                                              |
| ผู้ใช้ปลายทาง / ผู้บริโภค | [จัดการการยืนยันตัวตนและข้อมูลส่วนตัวของตนเอง](/end-user-flows)        | <ul><li>[Logto Management API](https://openapi.logto.io/)</li><li>[Account API](https://openapi.logto.io/operation/operation-getaccountcentersettings)</li></ul> |
