# สถาปัตยกรรมหลายผู้เช่าสำหรับบริการ B2B

## สถาปัตยกรรม \{#architecture}

แอป B2B มักใช้สถาปัตยกรรมแบบ [multi-tenant](https://auth.wiki/multi-tenancy) ในแอปเหล่านี้ ผู้ใช้เป็นเจ้าของบัญชีของตนเองและจัดการอัตลักษณ์และการยืนยันตัวตน โดยมีบุคคลที่สาม เช่น ธุรกิจหรือองค์กร เข้ามามีส่วนร่วม อัตลักษณ์ของผู้ใช้ปลายทางมักไม่ใช่ผู้บริโภคทั่วไป แต่เป็นพนักงานหรือผู้ร่วมงานในองค์กรธุรกิจ

## เชื่อมโยงความต้องการของคุณกับชุดเครื่องมือสนับสนุนของ Logto \{#connect-your-requirements-to-logtos-support-toolkit}

สถาปัตยกรรมนี้ประกอบด้วย 3 ฝ่ายหลักที่เกี่ยวข้องกับสถานการณ์การจัดการ ขึ้นอยู่กับความต้องการและเป้าหมายของคุณ อาจมีบางฝ่ายหรือทุกฝ่ายที่เกี่ยวข้อง

เราได้สรุปกรณีการใช้งานที่พบบ่อย โดยเน้นเป้าหมายหลักของแต่ละผู้ใช้ที่จัดการงาน และผลิตภัณฑ์กับ API ที่เกี่ยวข้องของเรา คุณสามารถจับคู่ความต้องการของคุณกับบริการของเราเพื่อเริ่มต้นได้อย่างรวดเร็ว

| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
