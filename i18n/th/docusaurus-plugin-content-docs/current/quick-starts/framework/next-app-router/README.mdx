### ดึงโทเค็นการเข้าถึงสำหรับทรัพยากร API \{#fetch-access-token-for-the-api-resource}

<FetchAccessTokenForApiResources />

หากคุณต้องการดึงโทเค็นการเข้าถึงใน server component สามารถใช้ฟังก์ชัน `getAccessTokenRSC` ได้ดังนี้:

```tsx title="app/page.tsx"
// ตัวอย่างนี้แสดงการดึง access token ใน server component
import { getAccessTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const accessToken = await getAccessTokenRSC(logtoConfig, 'https://shopping.your-app.com/api');

  return (
    <main>
      <p>Access token: {accessToken}</p>
    </main>
  );
}
```

:::tip[ข้อจำกัดการแคชโทเค็นใน RSC (RSC token caching limitation)]

React Server Components ไม่สามารถเขียนคุกกี้ได้ (ข้อจำกัดของ Next.js) แม้ว่า `getAccessTokenRSC` จะรีเฟรชโทเค็นที่หมดอายุโดยใช้ refresh token ได้ แต่ access token ใหม่จะไม่ถูกบันทึกลงใน session cookie ซึ่งหมายความว่า ทุกคำขอ RSC อาจต้องรีเฟรชโทเค็นใหม่หากโทเค็นที่แคชไว้หมดอายุ

**แนวทางแก้ไข:**

1. **ใช้ client component ร่วมกับ Server Actions** - เรียก `getAccessToken` จาก client component ผ่าน Server Actions ซึ่งสามารถอัปเดตคุกกี้ได้
2. **ใช้ external session storage** - กำหนดค่า `sessionWrapper` ด้วย Redis/KV storage โดยคุกกี้จะเก็บเพียง session ID ขณะที่ข้อมูลโทเค็นจะอยู่ใน storage ภายนอก ทำให้ RSC สามารถบันทึกโทเค็นที่รีเฟรชใหม่ได้ ดู [การใช้ external session storage](#use-external-session-storage) ด้านล่าง

:::

### ดึงโทเค็นองค์กร \{#fetch-organization-tokens}

<FetchOrganizationTokenForUser />

หากคุณต้องการดึงโทเค็นองค์กรใน server component สามารถใช้ฟังก์ชัน `getOrganizationTokenRSC` ได้ดังนี้:

```tsx title="app/page.tsx"
// ตัวอย่างนี้แสดงการดึง organization token ใน server component
import { getOrganizationTokenRSC } from '@logto/next/server-actions';
import { logtoConfig } from './logto';

export default async function Home() {
  const token = await getOrganizationTokenRSC(logtoConfig, 'organization-id');

  return (
    <main>
      <p>Organization token: {token}</p>
    </main>
  );
}
```

:::tip[ข้อจำกัดการแคชโทเค็นใน RSC (RSC token caching limitation)]

เหมือนกับ `getAccessTokenRSC` โทเค็นองค์กรที่รีเฟรชใหม่จะไม่ถูกบันทึกใน RSC ดู [แนวทางแก้ไขด้านบน](#fetch-access-token-for-the-api-resource)

:::

## การเปลี่ยนเส้นทางไปยังหน้าลงชื่อเข้าใช้อัตโนมัติเมื่อไม่ได้รับอนุญาต \{#automatic-sign-in-redirect\}

:::tip
ตัวช่วย `signIn` จะเปลี่ยนแปลงคุกกี้เพื่อสร้าง session การลงชื่อเข้าใช้ จึงไม่สามารถเรียกใช้โดยตรงใน React Server Component (RSC) ได้ หากต้องการให้ระบบเปลี่ยนเส้นทางไปยังหน้าลงชื่อเข้าใช้อัตโนมัติเมื่อ RSC ตรวจพบผู้ใช้ที่ไม่ได้รับอนุญาต ให้เรียก `signIn` ใน route handler เฉพาะ แล้ว redirect ไปยัง route นั้น
:::

```ts title="app/sign-in/route.ts"
// ตัวอย่าง route handler สำหรับ sign-in
import { signIn } from '@logto/next/server-actions';
import { logtoConfig } from '../../logto';

export async function GET() {
  await signIn(logtoConfig);
}
```

```tsx title="app/protected/page.tsx"
// ตัวอย่างการ redirect ไปยัง sign-in เมื่อไม่ได้รับอนุญาต
import { getLogtoContext } from '@logto/next/server-actions';
import { redirect } from 'next/navigation';
import { logtoConfig } from '../logto';

export default async function ProtectedPage() {
  const { isAuthenticated } = await getLogtoContext(logtoConfig);

  if (!isAuthenticated) {
    redirect('/sign-in');
  }

  return <div>Protected content</div>;
}
```
