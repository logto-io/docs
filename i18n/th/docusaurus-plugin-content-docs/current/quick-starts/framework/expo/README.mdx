## การแก้ไขปัญหา \{#troubleshooting}

### ไม่สามารถแก้ไข `@logto/client/shim` ได้ \{#unable-to-resolve-logtoclientshim}

:::note

- สำหรับผู้ใช้ที่ใช้ SDK เวอร์ชัน \<= v0.3.0 โปรดทำตามคำแนะนำด้านล่างเพื่อแก้ไขปัญหา
- สำหรับผู้ใช้ที่ใช้ SDK เวอร์ชัน \>= v0.4.0 โปรดข้ามส่วนนี้ไป ปัญหานี้ได้รับการแก้ไขแล้วในเวอร์ชันล่าสุด

:::

สำหรับโปรเจกต์ Expo หากคุณพบข้อผิดพลาด `Unable to resolve "@logto/client/shim" from "node_modules/@logto/rn/lib/index.js"` คุณสามารถแก้ไขได้โดยเพิ่มโค้ดต่อไปนี้ในไฟล์ `metro.config.js` ของคุณ:

```js title="metro.config.js"
// ...
const config = {
  // ...
  resolver: {
    unstable_enablePackageExports: true,
  },
};

module.exports = config;
```

ข้อผิดพลาดนี้หมายความว่าแพ็กเกจ `@logto/rn` ไม่สามารถแก้ไขโมดูล `@logto/client/shim` ได้

เนื่องจากในแพ็กเกจ `@logto/client` มีการใช้ node exports และ Metro bundler ไม่ได้เปิดใช้งาน package exports โดยค่าเริ่มต้น คุณจึงต้องเปิดใช้งานด้วยตนเอง

ดูรายละเอียดเพิ่มเติมได้ที่ [React Native package exports support](https://reactnative.dev/blog/2023/06/21/package-exports-support#enabling-package-exports-beta)
