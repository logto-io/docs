# ตัวเชื่อมต่อ SMS

การตั้งค่าตัวเชื่อมต่อ SMS ช่วยให้คุณสามารถส่ง [รหัสผ่านใช้ครั้งเดียว (OTP)](https://auth.wiki/otp) ไปยังหมายเลขโทรศัพท์ของผู้ใช้ได้ กลไกการยืนยันตัวตนแบบไม่ใช้รหัสผ่านนี้สามารถนำไปใช้ในหลายกรณี เช่น [สมัครสมาชิก](/end-user-flows/sign-up-and-sign-in/sign-up), [ลงชื่อเข้าใช้](/end-user-flows/sign-up-and-sign-in/sign-in), [ลืมรหัสผ่าน](/end-user-flows/sign-up-and-sign-in/reset-password), [กระบวนการเชื่อมโยงบัญชี](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking), [เชิญสมาชิก](/end-user-flows/organization-experience/invite-organization-members) และ [ยืนยันตัวตนของผู้ใช้](/end-user-flows/security-verification) ช่วยให้การยืนยันตัวตนของผู้ใช้เป็นไปอย่างราบรื่นและเพิ่มความปลอดภัยโดยลดความเสี่ยงจากการรั่วไหลที่เกี่ยวข้องกับรหัสผ่าน

## ขั้นตอนการตั้งค่า \{#configuration-steps}

ทำตามขั้นตอนด้านล่างเพื่อเพิ่มตัวเชื่อมต่อ SMS:

1. ไปที่ <CloudLink to="/connectors/passwordless">Console > Connector > Email and SMS connectors</CloudLink>
2. เพื่อเพิ่มตัวเชื่อมต่อ SMS ใหม่ ให้คลิกปุ่ม "Set up" และเลือกผู้ให้บริการ SMS ที่ต้องการ
3. อ่านเอกสาร README สำหรับผู้ให้บริการที่คุณเลือก
4. กรอกข้อมูลที่จำเป็นในส่วน "Parameter Configuration"
5. ปรับแต่งเทมเพลต SMS ด้วย JSON editor
6. ทดสอบการตั้งค่าของคุณโดยส่งรหัสยืนยันไปยังโทรศัพท์ของคุณโดยใช้เทมเพลต "Generic"
7. คลิก "Save and Done" เพื่อเสร็จสิ้น

## เปลี่ยนตัวเชื่อมต่อ SMS \{#change-sms-connector}

หากต้องการเปลี่ยนไปใช้ตัวเชื่อมต่อ SMS อื่น:

- เข้าสู่หน้ารายละเอียดตัวเชื่อมต่อแล้วคลิก "Change connector" ที่มุมขวาบน
- หรือ ลบตัวเชื่อมต่อเดิมแล้วสร้างตัวใหม่

หมายเหตุ: คุณสามารถเปิดใช้งาน _ตัวเชื่อมต่อ SMS_ ได้เพียง **หนึ่งตัวเท่านั้น**

เมื่อคุณมีตัวเชื่อมต่อ SMS อยู่แล้ว คุณจะสามารถเลือกได้เฉพาะ "Change SMS connector" เท่านั้น เมื่อดำเนินการนี้ในคอนโซล การเปลี่ยนแปลงจะมีผลทันทีหลังจากบันทึก ดังนั้นโปรดทดสอบการตั้งค่าก่อนบันทึกทุกครั้ง

เพื่อความเสถียรของบริการ ผู้ใช้ Cloud สามารถสร้าง [dev tenant](/logto-cloud/tenant-settings#development) สำหรับสภาพแวดล้อมการพัฒนา หลังจากทดสอบการตั้งค่าในสภาพแวดล้อม dev แล้ว ให้ "migrate" การตั้งค่าไปยัง [production](/logto-cloud/tenant-settings#production) แล้วจึงบันทึกการตั้งค่า

## เปิดใช้งานหมายเลขโทรศัพท์สำหรับ end-user flows \{#enable-phone-number-for-end-user-flows}

เมื่อคุณเพิ่มและตั้งค่าตัวเชื่อมต่อ SMS แล้ว คุณสามารถเปิดใช้งาน flow การยืนยันตัวตนด้วยโทรศัพท์ได้ดังนี้:

- [**ตั้งค่าวิธีสมัครสมาชิก**](/end-user-flows/sign-up-and-sign-in/sign-up): ไปที่ <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Sign-in experience > Sign-up and sign-in</CloudLink> ในส่วนสมัครสมาชิก ให้เพิ่ม "Phone number" หรือ "Email address or phone number" เป็นตัวระบุ หมายเหตุ: การยืนยันหมายเลขโทรศัพท์เป็นสิ่งจำเป็นในขั้นตอนสมัครสมาชิก
- [**ตั้งค่าวิธีลงชื่อเข้าใช้**](/end-user-flows/sign-up-and-sign-in/sign-in): ในส่วนลงชื่อเข้าใช้ ให้เพิ่ม "Phone number" เป็นวิธีการลงชื่อเข้าใช้ เลือกใช้รหัสผ่านหรือรหัสยืนยันทาง SMS สำหรับการยืนยันตัวตน คลิก "Add another" เพื่อเพิ่มตัวเลือกอื่น ๆ
- [**ตั้งค่าลืมรหัสผ่าน**](/end-user-flows/sign-up-and-sign-in/reset-password): ในส่วนลงชื่อเข้าใช้ หากเปิดใช้งานการลงชื่อเข้าใช้ด้วยรหัสผ่าน คุณจะเห็นการตั้งค่า "Forgot password" ให้เปิดใช้งาน "Phone verification code" เพื่อให้สามารถรีเซ็ตรหัสผ่านผ่าน SMS ได้
- [**ตั้งค่า SMS สำหรับ MFA**](/end-user-flows/mfa/sms-mfa): ไปที่ <CloudLink to="/mfa">Sign-in experience > Multi-factor auth</CloudLink> เปิดใช้งาน "SMS verification code" เป็นปัจจัยที่สองเพื่อเพิ่มความปลอดภัย
- [**ตั้งค่าการจัดการบัญชี**](/end-user-flows/account-settings/by-account-api#manage-phone): ใช้ [Account API](/end-user-flows/account-settings/by-account-api) เพื่อปรับแต่ง Account Center ของคุณและให้ผู้ใช้จัดการหมายเลขโทรศัพท์ของตนเอง

:::warning
เนื่องจากการตั้งค่าของเรามีผลทันที การลบตัวเชื่อมต่อ SMS ที่มีอยู่จะส่งผลต่อกระบวนการเข้าสู่ระบบและสมัครสมาชิกของผู้ใช้ปลายทาง

ตัวอย่างเช่น หากตัวระบุการลงชื่อเข้าใช้รวม "หมายเลขโทรศัพท์ + รหัสยืนยัน" การลบตัวเชื่อมต่อ SMS ที่มีอยู่จะทำให้ผู้ใช้ไม่ได้รับรหัสยืนยันทางอีเมลและจะเกิดข้อผิดพลาดใน flow การลงชื่อเข้าใช้

โปรดระวังความเสี่ยงที่อาจเกิดขึ้นเมื่อดำเนินการกับตัวเชื่อมต่อ
:::
