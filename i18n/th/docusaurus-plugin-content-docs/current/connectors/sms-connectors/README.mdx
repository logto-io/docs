# ตัวเชื่อมต่อ SMS

การตั้งค่าตัวเชื่อมต่อ SMS ช่วยให้คุณสามารถส่ง [รหัสผ่านใช้ครั้งเดียว (OTP)](https://auth.wiki/otp) ไปยังหมายเลขโทรศัพท์ของผู้ใช้ได้ กลไกการยืนยันตัวตนแบบไม่ใช้รหัสผ่านนี้สามารถนำไปใช้ในหลากหลายสถานการณ์ เช่น [การสมัครสมาชิก](/end-user-flows/sign-up-and-sign-in/sign-up), [การลงชื่อเข้าใช้](/end-user-flows/sign-up-and-sign-in/sign-in), [ลืมรหัสผ่าน](/end-user-flows/sign-up-and-sign-in/reset-password), [กระบวนการเชื่อมโยงบัญชี](/end-user-flows/sign-up-and-sign-in/social-sign-in#account-linking), [เชิญสมาชิก](/end-user-flows/organization-experience/invite-organization-members) และ [ยืนยันตัวตนของผู้ใช้](/end-user-flows/security-verification) ช่วยให้กระบวนการยืนยันตัวตนของผู้ใช้สะดวกขึ้นและเพิ่มความปลอดภัยโดยลดความเสี่ยงจากการรั่วไหลที่เกี่ยวข้องกับรหัสผ่าน

## ขั้นตอนการตั้งค่า \{#configuration-steps}

ทำตามขั้นตอนด้านล่างเพื่อเพิ่มตัวเชื่อมต่อ SMS:

1. ไปที่ <CloudLink to="/connectors/passwordless">Console > Connector > ตัวเชื่อมต่ออีเมลและ SMS</CloudLink>
2. เพื่อเพิ่มตัวเชื่อมต่อ SMS ใหม่ ให้คลิกปุ่ม "Set up" และเลือกผู้ให้บริการ SMS ที่ต้องการ
3. อ่านเอกสาร README ของผู้ให้บริการที่เลือก
4. กรอกข้อมูลที่จำเป็นในส่วน "Parameter Configuration"
5. ปรับแต่งเทมเพลต SMS ด้วย JSON editor
6. ทดสอบการตั้งค่าของคุณโดยส่งรหัสยืนยันไปยังโทรศัพท์ของคุณโดยใช้เทมเพลต "Generic"
7. คลิก "Save and Done" เพื่อเสร็จสิ้น

## เปลี่ยนตัวเชื่อมต่อ SMS \{#change-sms-connector}

หากต้องการเปลี่ยนไปใช้ตัวเชื่อมต่อ SMS อื่น:

- เข้าสู่หน้ารายละเอียดตัวเชื่อมต่อและคลิก "Change connector" ที่มุมขวาบน
- หรือ ลบตัวเชื่อมต่อเดิมแล้วสร้างตัวใหม่

หมายเหตุ: คุณสามารถเปิดใช้งาน _ตัวเชื่อมต่อ SMS_ ได้เพียงหนึ่งตัวเท่านั้น

เมื่อคุณมีตัวเชื่อมต่อ SMS อยู่แล้ว จะสามารถเลือกได้แค่ "Change SMS connector" เท่านั้น เมื่อดำเนินการนี้ในคอนโซล การเปลี่ยนแปลงจะมีผลทันทีหลังจากบันทึก ดังนั้นโปรดทดสอบการตั้งค่าก่อนบันทึกทุกครั้ง

เพื่อความเสถียรของบริการ ผู้ใช้ Cloud สามารถสร้าง [dev tenant](/logto-cloud/tenant-settings#development) สำหรับสภาพแวดล้อมการพัฒนา หลังจากทดสอบการตั้งค่าในสภาพแวดล้อม dev แล้ว ให้ "migrate" การตั้งค่าไปยัง [production](/logto-cloud/tenant-settings#production) แล้วจึงบันทึกการตั้งค่า

## เปิดใช้งานการสมัครหรือเข้าสู่ระบบด้วยหมายเลขโทรศัพท์ \{#enable-phone-number-sign-up-or-sign-in}

เมื่อคุณเพิ่มตัวเชื่อมต่อ SMS และตั้งค่าเสร็จสมบูรณ์แล้ว คุณสามารถเปิดใช้งานการตั้งค่าการลงชื่อเข้าใช้และสมัครสมาชิกที่ใช้หมายเลขโทรศัพท์ + รหัสยืนยัน ตามความต้องการของธุรกิจคุณ

1. **ไปที่**: <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Console > Sign-in experience > Sign-up and sign-in</CloudLink>
2. [**ตั้งค่าวิธีสมัครสมาชิก**](/end-user-flows/sign-up-and-sign-in/sign-up): เลือก "หมายเลขโทรศัพท์" หรือ "อีเมลหรือหมายเลขโทรศัพท์" เป็นตัวระบุการสมัครสมาชิก หมายเหตุ: ปัจจุบันการยืนยันหมายเลขโทรศัพท์เป็นสิ่งจำเป็นในขั้นตอนสมัครสมาชิก
3. [**ตั้งค่าวิธีลงชื่อเข้าใช้**](/end-user-flows/sign-up-and-sign-in/sign-in): ตัวระบุที่เลือกสำหรับสมัครสมาชิกจะกลายเป็นวิธีลงชื่อเข้าใช้โดยอัตโนมัติ คลิก "Add another" เพื่อเพิ่มตัวเลือกการลงชื่อเข้าใช้อื่น
4. **บันทึกและทดสอบ:** คลิก "Save changes" เพื่อใช้งานประสบการณ์ลงชื่อเข้าใช้ใหม่ อย่าลืมใช้ "[Live preview](/customization/live-preview)" เพื่อทดสอบฟังก์ชัน

:::warning
เนื่องจากการตั้งค่าของเรามีผลทันที การลบตัวเชื่อมต่อ SMS ที่มีอยู่จะส่งผลต่อกระบวนการเข้าสู่ระบบและสมัครสมาชิกของผู้ใช้ปลายทาง

ตัวอย่างเช่น หากตัวระบุการลงชื่อเข้าใช้รวม "หมายเลขโทรศัพท์ + รหัสยืนยัน" การลบตัวเชื่อมต่อ SMS ที่มีอยู่จะทำให้ผู้ใช้ไม่ได้รับรหัสยืนยันทางอีเมลและเกิดข้อผิดพลาดในขั้นตอนการลงชื่อเข้าใช้

โปรดระวังความเสี่ยงที่อาจเกิดขึ้นเมื่อดำเนินการกับตัวเชื่อมต่อ
:::
