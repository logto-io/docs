Logto มีบริการอีเมลในตัวเพื่อความสะดวกของคุณในสถานการณ์ต่อไปนี้:

1. สำรวจหรือทดสอบประสบการณ์การเข้าสู่ระบบด้วยอีเมลของ Logto ได้อย่างรวดเร็ว
2. ใช้งานโดยตรงกับผลิตภัณฑ์ออนไลน์ของคุณ เหมาะสำหรับสตาร์ทอัพใหม่ที่สะดวกใช้ `logto.email` เป็นโดเมนอีเมลผู้ส่ง

คุณสมบัติของบริการอีเมล Logto:

- **ใช้ฟรี:** ไม่มีค่าใช้จ่ายและไม่มีขีดจำกัดการใช้งานอีเมลต่อวัน ช่วยประหยัดต้นทุนของคุณ
- **ง่ายดาย:** ไม่ต้องตั้งค่ากับผู้ให้บริการอีเมลภายนอก เพียงปรับแต่งข้อมูลแบรนด์พื้นฐานสำหรับเทมเพลตอีเมล หากคุณยังไม่มีข้อมูลแบรนด์ของตนเอง สามารถเริ่มใช้งานได้ในไม่กี่คลิก
- **การส่งถึงมือผู้ใช้:** ด้วยบริการอีเมลของ Logto คุณจะได้รับบริการที่เสถียรและการส่งอีเมลที่เชื่อถือได้ เพื่อให้แน่ใจว่าผู้ใช้สามารถเข้าถึงผลิตภัณฑ์ของคุณได้

อย่างไรก็ตาม แม้จะสะดวกแต่มีข้อจำกัดบางประการที่ควรทราบ:

1. อีเมลจะถูกส่งจากที่อยู่อีเมลคงที่ `no-reply@logto.email`
2. ไม่สามารถเพิ่มลิงก์หรือเนื้อหากำหนดเองอื่น ๆ ลงในอีเมลได้

ขึ้นอยู่กับความต้องการทางธุรกิจที่เปลี่ยนแปลง คุณสามารถเลือกใช้ผู้ให้บริการอีเมลอื่นในภายหลังได้ เรามี [ตัวเชื่อมต่อบริการอีเมลสำเร็จรูป](/connectors/email-connectors#popular-email-providers) ให้เลือกหลากหลาย และยังรองรับ [SMTP](/integrations/smtp), [HTTP](/integrations/http-email) และ [Webhook](/developers/webhooks) สำหรับการส่งอีเมล คุณจึงมั่นใจได้ว่าจะพบวิธีที่เหมาะสมกับคุณ

:::note
บริการอีเมลฟรีในตัวของ Logto ขณะนี้มีให้เฉพาะผู้ใช้ [Cloud](https://cloud.logto.io/) เท่านั้น สำหรับผู้ใช้บริการโอเพ่นซอร์ส คุณสามารถตั้งค่าผู้ให้บริการอีเมลของคุณเองสำหรับการเข้าสู่ระบบด้วยอีเมลได้อย่างอิสระ
:::

## ขั้นตอนการตั้งค่า \{#configuration-steps}

ทำตามขั้นตอนเหล่านี้เพื่อกำหนดค่าบริการอีเมล Logto:

1. ไปที่ <CloudLink to="/connectors/passwordless">Connector > Email and SMS connectors</CloudLink>
2. หากต้องการเพิ่มตัวเชื่อมต่ออีเมลใหม่ ให้คลิกปุ่ม "**Set up**" และเลือกตัวเชื่อมต่อ "**Logto email service**"
3. เมื่อสร้างตัวเชื่อมต่อ "Logto email service" สำเร็จแล้ว คุณสามารถปรับแต่งข้อมูลแบรนด์พื้นฐานที่แสดงในเทมเพลตอีเมลได้
4. หลังจากแก้ไขข้อมูลแล้ว อย่าลืมส่งอีเมลทดสอบไปยังอีเมลของคุณก่อนบันทึกการเปลี่ยนแปลง

ตัวเลือกการปรับแต่ง:

- **From email:** อีเมลผู้ส่งถูกตั้งค่าเป็น `no-reply@logto.email` และไม่สามารถแก้ไขได้
- **Sender name:** กำหนดชื่อแบรนด์ของคุณเป็นชื่อผู้ส่ง เพื่อให้ผู้ใช้จดจำได้
- **ข้อมูลบริษัท:** แสดงชื่อบริษัท ที่อยู่ หรือรหัสไปรษณีย์ เพื่อเสริมความน่าเชื่อถือและเป็นไปตามข้อกำหนด _ไม่อนุญาตให้ใส่ URL_
- **โลโก้แอป:** อัปโหลดโลโก้แบรนด์ของแอป เพื่อให้คุณค่าสินค้าแสดงในอีเมลที่ผู้ใช้ได้รับ

## เทมเพลตอีเมลแบบรวมศูนย์ \{#unified-email-templates}

บริการอีเมล Logto ใช้เทมเพลตอีเมลแบบรวมศูนย์ที่ออกแบบมาเฉพาะสำหรับแต่ละสถานการณ์การยืนยันตัวตน:

| การใช้งาน                | สถานการณ์                                                                                                                                                                                                                                                                |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Register                 | ผู้ใช้สร้างบัญชีด้วยอีเมลและยืนยันโดยกรอกรหัสยืนยันที่ Logto ส่งไปยังอีเมลของตน                                                                                                                                                                                          |
| SignIn                   | ผู้ใช้เข้าสู่ระบบด้วยอีเมลและยืนยันโดยกรอกรหัสยืนยันแทนการกรอกรหัสผ่าน                                                                                                                                                                                                   |
| ForgotPassword           | หากผู้ใช้ลืมรหัสผ่านขณะเข้าสู่ระบบ สามารถเลือกยืนยันตัวตนด้วยอีเมลที่เคยยืนยันกับ Logto แล้ว                                                                                                                                                                             |
| Generic                  | เทมเพลตนี้ใช้เป็นตัวเลือกสำรองทั่วไปสำหรับหลายสถานการณ์ รวมถึงการทดสอบการตั้งค่าตัวเชื่อมต่อ ฯลฯ                                                                                                                                                                         |
| OrganizationInvitation   | ใช้เทมเพลตนี้เพื่อส่งลิงก์เชิญผู้ใช้เข้าร่วมองค์กร                                                                                                                                                                                                                       |
| UserPermissionValidation | ระหว่างการใช้งานแอป อาจมีบางการดำเนินการที่มีความเสี่ยงสูงหรือค่อนข้างสูงซึ่งต้องการการยืนยันผู้ใช้เพิ่มเติม เช่น โอนเงิน ลบทรัพยากรที่ใช้งานอยู่ หรือยกเลิกสมาชิก เทมเพลต `UserPermissionValidation` ใช้กำหนดเนื้อหาอีเมลรหัสยืนยันที่ผู้ใช้จะได้รับในสถานการณ์เหล่านี้ |
| BindNewIdentifier        | เมื่อผู้ใช้แก้ไขโปรไฟล์ อาจผูกอีเมลใหม่กับบัญชีปัจจุบัน ในกรณีนี้ใช้เทมเพลต `BindNewIdentifier` เพื่อปรับแต่งเนื้อหาอีเมลยืนยัน                                                                                                                                          |
| MfaVerification          | เมื่อเปิดใช้งาน MFA ทางอีเมล เทมเพลตนี้ใช้ส่งรหัสยืนยันให้ผู้ใช้ระหว่างกระบวนการยืนยันตัวตนหลายปัจจัย                                                                                                                                                                    |
| BindMfa                  | เมื่อเปิดใช้งาน MFA ทางอีเมล เทมเพลตนี้ใช้สำหรับตั้งค่ารหัสยืนยัน MFA ผู้ใช้จะได้รับรหัสนี้เมื่อผูกหรือกำหนดอีเมลเป็นปัจจัย MFA ให้กับบัญชีของตน                                                                                                                         |

ตัวอย่างเทมเพลตอีเมลสำหรับการใช้งานประเภท "Register" พร้อมข้อมูลแบรนด์ที่กำหนดเอง:

## คำถามที่พบบ่อย \{#faqs}

<summary>

### สามารถเพิ่ม CSS หรือ HTML กำหนดเองในบริการอีเมล Logto ได้หรือไม่ \{#is-it-possible-to-add-custom-css-or-html-to-logto-email-service}

</summary>

บริการอีเมลในตัวของ Logto ไม่รองรับ CSS หรือ HTML กำหนดเอง คุณสามารถปรับแต่งได้เฉพาะองค์ประกอบแบรนด์ทั่วไปเท่านั้น ข้อจำกัดนี้มีไว้เพื่อรักษาเสถียรภาพของบริการอีเมลในตัว เนื่องจากผู้เช่าทั้งหมดใช้ IP และที่อยู่อีเมลผู้ส่งเดียวกัน ดูรายละเอียดเพิ่มเติมได้ที่ "[ปัจจัยที่ช่วยปรับปรุงการส่งอีเมล](https://blog.logto.io/verification-email-delivery#factors-to-improve-email-delivery)"

หากต้องการปรับแต่งเทมเพลตอีเมล แนะนำให้ใช้ตัวเชื่อมต่ออีเมลอื่น เช่น AWS Direct Mail, SendGrid, Mailgun, Postmark หรือ SMTP

</details>
