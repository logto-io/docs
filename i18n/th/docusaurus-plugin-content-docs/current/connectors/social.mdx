# ตัวเชื่อมต่อโซเชียล

ทำให้การเริ่มต้นใช้งานของผู้ใช้เป็นเรื่องง่ายและเพิ่มอัตราการแปลงด้วยการเปิดใช้งาน [เข้าสู่ระบบโซเชียล](/end-user-flows/sign-up-and-sign-in/social-sign-in) ด้วย Logto ผู้ใช้สามารถลงชื่อเข้าใช้อย่างรวดเร็วและปลอดภัยโดยใช้บัญชีโซเชียลมีเดียที่มีอยู่ โดยไม่ต้องสร้างรหัสผ่านหรือผ่านขั้นตอนลงทะเบียนที่ซับซ้อน Logto มีตัวเชื่อมต่อโซเชียลสำเร็จรูปหลากหลายและรองรับการเชื่อมต่อแบบกำหนดเองเพื่อความยืดหยุ่นสูงสุด

## เลือกตัวเชื่อมต่อโซเชียลของคุณ \{#choose-your-social-connectors}

## ขั้นตอนการตั้งค่า \{#configuration-steps}

1. ไปที่ <CloudLink to="/connectors/social">Console > ตัวเชื่อมต่อ > ตัวเชื่อมต่อโซเชียล</CloudLink>
2. คลิก "เพิ่มตัวเชื่อมต่อโซเชียล" และเลือกประเภทที่ต้องการ
3. ทำตามคู่มือ README กรอกข้อมูลที่จำเป็นและปรับแต่งการตั้งค่า
4. คลิก "บันทึกและเสร็จสิ้น"
5. ทดสอบตัวเชื่อมต่อโดยเริ่มต้นเข้าสู่ระบบโซเชียล

โปรดทราบการตั้งค่าต่อไปนี้:

- **ชื่อผู้ให้บริการข้อมูลระบุตัวตน (Identity provider name)**: ตัวเชื่อมต่อโซเชียลแต่ละตัวจะมีชื่อผู้ให้บริการข้อมูลระบุตัวตน (IdP) เฉพาะเพื่อแยกแยะตัวตนของผู้ใช้ ตัวเชื่อมต่อทั่วไปจะใช้ชื่อ IdP คงที่ ส่วนตัวเชื่อมต่อแบบกำหนดเองต้องใช้ค่าที่ไม่ซ้ำกัน ดูข้อมูลเพิ่มเติมเกี่ยวกับ [ชื่อ IdP](/connectors/connector-data-structure#target-identity-provider-name)
- **ซิงค์โปรไฟล์ผู้ใช้ (Sync user profiles)**: เลือกเวลาที่จะซิงค์ [ข้อมูลโปรไฟล์ผู้ใช้](/user-management/user-data#social-identities) (เช่น รูปโปรไฟล์, ชื่อผู้ใช้) โดยค่าเริ่มต้นคือ "ซิงค์เฉพาะตอนลงทะเบียน" "ซิงค์ทุกครั้งที่ลงชื่อเข้าใช้" เป็นอีกทางเลือกหนึ่งแต่ข้อมูลผู้ใช้อาจถูกเขียนทับ
- **เปิดใช้งานการจัดเก็บโทเค็น (Enable token storage)**: สำหรับตัวเชื่อมต่อโซเชียลที่รองรับ คุณสามารถเปิดใช้งานการจัดเก็บโทเค็นเพื่อบันทึกโทเค็นการเข้าถึงและโทเค็นรีเฟรชอย่างปลอดภัยใน [Secret Vault](/secret-vault) ของ Logto เมื่อผู้ใช้ลงชื่อเข้าใช้ด้วยผู้ให้บริการโซเชียล วิธีนี้ช่วยให้แอปของคุณเข้าถึง API ของบุคคลที่สามในนามของผู้ใช้โดยไม่ต้องให้ผู้ใช้ยืนยันตัวตนซ้ำ ดูข้อมูลเพิ่มเติมเกี่ยวกับ [การจัดเก็บโทเค็นแบบ Federated](/secret-vault/federated-token-set)

## เปิดใช้งานการเข้าสู่ระบบโซเชียล \{#enable-social-sign-in}

เมื่อคุณสร้างตัวเชื่อมต่อโซเชียลสำเร็จแล้ว คุณสามารถเปิดใช้งานเป็นปุ่มเข้าสู่ระบบโซเชียล (เช่น ดำเนินการต่อด้วย Google) ในประสบการณ์การลงชื่อเข้าใช้

1. ไปที่ <CloudLink to="/sign-in-experience/sign-up-and-sign-in">Console > ประสบการณ์การลงชื่อเข้าใช้ > ลงทะเบียนและลงชื่อเข้าใช้</CloudLink>
2. (ไม่บังคับ) เลือก "ไม่ระบุ" สำหรับตัวระบุการลงทะเบียนหากต้องการใช้เฉพาะเข้าสู่ระบบโซเชียล
3. เพิ่มตัวเชื่อมต่อโซเชียลที่ตั้งค่าไว้ในส่วน "เข้าสู่ระบบโซเชียล"
4. จัดเรียงลำดับตัวเชื่อมต่อใหม่ตามต้องการ
5. คลิก "บันทึกการเปลี่ยนแปลง" และทดสอบ "ดูตัวอย่างสด"

ดูรายละเอียดเพิ่มเติมได้ที่ [เข้าสู่ระบบโซเชียล](/end-user-flows/sign-up-and-sign-in/social-sign-in)
