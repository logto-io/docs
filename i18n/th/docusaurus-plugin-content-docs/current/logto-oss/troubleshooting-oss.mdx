# การแก้ไขปัญหาสำหรับ OSS

<details>

<summary>

### เบราว์เซอร์ของฉันไม่สามารถโหลด Admin Console (แสดงข้อผิดพลาด `Crypto.subtle is unavailable...`) \{#my-browser-cannot-load-admin-console-showing-error-cryptosubtle-is-unavailable}

</summary>

Admin Console ใช้ [Web Crypto API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Crypto_API) ซึ่งต้องการ [secure contexts](https://developer.mozilla.org/en-US/docs/Web/Security/Secure_Contexts) เช่น HTTPS หรือ HTTP กับ `localhost`

หากคุณใช้ HTTP กับ IP address หรือโดเมนที่กำหนดเอง เบราว์เซอร์จะไม่สามารถโหลด Admin Console ได้

</details>

<details>

<summary>

### ฉันใช้โดเมนที่กำหนดเอง แต่เบราว์เซอร์ของฉันไม่สามารถโหลด Admin Console (แสดงข้อผิดพลาด `"code": "oidc.invalid_redirect_uri"`) \{#im-using-a-custom-domain-but-my-browser-cannot-load-admin-console-showing-error-code-oidcinvalid_redirect_uri}

</summary>

หากคุณใช้โดเมนที่กำหนดเองแทน `localhost` คุณต้องตั้งค่าตัวแปรสภาพแวดล้อม `ENDPOINT` ให้เป็น URL ของ Logto เนื่องจากข้อกำหนดที่เข้มงวดของ Redirect URI ใน OIDC ดูรายละเอียดที่ [การตั้งค่า](/concepts/core-service/configuration/)

</details>

<details>

<summary>

### ฉันใช้ HTTPS proxy (เช่น Nginx) หน้า Logto แต่ไม่สามารถลงชื่อเข้าใช้ได้ (แสดงข้อผิดพลาด `TypeError: Failed to fetch`) \{#im-using-an-https-proxy-eg-nginx-in-front-of-logto-but-i-failed-on-sign-in-showing-error-typeerror-failed-to-fetch}

</summary>

- ขั้นแรก ตรวจสอบว่าคุณได้ตั้งค่าตัวแปรสภาพแวดล้อมของ node `TRUST_PROXY_HEADER` เป็น `true` แล้ว ดูรายละเอียดที่ [การตั้งค่า](/concepts/core-service/configuration/#using-a-https-proxy)
- คุณต้องตั้งค่า header `X-Forwarded-Proto` เป็น `https` ในการตั้งค่า proxy ของคุณด้วย ดูรายละเอียดที่ [Trusting TLS offloading proxies](https://github.com/panva/node-oidc-provider/blob/main/docs/README.md#trusting-tls-offloading-proxies)

</details>

<details>

<summary>

### ฉันใช้ PgBouncer หรือ RDS Proxy แล้ว Logto เชื่อมต่อไม่สำเร็จ (error `unsupported startup parameter: statement_timeout`) \{#im-using-pgbouncer-or-rds-proxy-and-logto-fails-to-connect-error-unsupported-startup-parameter-statement-timeout}

</summary>

พร็อกซีฐานข้อมูลบางตัวจะปฏิเสธพารามิเตอร์เริ่มต้น `statement_timeout` ตั้งแต่ Logto v1.36.0 คุณสามารถปิดการส่งพารามิเตอร์นี้ได้โดยตั้งค่า:

```
DATABASE_STATEMENT_TIMEOUT=DISABLE_TIMEOUT
```

หากคุณต้องการตั้งค่า timeout เอง ให้ระบุเป็นตัวเลข (มิลลิวินาที) เช่น `5000` หากไม่ได้ตั้งค่าหรือไม่ถูกต้อง Logto จะใช้ค่าเริ่มต้นของ client คือ 60000 ms

</details>

<details>

<summary>

### ฉันพบปัญหา CORS \{#im-facing-cors-issues}

</summary>

- หากไม่ได้ระบุ `ADMIN_ENDPOINT` จะอนุญาตให้ `localhost:[admin-port]` ทำ Cross-Origin Resource Sharing (CORS) ใน Logto ได้
- หากระบุ `ADMIN_ENDPOINT` แล้ว จะอนุญาตเฉพาะคำขอจาก origin ของ `ADMIN_ENDPOINT` เท่านั้น

</details>

<details>

<summary>

### นี่คืออะไรอีกนะ? `Error: Invalid id token` \{#what-is-this-again-error-invalid-id-token}

</summary>

หากคุณไม่ได้ยุ่งกับโทเค็นของคุณ ส่วนใหญ่เกิดจากเวลาของเซิร์ฟเวอร์กับ client ไม่ตรงกัน ให้ซิงค์เวลาในเซิร์ฟเวอร์และ client แล้วลองใหม่

</details>

<details>

<summary>

### endpoint ของ Management API สำหรับ Logto OSS แตกต่างจากเวอร์ชัน Cloud หรือไม่? \{#does-the-management-api-endpoint-for-logto-oss-differ-from-the-cloud-version}

</summary>

ใช่ ตัวระบุ Management API ของ Logto แตกต่างกันระหว่าง Logto Cloud และเวอร์ชัน self-hosted (Open Source):

- Logto Cloud: `https://[tenant-id].logto.app/api`

- Logto Open Source: `https://default.logto.app/api`

</details>
