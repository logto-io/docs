# Logto OSS

Logto มีทั้งเวอร์ชันโอเพ่นซอร์ส (OSS) และ [คลาวด์](/logto-cloud) ส่วนนี้จะเน้นไปที่โซลูชัน OSS: <Url href="https://github.com/logto-io/logto">github.com/logto-io/logto</Url>

Logto OSS เป็นโซลูชันข้อมูลระบุตัวตนที่ใช้งานได้ฟรี ให้ความสามารถในการปรับแต่งอย่างเต็มที่และควบคุมข้อมูลของคุณได้อย่างสมบูรณ์ Logto OSS มีความยืดหยุ่นในการปรับแต่งโค้ดให้ตรงกับความต้องการเฉพาะ เหมาะสำหรับผู้ที่ให้ความสำคัญกับอธิปไตยของข้อมูลและต้องการโซลูชันผู้ให้บริการข้อมูลระบุตัวตนที่สามารถเติบโตไปพร้อมกับแอปพลิเคชันของตน

## ฟีเจอร์ที่รองรับโดย Logto OSS \{#feature-supported-by-logto-oss}

Logto OSS รองรับความสามารถหลักส่วนใหญ่ของบริการ Logto และมีการอัปเดตและดูแลอย่างสม่ำเสมอ

ฟีเจอร์ขั้นสูงบางอย่างมีเฉพาะในเวอร์ชัน Logto Cloud เท่านั้น ได้แก่:

| ข้อจำกัดของฟีเจอร์ใน OSS                                                         | คำอธิบาย                                                                                                                                     |
| -------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| [Logto console: หลาย tenant](/logto-cloud/tenant-settings)                       | สร้างและจัดการ Logto tenant หลายรายการในคอนโซลเดียว                                                                                          |
| [Logto console: เชิญผู้ร่วมงาน](/logto-cloud/tenant-member-management)           | เชิญและจัดการสมาชิกหลายคนใน tenant เดียวของ Logto                                                                                            |
| Logto console: เปิดใช้งาน MFA                                                    | เพิ่มความปลอดภัยโดยบังคับใช้การยืนยันตัวตนหลายปัจจัย (MFA) เมื่อเข้าสู่ Logto Console                                                        |
| [Logto Protected App](/integrate-logto/protected-app)                            | การผสานการยืนยันตัวตนแบบรวดเร็วและไม่ต้องใช้ SDK ด้วย Cloudflare                                                                             |
| [บริการอีเมลในตัวของ Logto](/connectors/email-connectors/built-in-email-service) | ใช้บริการอีเมลในตัวและฟรีสำหรับการส่งอีเมล                                                                                                   |
| [Bring your UI](/customization/bring-your-ui)                                    | ปรับแต่ง UI หรือ flow ของประสบการณ์การลงชื่อเข้าใช้ด้วยฟีเจอร์นี้ (OSS สามารถ fork โค้ดบน GitHub เพื่อปรับแต่งประสบการณ์ลงชื่อเข้าใช้โดยตรง) |
| [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso)            | เปิดใช้งาน enterprise single sign-on ที่เริ่มต้นโดยผู้ให้บริการข้อมูลระบุตัวตน (IdP)                                                         |
| [SAML apps](/integrate-logto/saml-app)                                           | ผสานแอปพลิเคชันผ่านโปรโตคอล SAML เวอร์ชัน OSS จำกัดที่ 3 SAML แอป                                                                            |
| [ซ่อนแบรนด์ Logto](/customization/match-your-brand#hide-logto-branding)          | ลบเครื่องหมาย "Powered by Logto" ออกจากประสบการณ์การลงชื่อเข้าใช้                                                                            |

เคล็ดลับ: Multi-tenancy, การเชิญสมาชิก และ MFA ไม่สามารถใช้กับทีมของคุณในการลงชื่อเข้าใช้ Logto console แบบโอเพ่นซอร์สได้ อย่างไรก็ตาม คุณสามารถนำฟีเจอร์เหล่านี้ไปใช้ในผลิตภัณฑ์ของคุณเองด้วย Logto OSS เพื่อให้ผู้ใช้ปลายทางของคุณใช้งานได้
