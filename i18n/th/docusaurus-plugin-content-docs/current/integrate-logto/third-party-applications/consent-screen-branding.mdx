# การปรับแต่งแบรนด์หน้าขอความยินยอม (Consent screen)

สิ่งสำคัญคือการแสดงข้อมูลแบรนด์และลิงก์ข้อกำหนดของบุคคลที่สามอย่างถูกต้องแก่ผู้ใช้ เมื่อพวกเขาถูกเปลี่ยนเส้นทางไปยังหน้าขอความยินยอมของแอปพลิเคชันบุคคลที่สาม

Logto อนุญาตให้คุณปรับแต่งข้อมูลแบรนด์ของแอปพลิเคชันบุคคลที่สามของคุณ รวมถึงชื่อแอป โลโก้ และลิงก์ข้อกำหนด

## ปรับแต่งข้อมูลแบรนด์ \{#customize-the-branding-information}

โปรดกำหนดค่าข้อมูลแบรนด์ของแอปพลิเคชันบุคคลที่สามของคุณให้เหมาะสม เพื่อให้ประสบการณ์การยืนยันตัวตน (Authentication) ของผู้ใช้มีความสอดคล้องและปลอดภัย

1. ไปที่ <CloudLink to="/applications/third-party-applications">Console > Application > Third-party apps</CloudLink> และเปิดหน้ารายละเอียดของแอปพลิเคชัน OIDC บุคคลที่สามที่ต้องการ

2. ไปที่แท็บ **Branding**

3. กำหนดค่าข้อมูลที่จะแสดงบนหน้าขอความยินยอม:

- **Display name**: ชื่อของแอปพลิเคชันบุคคลที่สามที่จะปรากฏบนหน้าขอความยินยอม โดยจะแสดงชื่อของแอปที่ร้องขอเข้าถึงข้อมูลของผู้ใช้คุณ หากเว้นว่างไว้จะใช้ **Application name**
- **App logo (Light)**: โลโก้ของแอปพลิเคชันบุคคลที่สามที่จะปรากฏบนหน้าขอความยินยอม โดยจะแสดงแบรนด์ของแอปที่ร้องขอเข้าถึงข้อมูลของผู้ใช้คุณ หากมีการกำหนดโลโก้ทั้งของแอปบุคคลที่สามและโลโก้ประสบการณ์การลงชื่อเข้าใช้แบบรวม (universal sign-in-experience) ทั้งสองจะถูกแสดงบนหน้าขอความยินยอม
- **App logo (Dark)**: ใช้ได้เฉพาะเมื่อเปิดใช้งานประสบการณ์การลงชื่อเข้าใช้แบบโหมดมืด (dark-mode) จัดการการตั้งค่า dark-mode ได้ที่หน้า <CloudLink to="/sign-in-experience/branding">Console > Sign-in & account > Branding</CloudLink>
- **Terms of use URL**: ลิงก์ข้อกำหนดการใช้งานของแอปพลิเคชันบุคคลที่สามที่จะแสดงบนหน้าขอความยินยอม
- **Privacy policy URL**: ลิงก์นโยบายความเป็นส่วนตัวของแอปพลิเคชันบุคคลที่สามที่จะแสดงบนหน้าขอความยินยอม
