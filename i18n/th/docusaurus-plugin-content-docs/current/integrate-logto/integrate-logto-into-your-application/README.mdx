# ผสานรวม Logto เข้ากับแอปพลิเคชันของคุณ

ทำตามขั้นตอนเหล่านี้เพื่อเพิ่มการยืนยันตัวตนให้กับแอปพลิเคชันของคุณด้วย Logto ไม่ว่าจะเป็นแอปสำหรับผู้ใช้หรือบริการเครื่องต่อเครื่อง (machine-to-machine):

1. ไปที่ <CloudLink to="/applications">Console > Applications</CloudLink>

2. คลิก "Create application" เพื่อเพิ่มแอปพลิเคชันใหม่

3. เลือก [เฟรมเวิร์กแอปพลิเคชัน](/quick-starts) ของคุณเพื่อเริ่มต้น หากไม่พบเฟรมเวิร์กของคุณ ให้คลิกปุ่ม "Create app without framework" ที่มุมขวาล่างของหน้าสร้างแอปพลิเคชันเพื่อสร้างแอปโดยเลือก [ประเภทแอปพลิเคชัน](/integrate-logto/application-data-structure/#application-types) หรือส่งคำขอฟีเจอร์หรือร่วมพัฒนา SDK ตาม [แนวทาง SDK ของเรา](/developers/sdk-conventions)

4. หลังจากเลือกเฟรมเวิร์กแล้ว คุณจะเห็นคู่มือเริ่มต้นอย่างรวดเร็วสำหรับ SDK ของเฟรมเวิร์กนั้น ให้ทำตามขั้นตอนเพื่อกำหนดค่าและผสานรวมแอปพลิเคชันของคุณ หากต้องการความเข้าใจเกี่ยวกับแนวคิดที่เกี่ยวข้องกับกระบวนการผสานรวม สามารถดูได้ที่ [ทำความเข้าใจ flow การยืนยันตัวตนของ Logto](/integrate-logto/integrate-logto-into-your-application/understand-authentication-flow/) เพื่อเข้าใจเชิงลึกเกี่ยวกับการผสานรวม

:::note
คู่มือใน console นี้เหมาะสำหรับเริ่มต้นอย่างรวดเร็วกับ Logto โดยใช้ SDK ของเราเท่านั้น สำหรับคู่มือการผสานรวมแบบสมบูรณ์ รวมถึงการใช้งาน SDK ขั้นสูง โปรดดูที่ส่วน [Quick starts](/quick-starts)
:::

5. เมื่อเสร็จสิ้นแล้ว คุณก็พร้อมที่จะสำรวจเพิ่มเติมเกี่ยวกับ Logto:

   <Url href="/end-user-flows">ประสบการณ์ (Experience) ของผู้ใช้ปลายทาง</Url>
   <Url href="/authorization">การอนุญาต (Authorization)</Url>
   <Url href="/organizations">องค์กร (Organizations)</Url>

## คำถามที่พบบ่อย \{#faqs}

<details>
  <summary>
    ### บริการ backend ของฉันจะตรวจสอบโทเค็นผู้ใช้และจัดการผู้ใช้กับ Logto ได้อย่างไร?
  </summary>
เพื่อยืนยันความถูกต้องของโทเค็นการเข้าถึง (access tokens) ใน API backend ของคุณ (เช่น Python, Node.js, Go, Java, PHP ฯลฯ) และเพื่อจัดการผู้ใช้แบบโปรแกรม โปรดดูคู่มือ: [วิธีตรวจสอบโทเค็นการเข้าถึงในบริการ API หรือ backend ของคุณ](/authorization/validate-access-tokens)

เอกสารนี้ครอบคลุม:

- วิธีตรวจสอบความถูกต้องของ bearer token ในทุกการเรียก API
- แนวปฏิบัติที่ดีที่สุดสำหรับการผสาน Logto กับแอป frontend หลายตัวและบริการ backend

</details>
