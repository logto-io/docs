# ผสานรวมการยืนยันตัวตน Logto

:::tip[ใช้ AI เพื่อผสานรวม (Logto Cloud)]
หากคุณใช้เครื่องมือพัฒนาแบบขับเคลื่อนด้วย AI ลองใช้ [Logto MCP Server](/logto-cloud/logto-mcp-server) เพื่อผสานรวม Logto ด้วยความช่วยเหลือจาก AI โดยจะตรวจจับ framework ของคุณ สร้างแอปพลิเคชัน และสร้างโค้ดที่ใช้งานได้จริง
:::

Logto มอบโซลูชันการยืนยันตัวตน (Authentication) ที่ครบถ้วนสำหรับแอปพลิเคชันเว็บ มือถือ และเดสก์ท็อป รองรับ [การยืนยันตัวตนระหว่างเครื่อง (Machine-to-Machine; M2M)](/quick-starts/m2m) ระหว่างบริการ, [device flow](/quick-starts/device-flow) สำหรับอุปกรณ์ที่มีข้อจำกัดด้านการป้อนข้อมูล และสามารถทำหน้าที่เป็นผู้ให้บริการข้อมูลระบุตัวตน (IdP) สำหรับ [แอปพลิเคชันของบุคคลที่สาม](/integrate-logto/third-party-applications) ผ่านโปรโตคอลมาตรฐาน เช่น [OpenID Connect (OIDC)](https://auth.wiki/openid-connect) และ [OAuth 2.0](https://auth.wiki/oauth-2.0)

เริ่มต้นการผสานรวมของคุณโดยเลือกโซลูชันที่ตรงกับความต้องการมากที่สุด:

## เพิ่มการยืนยันตัวตนให้กับแอปพลิเคชันของคุณ \{#add-authentication-for-your-applications}

ไม่ว่าคุณจะสร้างแอปพลิเคชันสำหรับผู้ใช้ปลายทาง (เช่น เว็บ มือถือ หรือเดสก์ท็อป) หรือแอปพลิเคชันแบบเครื่องต่อเครื่อง (M2M) สำหรับการสื่อสารระหว่างบริการ คุณสามารถนำเสนอฟีเจอร์ [การยืนยันตัวตน (Authentication)](/end-user-flows) และ [การจัดการผู้ใช้](/user-management) ได้อย่างรวดเร็วด้วยการผสานรวม Logto

Logto สร้างขึ้นบนมาตรฐาน OIDC ทำให้คุณสามารถใช้งาน **Omni sign-in** ข้ามทุกแอปพลิเคชันของคุณ เมื่อคุณผสานรวมหลายแอปกับ Logto พวกเขาจะใช้ระบบเอกลักษณ์และวิธีการยืนยันตัวตนเดียวกัน ซึ่งหมายความว่าผู้ใช้สามารถลงชื่อเข้าใช้เพียงครั้งเดียวและเข้าถึงทุกแอปที่เชื่อมต่อของคุณได้อย่างไร้รอยต่อด้วยประสบการณ์การยืนยันตัวตนแบบรวมศูนย์

ค้นหาคู่มือการผสานรวมสำหรับ framework หรือภาษาโปรแกรมที่คุณต้องการ:

## เพิ่มแอป SAML \{#add-saml-apps}

Logto (ในฐานะผู้ให้บริการข้อมูลระบุตัวตน) รองรับการผสานรวมกับแอปพลิเคชันที่ใช้โปรโตคอล SAML (ในฐานะ Service Provider) ผ่านการผสานรวมแอป SAML คุณสามารถมอบประสบการณ์ Single Sign-On (SSO) ที่ปลอดภัยและเป็นมาตรฐานให้กับผู้ใช้ระดับองค์กร

## เพิ่มแอป OIDC/OAuth ของบุคคลที่สาม \{#add-third-party-oidc-oauth-apps}

เมื่อคุณต้องการให้แอปพลิเคชันของบุคคลที่สามสามารถยืนยันตัวตนกับระบบเอกลักษณ์ของ Logto ได้ คุณสามารถกำหนดค่า Logto ให้เป็น [Identity Provider (IdP)](https://auth.wiki/identity-provider)

Logto สามารถใช้เป็น IdP ผ่านโปรโตคอลมาตรฐานอย่าง OAuth และ OIDC และบริการภายนอกสามารถใช้งาน “Sign in with Logto” ได้เช่นเดียวกับ “Sign in with Google” ทำให้ผู้ใช้ของพวกเขาสามารถยืนยันตัวตนผ่านระบบผู้ใช้ของคุณ

แตกต่างจากแอป first-party แอปของบุคคลที่สามสามารถร้องขอสิทธิ์ได้อย่างจำกัดเท่านั้น ระหว่างกระบวนการยืนยันตัวตน ผู้ใช้จะเห็น **หน้าขอความยินยอม (consent screen)** เพื่อทบทวนและอนุมัติข้อมูลและสิทธิ์ที่แอปบุคคลที่สามร้องขอก่อนอนุญาตการเข้าถึง

ด้วยการสร้างแอปพลิเคชันของบุคคลที่สามใน Logto คุณจะสร้างการเชื่อมต่อที่ปลอดภัยระหว่างระบบเอกลักษณ์ของคุณกับบริการภายนอก พร้อมให้ผู้ใช้ของคุณควบคุมการแบ่งปันข้อมูลของตนเอง

## ขยายฟีเจอร์ Logto ด้วย Management API \{#extend-logto-features-with-management-api}

Logto [Management API](https://openapi.logto.io/) ช่วยให้คุณควบคุมและทำงานอัตโนมัติของฟังก์ชัน Logto ในแอปพลิเคชันของคุณได้แบบโปรแกรม

เมื่อคุณต้องการขยายความสามารถของ Logto เพื่อเสริมศักยภาพผลิตภัณฑ์ (เช่น [จัดการองค์กรแบบหลายผู้เช่า](/organizations) หรือทำงานอัตโนมัติร่วมกับ Logto) คุณสามารถใช้ Logto Management API เพื่อสร้างฟีเจอร์ขั้นสูงให้เหมาะกับความต้องการของผลิตภัณฑ์
