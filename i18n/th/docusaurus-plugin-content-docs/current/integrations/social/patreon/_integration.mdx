## ลงชื่อเข้าใช้ด้วยบัญชี Patreon \{#sign-in-with-patreon-account}

ไปที่ [เว็บไซต์ Patreon](https://www.patreon.com/) และลงชื่อเข้าใช้ด้วยบัญชี Patreon ของคุณ คุณสามารถสมัครบัญชีใหม่ได้หากยังไม่มีบัญชี

## สร้างและกำหนดค่า OAuth app \{#create-and-configure-oauth-app}

ทำตามคู่มือ [การสร้าง Patreon OAuth App](https://www.patreon.com/portal/registration/register-clients) และลงทะเบียนแอปพลิเคชันใหม่

ตั้งชื่อ OAuth application ใหม่ของคุณในช่อง **App Name** และกรอก **App URL** ของแอป คุณสามารถเว้นว่างช่อง **App Description** ได้ และปรับแต่ง **Redirect URIs** เป็น `${your_logto_origin}/callback/${connector_id}` โดย `connector_id` สามารถดูได้ที่แถบด้านบนของหน้ารายละเอียดตัวเชื่อมต่อใน Logto Admin Console

:::note
หากคุณพบข้อความแสดงข้อผิดพลาด "The redirect_uri MUST match the registered callback URL for this application." ขณะเข้าสู่ระบบ ให้ลองปรับค่า Redirect URI ของ Patreon OAuth App และ URL สำหรับ redirect ของ Logto App ของคุณ (รวมถึง protocol) ให้ตรงกันเพื่อแก้ไขปัญหา
:::

## จัดการ OAuth apps \{#managing-oauth-apps}

ไปที่ [หน้า Clients & API Keys](https://www.patreon.com/portal/registration/register-clients) บน Patreon ซึ่งคุณสามารถเพิ่ม แก้ไข หรือลบ OAuth apps ที่มีอยู่ได้ คุณยังสามารถค้นหา `Client ID` และสร้าง `Client secrets` ได้ในหน้ารายละเอียด OAuth app

## กำหนดค่าตัวเชื่อมต่อของคุณ \{#configure-your-connector}

กรอกช่อง `clientId` และ `clientSecret` ด้วย _Client ID_ และ _Client Secret_ ที่คุณได้รับจากหน้ารายละเอียด OAuth app ตามที่กล่าวไว้ในส่วนก่อนหน้า

`scope` คือรายการ [ขอบเขต (scopes)](https://docs.patreon.com/#scopes) ที่คั่นด้วยช่องว่าง หากไม่ได้ระบุ scope จะใช้ค่าเริ่มต้นเป็น `identity identity[email]`

### ประเภทของ config \{#config-types}

| Name         | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
