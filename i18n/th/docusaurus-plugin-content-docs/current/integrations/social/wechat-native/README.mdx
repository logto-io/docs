# ตั้งค่าการเข้าสู่ระบบโซเชียลด้วย WeChat (Native) (Set up social login with WeChat (Native))

ตัวเชื่อมต่อ (Connector) อย่างเป็นทางการของ Logto สำหรับการเข้าสู่ระบบโซเชียล WeChat ในแอปเนทีฟ (iOS และ Android)

<GuideTip />

## เริ่มต้นใช้งาน \{#get-started}

หากคุณยังไม่เข้าใจแนวคิดของตัวเชื่อมต่อ หรือไม่รู้วิธีเพิ่มตัวเชื่อมต่อนี้ในประสบการณ์การลงชื่อเข้าใช้ของคุณ โปรดดู [คู่มือ Logto](/connectors/social-connectors)

ในแอปเนทีฟ คุณไม่สามารถใช้เว็บเป็นวิธีการลงชื่อเข้าใช้ของ WeChat ได้: ต้องนำทางไปยังแอป WeChat และยังต้องใช้ SDK ของพวกเขาด้วย

เราเข้าใจว่ามันอาจดูน่ากลัว แต่ไม่ต้องกังวล เราจะจัดการได้อย่างง่ายดายในคู่มือนี้

## สร้างแอปมือถือใน WeChat Open Platform \{#create-a-mobile-app-in-the-wechat-open-platform}

:::tip

คุณสามารถข้ามบางส่วนได้หากทำเสร็จแล้ว

:::

### สร้างบัญชี \{#create-an-account}

เปิด https://open.weixin.qq.com/ คลิกปุ่ม "Sign Up" ที่มุมขวาบน แล้วดำเนินการสมัครให้เสร็จสิ้น

### สร้างแอปมือถือ \{#create-a-mobile-app}

ลงชื่อเข้าใช้ด้วยบัญชีที่คุณเพิ่งสร้าง ในแท็บ "Mobile Application" (移动应用) คลิกปุ่มสีเขียวขนาดใหญ่ "Create a mobile app" (创建移动应用)

<img src="/img/assets/app-tabs.png" alt="App tabs" />

กรอกข้อมูลที่จำเป็นในแบบฟอร์มแอปพลิเคชัน

<img src="/img/assets/create-mobile-app.png" alt="Create mobile app" />

#### ข้อมูลพื้นฐาน \{#basic-info}

ส่วนใหญ่ค่อนข้างตรงไปตรงมา และเรามีเคล็ดลับดังนี้:

- หากคุณต้องการทดสอบการเข้าสู่ระบบ WeChat และแอปยังไม่ได้อยู่ใน App Store ในส่วน "App is available" ให้เลือก "No" เพื่อข้าม "App download link"
- "App operation flow chart" ดูเหมือนจะยุ่งยาก จากประสบการณ์ของเรา คุณควรเตรียมไดอะแกรมโฟลว์ง่าย ๆ และภาพหน้าจอแอปหลายภาพเพื่อเพิ่มโอกาสผ่านการตรวจสอบ

คลิก "Next step" เพื่อดำเนินการต่อ

#### ข้อมูลแพลตฟอร์ม \{#platform-info}

คุณสามารถกำหนดค่า iOS และ / หรือ Android เพื่อผสาน Logto กับการเข้าสู่ระบบ WeChat แบบเนทีฟ

**แอป iOS**

เลือก "iOS app" (iOS 应用) แล้วเลือกประเภทอุปกรณ์เป้าหมายของแอปคุณให้ถูกต้อง

<img src="/img/assets/platform.png" alt="App platform" />

หากคุณเลือก "No" สำหรับ App Store availability คุณสามารถข้ามการกรอก "AppStore download address" ได้

กรอก _Bundle ID_, _Test version Bundle ID_ และ _Universal Links_ (จริง ๆ แล้วต้องการแค่ลิงก์เดียว 😂)

:::note

_Bundle ID_ และ _Test version Bundle ID_ สามารถใช้ค่าเดียวกันได้

:::

:::tip

WeChat ต้องการ universal link สำหรับการเข้าสู่ระบบแบบเนทีฟ หากคุณยังไม่ได้ตั้งค่าหรือไม่รู้จัก universal link โปรดดู [เอกสารทางการของ Apple](https://developer.apple.com/ios/universal-links/)

:::

**แอป Android**

เลือก "Android app" (Android 应用)

<img src="/img/assets/platform-android-app.png" alt="Android app platform" />

กรอก _Application Signing Signature_ (应用签名) และ _Application Package Name_ (应用包名)

:::note

คุณต้องเซ็นแอปของคุณเพื่อให้ได้ signature ดูข้อมูลเพิ่มเติมได้ที่ [Sign your app](https://developer.android.com/studio/publish/app-signing)

:::

หลังจากเซ็นแอปเสร็จแล้ว คุณสามารถรันคำสั่ง `signingReport` เพื่อรับ signing signature

```bash
./gradlew your-android-project:signingReport
```

ค่า `MD5` ของ build variant ที่เกี่ยวข้องจะเป็น _Application Signing Signature_ (应用签名) แต่ต้องลบเครื่องหมายจุลภาคออกทั้งหมดและเปลี่ยนเป็นตัวพิมพ์เล็ก

เช่น `1A:2B:3C:4D` -> `1a2b3c4d`

#### รอผลการตรวจสอบ \{#waiting-for-the-review-result}

หลังจากกรอกข้อมูลแพลตฟอร์มเสร็จแล้ว คลิก "Submit Review" เพื่อดำเนินการต่อ โดยปกติการตรวจสอบจะรวดเร็ว มักเสร็จภายใน 1-2 วัน

เราสงสัยว่าผู้ตรวจสอบถูกสุ่มในแต่ละครั้งเพราะมาตรฐานไม่คงที่ คุณอาจถูกปฏิเสธครั้งแรก แต่ขอให้สู้ต่อ! แจ้งสถานะของคุณและสอบถามผู้ตรวจสอบว่าควรแก้ไขอย่างไร

## เปิดใช้งานการเข้าสู่ระบบ WeChat แบบเนทีฟในแอปของคุณ \{#enable-wechat-native-sign-in-in-your-app}

### iOS \{#ios}

เราสมมติว่าคุณได้ผสาน [Logto iOS SDK](/quick-starts/swift) ในแอปของคุณแล้ว ในกรณีนี้ ทุกอย่างค่อนข้างง่าย และคุณไม่จำเป็นต้องอ่านเอกสาร WeChat SDK ด้วยซ้ำ:

**1. กำหนดค่า universal link และ URL scheme ในโปรเจกต์ Xcode ของคุณ**

ในโปรเจกต์ Xcode -> แท็บ Signing & Capabilities ให้เพิ่มความสามารถ "Associated Domains" และ universal link ที่คุณตั้งค่าไว้ก่อนหน้านี้

<img src="/img/assets/universal-link.png" alt="Universal link" />

จากนั้นไปที่แท็บ "Info" เพิ่ม [custom URL scheme](https://developer.apple.com/documentation/xcode/defining-a-custom-url-scheme-for-your-app) ด้วย WeChat App ID

<img src="/img/assets/custom-url-scheme.png" alt="Custom URL scheme" />

สุดท้ายเปิด `Info.plist` ของคุณ เพิ่ม `weixinULAPI` และ `weixin` ใน `LSApplicationQueriesSchemes`

<img src="/img/assets/plist.png" alt="Plist" />

:::note

เราทราบว่าขั้นตอนเหล่านี้อาจดูไม่สมเหตุสมผลนัก แต่นี่คือวิธีที่น้อยที่สุดที่ใช้งานได้จริงที่เราพบ ดู [คู่มือทางการสุดมหัศจรรย์](https://developers.weixin.qq.com/doc/oplatform/en/Mobile_App/Access_Guide/iOS.html) สำหรับข้อมูลเพิ่มเติม

:::

**2. เพิ่ม `LogtoSocialPluginWechat` ในโปรเจกต์ Xcode ของคุณ**

เพิ่ม framework:

<img src="/img/assets/add-framework.png" alt="Add framework" />

และเพิ่ม `-ObjC` ใน Build Settings > Linking > Other Linker Flags:

<img src="/img/assets/linker-flags.png" alt="Linker flags" />

:::note

ปลั๊กอินนี้รวม WeChat Open SDK 1.9.2 คุณสามารถใช้ `import WechatOpenSDK` ได้ทันทีหลังจากนำเข้า

:::

**3. เพิ่มปลั๊กอินในตัวเลือก init ของ `LogtoClient` ของคุณ**

```swift
let logtoClient = LogtoClient(
  useConfig: config,
  socialPlugins: [LogtoSocialPluginWechat()]
)
```

**4. จัดการ `onOpenURL` ให้ถูกต้อง**

:::note

ฟังก์ชัน `LogtoClient.handle(url:)` จะจัดการตัวเชื่อมต่อเนทีฟทั้งหมดที่คุณเปิดใช้งาน คุณเพียงแค่เรียกใช้ครั้งเดียว

:::

```swift
// SwiftUI
YourRootView()
  .onOpenURL { url in
      LogtoClient.handle(url: url)
  }

// หรือ AppDelegate
func application(_ app: UIApplication, open url: URL, options: /*...*/) -> Bool {
  LogtoClient.handle(url: url)
}
```

### Android \{#android}

เราสมมติว่าคุณได้ผสาน [Logto Android SDK](/quick-starts/android) ในแอปของคุณแล้ว ในกรณีนี้ ทุกอย่างค่อนข้างง่าย และคุณไม่จำเป็นต้องอ่านเอกสาร WeChat SDK ด้วยซ้ำ:

**1. เพิ่ม `Wechat Open SDK` ในโปรเจกต์ของคุณ**

ตรวจสอบให้แน่ใจว่า repository `mavenCentral()` อยู่ใน repositories ของโปรเจกต์ Gradle ของคุณ:

```kotlin
repositories {
  // ...
  mavenCentral()
}
```

เพิ่ม Wechat Open SDK ใน dependencies ของคุณ:

```kotlin
dependencies {
  // ...
  api("com.tencent.mm.opensdk:wechat-sdk-android:6.8.0")  // kotlin-script
  // หรือ
  api 'com.tencent.mm.opensdk:wechat-sdk-android:6.8.0'   // groovy-script
}
```

**2. สร้าง `WXEntryActivity` ในโปรเจกต์ของคุณ**

สร้างแพ็กเกจ `wxapi` ใต้ root ของแพ็กเกจคุณ และเพิ่ม `WXEntryActivity` ในแพ็กเกจ `wxapi` (ตัวอย่างเช่น `com.sample.app`):

```kotlin
// WXEntryActivity.kt
package com.sample.app.wxapi

import io.logto.sdk.android.auth.social.wechat.WechatSocialResultActivity

class WXEntryActivity: WechatSocialResultActivity()
```

```java
// WXEntryActivity.java
package com.sample.app.wxapi

import io.logto.sdk.android.auth.social.wechat.WechatSocialResultActivity

public class WXEntryActivity extends WechatSocialResultActivity {}
```

ตำแหน่งสุดท้ายของ `WXEntryActivity` ในโปรเจกต์ควรเป็นดังนี้ (ตัวอย่าง Kotlin):

```bash
src/main/kotlin/com/sample/app/wxapi/WXEntryActivity.kt
```

**3. แก้ไข `AndroidManifest.xml`**

เพิ่มบรรทัดต่อไปนี้ใน `AndroidManifest.xml` ของคุณ:

```xml
\<?xml version="1.0" encoding="utf-8"?>
\<manifest xmlns:android="http://schemas.android.com/apk/res/android"
  package="com.sample.app">

  \<application>
    \<!-- line to be added -->
    \<activity android:name=".wxapi.WXEntryActivity" android:exported="true"/>
  \</application>

\</manifest>
```

## ทดสอบตัวเชื่อมต่อ WeChat แบบเนทีฟ \{#test-wechat-native-connector}

เรียบร้อยแล้ว อย่าลืม [เปิดใช้งานตัวเชื่อมต่อโซเชียลใน Sign-up และ sign-in](/connectors/social-connectors/#enable-social-sign-in)

เมื่อเปิดใช้งานตัวเชื่อมต่อ WeChat แบบเนทีฟแล้ว คุณสามารถ build และรันแอปของคุณเพื่อตรวจสอบการทำงาน

:::caution

WeChat ไม่มีแผนรองรับอุปกรณ์ที่ไม่ได้ติดตั้งแอป WeChat Logto จะซ่อนตัวเชื่อมต่อนี้ระหว่างการลงชื่อเข้าใช้หากเป็นเช่นนั้น (ซึ่งเป็นวิธีที่แนะนำจาก [คู่มือพัฒนาอย่างเป็นทางการ](https://developers.weixin.qq.com/doc/oplatform/en/Mobile_App/WeChat_Login/Development_Guide.html))

:::
