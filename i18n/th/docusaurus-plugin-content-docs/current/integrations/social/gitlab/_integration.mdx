## ลงชื่อเข้าใช้ด้วยบัญชี GitLab \{#sign-in-with-gitlab-account}

ไปที่ [เว็บไซต์ GitLab](https://gitlab.com/) และลงชื่อเข้าใช้ด้วยบัญชี GitLab ของคุณ คุณสามารถสมัครบัญชีใหม่ได้หากยังไม่มี

## สร้างและตั้งค่า OAuth app \{#create-and-configure-oauth-app}

ทำตามคู่มือ [การสร้าง GitLab OAuth App](https://docs.gitlab.com/ee/integration/oauth_provider.html) และลงทะเบียนแอปพลิเคชันใหม่

ตั้งชื่อ OAuth application ใหม่ของคุณในช่อง **Name** และกรอก **Redirect URI** ของแอป กำหนดค่า **Redirect URIs** เป็น `${your_logto_origin}/callback/${connector_id}` โดย `connector_id` สามารถดูได้ที่แถบด้านบนของหน้ารายละเอียดตัวเชื่อมต่อใน Logto Admin Console

ในส่วน scopes ให้เลือก `openid` คุณอาจต้องการเปิดใช้งาน `profile` และ `email` ด้วย โดย `profile` scope จำเป็นสำหรับการดึงข้อมูลโปรไฟล์ของผู้ใช้ และ `email` scope จำเป็นสำหรับการดึงที่อยู่อีเมลของผู้ใช้ ตรวจสอบให้แน่ใจว่าคุณได้อนุญาต scopes เหล่านี้ใน GitLab OAuth app ของคุณหากต้องการใช้งาน โดย scopes เหล่านี้จะถูกใช้ในการตั้งค่าตัวเชื่อมต่อของคุณในขั้นตอนถัดไปด้วย

:::note

- หากคุณใช้โดเมนแบบกำหนดเอง ให้เพิ่มทั้งโดเมนที่กำหนดเองและโดเมน Logto เริ่มต้นลงใน Redirect URIs เพื่อให้ OAuth flow ทำงานได้ถูกต้องกับทั้งสองโดเมน
- หากคุณพบข้อความแสดงข้อผิดพลาด "The redirect_uri MUST match the registered callback URL for this application." ขณะเข้าสู่ระบบ ให้ลองปรับ Redirect URI ของ GitLab OAuth App และ URL สำหรับ redirect ของ Logto App ของคุณ (รวมถึง protocol) ให้ตรงกันเพื่อแก้ไขปัญหา
  :::

## จัดการ OAuth apps \{#managing-oauth-apps}

ไปที่ [หน้าจัดการแอปพลิเคชัน](https://gitlab.com/-/profile/applications) บน GitLab ซึ่งคุณสามารถเพิ่ม แก้ไข หรือลบ OAuth apps ที่มีอยู่ได้ คุณยังสามารถดู `Application ID` และสร้าง `Secret` ได้ในหน้ารายละเอียด OAuth app

## ตั้งค่าตัวเชื่อมต่อของคุณ \{#configure-your-connector}

กรอกข้อมูลในช่อง `clientId` และ `clientSecret` ด้วย _Application ID_ และ _Secret_ ที่คุณได้รับจากหน้ารายละเอียด OAuth app ตามที่กล่าวไว้ในหัวข้อก่อนหน้า

`scope` คือรายการ [scopes](https://docs.gitlab.com/ee/integration/oauth_provider.html#authorized-applications) ที่คั่นด้วยช่องว่าง หากไม่ได้ระบุ scope จะถูกตั้งค่าเริ่มต้นเป็น `openid` สำหรับ GitLab connector, scopes ที่คุณอาจต้องการใช้คือ `openid`, `profile` และ `email` โดย `profile` scope จำเป็นสำหรับการดึงข้อมูลโปรไฟล์ของผู้ใช้ และ `email` scope จำเป็นสำหรับการดึงที่อยู่อีเมลของผู้ใช้ ตรวจสอบให้แน่ใจว่าคุณได้อนุญาต scopes เหล่านี้ใน GitLab OAuth app ของคุณ (ตั้งค่าในหัวข้อ [สร้างและตั้งค่า OAuth app](#create-and-configure-oauth-app))

### ประเภทของการตั้งค่า \{#config-types}

| Name         | Type   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |
| scope        | string |
