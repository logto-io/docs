iOS：集成 Logto Swift SDK

备注

本篇教程默认你已经在管理界面中成功创建了一个「原生应用」类型的应用。如果你还没有完成这一步操作，请参阅这篇教程，创建应用之后再继续。

将 Logto SDK 添加至依赖​

使用如下的 URL 将 Logto SDK 添加至 Swift Package Manager 的依赖中。

https://github.com/logto-io/swift.git

从 Xcode 11 开始，无需任何额外工具你就可以 引入一个 Swift package。

因为一些技术原因，我们暂时不支持 Carthage 和 CocoaPods。

Carthage

Carthage 需要创建一个 xcodeproj 文件才能编译，但是由于我们的内置的一些社交软件插件用到了二进制目标文件，导致使用 swift package generate-xcodeproj 命令时报错。我们会继续尝试寻求解决方案。

CocoaPods

CocoaPods 不支持本地依赖 和 monorepo，所以要在此工程创建 .podspec 文件使用 Cocoapods 的话将非常困难。

初始化 LogtoClient​

import Logto
import LogtoClient

let config = try? LogtoConfig(
  endpoint: "<your-logto-endpoint>", // 例如 http://localhost:3001
  appId: "<your-app-id>"
)
let client = LogtoClient(useConfig: config)

我们默认会把例如 ID Token 和 Refresh Token 这样的凭据存储在 Keychain 中，如此一来用户在重新打开应用的时候无需再次登录。

如果需要禁用这个行为，可将 usingPersistStorage 设置成 false：

let config = try? LogtoConfig(
  // ...
  usingPersistStorage: false
)

登录​

配置 Redirect URI​

让我们在这里切换到「管理控制台」的应用详情页。添加一个值为 io.logto://callback 的 Redirect URI 并点按「保存变更」。

Redirect URI 是一个 OAuth 2.0 的概念，用于表明在 authentication 后重定向至的位置。

信息

iOS SDK 中的 Redirect URI 仅用于内部。除非连接器有要求，否则 无需 在项目中添加 Custom URL Scheme。

通过浏览器登录​

备注

在调用 .signInWithBrowser(redirectUri:) 之前，请先确保已经在「管理控制台」中正确配置了 Redirect URI。

回到 Xcode，使用下面的代码实现登录：

do {
  try await client.signInWithBrowser(redirectUri: "io.logto://callback")
  print(client.isAuthenticated) // true
} catch let error as LogtoClientErrors.SignIn {
  // 登录过程中有错误发生
}

退出登录​

调用 .signOut() 将清除 Keychain 中所有 Logto 的数据（如果有）。

await client.signOut()
